🔷 What is an Acyclic Graph?
The Foundation

A graph is a collection of vertices (nodes) connected by edges (links). Imagine a city map — cities are vertices and roads are edges.

An acyclic graph is a graph that has no cycles. A cycle is a path that starts from a vertex and returns back to the same vertex without repeating any edge. If you can never walk in a circle on this graph, it's acyclic.

Two main types: Trees (undirected, connected, acyclic) and DAGs – Directed Acyclic Graphs (directed edges, no cycles). DAGs are extremely common in real-world software.

🗺️
Real-Life Analogy
Think of a university course prerequisite system. You must take Math101 before taking Physics201, and Physics201 before Quantum301. The structure forms a DAG — you never go back to a previous course. That's an acyclic directed graph!
✓
Properties of Acyclic Graphs
No path from any vertex leads back to itself
Can be directed (DAG) or undirected (Forest/Tree)
Topological sorting is always possible in a DAG
Used in compilers, scheduling, build systems
⚡
Real-World Uses
Git version control (commit graph)
Make/build tools (dependency resolution)
Spreadsheet cell dependencies
Neural networks (forward pass)
📊 Representation
Adjacency Matrix
The way we store and represent an acyclic graph in computer memory.

An adjacency matrix is a 2D array (table) of size V × V, where V is the number of vertices. If there is an edge from vertex i to vertex j, then matrix[i][j] = 1. Otherwise, it is 0.

For a directed graph (DAG): matrix[i][j]=1 means edge i→j exists, but matrix[j][i] may be 0.

For an undirected acyclic graph (tree/forest): matrix[i][j]=1 AND matrix[j][i]=1 always, so the matrix is symmetric.

Diagonal rule: In an acyclic graph, matrix[i][i] is always 0 (no self-loops).

📋
Real-Life Analogy
Think of a school attendance register. Rows are students, columns are days. A "1" means the student attended that day. Similarly, our matrix rows=source vertices, columns=destination vertices, and "1" means an edge exists between them.

C — Adjacency Matrix Structure
#include <stdio.h> #include <string.h> #define MAX 10 /* Maximum vertices allowed */ /* Graph structure using adjacency matrix */ struct Graph { int matrix[MAX][MAX]; /* 0 or 1: edge exists? */ int vertices; /* current number of vertices */ char label[MAX][20]; /* optional: vertex names */ }; /* Initialize an empty graph */ void initGraph(struct Graph *g, int v) { g->vertices = v; memset(g->matrix, 0, sizeof(g->matrix)); /* all zeros */ } /* Display the adjacency matrix */ void displayMatrix(struct Graph *g) { int i, j; printf(" "); for (i = 0; i < g->vertices; i++) printf("%2d", i); printf("\n"); for (i = 0; i < g->vertices; i++) { printf("%2d", i); for (j = 0; j < g->vertices; j++) printf("%2d", g->matrix[i][j]); printf("\n"); } } int main() { struct Graph g; initGraph(&g, 4); /* 4 vertices: 0,1,2,3 */ g.matrix[0][1] = 1; /* edge 0 → 1 */ g.matrix[1][2] = 1; /* edge 1 → 2 */ g.matrix[0][2] = 1; /* edge 0 → 2 */ g.matrix[2][3] = 1; /* edge 2 → 3 */ displayMatrix(&g); return 0; }
🔵 Vertex Operations
Insert Vertex

Inserting a vertex means adding a new node to the graph. In an adjacency matrix, this means expanding the matrix by one row and one column, both filled with zeros (no edges yet).

Since the matrix is pre-allocated (static array), we simply increment the vertex count and make sure all new cells in that row/column are 0.

Time Complexity: O(V) to initialize the new row/column. Space: No new memory allocation needed if we pre-allocate MAX size.

🏫
Real-Life Analogy
Adding a vertex is like a new student joining a class. At first, they have no connections (friends). You add a row and column for them in the class register — all zeros. Then later you add their connections (friendships) separately.

C — Insert Vertex
/* Insert a new vertex into the graph */ void insertVertex(struct Graph *g) { int i; if (g->vertices >= MAX) { printf("Graph is full! Cannot add more vertices.\n"); return; } int newV = g->vertices; /* index of the new vertex */ /* Set new row to all zeros */ for (i = 0; i <= newV; i++) g->matrix[newV][i] = 0; /* Set new column to all zeros */ for (i = 0; i <= newV; i++) g->matrix[i][newV] = 0; g->vertices++; /* increment vertex count */ printf("Vertex %d inserted. Total vertices: %d\n", newV, g->vertices); } /* Usage: */ insertVertex(&g); /* adds vertex 0 */ insertVertex(&g); /* adds vertex 1 */
🔵 Vertex Operations
Delete Vertex

Deleting a vertex means removing it along with all its edges. In an adjacency matrix, we must:

1. Remove the row and column for that vertex by shifting rows/columns left/up.

2. Decrement the vertex count.

This is expensive — O(V²) in the worst case — because shifting a 2D matrix requires moving many elements. That's a trade-off of adjacency matrix representation.

👋
Real-Life Analogy
Like removing a student from school. You don't just erase their name — you also remove all their friendships (edges) from the register, and then close the gap by shifting everyone's records up.

C — Delete Vertex
/* Delete vertex 'v' from the graph */ void deleteVertex(struct Graph *g, int v) { int i, j; if (v < 0 || v >= g->vertices) { printf("Invalid vertex!\n"); return; } /* Shift rows up: remove row 'v' */ for (i = v; i < g->vertices - 1; i++) for (j = 0; j < g->vertices; j++) g->matrix[i][j] = g->matrix[i+1][j]; /* Shift columns left: remove column 'v' */ for (i = 0; i < g->vertices - 1; i++) for (j = v; j < g->vertices - 1; j++) g->matrix[i][j] = g->matrix[i][j+1]; g->vertices--; /* one fewer vertex now */ printf("Vertex %d deleted. Remaining vertices: %d\n", v, g->vertices); }
🔵 Vertex Operations
Search Vertex

Searching for a vertex checks whether a vertex with a given index or label exists in the graph. In an adjacency matrix with V vertices, vertex indices are 0 to V-1. A simple bounds check tells us if a vertex is valid.

If vertices have labels (names), we search through the label array — O(V) time. By index alone, it's O(1).

📇
Real-Life Analogy
Searching a vertex is like looking up a contact in your phone book. If you know the page number (index), it's instant. If you search by name (label), you scan until you find it.

/* Search by index: O(1) */ int searchVertexByIndex(struct Graph *g, int v) { if (v >= 0 && v < g->vertices) { printf("Vertex %d FOUND in graph.\n", v); return 1; /* found */ } printf("Vertex %d NOT found.\n", v); return 0; /* not found */ } /* Search by label: O(V) */ int searchVertexByLabel(struct Graph *g, const char *name) { int i; for (i = 0; i < g->vertices; i++) { if (strcmp(g->label[i], name) == 0) { printf("Vertex '%s' found at index %d\n", name, i); return i; } } printf("Vertex '%s' not found!\n", name); return -1; }
🔵 Vertex Operations
Update Vertex

Updating a vertex means changing its label or data. The adjacency matrix structure (the connections) doesn't change — only the metadata associated with the vertex changes.

This is an O(1) operation if we know the vertex index. We simply update the label array at that index.

C — Update Vertex
#include <string.h> /* Update the label of vertex 'v' */ void updateVertex(struct Graph *g, int v, const char *newLabel) { if (v < 0 || v >= g->vertices) { printf("Error: Vertex %d does not exist!\n", v); return; } strcpy(g->label[v], newLabel); /* update the name */ printf("Vertex %d updated to '%s'\n", v, newLabel); } /* Example usage: */ updateVertex(&g, 0, "StartNode"); updateVertex(&g, 3, "EndNode");
🔵 Vertex Operations
Count & Display Vertices

Count Vertices: Simply return g->vertices — O(1). The count is maintained as we insert/delete.

Display Vertices: Loop from 0 to vertices-1 and print each. If labels exist, print those too. O(V) time.

C — Count & Display Vertices
/* Count vertices: O(1) */ int countVertices(struct Graph *g) { printf("Total vertices: %d\n", g->vertices); return g->vertices; } /* Display all vertices: O(V) */ void displayVertices(struct Graph *g) { int i; printf("Vertices in graph:\n"); for (i = 0; i < g->vertices; i++) { printf(" [%d]", i); if (strlen(g->label[i]) > 0) printf(" '%s'", g->label[i]); printf("\n"); } } /* Output example: Vertices in graph: [0] 'A' [1] 'B' [2] 'C' */
🔗 Edge Operations
Insert Edge

Adding an edge means setting matrix[src][dst] = 1. For a directed graph (DAG), only this one cell changes. For an undirected graph, we also set matrix[dst][src] = 1.

Acyclic constraint: Before inserting an edge in a DAG, we must verify that adding it doesn't create a cycle. We do this by checking if there's already a path from dst to src — if yes, adding src→dst would create a cycle.

Time Complexity: O(1) for the actual insertion, but O(V+E) if we do cycle checking first.

🛤️
Real-Life Analogy
Adding an edge is like building a one-way road between two cities. You mark it on the map (matrix[src][dst]=1). But in an acyclic city plan, you must ensure this new road doesn't create a loop in the road network!

C — Insert Edge (with cycle check)
/* Check if path exists from 'src' to 'dst' using DFS */ int pathExists(struct Graph *g, int src, int dst, int visited[]) { if (src == dst) return 1; /* reached destination */ visited[src] = 1; int i; for (i = 0; i < g->vertices; i++) { if (g->matrix[src][i] && !visited[i]) if (pathExists(g, i, dst, visited)) return 1; } return 0; } /* Insert edge src→dst in a DAG (acyclic check) */ void insertEdge(struct Graph *g, int src, int dst) { int visited[MAX] = {0}; if (src == dst) { printf("Self-loop not allowed in acyclic graph!\n"); return; } /* If dst can reach src, adding src→dst creates a cycle */ if (pathExists(g, dst, src, visited)) { printf("Adding %d→%d would create a cycle! Rejected.\n", src, dst); return; } g->matrix[src][dst] = 1; printf("Edge %d → %d inserted.\n", src, dst); }
🔗 Edge Operations
Delete Edge

Deleting an edge simply sets matrix[src][dst] = 0. This is an O(1) operation — one of the biggest advantages of an adjacency matrix. We just zero out that cell.

For undirected graphs, also set matrix[dst][src] = 0. Always check that the edge actually exists first to avoid redundant operations.

C — Delete Edge
/* Delete a directed edge from src to dst */ void deleteEdge(struct Graph *g, int src, int dst) { if (src < 0 || src >= g->vertices || dst < 0 || dst >= g->vertices) { printf("Invalid vertex index!\n"); return; } if (g->matrix[src][dst] == 0) { printf("Edge %d→%d does not exist!\n", src, dst); return; } g->matrix[src][dst] = 0; /* remove the edge: O(1) */ printf("Edge %d→%d deleted.\n", src, dst); } /* For undirected acyclic graph (forest/tree): */ void deleteUndirectedEdge(struct Graph *g, int u, int v) { g->matrix[u][v] = 0; g->matrix[v][u] = 0; /* remove both directions */ printf("Edge %d--%d deleted.\n", u, v); }
🔗 Edge Operations
Search Edge & Update Edge

Search Edge: Check if matrix[src][dst] == 1. This is O(1) — the fastest possible! This is the primary advantage of adjacency matrices over adjacency lists.

Update Edge: In a basic binary adjacency matrix (0 or 1), "updating" means toggling or adding weight. For a weighted graph, matrix values store the weight instead of just 0/1. Updating means changing that weight value.

C — Search & Update Edge
/* Search for edge: O(1) */ int searchEdge(struct Graph *g, int src, int dst) { if (g->matrix[src][dst] == 1) { printf("Edge %d → %d EXISTS.\n", src, dst); return 1; } printf("Edge %d → %d does NOT exist.\n", src, dst); return 0; } /* For weighted DAG: update edge weight */ void updateEdgeWeight(int wMatrix[][MAX], int src, int dst, int newWeight) { if (wMatrix[src][dst] == 0) { printf("Edge doesn't exist! Cannot update.\n"); return; } wMatrix[src][dst] = newWeight; printf("Edge %d→%d weight updated to %d\n", src, dst, newWeight); }
🔗 Edge Operations
Count & Display Edges

Count Edges: Scan the entire matrix and count cells where matrix[i][j]==1. Time: O(V²). For undirected, divide the count by 2.

Display Edges: Similarly scan and print every (i,j) pair where matrix[i][j]==1. Also O(V²).

C — Count & Display Edges
/* Count all directed edges: O(V²) */ int countEdges(struct Graph *g) { int i, j, count = 0; for (i = 0; i < g->vertices; i++) for (j = 0; j < g->vertices; j++) if (g->matrix[i][j] == 1) count++; printf("Total edges: %d\n", count); return count; } /* Display all directed edges: O(V²) */ void displayEdges(struct Graph *g) { int i, j; printf("Edges in graph:\n"); for (i = 0; i < g->vertices; i++) for (j = 0; j < g->vertices; j++) if (g->matrix[i][j] == 1) printf(" %d → %d\n", i, j); }
🌊 Traversal
Breadth First Search (BFS)
Explore all vertices level by level, using a queue.

BFS starts at a given vertex and visits all its neighbors first, then their neighbors, and so on — like ripples in a pond. It uses a Queue (FIFO) data structure.

Steps: 1) Initialize queue, 2) Enqueue start vertex, mark visited, 3) While queue not empty: dequeue vertex, process it, enqueue all unvisited adjacent vertices and mark them visited.

Time: O(V²) with adjacency matrix (we check every cell in each row). Space: O(V) for the queue and visited array.

💧
Real-Life Analogy
BFS is like spreading news in a city. Person A tells all their friends (level 1). Each of those friends tells their own friends (level 2), and so on. The news spreads outward in waves — nobody gets told before their "introducer" is told first.

#include <stdio.h> #define MAX 10 /* Simple array-based queue */ int queue[MAX], front = -1, rear = -1; void enqueue(int val) { queue[++rear] = val; if(front==-1)front=0; } int dequeue() { return queue[front++]; } int isQueueEmpty() { return (front == -1 || front > rear); } /* BFS traversal from start vertex */ void BFS(struct Graph *g, int start) { int visited[MAX] = {0}; int i; front = rear = -1; /* reset queue */ /* Step 1: Enqueue start, mark visited */ enqueue(start); visited[start] = 1; printf("BFS Order: "); /* Step 2: Process queue */ while (!isQueueEmpty()) { int v = dequeue(); /* Step 3: Dequeue */ printf("%d ", v); /* Step 4: Visit node */ /* Step 5: Traverse adjacent nodes */ for (i = 0; i < g->vertices; i++) { if (g->matrix[v][i] && !visited[i]) { enqueue(i); /* enqueue neighbor */ visited[i] = 1; /* Step 6: Mark visited */ } } } printf("\n"); } /* Output for DAG 0→1, 0→2, 1→3, 2→3 starting from 0: BFS Order: 0 1 2 3 */
🌀 Traversal
Depth First Search (DFS)
Dive deep into one path before backtracking.

DFS starts at a vertex and goes as deep as possible along one path before backtracking. Think of it as exploring a maze by always taking the first available turn.

Recursive DFS: Mark vertex as visited, recurse on each unvisited neighbor. Uses the call stack implicitly.

Stack DFS: Same logic but uses an explicit stack (useful when recursion depth is very large).

Backtracking: When a dead end is reached (no unvisited neighbors), DFS "backtracks" to the previous vertex and tries the next path.

Time: O(V²) with adjacency matrix. Space: O(V) for visited array + recursion stack.

🌲
Real-Life Analogy
DFS is like exploring a cave system. You always go as far down one tunnel as possible. When you hit a dead end, you back up to the last junction and try the next tunnel. You keep doing this until every tunnel is explored.

🛤️ Path Operations
Find Path & All Paths

Find Path: DFS-based search to find a path from source to destination. We track the path array and print it when we reach the destination.

All Paths: Similar to Find Path, but we don't stop at the first path. We use backtracking — after reaching destination, we unmark the current vertex as visited and continue DFS to find more paths.

Path Length Count: Track the number of edges in the current path using a counter.

In a DAG, finding all paths is safe because there are no cycles — the recursion always terminates.

🗺️
Real-Life Analogy
Finding all paths is like planning all possible routes from home to office. You list every possible combination of roads. Since the city map (DAG) has no circular routes, you'll eventually list them all without getting lost in a loop.

C — Find Path & All Paths (Backtracking)
int path[MAX], pathLen; int visited[MAX]; /* Find ONE path from src to dst */ int findPath(struct Graph *g, int src, int dst) { int i; path[pathLen++] = src; visited[src] = 1; if (src == dst) { /* reached destination! */ for (i = 0; i < pathLen; i++) printf("%d ", path[i]); printf("\n"); return 1; } for (i = 0; i < g->vertices; i++) if (g->matrix[src][i] && !visited[i]) if (findPath(g, i, dst)) return 1; pathLen--; /* backtrack */ return 0; } /* Find ALL paths from src to dst — backtracking */ void allPaths(struct Graph *g, int src, int dst) { int i; path[pathLen++] = src; visited[src] = 1; if (src == dst) { for (i = 0; i < pathLen; i++) printf("%d ", path[i]); printf("\n"); } else { for (i = 0; i < g->vertices; i++) if (g->matrix[src][i] && !visited[i]) allPaths(g, i, dst); /* recurse */ } pathLen--; /* backtrack: un-include src */ visited[src] = 0; /* unmark to allow other paths */ } /* Path Length Count */ int pathLengthCount() { return pathLen - 1; } /* edges = nodes - 1 */
⚡ Path Operations
Shortest Path in DAG

In an unweighted DAG, BFS gives the shortest path (fewest edges) naturally, since BFS explores level by level.

In a weighted DAG, we use Dynamic Programming on Topological Order. After doing a topological sort, we relax edges in topological order: dist[v] = min(dist[v], dist[u] + weight(u,v)).

This is more efficient than Dijkstra for DAGs — O(V+E) vs O((V+E) log V).

🚀
Real-Life Analogy
Shortest path in a DAG is like finding the fastest assembly line sequence. Each step (vertex) has a time cost. You process tasks in order of dependencies (topological order) and calculate the minimum total time to reach the final product.
C — Shortest Path via BFS (Unweighted DAG)
#define INF 99999 /* BFS-based shortest path (unweighted DAG) */ void shortestPath_BFS(struct Graph *g, int src, int dst) { int dist[MAX], prev[MAX], visited[MAX]; int queue[MAX], front = -1, rear = -1; int i; for (i = 0; i < g->vertices; i++) { dist[i] = INF; prev[i] = -1; visited[i] = 0; } dist[src] = 0; queue[++rear] = src; front = 0; visited[src] = 1; while (front <= rear) { int v = queue[front++]; for (i = 0; i < g->vertices; i++) { if (g->matrix[v][i] && !visited[i]) { visited[i] = 1; dist[i] = dist[v] + 1; prev[i] = v; queue[++rear] = i; } } } printf("Shortest path length %d → %d: %d edges\n", src, dst, dist[dst]); /* Print path by backtracking via prev[] */ int p = dst, trace[MAX], tlen = 0; while (p != -1) { trace[tlen++] = p; p = prev[p]; } printf("Path: "); for (i = tlen-1; i >= 0; i--) printf("%d ", trace[i]); printf("\n"); }
✅ Path Operations
Reachability Check & Path Length

Reachability Check: Can vertex B be reached from vertex A? We do a DFS/BFS from A and check if B is visited. O(V²) with adjacency matrix.

Transitive Closure: For all pairs (i,j), compute whether j is reachable from i. Result stored in a V×V boolean matrix — can be computed using Floyd-Warshall in O(V³) or by running DFS/BFS from each vertex.

C — Reachability & Transitive Closure
/* Check reachability from src to dst via DFS */ int isReachable(struct Graph *g, int src, int dst) { int vis[MAX] = {0}; return pathExists(g, src, dst, vis); } /* Transitive Closure using DFS from each vertex */ void transitiveClosure(struct Graph *g, int reach[][MAX]) { int i, j; /* Initialize from adjacency matrix */ for (i = 0; i < g->vertices; i++) for (j = 0; j < g->vertices; j++) reach[i][j] = g->matrix[i][j]; /* Floyd-Warshall style: O(V³) */ int k; for (k = 0; k < g->vertices; k++) for (i = 0; i < g->vertices; i++) for (j = 0; j < g->vertices; j++) if (reach[i][k] && reach[k][j]) reach[i][j] = 1; printf("Reachable vertices from each node:\n"); for (i = 0; i < g->vertices; i++) { printf(" From %d: ", i); for (j = 0; j < g->vertices; j++) if (reach[i][j]) printf("%d ", j); printf("\n"); } }
🔄 Acyclic Operations
Cycle Detection & Validation

Detecting Cycle Absence (verifying acyclicity) is crucial. For a directed graph, we use DFS with coloring:

• White (0): Not visited

• Gray (1): Currently in the DFS recursion stack

• Black (2): Fully processed

If we encounter a gray vertex during DFS, we found a back edge → cycle exists!

Edge Safety Check: Before inserting an edge u→v, verify v cannot reach u (no path from v to u). If v→u exists, adding u→v creates a cycle.

Validate Acyclic Structure: Run cycle detection and return true only if no cycle is found.

🚦
Real-Life Analogy
Cycle detection is like a traffic audit. You follow every road sequence and color roads as "currently on" (gray). If you ever hit a "currently on" road again, you've found a circular route — a cycle! For a DAG (like an airport transit map), this should never happen.

C — Cycle Detection (DFS Coloring) + Edge Safety
/* color: 0=white(unvisited), 1=gray(active), 2=black(done) */ int color[MAX]; int dfsHasCycle(struct Graph *g, int v) { int i; color[v] = 1; /* gray: currently exploring */ for (i = 0; i < g->vertices; i++) { if (g->matrix[v][i]) { if (color[i] == 1) return 1; /* back edge = cycle! */ if (color[i] == 0) if (dfsHasCycle(g, i)) return 1; } } color[v] = 2; /* black: fully done */ return 0; } /* Validate: is the graph truly acyclic? */ int isAcyclic(struct Graph *g) { int i; for (i = 0; i < g->vertices; i++) color[i] = 0; /* all white */ for (i = 0; i < g->vertices; i++) if (color[i] == 0) if (dfsHasCycle(g, i)) { printf("CYCLE DETECTED! Not acyclic.\n"); return 0; } printf("Graph is ACYCLIC (valid DAG/Tree).\n"); return 1; } /* Edge Safety Check: safe to add u→v? */ int edgeSafeCheck(struct Graph *g, int u, int v) { int vis[MAX] = {0}; if (pathExists(g, v, u, vis)) { printf("UNSAFE: Adding %d→%d would create a cycle!\n", u, v); return 0; } printf("SAFE: Edge %d→%d can be added.\n", u, v); return 1; }
🌳 Acyclic Operations
Check Tree, Forest & DAG Validation

Tree Structure: A connected, undirected, acyclic graph with exactly V-1 edges. Every pair of vertices has exactly one path between them.

Forest Structure: A collection of disjoint trees. Undirected, acyclic, but not necessarily connected. Has exactly V-E connected components (where E = V - number_of_components).

DAG (Directed Acyclic Graph): Directed graph with no directed cycles. Must pass cycle detection AND topological sort must succeed.

Check Rules: Tree = Acyclic + Connected + Undirected. Forest = Acyclic + Undirected (may be disconnected). DAG = Acyclic + Directed.

📋 Topological Operations
Topological Sort
Order vertices so every directed edge goes from earlier to later in the sequence.

Topological Sort is a linear ordering of all vertices in a DAG such that for every directed edge u→v, vertex u comes before vertex v in the ordering.

This only works on DAGs. If a cycle exists, topological sort is impossible.

Two methods:

• Kahn's Algorithm (BFS-based): Repeatedly pick vertices with in-degree 0, remove them, update neighbors' in-degrees.

• DFS-based: Run DFS; push vertex to a stack only after all its neighbors are processed. The stack gives reverse topological order.

👨‍🍳
Real-Life Analogy
Topological sort is like a recipe for cooking a multi-course meal. You must boil water before making pasta, make pasta before serving it. The topological order tells the exact sequence. If there's no circular dependency ("pasta requires itself"), the order is always achievable.

📦 Topological Operations
Dependency Resolution & Task Scheduling

Dependency Resolution: Given a set of tasks with dependencies (task A requires task B to be done first), find a valid execution order. This is directly a topological sort problem — tasks are vertices, dependencies are edges.

Task Scheduling (Ordering Vertices): Schedule tasks such that no task starts before all its dependencies complete. The topological order is the schedule.

Build Sequence: In build systems (like make, cmake, gradle), source files have compile-order dependencies. Topological sort gives the build sequence.

If topological sort fails (cycle detected), there's a circular dependency — an error in the project configuration.

C — Dependency Resolution with Kahn's Algorithm
/* Kahn's Algorithm: BFS-based topological sort Also detects cycles if result has fewer than V vertices */ void kahnAlgorithm(struct Graph *g) { int inDegree[MAX] = {0}; int queue[MAX], front=0, rear=0; int result[MAX], count=0; int i, j; /* Step 1: Compute in-degree of each vertex */ for (i=0; i<g->vertices; i++) for (j=0; j<g->vertices; j++) if (g->matrix[i][j]) inDegree[j]++; /* Step 2: Enqueue all vertices with in-degree 0 */ for (i=0; i<g->vertices; i++) if (inDegree[i]==0) queue[rear++]=i; /* Step 3: Process */ while (front < rear) { int v = queue[front++]; /* dequeue */ result[count++] = v; /* add to result */ for (j=0; j<g->vertices; j++) { if (g->matrix[v][j]) { inDegree[j]--; /* reduce in-degree */ if (inDegree[j]==0) queue[rear++]=j; } } } if (count != g->vertices) { /* not all processed = cycle */ printf("CIRCULAR DEPENDENCY DETECTED!\n"); return; } printf("Build/Task order: "); for (i=0; i<count; i++) printf("%d ", result[i]); printf("\n"); }
🔌 Connectivity Operations
Connectivity in Acyclic Graphs

Connected Graph Check: A graph is connected if every vertex is reachable from every other vertex. For directed graphs, we check strong connectivity (every vertex reaches every other) or weak connectivity (ignoring edge directions).

Connected Components: Groups of vertices that are connected to each other but not to vertices outside the group. Count them using repeated DFS — each new DFS start from an unvisited vertex = new component.

Reachable Vertices: From a given source, which vertices can be reached? Run DFS/BFS from that source and collect all visited vertices.

Isolation Check: A vertex is isolated if it has no edges (in-degree = 0 AND out-degree = 0). Check row and column of that vertex in the matrix — all zeros means isolated.

🏝️
Real-Life Analogy
Connected components are like islands in an ocean. Within each island, all towns are connected by bridges. But there's no bridge between different islands. An isolated vertex is a tiny island with no bridges at all — completely alone!

📐 Degree Operations
Vertex Degrees

Degree of a vertex is the number of edges connected to it.

• In-Degree: Number of edges coming INTO a vertex. In matrix: count 1s in the column of that vertex.

• Out-Degree: Number of edges going OUT from a vertex. In matrix: count 1s in the row of that vertex.

• Total Degree: In-Degree + Out-Degree (for directed) or just edge count (for undirected).

• Maximum Degree: Scan all vertices, find the one with the highest degree.

• Minimum Degree: Same, but find the lowest. In a connected graph, minimum degree ≥ 1.

Key fact for DAGs: Source vertices have in-degree = 0. Sink vertices have out-degree = 0. Kahn's algorithm exploits this!

✈️
Real-Life Analogy
In an airline network, a hub airport has a high degree (many incoming and outgoing flights). A small regional airport has a low degree. In-degree = flights arriving, out-degree = flights departing. A tiny airstrip with no connections is an isolated vertex!

🏗️ Structural Operations
Graph Size, Order & Structural Checks

Graph Order: The number of vertices (|V|). A.k.a. the "size" of the vertex set. g->vertices

Graph Size: The number of edges (|E|). Counted by scanning the matrix.

Sparse vs Dense Check:

• A graph is dense if E ≈ V² (most possible edges exist). Adjacency matrix is efficient here.

• A graph is sparse if E ≈ V (few edges). Adjacency list would be more memory-efficient.

• Rule of thumb: if E > V·log(V), consider it dense.

Tree Validation: Connected + Acyclic + Exactly V-1 edges.

Forest Validation: Acyclic + Exactly V-K edges (where K = number of components).

/* Graph Order (number of vertices) */ int graphOrder(struct Graph *g) { printf("Graph Order (vertices): %d\n", g->vertices); return g->vertices; } /* Graph Size (number of edges) */ int graphSize(struct Graph *g) { int e = countEdges(g); printf("Graph Size (edges): %d\n", e); return e; } /* Sparse / Dense Check */ void sparseDenseCheck(struct Graph *g) { int V = g->vertices; int E = countEdges(g); int maxEdges = V*(V-1); /* directed: max edges */ float density = (float)E / maxEdges; printf("Vertices: %d, Edges: %d\n", V, E); printf("Density: %.2f%%\n", density*100); if (density > 0.5) printf("Graph is DENSE\n"); else printf("Graph is SPARSE\n"); }

Longest Path in DAG — Theory

Use topological sort first. Then for each vertex u in topological order, for each neighbor v: dist[v] = max(dist[v], dist[u] + 1). Initialize all distances to 0. The longest path is max(dist[]).

This is O(V+E) — much better than Bellman-Ford which is O(V·E)!

C — Longest Path in DAG (DP)
/* Longest Path in DAG using Topological Sort + DP */ void longestPath(struct Graph *g) { int topoOrder[MAX], top=0; int dist[MAX], vis[MAX]={0}; int i, j; /* Step 1: Get topological order */ topoTop = -1; for(i=0; i<g->vertices; i++) topoVisited[i]=0; for(i=0; i<g->vertices; i++) if(!topoVisited[i]) topoSortDFS(g,i); /* Step 2: Initialize distances to 0 */ for(i=0; i<g->vertices; i++) dist[i]=0; /* Step 3: Process in topological order */ while(topoTop >= 0) { int u = topoStack[topoTop--]; for(j=0; j<g->vertices; j++) { if(g->matrix[u][j]) { if(dist[u]+1 > dist[j]) dist[j] = dist[u]+1; /* relax edge */ } } } /* Step 4: Find and print the longest path */ int maxDist=0; for(i=0; i<g->vertices; i++) if(dist[i]>maxDist) maxDist=dist[i]; printf("Longest path length: %d\n", maxDist); printf("Distances: "); for(i=0; i<g->vertices; i++) printf("dist[%d]=%d ", i, dist[i]); } /* DP on DAG: Count paths from source to each vertex */ void countPaths(struct Graph *g, int src) { int ways[MAX]={0}; int i,j; ways[src] = 1; /* 1 way to reach source */ /* Process in topological order (assumed stored in topoStack) */ while(topoTop >= 0) { int u = topoStack[topoTop--]; for(j=0;j<g->vertices;j++) if(g->matrix[u][j]) ways[j] += ways[u]; /* DP recurrence */ } printf("Paths from %d to each vertex:\n", src); for(i=0;i<g->vertices;i++) printf(" to %d: %d paths\n",i,ways[i]); }

Dependency Graph Processing

Build a dependency graph from pairs of (task, depends-on). Each pair becomes a directed edge. Then run Kahn's algorithm to get the execution order. If a cycle is detected, report circular dependency error. This is exactly how package managers (npm, pip, apt) work internally.

/* Build and process a dependency graph deps[][0] depends on deps[][1] Example: compile B before A */ void processDependencies(int deps[][2], int numDeps, int numTasks) { struct Graph g; initGraph(&g, numTasks); int i; /* Build the dependency DAG */ for(i=0; i<numDeps; i++) { int dependent = deps[i][0]; /* task that needs dep */ int dependency = deps[i][1]; /* task that must run first */ g.matrix[dependency][dependent] = 1; /* dep → dependent */ } /* Run Kahn's to get execution order */ printf("Execution order:\n"); kahnAlgorithm(&g); } /* Usage: int deps[][2] = {{1,0},{2,0},{3,1},{3,2}}; // Task 1 needs 0, task 2 needs 0, task 3 needs 1 and 2 processDependencies(deps, 4, 4); Output: 0 1 2 3 or 0 2 1 3 */
🛠️ Utility Operations
Utility Operations

These helper functions support the main operations and are essential for maintaining graph state:

• Is Empty: Check if the graph has 0 vertices or 0 edges.

• Clear Graph: Reset all matrix cells to 0 and set vertex count to 0. Equivalent to destroying and re-initializing the graph.

• Copy Graph: Create an exact duplicate of the graph (same matrix, same vertex count). Used before destructive operations.

• Display Graph: Pretty-print the adjacency matrix and/or edge list.

• Reset Visited Array: Set all cells in the visited[] array to 0. Done before each new DFS/BFS traversal to ensure clean state.
