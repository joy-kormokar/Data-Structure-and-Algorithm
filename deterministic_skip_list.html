📚 The Problem with Regular Linked Lists

A simple linked list lets you insert easily, but searching requires scanning every node from front to back — O(n) time. If you have 1 million items, you might check all 1 million.

🚀 Skip Lists — Express Lanes

A skip list adds extra layers above the base linked list. Higher layers are express lanes that skip over many nodes at once. Like a highway that skips small towns and only stops at cities.

🎲 Random vs Deterministic

Classic skip lists use a coin flip to decide which nodes get promoted. Deterministic skip lists use a fixed mathematical rule instead — typically every k-th node gets promoted to the next level. This gives predictable performance and structure.

⚡ Why Deterministic?

  • No randomness = reproducible structure every time
  • Guaranteed worst-case O(log n) performance
  • Easier to analyze, debug, and reason about
  • Used in databases and file systems for reliable indexing

/* ─── Maximum levels in our skip list ─── */
#define MAX_LEVEL 6
#define K_PROMOTE 2   /* Every 2nd node gets promoted */

/* ─── A single node in the skip list ─── */
typedef struct Node {
    int   key;                         /* the value stored */
    int   value;                       /* satellite data */
    int   level;                       /* how many levels this node spans */
    struct Node *forward[MAX_LEVEL];  /* array of next-pointers */
} Node;

/* ─── The skip list itself ─── */
typedef struct SkipList {
    Node *header;        /* sentinel head node */
    int   currentLevel;  /* highest active level */
    int   size;          /* total number of elements */
    int   insertCount;   /* used to determine promotion */
    int   k;             /* promotion rule: every k-th node */
} SkipList;

/* ─── Helper: create a new node ─── */
Node* createNode(int key, int value, int level) {
    Node *n = (Node*) malloc(sizeof(Node));
    n->key   = key;
    n->value = value;
    n->level = level;
    for (int i = 0; i < MAX_LEVEL; i++)
        n->forward[i] = NULL;
    return n;
}
💡 Real-Life Analogy
Think of a city highway system:

🏘️ Level 0 — Every house on every street (all elements)
🏙️ Level 1 — Main city roads (every 2nd important stop)
🛣️ Level 2 — Express highways (every 4th major hub)
✈️ Level 3 — Air routes (every 8th major city)

To travel from one end to another, you use highways first to cover large distances fast, then take local streets for the final stretch. A deterministic skip list does exactly this — it plans the "express lanes" in a predictable, formula-based way, not by flipping a coin.

What Happens During Initialization?

Before any data is stored, we must create the "skeleton" of the skip list. This involves three key steps:

  • Allocate Header: A special sentinel node that acts as the entry point. It has the smallest possible key (−∞) and is never removed.
  • Initialize Forward Pointers: Every level of the header points to NULL — meaning the list is empty.
  • Set Configuration: Define MAX_LEVEL (usually ⌊log₂ n⌋ + 1), current level (starts at 0), size (0), and the deterministic rule (k=2).
currentLevel = 0 | size = 0 | header→forward[i] = NULL for all i

Why a Header Node?

The header is a "dummy" node — it simplifies the code because we never have to handle an empty list as a special case. Insertions and searches always start from the header.

SkipList* createSkipList(int k) {
    /* Allocate memory for the skip list control struct */
    SkipList *sl = (SkipList*) malloc(sizeof(SkipList));

    /* ─── Allocate the header/sentinel node ─── */
    sl->header = createNode(-99999, 0, MAX_LEVEL - 1);

    /* ─── Initialize all forward pointers to NULL ─── */
    for (int i = 0; i < MAX_LEVEL; i++)
        sl->header->forward[i] = NULL;

    /* ─── Set initial state ─── */
    sl->currentLevel = 0;     /* no active levels yet */
    sl->size         = 0;     /* no elements */
    sl->insertCount  = 0;     /* counter for promotion rule */
    sl->k            = k;      /* promotion rule (e.g., k=2) */

    return sl;
}

    printf("Skip list created! Size: %d\n", sl->size); /* → 0 */
    return 0;
}

How Insertion Works — Step by Step

  1. 1
    Find Insert Position: Start from the highest active level, move right while the next key is smaller than target, then drop down. Record the predecessor at each level in an update[] array.
  2. 2
    Determine Node Level (Deterministically): Use the insertion count. If insertCount % k == 0, promote one more level. If insertCount % k² == 0, promote two levels, etc.
  3. 3
    Create New Node: Allocate memory and set the key/value.
  4. 4
    Splice In the Node: For each level 0 to nodeLevel, set:
    newNode→forward[i] = update[i]→forward[i]
    update[i]→forward[i] = newNode
  5. 5
    Update currentLevel if the new node's level is higher than current.

The Update Array

The update[] array is crucial — it stores the rightmost node at each level whose key is less than the new key. These are the "insertion points" where we splice in the new node.

nodeLevel = floor(log_k(insertCount)) capped at MAX_LEVEL

Cinsert() — Full Insertion with Deterministic Promotion
/* ─── Determine level deterministically based on insert count ─── */
int getNodeLevel(SkipList *sl) {
    int level = 0;
    int count = sl->insertCount;
    /* Promote if perfectly divisible by k, k², k³... */
    while (level < MAX_LEVEL - 1 && count % sl->k == 0) {
        level++;
        count /= sl->k;
    }
    return level;
}

/* ─── Main insert function ─── */
void insert(SkipList *sl, int key, int value) {
    Node *update[MAX_LEVEL];   /* predecessors at each level */
    Node *curr = sl->header;

    /* ─── Step 1: Traverse top-to-bottom to find position ─── */
    for (int i = sl->currentLevel; i >= 0; i--) {
        while (curr->forward[i] != NULL &&
               curr->forward[i]->key < key) {
            curr = curr->forward[i];  /* move forward */
        }
        update[i] = curr;  /* record predecessor */
    }

    /* ─── Skip if duplicate key ─── */
    curr = curr->forward[0];
    if (curr != NULL && curr->key == key) {
        curr->value = value;  /* update existing */
        return;
    }

    /* ─── Step 2: Determine level deterministically ─── */
    sl->insertCount++;
    int newLevel = getNodeLevel(sl);

    /* ─── Extend update array if new level is higher ─── */
    if (newLevel > sl->currentLevel) {
        for (int i = sl->currentLevel + 1; i <= newLevel; i++)
            update[i] = sl->header;
        sl->currentLevel = newLevel;
    }

    /* ─── Step 3: Create new node ─── */
    Node *newNode = createNode(key, value, newLevel);

    /* ─── Step 4: Splice new node into each level ─── */
    for (int i = 0; i <= newLevel; i++) {
        newNode->forward[i]  = update[i]->forward[i];
        update[i]->forward[i] = newNode;
    }

    sl->size++;  /* increment element count */
    printf("Inserted key=%d at level=%d\n", key, newLevel);
}

Like inserting a new employee into a company hierarchy:

🎖️ Every 8th hire becomes a Director (Level 3).

How Deletion Works

Deletion is the reverse of insertion. We must "unlink" the target node from all levels it participates in, then free its memory.

  1. 1
    Find Predecessors: Traverse top-to-bottom, building the update[] array of nodes that point to (or past) the target.
  2. 2
    Verify Target Exists: Check that update[0]→forward[0] is the target key. If not, the key doesn't exist — return.
  3. 3
    Remove Forward Links: For each level from 0 upward, if update[i]→forward[i] is our target, redirect it: update[i]→forward[i] = target→forward[i]
  4. 4
    Reduce Current Level: After removal, shrink currentLevel if top levels are now empty (header points to NULL).
  5. 5
    Free Memory: Call free() on the removed node. Decrease size.

Cdelete() — Unlinking a Node from All Levels
void deleteNode(SkipList *sl, int key) {
    Node *update[MAX_LEVEL];
    Node *curr = sl->header;

    /* ─── Step 1: Build the update array ─── */
    for (int i = sl->currentLevel; i >= 0; i--) {
        while (curr->forward[i] != NULL &&
               curr->forward[i]->key < key) {
            curr = curr->forward[i];
        }
        update[i] = curr;
    }

    /* ─── Step 2: Get candidate node ─── */
    Node *target = update[0]->forward[0];

    /* ─── Step 3: Verify it exists with matching key ─── */
    if (target == NULL || target->key != key) {
        printf("Key %d not found.\n", key);
        return;
    }

    /* ─── Step 4: Remove forward links level by level ─── */
    for (int i = 0; i <= sl->currentLevel; i++) {
        if (update[i]->forward[i] != target) break;
        update[i]->forward[i] = target->forward[i];
    }

    /* ─── Step 5: Reduce currentLevel if top levels empty ─── */
    while (sl->currentLevel > 0 &&
           sl->header->forward[sl->currentLevel] == NULL) {
        sl->currentLevel--;
    }

    /* ─── Step 6: Free memory and update count ─── */
    free(target);
    sl->size--;
    printf("Deleted key=%d successfully.\n", key);
}
💡 Real-Life Analogy
Like removing a stop from a bus route network:

🚌 Imagine a city with express buses, local buses, and walking paths (3 levels).
If you close a bus stop:
1. Update the express routes to skip that stop (remove from level 2)
2. Update the local routes to skip it (remove from level 1)
3. Remove the stop from the walking path too (level 0)
4. If the express route now has no stops, cancel the express route (reduce current level)
5. Demolish the physical stop (free memory)

The Search Algorithm

This is the core operation. Start at the top level (the fastest "highway") and work your way down:

  1. 1
    Start at Top Level: Begin at header→forward[currentLevel].
  2. 2
    Move Forward: While the next node's key is less than the target, advance forward on the current level.
  3. 3
    Drop Down: When you can't move forward (next key ≥ target), drop to the level below.
  4. 4
    Repeat: Continue until you reach Level 0 and can no longer move forward.
  5. 5
    Check: The next node at Level 0 is either our target (found!) or a larger key (not found).

Why is it O(log n)?

Each level skips approximately k nodes. With k=2, Level 1 skips 2 nodes, Level 2 skips 4, etc. So the total comparisons across all levels is about log₂(n).

Csearch() — Top-Down Skip List Search
/* ─── Returns pointer to node if found, NULL otherwise ─── */
Node* search(SkipList *sl, int key) {
    Node *curr = sl->header;

    /* ─── Start from the highest active level ─── */
    for (int i = sl->currentLevel; i >= 0; i--) {

        /* ─── Move forward while next key is smaller ─── */
        while (curr->forward[i] != NULL &&
               curr->forward[i]->key < key) {
            curr = curr->forward[i];
            printf("  Level %d: moved to key %d\n", i, curr->key);
        }
        /* Drop to lower level when can't go forward */
        printf("  Dropping from level %d to %d\n", i, i-1);
    }

    /* ─── Check the node at Level 0 ─── */
    curr = curr->forward[0];

    if (curr != NULL && curr->key == key) {
        printf("FOUND: key=%d, value=%d\n", curr->key, curr->value);
        return curr;
    }

    printf("NOT FOUND: key=%d\n", key);
    return NULL;
}

/* ─── Usage example ─── */
Node *result = search(sl, 42);
if (result) printf("Value: %d\n", result->value);
else        printf("Key not in list.\n");
💡 Real-Life Analogy
Like looking up a word in a dictionary using index tabs:

📖 The dictionary has alphabet tabs (top level — jump to 'M' section instantly).
📑 Then chapter dividers inside 'M' (level 1 — 'Ma', 'Me', 'Mi'...).

You don't start at page 1. You start at the highest-level tab that gets you close, then drill down. Skip lists do the same — they always start at the top and work downward, skipping massive swaths of data at higher levels.

Why Update is Simple

In a skip list, keys are immutable — once a node is placed, its position in the sorted order is fixed. Only the value (satellite data) can change. This means we just need to find the node and overwrite its value field.

Update Process

  1. 1
    Search: Run the normal search algorithm to find the node with the given key.
  2. 2
    Validate: If the key doesn't exist, return an error. Don't insert a new node.
  3. 3
    Overwrite: Set node→value = newValue. No pointer changes needed — the node stays in the same position at all levels.

Important Note

If you need to change the key (not the value), you must delete the old node and insert a new one, since keys determine position in the sorted structure.

Cupdate() — Modify a Node's Value In-Place
int update(SkipList *sl, int key, int newValue) {
    /* ─── Search for the node ─── */
    Node *target = search(sl, key);

    /* ─── Validate: does the key exist? ─── */
    if (target == NULL) {
        printf("Update failed: key %d not found.\n", key);
        return -1;  /* error code */
    }

    /* ─── Replace old data with new value ─── */
    int oldValue = target->value;
    target->value = newValue;

    printf("Updated key=%d: %d → %d\n", key, oldValue, newValue);
    return 0;  /* success */
}

/* ─── If you want to change the KEY itself: ─── */
void updateKey(SkipList *sl, int oldKey,
                int newKey, int value) {
    deleteNode(sl, oldKey);  /* remove old position */
    insert(sl, newKey, value);  /* insert at new position */
}

📒 The grade book is sorted by student ID (the key — never changes).

You don't re-sort the whole book just to update a grade. You find the student's row (search by ID), erase the old grade, write the new one. Done — the student stays in the same row. That's exactly what update does in a skip list.

/* ─── Level-wise display (shows skip list structure) ─── */

    printf("\n=== Skip List Structure ===\n");

    for (int i = sl->currentLevel; i >= 0; i--) {
        printf("Level %d: HEAD", i);
        Node *curr = sl->header->forward[i];

        while (curr != NULL) {
            printf(" --> [%d]", curr->key);
            curr = curr->forward[i];
        }
        printf(" --> NULL\n");
    }
}

    if (level < 0 || level > sl->currentLevel) {
        printf("Invalid level.\n");
        return;

    printf("Level %d nodes: ", level);
    Node *curr = sl->header->forward[level];
    while (curr) {
        printf("%d ", curr->key);
        curr = curr->forward[level];
    }
    printf("\n");
}

What Are Levels?

Each node spans one or more levels. A node at level L has forward pointers for levels 0, 1, ..., L. currentLevel tracks the highest level that has any node in it.

Promotion Rule (Deterministic)

With k=2: After inserting n nodes, the distribution should be:

  • Level 0: all n nodes
  • Level 1: every 2nd node → ~n/2 nodes
  • Level 2: every 4th node → ~n/4 nodes
  • Level L: ~n/2^L nodes
max useful level = ⌊log_k(n)⌋

Empty Level Removal

After deletion, if the top level's header forward pointer becomes NULL, that level is empty. We decrement currentLevel — this keeps searches efficient by not scanning useless empty levels.

/* ─── Check and reduce empty top levels ─── */
void removeEmptyLevels(SkipList *sl) {
    while (sl->currentLevel > 0 &&
           sl->header->forward[sl->currentLevel] == NULL) {
        sl->currentLevel--;
        printf("Removed empty level. Current: %d\n",

/* ─── Increase current level when adding tall node ─── */
void increaseLevel(SkipList *sl, int newLevel,
                   Node *update[]) {
    if (newLevel > sl->currentLevel) {
        for (int i = sl->currentLevel + 1; i <= newLevel; i++)
            update[i] = sl->header;  /* header is predecessor */
        sl->currentLevel = newLevel;
    }
}

/* ─── Determine ideal max level for n elements ─── */
int idealMaxLevel(int n, int k) {
    int level = 0;
    int count = n;
    while (count > 1) {
        count /= k;
        level++;
    }
    return level;
}

The Heart of Determinism

This is what makes a deterministic skip list special. Instead of flipping a coin, we use the insertion sequence number to decide a node's level.

Promotion Rule (k=2)

  • 1st insert: count=1 → 1%2≠0 → Level 0
  • 2nd insert: count=2 → 2%2=0 → Level 1; 2%4≠0 → stops
  • 3rd insert: count=3 → 3%2≠0 → Level 0
  • 4th insert: count=4 → 4%2=0 → Level 1; 4%4=0 → Level 2; 4%8≠0 → stops
  • 8th insert: → Level 0,1,2,3
level(insertCount) = v₂(insertCount) where v₂ = 2-adic valuation

Why This Works

The 2-adic valuation (number of times 2 divides n) gives exactly the binary tree structure we need. Every 2nd element gets 1 extra level, every 4th gets 2 extra, etc. — perfectly balanced.

CPromotion Rule — Deterministic Level Assignment
/* ─── 2-adic valuation: count how many times k divides n ─── */
int kAdicValuation(int n, int k) {
    int level = 0;
    while (n % k == 0 && level < MAX_LEVEL - 1) {
        level++;
        n /= k;
    }
    return level;
}

/* ─── Apply the promotion rule before each insert ─── */
int applyPromotionRule(SkipList *sl) {
    sl->insertCount++;  /* increment before deciding */
    int level = kAdicValuation(sl->insertCount, sl->k);
    printf("Insert #%d → Level %d\n", sl->insertCount, level);
    return level;
}

/* ─── Enforce structural constraints ─── */
int checkPromotionCondition(int insertCount, int k, int level) {
    /* Can this node be at 'level'? */
    int required = 1;
    for (int i = 0; i < level; i++) required *= k;
    return (insertCount % required == 0) ? 1 : 0;
}

/* ─── Example output for k=2:
   Insert  1 → Level 0  (1 is odd)
   Insert  2 → Level 1  (2 = 2¹)
   Insert  3 → Level 0
   Insert  4 → Level 2  (4 = 2²)
   Insert  5 → Level 0
   Insert  6 → Level 1  (6 = 2×3)
   Insert  8 → Level 3  (8 = 2³) ─── */

Forward Pointer Arrays

Each node has a forward[] array of size MAX_LEVEL. forward[0] links to the next node at the base level (like a regular linked list). forward[i] links to the next node that is also at level i or above.

Key Pointer Operations

  • Insert Splice: new→forward[i] = update[i]→forward[i]; update[i]→forward[i] = new;
  • Delete Redirect: update[i]→forward[i] = deleted→forward[i];
  • Broken Links: A NULL pointer means "end of list at this level". Never dereference a NULL forward pointer.
  • Initialization: All pointers start as NULL at allocation time.

Common Pointer Bugs

  • Forgetting to update a level (partial unlinking)
  • Dereferencing NULL (check before accessing)
  • Not initializing all MAX_LEVEL entries (use memset or a loop)

/* ─── Safely initialize all pointers in a node ─── */
void initPointers(Node *node) {
    for (int i = 0; i < MAX_LEVEL; i++)
        node->forward[i] = NULL;
}

/* ─── Splice node in at level i (insert) ─── */
void spliceIn(Node *pred, Node *newNode, int level) {
    newNode->forward[level] = pred->forward[level];
    pred->forward[level]    = newNode;
}

/* ─── Redirect pointer around deleted node ─── */
void redirectPointer(Node *pred, Node *deleted, int level) {
    if (pred->forward[level] == deleted)
        pred->forward[level] = deleted->forward[level];
}

/* ─── Check for and log any broken links ─── */
void validatePointers(SkipList *sl) {
    for (int i = 0; i <= sl->currentLevel; i++) {
        Node *curr = sl->header->forward[i];
        Node *prev = sl->header;
        while (curr != NULL) {
            if (prev->key >= curr->key) {
                printf("BROKEN LINK at level %d!\n", i);
            }
            prev = curr;
            curr = curr->forward[i];
        }
    }
    printf("Pointer validation complete.\n");
}

When Is Rebalancing Needed?

After many deletions, the skip list may lose its balance — the upper levels might become sparse or unbalanced, making searches slower than O(log n). Rebalancing rebuilds the structure from scratch.

Rebalancing Strategy

  1. 1
    Collect All Keys: Traverse Level 0 and store all keys in an array (in sorted order).
  2. 2
    Clear the List: Reset all pointers, levels, and counters.
  3. 3
    Re-insert with Fresh Counts: Insert each key back, resetting insertCount to 0. This reassigns levels deterministically based on position, not history.

Note on Deterministic Skip Lists

Well-maintained deterministic skip lists rarely need full rebalancing — if insertions follow the promotion rule faithfully, the structure stays balanced automatically. Rebalancing is mainly needed after bulk deletions or manual structural modifications.

/* ─── Rebalance by rebuilding from Level 0 ─── */
void rebalance(SkipList *sl) {
    /* Step 1: Collect all keys from Level 0 */
    int keys[1000], vals[1000], count = 0;
    Node *curr = sl->header->forward[0];
    while (curr != NULL) {
        keys[count] = curr->key;
        vals[count] = curr->value;
        count++;
        curr = curr->forward[0];
    }

    /* Step 2: Fully clear the skip list */
    clearSkipList(sl);  /* see Utility Operations */
    sl->insertCount = 0;
    sl->currentLevel = 0;
    sl->size = 0;

    /* Step 3: Re-insert all keys (fresh promotion counts) */
    for (int i = 0; i < count; i++) {
        insert(sl, keys[i], vals[i]);
    }
    printf("Rebalanced. %d nodes re-inserted.\n", count);
}

/* ─── Redistribute nodes after bulk deletion ─── */
void adjustNodeHeights(SkipList *sl) {
    /* Verify each node's level matches promotion rule */
    int count = 1;
    Node *curr = sl->header->forward[0];
    while (curr != NULL) {
        int expectedLevel = kAdicValuation(count, sl->k);
        if (curr->level != expectedLevel)
            printf("Node %d: level %d, expected %d\n",
                   curr->key, curr->level, expectedLevel);
        count++;
        curr = curr->forward[0];
    }
}

Essential Utility Functions

Every data structure needs basic utility operations for checking state and maintenance:

  • isEmpty: Returns 1 if the skip list has zero elements. Simply checks sl→size == 0 or header→forward[0] == NULL.
  • getSize: Returns the count of elements. Since we maintain sl→size, this is O(1) — no traversal needed.
  • clear: Removes all nodes and resets to initial state. Traverses Level 0 and frees all nodes except the header.
  • countNodes: An alternative O(n) count by actually traversing — useful for debugging to verify the size counter is accurate.

/* ─── Check if skip list is empty ─── */
int isEmpty(SkipList *sl) {

/* ─── Get number of elements ─── */

}

/* ─── Count nodes by traversal (for verification) ─── */
int countNodes(SkipList *sl) {
    int count = 0;
    Node *curr = sl->header->forward[0];
    while (curr != NULL) {
        count++;
        curr = curr->forward[0];
    }
    return count;  /* O(n) - for debugging */
}

/* ─── Clear skip list (remove all data nodes) ─── */
void clearSkipList(SkipList *sl) {
    Node *curr = sl->header->forward[0];
    while (curr != NULL) {
        Node *next = curr->forward[0];
        free(curr);    /* free the node */
        curr = next;
    }
    /* Reinitialize header */
    for (int i = 0; i < MAX_LEVEL; i++)
        sl->header->forward[i] = NULL;
    sl->currentLevel = 0;
    sl->size = 0;
    sl->insertCount = 0;
    printf("Skip list cleared.\n");
}

Memory in C — The Developer's Responsibility

Unlike Python or Java, C has no garbage collector. Every byte you allocate with malloc() must be explicitly released with free(). Failing to do so causes a memory leak.

Skip List Memory Layout

  • Each node: sizeof(Node) = sizeof(int)*3 + sizeof(Node*)*MAX_LEVEL
  • For MAX_LEVEL=6 on 64-bit: 3×4 + 6×8 = 12 + 48 = 60 bytes per node
  • Total for n nodes: O(n) space

Memory Leak Prevention

  • Always save the pointer before redirecting it
  • Free nodes after deletion (free(target))
  • Use a freeSkipList() function at program end
  • Use Valgrind to detect leaks during testing
CMemory Management — Allocate, Free, Prevent Leaks
#include <stdlib.h>

/* ─── Allocate a new node ─── */
Node* createNode(int key, int value, int level) {
    Node *n = (Node*) malloc(sizeof(Node));

    if (n == NULL) {
        printf("ERROR: malloc failed!\n");
        exit(1);  /* out of memory */
    }

    n->key   = key;
    n->value = value;
    n->level = level;
    for (int i = 0; i < MAX_LEVEL; i++)
        n->forward[i] = NULL;  /* always init! */
    return n;
}

/* ─── Free a single node ─── */
void freeNode(Node *node) {
    if (node != NULL) free(node);
}

/* ─── Free the entire skip list (no memory leaks!) ─── */
void freeSkipList(SkipList *sl) {
    Node *curr = sl->header;
    while (curr != NULL) {
        Node *next = curr->forward[0];
        free(curr);   /* free this node */
        curr = next;  /* move to next BEFORE freeing */
    }
    free(sl);  /* free the container struct */
    printf("All memory released.\n");
}

/* ─── BAD (memory leak!) ─── */
void badDelete(SkipList *sl, Node *target) {
    /* forgot to free — target's memory leaks! */
    update[0]->forward[0] = target->forward[0];
}

/* ─── GOOD (no leak) ─── */
void goodDelete(SkipList *sl, Node *target) {
    update[0]->forward[0] = target->forward[0];
    free(target);  /* always free the node! */
}

✅ Good practice = Always check out properly. Use Valgrind (a tool) to find forgotten checkouts.

/* ─── Count nodes at each level ─── */
void analyzeStructure(SkipList *sl) {
    printf("=== Structure Analysis ===\n");
    printf("Total nodes: %d\n", sl->size);
    printf("Active levels: %d\n", sl->currentLevel + 1);

    for (int i = sl->currentLevel; i >= 0; i--) {
        int count = 0;
        Node *curr = sl->header->forward[i];
        while (curr) { count++; curr = curr->forward[i]; }

        double expected = sl->size;
        for (int j = 0; j < i; j++) expected /= sl->k;

        printf("  Level %d: %d nodes (expected: %.1f)\n",
               i, count, expected);

/* ─── Measure actual search path length ─── */
int measureSearchComplexity(SkipList *sl, int key) {
    int comparisons = 0;
    Node *curr = sl->header;

    for (int i = sl->currentLevel; i >= 0; i--) {
        while (curr->forward[i] != NULL) {
            comparisons++;
            if (curr->forward[i]->key < key)
                curr = curr->forward[i];
            else break;
        }
    }
    printf("Search for %d: %d comparisons\n", key, comparisons);
    return comparisons;
}

/* ─── Check balance efficiency (ratio) ─── */
double checkBalanceEfficiency(SkipList *sl) {
    double ideal = 0, actual = 0;
    for (int i = 0; i <= sl->currentLevel; i++) {
        double exp = sl->size;
        for (int j = 0; j < i; j++) exp /= sl->k;
        ideal += exp;
        Node *c = sl->header->forward[i];
        while (c) { actual++; c = c->forward[i]; }
    }
    return actual / ideal;  /* 1.0 = perfectly balanced */
}

What Is an Iterator?

An iterator is an object (or struct in C) that keeps track of the current position in the skip list. It lets you visit nodes one by one without knowing the internal structure — clean and encapsulated.

Iterator Design

An iterator for a skip list is simple — it just holds a pointer to the current node at Level 0. Level 0 has all nodes in sorted order, so we always iterate at Level 0.

Iterator Operations

  • init: Point to the first real node (header→forward[0])
  • hasNext: Return true if current pointer is not NULL
  • next: Return current node, advance pointer to forward[0]
  • reset: Point back to header→forward[0]

CIterator Pattern — Struct-based Iterator in C
/* ─── Iterator struct ─── */
typedef struct {
    Node *current;   /* points to current node */
    Node *start;     /* first real node (for reset) */
} Iterator;

/* ─── Initialize iterator ─── */
Iterator iteratorInit(SkipList *sl) {
    Iterator it;
    it.current = sl->header->forward[0]; /* first real node */
    it.start   = sl->header->forward[0];
    return it;
}

/* ─── Check if there are more elements ─── */
int hasNext(Iterator *it) {
    return (it->current != NULL);
}

/* ─── Get current value and advance ─── */
Node* iterNext(Iterator *it) {
    if (!hasNext(it)) return NULL;
    Node *node  = it->current;
    it->current = it->current->forward[0]; /* advance */
    return node;
}

/* ─── Reset to beginning ─── */
void iterReset(Iterator *it) {
    it->current = it->start;
}

/* ─── Usage example ─── */
Iterator it = iteratorInit(sl);
while (hasNext(&it)) {
    Node *n = iterNext(&it);
    printf("Key: %d, Value: %d\n", n->key, n->value);
}
💡 Real-Life Analogy
Like a bookmark in a book:

🔖 The iterator struct is your bookmark — it remembers where you left off.
📖 iterNext() = Turn to the next page and read, then move the bookmark forward.
🔍 hasNext() = "Am I at the last page?" Check before turning.
↩️ iterReset() = Move the bookmark back to the first page.

You can hand your bookmark to someone else, and they can continue reading from where you left off. That's the beauty of the iterator pattern — it separates "where we are" from "the book itself."

Range Query — The Most Powerful Skip List Feature

A range query finds all elements with keys between L (low) and H (high). This is where skip lists shine — search runs in O(log n) to find L, then traverse Level 0 to reach H in O(k) steps.

Range Query Time = O(log n + k) where k = number of results

Min and Max — O(1) Access

Since Level 0 is sorted: Minimum = header→forward[0] (first node). Maximum = traverse Level 0 to last node.

Merge — Combining Two Skip Lists

Merge by traversing both Level 0s simultaneously (like merge sort), inserting into a new skip list. Time: O((m+n) log(m+n)).

Split — Dividing at a Key

Split into two lists: one with keys < splitKey, one with keys ≥ splitKey. Traverse Level 0, insert each node into the appropriate new list.

/* ─── RANGE QUERY: find all keys in [low, high] ─── */
void rangeQuery(SkipList *sl, int low, int high) {
    Node *curr = sl->header;
    /* Step 1: Use skip list search to jump to 'low' */
    for (int i = sl->currentLevel; i >= 0; i--)
        while (curr->forward[i] && curr->forward[i]->key < low)
            curr = curr->forward[i];
    curr = curr->forward[0];

    /* Step 2: Traverse Level 0 until key > high */
    printf("Range [%d, %d]: ", low, high);
    while (curr != NULL && curr->key <= high) {
        printf("%d ", curr->key);
        curr = curr->forward[0];
    }
    printf("\n");
}

/* ─── FIND MINIMUM (O(1)) ─── */
Node* findMin(SkipList *sl) {
    return sl->header->forward[0];  /* first node */
}

/* ─── FIND MAXIMUM (O(log n) using top level) ─── */
Node* findMax(SkipList *sl) {
    Node *curr = sl->header;
    for (int i = sl->currentLevel; i >= 0; i--)
        while (curr->forward[i] != NULL)
            curr = curr->forward[i];
    return (curr == sl->header) ? NULL : curr;
}

/* ─── MERGE two skip lists into a new one ─── */
SkipList* merge(SkipList *a, SkipList *b) {
    SkipList *result = createSkipList(a->k);
    Node *pA = a->header->forward[0];
    Node *pB = b->header->forward[0];

    while (pA && pB) {
        if (pA->key <= pB->key) {
            insert(result, pA->key, pA->value);
            pA = pA->forward[0];
        } else {
            insert(result, pB->key, pB->value);
            pB = pB->forward[0];
        }
    }
    while (pA) { insert(result, pA->key, pA->value); pA = pA->forward[0]; }
    while (pB) { insert(result, pB->key, pB->value); pB = pB->forward[0]; }
    return result;
}

/* ─── SPLIT at splitKey ─── */
void split(SkipList *sl, int splitKey,
            SkipList **left, SkipList **right) {
    *left  = createSkipList(sl->k);
    *right = createSkipList(sl->k);
    Node *curr = sl->header->forward[0];
    while (curr) {
        if (curr->key < splitKey)
            insert(*left,  curr->key, curr->value);
        else
            insert(*right, curr->key, curr->value);
        curr = curr->forward[0];
    }
}
