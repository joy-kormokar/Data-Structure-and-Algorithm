What is a Priority Queue?

Concepts, structure, and why sorted linked lists make sense

📋 Priority Queue

A special queue where every element has a priority value. Elements are served based on priority — not just arrival order. Highest (or lowest) priority goes first.

🔗 Sorted Linked List

We store nodes in a linked list, always keeping them sorted by priority. This makes peek O(1) — the highest-priority item is always at the front (or back).

🏗️ Node Structure

Each node holds a data value, a priority number, and a next pointer to the following node. Simple — yet powerful.

🏥
Real-Life Analogy — Hospital Emergency Room

Patients arrive at different times, but the most critically ill are treated first. A patient with priority 1 (critical) is always ahead of priority 5 (minor cut). The doctor always sees the head of the sorted queue — no searching needed. This is exactly how a sorted linked-list priority queue works.

Node Structure in C

C — Node Definition 
// Every node in our sorted linked list holds:
// 1) data   — the actual value stored
// 2) priority — determines order (lower number = higher priority here)
// 3) next   — pointer to the next node

struct Node {
    int data;       // the value we store
    int priority;   // smaller number = higher priority
    struct Node* next; // link to next node
};

// Our queue just needs to track the HEAD of the sorted list
struct Node* head = NULL; // empty queue to start

Priority Queue Initialization

Setting up the empty structure — creating the head pointer

🎯 What Happens

We simply declare a pointer called head and set it to NULL. A NULL head means the list is empty. That's the entire initialization!

📐 Sorting Rule

Decide upfront: ascending (priority 1 is highest) or descending (priority 10 is highest). We'll use ascending (lower = higher priority) throughout this guide.

📭
Analogy — Empty Mailbox

Initializing is like placing an empty mailbox outside your house. The mailbox exists (head pointer), but nothing is inside yet (NULL). Later, letters (nodes) arrive and are sorted by importance.

Insert / Enqueue Operation

Add a new element into its correct sorted position

1

Allocate memory for a new node using malloc()

2

Set data and priority values in the new node

3

Find the correct position — traverse until we find a node with lower priority (higher number)

4

Insert the node — adjust prev→next and newNode→next pointers

5

Handle edge cases — inserting at head (empty list, or highest priority)

🗂️
Analogy — Filing Cabinet

When you add a new document, you don't throw it on top. You flip through the folders until you find where it belongs by date/urgency, then slide it in. Every folder stays in order — this is exactly O(n) sorted insertion.

C — Insert (Enqueue) 
/* Insert a node in sorted order (ascending priority).
   Lower priority number = served first (highest priority). */

void enqueue(struct Node** head, int data, int priority) {

    // Step 1: Allocate memory for new node
    struct Node* newNode = (struct Node*) malloc(sizeof(struct Node));
    if (newNode == NULL) {
        printf("Memory allocation failed!\n");
        return;
    }

    // Step 2: Fill the new node
    newNode->data     = data;
    newNode->priority = priority;
    newNode->next     = NULL;

    // Step 3: Insert at head if list is empty OR new node has highest priority
    if (*head == NULL || (*head)->priority > priority) {
        newNode->next = *head;
        *head = newNode;
        return;
    }

    // Step 4: Find correct position by traversing
    struct Node* curr = *head;
    while (curr->next != NULL && curr->next->priority <= priority) {
        curr = curr->next;
    }

    // Step 5: Insert at found position
    newNode->next = curr->next;
    curr->next    = newNode;
}
Time: O(n) — must find correct position
Space: O(1) — only one new node
Edge: Insert into empty list

Delete Highest Priority Element

Remove the element that must be served next

⚡ O(1) Operation

Because the list is sorted, the highest-priority element is always at the head. Deleting it is simply moving the head pointer forward — no traversal needed!

🗑️ Free Memory

Always free() the deleted node's memory to avoid memory leaks. This is critical in C where memory is manually managed.

🚦
Analogy — Traffic Priority Lane

The ambulance is always at the front. When the traffic clears, the ambulance (highest priority) goes first. The next-most-urgent vehicle automatically becomes the new front. No one needs to rearrange — it's already sorted!

C — Delete Highest Priority (Dequeue) 
/* Remove and return the highest-priority element.
   It's always at the HEAD of the sorted list! */

int dequeueHigh(struct Node** head) {

    // Safety check: don't delete from empty queue
    if (*head == NULL) {
        printf("Queue is empty! Cannot delete.\n");
        return -1;
    }

    // Save the front node (highest priority)
    struct Node* temp = *head;
    int val = temp->data;

    // Move head to the next node
    *head = (*head)->next;

    // Free the removed node's memory
    free(temp);

    return val; // return the deleted value
}

// Usage:
int served = dequeueHigh(&head);
printf("Served: %d\n", served);

Delete Lowest Priority Element

Remove the element at the back of the sorted list

🔚 Tail Operation

The lowest-priority element sits at the tail of our sorted list. We must traverse to find the second-to-last node, then unlink and free the last.

⏱️ O(n) Time

Unlike deleting from the head, removing from the tail requires a full traversal to find the node before the last one.

🎭
Analogy — Removing the Least Urgent Task

Imagine a to-do list sorted from most to least urgent. Sometimes you decide "I'll never get to this" and cross off the least urgent item at the bottom. You have to flip to the last page to find it — that's O(n) traversal.

Interactive — Delete Lowest Priority
[ Load sample data first ]
—
C — Delete Lowest Priority 
/* Remove the element with the LOWEST priority (at the TAIL). */

int dequeueLow(struct Node** head) {

    if (*head == NULL) {
        printf("Queue is empty!\n");
        return -1;
    }

    // Special case: only one node in the list
    if ((*head)->next == NULL) {
        int val = (*head)->data;
        free(*head);
        *head = NULL;
        return val;
    }

    // Traverse to find the second-to-last node
    struct Node* curr = *head;
    while (curr->next->next != NULL) {
        curr = curr->next;
    }

    // curr is now the second-to-last node
    int val = curr->next->data;
    free(curr->next); // free the tail
    curr->next = NULL; // mark new tail

    return val;
}

Peek Highest Priority Element

Look at the top element without removing it

👀 Non-Destructive

Peek reads without deleting. Think of it as "who's next?" without actually calling them in.

⚡ O(1) Always

The highest-priority element is always at head. Peeking is just head→data — instant!

👁️
Analogy — Checking Next Patient

The nurse checks the front of the waiting list to see who's next — without actually calling them in yet. The doctor might be finishing up; the nurse just needs to know who's coming next.

C — Peek Highest Priority 
/* Peek at highest priority element WITHOUT removing it. */

int peekHigh(struct Node* head) {
    if (head == NULL) {
        printf("Queue is empty!\n");
        return -1;
    }
    // Highest priority is always at HEAD — O(1)!
    return head->data;
}

// Usage:
printf("Next to be served: %d (priority %d)\n",
       head->data, head->priority);
06

Peek Lowest Priority Element

Look at the last element without removing it

🔚 Tail Access

The lowest-priority element is at the tail. We traverse to the end — O(n) operation unless we maintain a tail pointer.

💡 Optimization Tip

Keep a tail pointer alongside head to make this O(1). Advanced versions of this data structure maintain both.

C — Peek Lowest Priority 
/* Peek at lowest priority element (tail of sorted list). */

int peekLow(struct Node* head) {
    if (head == NULL) {
        printf("Queue is empty!\n");
        return -1;
    }

    // Traverse to the last node
    struct Node* curr = head;
    while (curr->next != NULL) {
        curr = curr->next;
    }

    return curr->data; // lowest priority element
}

// --- OPTIMIZED VERSION with tail pointer ---
struct Node* tail = NULL; // maintain a tail pointer

int peekLowFast() {
    if (tail == NULL) return -1;
    return tail->data; // O(1) with tail pointer!
}

IsEmpty Operation

Check whether the queue contains any elements

📭
Analogy — Checking the Mailbox

Before reaching into the mailbox, you peek inside. If it's empty, you don't bother. In code, we check head == NULL before any operation to prevent crashes.

C — IsEmpty 
/* Returns 1 (true) if queue is empty, 0 (false) if not. */

int isEmpty(struct Node* head) {
    return (head == NULL); // 1 = empty, 0 = not empty
}

// Usage — always check before delete/peek:
if (isEmpty(head)) {
    printf("Queue is empty! Nothing to process.\n");
} else {
    printf("Queue has elements. Ready to serve.\n");
}

C — Display 
/* Print all elements from highest to lowest priority. */

void display(struct Node* head) {
    if (head == NULL) {
        printf("Queue is empty.\n");
        return;
    }

    printf("Priority Queue (High → Low):\n");
    printf("HEAD → ");

    struct Node* curr = head;
    while (curr != NULL) {
        printf("[data:%d, prio:%d] → ", curr->data, curr->priority);
        curr = curr->next;
    }

    printf("NULL\n");
}

Traverse Priority Queue

Visit every node sequentially (forward traversal)

➡️ Forward Only

Singly linked lists can only be traversed forward (head → tail). Each node visits its next pointer until reaching NULL.

🔁 Common Pattern

The traversal pattern curr = curr→next is the most fundamental loop in linked list programming. Master it early!

C — Traverse (Forward) 
/* Standard forward traversal — visits every node once. */

void traverse(struct Node* head) {
    struct Node* curr = head; // start at head

    int position = 1;
    while (curr != NULL) {  // stop when we hit NULL
        printf("Position %d: data=%d, priority=%d\n",
               position, curr->data, curr->priority);
        curr = curr->next;   // move to next node
        position++;
    }
    printf("Traversal complete. Total: %d nodes.\n", position - 1);
}

/* Key insight:
   - curr starts at HEAD
   - Each iteration: process curr, then move curr = curr->next
   - Stop when curr == NULL (end of list)
*/

Search Element

Find a node by data value using linear traversal

🔍 Linear Search

We compare each node's data value as we traverse. Worst case: O(n) — the item is at the tail or doesn't exist.

🎯 Search by Data vs Priority

You can search by data (what's stored) or by priority (its rank). The approach is the same — just compare a different field.

🔎
Analogy — Finding a Student in a Sorted Roll Call

The teacher calls names alphabetically. To find "John", she reads through the list until "John" appears. She's doing linear search — exactly what we do with linked list traversal.

Interactive — Search
[ Load sample data first ]

C — Search Element 
/* Search for a node by its data value.
   Returns the node pointer if found, NULL if not found. */

struct Node* search(struct Node* head, int target) {
    struct Node* curr = head;
    int position = 1;

    while (curr != NULL) {
        if (curr->data == target) {
            printf("Found %d at position %d (priority: %d)\n",
                   target, position, curr->priority);
            return curr; // found!
        }
        curr = curr->next;
        position++;
    }

    printf("%d not found in queue.\n", target);
    return NULL; // not found
}

// Usage:
struct Node* found = search(head, 42);
if (found != NULL)
    printf("Priority of found element: %d\n", found->priority);

Update Element Priority

Change a node's priority and reinsert it in sorted position

1

Search for the target node by data value

2

Remove the node from its current position (track prev pointer)

3

Update the priority value in the node

4

Reinsert the node using enqueue() logic to maintain sort order

🔄
Analogy — Patient Condition Change

A patient in the waiting room's condition suddenly worsens. The nurse pulls them from their current spot in the queue and moves them closer to the front based on the new urgency. The queue must remain sorted after the update.

C — Update Element Priority 
/* Update a node's priority: remove, modify, reinsert. */

void updatePriority(struct Node** head, int target, int newPriority) {
    if (*head == NULL) { printf("Empty queue.\n"); return; }

    struct Node* curr = *head;
    struct Node* prev = NULL;

    // Step 1: Find the target node
    while (curr != NULL && curr->data != target) {
        prev = curr;
        curr = curr->next;
    }

    if (curr == NULL) { printf("Element %d not found.\n", target); return; }

    // Step 2: Remove node from current position
    if (prev == NULL)
        *head = curr->next;          // removing head
    else
        prev->next = curr->next;     // bypassing curr

    // Step 3: Update the priority
    curr->priority = newPriority;
    curr->next = NULL;

    // Step 4: Re-insert in sorted position
    if (*head == NULL || (*head)->priority > newPriority) {
        curr->next = *head;
        *head = curr;
        return;
    }
    struct Node* it = *head;
    while (it->next != NULL && it->next->priority <= newPriority)
        it = it->next;
    curr->next = it->next;
    it->next = curr;
}

Count / Size Operation

Determine how many elements are currently in the queue

🔢 Dynamic Counting

Traverse from head to tail counting each node. O(n) unless you maintain a separate size counter variable that you increment/decrement on each insert/delete.

💡 Optimization

Maintain a global int size = 0 and update it on every enqueue/dequeue. Then getSize() becomes O(1).

C — Count / Size 
/* Method 1: Traverse and count — O(n) */
int countNodes(struct Node* head) {
    int count = 0;
    struct Node* curr = head;
    while (curr != NULL) {
        count++;
        curr = curr->next;
    }
    return count;
}

/* Method 2: Maintain a size counter — O(1) size checks */
int size = 0; // global counter

void enqueue(struct Node** head, int data, int priority) {
    // ... insert logic ...
    size++; // increment on every insert
}

int dequeueHigh(struct Node** head) {
    // ... delete logic ...
    size--; // decrement on every delete
    return 0;
}

int getSize() { return size; } // O(1)!

Clear / Destroy Priority Queue

Remove all nodes and reset the queue to empty

⚠️ Memory Leaks!

In C, simply setting head = NULL does NOT free the memory — nodes are still allocated on the heap. You must free each node individually.

🔁 Traverse & Free

Walk through every node, save its next pointer, then free() it. After the loop, set head = NULL.

🗑️
Analogy — Clearing a Board Game

Putting away a board game: you pick up each piece one by one and return it to the box. You can't just flip the board (set head = NULL) because pieces (memory) are still on the table (heap). Each must be individually returned (freed).

C — Clear / Destroy 
/* Free ALL nodes and reset the queue. MUST use this — not just head = NULL! */

void clearQueue(struct Node** head) {
    struct Node* curr = *head;
    struct Node* nextNode;

    while (curr != NULL) {
        nextNode = curr->next; // save next BEFORE freeing
        free(curr);            // free current node
        curr = nextNode;       // move to saved next
    }

    *head = NULL; // reset head to NULL
    printf("Queue cleared. All memory freed.\n");
}

/* ❌ WRONG — memory leak! Don't do this:
   head = NULL;   // nodes still exist in memory!
*/

Reverse Priority Queue

Flip the order: ascending ↔ descending priorities

1

Use three pointers: prev, curr, and next

2

At each step: save curr→next, then point curr→next to prev

3

Advance prev = curr, then curr = next

4

When curr == NULL, set head = prev (the new front)

🔄
Analogy — Reversing a Line of People

Ask everyone to turn around and face the other direction. Now the last person in line is first. In code, we reverse each pointer one by one — each person faces backward (to who was behind them, now in front).

Interactive — Reverse
[ Load sample data first ]
—
C — Reverse 
/* Reverse the linked list in-place using 3 pointers. */

void reverseQueue(struct Node** head) {
    struct Node* prev = NULL;
    struct Node* curr = *head;
    struct Node* next = NULL;

    while (curr != NULL) {
        next       = curr->next;  // Step 1: save next
        curr->next = prev;        // Step 2: reverse the link
        prev       = curr;        // Step 3: advance prev
        curr       = next;        // Step 4: advance curr
    }

    *head = prev; // prev is the new head
    printf("Queue reversed: now lowest priority is first.\n");
}

/* After reversing ascending list → descending list
   [1]→[3]→[5]→NULL   becomes   [5]→[3]→[1]→NULL
*/

Merge / Concatenate Priority Queues

Combine two sorted queues into one sorted queue

🔀 Merge Sort Style

Compare the heads of both lists. Take the smaller-priority node and advance that list's pointer. Repeat until one list is exhausted, then append the rest.

⏱️ O(m + n)

Where m and n are the sizes of the two queues. We visit each node at most once — very efficient!

🤝
Analogy — Merging Two Hospital Departments

Two hospital departments each have their own sorted patient queues. When they merge, you interleave both lists: take the most urgent patient from either queue, then the next, keeping the combined queue sorted at all times.

C — Merge Two Sorted Priority Queues 
/* Merge two sorted linked lists into one sorted list.
   Similar to the merge step in merge sort! */

struct Node* mergePQ(struct Node* h1, struct Node* h2) {
    // Base cases
    if (h1 == NULL) return h2;
    if (h2 == NULL) return h1;

    // Use a dummy head to simplify logic
    struct Node dummy;
    struct Node* tail = &dummy;
    dummy.next = NULL;

    while (h1 != NULL && h2 != NULL) {
        if (h1->priority <= h2->priority) {
            tail->next = h1;  // take from list 1
            h1 = h1->next;
        } else {
            tail->next = h2;  // take from list 2
            h2 = h2->next;
        }
        tail = tail->next;
    }

    // Attach remaining nodes
    tail->next = (h1 != NULL) ? h1 : h2;

    return dummy.next; // head of merged list
}

// Usage:
// struct Node* merged = mergePQ(head1, head2);

Copy Priority Queue

Create a deep copy — completely independent duplicate

🧬 Deep Copy vs Shallow Copy

A shallow copy just copies the head pointer — both lists share the same nodes. A deep copy allocates new nodes so the two queues are fully independent.

⚠️ Always Deep Copy

For data structures in C, always deep copy. Shallow copies mean modifying one queue accidentally modifies the other — a nasty bug!

📄
Analogy — Photocopying a Document

Copying a sorted patient queue is like photocopying a document. The copy looks identical, but changes to the original don't affect the copy, and vice versa. Each node is "photocopied" into new memory.

C — Deep Copy Priority Queue 
/* Deep copy: creates completely new nodes. O(n) time and space. */

struct Node* copyQueue(struct Node* head) {
    if (head == NULL) return NULL;

    // Create a dummy node to simplify building the copy
    struct Node dummy;
    struct Node* copyTail = &dummy;
    dummy.next = NULL;

    struct Node* curr = head;

    while (curr != NULL) {
        // Allocate a new node
        struct Node* newNode = (struct Node*) malloc(sizeof(struct Node));
        newNode->data     = curr->data;
        newNode->priority = curr->priority;
        newNode->next     = NULL;

        // Append to copy list
        copyTail->next = newNode;
        copyTail = newNode;

        curr = curr->next;
    }

    return dummy.next; // head of the copy
}

// Usage:
// struct Node* backup = copyQueue(head);
// Now backup is a fully independent copy!

Find Maximum Priority Node

Return the node with the highest priority (most urgent)

⚡ O(1) — Already at HEAD

Since our list is sorted in ascending order (lower number = higher priority), the maximum priority node is always HEAD. No traversal needed!

📝 Convention Matters

"Maximum priority" can mean different things. In our convention, priority 1 is the highest urgency. Always document your sorting convention clearly.

🥇
Analogy — Gold Medal Winner

In a race sorted by finishing time (lowest = fastest), the winner is always first in the sorted list. Finding the maximum priority node is like asking "who won?" — they're always at the front of the podium.

C — Find Maximum Priority Node 
/* In ascending-sorted list (priority 1 = highest urgency),
   the MAXIMUM priority node is always at the HEAD. O(1)! */

struct Node* findMax(struct Node* head) {
    if (head == NULL) {
        printf("Queue is empty.\n");
        return NULL;
    }
    // Highest priority (lowest number) is always HEAD!
    printf("Max priority node: data=%d, priority=%d\n",
           head->data, head->priority);
    return head;
}

/* If your queue uses descending order (high number = high priority),
   the maximum is at the HEAD still — just adjust your sort direction
   in enqueue(). The findMax() function itself stays the same! */

// Usage:
// struct Node* best = findMax(head);
// printf("Next to serve: %d\n", best->data);

Find Minimum Priority Node

Return the node with the lowest priority (least urgent)

🔚 Always at TAIL

The minimum priority node is at the tail of our ascending-sorted list. Requires traversal — O(n) — unless a tail pointer is maintained.

💡 Tail Pointer Pattern

Keeping a tail pointer makes this O(1). Update tail on every insert and delete to always know where the last node is.

🥉
Analogy — Last in Line

In a sorted priority queue, the minimum priority element is like the person at the very back of the line — last to be served. To find them, you walk to the end of the queue. With a tail pointer, you already know exactly where they stand.

C — Find Minimum Priority Node 
/* Find the node with the LOWEST priority (least urgent).
   It's always at the TAIL of the ascending-sorted list. */

struct Node* findMin(struct Node* head) {
    if (head == NULL) {
        printf("Queue is empty.\n");
        return NULL;
    }

    // Traverse to the last node — O(n)
    struct Node* curr = head;
    while (curr->next != NULL)
        curr = curr->next;

    printf("Min priority node: data=%d, priority=%d\n",
           curr->data, curr->priority);
    return curr; // tail = lowest priority
}

/* --- O(1) version with tail pointer --- */
struct Node* tail = NULL; // maintain alongside head

struct Node* findMinFast() {
    return tail; // O(1)! Always the minimum priority node
}
