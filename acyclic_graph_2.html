Foundations

What is an Acyclic Graph?

An acyclic graph is a graph with no cycles — you can never start at a node and follow edges back to where you began. The most important type is a Directed Acyclic Graph (DAG).

🌐
Core Concepts

Key Definitions

  • Graph: A set of vertices (nodes) connected by edges.
  • Directed Graph (Digraph): Edges have a direction — from one node to another.
  • Acyclic: No path leads back to the starting node — no cycles.
  • DAG (Directed Acyclic Graph): A directed graph with no cycles. The most commonly used acyclic structure.
  • Tree: An undirected acyclic connected graph (special case).
  • Forest: Multiple disconnected trees.

Properties of a DAG

  • Every DAG has at least one source (in-degree = 0) and one sink (out-degree = 0).
  • A DAG always has a valid topological ordering.
  • A DAG with V vertices has at most V*(V-1)/2 edges.
  • If we detect a back-edge during DFS, the graph has a cycle — not acyclic.

🗺️ Real-Life Analogy: Course Prerequisites

Think of a university course catalog. Math 101 must be taken before Physics 201, which must come before Engineering 301. You can never take Engineering 301 first and then somehow end up needing to take itself — there's no cycle. This is exactly how a DAG works!

📦 More Real-World Uses

Build systems (Makefile, npm), spreadsheet calculations, compiler instruction scheduling, version control (Git commits), event-driven workflows, neural network computation graphs.

Representation

Adjacency List

An adjacency list stores each vertex's neighbors in a linked list. It is the most memory-efficient representation for sparse graphs.

📋
Adjacency List Representation

How It Works

An array of size V (vertices) stores pointers. Each pointer points to a linked list of all neighbors of that vertex.

  • Space: O(V + E) — much better than O(V²) for sparse graphs
  • Add edge: O(1)
  • Check edge: O(degree of vertex)
  • Traverse neighbors: O(degree of vertex)
Space
O(V+E)
Add Edge
O(1)
Find Edge
O(deg)
Traverse
O(V+E)
C
#include <stdio.h>
#include <stdlib.h>

#define MAX_VERTICES 100

/* Node in the adjacency linked list */
struct Node {
    int dest;               /* destination vertex */
    struct Node* next;     /* pointer to next neighbor */
};

/* Graph structure */
struct Graph {
    int V;                          /* number of vertices */
    struct Node* adj[MAX_VERTICES]; /* adjacency list array */
};

/* Create a new graph with V vertices */
struct Graph* createGraph(int V) {
    struct Graph* g = (struct Graph*)malloc(sizeof(struct Graph));
    g->V = V;
    for (int i = 0; i < V; i++)
        g->adj[i] = NULL;  /* start with empty lists */
    return g;
}

📒 Analogy: Phone Contact Book

Imagine a contact book where each person has a list of people they can call directly. Vertex = person. Adjacency list = that person's direct contacts. Instead of a huge grid showing "can A call B?" for every pair, we just list each person's direct contacts — much more practical!

Vertex Operations

Vertex Operations

Operations to manage vertices: insert, delete, search, update, count, and display.

🔵
All Vertex Operations

Insert Vertex

Add a new vertex to the graph. In adjacency list, simply increment V and set adj[V-1] = NULL. Time: O(1).

Delete Vertex

Remove a vertex and all edges touching it. Free the linked list for that vertex, then remove any edges pointing TO it from all other lists. Time: O(V + E).

Search Vertex

Check if vertex with ID v exists. Simply verify v < graph->V. Time: O(1).

Update, Count, Display

  • Update: Vertex labels/data can be stored in a separate array, updated in O(1).
  • Count: Return graph->V — O(1).
  • Display: Print each vertex and its neighbor list — O(V + E).
C
/* Insert a new vertex */
void insertVertex(struct Graph* g) {
    if (g->V >= MAX_VERTICES) {
        printf("Graph is full!\n");
        return;
    }
    g->adj[g->V] = NULL; /* new vertex has no neighbors */
    g->V++;
    printf("Vertex %d inserted.\n", g->V - 1);
}

/* Delete a vertex and all its edges */
void deleteVertex(struct Graph* g, int v) {
    if (v < 0 || v >= g->V) { printf("Invalid vertex!\n"); return; }
    /* Free the vertex's own neighbor list */
    struct Node* cur = g->adj[v];
    while (cur) {
        struct Node* tmp = cur->next;
        free(cur); cur = tmp;
    }
    g->adj[v] = NULL;
    /* Remove edges pointing TO v from other lists */
    for (int i = 0; i < g->V; i++) {
        if (i == v) continue;
        struct Node** pp = &g->adj[i];
        while (*pp) {
            if ((*pp)->dest == v) {
                struct Node* tmp = *pp;
                *pp = (*pp)->next;
                free(tmp);
            } else pp = &(*pp)->next;
        }
    }
}

/* Search for a vertex */
int searchVertex(struct Graph* g, int v) {
    return (v >= 0 && v < g->V);
}

/* Count vertices */
int countVertices(struct Graph* g) {
    return g->V;
}

/* Display all vertices and their adjacency lists */
void displayVertices(struct Graph* g) {
    for (int i = 0; i < g->V; i++) {
        printf("Vertex %d: ", i);
        struct Node* cur = g->adj[i];
        while (cur) { printf("%d -> ", cur->dest); cur = cur->next; }
        printf("NULL\n");
    }
}

🏙️ Analogy: City Map

Vertices are like cities on a map. Inserting a vertex = founding a new city. Deleting one = removing a city and closing all roads to it. Searching = checking if a city exists. Counting = counting how many cities exist.

Edge Operations

Edge Operations

Managing edges: insert, delete, search, update, count edges.

➡️
All Edge Operations

Insert Edge (u → v)

Create a new node with dest = v, prepend it to adj[u]. O(1). For DAGs we must verify this does not create a cycle (use DFS reachability check).

Delete Edge (u → v)

Walk the linked list of adj[u], find the node with dest == v, unlink and free it. O(degree of u).

Search / Update / Count Edges

  • Search: Scan adj[u] for a node with dest == v. O(degree).
  • Update: Change edge weight — scan and modify the node's weight field.
  • Count edges: Sum lengths of all adjacency lists — O(V + E).
C
/* Create a new adjacency list node */
struct Node* newNode(int dest) {
    struct Node* n = (struct Node*)malloc(sizeof(struct Node));
    n->dest = dest;
    n->next = NULL;
    return n;
}

/* Insert directed edge u -> v */
void insertEdge(struct Graph* g, int u, int v) {
    struct Node* n = newNode(v);
    n->next = g->adj[u]; /* prepend to list */
    g->adj[u] = n;
}

/* Delete directed edge u -> v */
void deleteEdge(struct Graph* g, int u, int v) {
    struct Node** pp = &g->adj[u];
    while (*pp) {
        if ((*pp)->dest == v) {
            struct Node* tmp = *pp;
            *pp = (*pp)->next;
            free(tmp);
            printf("Edge %d->%d deleted.\n", u, v);
            return;
        }
        pp = &(*pp)->next;
    }
    printf("Edge not found!\n");
}

/* Search for edge u -> v */
int searchEdge(struct Graph* g, int u, int v) {
    struct Node* cur = g->adj[u];
    while (cur) {
        if (cur->dest == v) return 1; /* found */
        cur = cur->next;
    }
    return 0; /* not found */
}

/* Count total edges in graph */
int countEdges(struct Graph* g) {
    int count = 0;
    for (int i = 0; i < g->V; i++) {
        struct Node* cur = g->adj[i];
        while (cur) { count++; cur = cur->next; }
    }
    return count;
}

✈️ Analogy: Flight Routes

Vertices = airports. Edges = direct flight routes. Adding an edge = opening a new direct flight from city A to city B. Removing = canceling that route. Searching = checking if a direct flight exists between two airports.

Traversal

Graph Traversal: BFS & DFS

Two fundamental ways to visit every node in a graph — BFS explores layer by layer, DFS goes as deep as possible first.

🔍
Breadth-First Search (BFS)

BFS Algorithm

  1. Initialize a queue. Mark the start node as visited and enqueue it.
  2. While the queue is not empty: dequeue a node, process it, enqueue all unvisited neighbors.
  3. Repeat until all reachable nodes are visited.
Time
O(V+E)
Space
O(V)
Order
Level-by-level
Data Structure
Queue
C
#define MAX 100

int queue[MAX], front = 0, rear = 0;
int visited[MAX];

void enqueue(int v) { queue[rear++] = v; }
int  dequeue()      { return queue[front++]; }
int  isEmpty()      { return front == rear; }

void BFS(struct Graph* g, int start) {
    /* Mark all as unvisited */
    for (int i = 0; i < g->V; i++) visited[i] = 0;
    front = rear = 0;

    /* Visit start node */
    visited[start] = 1;
    enqueue(start);

    while (!isEmpty()) {
        int v = dequeue();
        printf("%d ", v);  /* process node */

        /* Visit all unvisited neighbors */
        struct Node* cur = g->adj[v];
        while (cur) {
            if (!visited[cur->dest]) {
                visited[cur->dest] = 1;
                enqueue(cur->dest);
            }
            cur = cur->next;
        }
    }
}

🌊 Analogy: Spreading Ripples

Drop a stone in a pond. Ripples spread out in concentric circles — first the nodes closest to you, then the next level, and so on. BFS works exactly like this: visit all nodes at distance 1 first, then distance 2, then 3...

🔎
Depth-First Search (DFS) — Recursive & Stack

DFS Algorithm

  1. Mark start node as visited. Process it.
  2. For each unvisited neighbor, recursively apply DFS.
  3. Backtracking: When no unvisited neighbors remain, return to the previous node.
Time
O(V+E)
Space
O(V)
Order
Deep-first
Data Structure
Stack/Recursion
C
/* Recursive DFS */
int visited[MAX] = {0};

void DFS_recursive(struct Graph* g, int v) {
    visited[v] = 1;
    printf("%d ", v);  /* visit node */

    struct Node* cur = g->adj[v];
    while (cur) {
        if (!visited[cur->dest])
            DFS_recursive(g, cur->dest); /* go deeper */
        cur = cur->next;
    }
    /* Backtrack: function returns here automatically */
}

/* Iterative DFS using a stack */
void DFS_stack(struct Graph* g, int start) {
    int stack[MAX], top = -1;
    int vis[MAX] = {0};

    stack[++top] = start;

    while (top >= 0) {
        int v = stack[top--];  /* pop */
        if (vis[v]) continue;
        vis[v] = 1;
        printf("%d ", v);

        struct Node* cur = g->adj[v];
        while (cur) {
            if (!vis[cur->dest])
                stack[++top] = cur->dest; /* push */
            cur = cur->next;
        }
    }
}

🧩 Analogy: Maze Exploration

Imagine exploring a maze. You always go as far as you can in one direction. When you hit a dead end, you backtrack to the last junction and try another path. DFS does exactly this — go deep, then backtrack.

Path Operations

Path Operations

Finding paths, shortest paths, all paths, and checking reachability between nodes.

🛤️
Find Path, Shortest Path, All Paths, Reachability

Find Path (DFS-based)

Use DFS with a path array. When we reach the destination, print the path array. Time: O(V + E).

Shortest Path in DAG (BFS)

BFS finds shortest path in terms of number of edges. For weighted DAGs, use Dijkstra's or DP on topological order. BFS time: O(V + E).

All Paths

Use DFS with backtracking — when we reach destination, record the path. Continue exploring. Time: O(2^V) worst case (exponential).

Reachability Check

Can we reach vertex v from u? Run DFS/BFS from u. If v gets visited, it is reachable. O(V + E).

Path Length Count

Track the current path length in DFS. When destination reached, record the length. Can also find min/max path lengths.

C
int path[MAX], pathLen = 0;
int vis[MAX];

/* Find ONE path from src to dest */
int findPath(struct Graph* g, int src, int dest) {
    vis[src] = 1;
    path[pathLen++] = src;
    if (src == dest) {
        for (int i = 0; i < pathLen; i++) printf("%d ", path[i]);
        return 1;
    }
    struct Node* cur = g->adj[src];
    while (cur) {
        if (!vis[cur->dest])
            if (findPath(g, cur->dest, dest)) return 1;
        cur = cur->next;
    }
    pathLen--; /* backtrack */
    return 0;
}

/* Print ALL paths from src to dest */
void printAllPaths(struct Graph* g, int src, int dest) {
    vis[src] = 1;
    path[pathLen++] = src;
    if (src == dest) {
        for (int i = 0; i < pathLen; i++) printf("%d ", path[i]);
        printf("\n");
    } else {
        struct Node* cur = g->adj[src];
        while (cur) {
            if (!vis[cur->dest])
                printAllPaths(g, cur->dest, dest);
            cur = cur->next;
        }
    }
    pathLen--;
    vis[src] = 0; /* unmark for other paths */
}

/* Check if dest is reachable from src */
int isReachable(struct Graph* g, int src, int dest) {
    int vis2[MAX] = {0};
    /* BFS from src */
    int q[MAX], f = 0, r = 0;
    vis2[src] = 1; q[r++] = src;
    while (f < r) {
        int v = q[f++];
        if (v == dest) return 1;
        struct Node* c = g->adj[v];
        while (c) { if(!vis2[c->dest]){vis2[c->dest]=1;q[r++]=c->dest;} c=c->next; }
    }
    return 0;
}

🗺️ Analogy: GPS Navigation

Your GPS finding routes is exactly path operations. Find Path = find any route from A to B. Shortest Path = fastest/shortest route. All Paths = all possible routes. Reachability = can I even get there?

Acyclic Operations

Acyclic Operations

The core operations that validate and maintain the acyclic property — the heart of what makes a DAG different from a regular directed graph.

🚫
Cycle Detection & DAG Validation

Cycle Detection (DFS with Color Marking)

Use 3 states for each node:

  • WHITE (0): Not visited yet
  • GRAY (1): Currently in the DFS recursion stack (being explored)
  • BLACK (2): Fully explored, all descendants visited

If we find a GRAY neighbor during DFS → a back-edge exists → cycle detected!

Edge Safety Check

Before adding edge u → v to a DAG: check if v can already reach u. If yes, adding this edge creates a cycle. So refuse it. O(V + E).

Check Tree Structure

A directed tree (rooted): one root (in-degree 0), all others have in-degree 1, no cycles. Check: count of vertices with in-degree 0 == 1, total edges == V-1.

Check Forest Structure

A directed forest: each connected component is a tree. Check: no node has in-degree > 1, no cycles, possibly multiple roots.

DAG Validation

A directed graph is a DAG if and only if it has no cycle. Run topological sort (Kahn's): if all vertices are processed → DAG. If some remain → cycle exists.

C
#define WHITE 0
#define GRAY  1
#define BLACK 2

int color[MAX];

/* DFS helper for cycle detection */
int dfs_cycle(struct Graph* g, int v) {
    color[v] = GRAY;  /* mark: currently exploring */

    struct Node* cur = g->adj[v];
    while (cur) {
        if (color[cur->dest] == GRAY)
            return 1;  /* back-edge found! CYCLE! */
        if (color[cur->dest] == WHITE)
            if (dfs_cycle(g, cur->dest)) return 1;
        cur = cur->next;
    }
    color[v] = BLACK;  /* fully explored */
    return 0;
}

/* Main function: detect cycle in directed graph */
int hasCycle(struct Graph* g) {
    for (int i = 0; i < g->V; i++) color[i] = WHITE;
    for (int i = 0; i < g->V; i++)
        if (color[i] == WHITE)
            if (dfs_cycle(g, i)) return 1; /* cycle found */
    return 0; /* no cycle = it's a DAG! */
}

/* Validate if graph is a DAG */
int isDAG(struct Graph* g) {
    return !hasCycle(g);
}

/* Edge safety check before inserting u->v */
int isSafeEdge(struct Graph* g, int u, int v) {
    /* Adding u->v is safe if v cannot reach u */
    return !isReachable(g, v, u);
}

🔄 Analogy: Debt Cycle

If Alice owes Bob, Bob owes Carol, and Carol owes Alice — that's a cycle! None of them can ever settle the debt because it goes in a circle forever. A DAG ensures this never happens: debts always flow in one direction, with a clear start and end.

Topological Operations

Topological Sort & Scheduling

Topological sort produces a linear ordering of vertices such that for every edge u→v, vertex u comes before v. Only possible in a DAG.

🗂️
Kahn's Algorithm & DFS Topological Sort

Kahn's Algorithm (BFS-based)

  1. Compute in-degree for every vertex.
  2. Enqueue all vertices with in-degree 0 (no prerequisites).
  3. While queue not empty: dequeue vertex v, add to result. For each neighbor w: decrement in-degree. If in-degree becomes 0, enqueue w.
  4. If result contains all vertices → DAG with valid topological order. Else → cycle!

DFS-based Topological Sort

Run DFS. After fully exploring a vertex (all descendants done), push it onto a stack. Print the stack in reverse. This gives topological order. Time: O(V + E).

Applications

  • Task Scheduling: Run tasks in dependency order
  • Build Systems: Compile files in correct order
  • Dependency Resolution: npm/pip package installs
  • Course Scheduling: Take courses in prerequisite order
Topological Order:
C — Kahn's Algorithm
void topologicalSort_Kahn(struct Graph* g) {
    int inDegree[MAX] = {0};

    /* Step 1: Compute in-degree of each vertex */
    for (int i = 0; i < g->V; i++) {
        struct Node* cur = g->adj[i];
        while (cur) {
            inDegree[cur->dest]++;
            cur = cur->next;
        }
    }

    /* Step 2: Enqueue all vertices with in-degree 0 */
    int queue[MAX], front = 0, rear = 0;
    for (int i = 0; i < g->V; i++)
        if (inDegree[i] == 0) queue[rear++] = i;

    int count = 0;
    printf("Topological Order: ");

    /* Step 3: Process queue */
    while (front < rear) {
        int v = queue[front++];
        printf("%d ", v);
        count++;

        struct Node* cur = g->adj[v];
        while (cur) {
            inDegree[cur->dest]--;
            if (inDegree[cur->dest] == 0)
                queue[rear++] = cur->dest;
            cur = cur->next;
        }
    }

    if (count != g->V)
        printf("\nCycle detected! Not a DAG.\n");
}

/* DFS-based topological sort */
int topoStack[MAX], stackTop = -1;

void topo_dfs(struct Graph* g, int v, int vis[]) {
    vis[v] = 1;
    struct Node* cur = g->adj[v];
    while (cur) {
        if (!vis[cur->dest])
            topo_dfs(g, cur->dest, vis);
        cur = cur->next;
    }
    topoStack[++stackTop] = v; /* push after exploring */
}

void topologicalSort_DFS(struct Graph* g) {
    int vis[MAX] = {0};
    stackTop = -1;
    for (int i = 0; i < g->V; i++)
        if (!vis[i]) topo_dfs(g, i, vis);
    printf("Topological Order: ");
    while (stackTop >= 0)
        printf("%d ", topoStack[stackTop--]);
}

🍳 Analogy: Recipe Steps

To make a cake: you must mix batter before pouring, preheat oven before baking, bake before frosting. Topological sort gives you a valid order to follow these steps — respecting all the "must come before" rules.

Connectivity

Connectivity Operations

Check if a graph is connected, find connected components, and identify isolated vertices.

🔗
Connected Components & Isolation Check

Connected Graph Check (Weakly Connected)

A directed graph is weakly connected if the underlying undirected graph is connected. Run DFS/BFS ignoring edge direction — if all nodes are visited, it's weakly connected.

Connected Components

Run DFS. Each time DFS is called on an unvisited node, a new component starts. Count these calls = number of components. O(V + E).

Reachable Vertices from a Source

Run BFS/DFS from source — all visited nodes form the set of reachable vertices. O(V + E).

Isolation Check

A vertex is isolated if it has no edges (in-degree = 0 AND out-degree = 0 in directed graphs). Check: adj[v] == NULL and no other vertex points to v.

C
int compID[MAX]; /* component label for each vertex */

void dfs_component(struct Graph* g, int v, int comp) {
    compID[v] = comp;
    struct Node* cur = g->adj[v];
    while (cur) {
        if (compID[cur->dest] == -1)
            dfs_component(g, cur->dest, comp);
        cur = cur->next;
    }
}

/* Count connected components */
int countComponents(struct Graph* g) {
    for (int i = 0; i < g->V; i++) compID[i] = -1;
    int comp = 0;
    for (int i = 0; i < g->V; i++)
        if (compID[i] == -1)
            dfs_component(g, i, comp++);
    return comp;
}

/* Is the graph (weakly) connected? */
int isConnected(struct Graph* g) {
    return countComponents(g) == 1;
}

/* Check if a vertex is isolated */
int isIsolated(struct Graph* g, int v) {
    if (g->adj[v] != NULL) return 0; /* has outgoing edges */
    for (int i = 0; i < g->V; i++) {
        struct Node* cur = g->adj[i];
        while (cur) {
            if (cur->dest == v) return 0; /* incoming edge */
            cur = cur->next;
        }
    }
    return 1; /* truly isolated */
}

🏝️ Analogy: Islands & Bridges

Think of vertices as islands and edges as bridges. A connected component is a group of islands all connected by bridges. An isolated vertex is an island with no bridges at all. Connectivity check = "Can I reach every island from any other island?"

Degree Operations

Degree Operations

Degree measures how many edges connect to a vertex — a fundamental property for graph analysis.

📊
In-Degree, Out-Degree, Max/Min Degree

Types of Degree

  • Out-degree of v: Number of edges going OUT from v. = Length of adj[v]. O(degree).
  • In-degree of v: Number of edges coming IN to v. Must scan all adjacency lists. O(V + E).
  • Total degree of v: in-degree + out-degree.
  • Maximum degree: The highest degree among all vertices.
  • Minimum degree: The lowest degree among all vertices.

DAG Property

In any DAG: a source has in-degree 0, a sink has out-degree 0. Every DAG has at least one source and one sink. The sum of all out-degrees = total number of edges.

C
/* Out-degree: count edges going out from v */
int outDegree(struct Graph* g, int v) {
    int count = 0;
    struct Node* cur = g->adj[v];
    while (cur) { count++; cur = cur->next; }
    return count;
}

/* In-degree: count edges coming into v */
int inDegree(struct Graph* g, int v) {
    int count = 0;
    for (int i = 0; i < g->V; i++) {
        struct Node* cur = g->adj[i];
        while (cur) {
            if (cur->dest == v) count++;
            cur = cur->next;
        }
    }
    return count;
}

/* Find vertex with maximum out-degree */
int maxDegreeVertex(struct Graph* g) {
    int maxDeg = -1, maxV = 0;
    for (int i = 0; i < g->V; i++) {
        int d = outDegree(g, i);
        if (d > maxDeg) { maxDeg = d; maxV = i; }
    }
    printf("Max degree vertex: %d (out-degree: %d)\n", maxV, maxDeg);
    return maxV;
}

/* Print degree table for all vertices */
void printDegreeTable(struct Graph* g) {
    printf("%-8s %-12s %-12s\n", "Vertex", "In-Degree", "Out-Degree");
    for (int i = 0; i < g->V; i++)
        printf("%-8d %-12d %-12d\n", i, inDegree(g,i), outDegree(g,i));
}

🤝 Analogy: Social Media

Out-degree = number of people you follow. In-degree = number of people who follow you (your "follower count"). A celebrity has very high in-degree. An influencer has high out-degree too. An isolated account has both = 0.

Structural Operations

Structural Operations

Analyzing the overall structure — graph size, density, and validating tree/forest properties.

🏗️
Size, Order, Density, Tree & Forest Validation

Graph Size vs Order

  • Order: Number of vertices (V). graph->V
  • Size: Number of edges (E). Sum of all adjacency list lengths.

Sparse vs Dense

  • Sparse graph: E ≈ O(V) — few edges relative to vertices. Adjacency list is ideal.
  • Dense graph: E ≈ O(V²) — many edges. Adjacency matrix may be better.
  • Density ratio: E / (V*(V-1)) for directed graphs. Close to 1 = dense, close to 0 = sparse.

Tree Validation (Directed)

A directed rooted tree has:

  • Exactly one root (in-degree = 0)
  • All other vertices have in-degree = 1
  • No cycles (acyclic)
  • V - 1 edges total

Forest Validation

A directed forest is a collection of trees. Check: no cycles, each vertex has in-degree ≤ 1, total edges = V - (number of trees).

C
/* Graph order (number of vertices) */
int graphOrder(struct Graph* g) { return g->V; }

/* Graph size (number of edges) */
int graphSize(struct Graph* g) {
    int e = 0;
    for (int i = 0; i < g->V; i++) {
        struct Node* c = g->adj[i];
        while (c) { e++; c = c->next; }
    }
    return e;
}

/* Is the graph sparse? (density < 0.5) */
int isSparse(struct Graph* g) {
    int V = g->V, E = graphSize(g);
    int maxE = V * (V - 1);
    if (maxE == 0) return 1;
    return (E * 2 < maxE); /* density < 0.5 */
}

/* Validate directed tree */
int isTree(struct Graph* g) {
    if (hasCycle(g)) return 0;
    if (graphSize(g) != g->V - 1) return 0;
    int roots = 0;
    for (int i = 0; i < g->V; i++)
        if (inDegree(g, i) == 0) roots++;
    return roots == 1; /* exactly one root */
}

/* Validate directed forest */
int isForest(struct Graph* g) {
    if (hasCycle(g)) return 0;
    for (int i = 0; i < g->V; i++)
        if (inDegree(g, i) > 1) return 0;
    return 1;
}

🌲 Analogy: Family Tree vs Forest

A single family tree (one ancestor at the top, everyone below) = directed tree. Multiple separate family trees = directed forest. A sparse graph is like a lightly-connected neighborhood. A dense graph is like a party where everyone knows everyone.

🚀
Longest Path in a DAG

Algorithm (DP + Topological Sort)

  1. Run topological sort to get order.
  2. Initialize dist[src] = 0, all others = -INFINITY.
  3. Process vertices in topological order. For each vertex v and each neighbor u: if dist[v] + weight(v,u) > dist[u], update dist[u].
  4. Answer = max value in dist array.

Time: O(V + E) — much better than general graphs (where longest path is NP-hard)!

C — Longest Path in DAG
#define NEG_INF -999999

/* Node with weight */
struct WNode { int dest, weight; struct WNode* next; };

void topoSort(struct Graph* g, int v, int vis[], int stk[], int* top) {
    vis[v] = 1;
    struct Node* c = g->adj[v];
    while (c) { if(!vis[c->dest]) topoSort(g,c->dest,vis,stk,top); c=c->next; }
    stk[++(*top)] = v;
}

void longestPath(struct Graph* g, int src) {
    int stk[MAX], top = -1, vis[MAX] = {0}, dist[MAX];
    for (int i = 0; i < g->V; i++) dist[i] = NEG_INF;
    dist[src] = 0;

    /* Get topological order */
    for (int i = 0; i < g->V; i++)
        if (!vis[i]) topoSort(g, i, vis, stk, &top);

    /* Relax edges in topological order */
    while (top >= 0) {
        int v = stk[top--];
        if (dist[v] == NEG_INF) continue;
        struct Node* cur = g->adj[v];
        while (cur) {
            if (dist[v] + 1 > dist[cur->dest])
                dist[cur->dest] = dist[v] + 1;
            cur = cur->next;
        }
    }

    int longest = 0;
    for (int i = 0; i < g->V; i++)
        if (dist[i] != NEG_INF && dist[i] > longest) longest = dist[i];
    printf("Longest path length: %d\n", longest);
}

🎯 Analogy: Critical Path in a Project

In project management, the critical path is the longest chain of dependent tasks. It determines the minimum total project duration. Delay any task on the critical path — and the whole project gets delayed. That's exactly what "longest path in a DAG" computes!

⚡
Dynamic Programming on DAG

Why DP Works on DAGs

DAGs have a topological order. This means we can compute answers for sub-problems in a natural order — earlier vertices first. No circular dependencies, so memoization is perfect. Common DP problems on DAGs:

  • Number of paths from source to any vertex
  • Shortest/longest path with weights
  • Max profit / min cost paths
  • Counting paths to a target

Template: Count Paths to Target

Process vertices in topological order. For each vertex, dp[v] = sum of dp[u] for all predecessors u. Initialize dp[src] = 1. O(V + E).

C — Count Paths in DAG
/* Count number of paths from src to every vertex */
void countPaths(struct Graph* g, int src) {
    int topoOrder[MAX], idx = 0;
    /* Step 1: Get topological order using Kahn's */
    int inDeg[MAX] = {0};
    for (int i = 0; i < g->V; i++) {
        struct Node* c = g->adj[i];
        while (c) { inDeg[c->dest]++; c = c->next; }
    }
    int q[MAX], f = 0, r = 0;
    for (int i = 0; i < g->V; i++)
        if (!inDeg[i]) q[r++] = i;
    while (f < r) {
        int v = q[f++];
        topoOrder[idx++] = v;
        struct Node* c = g->adj[v];
        while (c) { if(!--inDeg[c->dest]) q[r++]=c->dest; c=c->next; }
    }

    /* Step 2: DP in topological order */
    long long dp[MAX] = {0};
    dp[src] = 1; /* 1 path from source to itself */

    for (int i = 0; i < idx; i++) {
        int v = topoOrder[i];
        struct Node* c = g->adj[v];
        while (c) {
            dp[c->dest] += dp[v]; /* accumulate path counts */
            c = c->next;
        }
    }

    printf("Paths from vertex %d:\n", src);
    for (int i = 0; i < g->V; i++)
        printf("  to %d: %lld paths\n", i, dp[i]);
}

/* DP: Maximum weighted path from src */
void maxWeightedPath(struct Graph* g, int src, int weight[]) {
    /* Similar structure using max instead of sum */
    int dp[MAX];
    for (int i = 0; i < g->V; i++) dp[i] = NEG_INF;
    dp[src] = weight[src];
    /* Process in topological order, update: dp[v] + weight[u] */
}

🧮 Analogy: Spreadsheet Calculation

A spreadsheet is a DAG! Cell C1 = A1 + B1. Cell D1 = C1 * 2. The spreadsheet computes them in order (A1 and B1 first, then C1, then D1). That's DP on a DAG — compute in topological order, reuse results.

Dependency Graph Processing

What is a Dependency Graph?

A DAG where edge u → v means "u depends on v" (v must be done before u). Used in build systems, package managers, task schedulers.

  • Find dependencies of X: BFS/DFS from X following edges.
  • Build order: Reverse topological sort (dependencies processed first).
  • Circular dependency detection: Cycle detection on the DAG.
  • Impact analysis: Reverse graph — find all things that depend on X.
C — Dependency Resolution
/*
 * Dependency Graph: edge u->v means "u requires v"
 * Build order = topological sort (dependencies first)
 */
void resolveDependencies(struct Graph* g) {
    /* Check for circular dependencies first */
    if (hasCycle(g)) {
        printf("ERROR: Circular dependency detected!\n");
        return;
    }

    /* Topological sort gives correct build order */
    printf("Build order (dependencies first):\n");
    topologicalSort_Kahn(g);
}

/* Find all direct and indirect dependencies of vertex v */
void findAllDependencies(struct Graph* g, int v) {
    int vis[MAX] = {0};
    /* DFS from v to find all reachable vertices */
    printf("Dependencies of %d: ", v);
    vis[v] = 1;
    struct Node* c = g->adj[v];
    while (c) {
        if (!vis[c->dest]) {
            printf("%d ", c->dest);
            DFS_recursive(g, c->dest); /* recursive DFS */
        }
        c = c->next;

/* Impact analysis: what depends on vertex v? */
/* Build reverse graph, then DFS from v */
struct Graph* reverseGraph(struct Graph* g) {
    struct Graph* rev = createGraph(g->V);
    for (int i = 0; i < g->V; i++) {
        struct Node* c = g->adj[i];
        while (c) {
            insertEdge(rev, c->dest, i); /* reverse the edge */
            c = c->next;
        }
    }
    return rev;
}

📦 Analogy: npm Package Manager

When you run npm install react, npm builds a dependency graph. React depends on other libraries, which depend on others. npm resolves them all using topological sort — installing dependencies before the packages that need them. A "circular dependency" error = a cycle in this DAG!

Utility Operations

Utility Operations

Essential helper operations: isEmpty, clear, copy, display, and reset visited array.

🛠️
Is Empty, Clear, Copy, Display, Reset

Is Empty

Check if the graph has no vertices: g->V == 0. O(1).

Clear Graph

Free all adjacency lists and set V = 0. Releases all memory. O(V + E).

Copy Graph

Create a new graph, then for each vertex, deep-copy its adjacency list. O(V + E).

Display Graph

Print each vertex followed by its adjacency list: 0 → 1 → 2 → NULL. O(V + E).

Reset Visited Array

Set all entries of the visited[] array to 0. Required before each new traversal. O(V). Often implemented as memset(visited, 0, sizeof(visited)).

C
#include <string.h>

/* Check if graph is empty */
int isEmpty(struct Graph* g) {
    return g->V == 0;
}

/* Clear all edges and vertices from graph */
void clearGraph(struct Graph* g) {
    for (int i = 0; i < g->V; i++) {
        struct Node* cur = g->adj[i];
        while (cur) {
            struct Node* tmp = cur->next;
            free(cur);
            cur = tmp;
        }
        g->adj[i] = NULL;
    }
    g->V = 0;
    printf("Graph cleared.\n");
}

/* Deep copy: create an identical graph */
struct Graph* copyGraph(struct Graph* src) {
    struct Graph* dest = createGraph(src->V);
    for (int i = 0; i < src->V; i++) {
        /* Reverse list to maintain order */
        int tmp[MAX]; int len = 0;
        struct Node* c = src->adj[i];
        while (c) { tmp[len++] = c->dest; c = c->next; }
        for (int j = len - 1; j >= 0; j--)
            insertEdge(dest, i, tmp[j]);
    }
    return dest;
}

/* Display the full graph */
void displayGraph(struct Graph* g) {
    printf("Graph (V=%d):\n", g->V);
    for (int i = 0; i < g->V; i++) {
        printf("  [%d] -> ", i);
        struct Node* c = g->adj[i];
        while (c) { printf("%d -> ", c->dest); c = c->next; }
        printf("NULL\n");
    }
}

/* Reset visited array before each traversal */
void resetVisited(int visited[], int n) {
    memset(visited, 0, n * sizeof(int));
    /* Or manually: for(int i=0;i<n;i++) visited[i]=0; */
}
