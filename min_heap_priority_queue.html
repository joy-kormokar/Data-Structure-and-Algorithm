Introduction & Core Concepts
// complete binary tree · array representation · heap ordering property
What is a Min Heap?

A Min Heap is a special tree-based data structure where every parent node holds a value smaller than or equal to its children. The smallest value always sits at the root (top).

Think of it as a "priority-sorted tree" that always lets you access the most important (minimum) element instantly.

A Priority Queue is an abstract data type. A Min Heap is the most efficient way to implement it.

3 Core Rules (Must Know!)

1. Complete Binary Tree: Every level is fully filled except possibly the last, which fills from left to right.

2. Heap Ordering Property: parent ≤ both children (always, at every level).

3. Array Representation: A heap is stored in a simple array — no pointers needed! For node at index i:

  • Parent: (i-1)/2
  • Left child: 2*i+1
  • Right child: 2*i+2
Complete Binary Tree Array Stored (No Pointers!) Root = Minimum Always O(log n) Insert & Delete O(1) Peek Minimum
🌳 Heap Tree + Array Representation
The tree above and the array below represent the same data. Every element's position in the array maps to a node in the tree.
C Core Heap Structure Definition 
#include <stdio.h>
#include <stdlib.h>
#include <limits.h>   /* For INT_MIN */

#define MAX_SIZE 100  /* Maximum number of elements */

/* ─── The Min Heap Structure ─────────────────────────────────
   We use an ARRAY (not linked list) to store the heap.
   This is efficient because:
   - No extra memory for pointers
   - Cache-friendly (sequential memory)
   - Parent/child indices are simple math
─────────────────────────────────────────────────────────── */
typedef struct {
    int data[MAX_SIZE];  /* The actual array storing heap elements */
    int size;            /* Current number of elements */
    int capacity;        /* Maximum allowed elements */
} MinHeap;

/* Index Math (memorize these!):
   Parent of node i     → (i - 1) / 2
   Left child of node i → 2 * i + 1
   Right child of node i→ 2 * i + 2

   Example: for index 3
   Parent = (3-1)/2 = 1
   Left   = 2*3+1  = 7
   Right  = 2*3+2  = 8
*/
🏥

Real-Life Analogy: Hospital Emergency Room

Imagine an ER waiting room. Patients are not served in order of arrival — the most critical patient (lowest priority number = most urgent) is always treated first. When a new patient arrives, they are placed based on their severity. This is exactly how a Min Heap Priority Queue works: the element with the smallest value always gets served first, regardless of when it was added.

01
Create / Initialize Min Heap
// define structure · allocate memory · set capacity · size = 0
Theory

Before using a heap, we must initialize it. This involves setting up the container (array), defining the maximum capacity, and starting with size = 0 (empty heap).

In C, we define a struct to bundle the array, size, and capacity together. This struct IS the heap. We then write a function to initialize all fields to their starting values.

Key point: An initialized heap with size=0 is a valid, ready-to-use empty heap. No memory is wasted — elements are only stored when inserted.

⚙️ Initialization State
An initialized heap: size=0, capacity=10, data array is empty (waiting for elements)
C Create / Initialize Min Heap 
/* Method 1: Using a pointer (dynamic allocation) */
MinHeap* createMinHeap() {
    MinHeap* heap = (MinHeap*) malloc(sizeof(MinHeap));
    
    if (heap == NULL) {           /* Always check malloc succeeded */
        printf("Memory allocation failed!\n");
        return NULL;
    }
    
    heap->size     = 0;         /* Heap starts empty */
    heap->capacity = MAX_SIZE;    /* Can hold MAX_SIZE elements */
    
    return heap;
}

/* Method 2: Using a local variable (stack allocation) */
void initMinHeap(MinHeap* heap) {
    heap->size     = 0;
    heap->capacity = MAX_SIZE;
    /* data[] array exists but we ignore it until size > 0 */
}

/* Usage in main() */
int main() {
    /* Option A: Dynamic */
    MinHeap* h1 = createMinHeap();
    
    /* Option B: Stack */
    MinHeap h2;
    initMinHeap(&h2);
    
    printf("Heap created! Size=%d, Capacity=%d\n",
           h1->size, h1->capacity);  /* Output: Size=0, Capacity=100 */
    
    free(h1);  /* Always free dynamic memory! */
    return 0;
}

02
Insert Element
// place at end · increase size · heapify-up (bubble-up)
Theory

To insert a new element: Step 1 — Check if the heap is full. Step 2 — Place the new element at the last position (end of array). Step 3 — Increase size by 1. Step 4 — Perform Heapify-Up (also called bubble-up or sift-up) to restore the min-heap property.

Heapify-Up compares the new element with its parent. If it's smaller, they swap. This repeats upward until either the element reaches the root or it's larger than its parent. This takes at most O(log n) swaps since tree height = log n.

Time: O(log n) Space: O(1) New element goes to END first Then bubbles UP if needed

C Insert Element into Min Heap 
void insert(MinHeap* heap, int value) {
    /* Step 1: Check for overflow */
    if (heap->size >= heap->capacity) {
        printf("Heap is full! Cannot insert %d\n", value);
        return;
    }
    
    /* Step 2 & 3: Place at last position, increase size */
    heap->data[heap->size] = value;
    heap->size++;
    
    /* Step 4: Restore min-heap property by bubbling up */
    heapifyUp(heap, heap->size - 1);
    
    printf("Inserted %d successfully. Heap size: %d\n",
           value, heap->size);
}

/* Heapify Up — moves new element to correct position */
void heapifyUp(MinHeap* heap, int index) {
    while (index > 0) {
        int parent = (index - 1) / 2;  /* Find parent index */
        
        if (heap->data[index] < heap->data[parent]) {
            /* Swap current with parent */
            int temp         = heap->data[index];
            heap->data[index]  = heap->data[parent];
            heap->data[parent] = temp;
            
            index = parent;  /* Move up to parent's position */
        } else {
            break;  /* Current is >= parent, heap property satisfied! */
        }
    }
}

/* Example usage:
   insert(heap, 10);  → Heap: [10]
   insert(heap, 20);  → Heap: [10, 20]
   insert(heap, 5);   → Place 5 at end: [10, 20, 5]
                         5 < parent(10)? Yes → swap: [5, 20, 10]
                         5 at root, done!
*/
📋

Analogy: New Employee Joining a Company Hierarchy

A new employee always starts at the lowest level (end of heap). But if they are more qualified (smaller priority number) than their manager, they get promoted (swap). This keeps happening until they find a manager more qualified than them — just like heapify-up!

03
Extract Minimum (Delete Highest Priority)
// remove root · move last to root · decrease size · heapify-down
Theory

Extract Minimum removes and returns the root element (the minimum). Step 1 — Check for underflow (empty heap). Step 2 — Save the root value. Step 3 — Move the last element to the root. Step 4 — Decrease size by 1. Step 5 — Perform Heapify-Down (sift-down) from the root to restore order.

We never "shift" the entire array — that would be O(n). Instead, we use the last element as a temporary root and then sink it to its correct place in O(log n) time.

Time: O(log n) Returns minimum value Last element fills root gap Then sinks DOWN to correct spot

C Extract Minimum from Min Heap 
int extractMin(MinHeap* heap) {
    /* Step 1: Check for underflow */
    if (heap->size == 0) {
        printf("Heap is empty! Nothing to extract.\n");
        return -1;
    }
    
    /* Step 2: Save the minimum (root) */
    int minimum = heap->data[0];
    
    /* Step 3 & 4: Move last element to root, shrink size */
    heap->data[0] = heap->data[heap->size - 1];
    heap->size--;
    
    /* Step 5: Restore min-heap property by sinking down */
    if (heap->size > 0) {
        heapifyDown(heap, 0);  /* Start from root (index 0) */
    }
    
    printf("Extracted minimum: %d\n", minimum);
    return minimum;
}

/* Heapify Down — sinks an element to its correct position */
void heapifyDown(MinHeap* heap, int index) {
    while (1) {
        int left    = 2 * index + 1;  /* Left child index  */
        int right   = 2 * index + 2;  /* Right child index */
        int smallest = index;           /* Assume current is smallest */
        
        /* Check if left child is smaller */
        if (left < heap->size &&
            heap->data[left] < heap->data[smallest]) {
            smallest = left;
        }
        
        /* Check if right child is smaller */
        if (right < heap->size &&
            heap->data[right] < heap->data[smallest]) {
            smallest = right;
        }
        
        /* If smallest is not the current node, swap and continue */
        if (smallest != index) {
            int temp           = heap->data[index];
            heap->data[index]   = heap->data[smallest];
            heap->data[smallest] = temp;
            index = smallest;
        } else {
            break;  /* Current is smallest among parent+children, done! */
        }
    }
}

When the CEO (root = minimum) leaves, you can't just leave the top empty. You temporarily promote the most junior employee (last element) to CEO. Then they "sink down" by competing with their direct reports — always getting replaced by the better-qualified child — until they reach the level where they truly belong.

04
Peek Minimum / Get Highest Priority
// check empty · access root · return minimum value
Theory

Peek is the simplest operation: just look at heap->data[0] (the root). Don't remove it — just read it. This is always O(1) because the minimum is always at index 0 by the heap property.

Unlike Extract Min (which removes the element), Peek is purely a read operation. The heap remains unchanged. This is useful when you want to check the priority without committing to removal.

Time: O(1) — Fastest operation! Heap unchanged Root = index 0 = minimum
👁️ Peek — Root Always Contains Minimum
The root (index 0) is always the minimum — highlighted in green. Peek simply reads this value.
C Peek Minimum 
/* Peek: Look at minimum WITHOUT removing it */
int peekMin(MinHeap* heap) {
    /* Step 1: Check if heap is empty */
    if (heap->size == 0) {
        printf("Heap is empty! No minimum to peek.\n");
        return -1;
    }
    
    /* Step 2: Return root element (always the minimum!) */
    return heap->data[0];  /* O(1) — instant! */
}

/* Usage */
int main() {
    MinHeap* heap = createMinHeap();
    insert(heap, 15);
    insert(heap, 3);
    insert(heap, 8);
    
    int min = peekMin(heap);
    printf("Minimum is: %d\n", min);  /* Output: 3 */
    printf("Heap size still: %d\n", heap->size); /* Still 3! */
    
    free(heap);
    return 0;
}
👀

Analogy: Checking the Top Card of a Sorted Deck

Peek is like looking at the top card of a sorted deck without picking it up. You see what the next card is (the minimum), but you don't remove it from the deck. The deck (heap) stays exactly the same after peeking.

05
Heapify Up (Bubble Up / Sift Up)
// calculate parent · compare · swap if smaller · continue upward
Theory

Heapify-Up is a helper algorithm that moves a node upward in the tree until it reaches its correct position. It's used after inserting a new element (which is placed at the end/bottom).

Algorithm: While the current node is not the root AND its value is less than its parent's value → swap with parent → move up. Stop when parent ≤ current, or when we reach the root.

The height of a complete binary tree with n nodes is ⌊log₂ n⌋, so at most log₂ n swaps occur. This gives us O(log n) time complexity.

C Heapify Up — Detailed with Comments 
/* Helper function to swap two elements */
void swap(int* a, int* b) {
    int temp = *a;
    *a = *b;
    *b = temp;
}

/* Heapify Up: restores heap property after insert */
void heapifyUp(MinHeap* heap, int index) {
    /* Base case: if we're at the root (index 0), stop */
    if (index == 0) return;
    
    while (index > 0) {
        /* Formula: parent of node at index i is at (i-1)/2 */
        int parentIdx = (index - 1) / 2;
        
        printf("Comparing data[%d]=%d with parent data[%d]=%d\n",
               index, heap->data[index],
               parentIdx, heap->data[parentIdx]);
        
        if (heap->data[index] < heap->data[parentIdx]) {
            /* Current is smaller than parent — SWAP and move up */
            swap(&heap->data[index], &heap->data[parentIdx]);
            printf("Swapped! Moving up to index %d\n", parentIdx);
            index = parentIdx;  /* Continue from parent's position */
        } else {
            /* Current >= parent — heap property satisfied! */
            printf("No swap needed. Heap property restored!\n");
            break;
        }
    }
}

/* Recursive version (equivalent, easier to understand) */
void heapifyUpRecursive(MinHeap* heap, int index) {
    if (index == 0) return;  /* Reached root, done */
    
    int parent = (index - 1) / 2;
    
    if (heap->data[index] < heap->data[parent]) {
        swap(&heap->data[index], &heap->data[parent]);
        heapifyUpRecursive(heap, parent);  /* Recurse upward */
    }
}
🎈

Analogy: Bubble Rising in Water

Think of inserting a very light (small value) element at the bottom of a container of water. Because it's lighter than the water above it, it bubbles upward. It keeps rising until it meets something equally or less dense above it — then it stops. That's heapify-up!

06
Heapify Down (Bubble Down / Sift Down)
// compare with children · swap with smallest child · continue downward
Theory

Heapify-Down moves a node downward toward the leaves until it reaches its correct position. It's used after extracting the minimum (when the last element is moved to the root).

Algorithm: Compare current node with both children. Find the smallest among the three (current, left child, right child). If the smallest is NOT the current node, swap with the smallest child and repeat from the child's position. Stop when current is smaller than both children or has no children (leaf node).

Always swap with the smaller child (not just any child) to maintain the min-heap property for both subtrees.

C Heapify Down — Detailed with Comments 
void heapifyDown(MinHeap* heap, int index) {
    int size = heap->size;
    
    while (1) {
        /* Calculate child indices */
        int left    = 2 * index + 1;
        int right   = 2 * index + 2;
        int smallest = index;  /* Start: assume current is smallest */
        
        /* Compare with LEFT child (only if it exists) */
        if (left < size && heap->data[left] < heap->data[smallest]) {
            smallest = left;
            printf("Left child[%d]=%d is smaller\n",
                   left, heap->data[left]);
        }
        
        /* Compare with RIGHT child (only if it exists) */
        if (right < size && heap->data[right] < heap->data[smallest]) {
            smallest = right;
            printf("Right child[%d]=%d is even smaller\n",
                   right, heap->data[right]);
        }
        
        /* If smallest is not the current node, swap and continue */
        if (smallest != index) {
            printf("Swapping data[%d]=%d with data[%d]=%d\n",
                   index, heap->data[index],
                   smallest, heap->data[smallest]);
            swap(&heap->data[index], &heap->data[smallest]);
            index = smallest;  /* Move down to swapped child */
        } else {
            printf("data[%d]=%d is in correct position!\n",
                   index, heap->data[index]);
            break;  /* Heap property restored! */
        }
    }
}
🪨

Analogy: Heavy Rock Sinking in Water

A heavy (large value) element placed at the top sinks down. At each level, it compares with both children and swaps with the lighter (smaller) one. It keeps sinking until it's heavier than everything above it and lighter than everything below — just like heapify-down!

07
Build Min Heap
// bulk insert · start from last non-leaf · heapify-down repeatedly
Theory

Building a heap from an arbitrary array: instead of inserting one-by-one (O(n log n)), we use Floyd's Algorithm to build in O(n).

Key Insight: Leaf nodes (bottom half of array) are already valid single-element heaps. So we only need to heapify-down non-leaf nodes. Non-leaf nodes exist from index 0 to n/2 - 1.

Algorithm: Start from the last non-leaf node (index n/2 - 1) and go backward to index 0, calling heapifyDown on each. This efficiently converts any array into a valid min-heap.

Time: O(n) — More efficient than n inserts! Floyd's Algorithm Start from n/2-1 (last non-leaf) Go backward to index 0
🏗️ Build Min Heap from Array

C Build Min Heap — Floyd's Algorithm 
void buildMinHeap(MinHeap* heap, int arr[], int n) {
    /* Step 1: Copy all elements into heap's data array */
    for (int i = 0; i < n; i++) {
        heap->data[i] = arr[i];
    }
    heap->size = n;
    
    /* Step 2: Find the last non-leaf node
       In a 0-indexed array: last non-leaf = n/2 - 1
       
       Why? Leaf nodes are at indices n/2 to n-1.
       They're already valid heaps (single elements)!
       We only need to fix non-leaf nodes.
    */
    int lastNonLeaf = n / 2 - 1;
    
    printf("Starting heapify from index %d (last non-leaf)\n",
           lastNonLeaf);
    
    /* Step 3: Heapify-down each non-leaf node (right to left) */
    for (int i = lastNonLeaf; i >= 0; i--) {
        printf("Heapifying node at index %d (value=%d)\n",
               i, heap->data[i]);
        heapifyDown(heap, i);
    }
    
    printf("Build complete! Min-heap is ready.\n");
}

/* Usage:
   int arr[] = {30, 15, 25, 10, 5, 8, 20};
   buildMinHeap(heap, arr, 7);
   
   Initial:  [30, 15, 25, 10, 5, 8, 20]
   After i=2:[30, 15,  8, 10, 5, 25, 20]  (heapify index 2)
   After i=1:[30,  5,  8, 10, 15,25, 20]  (heapify index 1)
   After i=0:[ 5, 10,  8, 30, 15,25, 20]  (heapify index 0)
   Result: Valid min-heap! Root = 5 (minimum)
*/

By the time we reach the CEO position, everything below is already properly organized.

08
Decrease Key / Increase Priority
// locate node · reduce value · heapify-up · restore property
Theory

Decrease Key reduces the value of an existing element. Since the value gets smaller, the element might now be smaller than its parent — violating the min-heap property. We fix this by calling heapify-up from that position.

This operation is crucial for algorithms like Dijkstra's shortest path, where we discover shorter paths and need to update distances (decrease keys). The new value must be smaller than the current value.

Time: O(log n) New value must be SMALLER Calls heapify-up after Used in Dijkstra's algorithm
C Decrease Key 
void decreaseKey(MinHeap* heap, int index, int newValue) {
    /* Validate index */
    if (index < 0 || index >= heap->size) {
        printf("Invalid index %d\n", index);
        return;
    }
    
    /* New value must be SMALLER (this is decrease-key!) */
    if (newValue > heap->data[index]) {
        printf("Error: New value %d is larger than current %d\n",
               newValue, heap->data[index]);
        printf("Use increaseKey() for larger values.\n");
        return;
    }
    
    /* Update the value */
    printf("Decreasing data[%d] from %d to %d\n",
           index, heap->data[index], newValue);
    heap->data[index] = newValue;
    
    /* Heapify-Up: smaller value might need to bubble up */
    heapifyUp(heap, index);
}

/* Practical example: Dijkstra's shortest path update
   Suppose we found a shorter path to node at index 3.
   Its distance was 15, now it's 7.
*/
void dijkstraExample() {
    MinHeap* heap = createMinHeap();
    insert(heap, 5);   /* Node A: distance 5  */
    insert(heap, 15);  /* Node B: distance 15 */
    insert(heap, 10);  /* Node C: distance 10 */
    insert(heap, 20);  /* Node D: distance 20 */
    
    /* Found shorter path to node at index 1 (was 15, now 3) */
    decreaseKey(heap, 1, 3);
    
    /* Now 3 bubbles up to become new minimum */
    printf("New minimum: %d\n", peekMin(heap));  /* Output: 3 */
    free(heap);
}
🎖️

Analogy: Emergency Upgrade in Hospital Queue

A patient's condition suddenly worsens (their priority number decreases — more urgent). They must be moved to a higher position in the queue. The system upgrades their priority (decrease key) and the queue re-sorts to ensure they're seen earlier. This is exactly decrease-key + heapify-up.

09
Increase Key
// locate node · increase value · heapify-down · restore structure
Theory

Increase Key is the opposite of Decrease Key. We increase an element's value, which might now be larger than its children — violating the min-heap property. We fix this by calling heapify-down from that position.

The new value must be greater than the current value. After changing it, we sink the element down to its correct position.

Time: O(log n) New value must be LARGER Calls heapify-down after Element sinks to correct position
C Increase Key 
void increaseKey(MinHeap* heap, int index, int newValue) {
    /* Validate index */
    if (index < 0 || index >= heap->size) {
        printf("Invalid index %d\n", index);
        return;
    }
    
    /* New value must be LARGER (this is increase-key!) */
    if (newValue < heap->data[index]) {
        printf("Error: New value %d is smaller than current %d\n",
               newValue, heap->data[index]);
        printf("Use decreaseKey() for smaller values.\n");
        return;
    }
    
    /* Update the value */
    printf("Increasing data[%d] from %d to %d\n",
           index, heap->data[index], newValue);
    heap->data[index] = newValue;
    
    /* Heapify-Down: larger value might need to sink down */
    heapifyDown(heap, index);
}

/* Example: Update a task's priority (make it less urgent) */
void example() {
    MinHeap* heap = createMinHeap();
    insert(heap, 1);   /* Critical task   */
    insert(heap, 5);   /* Important task  */
    insert(heap, 3);   /* Moderate task   */
    
    /* Index 0 (root, value=1) becomes less urgent → 10 */
    increaseKey(heap, 0, 10);
    
    /* 10 sinks down. New minimum = 3 */
    printf("New min: %d\n", peekMin(heap));  /* Output: 3 */
    free(heap);
}
📉

Analogy: Rescheduling a Meeting to Later

A high-priority meeting (low priority number) gets postponed (key increases — less urgent). It moves to a later slot in the schedule. Other meetings that were scheduled after it might now need to happen first. The schedule (heap) re-organizes itself — that's increase-key + heapify-down.

10
Change Priority
// search target · compare old vs new · call decrease-key or increase-key · rebalance
Theory

Change Priority is a smart wrapper that decides whether to call decreaseKey or increaseKey based on the new value compared to the old value.

If new value < old value → call decreaseKey (heapify-up). If new value > old value → call increaseKey (heapify-down). If equal → no action needed. This unified interface makes code cleaner.

C Change Priority — Smart Wrapper 
void changePriority(MinHeap* heap, int index, int newValue) {
    /* Validate index */
    if (index < 0 || index >= heap->size) {
        printf("Invalid index!\n");
        return;
    }
    
    int oldValue = heap->data[index];
    
    if (newValue == oldValue) {
        printf("No change needed. Value is already %d\n", oldValue);
        return;
    }
    
    /* Update the value first */
    heap->data[index] = newValue;
    
    if (newValue < oldValue) {
        /* New value is smaller → might need to bubble UP */
        printf("Value decreased %d→%d, heapifying up...\n",
               oldValue, newValue);
        heapifyUp(heap, index);
    } else {
        /* New value is larger → might need to sink DOWN */
        printf("Value increased %d→%d, heapifying down...\n",
               oldValue, newValue);
        heapifyDown(heap, index);
    }
}

/* Example: Dynamic task scheduler */
void taskSchedulerExample() {
    MinHeap* heap = createMinHeap();
    insert(heap, 4);
    insert(heap, 8);
    insert(heap, 6);
    insert(heap, 12);
    
    printf("Before: min=%d\n", peekMin(heap));  /* 4 */
    
    changePriority(heap, 3, 1);  /* Index 3 (was 12) → 1 */
    
    printf("After:  min=%d\n", peekMin(heap));  /* Now 1! */
    free(heap);
}
🔄

Analogy: Flight Seat Upgrade or Downgrade

Imagine airline boarding priority. A passenger can be upgraded (priority decreases — boards sooner) or downgraded (priority increases — boards later). The system automatically moves them to the right position in the boarding group — just like changePriority calling the correct heapify function.

11
Delete Specific Element
// locate node · replace with last · decrease size · heapify-up or down
Theory

Deleting a specific element (not necessarily the minimum) uses a clever trick: decrease the element's key to −∞ (INT_MIN), making it the smallest in the heap. Then it naturally bubbles to the root. Finally, call extractMin() to remove it.

Alternatively: Replace the target with the last element, reduce size, then heapify up or down from that position depending on whether the replacement is smaller or larger than its neighbors.

C Delete Specific Element 
/* Method 1: Using INT_MIN trick (simple & clean) */
void deleteElement(MinHeap* heap, int index) {
    if (index < 0 || index >= heap->size) {
        printf("Invalid index!\n");
        return;
    }
    
    printf("Deleting element %d at index %d\n",
           heap->data[index], index);
    
    /* Step 1: Make this element the smallest (INT_MIN) */
    heap->data[index] = INT_MIN;
    
    /* Step 2: Bubble it up to the root */
    heapifyUp(heap, index);
    
    /* Step 3: Now it's at root, extract it! */
    extractMin(heap);
    
    printf("Element deleted successfully!\n");
}

/* Method 2: Replace with last element */
void deleteElementV2(MinHeap* heap, int index) {
    if (index < 0 || index >= heap->size) return;
    
    int oldVal = heap->data[index];
    
    /* Replace with last element */
    heap->data[index] = heap->data[heap->size - 1];
    heap->size--;
    
    if (heap->size == 0 || index == heap->size) return;
    
    /* Decide direction: up or down? */
    if (heap->data[index] < oldVal) {
        heapifyUp(heap, index);   /* Replacement is smaller */
    } else {
        heapifyDown(heap, index); /* Replacement is larger  */
    }
}

/* Example:
   Heap: [1, 5, 3, 8, 10, 6]
   Delete index 2 (value=3):
   → Set to INT_MIN: [1, 5, INT_MIN, 8, 10, 6]
   → Heapify up: [INT_MIN, 5, 1, 8, 10, 6]
      → Wait, INT_MIN < 1 (root), so swap: [INT_MIN, 5, 1, 8, 10, 6]
      (Actually INT_MIN bubbles all the way to root)
   → ExtractMin removes INT_MIN
   → Result: [1, 5, 6, 8, 10]
*/

To remove someone from the middle of a priority queue: first give them VIP status (INT_MIN = highest priority), they naturally move to the front. Then serve them (extract min), removing them from the queue. The remaining people reorganize by priority.

Search Element
// traverse heap array · compare target · return index if found
Theory

Unlike Binary Search Trees, a heap does NOT support efficient searching. The heap property only guarantees parent ≤ children — there's no ordering between siblings or cousins. So we must do a linear scan of the array.

Time complexity is O(n) — we may need to check every element. This is a known limitation of heaps. If frequent searching is needed, consider using a hash map alongside the heap to track indices.

Time: O(n) — Linear scan required! No ordering between siblings Returns index of found element Use hash map for O(1) lookup

    for (int i = 0; i < heap->size; i++) {
        if (heap->data[i] == target) {
            printf("Found %d at index %d!\n", target, i);
            return i;  /* Return the index */
        }
    }
    
    printf("%d not found in heap.\n", target);
    return -1;  /* -1 means not found */
}

/* Optimized: Early termination hint
   If target > parent in any path, it might not exist in that subtree.
   But this is NOT guaranteed due to heap structure.
   For general use, always do full linear scan.
*/
int searchAll(MinHeap* heap, int target, int* indices, int* count) {
    *count = 0;
    for (int i = 0; i < heap->size; i++) {
        if (heap->data[i] == target) {
            indices[(*count)++] = i;  /* Collect all occurrences */
        }
    }
    return *count;  /* Return total occurrences found */
}

Merge Min Heaps
// combine arrays · rebuild min heap · restore heap order
Theory

Merging two min-heaps: combine both arrays into one, then rebuild the heap using Floyd's algorithm. This is the simplest approach and runs in O(n) where n is the total number of elements.

More advanced heap types (Fibonacci Heap, Binomial Heap) can merge in O(log n) or even O(1). But for a standard binary min-heap, the rebuild approach is straightforward and effective.

Time: O(n) using buildHeap Simple: concat + rebuild Both heaps remain valid input Fibonacci Heap merges in O(1)
C Merge Two Min Heaps 
MinHeap* mergeHeaps(MinHeap* h1, MinHeap* h2) {
    /* Create new heap with combined capacity */
    MinHeap* merged = (MinHeap*) malloc(sizeof(MinHeap));
    merged->capacity = h1->size + h2->size;
    merged->size     = 0;
    
    if (merged->capacity > MAX_SIZE) {
        printf("Merged heap too large!\n");
        free(merged);
        return NULL;
    }
    
    /* Step 1: Copy all elements from heap 1 */
    for (int i = 0; i < h1->size; i++) {
        merged->data[merged->size++] = h1->data[i];
    }
    
    /* Step 2: Copy all elements from heap 2 */
    for (int i = 0; i < h2->size; i++) {
        merged->data[merged->size++] = h2->data[i];
    }
    
    /* Step 3: Rebuild to form valid min-heap */
    for (int i = merged->size / 2 - 1; i >= 0; i--) {
        heapifyDown(merged, i);
    }
    
    printf("Merged heap size: %d, min: %d\n",
           merged->size, merged->data[0]);
    return merged;
}

/* Usage */
int main() {
    MinHeap* h1 = createMinHeap();
    MinHeap* h2 = createMinHeap();
    
    insert(h1, 2); insert(h1, 8); insert(h1, 4);
    insert(h2, 1); insert(h2, 6); insert(h2, 3);
    
    MinHeap* combined = mergeHeaps(h1, h2);
    /* combined → [1, 2, 3, 8, 6, 4] (valid min-heap) */
    
    free(h1); free(h2); free(combined);
    return 0;
}

Analogy: Merging Two Hospital Waiting Rooms

Two hospitals merge. All patients from both waiting rooms are combined into one room. The staff then re-sorts the combined group by urgency (rebuild heap). The most critical patient across both original rooms is now clearly at the front of the new queue.

Traverse Heap
// level-order traversal · sequential array traversal · display contents
Theory

Heap traversal can be done two ways: Array traversal (simply print data[0] to data[size-1]) gives sequential access. Level-order traversal visits nodes level by level (which is the same as array order since heap IS an array!). Both are O(n).

The array representation makes traversal trivially simple — there's no pointer-following needed. Index 0 is level 0 (root), indices 1-2 are level 1, indices 3-6 are level 2, and so on.

C Traverse Heap — Array & Level-Order 
/* Method 1: Simple array traversal (O(n)) */
void traverseArray(MinHeap* heap) {
    printf("Heap array: [ ");
    for (int i = 0; i < heap->size; i++) {
        printf("%d ", heap->data[i]);
    }
    printf("]  (size=%d)\n", heap->size);
}

/* Method 2: Level-order traversal (same order as array!) */
void traverseLevelOrder(MinHeap* heap) {
    if (isEmpty(heap)) {
        printf("Heap is empty!\n");
        return;
    }
    
    int level = 0;
    int levelStart = 0;    /* First index of current level */
    int levelSize  = 1;    /* Nodes in current level */
    
    while (levelStart < heap->size) {
        printf("Level %d: [ ", level);
        
        for (int i = levelStart;
             i < levelStart + levelSize && i < heap->size; i++) {
            printf("%d ", heap->data[i]);
        }
        printf("]\n");
        
        levelStart += levelSize;    /* Move to next level */
        levelSize  *= 2;           /* Next level has 2x nodes */
        level++;
    }
}

/* Example output for heap [1, 3, 5, 8, 10, 7, 9]:
   Level 0: [ 1 ]
   Level 1: [ 3 5 ]
   Level 2: [ 8 10 7 9 ]
*/

Clear Heap
// remove all elements · reset size = 0 · reinitialize structure
Theory

Clearing a heap is surprisingly simple: just set size = 0. Since the heap only uses elements from index 0 to size-1, setting size to 0 effectively "removes" all elements. The actual data remains in memory but will be overwritten on future inserts.

This is O(1) — much faster than freeing and reallocating. If you need to guarantee data erasure (for security), you can also zero out the array.

C Clear Heap 
/* Method 1: Fast clear — just reset size (O(1)) */

    heap->size = 0;
    printf("Heap cleared! Ready to accept new elements.\n");
}

/* Method 2: Secure clear — zero out data too (O(n)) */
void clearHeapSecure(MinHeap* heap) {
    for (int i = 0; i < heap->size; i++) {
        heap->data[i] = 0;  /* Overwrite with zeros */
    }
    heap->size = 0;
    printf("Heap securely cleared!\n");
}

/* Method 3: Free and destroy (if dynamically allocated) */
void destroyHeap(MinHeap** heap) {
    if (*heap != NULL) {
        free(*heap);     /* Free the memory */
        *heap = NULL;    /* Set pointer to NULL (safe!) */
        printf("Heap destroyed and memory freed.\n");
    }
}

Validate Min Heap Property
// verify parent-child relationships · ensure parent ≤ children · return result
Theory

Validation checks that every parent is ≤ both its children throughout the entire heap. We iterate through all non-leaf nodes (indices 0 to n/2-1) and verify each one against its children. This is used for debugging and testing.

A valid min-heap must satisfy: for every node i, data[i] ≤ data[2i+1] and data[i] ≤ data[2i+2] (when children exist). This check is O(n).

C Validate Min Heap Property 
/* Returns 1 if valid min-heap, 0 if invalid */
int validateMinHeap(MinHeap* heap) {
    if (heap->size <= 1) {
        printf("Heap with 0 or 1 element is always valid!\n");
        return 1;
    }
    
    int isValid = 1;
    
    /* Check all non-leaf nodes (0 to size/2-1) */
    for (int i = 0; i < heap->size / 2; i++) {
        int left  = 2 * i + 1;
        int right = 2 * i + 2;
        
        /* Check left child */
        if (left < heap->size &&
            heap->data[i] > heap->data[left]) {
            printf("VIOLATION at index %d: parent=%d > left=%d\n",
                   i, heap->data[i], heap->data[left]);
            isValid = 0;
        }
        
        /* Check right child */
        if (right < heap->size &&
            heap->data[i] > heap->data[right]) {
            printf("VIOLATION at index %d: parent=%d > right=%d\n",
                   i, heap->data[i], heap->data[right]);
            isValid = 0;
        }
    }
    
    if (isValid) {
        printf("✓ Valid min-heap! All parent ≤ children.\n");
    } else {
        printf("✗ INVALID heap! Violations found above.\n");
    }
    return isValid;
}

   Valid:   [1, 3, 5, 8, 10]  → all parents ≤ children ✓
   Invalid: [1, 3, 2, 8, 10]  → 3 > 2 VIOLATION! ✗
                                 (3 at index 1, child 2 at index 2)
*/

Convert to Priority Queue View
// repeatedly extract minimum · display ascending order · simulate PQ processing
Theory

A Priority Queue view shows all elements in their processing order — smallest first. We achieve this by repeatedly calling extractMin() on a copy of the heap until it's empty. This is essentially heap sort!

Important: We must work on a copy to preserve the original heap. This operation is O(n log n) since each of the n extractions takes O(log n).

void priorityQueueView(MinHeap* heap) {

    }
