Traversal
// display_list

What is a Circular Linked List?

A Circular Linked List (CLL) is a type of linked list where the last node does NOT point to NULL. Instead, it points back to the first node, forming a circle.

Every node has two parts: data (the value stored) and next (the address of the next node). The last node's next points to the head, completing the circle.

Traversal — Display List

Traversal means visiting every node exactly once. In a CLL, we stop when we come back to the starting node (head).

  • 1Start from the head node.
  • 2Print the data of the current node.
  • 3Move to the next node.
  • 4Repeat until you reach head again.
  • 5Stop when current == head.
⚠️ Key Rule: Always check if the list is empty (head == NULL) before traversing.

#include <stdio.h>
#include <stdlib.h>

struct Node {
    int data;
    struct Node *next;  /* last node's next → head */
};

struct Node* createNode(int val) {
    struct Node *n = (struct Node*)malloc(sizeof(struct Node));
    n->data = val;
    n->next = n;  /* points to itself */
    return n;
}

void displayList(struct Node *head) {
    if (head == NULL) { printf("Empty!\n"); return; }
    struct Node *curr = head;
    do {
        printf("%d → ", curr->data);
        curr = curr->next;
    } while (curr != head);
    printf("(back to %d)\n", head->data);
}

int main() {
    struct Node *head = createNode(10);
    struct Node *n2 = createNode(20);
    struct Node *n3 = createNode(30);
    head->next = n2; n2->next = n3; n3->next = head;
    displayList(head);
    return 0;
}
/* Output: 10 → 20 → 30 → (back to 10) */

Real-life Analogy — Merry-Go-Round

A merry-go-round has seats in a circle — each seat points to the next, and the last seat points back to the first. Watching the ride is like traversal: you note each child as they pass and stop when you see the first child again.

Real Use Case: Music playlists on loop, round-robin CPU scheduling, token passing in networks.

02
Insertion
// at_beginning · at_end · at_position

📌 At Beginning

Insert before the head. New node's next → old head. Update last node's next → new head. New node becomes head.

📌 At End

Traverse to the last node (whose next == head). Set last→new node, new node→head.

📌 At Specific Position

Traverse to position-1, insert the new node between that node and its successor.

⚠️ Common Mistake: Forgetting to update the last node's next to the new head after inserting at beginning!

/* Insert at BEGINNING */
struct Node* insertAtBeginning(struct Node *head, int val) {
    struct Node *newNode = createNode(val);
    if (head == NULL) return newNode;
    struct Node *last = head;
    while (last->next != head) last = last->next;
    newNode->next = head;
    last->next = newNode;
    return newNode;
}

/* Insert at END */
struct Node* insertAtEnd(struct Node *head, int val) {
    struct Node *newNode = createNode(val);
    if (head == NULL) return newNode;
    struct Node *last = head;
    while (last->next != head) last = last->next;
    last->next = newNode;
    newNode->next = head;
    return head;
}

/* Insert at POSITION (1-based) */
struct Node* insertAtPosition(struct Node *head, int val, int pos) {
    if (pos == 1) return insertAtBeginning(head, val);
    struct Node *newNode = createNode(val);
    struct Node *curr = head;
    int i;
    for (i = 1; i < pos - 1; i++) {
        curr = curr->next;
        if (curr == head) { printf("Out of range!\n"); return head; }
    }
    newNode->next = curr->next;
    curr->next = newNode;
    return head;
}

03
Deletion
// from_beginning · from_end · specific_node · by_value

🗑️ From Beginning

Find the last node, point it to head's next. Free the old head.

🗑️ From End

Traverse to the second-to-last node. Set its next to head. Free the last node.

🗑️ Specific Node

Find the node before target. Update its next to skip the target. Free target.

🗑️ By Value

Search for a node with the given value. Bypass it using the previous node's next pointer, then free it.

Memory Management

In C, deleted nodes must be free()d manually to avoid memory leaks. Always save the pointer before overwriting it, then call free().

/* Delete from BEGINNING */
struct Node* deleteFromBeginning(struct Node *head) {
    if (head == NULL) return NULL;
    if (head->next == head) { free(head); return NULL; }
    struct Node *last = head;
    while (last->next != head) last = last->next;
    struct Node *newHead = head->next;
    last->next = newHead;
    free(head);
    return newHead;
}

/* Delete from END */
struct Node* deleteFromEnd(struct Node *head) {
    if (head == NULL) return NULL;
    if (head->next == head) { free(head); return NULL; }
    struct Node *prev = head;
    while (prev->next->next != head) prev = prev->next;
    free(prev->next);
    prev->next = head;
    return head;
}

/* Delete by VALUE */
struct Node* deleteByValue(struct Node *head, int val) {
    if (head == NULL) return NULL;
    if (head->data == val) return deleteFromBeginning(head);
    struct Node *prev = head, *curr = head->next;
    do {
        if (curr->data == val) {
            prev->next = curr->next;
            free(curr);
            return head;
        }
        prev = curr; curr = curr->next;
    } while (curr != head);
    printf("Value %d not found!\n", val);
    return head;
}

Searching
// search_by_value · search_by_position

🔍 Search by Value

Traverse the list and compare each node's data with the target. Return position when found. Stop when we loop back to head.

🔍 Search by Position

Traverse while counting. When counter equals desired position, return that node's data.

Linear Search in CLL

CLL does not support binary search (no random access like arrays). We must perform a linear search — checking one node at a time. Worst case O(n).

int searchByValue(struct Node *head, int val) {
    if (head == NULL) return -1;

    int pos = 1;
    do {
        if (curr->data == val) { printf("Found at pos %d\n", pos); return pos; }

int searchByPosition(struct Node *head, int pos) {
    if (head == NULL || pos < 1) return -1;
    struct Node *curr = head;
    int i;
    for (i = 1; i < pos; i++) {
        curr = curr->next;
        if (curr == head) { printf("Out of range!\n"); return -1; }
    }
    printf("Position %d = %d\n", pos, curr->data);
    return curr->data;
}

Updating
// modify_node_data · replace_value

✏️ Modify Node Data

Traverse to a given position, then change the data field. Pointers stay unchanged — no circular structure risk.

✏️ Replace Value

Search for all nodes with a specific old value and update them to a new value in a single traversal pass.

Updating is Safe

Unlike insertion or deletion, updating only changes the data field — not any pointers. The circular structure is never at risk. Simple and O(n).

void replaceValue(struct Node *head, int oldVal, int newVal) {
    if (!head) return;
    struct Node *curr = head;
    int count = 0;
    do {
        if (curr->data == oldVal) { curr->data = newVal; count++; }
        curr = curr->next;
    } while (curr != head);
    printf("Replaced %d occurrence(s)\n", count);
}

07
Reversing
// full_reverse

Full Reverse

Make every node's next point to the previous node. The original last node becomes the new head. Use three pointers: prev, current, next.

  • 1Store original head to detect when we've gone all the way around.
  • 2For each node: save current->next, set current->next = prev.
  • 3Move prev and current forward.
  • 4After loop, new head is prev. Set head->next = prev to complete the circle.
OperationTimeSpace
Full ReverseO(n)O(1)
REVERSE VISUALIZER

CReverse CLL
struct Node* reverseCLL(struct Node *head) {
    if (!head || head->next == head) return head;
    struct Node *prev = NULL, *curr = head, *next, *orig = head;
    do {
        next = curr->next;
        curr->next = prev;
        prev = curr;
        curr = next;
    } while (curr != orig);
    head->next = prev;   /* original head (now last) → new head */
    return prev;
}
/* Before: 1→2→3→4→(1)  After: 4→3→2→1→(4) */

A circular railway loop where trains go clockwise. Reversing the list = making all trains go counter-clockwise. The engine that was last now leads.

Real Use Case: Reversing undo history, reversing circular buffers for playback.

Joining Two CLLs

Find the last nodes of both lists, then swap two pointers to merge them into one big circle.

  • 1Find last1 (last of List 1).
  • 2Find last2 (last of List 2).
  • 3Set last1->next = head2.
  • 4Set last2->next = head1.

struct Node* concatenate(struct Node *h1, struct Node *h2) {
    if (!h1) return h2;
    if (!h2) return h1;
    struct Node *last1 = h1;
    while (last1->next != h1) last1 = last1->next;
    struct Node *last2 = h2;
    while (last2->next != h2) last2 = last2->next;
    last1->next = h2;    /* end of L1 → start of L2 */
    last2->next = h1;    /* end of L2 → start of L1 */
    return h1;
}
/* L1: 1→2→3→(1)  L2: 4→5→6→(4)
   Result: 1→2→3→4→5→6→(1) */

CSplit CLL into Two Halves
void splitIntoHalves(struct Node *head,
                     struct Node **h1, struct Node **h2) {
    if (!head) { *h1 = *h2 = NULL; return; }
    struct Node *slow = head, *fast = head;
    while (fast->next != head && fast->next->next != head) {
        fast = fast->next->next;
        slow = slow->next;
    }
    if (fast->next->next == head) fast = fast->next;
    *h1 = head;
    *h2 = slow->next;
    fast->next = *h2;   /* fix second half's circular end */
    slow->next = *h1;   /* fix first half's circular end */
}

10
Rotation
// left_rotation · right_rotation

↰ Left Rotation (by k)

Move the head pointer forward by k steps. Node at position k+1 becomes new head. Circular structure stays intact.

↱ Right Rotation (by k)

Move head backward by k steps (= forward by n-k steps). Opposite of left rotation.

Key Insight

In a CLL, rotation just changes where "head" points. Actual nodes and links don't change. Use k = k % n to avoid unnecessary full cycles.

OperationTimeSpace
Left Rotation by kO(k)O(1)
Right Rotation by kO(n-k)O(1)

CRotation Operations
struct Node* rotateLeft(struct Node *head, int k) {
    if (!head || k == 0) return head;
    int n = countNodes(head);
    k = k % n;
    if (k == 0) return head;
    struct Node *curr = head;
    int i;
    for (i = 0; i < k - 1; i++) curr = curr->next;
    struct Node *newHead = curr->next;
    return newHead;
}

/* Right rotation = Left rotation by (n - k) */
struct Node* rotateRight(struct Node *head, int k) {
    if (!head || k == 0) return head;
    int n = countNodes(head);
    return rotateLeft(head, n - (k % n));
}
🎰

Real-life Analogy — Rotating a Combination Lock

A combination lock dial is circular. Rotating left or right doesn't change which numbers exist — it just changes which is at the top (the "head"). Numbers stay in the same relative order.

Real Use Case: Circular menus, token rotation in Token Ring networks.

Count total nodes n, then traverse n/2 steps to find the middle. Simple but requires 2 passes.

Sorting
// bubble_sort · selection_sort · merge_sort

🫧 Bubble Sort

Compare adjacent nodes; if left > right, swap data values. Repeat until no swaps occur. O(n²) but simple.

✔️ Selection Sort

Find minimum in unsorted portion, swap it to the front. O(n²) but fewer swaps than bubble sort.

⚡ Merge Sort

Divide into halves, recursively sort each, merge in sorted order. O(n log n) — most efficient.

💡 For CLL: Bubble and Selection sort swap only DATA (not pointers), which is simpler to implement.

/* Bubble Sort — swap data, not pointers */
void bubbleSort(struct Node *head) {
    if (!head) return;
    int swapped;
    struct Node *ptr, *last = NULL;
    do {
        swapped = 0; ptr = head;
        while (ptr->next != head && ptr->next != last) {
            if (ptr->data > ptr->next->data) {
                int tmp = ptr->data;
                ptr->data = ptr->next->data;
                ptr->next->data = tmp;
                swapped = 1;
            }
            ptr = ptr->next;
        }
        last = ptr;
    } while (swapped);
}

/* Selection Sort */
void selectionSort(struct Node *head) {
    if (!head) return;
    struct Node *i = head, *j, *minN;
    do {
        minN = i; j = i->next;
        while (j != head) {
            if (j->data < minN->data) minN = j;
            j = j->next;
        }
        if (minN != i) { int tmp = i->data; i->data = minN->data; minN->data = tmp; }
        i = i->next;
    } while (i != head);
}

struct Node* mergeSorted(struct Node *h1, struct Node *h2) {
    if (!h1) return h2;
    if (!h2) return h1;
    struct Node *result = NULL, *tail = NULL;
    struct Node *p1 = h1, *p2 = h2;
    int d1 = 0, d2 = 0;
    while (!d1 || !d2) {
        int val;
        if (d1) { val = p2->data; p2=p2->next; if(p2==h2) d2=1; }
        else if (d2) { val = p1->data; p1=p1->next; if(p1==h1) d1=1; }
        else if (p1->data <= p2->data) { val=p1->data; p1=p1->next; if(p1==h1) d1=1; }
        else { val=p2->data; p2=p2->next; if(p2==h2) d2=1; }
        struct Node *nd = createNode(val);
        if (!result) { result=tail=nd; nd->next=nd; }
        else { tail->next=nd; nd->next=result; tail=nd; }
    }
    return result;
}

📎 Consecutive Duplicates

Works on sorted list. If current->data == current->next->data, delete current->next. Single pass: O(n).

📎 Remove All Duplicates

For each node, scan the rest and remove matching nodes. Nested loop: O(n²). Or use a seen-set for O(n).

⚠️ After removing duplicates, always ensure the last node's next still points to head — never break the circular structure!

/* Remove CONSECUTIVE duplicates (sorted list) */
void removeConsecutiveDups(struct Node *head) {
    if (!head) return;
    struct Node *curr = head;
    do {
        while (curr->next != head && curr->data == curr->next->data) {
            struct Node *dup = curr->next;
            curr->next = dup->next;
            free(dup);
        }
        curr = curr->next;
    } while (curr != head);
}

/* Remove ALL duplicates (any order) */
void removeAllDups(struct Node *head) {
    if (!head) return;
    struct Node *c1 = head;
    do {
        struct Node *c2 = c1;
        while (c2->next != head) {
            if (c2->next->data == c1->data) {
                struct Node *d = c2->next;
                c2->next = d->next; free(d);
            } else c2 = c2->next;
        }
        c1 = c1->next;
    } while (c1 != head);
}

Josephus Problem
// circular_elimination_logic · survivor_finding

What is the Josephus Problem?

n people stand in a circle. Starting from person 1, every k-th person is eliminated. The circle shrinks after each elimination until only one — the survivor — remains. Perfectly modeled by a CLL.

⚡ Circular Elimination

Start at head, count k steps, delete that node. Move to the next node and repeat until only one remains.

🏆 Survivor Finding

Last remaining node. Its next points to itself (single-node CLL). This is the winner.

int josephus(int n, int k) {
    /* Build circular linked list 1 to n */
    struct Node *head = createNode(1), *tail = head;
    int i;
    for (i = 2; i <= n; i++) { tail->next = createNode(i); tail = tail->next; }
    tail->next = head;

    struct Node *curr = head, *prev;
    while (curr->next != curr) {
        for (i = 1; i < k; i++) { prev = curr; curr = curr->next; }
        printf("Eliminated: %d\n", curr->data);
        prev->next = curr->next;
        free(curr);
        curr = prev->next;
    }
    printf("Survivor: %d\n", curr->data);
    return curr->data;
}
/* josephus(7,3) → Eliminated: 3,6,2,7,5,1 → Survivor: 4 */

Jewish historian Flavius Josephus and 40 soldiers stood in a circle, eliminating every third person. Josephus calculated the safe position and survived. A CLL models this perfectly — nodes are eliminated one by one, the circle shrinks.

Reverse the list in chunks of k nodes. Each chunk is reversed internally; chunks' relative order stays the same.

  • 2Reverse those k nodes.
  • 3Connect reversed block's tail to next group.
  • 4Repeat for each group.
  • 5Close the circular structure at the end.

CReverse in k-Sized Groups
struct Node* reverseInGroups(struct Node *head, int k) {
    if (!head || k == 1) return head;
    int n = countNodes(head), processed = 0;
    struct Node *curr = head, *prevTail = NULL, *newHead = NULL;
    while (processed < n) {
        struct Node *gHead = curr, *prev = NULL;
        int cnt = 0;
        while (cnt < k && processed < n) {
            struct Node *nxt = curr->next;
            curr->next = prev; prev = curr; curr = nxt;
            cnt++; processed++;
        }
        if (!newHead) newHead = prev;
        if (prevTail) prevTail->next = prev;
        prevTail = gHead;
    }
    prevTail->next = newHead;  /* restore circular */
    return newHead;
}

An orchestra in a circle — a conductor reverses players in groups of 3: the third plays first, then second, then first — in each group of three. Groups keep their relative positions.

Real Use Case: Processing circular buffers in fixed-size blocks, audio effects processing.

int hasLoop(struct Node *head) {
    struct Node *slow = head, *fast = head;
    while (fast && fast->next) {

}

/* Find intersection node of two lists */
struct Node* findIntersection(struct Node *h1, struct Node *h2) {
    int l1 = countNodes(h1), l2 = countNodes(h2);
    struct Node *p1 = h1, *p2 = h2;
    int i;
    if (l1 > l2) for(i=0;i<l1-l2;i++) p1=p1->next;
    else for(i=0;i<l2-l1;i++) p2=p2->next;
    int steps = 0;
    while (p1 != p2 && steps < l1+l2) {
        p1=p1->next; p2=p2->next; steps++;
    }
    if (p1 == p2) {
        printf("Intersection at: %d\n", p1->data);
        return p1;
    }
    printf("No intersection.\n");
    return NULL;
}

int hasCommonCycle(struct Node *h1, struct Node *h2) {
    struct Node *c = h1;
    do {
        if (c == h2) { printf("Common cycle!\n"); return 1; }
        c = c->next;
    } while (c != h1);
    return 0;
}

Cloning / Copying
// shallow_copy · deep_copy

📋 Shallow Copy

Create a new head pointer pointing to the same nodes. Changes in the copy affect the original! Fast but dangerous.

📋 Deep Copy

Create all-new nodes with the same data values. Completely independent — changes don't affect the original.

Key Difference

Shallow copy: Copies the pointer. Same memory — dangerous for parallel modifications.
Deep copy: Allocates new memory for every node. Completely independent and safe.

OperationTimeSpace
Shallow CopyO(1)O(1)
Deep CopyO(n)O(n)

CShallow and Deep Copy
/* SHALLOW COPY — just copy the pointer */
struct Node* shallowCopy(struct Node *head) {
    return head;  /* same memory! modifying copy changes original */
}

/* DEEP COPY — create brand new nodes */
struct Node* deepCopy(struct Node *head) {
    if (!head) return NULL;
    struct Node *newHead = createNode(head->data);
    struct Node *copyTail = newHead, *orig = head->next;
    while (orig != head) {
        struct Node *nd = createNode(orig->data);
        copyTail->next = nd; copyTail = nd;
        orig = orig->next;
    }
    copyTail->next = newHead;  /* close the new circle */
    return newHead;  /* completely independent */
}

int main() {
    struct Node *orig = createNode(1);
    orig->next = createNode(2); orig->next->next = orig;
    struct Node *copy = deepCopy(orig);
    copy->data = 999;
    printf("Original: %d, Copy: %d\n", orig->data, copy->data);
    /* Output: Original: 1, Copy: 999 */
    return 0;
}

Real-life Analogy — Shortcut vs Photocopy

Shallow copy = desktop shortcut to a file. Both point to the same file — editing one changes the other. Deep copy = making a photocopy. Scribbling on the photocopy doesn't affect the original at all.

Real Use Case: Undo/redo systems, backups of circular data structures, forking processes in OS.

Flattening
// convert_multi_level_to_single_level

What is a Multi-Level CLL?

A multi-level CLL has nodes with an extra down pointer, each pointing to another sub-list. Flattening collapses all levels into a single CLL.

Node structure: data | next (horizontal) | down (vertical/child)

  • 1Traverse the top-level list.
  • 2For each node with a down pointer, recursively flatten that sub-list.
  • 3Insert the flattened sub-list after the current node.
  • 4Continue until all nodes are on one level.
  • 5Restore the circular structure.
OperationTimeSpace
Flatten All LevelsO(total nodes)O(depth) recursion
MULTI-LEVEL FLATTENING
Click Flatten to collapse the multi-level list into one.
CFlatten Multi-Level CLL
struct MultiNode {
    int data;
    struct MultiNode *next;  /* horizontal */
    struct MultiNode *down;  /* vertical/child */
};

struct MultiNode* getTail(struct MultiNode *h) {
    if (!h) return NULL;
    struct MultiNode *c = h;
    while (c->next && c->next != h) c = c->next;
    return c;
}

struct MultiNode* flatten(struct MultiNode *head) {
    if (!head) return NULL;
    struct MultiNode *curr = head;
    do {
        if (curr->down) {
            struct MultiNode *child = flatten(curr->down);
            curr->down = NULL;
            struct MultiNode *childTail = getTail(child);
            struct MultiNode *nxt = curr->next;
            curr->next = child;
            childTail->next = nxt;
        }
        curr = curr->next;
    } while (curr != head);
    return head;
}
/* L1: 1→2→3→(1)  with 2↓[A→B] and 3↓[D→E]
   Flattened: 1→2→A→B→3→D→E→(1) */
📚

Real-life Analogy — Flattening a Table of Contents

A book has chapters (level 1), each with sub-chapters (level 2), possibly with sub-sub-chapters (level 3). Flattening expands the entire table of contents into a single continuous numbered list — all levels collapsed into one sequence.

Real Use Case: Flattening nested menus, processing hierarchical file systems into a linear list, converting multi-level task hierarchies into a flat queue.
