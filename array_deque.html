01
Insert Front / Enqueue Front
Add an element at the front (left) end of the deque
THEORY How Insert Front Works

In an array-based deque we keep two pointers: front and rear, both starting at −1. When we insert at the front:

Overflow check — if the deque is already full (size == MAX), we cannot insert. Print error and return.

First element case — if front == -1 (deque is empty), set both front = 0 and rear = 0, then store the value.

Wrap-around — if front == 0, moving further left wraps to MAX-1 (circular array). Otherwise, simply do front = front - 1.

Store the new value at arr[front]. The element now occupies the leftmost slot.

💡 We use a circular array so the front can wrap around without wasting space. This is optional for a simple linear deque but essential for efficient circular deques.
VISUALIZER Insert Front — Step by Step
Click "Insert Front" to watch the front pointer move.
C CODE Insert Front Function
C
#define MAX 8

int arr[MAX];
int front = -1, rear = -1;

/* Insert element at front of deque */
void insertFront(int value) {
    /* Step 1: Check overflow */
    if ((front == 0 && rear == MAX - 1) ||
        (front == rear + 1)) {
        printf("Overflow! Deque is full.\n");
        return;
    }

    /* Step 2: First element */
    if (front == -1) {
        front = 0;
        rear  = 0;
    }
    /* Step 3: Wrap around to end of array */
    else if (front == 0) {
        front = MAX - 1;
    }
    /* Step 4: Normal case - move front left */
    else {
        front = front - 1;
    }

    arr[front] = value;   /* Store value */
    printf("Inserted %d at front. front=%d\n", value, front);
}

🎟️ Imagine a cinema ticket queue. Normally people join at the back. But a VIP guest is allowed to walk straight to the front of the line. That's exactly Insert Front — adding a new person at the very beginning of the queue without disturbing anyone else.

02
Insert Rear / Enqueue Rear
Add an element at the rear (right) end of the deque
THEORY How Insert Rear Works

Insert Rear is the most familiar operation — it works exactly like a regular queue's enqueue. The rear pointer moves forward (right) before placing the element.

Overflow check — stop if full.

First element — if front == -1, set front = 0 and rear = 0, then store.

Wrap-around — if rear == MAX-1 (at the last index), wrap to 0. Otherwise do rear = rear + 1.

Store the value at arr[rear].

void insertRear(int value) {

    if ((front == 0 && rear == MAX - 1) ||
        (front == rear + 1)) {
        printf("Overflow! Deque is full.\n");
        return;

    /* Step 2: First element ever */
    if (rear == -1) {
        front = 0;
        rear  = 0;
    }
    /* Step 3: Wrap around to start of array */
    else if (rear == MAX - 1) {
        rear = 0;
    }
    /* Step 4: Normal case - move rear right */
    else {
        rear = rear + 1;
    }

    arr[rear] = value;
    printf("Inserted %d at rear. rear=%d\n", value, rear);
}

03
Delete Front / Dequeue Front
Remove the element from the front (left) end
THEORY How Delete Front Works

Removing from the front means taking the first element and advancing the front pointer forward.

Underflow check — if front == -1 the deque is empty; nothing to remove. Print error.

Save arr[front] in a temporary variable so you can return/print it.

Only one element? If front == rear, reset both to -1. Deque becomes empty.

Wrap-around — if front == MAX-1, wrap to 0. Otherwise do front = front + 1.

⚠️ Underflow is the opposite of overflow. Attempting to delete from an empty deque is a logic error that must always be checked first.
VISUALIZER Delete Front — Watch the front pointer advance
Add elements, then delete from front.
C CODE Delete Front Function
C
int deleteFront() {
    /* Step 1: Underflow check */
    if (front == -1) {
        printf("Underflow! Deque is empty.\n");
        return -1;
    }

    int removed = arr[front];  /* Save value */

    /* Step 2: Only one element left */
    if (front == rear) {
        front = -1;
        rear  = -1;
    }
    /* Step 3: Wrap around */
    else if (front == MAX - 1) {
        front = 0;
    }
    /* Step 4: Move front right */
    else {
        front = front + 1;
    }

    printf("Removed %d from front.\n", removed);
    return removed;
}
ANALOGY Real-Life Example

🎠 Picture a roller-coaster ride queue. When the ride is ready, the person at the very front of the line gets on. They leave the queue; everyone behind them stays. That's Delete Front — serving (removing) the element that has been waiting the longest.

04
Delete Rear / Dequeue Rear
Remove the element from the rear (right) end
THEORY How Delete Rear Works

Delete Rear is a unique ability that a normal queue does NOT have. It removes the most recently added element from the back.

Underflow check — if empty (front == -1), print error and return.

Save arr[rear] to return it.

Only one element? Reset both pointers to -1.

Wrap-around — if rear == 0, wrap to MAX-1. Otherwise rear = rear - 1.

VISUALIZER Delete Rear — Watch the rear pointer shrink
Add elements, then delete from rear.
C CODE Delete Rear Function
C
int deleteRear() {
    if (front == -1) {
        printf("Underflow! Deque is empty.\n");
        return -1;
    }

    int removed = arr[rear];

    if (front == rear) {      /* Last element */
        front = -1;
        rear  = -1;
    }
    else if (rear == 0) {   /* Wrap around */
        rear = MAX - 1;
    }
    else {                    /* Move rear left */
        rear = rear - 1;
    }

    printf("Removed %d from rear.\n", removed);
    return removed;
}
ANALOGY Real-Life Example

🃏 Think of a deck of playing cards. A dealer can draw a card from the bottom of the deck (rear) just as easily as from the top. Delete Rear is like drawing from the bottom — it's a unique power not found in ordinary queues.

05–06
Get Front & Get Rear / Peek
View elements without removing them
THEORY Peek Operations Explained

Get Front (Peek Front) — looks at arr[front] and returns the value. The front pointer does not change. The element stays in the deque.

Get Rear (Peek Rear) — same idea but for arr[rear]. Returns the last element without removing it.

✅ Peek operations are always O(1). They never modify the deque; they only read it. Always check for underflow (empty deque) before peeking.

Add elements, then peek front or rear.
C CODE Peek Front & Peek Rear
C
/* View front element — NO deletion */
int getFront() {
    if (front == -1) {
        printf("Deque is empty.\n");
        return -1;
    }
    printf("Front element: %d\n", arr[front]);
    return arr[front];   /* No pointer change! */
}

/* View rear element — NO deletion */
int getRear() {
    if (rear == -1) {
        printf("Deque is empty.\n");
        return -1;
    }
    printf("Rear element: %d\n", arr[rear]);
    return arr[rear];    /* No pointer change! */
}
ANALOGY Real-Life Example

📦 Imagine a conveyor belt in a factory. A quality inspector can look at the item at the start or end of the belt to check it — without actually taking it off the belt. That's exactly what Peek does: observe without disturbing.

07
isEmpty ()
Check if the deque contains zero elements
THEORY How isEmpty Works

The rule is simple: when both front and rear are -1, the deque holds no elements and is considered empty. This is the initial state.

Other operations (Delete, Peek) must always call isEmpty() first before doing anything. It is the guard condition that prevents underflow errors.

Returns TRUE when

front == -1 (and rear == -1). No elements present.

Returns FALSE when

front ≥ 0. At least one element exists.

Time Complexity

O(1) — just a single comparison.

C CODE isEmpty Function

/* Returns 1 (true) if deque is empty, 0 (false) otherwise */
int isEmpty() {
    if (front == -1) {
        return 1;   /* empty */
    }
    return 0;       /* not empty */
}

/* Usage example */
if (isEmpty()) {
    printf("Deque is empty!\n");
} else {
    printf("Deque has elements.\n");
}

08
isFull ()
Check if the deque has reached its maximum capacity
THEORY How isFull Works

An array-based deque has a fixed size MAX. When the number of elements equals MAX, no more insertions are possible. There are two conditions that indicate fullness in a circular array:

Front is 0 AND rear is MAX-1 — the deque occupies the entire array from left to right with no wrap-around.

Front = rear + 1 — the deque has wrapped around and the front pointer is just one ahead of rear, meaning no gap exists.

⚠️ Always call isFull() inside Insert functions before modifying any pointers. Inserting into a full array causes an overflow and overwrites existing data.
VISUALIZER isFull — Fill the deque to capacity
Fill all 6 slots, then check isFull.
C CODE isFull Function
C
/* Returns 1 (true) if deque is full, 0 (false) otherwise */
int isFull() {
    /* Condition 1: normal full (front=0, rear=MAX-1) */
    if (front == 0 && rear == MAX - 1) {
        return 1;
    }
    /* Condition 2: circular full (front = rear + 1) */
    if (front == rear + 1) {
        return 1;
    }
    return 0;   /* not full */
}
ANALOGY Real-Life Example

🛗 An elevator has a maximum weight or person limit. Before anyone can step in, the sensor checks if the elevator is already at capacity. If full, it refuses entry and beeps. isFull() is that capacity sensor — it prevents overfilling the deque.

09
Size / Count
Find the total number of elements currently in the deque
THEORY Calculating Size

The size calculation depends on where front and rear sit relative to each other:

Empty Deque

front == -1 → size = 0

Normal (rear ≥ front)

size = rear − front + 1

Wrapped Around (rear < front)

size = MAX − front + rear + 1

VISUALIZER Size — Count elements dynamically
Add and remove elements, then check size.
C CODE Size Function
C
int size() {
    if (front == -1) {
        return 0;   /* empty */
    }

    /* Normal case: rear is to the right of front */
    if (rear >= front) {
        return rear - front + 1;
    }

    /* Wrapped-around case */
    return MAX - front + rear + 1;
}

/* Print size */
printf("Current size: %d\n", size());
ANALOGY Real-Life Example

🧺 Imagine counting items in a shopping basket. You don't need to remove them — just count them in place. size() tells you exactly how many items (elements) are in your basket (deque) at any moment, without disturbing their order.

10
Traverse / Display
Print all elements from front to rear
THEORY How Traverse Works

Traverse visits every element from front to rear in order, printing each one. The tricky part is handling the circular wrap-around properly:

If empty (front == -1), print "Empty deque" and return.

Start a loop at index i = front.

Print arr[i], then move i = (i + 1) % MAX. The modulo handles the wrap-around automatically.

Stop when i == rear. Print arr[rear] last.

VISUALIZER Traverse — Watch each cell light up in order
Add elements then click Traverse to animate.
C CODE Traverse / Display Function
C
void display() {
    if (front == -1) {
        printf("Deque is empty.\n");
        return;
    }

    printf("Deque [ FRONT -> REAR ]: ");

    int i = front;
    while (1) {
        printf("%d ", arr[i]);

        if (i == rear) break;        /* reached the end */

        i = (i + 1) % MAX;           /* move forward, wrap if needed */
    }

    printf("\n");
}
ANALOGY Real-Life Example

📖 Like reading a book from the first page to the last. You visit every page in order without tearing any out. Traverse simply "reads" every element from front to rear and shows its value — perfect for debugging or displaying state.

11
Clear / Reset
Remove all elements and reset the deque to initial state
THEORY How Clear Works

Clear resets the deque to an empty state instantly. There is no need to physically erase array values — simply resetting front and rear to -1 logically empties the deque. Any old data will be overwritten on the next insertion.

After Clear

front = -1, rear = -1. Deque appears empty to all other operations.

Time Complexity

O(1) — just two assignments. Very fast regardless of how full the deque was.

VISUALIZER Clear — Instant reset
Fill the deque, then hit Clear.
C CODE Clear Function
C
/* Clears all elements — O(1) operation */
void clearDeque() {
    front = -1;
    rear  = -1;
    printf("Deque cleared. front=%d, rear=%d\n", front, rear);
}

/* Optional: also zero-out the array (for security) */
void clearDequeSecure() {
    int i;
    for (i = 0; i < MAX; i++) {
        arr[i] = 0;
    }
    front = -1;
    rear  = -1;
    printf("Deque securely cleared.\n");
}
ANALOGY Real-Life Example

🧽 Imagine a whiteboard filled with notes. Instead of erasing each word one by one, you pick up the eraser and wipe the entire board in one sweep. clear() is that one sweep — it discards everything instantly and gives you a fresh start.

12
Search ()
Find a specific element and report its position
THEORY How Search Works

Search scans through the deque from front to rear, comparing each element with the target value. It uses the same circular traversal pattern as Display.

Since elements are not sorted, we must use Linear Search — O(n) worst case. For each position, if arr[i] == target, we report it found and return its index.

💡 Search returns the position from front (0-indexed), not the raw array index. This is more intuitive: position 0 = front, position 1 = one after front, etc.
VISUALIZER Search — Watch cells light up during scan
Add elements (auto-numbered), then search for a value.
C CODE Search Function
C
/* Search for a value. Returns position from front (0-based)
   or -1 if not found. */
int search(int target) {
    if (front == -1) {
        printf("Deque is empty.\n");
        return -1;
    }

    int i = front, pos = 0;

    while (1) {
        if (arr[i] == target) {
            printf("Found %d at position %d\n", target, pos);
            return pos;
        }
        if (i == rear) break;      /* end of deque */
        i = (i + 1) % MAX;
        pos++;
    }

    printf("%d not found in deque.\n", target);
    return -1;
}
ANALOGY Real-Life Example

🕵️ Like looking for a specific item in a line of people. You start from the front and check each person one by one until you find who you're looking for (or reach the end). That's linear search — simple, reliable, and complete.

13
Update / Modify
Change the value at a specific position in the deque
THEORY How Update Works

Update finds the element at a given position from the front (0-indexed) and replaces its value. No elements are added or removed — only the value changes.

Check if deque is empty. If so, print error.

Validate the position. If pos < 0 or pos >= size(), it's out of range — print error.

Calculate the actual array index: actual = (front + pos) % MAX. This handles wrap-around correctly.

Assign arr[actual] = newValue. Done!

VISUALIZER Update — Modify a specific cell
Add elements, enter position & new value, then Update.
C CODE Update Function
C
/* Update element at position 'pos' from front (0-indexed) */
void update(int pos, int newValue) {
    if (isEmpty()) {
        printf("Deque is empty.\n");
        return;
    }

    if (pos < 0 || pos >= size()) {
        printf("Invalid position %d.\n", pos);
        return;
    }

    /* Calculate actual index with wrap-around */
    int actualIndex = (front + pos) % MAX;

    printf("Updated arr[%d]: %d -> %d\n",
           actualIndex, arr[actualIndex], newValue);

    arr[actualIndex] = newValue;
}
ANALOGY Real-Life Example

✍️ Like a teacher who has a seating chart. They know student "position 2 from the front" and they can update that student's grade directly. The student stays in their seat; only their grade (value) changes. Update does exactly this — targeted in-place modification.

14
Reverse Deque
Reverse the order of all elements in place
THEORY How Reverse Works

Reversing a deque means the front element becomes the rear and vice versa. We use the two-pointer technique: one pointer starts at front, one at rear, and they swap elements while moving toward the center.

Set left = front, right = rear.

While left != right and they haven't crossed: swap arr[left] and arr[right].

Move left = (left + 1) % MAX and right = (right - 1 + MAX) % MAX.

Continue until both pointers meet in the middle. The deque is now reversed — front and rear pointers themselves do NOT change.

✅ The front and rear pointers stay the same! Only the values inside the array slots are swapped. The deque's structure is preserved.
VISUALIZER Reverse — Watch the swap animation
Add elements, then click Reverse to watch them swap.
C CODE Reverse Deque Function
C
/* Helper: swap two integers */
void swapInt(int *a, int *b) {
    int temp = *a;
    *a = *b;
    *b = temp;
}

/* Reverse the deque in-place using two pointers */
void reverseDeque() {
    if (isEmpty()) {
        printf("Deque is empty.\n");
        return;
    }

    int left  = front;
    int right = rear;
    int count = size() / 2;   /* only need half */
    int i;

    for (i = 0; i < count; i++) {
        swapInt(&arr[left], &arr[right]);

        left  = (left  + 1)       % MAX;
        right = (right - 1 + MAX) % MAX;
    }

    printf("Deque reversed.\n");
    display();
}
ANALOGY Real-Life Example

🪞 Imagine a line of dancers facing a mirror. To reverse the line: the first and last dancer swap places, then the second and second-to-last swap, and so on — until everyone is in reversed order. No one leaves the dance floor; they just swap positions. That's Reverse Deque with the two-pointer technique.

SUMMARY Key Takeaways

9 O(1) Operations

Insert, Delete, Peek, isEmpty, isFull, Size, Clear, and Update are all constant time — the fastest possible.

3 O(n) Operations

Traverse, Search, and Reverse must visit multiple elements. Their time grows linearly with deque size.

O(1) Extra Space

All operations use only a constant amount of extra variables — no extra arrays needed (Reverse is in-place).

Fixed Capacity

Array-based deque cannot grow dynamically. For variable size, use a linked-list based deque.

#define MAX 8

int arr[MAX];
int front = -1, rear = -1;

int  isEmpty()  { return front == -1; }
int  isFull()   { return (front==0&&rear==MAX-1)||(front==rear+1); }
int  size()     {
    if(isEmpty()) return 0;
    return rear>=front ? rear-front+1 : MAX-front+rear+1;
}

void insertFront(int v) {
    if(isFull()){ puts("Overflow!"); return; }
    if(front==-1){ front=rear=0; }
    else if(front==0) front=MAX-1;
    else front--;
    arr[front]=v;
}
void insertRear(int v) {
    if(isFull()){ puts("Overflow!"); return; }
    if(rear==-1){ front=rear=0; }
    else if(rear==MAX-1) rear=0;
    else rear++;
    arr[rear]=v;
}
int deleteFront() {
    if(isEmpty()){ puts("Underflow!"); return -1; }
    int v=arr[front];
    if(front==rear){ front=rear=-1; }
    else if(front==MAX-1) front=0;
    else front++;
    return v;
}
int deleteRear() {
    if(isEmpty()){ puts("Underflow!"); return -1; }
    int v=arr[rear];
    if(front==rear){ front=rear=-1; }
    else if(rear==0) rear=MAX-1;
    else rear--;
    return v;
}
int  getFront()  { return isEmpty() ? -1 : arr[front]; }
int  getRear()   { return isEmpty() ? -1 : arr[rear]; }
void clearDeque(){ front=rear=-1; }

void display() {
    if(isEmpty()){ puts("Empty"); return; }
    int i=front;
    printf("[ ");
    while(1){ printf("%d ",arr[i]); if(i==rear)break; i=(i+1)%MAX; }
    printf("]\n");
}
int search(int t) {
    if(isEmpty()) return -1;
    int i=front,p=0;
    while(1){ if(arr[i]==t)return p; if(i==rear)break; i=(i+1)%MAX; p++; }
    return -1;
}
void update(int pos, int v) {
    if(pos<0||pos>=size()){ puts("Invalid pos"); return; }
    arr[(front+pos)%MAX]=v;
}
void reverseDeque() {
    int l=front,r=rear,c=size()/2,i,t;
    for(i=0;i<c;i++){
        t=arr[l]; arr[l]=arr[r]; arr[r]=t;
        l=(l+1)%MAX; r=(r-1+MAX)%MAX;

    insertFront(5);

    printf("Size:  %d\n", size());
    reverseDeque();
    display();       /* [ 30 20 10 5 ] */

}
