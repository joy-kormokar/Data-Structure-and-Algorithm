// 01 · Introduction

What is a Connected Graph?

graph theory foundations

🔗
Graph Fundamentals
THEORY

Theory

A graph is a collection of vertices (nodes) and edges (connections between nodes).

A graph is called connected if there is a path between every pair of vertices — no vertex is isolated from the rest.

Key terminology:

  • Vertex (Node) — a single point/entity in the graph
  • Edge — a link between two vertices
  • Adjacent — two vertices connected by an edge
  • Degree — number of edges at a vertex
  • Path — sequence of vertices connected by edges
  • Directed — edges have a direction (A→B ≠ B→A)
  • Undirected — edges go both ways (A–B = B–A)
Real-life Analogy

Think of a city road network. Each intersection is a vertex, each road is an edge. A connected graph means you can drive from any intersection to any other — no dead-end isolated islands.

// 02 · Representation

Adjacency Matrix

how we store a graph in memory

🗃️
Adjacency Matrix Representation
FUNDAMENTAL

Theory

An adjacency matrix is a 2D array of size V × V, where V is the number of vertices.

matrix[i][j] = 1 means there is an edge from vertex i to vertex j. Otherwise it's 0.

For undirected graphs, the matrix is always symmetric: if matrix[i][j] = 1 then matrix[j][i] = 1.

For weighted graphs, store the weight instead of 1.

Space: O(V²)
Edge check: O(1)
Add vertex: O(V²)
Real-life Analogy

Imagine a contact table where rows and columns are people. A "1" means those two people are friends. A "0" means they're not connected. This friendship table is exactly an adjacency matrix!

/* Adjacency Matrix in C */
#define MAX 10

struct Graph {
    int vertices;
    int adj[MAX][MAX];
};

/* Initialize empty graph */
void initGraph(struct Graph *g, int v) {
    g->vertices = v;
    for (int i = 0; i < v; i++)
        for (int j = 0; j < v; j++)
            g->adj[i][j] = 0;
}
// 03 · Vertex Operations

Vertex Operations

insert · delete · search · update · count · display

➕
Insert Vertex
VERTEX OP

Theory

Adding a new vertex expands the graph. In an adjacency matrix, we increase the vertex count and initialize all new row/column cells to 0 (no edges yet).

The new vertex starts isolated — it has no connections until edges are inserted.

Analogy

Adding a new person to a contact list. They exist in the table but have no friends yet — all their row and column entries are 0.

C Code

/* Insert a new vertex */
int insertVertex(struct Graph *g) {
    if (g->vertices >= MAX) {
        printf("Graph full!\n");
        return -1;
    }
    int v = g->vertices; /* new vertex index */
    g->vertices++;

    /* Initialize new row and column to 0 */
    for (int i = 0; i < g->vertices; i++) {
        g->adj[v][i] = 0;
        g->adj[i][v] = 0;
    }
    printf("Vertex %d inserted.\n", v);
    return v;
}
Time: O(V)
Space: O(1)
🗑️
Delete Vertex
VERTEX OP

Theory

Deleting a vertex removes it and all its edges. In an adjacency matrix, we shift all rows and columns after the deleted vertex one step left/up to fill the gap.

  1. Remove the row of the deleted vertex
  2. Remove the column of the deleted vertex
  3. Shift remaining rows/columns up/left
  4. Decrement vertex count
Analogy

Removing a person from the contact table. Every row and column entry for that person disappears, and the table shrinks.

C Code

void deleteVertex(struct Graph *g, int v) {
    if (v >= g->vertices) {
        printf("Invalid vertex!\n");
        return;
    }
    /* Shift rows up */
    for (int i = v; i < g->vertices - 1; i++)
        for (int j = 0; j < g->vertices; j++)
            g->adj[i][j] = g->adj[i+1][j];

    /* Shift columns left */
    for (int i = 0; i < g->vertices - 1; i++)
        for (int j = v; j < g->vertices - 1; j++)
            g->adj[i][j] = g->adj[i][j+1];

    g->vertices--;
    printf("Vertex %d deleted.\n", v);
}
Time: O(V²)
🔍
Search Vertex
VERTEX OP

Theory

Searching for a vertex checks whether a given vertex index exists in the graph. Since vertices are indexed 0 to V-1, a simple range check suffices.

Analogy

Looking up if "Person 3" exists in your contact table. If the table has 5 people (0–4), then Person 3 exists. Person 7 does not.

C Code

int searchVertex(struct Graph *g, int v) {
    if (v >= 0 && v < g->vertices) {
        printf("Vertex %d found.\n", v);
        return 1; /* found */
    }
    printf("Vertex %d not found.\n", v);
    return 0; /* not found */
}
Time: O(1)

🔎
Search Edge
EDGE OP

Theory

Checking if an edge exists between vertices u and v is one of the biggest advantages of adjacency matrices — it's an instant O(1) lookup!

Just check: matrix[u][v] == 1

Analogy

Looking up if Alice and Bob are friends in the table. Jump directly to row "Alice", column "Bob" — if it's 1, they're connected. Instant!

C Code

int searchEdge(struct Graph *g,
               int u, int v)
{
    if (u >= g->vertices ||
        v >= g->vertices) {
        printf("Invalid vertex!\n");
        return -1;
    }
    if (g->adj[u][v] == 1) {
        printf("Edge %d-%d EXISTS\n",u,v);
        return 1;
    }
    printf("Edge %d-%d NOT found\n",u,v);
    return 0;
}
/* Time complexity: O(1) — fastest! */
Time: O(1)

Count Edges

Count all 1s in the upper triangle of the matrix (for undirected), or the entire matrix (for directed).

int countEdges(
    struct Graph *g)
{
    int count = 0;
    for(int i=0;i<g->vertices;i++)
      for(int j=i+1;j<g->vertices;j++)
        if(g->adj[i][j])
          count++;
    return count;
}

Print all existing edges by scanning the upper triangle of the adjacency matrix.

{

   for(int j=i+1;j<g->vertices;j++)
    if(g->adj[i][j])
     printf("%d -- %d\n",i,j);
}
// 05 · Traversal

Breadth First Search (BFS)

queue initialization · visit node · mark visited · traverse adjacent

🌊
BFS — Breadth First Search
TRAVERSAL

Theory

BFS explores the graph level by level — first visiting all neighbors of the starting vertex, then their neighbors, and so on.

It uses a Queue (FIFO) to keep track of which vertices to visit next.

  1. Enqueue the start vertex, mark as visited
  2. Dequeue a vertex, process it
  3. Enqueue all unvisited neighbors of that vertex
  4. Repeat until queue is empty
Time: O(V²)
Space: O(V)
Analogy

Spreading news in a social network. You tell your friends (level 1), they tell their friends (level 2), and so on — rippling outward like waves on water.

BFS Animation

#define MAX 10
int queue[MAX], front = -1, rear = -1;
int visited[MAX];

void BFS(struct Graph *g, int start) {
    /* Init visited array */
    for(int i=0;i<g->vertices;i++)
        visited[i] = 0;

    /* Enqueue start vertex */
    queue[++rear] = start;
    visited[start] = 1;

    while(front != rear) {
        int v = queue[++front]; /* dequeue */
        printf("Visited: %d\n", v);

        /* Enqueue unvisited neighbors */
        for(int j=0;j<g->vertices;j++) {
            if(g->adj[v][j]==1 &&
               !visited[j]) {
                queue[++rear] = j;
                visited[j] = 1;
            }
        }
    }
}
🔬
DFS — Depth First Search (Recursive + Stack)
TRAVERSAL

Theory

DFS explores the graph by going as deep as possible along each branch before backtracking.

Two implementations:

  • Recursive DFS — uses the call stack for backtracking (simpler code)
  • Stack-based DFS — uses an explicit stack (avoids stack overflow on large graphs)
Time: O(V²)
Space: O(V)
Analogy

Exploring a maze: go as far down one path as possible. When you hit a dead end, backtrack and try another path. You explore one "tunnel" fully before starting another.

DFS Animation

Press ▶ Run DFS to start from vertex 0.
int visited[MAX];

/* Recursive DFS */
void DFS_recursive(struct Graph *g,
                    int v)
{
    visited[v] = 1;
    printf("Visited: %d\n", v);

    for(int j=0;j<g->vertices;j++) {
        if(g->adj[v][j]==1 &&
           !visited[j])
            DFS_recursive(g, j);
    }
}

/* Stack-based DFS */
void DFS_stack(struct Graph *g,
               int start)
{
    int stack[MAX], top = -1;
    stack[++top] = start;
    visited[start] = 1;

    while(top >= 0) {
        int v = stack[top--]; /* pop */
        printf("Visited: %d\n", v);

        for(int j=0;j<g->vertices;j++)
            if(g->adj[v][j] &&
               !visited[j]) {
                stack[++top] = j;
                visited[j] = 1;
            }
    }
}
// 06 · Connectivity Operations

Connectivity Operations

connected check · strong · weak · reachable · validation

🔗
Check Connected Graph
CONNECTIVITY

Theory

A graph is connected if starting a BFS or DFS from any vertex visits all vertices.

Algorithm: Run DFS from vertex 0. If the visited count equals total vertices → connected. Otherwise → disconnected.

  1. Initialize visited array to all 0
  2. Run DFS from vertex 0
  3. Count visited vertices
  4. If count == total vertices → connected
Analogy

Can you travel from any city to every other city in a road network? If yes, the network is connected. If some cities are unreachable, it's disconnected.

C Code

void dfsHelper(struct Graph *g,
               int v,int *vis) {
    vis[v] = 1;
    for(int j=0;j<g->vertices;j++)
        if(g->adj[v][j] && !vis[j])
            dfsHelper(g,j,vis);
}

int isConnected(struct Graph *g) {
    int vis[MAX] = {0};
    dfsHelper(g, 0, vis);

    for(int i=0;i<g->vertices;i++)
        if(!vis[i]) {
            printf("DISCONNECTED\n");
            return 0;
        }
    printf("CONNECTED\n");
    return 1;
}
Time: O(V²)
💪
Strong Connectivity · Weak Connectivity · Find Reachable Vertices
CONNECTIVITY

Strong Connectivity

A directed graph is strongly connected if there is a path from every vertex to every other vertex.

Algorithm: Run DFS from any vertex in original graph, then run DFS on the transposed (reversed) graph. If both cover all vertices → strongly connected.

/* Transpose the matrix */
void transpose(
    struct Graph *g,
    int t[MAX][MAX])
{
  for(int i=0;i<g->vertices;i++)
   for(int j=0;j<g->vertices;j++)
    t[i][j] = g->adj[j][i];
}

Weak Connectivity

A directed graph is weakly connected if treating all directed edges as undirected makes the graph connected.

Weaker condition: you don't need bidirectional paths, just an underlying connected structure.

/* Check weak: symmetrize
   then check connectivity */
int isWeaklyConnected(
    struct Graph *g)
{
  int sym[MAX][MAX];
  for(int i=0;i<g->vertices;i++)
   for(int j=0;j<g->vertices;j++)
    sym[i][j]=g->adj[i][j]
             |g->adj[j][i];
  /* run DFS on sym matrix */
}

Find Reachable Vertices

From a given source vertex, find all vertices that can be reached by following edges. Uses BFS or DFS and collects all visited vertices.

void findReachable(
    struct Graph *g,
    int src)
{
  int vis[MAX]={0};
  dfsHelper(g,src,vis);
  printf("Reachable from %d:\n",src);
  for(int i=0;i<g->vertices;i++)
    if(vis[i])
      printf("  %d\n",i);
}
Analogy

Strong: A city network where you can drive in both directions — from A to B AND from B to A. Weak: You can reach everywhere if you ignore one-way streets. Reachable: From your home, which places can you visit today?

CONNECTIVITY

Count Connected Components

A component is a maximal connected subgraph. Run DFS from every unvisited vertex — each DFS call that starts fresh discovers one new component.

int countComponents(
    struct Graph *g)
{
  int vis[MAX]={0},cnt=0;
  for(int i=0;i<g->vertices;i++)
    if(!vis[i]) {
      dfsHelper(g,i,vis);
      cnt++;
    }
  printf("Components: %d\n",cnt);
  return cnt;
}

Detect Isolated Vertex

An isolated vertex has no edges — its entire row in the adjacency matrix is all zeros. Check by summing the row.

int isIsolated(
    struct Graph *g,
    int v)
{
  for(int j=0;j<g->vertices;j++)
    if(g->adj[v][j])
      return 0; /* has edge */
  printf("V%d is isolated\n",v);
  return 1;
}

Connectivity Restoration

If the graph is disconnected, restore connectivity by adding minimum edges to connect isolated components — finding the "bridge" vertices between components.

/* Connect components by adding
   edges between their first 
   vertices */
void restoreConn(
    struct Graph *g)
{
  int vis[MAX]={0},last=-1;
  for(int i=0;i<g->vertices;i++)
    if(!vis[i]) {
      dfsHelper(g,i,vis);
      if(last>=0)
        insertEdge(g,last,i);
      last=i;
    }
}
// 07 · Path Operations

Path Operations

find path · shortest path · all paths · reachability · path length

🛤️
Find Path · Shortest Path (BFS)
PATH

Theory

Find Path: Check if any path exists between source and destination using DFS. Track the path using a parent array.

Shortest Path: In an unweighted graph, BFS naturally finds the shortest path because it explores level by level. Level = distance from source.

  1. Run BFS from source
  2. Track parent of each visited vertex
  3. When destination is found, trace back via parent array
  4. Print path in reverse
Find Path: O(V²)
Shortest (BFS): O(V²)
Analogy

GPS navigation: Find Path = "Can I get to the airport?" Shortest Path = "What is the quickest route to the airport with fewest turns?"

C Code — Shortest Path with BFS

void shortestPath(struct Graph *g,
                   int src, int dest)
{
    int vis[MAX]={0};
    int parent[MAX], dist[MAX];
    int queue[MAX], front=-1, rear=-1;

    for(int i=0;i<g->vertices;i++)
        parent[i]=-1, dist[i]=-1;

    queue[++rear]=src;
    vis[src]=1; dist[src]=0;

    while(front!=rear) {
        int v=queue[++front];
        for(int j=0;j<g->vertices;j++)
            if(g->adj[v][j]&&!vis[j]) {
                queue[++rear]=j;
                vis[j]=1;
                parent[j]=v;
                dist[j]=dist[v]+1;
            }
    }
    /* Trace back path */
    if(dist[dest]==-1) {
        printf("No path!\n"); return;
    }
    printf("Dist: %d\n",dist[dest]);
    int path[MAX], idx=0, cur=dest;
    while(cur!=-1)
        path[idx++]=cur, cur=parent[cur];
    for(int k=idx-1;k>=0;k--)
        printf("%d ",path[k]);
}

All Paths · Reachability Check · Path Length Count
PATH

All Paths

Find every possible path between source and destination using recursive DFS with backtracking. Mark a vertex visited before recursing, unmark after returning.

int path[MAX], idx=0;

void allPaths(
    struct Graph *g,
    int v,int dest,
    int *vis)
{
  vis[v]=1;
  path[idx++]=v;
  if(v==dest) {
    for(int k=0;k<idx;k++)
      printf("%d ",path[k]);
    printf("\n");
  } else
    for(int j=0;j<g->vertices;j++)
      if(g->adj[v][j]&&!vis[j])
        allPaths(g,j,dest,vis);
  idx--; vis[v]=0; /* backtrack */
}

Cycle Operations

🔄
Detect Cycle · Count Cycles · Check Cyclic Structure · Remove Cycle
CYCLE

Detect Cycle

A cycle exists if during DFS we encounter an already-visited vertex that isn't the parent (for undirected) or a back edge (for directed).

For undirected graphs: Use DFS, pass the parent. If a neighbor is visited and not the parent → cycle detected.

/* Detect cycle - undirected */
int hasCycleDFS(struct Graph *g,
               int v,int parent,
               int *vis)
{
  vis[v] = 1;
  for(int j=0;j<g->vertices;j++) {
    if(g->adj[v][j]) {
      if(!vis[j]) {
        if(hasCycleDFS(g,j,v,vis))
          return 1;
      } else if(j != parent)
          return 1; /* cycle! */
    }
  }
  return 0;
}

int detectCycle(struct Graph *g) {
  int vis[MAX]={0};
  for(int i=0;i<g->vertices;i++)
    if(!vis[i])
      if(hasCycleDFS(g,i,-1,vis))
        { printf("Cycle found!\n");
          return 1; }
  printf("No cycle.\n");
  return 0;
}

Remove Cycle / Count Cycles

Remove Cycle: Find the back edge causing the cycle and delete it. This makes the graph a tree (acyclic).

Count Cycles: Complex — uses the formula: Cycles = Edges - Vertices + Connected_Components (for undirected simple graphs).

Formula: C = E - V + CC

/* Count independent cycles
   using: C = E - V + CC
   (Circuit Rank / Cyclomatic) */
int countCycles(
    struct Graph *g)
{
  int E = countEdges(g);
  int V = g->vertices;
  int CC = countComponents(g);
  int cycles = E - V + CC;
  printf("Independent cycles: %d\n",
         cycles);
  return cycles;
}

/* Remove cycle: delete the 
   back-edge found during DFS */
void removeCycleEdge(
    struct Graph *g,
    int u, int v)
{
  g->adj[u][v] = 0;
  g->adj[v][u] = 0;
}
Analogy

A cycle is like a roundabout on a road network — you can loop endlessly. Removing a cycle edge breaks the loop and turns it into a dead-end road instead.

// 09 · Degree Operations

Degree Operations

find degree · in-degree · out-degree · max · min

📊
Find Degree · In-Degree · Out-Degree · Maximum · Minimum
DEGREE

Theory

The degree of a vertex is the number of edges connected to it.

  • Degree (undirected) — sum of row (or column) in matrix
  • In-degree (directed) — sum of the column (edges coming in)
  • Out-degree (directed) — sum of the row (edges going out)
  • Max degree — vertex with the most connections
  • Min degree — vertex with the fewest connections
Single degree: O(V)
Max/Min: O(V²)
Analogy

In a social network: degree = number of friends. In-degree = how many people follow you. Out-degree = how many people you follow. Max degree = the most popular person.

C Code

/* Degree for undirected graph */
int getDegree(struct Graph *g,
              int v)
{
  int deg=0;
  for(int j=0;j<g->vertices;j++)
    deg += g->adj[v][j];
  return deg;
}

/* In-degree (directed) */
int inDegree(struct Graph *g,
             int v) {
  int deg=0;
  for(int i=0;i<g->vertices;i++)
    deg += g->adj[i][v];
  return deg;
}

/* Out-degree (directed) */
int outDegree(struct Graph *g,
              int v) {
  int deg=0;
  for(int j=0;j<g->vertices;j++)
    deg += g->adj[v][j];
  return deg;
}

/* Maximum degree vertex */
int maxDegreeVertex(
    struct Graph *g)
{
  int maxD=-1, maxV=0;
  for(int i=0;i<g->vertices;i++) {
    int d=getDegree(g,i);
    if(d>maxD) { maxD=d; maxV=i; }
  }
  printf("Max degree: V%d (%d)\n",
         maxV,maxD);
  return maxV;
}

/* Minimum degree vertex */
int minDegreeVertex(
    struct Graph *g)
{
  int minD=9999, minV=0;
  for(int i=0;i<g->vertices;i++) {
    int d=getDegree(g,i);
    if(d<minD){ minD=d; minV=i; }
  }
  printf("Min degree: V%d (%d)\n",
         minV,minD);
  return minV;
}

STRUCTURAL

Graph Size

The size of a graph is the number of edges.

/* Size = edge count */
int graphSize(
    struct Graph *g)
{
  return countEdges(g);
}

Graph Order

The order of a graph is the number of vertices.

/* Order = vertex count */
int graphOrder(
    struct Graph *g)
{
  return g->vertices;
}

Density

Density = actual edges / maximum possible edges. Max edges for undirected = V*(V-1)/2.

float density(
    struct Graph *g)
{
  int V=g->vertices;
  int E=countEdges(g);
  float maxE=V*(V-1)/2.0;
  return E/maxE;
}

Sparse/Dense

Sparse: density < 0.5. Dense: density ≥ 0.5. Adjacency matrix is efficient for dense graphs.

void checkDense(
    struct Graph *g)
{
  float d=density(g);
  if(d>=0.5)
    printf("Dense\n");
  else
    printf("Sparse\n");
}
Analogy

Order = number of cities. Size = number of roads. Dense = a city center where almost every street connects to every other (like Manhattan). Sparse = rural areas with few roads between towns.

Complete Graph Check

A graph is complete (K_n) if every vertex is directly connected to every other vertex. Total edges = V*(V-1)/2 for undirected.

Simply check if actual edges == V*(V-1)/2.

Analogy

A group where everyone knows everyone else directly — no introductions needed. A complete friend network with direct connections for all pairs.

Tree Validation + Complete Graph

/* Complete graph: edges = V*(V-1)/2 */
int isComplete(struct Graph *g) {
    int V = g->vertices;
    int E = countEdges(g);
    if(E == V*(V-1)/2) {
        printf("Complete graph K%d\n",V);
        return 1;
    }
    printf("Not complete.\n");
    return 0;
}

/* Tree: connected + no cycles
   A tree has exactly V-1 edges */
int isTree(struct Graph *g) {
    int V = g->vertices;
    int E = countEdges(g);

    /* Must have exactly V-1 edges */
    if(E != V - 1) {
        printf("Not a tree.\n");
        return 0;
    }
    /* Must be connected */
    if(!isConnected(g)) {
        printf("Not a tree.\n");
        return 0;
    }
    printf("Valid tree!\n");
    return 1;
}

Minimum Spanning Tree (MST) — Overview
ADVANCED

Theory

A Minimum Spanning Tree is a subset of edges that connects all vertices in a weighted graph with the minimum total edge weight, without any cycles.

Properties of MST:

  • Contains exactly V - 1 edges
  • Connects all V vertices (spanning)
  • Has minimum total weight (minimum)
  • Has no cycles (tree)
Analogy

You want to lay cables to connect 5 cities with minimum total cable length. The MST tells you which roads to lay cables along so every city gets connected using the least total wire.

Kruskal's Algorithm
MST

Theory

Kruskal's algorithm builds the MST by greedily picking the smallest weight edge that doesn't create a cycle.

  1. Extract all edges and sort by weight (ascending)
  2. Pick the smallest edge
  3. If adding it doesn't create a cycle → add to MST
  4. Use Union-Find to detect cycles efficiently
  5. Repeat until V-1 edges added
Time: O(E log E)
Best for: Sparse graphs

C Code — Kruskal's

struct Edge { int u,v,w; };

int parent[MAX];
int find(int x) {
    if(parent[x]!=x)
        parent[x]=find(parent[x]);
    return parent[x];
}
void unite(int a, int b) {
    parent[find(a)] = find(b);
}

/* Compare edges by weight (for sort) */
int cmp(const void *a,
        const void *b) {
    return ((struct Edge*)a)->w -
           ((struct Edge*)b)->w;
}

void kruskal(struct Graph *g,
              int weight[MAX][MAX])
{
    struct Edge edges[MAX*MAX];
    int eCount=0, V=g->vertices;

    /* Extract all edges */
    for(int i=0;i<V;i++)
      for(int j=i+1;j<V;j++)
        if(g->adj[i][j])
          edges[eCount++]=((struct Edge)
            {i,j,weight[i][j]});

    qsort(edges,eCount,
          sizeof(struct Edge),cmp);

    for(int i=0;i<V;i++) parent[i]=i;

    int mstW=0, edgesUsed=0;
    for(int i=0;i<eCount;i++) {
        int u=edges[i].u, v=edges[i].v;
        if(find(u)!=find(v)) {
            unite(u,v);
            mstW+=edges[i].w;
            printf("%d-%d w=%d\n",
                   u,v,edges[i].w);
            if(++edgesUsed==V-1)
                break;
        }
    }
    printf("MST weight: %d\n",mstW);
}
🌿
Prim's Algorithm
MST

Theory

Prim's algorithm grows the MST one vertex at a time. Start with any vertex, always add the cheapest edge that connects a vertex outside the MST to one inside it.

  1. Start with vertex 0 in the MST set
  2. Find the minimum weight edge from MST set to non-MST set
  3. Add that edge and vertex to the MST
  4. Repeat V-1 times
Time: O(V²)
Best for: Dense graphs
Analogy

Building a phone network city by city: start from one city, always connect the nearest unconnected city next. Grow the network outward from your starting point.

C Code — Prim's

void prim(struct Graph *g,
          int weight[MAX][MAX])
{
    int V=g->vertices;
    int key[MAX],parent[MAX];
    int inMST[MAX];

    for(int i=0;i<V;i++) {
        key[i]=99999;
        inMST[i]=0;
        parent[i]=-1;
    }
    key[0]=0; /* Start vertex 0 */

    for(int count=0;count<V-1;count++) {
        /* Pick min key not in MST */
        int u=-1;
        for(int i=0;i<V;i++)
          if(!inMST[i]&&
             (u==-1||key[i]<key[u]))
            u=i;

        inMST[u]=1;

        /* Update neighbors */
        for(int v=0;v<V;v++)
          if(g->adj[u][v] &&
             !inMST[v] &&
             weight[u][v]<key[v]) {
            key[v]=weight[u][v];
            parent[v]=u;
          }
    }
    /* Print MST edges */
    int total=0;
    for(int i=1;i<V;i++) {
      printf("%d-%d w=%d\n",
             parent[i],i,key[i]);
      total+=key[i];
    }
    printf("MST total: %d\n",total);
}

Strongly Connected Components (Kosaraju's)

SCCs are maximal groups where every vertex can reach every other vertex within the group (directed graph).

Kosaraju's Algorithm:

  1. Run DFS on original graph, push vertices to a stack in finish order
  2. Transpose (reverse) all edges
  3. Pop from stack, run DFS on transposed graph — each DFS tree is one SCC
Time: O(V²)
Analogy

Cliques on social media — groups where everyone follows everyone else. Each clique (mutual follow group) is a strongly connected component.

Graph Coloring (Greedy)

/* Graph Coloring: assign minimum
   colors so no adjacent vertices
   share a color */
void greedyColoring(
    struct Graph *g)
{
    int V=g->vertices;
    int color[MAX];
    int used[MAX]; /* temp array */

    color[0]=0; /* First vertex gets 0 */
    for(int i=1;i<V;i++)
        color[i]=-1;

    for(int u=1;u<V;u++) {
        /* Mark colors used by neighbors */
        for(int c=0;c<V;c++)
            used[c]=0;
        for(int v=0;v<V;v++)
            if(g->adj[u][v]&&color[v]!=-1)
                used[color[v]]=1;

        /* Assign smallest unused color */
        int c;
        for(c=0;c<V;c++)
            if(!used[c]) break;
        color[u]=c;
    }

    printf("Vertex colors:\n");
    for(int i=0;i<V;i++)
        printf("  V%d → Color %d\n",
               i,color[i]);
}
