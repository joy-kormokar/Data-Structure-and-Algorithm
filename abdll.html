Basic Operations

Foundation

An array-based doubly linked list stores node data inside a fixed-size array. Each element holds a data value, an index pointing to the next node, and an index pointing to the previous node — instead of memory pointers, we use array indices. The special value -1 means "no link" (like NULL in pointer-based lists).

What is a Doubly Linked List?

A linked list where each node connects to both its neighbor in front and behind. This lets us travel in both directions — forward and backward — unlike singly linked lists.

Why Array-Based?

Instead of dynamic memory allocation (malloc/free), nodes live in a fixed array. The array indices act as "pointers". This avoids memory fragmentation and is cache-friendly.

Node Structure (The Building Block)

node_structure.cC
/* Maximum number of nodes our list can hold */
#define MAX 10

/* Each node has 3 fields: data, next-index, prev-index */
struct Node {
    int data;   /* The value stored in this node        */
    int next;   /* Index of next node (-1 = no next)   */
    int prev;   /* Index of prev node (-1 = no prev)   */
};

/* The whole list lives in a single array! */
struct Node list[MAX];
int head = -1;  /* Index of first node (-1 = empty)    */
int size =  0;  /* How many nodes are currently used   */

void initList() {
    int i;

Insertion Operations

4 Methods

Insertion adds a new node into the list. In an array-based DLL, we must find an unused array slot, put the data there, then update the next and prev links of the neighboring nodes.

💡 Golden Rule: Before inserting, always check if the list is full using isFull(). If size == MAX, no more nodes can be added.

2.1 Insert at Front

The new node becomes the new head. Its next points to the old head. The old head's prev now points back to the new node.

insert_front.cC
void insertFront(int value) {
    if (isFull()) { printf("List is full!\n"); return; }

    int slot = size;           /* Use next available slot */
    list[slot].data = value;

    if (head == -1) {         /* List was empty */
        list[slot].next = -1;
        list[slot].prev = -1;
        head = slot;
    } else {                  /* List had nodes */
        list[slot].next = head; /* New → old head */
        list[slot].prev = -1; /* New has no prev */
        list[head].prev = slot; /* Old head ← New */
        head = slot;            /* New is now head */
    }
    size++;
    printf("Inserted %d at front.\n", value);
}

2.2 Insert at Rear

insert_rear.cC
void insertRear(int value) {
    if (isFull()) { printf("List is full!\n"); return; }

    int slot = size;
    list[slot].data = value;
    list[slot].next = -1;  /* Rear has no next */

    if (head == -1) {      /* Empty list */
        list[slot].prev = -1;
        head = slot;
    } else {
        /* Walk to the last node */
        int cur = head;
        while (list[cur].next != -1)
            cur = list[cur].next;

        list[cur].next  = slot; /* Last → New      */
        list[slot].prev = cur;  /* New ← Last      */
    }
    size++;
    printf("Inserted %d at rear.\n", value);
}

2.3 Insert Before Position & 2.4 Insert After Position

insert_position.cC
/* Insert AFTER the node at given position (0-based) */
void insertAfter(int pos, int value) {
    if (isFull()) { printf("Full!\n"); return; }
    if (pos < 0 || pos >= size) { printf("Bad position!\n"); return; }

    /* Step 1: Navigate to the target position */
    int cur = head, i;
    for (i = 0; i < pos; i++) cur = list[cur].next;

    /* Step 2: Put new node in a free slot */
    int slot = size;
    list[slot].data = value;

    /* Step 3: Link new node between cur and cur's next */
    list[slot].next = list[cur].next;
    list[slot].prev = cur;

    if (list[cur].next != -1)
        list[list[cur].next].prev = slot; /* next ← new */

    list[cur].next = slot;  /* cur → new */
    size++;
}

/* Insert BEFORE: navigate to pos, then insertAfter(pos-1) */
void insertBefore(int pos, int value) {
    if (pos == 0) { insertFront(value); return; }
    insertAfter(pos - 1, value);
}

Deletion Operations

4 Methods

Deletion removes a node from the list and re-connects the neighboring nodes so the list stays intact. In array-based DLL, we don't free memory — instead we "shift" the remaining nodes to keep the array compact (or mark slots as free).

⚠️ Always check: call isEmpty() before any deletion. Deleting from an empty list causes logical errors.

3.1 Delete Front Element

delete_front.cC
void deleteFront() {
    if (isEmpty()) { printf("List is empty!\n"); return; }

    int deleted_val = list[head].data;
    int new_head    = list[head].next;

    if (new_head != -1)
        list[new_head].prev = -1; /* New head has no prev */

    /* Shift array: move each node one index back */
    int i, j;
    for (i = head; i < size - 1; i++) {
        list[i] = list[i + 1];
        /* Fix references that pointed to i+1 */
        if (list[i].next != -1) list[list[i].next].prev = i;
        if (list[i].prev != -1) list[list[i].prev].next = i;
    }
    head = (size > 1) ? 0 : -1;
    size--;
    printf("Deleted %d from front.\n", deleted_val);
}

3.2 Delete Rear Element

delete_rear.cC
void deleteRear() {
    if (isEmpty()) { printf("List is empty!\n"); return; }

    /* Walk to the last node */
    int cur = head;
    while (list[cur].next != -1)
        cur = list[cur].next;

    int prev_idx = list[cur].prev;
    if (prev_idx != -1)
        list[prev_idx].next = -1; /* Second-to-last now ends the list */
    else
        head = -1;                  /* List had only one node */

    size--;
    printf("Deleted rear node.\n");
}

3.3 Delete by Value & 3.4 Delete Specific Position

delete_value_pos.cC
/* Delete first node that has matching value */
void deleteByValue(int value) {
    int cur = head;
    while (cur != -1) {
        if (list[cur].data == value) {
            /* Unlink this node */
            if (list[cur].prev != -1)
                list[list[cur].prev].next = list[cur].next;
            else
                head = list[cur].next; /* Was the head */

            if (list[cur].next != -1)
                list[list[cur].next].prev = list[cur].prev;

            size--;
            printf("Deleted node with value %d.\n", value);
            return;
        }
        cur = list[cur].next;
    }
    printf("Value %d not found!\n", value);
}

/* Delete node at specific position (0-based) */
void deletePosition(int pos) {
    if (pos == 0) { deleteFront(); return; }
    int cur = head, i;
    for (i = 0; i < pos && cur != -1; i++) cur = list[cur].next;
    if (cur == -1) { printf("Position out of range!\n"); return; }
    deleteByValue(list[cur].data); /* Reuse deleteByValue */
}

Traversal Operations

3 Methods

Traversal means visiting every node in the list, one by one. The superpower of a doubly linked list is that we can go both ways — forward from head, or backward from tail.

Forward Traversal

Start at head, keep following .next until you reach -1. This visits every node left to right.

Backward Traversal

First find the tail (last node), then keep following .prev until you reach -1. Visits right to left.

traversal.cC
/* 4.1 Forward Traversal — head → tail */
void forwardTraverse() {
    if (isEmpty()) { printf("List is empty!\n"); return; }
    int cur = head;
    printf("Forward: ");
    while (cur != -1) {
        printf("%d", list[cur].data);
        if (list[cur].next != -1) printf(" ↔ ");
        cur = list[cur].next;       /* Move to next node */
    }
    printf("\n");
}

/* 4.2 Backward Traversal — tail → head */
void backwardTraverse() {
    if (isEmpty()) { printf("List is empty!\n"); return; }

    /* Step 1: Go to the very last node */
    int cur = head;
    while (list[cur].next != -1) cur = list[cur].next;

    /* Step 2: Walk backward using .prev */
    printf("Backward: ");
    while (cur != -1) {
        printf("%d", list[cur].data);
        if (list[cur].prev != -1) printf(" ↔ ");
        cur = list[cur].prev;       /* Move to previous node */
    }
    printf("\n");
}

/* 4.3 Loop-Based Traversal (count iterations) */
void loopTraverse() {
    int cur = head, step = 0;
    while (cur != -1) {
        printf("Step %d → data=%d\n", step, list[cur].data);
        cur = list[cur].next;
        step++;
    }
}

Searching Operations

3 Methods

Searching finds a specific value (or position) in the list. With a doubly linked list, we can search from either end, which can be faster if we know roughly where the item is.

search.cC
/* 5.1 Forward Search — search from head toward tail */
int forwardSearch(int value) {
    int cur = head, pos = 0;
    while (cur != -1) {
        if (list[cur].data == value) {
            printf("Found %d at position %d\n", value, pos);
            return pos;
        }
        cur = list[cur].next;
        pos++;
    }
    printf("%d not found!\n", value);
    return -1;
}

/* 5.2 Backward Search — search from tail toward head */
int backwardSearch(int value) {
    /* First go to the tail */
    int cur = head;
    while (cur != -1 && list[cur].next != -1)
        cur = list[cur].next;

    int pos = size - 1;
    while (cur != -1) {
        if (list[cur].data == value) {
            printf("Found %d at position %d (from rear)\n", value, pos);
            return pos;
        }
        cur = list[cur].prev;
        pos--;
    }
    return -1;
}

/* 5.3 Search by Position — get value at position n */
int searchByPosition(int pos) {
    if (pos < 0 || pos >= size) { printf("Invalid!\n"); return -1; }
    int cur = head, i;
    for (i = 0; i < pos; i++) cur = list[cur].next;
    printf("Position %d has value %d\n", pos, list[cur].data);
    return list[cur].data;
}

Update Operations

3 Methods

Update operations modify existing node data without changing the list structure. The links remain exactly the same — only data values change.

update.cC
/* 6.1 Replace Value — find old_val, replace with new_val */
void replaceValue(int old_val, int new_val) {
    int cur = head;
    while (cur != -1) {
        if (list[cur].data == old_val) {
            list[cur].data = new_val;
            printf("Replaced %d with %d\n", old_val, new_val);
            return;
        }
        cur = list[cur].next;
    }
    printf("%d not found!\n", old_val);
}

/* 6.2 Modify Data — update node at position pos */
void modifyData(int pos, int new_val) {
    if (pos < 0 || pos >= size) { printf("Bad position!\n"); return; }
    int cur = head, i;
    for (i = 0; i < pos; i++) cur = list[cur].next;
    list[cur].data = new_val;
    printf("Position %d updated to %d\n", pos, new_val);
}

/* 6.3 Swap Nodes — exchange data of two positions */
void swapNodes(int pos1, int pos2) {
    int a = head, b = head, i, temp;
    for (i = 0; i < pos1; i++) a = list[a].next;
    for (i = 0; i < pos2; i++) b = list[b].next;
    temp = list[a].data;   /* Classic swap */
    list[a].data = list[b].data;
    list[b].data = temp;
    printf("Swapped positions %d and %d\n", pos1, pos2);
}
📝 Real-Life Analogy — Editing a Spreadsheet

Think of the list as spreadsheet rows. Replace Value finds a cell with a specific number and changes it. Modify Data jumps straight to row N and edits it. Swap Nodes exchanges the contents of two rows — like cut-paste between cells — without moving the rows themselves.

Access Operations

3 Methods

Access operations retrieve values without modifying the list. They are read-only "peek" operations.

access.cC
/* 7.1 Peek Front — see first element without removing */
int peekFront() {
    if (isEmpty()) { printf("Empty!\n"); return -1; }
    printf("Front value: %d\n", list[head].data);
    return list[head].data;
}

/* 7.2 Peek Rear — see last element without removing */
int peekRear() {
    if (isEmpty()) { printf("Empty!\n"); return -1; }
    int cur = head;
    while (list[cur].next != -1) cur = list[cur].next;
    printf("Rear value: %d\n", list[cur].data);
    return list[cur].data;
}

/* 7.3 Get Element by Position */
int getElement(int pos) {
    if (pos < 0 || pos >= size) { printf("Out of bounds!\n"); return -1; }
    int cur = head, i;
    for (i = 0; i < pos; i++) cur = list[cur].next;
    printf("Element at %d = %d\n", pos, list[cur].data);
    return list[cur].data;
}
🍽️ Real-Life Analogy — Restaurant Order Queue

Peek Front is like a kitchen display showing the next order to prepare — you can see it without removing it. Peek Rear is checking the last order placed. Get Element by Position is the manager looking at order number 3 in the queue without disturbing any others.

Structural Operations — Count Elements

3 Methods

These operations measure the list — how many nodes exist, counted from either direction (both should give the same answer for a correct list).

count.cC
/* 8.1 Count Elements — use the size variable (O(1) fast!) */
int countElements() {
    printf("Total elements: %d\n", size);
    return size;
}

/* 8.2 Count Forward — traverse and count (verifies integrity) */
int countForward() {
    int cur = head, count = 0;
    while (cur != -1) {
        count++;
        cur = list[cur].next;
    }
    printf("Forward count: %d\n", count);
    return count;
}

/* 8.3 Count Backward — traverse backward and count */
int countBackward() {
    if (isEmpty()) return 0;
    int cur = head, count = 0;
    /* Go to tail first */
    while (list[cur].next != -1) cur = list[cur].next;
    /* Walk backward counting */
    while (cur != -1) { count++; cur = list[cur].prev; }
    printf("Backward count: %d\n", count);
    return count;
}

/* Integrity check: both counts must match! */
void integrityCheck() {
    int fwd = countForward();
    int bwd = countBackward();
    if (fwd == bwd) printf("✓ List is consistent!\n");
    else           printf("✗ List is CORRUPTED!\n");
}
💡 Pro Tip: Always maintain a size variable updated on every insert/delete. This makes countElements() O(1) — instant! countForward() and countBackward() are O(n) traversals used mainly for debugging.
🧮 Real-Life Analogy — Counting People in a Line

Count Elements is like the queue manager's counter — they always know how many are in line (they update it every time someone joins or leaves). Count Forward is a security guard walking front-to-back counting heads. Count Backward is the guard walking back-to-front. Both should give the same number — if they don't, someone sneaked in!

Link Management Operations

3 Methods

These low-level operations directly manipulate the next and prev indices. They are the building blocks used by insertion, deletion, and other higher-level operations.

link_management.cC
/* 9.1 Update Previous Link — set node[idx].prev = new_prev */
void updatePrevLink(int idx, int new_prev) {
    if (idx < 0 || idx >= size) { printf("Invalid index!\n"); return; }
    list[idx].prev = new_prev;
    printf("Node[%d].prev updated to %d\n", idx, new_prev);
}

/* 9.2 Update Next Link — set node[idx].next = new_next */
void updateNextLink(int idx, int new_next) {
    if (idx < 0 || idx >= size) { printf("Invalid index!\n"); return; }
    list[idx].next = new_next;
    printf("Node[%d].next updated to %d\n", idx, new_next);
}

/* 9.3 Relink Nodes — connect node A → C, bypassing B
   Before: A ↔ B ↔ C
   After:  A ↔ C (B removed from chain)             */
void relinkNodes(int idxA, int idxC) {
    if (idxA != -1) list[idxA].next = idxC; /* A → C */
    if (idxC != -1) list[idxC].prev = idxA; /* A ← C */
    printf("Relinked: node %d now connects to node %d\n", idxA, idxC);
}

/* How relinkNodes is used inside deletion:
   To delete node B between A and C:
   1. idxA = list[B].prev
   2. idxC = list[B].next
   3. relinkNodes(idxA, idxC)    ← skips B       */
🔗 Real-Life Analogy — Changing Contacts in a Phone Chain

Imagine a "phone chain" where each person has the number of the next and previous person to call during an emergency. Update Previous Link is changing who calls you. Update Next Link is changing who you call. Relink Nodes is removing someone from the chain — you now call the person they used to call, and that person now waits for your call instead.

Utility Operations

3 Methods

Utility operations perform structural changes on the whole list — clearing it, making a copy, or reversing the order of nodes.

utility.cC
/* 10.1 Clear List — remove all nodes */
void clearList() {
    int i;
    for (i = 0; i < MAX; i++) {
        list[i].data = 0;
        list[i].next = -1;
        list[i].prev = -1;
    }
    head = -1;
    size = 0;
    printf("List cleared!\n");
}

/* 10.2 Clone List — copy all nodes into a second list */
void cloneList(struct Node dest[], int *dest_head, int *dest_size) {
    int i;
    for (i = 0; i < size; i++) dest[i] = list[i]; /* Copy all slots */
    *dest_head = head;
    *dest_size = size;
    printf("List cloned! %d nodes copied.\n", size);
}

/* 10.3 Reverse List — flip the direction of all links */
void reverseList() {
    int cur = head, temp;
    while (cur != -1) {
        /* Swap next and prev for every node */
        temp           = list[cur].next;
        list[cur].next = list[cur].prev;
        list[cur].prev = temp;

        /* Move to what was next (now stored in temp) */
        cur = temp;
    }
    /* Head is now the old tail — find it */
    int t = head;
    if (t != -1) {
        while (list[t].prev != -1) t = list[t].prev;
        head = t;
    }
    printf("List reversed!\n");
}

Sorting Operations

3 Algorithms

Sorting arranges the list's values in ascending (or descending) order. For array-based DLL, we swap data values (not node positions) for simplicity. All three classic O(n²) sorting algorithms are covered here.

Time Complexity

AlgorithmBestWorst
Bubble SortO(n)O(n²)
Selection SortO(n²)O(n²)
Insertion SortO(n)O(n²)

When to Use Each

Bubble Sort: Easy to understand, good for nearly-sorted data.

Selection Sort: Minimal swaps, good when writing is expensive.

Insertion Sort: Best for small or nearly-sorted lists.

11.1 Bubble Sort

bubble_sort.cC
/* Bubble Sort — repeatedly swap adjacent elements if out of order */
void bubbleSort() {
    int i, j, temp, swapped;
    for (i = 0; i < size - 1; i++) {
        swapped = 0;
        int cur = head;
        for (j = 0; j < size - i - 1; j++) {
            int nxt = list[cur].next;
            if (list[cur].data > list[nxt].data) {
                /* Swap the data values, not the nodes */
                temp = list[cur].data;
                list[cur].data = list[nxt].data;
                list[nxt].data = temp;
                swapped = 1;
            }
            cur = list[cur].next;
        }
        if (!swapped) break; /* Already sorted — stop early! */
    }
    printf("Bubble Sort done.\n");
}

11.2 Selection Sort

selection_sort.cC
/* Selection Sort — find minimum, put it at front, repeat */
void selectionSort() {
    int i, j, temp;
    int cur = head;
    while (cur != -1) {
        int min_idx = cur;          /* Assume current is min */
        int scan = list[cur].next;

        while (scan != -1) {
            if (list[scan].data < list[min_idx].data)
                min_idx = scan;     /* Found a smaller one */
            scan = list[scan].next;
        }

        if (min_idx != cur) {
            /* Swap data between cur and min_idx */
            temp = list[cur].data;
            list[cur].data = list[min_idx].data;
            list[min_idx].data = temp;
        }
        cur = list[cur].next;       /* Move forward one step */
    }
    printf("Selection Sort done.\n");
}

11.3 Insertion Sort

insertion_sort.cC
/* Insertion Sort — take one node, insert it in sorted position */
void insertionSort() {
    int cur = list[head].next;   /* Start from 2nd element */
    while (cur != -1) {
        int key = list[cur].data;
        int prev = list[cur].prev;

        /* Shift larger elements one step right */
        while (prev != -1 && list[prev].data > key) {
            list[list[prev].next].data = list[prev].data;
            prev = list[prev].prev;
        }

        /* Place key in its correct spot */
        if (prev == -1)
            list[head].data = key;      /* Goes to front */
        else
            list[list[prev].next].data = key;

        cur = list[cur].next;
    }
    printf("Insertion Sort done.\n");
}

Merge & Split Operations

3 Methods

Merge combines two lists into one. Split divides one list into two. Concatenate appends one list to the end of another. These are powerful structural operations that require careful link management.

merge_split.cC
/* 12.1 Merge Lists — merge two sorted lists into one sorted list */
void mergeLists(struct Node listB[], int headB, int sizeB,
                struct Node result[], int *res_head) {
    int a = head, b = headB, r = -1, prev_r = -1, idx = 0;
    while (a != -1 && b != -1) {
        if (list[a].data <= listB[b].data) {
            result[idx].data = list[a].data; a = list[a].next;
        } else {
            result[idx].data = listB[b].data; b = listB[b].next;
        }
        result[idx].prev = prev_r;
        if (prev_r != -1) result[prev_r].next = idx;
        else *res_head = idx;
        result[idx].next = -1;
        prev_r = idx++;
    }
    /* Copy remaining nodes from either list */
    while (a != -1) { result[idx].data = list[a].data;   result[idx].prev = prev_r; result[prev_r].next = idx; result[idx].next = -1; prev_r = idx++; a = list[a].next; }
    while (b != -1) { result[idx].data = listB[b].data; result[idx].prev = prev_r; result[prev_r].next = idx; result[idx].next = -1; prev_r = idx++; b = listB[b].next; }
}

/* 12.2 Split List — divide at position pos into two lists */
void splitList(int pos, int *head2, int *size2) {
    int cur = head, i;
    for (i = 0; i < pos - 1; i++) cur = list[cur].next;
    *head2 = list[cur].next;        /* Second list starts here */
    if (*head2 != -1) list[*head2].prev = -1;
    list[cur].next = -1;            /* First list ends here   */
    *size2 = size - pos;
    size   = pos;
    printf("Split at pos %d. List1: %d nodes, List2: %d nodes\n", pos, size, *size2);
}

/* 12.3 Concatenate Lists — append listB to end of list */
void concatenateLists(struct Node listB[], int headB, int sizeB) {
    if (headB == -1) return; /* listB is empty */
    int tail = head;
    while (list[tail].next != -1) tail = list[tail].next;
    /* Copy listB nodes into our array starting at slot 'size' */
    int b = headB, offset = size;
    while (b != -1) {
        list[offset] = listB[b];
        if (listB[b].next != -1) list[offset].next = listB[b].next + offset;
        if (listB[b].prev != -1) list[offset].prev = listB[b].prev + offset;
        else { list[offset].prev = tail; list[tail].next = offset; }
        b = listB[b].next; offset++;
    }
    size += sizeB;
}
🚂 Real-Life Analogy — Combining Train Carriages

Concatenate: Couple train B to the end of train A — they become one long train. Split: Uncouple carriage number 5, creating two separate trains. Merge (sorted): Like merging two queues at a theme park by always taking the person with a shorter wait time — the result is one ordered queue.

Recursive Operations

3 Methods

Recursion solves a problem by having a function call itself with a smaller version of the same problem. For linked lists, this means: process current node, then recurse on the rest of the list.

💡 Key Concept: Every recursive function must have a base case (when to stop) and a recursive case (call itself on a smaller problem). Without a base case, you get infinite recursion and a stack overflow crash!
recursive.cC
/* 13.1 Recursive Traversal — print all nodes recursively */
void recursiveTraverse(int cur) {
    if (cur == -1) return;          /* Base case: end of list */

    printf("%d ", list[cur].data);   /* Process current node */
    recursiveTraverse(list[cur].next); /* Recurse on rest    */
}
/* Call: recursiveTraverse(head);
   Trace for list [10→20→30]:
     recursiveTraverse(0) → print 10 → recursiveTraverse(1)
       recursiveTraverse(1) → print 20 → recursiveTraverse(2)
         recursiveTraverse(2) → print 30 → recursiveTraverse(-1)
           recursiveTraverse(-1) → BASE CASE → return              */

/* 13.2 Recursive Search */
int recursiveSearch(int cur, int value) {
    if (cur == -1) return -1;        /* Base: not found */
    if (list[cur].data == value) {
        printf("Found at index %d!\n", cur);
        return cur;                  /* Base: found it! */
    }
    return recursiveSearch(list[cur].next, value); /* Recurse */
}
/* Call: recursiveSearch(head, 20); */

/* 13.3 Recursive Count */
int recursiveCount(int cur) {
    if (cur == -1) return 0;         /* Base: empty list = 0 */
    return 1 + recursiveCount(list[cur].next);  /* 1 + rest */
}
/* Count trace for [10→20→30]:
   count(0) = 1 + count(1)
            = 1 + 1 + count(2)
            = 1 + 1 + 1 + count(-1)
            = 1 + 1 + 1 + 0 = 3 ✓    */
🪞 Real-Life Analogy — Russian Nesting Dolls (Matryoshka)

Each doll calls out what's inside it, then tells the next doll to do the same. "I'm doll 1, I contain doll 2 — hey doll 2, what do you contain?" Doll 2 reports itself and asks doll 3. The last (smallest) doll says "I'm empty — I'm done!" This is exactly how recursion works: each call handles the current item and delegates the rest.

Iterative Operations

3 Methods

Iterative operations use loops instead of function calls (no recursion). They are generally more memory-efficient because they don't use the call stack. In C, iteration is usually preferred for linked list traversal to avoid stack overflow on large lists.

iterative.cC
/* 14.1 Iterative Traversal — same as forwardTraverse() */
void iterativeTraverse() {
    int cur = head;  /* cur = "current" pointer */
    printf("Iterative: ");
    while (cur != -1) {        /* Loop until end */
        printf("%d ", list[cur].data);
        cur = list[cur].next;   /* Advance manually */
    }
    printf("\n");
}

/* 14.2 Iterative Search */
int iterativeSearch(int value) {
    int cur = head, pos = 0;
    while (cur != -1) {
        if (list[cur].data == value) {
            printf("Found %d at position %d\n", value, pos);
            return pos;
        }
        cur = list[cur].next;
        pos++;
    }
    return -1; /* Not found */
}

/* 14.3 Reverse Iteration — tail to head, using .prev links */
void reverseIteration() {
    if (isEmpty()) { printf("Empty!\n"); return; }

    /* Step 1: Go to the tail using a while loop */
    int cur = head;
    while (list[cur].next != -1)
        cur = list[cur].next;

    /* Step 2: Walk backward using .prev links */
    printf("Reverse: ");
    while (cur != -1) {
        printf("%d ", list[cur].data);
        cur = list[cur].prev;   /* Go backward */
    }
    printf("\n");
}

🔄 Real-Life Analogy — Conveyor Belt vs Factory Robot Arm

Iterative traversal is like a conveyor belt — items pass by one at a time in a steady loop until the belt ends. Recursive traversal is like a robot arm that picks up an item, processes it, then calls an identical robot arm to handle the rest of the belt. The loop (conveyor) is simpler and doesn't require dozens of robot arms stacked on top of each other.
