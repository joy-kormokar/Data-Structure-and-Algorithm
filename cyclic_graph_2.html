What is a Cyclic Graph?
Introduction & core concepts
🔵 Introduction to Cyclic Graphs Beginner ▼

A Graph is a collection of vertices (nodes) connected by edges (links).

A Cyclic Graph is a graph that contains at least one cycle — a path that starts and ends at the same vertex without repeating any edge.

Key terms:

  • Vertex (Node): A point in the graph (e.g., a city)
  • Edge: A connection between two vertices (e.g., a road)
  • Cycle: A closed loop — going from A→B→C→A
  • Directed Graph: Edges have direction (one-way streets)
  • Undirected Graph: Edges go both ways (two-way roads)

Every cyclic graph has at least one cycle. Trees, in contrast, are acyclic (no cycles).

C — Graph Structure Definition
#include <stdio.h>
#include <stdlib.h>

#define MAX 100  // Maximum number of vertices

/* --- Node in adjacency list --- */
struct Node {
    int vertex;             // Neighbor vertex number
    struct Node* next;     // Pointer to next neighbor
};

/* --- Graph using Adjacency List --- */
struct Graph {
    int numVertices;                 // Total number of vertices
    struct Node* adjList[MAX];       // Array of linked lists
};

/* --- Create a new graph --- */
struct Graph* createGraph(int vertices) {
    struct Graph* g = (struct Graph*)malloc(sizeof(struct Graph));
    g->numVertices = vertices;
    for (int i = 0; i < vertices; i++)
        g->adjList[i] = NULL;  // No edges yet
    return g;
}

int main() {
    struct Graph* g = createGraph(5);
    printf("Graph created with %d vertices\n", g->numVertices);
    return 0;
}
🌍 Real-life Analogy

Think of a city road map. Each city is a vertex. Each road connecting cities is an edge. If you can drive from City A → City B → City C → back to City A, that's a cycle! Most real road networks are cyclic — you can always find a round-trip route.

Other examples: Social networks (A follows B, B follows C, C follows A = cycle), Flight routes, Electric circuits.

Adjacency List Representation
How a graph is stored in memory
📌 Adjacency List Beginner ▼

An Adjacency List stores a graph as an array of linked lists. Each index of the array represents a vertex, and its linked list contains all vertices it connects to.

Why Adjacency List over Matrix?

  • Space efficient: Uses O(V + E) space vs O(V²) for matrix
  • Better for sparse graphs (fewer edges)
  • Faster neighbor traversal

Example: If graph has edges 0→1, 0→2, 1→2, 2→0 (a cycle!):

  • adjList[0] → 1 → 2 → NULL
  • adjList[1] → 2 → NULL
  • adjList[2] → 0 → NULL (cycle back!)

C — Adjacency List with Edge Insertion
/* Create a new node for the adjacency list */
struct Node* createNode(int v) {
    struct Node* newNode;
    newNode = (struct Node*)malloc(sizeof(struct Node));
    newNode->vertex = v;
    newNode->next = NULL;
    return newNode;
}

/* Add a directed edge from src to dest */
void addEdge(struct Graph* g, int src, int dest) {
    // Create new node for dest
    struct Node* newNode = createNode(dest);
    // Insert at front of src's list
    newNode->next = g->adjList[src];
    g->adjList[src] = newNode;
}

/* Print the adjacency list */
void printGraph(struct Graph* g) {
    for (int v = 0; v < g->numVertices; v++) {
        struct Node* temp = g->adjList[v];
        printf("Vertex %d: ", v);
        while (temp) {
            printf("%d → ", temp->vertex);
            temp = temp->next;
        }
        printf("NULL\n");
    }
}

int main() {
    struct Graph* g = createGraph(3);
    addEdge(g, 0, 1);
    addEdge(g, 1, 2);
    addEdge(g, 2, 0);  // This creates a cycle!
    printGraph(g);
    return 0;
}
🌍 Real-life Analogy

Think of a phone contacts list. Each person has a list of their direct contacts. Instead of a giant table showing all possible pairs (like an adjacency matrix), you only store who is actually connected. This is exactly what an adjacency list does — save space, store only real connections.

Vertex Operations
Insert, Delete, Search, Update, Count, Display
➕ Insert Vertex Beginner ▼

Inserting a vertex means adding a new node to the graph. In an adjacency list, you simply increase numVertices and initialize its list to NULL.

  • The new vertex starts with no edges
  • Time complexity: O(1)
  • Space: One new pointer entry in the array
Click "Add Vertex" to see it inserted step by step
C — Insert Vertex
/* Insert a new vertex into the graph */
void insertVertex(struct Graph* g) {
    if (g->numVertices >= MAX) {
        printf("Graph is full! Cannot add more vertices.\n");
        return;
    }
    // Initialize the new vertex's adjacency list to NULL
    g->adjList[g->numVertices] = NULL;
    g->numVertices++;  // Increase vertex count
    printf("New vertex %d added.\n", g->numVertices - 1);
}
🌍 Analogy

Like adding a new city to a map. The city exists, but initially has no roads connecting it. Later, you can add roads (edges) to connect it to other cities.

🗑️ Delete Vertex Intermediate ▼

Deleting a vertex requires: (1) freeing the vertex's own adjacency list, and (2) removing all edges pointing to this vertex from all other lists.

  • Must scan all other vertices' lists — O(V + E)
  • Shifts subsequent vertex indices (if using array)
C — Delete Vertex
/* Remove all edges TO vertex 'v' from all lists */
void removeEdgesTo(struct Graph* g, int v) {
    for (int i = 0; i < g->numVertices; i++) {
        struct Node** curr = &g->adjList[i];
        while (*curr) {
            if ((*curr)->vertex == v) {
                struct Node* temp = *curr;
                *curr = (*curr)->next;
                free(temp);
            } else {
                curr = &(*curr)->next;
            }
        }
    }
}

/* Delete vertex v completely */
void deleteVertex(struct Graph* g, int v) {
    // Step 1: Free v's own adjacency list
    struct Node* curr = g->adjList[v];
    while (curr) {
        struct Node* temp = curr;
        curr = curr->next;
        free(temp);
    }
    g->adjList[v] = NULL;
    // Step 2: Remove all edges pointing to v
    removeEdgesTo(g, v);
    g->numVertices--;
    printf("Vertex %d deleted.\n", v);
}
🌍 Analogy

Like demolishing a city. First, remove the city from the map. Then close all roads that led to it. Without step 2, other cities would have roads pointing to nowhere!

🔍 Search Vertex Beginner ▼

Searching for a vertex checks if a given vertex number exists in the graph. Since vertices are stored as indices in an array, this is a simple bounds check — O(1).

C — Search Vertex
/* Check if vertex v exists in graph */
int searchVertex(struct Graph* g, int v) {
    if (v >= 0 && v < g->numVertices) {
        printf("Vertex %d found!\n", v);
        return 1;  // Found
    }
    printf("Vertex %d not found.\n", v);
    return 0;  // Not found
}
🌍 Analogy

Like checking if a city exists on a numbered list. If the city number is within range, it exists!

Edge Operations
Insert, Delete, Search, Update, Count, Display
➕ Insert Edge Beginner▼

Add an edge from vertex src to vertex dest. In an adjacency list, create a new node with value dest and prepend it to adjList[src].

  • Directed: Only add dest to src's list
  • Undirected: Also add src to dest's list
  • Time: O(1)
Click to watch an edge being inserted
C — Insert Edge (Directed & Undirected)
/* Insert directed edge: src → dest */
void insertEdgeDirected(struct Graph* g, int src, int dest) {
    struct Node* newNode = createNode(dest);
    newNode->next = g->adjList[src];
    g->adjList[src] = newNode;
    printf("Edge %d → %d added.\n", src, dest);
}

/* Insert undirected edge: src ↔ dest */
void insertEdgeUndirected(struct Graph* g, int src, int dest) {
    insertEdgeDirected(g, src, dest);  // src → dest
    insertEdgeDirected(g, dest, src);  // dest → src
}
🌍 Analogy

Building a one-way road (directed) or a two-way road (undirected) between two cities. One-way: only drivers from City A can reach City B directly. Two-way: both can reach each other.

🗑️ Delete Edge Intermediate▼

To delete edge src→dest, traverse src's list, find the node with value dest, and unlink it. Then free the memory. O(degree(src)).

C — Delete Edge
void deleteEdge(struct Graph* g, int src, int dest) {
    struct Node** curr = &g->adjList[src];
    while (*curr) {
        if ((*curr)->vertex == dest) {
            struct Node* temp = *curr;
            *curr = (*curr)->next;  // Bypass the node
            free(temp);
            printf("Edge %d→%d deleted.\n", src, dest);
            return;
        }
        curr = &(*curr)->next;
    }
    printf("Edge not found!\n");
}
🌍 Analogy

Like closing a road between two cities. The cities still exist, but you can no longer travel that specific route.

🔍 Search Edge Beginner▼

Check if edge src→dest exists. Traverse src's adjacency list looking for dest. O(degree(src)).

C — Search Edge
int searchEdge(struct Graph* g, int src, int dest) {
    struct Node* curr = g->adjList[src];
    while (curr) {
        if (curr->vertex == dest) {
            printf("Edge %d→%d exists!\n", src, dest);
            return 1;
        }
        curr = curr->next;
    }
    printf("Edge %d→%d not found.\n", src, dest);
    return 0;
}
✏️ Update Edge Intermediate▼

Update an edge by changing its destination (or weight in a weighted graph). Simplest approach: delete old edge + insert new edge. In weighted graphs, just find the node and update the weight field.

C — Update Edge (Weighted Graph)
struct Node {
    int vertex;
    int weight;  // Added weight field
    struct Node* next;
};

void updateEdgeWeight(struct Graph* g, int src, int dest, int newW) {
    struct Node* curr = g->adjList[src];
    while (curr) {
        if (curr->vertex == dest) {
            curr->weight = newW;
            printf("Edge %d→%d weight updated to %d\n", src, dest, newW);
            return;
        }
        curr = curr->next;
    }
}
🔢 Count Edges Beginner▼

Count all edges by summing the length of every vertex's adjacency list. For undirected graphs, divide by 2 (each edge is stored twice). O(V + E). Called the "size" of the graph.

C — Count Edges
int countEdges(struct Graph* g, int directed) {
    int count = 0;
    for (int i = 0; i < g->numVertices; i++) {
        struct Node* curr = g->adjList[i];
        while (curr) { count++; curr = curr->next; }
    }
    if (!directed) count /= 2;  // Undirected: each edge counted twice
    printf("Total edges: %d\n", count);
    return count;
}
📋 Display Edges Beginner▼

Print all edges. For each vertex, traverse its list and print each (src, dest) pair.

C — Display All Edges
void displayEdges(struct Graph* g) {
    printf("All edges:\n");
    for (int i = 0; i < g->numVertices; i++) {
        struct Node* curr = g->adjList[i];
        while (curr) {
            printf("  %d → %d\n", i, curr->vertex);
            curr = curr->next;
        }
    }
}

Traversal Operations
BFS and DFS — visiting every node systematically
🌊 Breadth First Search (BFS) Intermediate▼

BFS explores all neighbors at the current depth before moving deeper. It uses a Queue (FIFO).

Steps:

  • 1. Mark start vertex as visited, enqueue it
  • 2. Dequeue a vertex, print it
  • 3. Enqueue all unvisited neighbors, mark them visited
  • 4. Repeat until queue is empty

Time: O(V + E) | Space: O(V) | Used for: Shortest path (unweighted), level-order traversal

Click "Run BFS" to animate Breadth-First Search
C — BFS with Queue
#define MAX 100
int queue[MAX], front = 0, rear = 0;
int visited[MAX] = {0};

void enqueue(int v) { queue[rear++] = v; }
int  dequeue()       { return queue[front++]; }
int  isEmpty()       { return front == rear; }

void BFS(struct Graph* g, int start) {
    // Step 1: Initialize
    front = rear = 0;
    visited[start] = 1;
    enqueue(start);

    printf("BFS order: ");
    while (!isEmpty()) {
        // Step 2: Visit current node
        int v = dequeue();
        printf("%d ", v);

        // Step 3: Enqueue unvisited neighbors
        struct Node* curr = g->adjList[v];
        while (curr) {
            if (!visited[curr->vertex]) {
                visited[curr->vertex] = 1;
                enqueue(curr->vertex);
            }
            curr = curr->next;
        }
    }
    printf("\n");
}
🌍 Analogy

Like ripples in a pond. Throw a stone (start vertex) — the wave spreads outward layer by layer. All nodes at distance 1 are visited first, then distance 2, etc. BFS is perfect for finding the closest path.

🔦 Depth First Search (DFS) Intermediate▼

DFS goes as deep as possible before backtracking. Uses recursion (or an explicit Stack).

Steps:

  • 1. Mark current vertex as visited
  • 2. Recursively visit each unvisited neighbor
  • 3. Backtrack when no unvisited neighbors remain

Time: O(V + E) | Used for: Cycle detection, topological sort, finding paths

Click "Run DFS" to animate Depth-First Search
C — DFS Recursive + Stack-based
int vis[MAX] = {0};

/* Recursive DFS */
void DFS_Recursive(struct Graph* g, int v) {
    vis[v] = 1;
    printf("%d ", v);
    struct Node* curr = g->adjList[v];
    while (curr) {
        if (!vis[curr->vertex])
            DFS_Recursive(g, curr->vertex);  // Go deeper!
        curr = curr->next;
    }
}  /* Automatically backtracks when function returns */

/* Stack-based DFS (iterative) */
void DFS_Stack(struct Graph* g, int start) {
    int stack[MAX], top = -1, vis2[MAX] = {0};
    stack[++top] = start;
    printf("DFS: ");
    while (top >= 0) {
        int v = stack[top--];
        if (vis2[v]) continue;
        vis2[v] = 1;
        printf("%d ", v);
        struct Node* curr = g->adjList[v];
        while (curr) {
            if (!vis2[curr->vertex])
                stack[++top] = curr->vertex;
            curr = curr->next;
        }
    }
    printf("\n");
}
🌍 Analogy

Like exploring a maze. You keep going straight until you hit a dead end. Then you backtrack and try a different path. DFS dives deep before exploring wide.

Cycle Operations
Detect, count, find, check, and remove cycles
🔄 Detect Cycle Intermediate▼

A cycle is detected when DFS encounters a back edge — an edge leading to an already-visited vertex that is currently in the DFS recursion stack.

For directed graphs: Track a recStack[] (recursion stack). If a neighbor is in recStack → cycle found.

For undirected graphs: If a neighbor is visited and is NOT the parent → cycle found.

Click to run cycle detection on the graph
C — Detect Cycle (Directed Graph)
int visited2[MAX], recStack[MAX];

int detectCycleUtil(struct Graph* g, int v) {
    visited2[v] = 1;
    recStack[v] = 1;  // Mark in current path

    struct Node* curr = g->adjList[v];
    while (curr) {
        int n = curr->vertex;
        if (!visited2[n] && detectCycleUtil(g, n))
            return 1;  // Cycle in deeper call
        if (recStack[n])
            return 1;  // Back edge found — CYCLE!
        curr = curr->next;
    }
    recStack[v] = 0;  // Remove from path on backtrack
    return 0;
}

int hasCycle(struct Graph* g) {
    for (int i = 0; i < g->numVertices; i++)
        visited2[i] = recStack[i] = 0;
    for (int i = 0; i < g->numVertices; i++)
        if (!visited2[i] && detectCycleUtil(g, i))
            return 1;
    return 0;
}
🌍 Analogy

Like leaving breadcrumbs while exploring. If you're walking through a forest and you find your own breadcrumbs, you've walked in a circle — that's a cycle! The recStack[] is your trail of breadcrumbs.

🔢 Count Cycles Advanced▼

Counting all distinct cycles is more complex. One approach: count back edges found during DFS — each back edge represents at least one cycle. For exact counts in undirected graphs: Cycles = E - V + 1 (for connected graph). O(V + E).

C — Count Cycles (Back Edge Method)
int cycleCount = 0;
int vis3[MAX], rec3[MAX];

void countCyclesUtil(struct Graph* g, int v) {
    vis3[v] = rec3[v] = 1;
    struct Node* curr = g->adjList[v];
    while (curr) {
        int n = curr->vertex;
        if (!vis3[n])
            countCyclesUtil(g, n);
        else if (rec3[n])
            cycleCount++;  // Found a back edge = one cycle
        curr = curr->next;
    }
    rec3[v] = 0;  // Backtrack
}

/* Formula for undirected connected graph */
int countCyclesFormula(int V, int E) {
    return E - V + 1;  // Cyclomatic number
}
🗺️ Find Cycle Path Advanced▼

Find the actual vertices forming a cycle. Use DFS with a path array. When a back edge is found, trace back through the path array from the current node to the ancestor.

C — Find and Print Cycle Path
int parent[MAX];
int vis4[MAX], rec4[MAX];

void printCycle(int start, int end) {
    printf("Cycle: %d", end);
    int curr = start;
    while (curr != end) {
        printf(" → %d", curr);
        curr = parent[curr];
    }
    printf(" → %d\n", end);
}

int findCycleUtil(struct Graph* g, int v) {
    vis4[v] = rec4[v] = 1;
    struct Node* curr = g->adjList[v];
    while (curr) {
        int n = curr->vertex;
        parent[n] = v;
        if (!vis4[n] && findCycleUtil(g, n)) return 1;
        if (rec4[n]) { printCycle(v, n); return 1; }
        curr = curr->next;
    }
    rec4[v] = 0;
    return 0;
}
🌍 Analogy

Like GPS tracking a delivery truck. When the truck returns to a city it already visited, you can print the exact route it took to form that loop.

🔁 Check Self Loop Beginner▼

A self-loop is an edge from a vertex to itself (e.g., 3→3). Check if any vertex's adjacency list contains itself as a neighbor. O(V + E).

C — Check Self Loop
int hasSelfLoop(struct Graph* g) {
    for (int i = 0; i < g->numVertices; i++) {
        struct Node* curr = g->adjList[i];
        while (curr) {
            if (curr->vertex == i) {  // Points to itself!
                printf("Self-loop at vertex %d\n", i);
                return 1;
            }
            curr = curr->next;
        }
    }
    return 0;
}
🌍 Analogy

Like a person who sends an email to themselves. The email goes from them and returns to them — that's a self-loop!

↩️ Check Back Edge Intermediate▼

A back edge connects a vertex to one of its DFS ancestors. Back edges always form cycles in directed graphs. The recStack[] tracks the current DFS path to identify ancestors.

C — Detect Back Edge
int vis5[MAX], rec5[MAX];

int hasBackEdge(struct Graph* g, int v) {
    vis5[v] = rec5[v] = 1;
    struct Node* curr = g->adjList[v];
    while (curr) {
        int n = curr->vertex;
        if (!vis5[n] && hasBackEdge(g, n)) return 1;
        if (rec5[n]) {
            printf("Back edge: %d → %d\n", v, n);
            return 1;  // This IS a back edge
        }
        curr = curr->next;
    }
    rec5[v] = 0;
    return 0;
}
➡️ Detect Directed Cycle Intermediate▼

In a directed graph, a cycle means you can follow directed edges and return to the starting vertex. Use DFS + recStack[]. A back edge (to a node in recStack) indicates a directed cycle.

This is the same as the hasCycle() function shown above — since that already handles directed graphs.

C — Directed Cycle Detection
/* Uses recStack to track current DFS path */
int visD[MAX], recD[MAX];

int dirCycleUtil(struct Graph* g, int v) {
    visD[v] = recD[v] = 1;
    struct Node* curr = g->adjList[v];
    while (curr) {
        int n = curr->vertex;
        if (!visD[n] && dirCycleUtil(g, n)) return 1;
        else if (recD[n]) return 1;  // Directed cycle!
        curr = curr->next;
    }
    recD[v] = 0;
    return 0;
}

int hasDirectedCycle(struct Graph* g) {
    for (int i=0;i<g->numVertices;i++) visD[i]=recD[i]=0;
    for (int i=0;i<g->numVertices;i++)
        if (!visD[i] && dirCycleUtil(g,i)) return 1;
    return 0;
}
↔️ Detect Undirected Cycle Intermediate▼

In an undirected graph, every edge appears in both directions. During DFS, if we visit a neighbor that's already visited and it's NOT our parent — that means a cycle exists. We must track the parent to avoid false positives from the back-edge to parent.

C — Undirected Cycle Detection
int visU[MAX];

int undirCycleUtil(struct Graph* g, int v, int parent) {
    visU[v] = 1;
    struct Node* curr = g->adjList[v];
    while (curr) {
        int n = curr->vertex;
        if (!visU[n]) {
            if (undirCycleUtil(g, n, v)) return 1;
        } else if (n != parent) {
            // Visited and not parent = cycle!
            return 1;
        }
        curr = curr->next;
    }
    return 0;
}
✂️ Remove Cycle Advanced▼

To remove a cycle: find the back edge that creates it (using DFS), then delete that edge. Repeat until no cycles remain. This turns the cyclic graph into a DAG or tree.

C — Remove Cycle by Breaking Back Edge
int visR[MAX], recR[MAX];

int removeCycleUtil(struct Graph* g, int v) {
    visR[v] = recR[v] = 1;
    struct Node* curr = g->adjList[v];
    while (curr) {
        int n = curr->vertex;
        if (!visR[n] && removeCycleUtil(g, n)) return 1;
        if (recR[n]) {
            // Delete this back edge to break the cycle
            deleteEdge(g, v, n);
            printf("Removed back edge %d→%d to break cycle\n", v, n);
            return 1;
        }
        curr = curr->next;
    }
    recR[v] = 0;
    return 0;
}

void removeAllCycles(struct Graph* g) {
    while (hasCycle(g)) {
        for (int i=0;i<g->numVertices;i++) visR[i]=recR[i]=0;
        for (int i=0;i<g->numVertices;i++)
            if(!visR[i]) removeCycleUtil(g,i);
    }
    printf("All cycles removed!\n");
}
🌍 Analogy

Like cutting one road in a city's circular route to make it a one-way dead-end. The cities still exist, but the circular loop is broken.

Path Operations
Finding and measuring paths between vertices
🛤️ Find Path Intermediate▼

Find if a path exists from vertex src to vertex dest, and print it. Use DFS with a path array. When dest is reached, print the recorded path.

C — Find Path using DFS
int pathArr[MAX], pathLen = 0;
int visP[MAX];

int findPathDFS(struct Graph* g, int curr, int dest) {
    visP[curr] = 1;
    pathArr[pathLen++] = curr;

    if (curr == dest) {
        printf("Path: ");
        for (int i=0;i<pathLen;i++) printf("%d%s",pathArr[i],i<pathLen-1?"→":"\n");
        return 1;
    }

    struct Node* c = g->adjList[curr];
    while (c) {
        if (!visP[c->vertex] && findPathDFS(g, c->vertex, dest))
            return 1;
        c = c->next;
    }
    pathLen--;  // Backtrack
    return 0;
}
🌍 Analogy

Like Google Maps finding any route between two places. It doesn't need to be the shortest — just a valid path that gets you there.

🏃 Shortest Path (BFS) Intermediate▼

For unweighted graphs, BFS naturally finds the shortest path because it explores level by level. Track each vertex's distance from the source and its parent to reconstruct the path.

C — Shortest Path using BFS
void shortestPath(struct Graph* g, int src, int dest) {
    int dist[MAX], par[MAX], vis[MAX];
    int q[MAX], front=0, rear=0;

    for(int i=0;i<MAX;i++) dist[i]=par[i]=vis[i]=-1;

    vis[src]=1; dist[src]=0; q[rear++]=src;

    while(front<rear) {
        int v = q[front++];
        struct Node* c = g->adjList[v];
        while(c) {
            if(!vis[c->vertex]) {
                vis[c->vertex]=1;
                dist[c->vertex]=dist[v]+1;
                par[c->vertex]=v;
                q[rear++]=c->vertex;
            }
            c=c->next;
        }
    }

    if(dist[dest]==-1) { printf("No path!\n"); return; }
    printf("Shortest distance: %d\nPath: ", dist[dest]);

    /* Reconstruct path using parent array */
    int path[MAX], plen=0, cur=dest;
    while(cur!=-1) { path[plen++]=cur; cur=par[cur]; }
    for(int i=plen-1;i>=0;i--)
        printf("%d%s",path[i],i?"→":"\n");
}
🌍 Analogy

Like finding the fewest number of bus stops to get from home to work. BFS counts hops level by level, so the first time it reaches the destination, it found the minimum-hop route.

📊 Density Check Intermediate▼

Density = ratio of actual edges to maximum possible edges.

  • Directed: max edges = V×(V-1)
  • Undirected: max edges = V×(V-1)/2
  • Density = actual_edges / max_possible_edges
  • Range: 0.0 (empty) to 1.0 (complete)
C — Density Calculation
float graphDensity(struct Graph* g, int directed) {
    int V = g->numVertices;
    int E = graphSize(g);
    int maxEdges = directed ? V*(V-1) : V*(V-1)/2;
    float density = (float)E / maxEdges;
    printf("Density: %.4f\n", density);
    return density;
}

⚙️ Tarjan's Algorithm Advanced▼

Tarjan's algorithm finds all Strongly Connected Components (SCCs) in a directed graph in a single DFS pass. O(V + E).

Key idea: Two arrays — disc[] (discovery time) and low[] (lowest reachable disc time) — and a stack.

  • When disc[v] == low[v], pop the stack to get one complete SCC
  • More efficient than Kosaraju (one DFS pass vs two)
Click to visualize Tarjan's SCC algorithm
C — Tarjan's SCC Algorithm
int disc[MAX], low[MAX], onStack[MAX];
int stkT[MAX], stkTop=-1, timer=0;

void tarjanUtil(struct Graph* g, int v) {
    disc[v] = low[v] = timer++;
    stkT[++stkTop] = v;
    onStack[v] = 1;

    struct Node* curr = g->adjList[v];
    while(curr) {
        int n = curr->vertex;
        if(disc[n]==-1) {
            tarjanUtil(g, n);
            // Update low value after DFS
            low[v] = low[v]<low[n] ? low[v] : low[n];
        } else if(onStack[n]) {
            low[v] = low[v]<disc[n] ? low[v] : disc[n];
        }
        curr = curr->next;
    }

    /* v is root of an SCC when disc[v] == low[v] */
    if(low[v] == disc[v]) {
        printf("SCC: ");
        while(stkT[stkTop] != v) {
            printf("%d ", stkT[stkTop]);
            onStack[stkT[stkTop--]] = 0;
        }
        printf("%d\n", stkT[stkTop]);
        onStack[stkT[stkTop--]] = 0;
    }
}

void tarjan(struct Graph* g) {
    for(int i=0;i<g->numVertices;i++) disc[i]=low[i]=onStack[i]=-1;
    timer=0; stkTop=-1;
    for(int i=0;i<g->numVertices;i++)
        if(disc[i]==-1) tarjanUtil(g,i);
}
🌍 Analogy

Like finding tightly-knit friend groups on social media. An SCC is a group where everyone can reach everyone else. Tarjan's algorithm identifies these groups efficiently in one traversal of the network.

🔀 Kosaraju's Algorithm Advanced▼

Kosaraju's algorithm finds SCCs in two DFS passes. O(V + E).

Steps:

  • 1. Do DFS on original graph. Push vertices to stack in finish order
  • 2. Reverse all edges of the graph
  • 3. Do DFS on reversed graph in stack's order. Each complete DFS = one SCC
C — Kosaraju's Algorithm
int visK[MAX], finStack[MAX], finTop=-1;

/* Step 1: DFS and push in finish order */
void dfsFinish(struct Graph* g, int v) {
    visK[v]=1;
    struct Node* c=g->adjList[v];
    while(c){if(!visK[c->vertex])dfsFinish(g,c->vertex);c=c->next;}
    finStack[++finTop]=v;  // Push when done
}

/* Step 3: DFS on reversed graph to print SCC */
int visR2[MAX];
void dfsPrint(struct Graph* rev, int v) {
    visR2[v]=1; printf("%d ",v);
    struct Node* c=rev->adjList[v];
    while(c){if(!visR2[c->vertex])dfsPrint(rev,c->vertex);c=c->next;}
}

void kosaraju(struct Graph* g) {
    for(int i=0;i<g->numVertices;i++) visK[i]=0;
    for(int i=0;i<g->numVertices;i++)
        if(!visK[i]) dfsFinish(g,i);
    struct Graph* rev=reverseGraph(g);
    for(int i=0;i<g->numVertices;i++) visR2[i]=0;
    while(finTop>=0) {
        int v=finStack[finTop--];
        if(!visR2[v]){printf("SCC: ");dfsPrint(rev,v);printf("\n");}
    }
}
🌍 Analogy

Like finding two-way communication groups in a network. First, find who finishes "talking" last. Then reverse all communication channels and see who can still hear each other — those form a strongly connected group.

🐢 Floyd's Cycle Detection Advanced▼

Floyd's Tortoise and Hare algorithm detects cycles using two pointers. Works best on linked list-like structures / functional graphs (each node has exactly one outgoing edge).

  • Slow pointer: moves 1 step at a time
  • Fast pointer: moves 2 steps at a time
  • If they meet → cycle exists
  • O(n) time, O(1) space
Click to see the Tortoise and Hare algorithm
C — Floyd's Tortoise and Hare
/* Works on functional graph (next[] array = 1 outgoing edge/node) */
int next_node[MAX];  // next_node[i] = where node i points

int floydCycleDetect(int start) {
    int slow = start;   // Tortoise
    int fast = start;   // Hare

    do {
        slow = next_node[slow];        // 1 step
        fast = next_node[next_node[fast]]; // 2 steps
    } while (slow != fast);

    printf("Cycle detected! Meeting point: %d\n", slow);

    /* Find start of cycle */
    slow = start;
    while (slow != fast) {
        slow = next_node[slow];
        fast = next_node[fast];
    }
    printf("Cycle starts at: %d\n", slow);
    return slow;
}
🌍 Analogy

A tortoise and a hare running on a circular track. The hare moves twice as fast. If the track is circular (cyclic), the hare will eventually lap the tortoise and they'll meet. If there's no circle (no cycle), the hare reaches the end first and they never meet.

🔵 Strongly Connected Components (SCC) Advanced▼

An SCC is a maximal subset of vertices where every vertex can reach every other vertex via directed edges. A single vertex is always its own SCC.

Applications:

  • Finding circular dependencies in build systems
  • Identifying feedback loops in systems
  • Web page clustering (connected website groups)
  • Social network community detection

Use Tarjan's or Kosaraju's algorithm (both shown above).

C — Count & Print All SCCs (using Tarjan)
/* sccCount tracks number of SCCs found */
int sccCount = 0;
int disc2[MAX], low2[MAX], onStk[MAX];
int stk2[MAX], stk2Top=-1, timer2=0;

void sccUtil(struct Graph* g, int v) {
    disc2[v]=low2[v]=timer2++;
    stk2[++stk2Top]=v; onStk[v]=1;
    struct Node* c=g->adjList[v];
    while(c){
        int n=c->vertex;
        if(disc2[n]==-1){sccUtil(g,n);low2[v]=low2[v]<low2[n]?low2[v]:low2[n];}
        else if(onStk[n]) low2[v]=low2[v]<disc2[n]?low2[v]:disc2[n];
        c=c->next;
    }
    if(low2[v]==disc2[v]) {
        sccCount++;
        printf("SCC %d: ", sccCount);
        while(stk2[stk2Top]!=v)
            {printf("%d ",stk2[stk2Top]);onStk[stk2[stk2Top--]]=0;}
        printf("%d\n",stk2[stk2Top]);onStk[stk2[stk2Top--]]=0;
    }
}

}
🌍 Analogy

Islands of mutual reachability. In a one-way flight network, an SCC is a group of cities where you can fly from any city in the group to any other city in the group (possibly via other cities in the group). Cities outside cannot be part of this loop.

🎨 Graph Coloring Advanced▼

Graph Coloring assigns colors to vertices so that no two adjacent vertices share the same color. The minimum number of colors needed = chromatic number χ(G).

Greedy Coloring Algorithm:

  • For each vertex, find colors used by neighbors
  • Assign the smallest color NOT used by any neighbor
  • Not always optimal, but simple and effective

Applications: Register allocation in compilers, scheduling problems, map coloring, Sudoku solving.

Click to watch the greedy coloring algorithm in action
C — Greedy Graph Coloring
void graphColoring(struct Graph* g) {
    int color[MAX];           // Color assigned to each vertex
    int available[MAX];       // Colors used by neighbors

    for(int i=0;i<g->numVertices;i++) color[i]=-1;
    color[0] = 0;  // Color first vertex with color 0

    for(int v=1;v<g->numVertices;v++) {
        /* Mark colors of adjacent vertices as unavailable */
        for(int c=0;c<g->numVertices;c++) available[c]=0;

        struct Node* curr=g->adjList[v];
        while(curr) {
            if(color[curr->vertex]!=-1)
                available[color[curr->vertex]]=1;
            curr=curr->next;
        }

        /* Find first available color */
        int c=0;
        while(available[c]) c++;
        color[v]=c;
    }

    printf("Vertex → Color:\n");
    for(int i=0;i<g->numVertices;i++)
        printf("  Vertex %d → Color %d\n", i, color[i]);
}
🌍 Analogy

Like coloring a map so that no two neighboring countries share the same color. The famous "Four Color Theorem" proves any map needs at most 4 colors. Graph coloring also models exam scheduling: students sharing a class conflict — assign different time slots (colors) so no two conflicting classes happen simultaneously.

After traversal algorithms (BFS/DFS), the visited array must be reset to 0 for the next traversal. Otherwise, subsequent traversals will skip already-visited nodes incorrectly.

C — Reset Visited Array
int visited[MAX];

/* Reset all entries to 0 */
void resetVisited(int n) {
    for(int i=0;i<n;i++)
        visited[i] = 0;
    printf("Visited array reset.\n");
}

/* Alternative: use memset for efficiency */
#include <string.h>
void resetVisitedFast(int n) {
    memset(visited, 0, n * sizeof(int));
}
