Basic Operations
🧱 Create / Initialize a 3D Array FOUNDATION

📖 Theory

A 3D array is like a box of layers (like floors in a building). Each floor has rows, and each row has columns. So to find any value, you need THREE coordinates: [layer][row][column].

  • Layer (x) → which floor?
  • Row (y) → which row on that floor?
  • Column (z) → which cell in that row?

Syntax: data_type arr[L][R][C]; — where L = layers, R = rows, C = columns

💡 Real-Life Analogy

🏢 Think of a 3D array as a multi-storey hotel. Each floor = a layer, each room row = a row, each room = a cell. To reach Room 204, you go Floor 2, Row 0, Room 4 → arr[2][0][4]

🔮 3D Structure Visualization

Click "Animate Init" to see initialization step by step.
C · Declare & Initialize
#include <stdio.h>

int main() {

  // 1. Declare (no values yet)
  int arr[2][3][4];
  // 2 layers, 3 rows, 4 cols

  // 2. Static Initialization
  int b[2][2][2] = {
    { {1,2},{3,4} },  // layer 0
    { {5,6},{7,8} }   // layer 1
  };

  // 3. Assign Default Values (0)
  int c[2][2][2] = {0};

  // 4. Input Elements from user
  int d[2][2][2];
  for(int l=0;l<2;l++)
    for(int r=0;r<2;r++)
      for(int c=0;c<2;c++)
        scanf("%d", &d[l][r][c]);

  return 0;
}
Total Elements = Layers × Rows × Columns  |  Memory (bytes) = L × R × C × sizeof(int)

Insertion Operations
➕ Layer Insertion INSERT

📖 Theory

Since C arrays have a fixed size, "insertion" means shifting elements to make room. To insert a new layer, you shift all layers after the insertion point one step forward, then fill the new layer. You must declare the array with extra space first.

  • Insert at Beginning → shift all layers right by 1, put new layer at index 0
  • Insert at Middle → shift layers from position p to end, put new at p
  • Insert at End → simply add to the next empty layer slot

💡 Analogy

🏗️ Adding a new floor to a building — everything above the new floor must be moved up first to create space.

🔮 Layer Insertion Visual

Choose where to insert a new layer.
C · Layer Insertion
#define LMAX 5
#define R 2
#define C 2
int arr[LMAX][R][C], n=3;

// Insert Layer at Beginning (pos=0)
void insertLayerAt(int pos,
                    int newL[R][C]){
  // shift layers right
  for(int l=n;l>pos;l--)
    for(int r=0;r<R;r++)
      for(int c=0;c<C;c++)
        arr[l][r][c]=arr[l-1][r][c];
  // copy new layer
  for(int r=0;r<R;r++)
    for(int c=0;c<C;c++)
      arr[pos][r][c]=newL[r][c];
  n++;
}

// Insert at End — just append
void insertLayerEnd(int newL[R][C]){
  for(int r=0;r<R;r++)
    for(int c=0;c<C;c++)
      arr[n][r][c]=newL[r][c];
  n++;
}
📋 Row, Column & Element Insertion INSERT

📖 Theory

Similar to layer insertion, row and column insertions require shifting data within a specific layer to create space. Element insertion at a specific position shifts elements within a row.

💡 Analogy

📊 Like inserting a row in an Excel sheet on a specific page — other rows slide down to make room.

C · Row, Column & Element Insertion
// Insert Row in Specific Layer (layer=0, pos=1)
void insertRow(int layer, int pos,
               int newRow[C], int rows) {
  // shift existing rows down
  for(int r=rows;r>pos;r--)
    for(int c=0;c<C;c++)
      arr[layer][r][c]=arr[layer][r-1][c];
  for(int c=0;c<C;c++)
    arr[layer][pos][c]=newRow[c];
}

// Insert Column in Specific Layer (layer=0, pos=1)
void insertCol(int layer, int pos,
               int newCol[R], int cols){
  for(int r=0;r<R;r++){
    for(int c=cols;c>pos;c--)
      arr[layer][r][c]=arr[layer][r][c-1];
    arr[layer][r][pos]=newCol[r];
  }
}

// Insert Element at Specific Position
void insertElem(int l,int r,int pos,
                int val,int cols){
  for(int c=cols;c>pos;c--)
    arr[l][r][c]=arr[l][r][c-1];
  arr[l][r][pos]=val;
}
03
Deletion Operations
🗑️ Layer, Row, Column & Element Deletion DELETE

📖 Theory

Deletion in a static C array means shifting elements left/up to fill the gap, then decrementing the count. The array size doesn't shrink — we just track fewer "valid" elements. Key operations:

  • Delete First/Middle/Last Layer → shift layers left starting from deletion point
  • Delete Row from Layer → shift rows up within that layer
  • Delete Column from Layer → shift columns left in every row
  • Delete Specific Element / Reset → shift elements left in that row, or set to 0

💡 Analogy

🏢 Demolishing a floor of a building — all floors above it must shift down to close the gap.

🔮 Deletion Visual

Click "Del Layer" to remove a layer (shift remaining).
C · Deletion Operations
// Delete Layer at position 'pos'
void deleteLayer(int pos, int *n){
  for(int l=pos;l<*n-1;l++)
    for(int r=0;r<R;r++)
      for(int c=0;c<C;c++)
        arr[l][r][c]=arr[l+1][r][c];
  (*n)--;
}

// Delete Row from Layer
void deleteRow(int layer, int row,
               int *rows){
  for(int r=row;r<*rows-1;r++)
    for(int c=0;c<C;c++)
      arr[layer][r][c]=arr[layer][r+1][c];
  (*rows)--;
}

// Delete Column from Layer
void deleteCol(int layer, int col,
               int *cols){
  for(int r=0;r<R;r++)
    for(int c=col;c<*cols-1;c++)
      arr[layer][r][c]=arr[layer][r][c+1];
  (*cols)--;
}

// Reset Element (set to 0)
arr[l][r][c] = 0;

// Delete Specific Element (shift left)
for(int i=col;i<C-1;i++)
  arr[l][r][i]=arr[l][r][i+1];

Searching Operations
🔎 Linear & Specialized Searching SEARCH

📖 Theory

Linear Search scans every element one by one until the target is found. Specialized searches exploit structure — sorted arrays can use binary search on each layer, and coordinate-based search targets a specific region.

  • Search Single Value → scan all elements, return first found coordinates
  • Search Multiple Occurrences → collect ALL coordinates where value exists
  • Layer-wise Search → check only elements within a specific layer
  • Coordinate-based Search → check value at exact [l][r][c]

💡 Analogy

🔦 Like searching for a missing book in a library — go floor by floor, shelf by shelf, slot by slot. Binary search is like knowing books are sorted alphabetically — jump to the right section.

🔮 Search Visual

Enter a value and click Search.
C · Searching
// Search Single Value (Linear)
void searchSingle(int arr[L][R][C],
                  int target){
  for(int l=0;l<L;l++)
    for(int r=0;r<R;r++)
      for(int c=0;c<C;c++)
        if(arr[l][r][c]==target){
          printf("Found at [%d][%d][%d]\n",
                 l,r,c);
          return;
        }
  printf("Not found\n");
}

// Search Multiple Occurrences
void searchAll(int arr[L][R][C],
               int target){
  int found=0;
  for(int l=0;l<L;l++)
    for(int r=0;r<R;r++)
      for(int c=0;c<C;c++)
        if(arr[l][r][c]==target){
          printf("[%d][%d][%d] ",l,r,c);
          found++;
        }
  printf("\nTotal: %d\n",found);
}

// Layer-wise Search
void searchInLayer(int l, int target){
  for(int r=0;r<R;r++)
    for(int c=0;c<C;c++)
      if(arr[l][r][c]==target)
        printf("Layer %d:[%d][%d]\n",l,r,c);
}

Traversal Operations
🚶 Standard, Directional & Pattern Traversal TRAVERSE

📖 Theory

Traversal means visiting every element in a specific order. Different traversal orders serve different purposes:

  • Layer-wise → process layer by layer (e.g., process each floor of building)
  • Row-wise → across all layers, row by row
  • Column-wise → column-first order
  • Front/Back, Top/Bottom → directional, for graphics/simulation
  • Spiral → snaking pattern through the array
  • Zigzag → alternating direction each row
  • Diagonal → along diagonals of each layer

💡 Analogy

🌀 Like reading a book — layer-wise is chapter by chapter, row-wise is scanning each line across all chapters, zigzag is like a snake game path.

🔮 Traversal Animation

Choose a traversal method to animate.
C · All Traversal Types
// Layer-wise Traversal
for(int l=0;l<L;l++)
  for(int r=0;r<R;r++)
    for(int c=0;c<C;c++)
      printf("%d ",arr[l][r][c]);

// Row-wise Traversal
for(int r=0;r<R;r++)
  for(int l=0;l<L;l++)
    for(int c=0;c<C;c++)
      printf("%d ",arr[l][r][c]);

// Back to Front
for(int l=L-1;l>=0;l--)
  for(int r=0;r<R;r++)
    for(int c=0;c<C;c++)
      printf("%d ",arr[l][r][c]);

// Zigzag Traversal (per layer)
for(int l=0;l<L;l++)
  for(int r=0;r<R;r++){
    if(r%2==0) // left to right
      for(int c=0;c<C;c++)
        printf("%d ",arr[l][r][c]);
    else        // right to left
      for(int c=C-1;c>=0;c--)
        printf("%d ",arr[l][r][c]);
  }

// Diagonal Traversal (layer 0)
for(int i=0;i<R&&i<C;i++)
  printf("%d ",arr[0][i][i]);

Transformation Operations
🔄 Structural Transformations & Reshaping TRANSFORM

📖 Theory

Transformations change the structure or orientation of the array without necessarily changing the values. Reshaping changes the dimensional representation.

  • Rotate Layers → rotate 2D grid of each layer 90° clockwise
  • Flip Horizontally → reverse each row's columns
  • Flip Vertically → reverse row order in each layer
  • Transpose 2D Layers → swap [r][c] ↔ [c][r] in each layer
  • Flatten 3D to 1D → unroll all elements into a single array
  • Convert 1D to 3D → pack a 1D array back into 3D shape

💡 Analogy

🎲 Like rotating, flipping, or unfolding a Rubik's cube — the pieces are the same, just rearranged. Flatten is like unfolding a box into a flat sheet.

🔮 Transformation Demo

Click a transformation to apply it to Layer 0.
C · Transformations
// Rotate Layer 90° Clockwise
void rotateLayer(int arr[R][C],
                  int tmp[C][R]){
  for(int r=0;r<R;r++)
    for(int c=0;c<C;c++)
      tmp[c][R-1-r]=arr[r][c];
}

// Flip Horizontally (mirror rows)
for(int l=0;l<L;l++)
  for(int r=0;r<R;r++){
    int lo=0, hi=C-1;
    while(lo<hi){
      int t=arr[l][r][lo];
      arr[l][r][lo++]=arr[l][r][hi];
      arr[l][r][hi--]=t;
    }
  }

// Transpose Layer (square only)
for(int l=0;l<L;l++)
  for(int r=0;r<R;r++)
    for(int c=r+1;c<C;c++){
      int t=arr[l][r][c];
      arr[l][r][c]=arr[l][c][r];
      arr[l][c][r]=t;
    }

// Flatten 3D to 1D
int flat[L*R*C], idx=0;
for(int l=0;l<L;l++)
  for(int r=0;r<R;r++)
    for(int c=0;c<C;c++)
      flat[idx++]=arr[l][r][c];

// Convert 1D to 3D
idx=0;
for(int l=0;l<L;l++)
  for(int r=0;r<R;r++)
    for(int c=0;c<C;c++)
      arr[l][r][c]=flat[idx++];

Mathematical Operations
➗ Arithmetic, Statistical & Layer Calculations MATH

📖 Theory

Mathematical operations on 3D arrays work element-wise. Arrays must have the same dimensions for addition/subtraction. Scalar multiplication applies one value to every element.

  • Addition / Subtraction → result[l][r][c] = a[l][r][c] ± b[l][r][c]
  • Element-wise Multiply → result[l][r][c] = a[l][r][c] × b[l][r][c]
  • Sum / Average / Max / Min → aggregate statistics over all elements
  • Layer/Row/Column Sum → partial aggregations

💡 Analogy

📊 Like combining sales data from two identical store layouts — same floor, same shelf, same slot — add corresponding entries. Layer sum = total sales per floor.

for(int l=0;l<L;l++)

// Scalar Multiplication
int scalar=2;
for(int l=0;l<L;l++)
  for(int r=0;r<R;r++)
    for(int c=0;c<C;c++)
      arr[l][r][c]*=scalar;

// Find Sum of All
int findSum(int arr[L][R][C]){
  int sum=0;
  for(int l=0;l<L;l++)
    for(int r=0;r<R;r++)
      for(int c=0;c<C;c++)
        sum+=arr[l][r][c];
  return sum;
}

// Find Maximum
int findMax(int arr[L][R][C]){
  int mx=arr[0][0][0];
  for(int l=0;l<L;l++)
    for(int r=0;r<R;r++)
      for(int c=0;c<C;c++)
        if(arr[l][r][c]>mx)
          mx=arr[l][r][c];
  return mx;
}

// Layer Sum
int layerSum(int arr[L][R][C], int l){
  int s=0;
  for(int r=0;r<R;r++)
    for(int c=0;c<C;c++)
      s+=arr[l][r][c];
  return s;
}

Sorting Operations
📊 Layer, Row, Column & Global Sorting SORT

📖 Theory

Sorting can be applied at different granularities. We use simple Bubble Sort to keep it beginner-friendly:

  • Sort Individual Layers → treat each layer's 2D grid as a sequence and sort it
  • Sort Rows in Each Layer → sort each individual row within every layer
  • Sort Columns in Each Layer → sort values in each column across rows
  • Flatten and Sort Entire Array → unroll to 1D, sort, refill 3D structure

💡 Analogy

📚 Like sorting books — you can sort each shelf individually, or take all books out, sort globally, then put them back in order.

🔮 Sort Animation

Click a sort to organize the array.
C · Sorting Operations
// Bubble Sort Helper
void bubbleSort(int *a, int n){
  for(int i=0;i<n-1;i++)
    for(int j=0;j<n-1-i;j++)
      if(a[j]>a[j+1]){
        int t=a[j];a[j]=a[j+1];a[j+1]=t;
      }
}

// Sort Rows in Each Layer
for(int l=0;l<L;l++)
  for(int r=0;r<R;r++)
    bubbleSort(arr[l][r], C);

// Sort Columns in Each Layer
int col[R];
for(int l=0;l<L;l++)
  for(int c=0;c<C;c++){
    for(int r=0;r<R;r++) col[r]=arr[l][r][c];
    bubbleSort(col,R);
    for(int r=0;r<R;r++) arr[l][r][c]=col[r];
  }

// Flatten and Sort Entire 3D Array
int flat[L*R*C], idx=0;
for(int l=0;l<L;l++)
  for(int r=0;r<R;r++)
    for(int c=0;c<C;c++)
      flat[idx++]=arr[l][r][c];
bubbleSort(flat, L*R*C);
idx=0;
for(int l=0;l<L;l++)
  for(int r=0;r<R;r++)
    for(int c=0;c<C;c++)
      arr[l][r][c]=flat[idx++];

Validation Operations
✅ Structure Validation & Comparison VALIDATE

📖 Theory

Validation ensures an array meets expected conditions before processing. This prevents bugs and crashes:

  • Check Empty → verify all elements are 0 (or no valid count)
  • Check Cubic → verify L == R == C (all dimensions equal)
  • Check Equal Dimensions → compare two arrays for same shape
  • Compare Two Arrays → element-by-element equality check

💡 Analogy

🔬 Like a quality check in manufacturing — verify a part has the right dimensions before installing it. Compare two parts to confirm they're identical.

C · Validation Operations
// Check Empty 3D Array (all zeros)
int isEmpty(int arr[L][R][C]){
  for(int l=0;l<L;l++)
    for(int r=0;r<R;r++)
      for(int c=0;c<C;c++)
        if(arr[l][r][c]!=0) return 0;
  return 1;
}

// Check Cubic Structure (L==R==C)
int isCubic(int l, int r, int c){
  return (l==r && r==c);
}

// Check Equal Dimensions
int equalDims(int l1,int r1,int c1,
               int l2,int r2,int c2){
  return(l1==l2 && r1==r2 && c1==c2);
}

// Compare Two 3D Arrays (Check Equality)
int areEqual(int a[L][R][C],
              int b[L][R][C]){
  for(int l=0;l<L;l++)
    for(int r=0;r<R;r++)
      for(int c=0;c<C;c++)
        if(a[l][r][c]!=b[l][r][c])
          return 0; // not equal
  return 1; // fully equal
}

Memory Operations
🧠 Static & Dynamic Memory, Copy, Clone, Free MEMORY

📖 Theory

Memory management is critical in C since there is no garbage collector. You must manually allocate and free memory for dynamic arrays.

  • Static Allocation → fixed size at compile time: int arr[2][3][4]
  • Dynamic Allocation → use malloc/calloc at runtime for flexible sizes
  • Copy → element-by-element copy (shallow)
  • Clone → allocate new memory and deep copy all values
  • Resize → allocate new bigger/smaller block, copy data, free old
  • Free → release each malloc'd level (pointer to pointer to pointer)

💡 Analogy

🏠 Static = buying a fixed house; Dynamic = renting a flexible space. You must always "return the keys" (free) when done, or memory leaks!

Address of arr[l][r][c] = BaseAddr + (l×R×C + r×C + c) × sizeof(int)

🔮 Memory Layout

Click to visualize memory layout.
C · Memory Ops (Dynamic 3D)
#include <stdlib.h>

// Dynamic Allocation (L layers, R rows, C cols)
int ***alloc3D(int L,int R,int C){
  int ***arr = malloc(L*sizeof(int**));
  for(int l=0;l<L;l++){
    arr[l] = malloc(R*sizeof(int*));
    for(int r=0;r<R;r++)
      arr[l][r] = malloc(C*sizeof(int));
  }
  return arr;
}

// Copy 3D Array
void copy3D(int ***src, int ***dst,
             int L,int R,int C){
  for(int l=0;l<L;l++)
    for(int r=0;r<R;r++)
      for(int c=0;c<C;c++)
        dst[l][r][c]=src[l][r][c];
}

// Clone (allocate + copy)
int ***clone3D(int***src,
               int L,int R,int C){
  int ***dst=alloc3D(L,R,C);
  copy3D(src,dst,L,R,C);
  return dst;
}

// Free Memory — CRITICAL!
void free3D(int***arr,int L,int R){
  for(int l=0;l<L;l++){
    for(int r=0;r<R;r++)
      free(arr[l][r]); // free rows
    free(arr[l]);      // free layer
  }
  free(arr);           // free outer
}

Conversion Operations
🔀 Representation & Specialized Conversion CONVERT

📖 Theory

Conversion changes HOW the data is stored or represented — not the data itself:

  • 3D to 2D → collapse layers into a longer 2D array
  • 3D to 1D → fully flatten to linear array (row-major order)
  • 3D to Linked Structure → each element becomes a node with 3D coordinates
  • Sparse Representation → store only non-zero elements with coordinates to save memory
  • Tensor Representation → multi-dimensional array used in machine learning (same concept!)

💡 Analogy

🗜️ Sparse = a checklist of only non-empty boxes instead of listing every empty box too. Saves enormous space when most values are zero (like a mostly empty warehouse).

Click "Show Sparse" to extract non-zero elements.

// Convert 3D to 2D (collapse layers)
int arr2D[L*R][C];

  for(int r=0;r<R;r++){
    for(int c=0;c<C;c++)
      arr2D[row][c]=arr[l][r][c];
    row++;
  }

// Convert 3D to 1D (flatten)
int flat[L*R*C], idx=0;
for(int l=0;l<L;l++)
  for(int r=0;r<R;r++)
    for(int c=0;c<C;c++)
      flat[idx++]=arr[l][r][c];

// Sparse 3D Representation
typedef struct {
  int l,r,c,val;
} SparseNode;

SparseNode sparse[L*R*C];
int cnt=0;
for(int l=0;l<L;l++)
  for(int r=0;r<R;r++)
    for(int c=0;c<C;c++)
      if(arr[l][r][c]!=0)
        sparse[cnt++]=(SparseNode)
          {l,r,c,arr[l][r][c]};

// 3D Array to Linked List Node
struct Node{

  struct Node *next;
};

Utility Operations
🛠️ Swapping & Display Operations UTILITY

📖 Theory

Utility operations are frequently-needed helpers. Swapping exchanges content between layers/rows/columns/elements. Display operations present the array in human-readable formats.

  • Swap Layers → exchange all elements between two complete layers
  • Swap Rows → exchange two rows within the same or different layers
  • Swap Columns → exchange column values across all rows in a layer
  • Swap Elements → exchange two individual values
  • Print / Display Layer-wise → formatted console output per layer
  • Visual Cube Display → ASCII art representation of the 3D structure

💡 Analogy

🏨 Swap floors = move all guests from Floor 1 to Floor 2 and vice versa. Swap rooms = two specific guests switch rooms.

🌐 3D Grid Simulation & Voxel Manipulation ADVANCED

📖 Theory

A 3D Grid Simulation uses the array to model a physical or logical 3D space — each cell holds a state (occupied/empty, temperature, etc.). A Voxel (volume pixel) is exactly one cell in a 3D grid — the 3D equivalent of a pixel.

  • Game worlds (Minecraft-style terrain) → each voxel = a block type
  • Medical imaging (CT/MRI) → each voxel = tissue density
  • Fluid simulation → each voxel = pressure value
  • Cellular automaton (Conway's Game of Life 3D)

💡 Analogy

🎮 Minecraft! Every block in the game world is one voxel in a massive 3D array. Setting arr[x][y][z] = 1 places a stone block; arr[x][y][z] = 0 means air.

Cellular automaton: cells live/die based on neighbors.
C · Voxel & Grid Simulation
#define N 10
int grid[N][N][N]; // 0=empty,1=filled

// Set a Voxel
void setVoxel(int x,int y,int z,
               int val){
  if(x>=0&&x<N&&y>=0&&y<N
     &&z>=0&&z<N)
    grid[x][y][z]=val;
}

// Count Neighbor Voxels (6-connected)
int neighbors(int x,int y,int z){
  int n=0;
  int dx[]={-1,1,0,0,0,0};
  int dy[]={0,0,-1,1,0,0};
  int dz[]={0,0,0,0,-1,1};
  for(int i=0;i<6;i++){
    int nx=x+dx[i];
    int ny=y+dy[i];
    int nz=z+dz[i];
    if(nx>=0&&nx<N&&ny>=0
       &&ny<N&&nz>=0&&nz<N)
      n+=grid[nx][ny][nz];
  }
  return n;
}

// 3D Game of Life Step
void simStep(){
  int next[N][N][N]={0};
  for(int x=0;x<N;x++)
    for(int y=0;y<N;y++)
      for(int z=0;z<N;z++){
        int nb=neighbors(x,y,z);
        if(grid[x][y][z])
          next[x][y][z]=(nb==4||nb==5)?1:0;
        else
          next[x][y][z]=(nb==5)?1:0;
      }
  memcpy(grid,next,sizeof(grid));
}
🎨 Graphics Operations — Volume Rendering & 3D Pixel Processing GRAPHICS

📖 Theory

Volume Rendering converts a 3D array of voxel values (density, color, opacity) into a 2D image by casting rays through the volume. 3D Pixel Processing treats a 3D array as a stack of image frames and applies filters (brightness, blur) frame by frame.

  • arr[frame][row][col] = pixel value → 3D video buffer
  • arr[z][y][x] = density → medical CT scan volume
  • Maximum Intensity Projection (MIP): for each (x,y), find max z-value

💡 Analogy

🩻 CT scans are literally 3D arrays! Each slice (layer) is a 2D X-ray image. The full scan is arr[slice][row][column] = density. Doctors "render" by viewing slices or projections.

C · Volume Rendering & Pixel Processing
#define Z 10  // depth slices
#define H 100 // height
#define W 100 // width

int volume[Z][H][W]; // voxel density

// Maximum Intensity Projection (MIP)
// Projects 3D volume onto 2D image
void maxIntensityProj(int proj[H][W]){
  for(int y=0;y<H;y++)
    for(int x=0;x<W;x++){
      int mx=0;
      for(int z=0;z<Z;z++)
        if(volume[z][y][x]>mx)
          mx=volume[z][y][x];
      proj[y][x]=mx;
    }
}

// Brightness Filter on 3D Pixel Buffer
int video[30][H][W]; // 30 frames
void brighten(int amount){
  for(int f=0;f<30;f++)
    for(int y=0;y<H;y++)
      for(int x=0;x<W;x++){
        video[f][y][x]+=amount;
        if(video[f][y][x]>255)
          video[f][y][x]=255;
      }
}

// 3x3x3 Box Blur (averaging neighbors)
int boxBlur(int arr[Z][H][W],
             int z,int y,int x){
  int sum=0, cnt=0;
  for(int dz=-1;dz<=1;dz++)
    for(int dy=-1;dy<=1;dy++)
      for(int dx=-1;dx<=1;dx++){
        int nz=z+dz,ny=y+dy,nx=x+dx;
        if(nz>=0&&nz<Z&&ny>=0
           &&ny<H&&nx>=0&&nx<W){
          sum+=arr[nz][ny][nx]; cnt++;
        }
      }
  return sum/cnt;
}
🔬 Scientific Computing & Dynamic Programming ADVANCED

📖 Theory

Tensor Calculations extend matrix operations to 3D (and beyond). A 3D array IS a rank-3 tensor. Matrix Stack Operations apply 2D matrix operations (multiply, invert) to each layer.

Dynamic Programming (DP) stores solutions to sub-problems in a table to avoid recomputing. 3D DP tables handle problems with THREE independent dimensions, such as:

  • 3D Knapsack → weight + volume + count constraints
  • Edit Distance with wildcards → 3 string parameters
  • Path in 3D grid → minimum cost from corner to corner

💡 Analogy

📝 3D DP is like a 3D memo pad. Each cell dp[i][j][k] stores the answer to a subproblem. Instead of solving the same puzzle again, just look it up in the table!

// Minimum Cost Path in 3D Grid
// dp[l][r][c] = min cost to reach [l][r][c]
int cost[L][R][C]; // given costs
int dp[L][R][C];

void minPath3D(){
  dp[0][0][0]=cost[0][0][0];
  for(int l=0;l<L;l++)
    for(int r=0;r<R;r++)
      for(int c=0;c<C;c++){
        if(l==0&&r==0&&c==0) continue;
        int mn=INT_MAX;
        if(l>0) mn=min(mn,dp[l-1][r][c]);
        if(r>0) mn=min(mn,dp[l][r-1][c]);
        if(c>0) mn=min(mn,dp[l][r][c-1]);
        dp[l][r][c]=mn+cost[l][r][c];
      }
}

// Multi-dimensional Memoization
// (top-down DP with memo table)
int memo[N][N][N];
int vis[N][N][N]={0};

int solve(int i,int j,int k){
  if(i==N-1&&j==N-1&&k==N-1)
    return cost[i][j][k];
  if(vis[i][j][k]) return memo[i][j][k];
  vis[i][j][k]=1;
  int ans=INT_MAX;
  if(i+1<N) ans=min(ans,solve(i+1,j,k));
  if(j+1<N) ans=min(ans,solve(i,j+1,k));
  if(k+1<N) ans=min(ans,solve(i,j,k+1));
  return memo[i][j][k]=ans+cost[i][j][k];
}

// Matrix Stack: multiply each layer
// by a fixed 2D matrix M
void matStackMul(int arr[L][R][C],
                  int M[R][C],
                  int out[L][R][C]){
  for(int l=0;l<L;l++)
    for(int r=0;r<R;r++)
      for(int c=0;c<C;c++){
        out[l][r][c]=0;
        for(int k=0;k<C;k++)
          out[l][r][c]+=arr[l][r][k]*M[k][c];
      }
}
