What is a Priority Queue?

A Priority Queue is a special data structure where each element has a priority value. Elements are served based on their priority, not the order they arrived. The element with the highest (or lowest) priority is always processed first.

In a Sorted Array implementation, we maintain the array in sorted order at all times — either ascending or descending — so that the highest priority element is always at a known, fixed position.

Ascending vs Descending Order

Ascending (min→max) Lowest priority value = first element. Highest priority element = last element (index size-1). Deletion of highest is O(1), insertion is O(n).

Descending (max→min) Highest priority value = first element. Deletion of highest is O(1) (front), insertion is O(n) to find correct spot.

🏥
Real-World Analogy
Think of a hospital emergency room. Patients are not treated in the order they arrive — a critical heart attack patient (priority 10) is treated before someone with a minor sprain (priority 2). The sorted priority queue works exactly like this: elements are always kept in sorted order by priority, and the most critical (highest priority) is always served first.

Priority Queue Initialization
// Setting up the structure before use
Theory

Before we can use a priority queue, we must initialize it. This means:

• Defining a fixed-size array to hold elements and their priorities.

• Setting size = 0 to indicate the queue is empty.

• Defining MAX_CAPACITY so we never overflow.

• Deciding if we want ascending (min→max) or descending (max→min) order.

In C, we use a struct to bundle the array, size, and capacity together into one clean unit.

Key Concepts

MAX_SIZE Maximum elements allowed. Cannot grow dynamically in a basic array implementation.

size Current number of elements. Starts at 0. Goes up on insert, down on delete.

Order Rule Ascending = arr[0] is minimum. Descending = arr[0] is maximum. Decide once, keep consistent.

📋
Real-World Analogy
Initialization is like setting up a new numbered ticket booth. You place an empty ticket rack (array) on the counter, hang a sign saying "Max 10 tickets" (MAX_SIZE), and flip the counter to zero (size = 0). Now you're ready to serve customers in priority order.

C Language
/* ===== Priority Queue Initialization ===== */
#include <stdio.h>

#define MAX_SIZE 10   /* Maximum capacity of the queue */

/* Structure to hold one element */
struct Element {
    int data;      /* The actual value/data */
    int priority;  /* Priority (higher = served first) */
};

/* Structure for the entire Priority Queue */
struct PriorityQueue {
    struct Element arr[MAX_SIZE];  /* Array of elements */
    int size;                      /* Current number of elements */
    int order;  /* 0 = Ascending, 1 = Descending */
};

/* Initialize the priority queue */
void initQueue(struct PriorityQueue *pq, int order) {
    pq->size = 0;    /* Start empty */
    pq->order = order;
    printf("Queue initialized. Max=%d, Order=%s\n",
           MAX_SIZE, order ? "DESCENDING" : "ASCENDING");
}

int main() {
    struct PriorityQueue pq;
    initQueue(&pq, 0);  /* 0 = Ascending order */
    return 0;
}
02
Insert / Enqueue Operation
// Adding elements while maintaining sorted order
Theory — How Insertion Works

Inserting into a sorted priority queue has 3 steps:

Step 1: Check if the queue is full. If yes, reject insertion.

Step 2: Find the correct position for the new element by scanning the array and comparing priorities.

Step 3: Shift all elements after that position one place to the right, then place the new element. This costs O(n) time in the worst case.

The insert is called stable if elements with equal priority maintain their original insertion order.

📬
Real-World Analogy
Imagine a sorted stack of exam papers ordered by score. When a new paper arrives with score 78, you slide through the pile to find where 78 fits, push the lower-scored papers aside, and slot it in. The pile stays sorted every time — but you have to do some shifting work each time you add.

C Language
/* ===== Insert / Enqueue (Ascending Order by Priority) ===== */

void insert(struct PriorityQueue *pq, int data, int priority) {
    /* Step 1: Check if queue is full */
    if (pq->size == MAX_SIZE) {
        printf("Queue is FULL! Cannot insert.\n");
        return;
    }

    /* Step 2: Find the correct position */
    int i = pq->size - 1;

    /* Step 3: Shift elements right until we find correct spot */
    while (i >= 0 && pq->arr[i].priority > priority) {
        pq->arr[i + 1] = pq->arr[i];  /* Shift right */
        i--;
    }

    /* Step 4: Place new element at correct position */
    pq->arr[i + 1].data = data;
    pq->arr[i + 1].priority = priority;
    pq->size++;

    printf("Inserted: data=%d, priority=%d\n", data, priority);
}
03
Delete Highest Priority Element
// O(1) — the power of sorted storage
Theory

Since we keep the array sorted in ascending order (low→high priority), the highest priority element is always at index size - 1 (the last position).

Deleting it is incredibly simple: just read the last element, then decrease size by 1. No shifting needed! This is O(1) time.

If using descending order, the highest is at index 0 (front), and after removal, all elements must shift left — making deletion O(n).

Step by Step

1. Check if queue is empty. If yes, report underflow.

2. Read the element at arr[size-1].

3. Decrease size by 1. The element is logically removed.

4. Return the deleted element's data (optional).

🚑
Real-World Analogy
The doctor always takes the patient at the top of the sorted waiting list (the most critical one). Because the list is already sorted, the doctor doesn't need to search — they just take the last card off the top of the sorted deck. Fast, instant, no rearranging!

C Language
/* ===== Delete Highest Priority Element ===== */

int deleteHighest(struct PriorityQueue *pq) {
    /* Check if empty */
    if (pq->size == 0) {
        printf("Queue is EMPTY! Underflow.\n");
        return -1;
    }

    /* In ascending order: highest priority is at last index */
    int highIdx = pq->size - 1;
    int deletedData = pq->arr[highIdx].data;
    int deletedPri  = pq->arr[highIdx].priority;

    /* Simply reduce size — O(1)! */
    pq->size--;

    printf("Deleted: data=%d, priority=%d\n",
           deletedData, deletedPri);
    return deletedData;
}

/* Time Complexity: O(1) — No shifting needed! */
04
Delete Lowest Priority Element
// O(n) — shifting required from the front
Theory

In ascending order, the lowest priority element is at index 0 (the front of the array).

Removing it requires shifting all remaining elements one position to the left to fill the gap. This costs O(n) time.

Example: Array is [P1, P3, P5, P8]. Remove P1 → shift [P3, P5, P8] left → result [P3, P5, P8]. Three elements moved!

🎟️
Real-World Analogy
Imagine a cinema queue where all people stand in order of ticket priority. If the lowest-priority person at the back leaves, the queue is fine. But if we remove the person at the very front (lowest priority), everyone else must step forward — that's the O(n) shift!

C Language
/* ===== Delete Lowest Priority Element ===== */

int deleteLowest(struct PriorityQueue *pq) {
    if (pq->size == 0) {
        printf("Queue is EMPTY!\n");
        return -1;
    }

    /* In ascending order: lowest is at index 0 */
    int deletedData = pq->arr[0].data;
    int deletedPri  = pq->arr[0].priority;

    /* Shift all elements left to fill the gap — O(n) */
    int i;
    for (i = 0; i < pq->size - 1; i++) {
        pq->arr[i] = pq->arr[i + 1];  /* Shift left */
    }
    pq->size--;

    printf("Deleted lowest: data=%d, priority=%d\n",
           deletedData, deletedPri);
    return deletedData;
}

/* Time Complexity: O(n) — must shift all elements */
05
Peek Highest Priority Element
// Read without removing — O(1)
Theory

Peek means look without touching. We read the highest priority element but do not remove it from the queue.

In ascending order, the highest priority is at arr[size-1]. Simply read and return it — no modification to the queue at all. This is O(1).

This is very useful when you want to check what will be served next before deciding to actually remove it.

Read-Only Rule

The key difference between Peek and Delete:

• Peek: size stays the same. Element stays in queue. Just returns the value.

• Delete: size decreases by 1. Element is gone forever.

👀
Real-World Analogy
A restaurant chef peeks at the next order ticket on the wheel without removing it — just to see what's coming next. The ticket stays pinned until the chef is ready to cook it. Peeking costs no effort (O(1)) because the sorted list keeps the most urgent task at a known position.

Peek Lowest Priority Element
// Also O(1) — known position at index 0
Theory

In ascending order, the lowest priority element is always at index 0 (the beginning of the array).

Just like peeking the highest, this is a read-only operation. We access arr[0] directly and return its value without changing anything. This is O(1).

The beauty of sorted storage: both ends are always accessible in constant time!

In a sorted filing cabinet, the least urgent folder is always at the very front. You can glance at its label in one second (O(1)) without moving anything. The sorted order guarantees you always know exactly where it is.

IsEmpty Operation
// Guard before every delete or peek
Theory

IsEmpty checks whether the priority queue has zero elements. This is a safety check used before any peek or delete operation.

If you try to delete from an empty queue, you get an underflow error. Always check IsEmpty first!

Implementation: simply check if size == 0. Returns 1 (true) if empty, 0 (false) otherwise. O(1).

🪣
Real-World Analogy
Before a barista tries to pour from a coffee pot, they glance at it: "Is there anything left?" If the pot is empty and they pour anyway, nothing comes out (underflow!). IsEmpty is that quick glance — cheap, O(1), and prevents errors.
C Language
/* ===== IsEmpty Operation ===== */

int isEmpty(struct PriorityQueue *pq) {
    return (pq->size == 0);  /* Returns 1 if empty, 0 if not */
}

/* Safe usage pattern: */
void safePeek(struct PriorityQueue *pq) {
    if (isEmpty(pq)) {
        printf("Cannot peek: queue is empty!\n");
        return;
    }
    /* Safe to peek now */
    printf("Top priority: %d\n",
           pq->arr[pq->size-1].priority);
}

/* Time Complexity: O(1) */
08
IsFull Operation
// Prevent overflow before insert
Theory

IsFull checks if the array has reached its maximum capacity. Just as IsEmpty guards against underflow, IsFull guards against overflow.

If you try to insert when the array is already full, you will write past the array boundary — causing undefined behavior and crashes in C. Always check IsFull before insertion!

Implementation: check if size == MAX_SIZE. Returns 1 (true) if full. O(1).

🚗
Real-World Analogy
A parking lot with 10 spaces has a sensor at the entrance. If all 10 spots are taken (IsFull = true), the barrier stays down and a "FULL" sign lights up. No new car can enter — preventing a crash (literally). IsFull is that sensor check.
C Language
/* ===== IsFull Operation ===== */

int isFull(struct PriorityQueue *pq) {
    return (pq->size == MAX_SIZE);
}

/* Safe insert pattern: */
void safeInsert(struct PriorityQueue *pq,
                int data, int priority) {
    if (isFull(pq)) {
        printf("Queue FULL! Cannot insert data=%d\n", data);
        return;
    }
    insert(pq, data, priority);
}

/* Time Complexity: O(1) */

// Print all elements with priorities clearly
Theory

Display prints all elements in the priority queue in their current sorted order. It's a read-only traversal that helps with debugging and visualization.

We loop from index 0 to size-1, printing each element's data and priority. In ascending order, arr[0] is shown first (lowest priority) and arr[size-1] last (highest). Time: O(n).

Traverse Priority Queue
// Forward and reverse traversal
Theory

Traversal means visiting each element exactly once. Unlike display (which just prints), traversal can be used to perform any operation on each element — sum all priorities, find an element, apply a transformation, etc.

Forward traversal: index 0 → size-1 (low priority to high priority). Use to process least urgent first.

Reverse traversal: index size-1 → 0 (high priority to low priority). Use to process most urgent first. Time: O(n) for both.

void traverseReverse(struct PriorityQueue *pq) {
    printf("Reverse traversal:\n");
    int i;
    for (i = pq->size - 1; i >= 0; i--) {
        printf("  [%d] data=%d pri=%d\n",
               i, pq->arr[i].data,
               pq->arr[i].priority);
    }
}

11
Search Element
// Linear scan — O(n)
Theory

Search scans the array from left to right looking for an element by its data value or priority. Since we sort by priority and search by data, we cannot do binary search on data — we must do a linear search.

If we search by priority, we could use binary search (since priorities are sorted) for O(log n). But generally, search is O(n).

Returns the index of the found element, or -1 if not found.

🔎
Real-World Analogy
Searching for a patient named "John" in a hospital queue sorted by urgency — you can't skip ahead just because it's sorted by priority (not name). You must check each patient card one by one until you find John. That's linear search: O(n) in the worst case.

/* ===== Search Element by Data Value ===== */

int search(struct PriorityQueue *pq, int targetData) {
    int i;
    for (i = 0; i < pq->size; i++) {
        if (pq->arr[i].data == targetData) {
            printf("Found at index %d: data=%d, priority=%d\n",

                   pq->arr[i].priority);
            return i;  /* Return position */
        }
    }
    printf("Element %d NOT FOUND.\n", targetData);
    return -1;  /* Not found */
}

12
Update Element Priority
// Search → Remove → Re-insert in sorted position
Theory — 3-Step Process

Updating a priority is a 3-phase operation that uses operations we already know:

Phase 1 — Search: Find the element by its data value. O(n).

Phase 2 — Remove: Delete the old entry at that index by shifting elements left. O(n).

Phase 3 — Re-insert: Insert the element with its new priority at the correct sorted position. O(n).

Total: O(n) time. We cannot just change the priority in place because that would break sorted order.

🏥
Real-World Analogy
A patient who arrived with a minor injury (priority 2) suddenly develops complications and is now critical (priority 9). The nurse removes their card from its current position in the sorted list, and places it near the top — near the most urgent cases. The whole list stays sorted throughout.

C Language
/* ===== Update Element Priority ===== */

void updatePriority(struct PriorityQueue *pq,
                    int targetData, int newPriority) {
    /* Phase 1: Search for the element */
    int pos = -1, i;
    for (i = 0; i < pq->size; i++) {
        if (pq->arr[i].data == targetData) {
            pos = i; break;
        }
    }
    if (pos == -1) {
        printf("Element %d not found!\n", targetData);
        return;
    }

    /* Phase 2: Remove old entry (shift left) */
    for (i = pos; i < pq->size - 1; i++) {
        pq->arr[i] = pq->arr[i + 1];
    }
    pq->size--;

    /* Phase 3: Re-insert with new priority */
    insert(pq, targetData, newPriority);
    printf("Updated data=%d to priority=%d\n",
           targetData, newPriority);
}

Count / Size Operation
// Instant size retrieval — O(1)
Theory

The count operation returns the total number of elements currently in the priority queue. Since we maintain the size variable and update it on every insert and delete, this is a simple return of that variable.

This is O(1) — no need to count elements one by one. The size field always knows the current count.

Complexity Analysis
ApproachTime
Using size field (our method)O(1)
Counting by loop (bad)O(n)

Always maintain a separate size counter — counting by looping each time is wasteful.

C Language
/* ===== Count / Size Operation ===== */

int count(struct PriorityQueue *pq) {
    return pq->size;  /* O(1) — just return the field */
}

/* Usage examples: */
void demo(struct PriorityQueue *pq) {

    printf("Slots remaining: %d\n", MAX_SIZE - count(pq));
    printf("Usage: %.1f%%\n",
           (float)count(pq) / MAX_SIZE * 100);
}

Clear / Destroy Priority Queue
// Reset everything back to zero
Theory

Clear (or Destroy) removes all elements from the queue and resets it to its initial empty state. This is useful when you want to reuse the queue structure without creating a new one.

For a static array-based queue in C, clearing is as simple as setting size = 0. The old data is still in memory but logically invisible — subsequent insertions will overwrite those positions. O(1).

For safety, you can also overwrite old values with zeros (optional, O(n)).

🗑️
Real-World Analogy
Clearing a priority queue is like erasing a whiteboard — you don't remove the whiteboard itself, you just wipe all the writing off. The board is ready for new content immediately. Setting size=0 is that eraser swipe: instant and O(1).

15
Sort Priority Queue
// Verify or enforce sorted order
Theory

In a sorted array-based priority queue, the array should always be sorted — the insert operation maintains this. However, if data is loaded externally or corruption occurs, we may need to re-sort.

We use Insertion Sort here because it's simple, works in-place, and is efficient when the array is nearly sorted (O(n) best case). For a completely unsorted array, it costs O(n²).

We can sort in ascending (min-first) or descending (max-first) order by changing the comparison operator.

C Language
/* ===== Sort Priority Queue (Insertion Sort) ===== */

/* Ascending: arr[0]=min, arr[size-1]=max */
void sortAscending(struct PriorityQueue *pq) {
    int i, j;
    struct Element key;

    for (i = 1; i < pq->size; i++) {
        key = pq->arr[i];
        j = i - 1;

        /* Shift larger elements right */
        while (j >= 0 &&
               pq->arr[j].priority > key.priority) {
            pq->arr[j + 1] = pq->arr[j];
            j--;
        }
        pq->arr[j + 1] = key;
    }
    printf("Sorted in ascending order.\n");
}

/* Descending: change > to < in the while condition */
void sortDescending(struct PriorityQueue *pq) {
    int i, j;
    struct Element key;
    for (i = 1; i < pq->size; i++) {
        key = pq->arr[i];
        j = i - 1;
        while (j >= 0 &&
               pq->arr[j].priority < key.priority) {
            pq->arr[j + 1] = pq->arr[j];
            j--;
        }
        pq->arr[j + 1] = key;
    }
    printf("Sorted in descending order.\n");
}

Merge / Concatenate Priority Queues
// Two-pointer merge — O(n+m)
Theory — Two-Pointer Merge

Merging two sorted priority queues into one sorted result uses the classic two-pointer technique:

• Maintain one pointer (i) at the start of Queue A, and another (j) at the start of Queue B.

• Compare priorities at A[i] and B[j]. Copy the smaller one into the result and advance that pointer.

• When one queue runs out, copy remaining elements of the other.

Time: O(n + m) where n and m are the sizes of the two queues. Much better than re-inserting all elements!

🃏
Real-World Analogy
Two sorted card decks are being merged into one. One person holds deck A, another holds deck B. They compare their top cards, place the lower one into the merged pile, and draw the next. This two-person collaboration is the two-pointer merge — efficient because both decks are already sorted!

C Language
/* ===== Merge Two Sorted Priority Queues ===== */

void mergeQueues(
    struct PriorityQueue *a,
    struct PriorityQueue *b,
    struct PriorityQueue *result)
{
    int i = 0, j = 0, k = 0;

    /* Two-pointer: compare and pick smaller priority */
    while (i < a->size && j < b->size) {
        if (a->arr[i].priority <=
            b->arr[j].priority) {
            result->arr[k++] = a->arr[i++];
        } else {
            result->arr[k++] = b->arr[j++];
        }
    }

    /* Copy leftover elements from A */
    while (i < a->size)
        result->arr[k++] = a->arr[i++];

    /* Copy leftover elements from B */
    while (j < b->size)
        result->arr[k++] = b->arr[j++];

    result->size = k;
    printf("Merged: %d elements\n", k);
}

/* Time Complexity: O(n + m) */
17
Copy Priority Queue
// Deep copy — fully independent duplicate
Theory — Deep Copy

Copying creates a fully independent duplicate of the priority queue. In C, this means copying every element's data and priority, and the size field.

A shallow copy (just copying the struct pointer) would make both variables point to the same memory — modifying one would corrupt the other!

A deep copy copies all the actual data. Changes to the copy don't affect the original. Time: O(n).

Shallow vs Deep Copy
❌ Shallow (WRONG in C):
struct PQ *copy = original;
// Both point to same data!
✓ Deep (CORRECT):
copy every field manually
copy.arr[i] = original.arr[i]
📄
Real-World Analogy
Photocopying a patient priority list creates a completely new, independent document. Writing on the photocopy doesn't change the original. That's deep copy. If you just pointed two doctors to the same single sheet, any one doctor's edits would confuse the other — that's shallow copy.
C Language
/* ===== Deep Copy Priority Queue ===== */

void copyQueue(struct PriorityQueue *source,
               struct PriorityQueue *destination) {
    int i;

    /* Copy the size and order fields */
    destination->size  = source->size;
    destination->order = source->order;

    /* Copy every element — element by element (deep copy) */
    for (i = 0; i < source->size; i++) {
        destination->arr[i].data     =
            source->arr[i].data;
        destination->arr[i].priority =
            source->arr[i].priority;
    }

    printf("Copied %d elements to destination.\n",
           source->size);
}

/* Time Complexity: O(n) — copies n elements */
18
Reverse Priority Queue
// Flip ascending ↔ descending in O(n/2)
Theory

Reversing swaps the order of all elements: an ascending queue becomes descending, and vice versa. This effectively changes which end holds the highest priority element.

We use the classic two-pointer swap technique: one pointer starts at index 0, the other at index size-1. They swap elements and move toward each other until they meet in the middle.

Only n/2 swaps are needed! Time: O(n). No extra array needed.

🔄
Real-World Analogy
Reversing a sorted queue is like flipping a ranked list upside down. The first person becomes last, the last becomes first. You grab person #1 and person #N, swap their positions, then do the same for #2 and #N-1, working your way inward until you've re-ordered the entire line in reverse.

C Language
/* ===== Reverse Priority Queue ===== */

void reverseQueue(struct PriorityQueue *pq) {
    int left  = 0;
    int right = pq->size - 1;
    struct Element temp;

    /* Two-pointer swap from both ends toward center */
    while (left < right) {
        /* Swap elements at left and right */
        temp             = pq->arr[left];
        pq->arr[left]   = pq->arr[right];
        pq->arr[right]  = temp;

        left++;   /* Move inward from left */
        right--;  /* Move inward from right */
    }

    /* Toggle the order flag */
    pq->order = !pq->order;
    printf("Reversed. Now in %s order.\n",
           pq->order ? "DESCENDING" : "ASCENDING");
}

/* Time Complexity: O(n/2) = O(n), Space: O(1) */
