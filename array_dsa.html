Section 1 — Basic Operations
Create / Initialize Array
BEGINNER
📖 Theory

An array is a collection of elements stored in contiguous (side-by-side) memory locations. All elements must be of the same data type. You access any element instantly using its index (position number, starting at 0). Declaration tells the compiler: "Reserve this much space." Initialization fills that space with values.

Click a button to initialize the array ↑
array_init.c
// ===== Declare Array =====
int arr[5];               // declares array of 5 integers (garbage values)

// ===== Static Initialization =====
int arr2[5] = {10, 20, 30, 40, 50};  // all values set at declaration

// ===== Dynamic Initialization =====
int n = 5;
int arr3[n];              // VLA - size decided at runtime
for(int i = 0; i < n; i++) {
    scanf("%d", &arr3[i]);  // read from user
}

// ===== Assign Default Values (all zeros) =====
int arr4[5] = {0};        // first element 0, rest auto-0
// OR use a loop:
for(int i = 0; i < 5; i++) arr[i] = 0;
🌍 Real-Life Analogy

Think of an array like a row of lockers in a school. Each locker has a fixed number (index: 0,1,2…) and holds one item. You can open locker #3 instantly — you don't check every locker one by one.

Access Operations
BEGINNER
📖 Theory

Access by index means reading or retrieving the value at a specific position. In C, arr[i] takes you directly to the i-th element. The first element is arr[0] and the last is arr[n-1]. Access time is O(1) — instant!

Like a numbered egg carton: You don't look through all eggs — you say "give me egg #3" and reach directly to slot 3.

Update Operations
BEGINNER
📖 Theory

Updating means replacing an element's value at a given index. arr[i] = newValue overwrites the old value. You can update a single element or loop through to modify multiple elements at once. Time: O(1) for single, O(n) for all.

Section 2 — Insertion Operations
Position-based Insertion (Beginning, Middle, End, Specific)
INTERMEDIATE
📖 Theory

Inserting into an array requires shifting elements to make room. Inserting at the end is O(1). Inserting at the beginning or middle is O(n) because all elements after the position must shift right. The array must have enough capacity (declared size must be larger than current count).

Insertion Visualizer (Max size: 8)
Array starts with 5 elements. Insert more!
insertion.c
// Array: {10,20,30,40,50}, size=5, max_size=8
int arr[8] = {10,20,30,40,50};
int n = 5;  // current count

// === Insert at End === O(1)
arr[n] = 60;
n++;        // {10,20,30,40,50,60}

// === Insert at Beginning === O(n)
int newVal = 5;
for(int i = n; i > 0; i--)
    arr[i] = arr[i-1];   // shift right
arr[0] = newVal;
n++;        // {5,10,20,30,40,50,60}

// === Insert at Specific Position pos=2 === O(n)
int pos = 2, val = 99;
if(pos <= n && n < 8) {
    for(int i = n; i > pos; i--)
        arr[i] = arr[i-1]; // shift right from pos
    arr[pos] = val;
    n++;
}

// === Insert Before a Value ===
int target = 30;
for(int i = 0; i < n; i++) {
    if(arr[i] == target) {
        // shift and insert at i
        for(int j = n; j > i; j--) arr[j] = arr[j-1];
        arr[i] = 25; n++;
        break;
    }
}
🌍 Analogy

Like inserting a person into a queue: If someone cuts to position 3, everyone from position 3 onwards must step one place back to make room.

Bulk Insertion — Insert Multiple Elements
INTERMEDIATE
📖 Theory

Inserting multiple elements at once involves shifting elements enough times to accommodate all new values. It's more efficient to calculate the total shift needed and do it in one pass rather than inserting one by one.

bulk_insert.c
// Insert multiple values {71,72,73} at position 2
int arr[20] = {10,20,30,40,50};
int n = 5;
int newElems[] = {71,72,73};
int m = 3;   // count of new elements
int pos = 2;

// Shift existing elements right by m places
for(int i = n - 1; i >= pos; i--)
    arr[i + m] = arr[i];

// Place new elements
for(int i = 0; i < m; i++)
    arr[pos + i] = newElems[i];

n += m;   // update count
// Result: {10,20,71,72,73,30,40,50}
🌍 Analogy

Like adding multiple slides to a presentation: You select where to insert, push the rest back, and drop in the new slides together.

Section 3 — Deletion Operations
Position-based Deletion (Beginning, Middle, End, Specific)
INTERMEDIATE
📖 Theory

Arrays have a fixed size in memory. Deletion doesn't free memory — it shifts elements left to cover the gap and decrements the count. Deleting from the end: O(1). Deleting from beginning/middle: O(n) due to shifting.

Deletion Visualizer
Array has 6 elements. Click to delete!
deletion.c
int arr[6] = {10,20,30,40,50,60};
int n = 6;

// === Delete from End === O(1)
n--;        // just reduce count, {10,20,30,40,50}

// === Delete from Beginning === O(n)
for(int i = 0; i < n - 1; i++)
    arr[i] = arr[i + 1];  // shift left
n--;       // {20,30,40,50}

// === Delete at Specific Position pos=1 === O(n)
int pos = 1;
for(int i = pos; i < n - 1; i++)
    arr[i] = arr[i + 1];
n--;

// === Delete by Value ===
int target = 30;
for(int i = 0; i < n; i++) {
    if(arr[i] == target) {
        for(int j = i; j < n - 1; j++)
            arr[j] = arr[j + 1];
        n--; break;
    }
}

// === Delete Multiple Occurrences of value 20 ===
int val = 20, write = 0;
for(int i = 0; i < n; i++)
    if(arr[i] != val) arr[write++] = arr[i];
n = write; // all 20s removed

// === Clear Entire Array ===
n = 0; // logical clear — just reset count
🌍 Analogy

Like removing a book from a shelf: Pull out the book, then slide all books to its right one position left to fill the gap. The shelf doesn't shrink, but the count of books decreases.

Section 4 — Searching Operations
Linear Search
BEGINNER
📖 Theory

Linear search checks every element one by one from left to right until it finds the target or reaches the end. Works on both sorted and unsorted arrays. Time: O(n) worst case.

Linear Search Live
Enter a value and click Search
linear_search.c
// Search Single Value - returns index or -1
int linearSearch(int arr[], int n, int target) {
    for(int i = 0; i < n; i++) {
        if(arr[i] == target)
            return i;      // found!
    }
    return -1;             // not found
}

// Search Multiple Occurrences
void findAll(int arr[], int n, int target) {
    for(int i = 0; i < n; i++) {
        if(arr[i] == target)
            printf("Found at index: %d\n", i);
    }
}

// Find First Occurrence
int findFirst(int arr[], int n, int t) {
    for(int i = 0; i < n; i++)
        if(arr[i] == t) return i;
    return -1;
}

// Find Last Occurrence
int findLast(int arr[], int n, int t) {
    for(int i = n-1; i >= 0; i--)
        if(arr[i] == t) return i;
    return -1;
}
🌍 Analogy

Like searching for a word in a book without an index: You read every page until you find it. Simple but slow for large books.

Binary Search (Iterative & Recursive)
INTERMEDIATE
📖 Theory

Binary search works only on sorted arrays. It repeatedly halves the search space: compare target with the middle element, eliminate the half that can't contain it. Time: O(log n) — searching 1 million elements takes at most 20 steps!

Binary Search Live (sorted array)
Array must be sorted. Enter value and search!
binary_search.c
// Iterative Binary Search
int binarySearch(int arr[], int n, int target) {
    int low = 0, high = n - 1;
    while(low <= high) {
        int mid = low + (high - low) / 2; // avoids overflow
        if(arr[mid] == target)   return mid;
        else if(arr[mid] < target) low = mid + 1;
        else                     high = mid - 1;
    }
    return -1;  // not found
}

// Recursive Binary Search
int binaryRec(int arr[], int low, int high, int t) {
    if(low > high) return -1;
    int mid = (low + high) / 2;
    if(arr[mid] == t) return mid;
    if(arr[mid] < t) return binaryRec(arr, mid+1, high, t);
    return binaryRec(arr, low, mid-1, t);
}

// Find Duplicates
void findDuplicates(int arr[], int n) {
    for(int i = 0; i < n - 1; i++)
        if(arr[i] == arr[i+1])  // works on sorted
            printf("Dup: %d\n", arr[i]);
}

// Find Missing Element (1 to n)
int findMissing(int arr[], int n) {
    int expected = n*(n+1)/2, sum = 0;
    for(int i = 0; i < n-1; i++) sum += arr[i];
    return expected - sum;
}
🌍 Analogy

Like finding a word in a dictionary: You open to the middle, if the word is before that page you search the first half, otherwise the second half. You narrow it down rapidly.

Section 5 — Traversal Operations
All Traversal Types (L→R, R→L, Even/Odd Index, Even/Odd Value)
BEGINNER
📖 Theory

Traversal means visiting every required element in the array. Standard traversal visits all elements. Conditional traversal visits only elements meeting a condition (even index, odd value, etc.). All traversals use a for/while loop — the difference is just the loop condition or the step size.

    printf("%d ", arr[i]);

// Right to Left Traversal
for(int i = n - 1; i >= 0; i--)
    printf("%d ", arr[i]);

// Traverse Even Indexes (0,2,4,6...)
for(int i = 0; i < n; i += 2)
    printf("%d ", arr[i]);

// Traverse Odd Indexes (1,3,5,7...)
for(int i = 1; i < n; i += 2)
    printf("%d ", arr[i]);

// Traverse Even Values
for(int i = 0; i < n; i++)
    if(arr[i] % 2 == 0)
        printf("%d ", arr[i]);

// Traverse Odd Values
for(int i = 0; i < n; i++)
    if(arr[i] % 2 != 0)
        printf("%d ", arr[i]);
🌍 Analogy

Like a security camera scanning a hallway: A full sweep visits every room. You can also program it to check only even-numbered rooms, or only rooms with the light on (conditional).

Section 6 — Sorting Operations
Sorting Algorithms — Bubble, Selection, Insertion, Merge, Quick, Heap
INTERMEDIATE
📖 Theory

Sorting arranges elements in a specific order (ascending or descending). Each algorithm has trade-offs between speed and simplicity. Bubble/Selection/Insertion: simple, O(n²), good for small arrays. Merge/Quick/Heap: efficient, O(n log n), used in real systems. Stable sort preserves original order of equal elements. Unstable doesn't guarantee this.

sorting.c
// === Bubble Sort — O(n²) — Stable ===
void bubbleSort(int arr[], int n) {
    for(int i = 0; i < n-1; i++)
        for(int j = 0; j < n-i-1; j++)
            if(arr[j] > arr[j+1]) {
                int t = arr[j]; arr[j]=arr[j+1]; arr[j+1]=t;
            }
}

// === Selection Sort — O(n²) — Unstable ===
void selectionSort(int arr[], int n) {
    for(int i = 0; i < n-1; i++) {
        int minIdx = i;
        for(int j = i+1; j < n; j++)
            if(arr[j] < arr[minIdx]) minIdx = j;
        int t = arr[i]; arr[i]=arr[minIdx]; arr[minIdx]=t;
    }
}

// === Insertion Sort — O(n²) — Stable ===
void insertionSort(int arr[], int n) {
    for(int i = 1; i < n; i++) {
        int key = arr[i], j = i - 1;
        while(j >= 0 && arr[j] > key)
            arr[j+1] = arr[j--];
        arr[j+1] = key;
    }
}

// === Merge Sort — O(n log n) — Stable ===
void merge(int a[], int l, int m, int r) {
    int n1=m-l+1, n2=r-m;
    int L[n1], R[n2];
    for(int i=0;i<n1;i++) L[i]=a[l+i];
    for(int j=0;j<n2;j++) R[j]=a[m+1+j];
    int i=0,j=0,k=l;
    while(i<n1 && j<n2) a[k++]=(L[i]<=R[j])?L[i++]:R[j++];
    while(i<n1) a[k++]=L[i++];
    while(j<n2) a[k++]=R[j++];
}
void mergeSort(int a[], int l, int r) {
    if(l<r) { int m=(l+r)/2; mergeSort(a,l,m); mergeSort(a,m+1,r); merge(a,l,m,r); }
}

// === Quick Sort — O(n log n) avg — Unstable ===
int partition(int a[], int low, int high) {
    int pivot=a[high], i=low-1;
    for(int j=low;j<high;j++)
        if(a[j]<=pivot){ i++; int t=a[i];a[i]=a[j];a[j]=t; }
    int t=a[i+1];a[i+1]=a[high];a[high]=t;
    return i+1;
}
void quickSort(int a[], int l, int h) {
    if(l<h){ int p=partition(a,l,h); quickSort(a,l,p-1); quickSort(a,p+1,h); }
}

// Reverse (Descending) Sort: sort ascending, then reverse
for(int i=0,j=n-1;i<j;i++,j--){
    int t=arr[i];arr[i]=arr[j];arr[j]=t;
}
🌍 Analogy

Bubble Sort = Sorting playing cards by repeatedly swapping adjacent mismatched cards. Quick Sort = Picking a pivot card, putting smaller ones left, bigger ones right, then repeating. Merge Sort = Splitting the deck in halves, sorting each pile, then merging them back.

Section 7 — Rearrangement Operations
Reverse, Rotate Left/Right, Shift Left/Right
INTERMEDIATE
📖 Theory

Reordering operations rearrange elements without adding or removing. Reverse flips the array. Rotate cyclically moves elements (left rotation: first element goes to end). Shift moves elements and fills with 0. Rotation by k = k single rotations, or use the "reversal algorithm" for O(n) efficiency.

Reorder Visualizer
Pick an operation to rearrange!
rearrange.c
int arr[6] = {1,2,3,4,5,6};
int n = 6;

// === Reverse Array ===
int l=0, r=n-1;
while(l<r){ int t=arr[l];arr[l]=arr[r];arr[r]=t; l++;r--; }

// === Rotate Left by 1 ===
int first = arr[0];
for(int i=0;i<n-1;i++) arr[i]=arr[i+1];
arr[n-1] = first;

// === Rotate Right by 1 ===
int last = arr[n-1];
for(int i=n-1;i>0;i--) arr[i]=arr[i-1];
arr[0] = last;

// === Shift Left by 1 (fill 0) ===
for(int i=0;i<n-1;i++) arr[i]=arr[i+1];
arr[n-1] = 0;

// === Shift Right by 1 (fill 0) ===
for(int i=n-1;i>0;i--) arr[i]=arr[i-1];
arr[0] = 0;
🌍 Analogy

Rotate Left = In a circular queue at a theme park, the person at the front leaves, everyone moves up, and the spot at the back is given to that person. Shift = Same but the leaving person doesn't re-enter (replaced by empty).

Partition (Even/Odd, Positive/Negative) & Shuffle
INTERMEDIATE
📖 Theory

Partitioning rearranges elements so one group comes first and another follows — without sorting within groups. The two-pointer technique does this in O(n) with O(1) extra space. Fisher-Yates shuffle randomly rearranges elements in O(n).

Partition & Shuffle Demo
Pick a partition operation!
partition.c
// === Partition Even/Odd (evens first) ===
void partitionEvenOdd(int arr[], int n) {
    int l=0, r=n-1;
    while(l < r) {
        while(l<n && arr[l]%2==0) l++;   // skip evens at left
        while(r>=0 && arr[r]%2!=0) r--;  // skip odds at right
        if(l < r) { int t=arr[l]; arr[l]=arr[r]; arr[r]=t; }
    }
}

// === Partition Positive/Negative ===
void partitionPosNeg(int arr[], int n) {
    int l=0, r=n-1;
    while(l < r) {
        while(l<n && arr[l]>=0) l++;   // skip positives
        while(r>=0 && arr[r]<0) r--;   // skip negatives
        if(l < r) { int t=arr[l]; arr[l]=arr[r]; arr[r]=t; }
    }
}

// === Fisher-Yates Shuffle ===
// (requires srand(time(NULL)) at start)
for(int i=n-1; i>0; i--) {
    int j = rand() % (i+1);
    int t=arr[i]; arr[i]=arr[j]; arr[j]=t;
}
🌍 Analogy

Partition Even/Odd = Sorting a basket of fruits: put round ones (even) on one side and odd-shaped ones on the other — no need for full ordering. Shuffle = Shuffling a deck of cards randomly.

Section 8 — Mathematical Operations
Sum, Average, Max, Min, Range, Median, Mode, Variance, Std Dev & Counting
INTERMEDIATE
📖 Theory

Mathematical operations analyze array data numerically. Sum/Average/Max/Min: single pass O(n). Median: sort first, then pick middle. Mode: element with highest frequency. Variance: average of squared differences from mean. Std Dev: square root of variance — measures spread of data.

math_ops.c
#include <stdio.h>
#include <math.h>

int arr[8] = {4,7,2,9,6,1,8,3};
int n = 8;

// Sum
int sum=0;
for(int i=0;i<n;i++) sum+=arr[i];

// Average
double avg = (double)sum / n;

// Maximum
int mx=arr[0];
for(int i=1;i<n;i++) if(arr[i]>mx) mx=arr[i];

// Minimum
int mn=arr[0];
for(int i=1;i<n;i++) if(arr[i]<mn) mn=arr[i];

// Range
int range = mx - mn;

// Median (sort first, then pick middle)
// (after sorting arr)
double median;
if(n%2==0) median=(arr[n/2-1]+arr[n/2])/2.0;
else       median=arr[n/2];

// Variance and Standard Deviation
double var=0;
for(int i=0;i<n;i++) var += (double)(arr[i]-avg)*(arr[i]-avg);
var /= n;
double stddev = sqrt(var);

// Count Even/Odd/Positive/Negative
int evens=0, odds=0, pos=0, neg=0;
for(int i=0;i<n;i++) {
    if(arr[i]%2==0) evens++; else odds++;
    if(arr[i]>0) pos++; else if(arr[i]<0) neg++;
}
🌍 Analogy

Like a grade sheet: Sum = total marks, Average = class average, Max = highest score, Min = lowest, Variance = how spread out the scores are, Std Dev = tells if scores are clustered (low) or all over the place (high).

Section 9 — Comparison Operations
Compare Arrays, Common Elements, Union, Intersection, Unique Elements

Set operations on arrays: Intersection = elements in BOTH arrays. Union = all elements from both (no duplicates). Difference/Unique = elements in one but not the other. Brute force: O(n²). Sort + two-pointer: O(n log n). Hash-based: O(n).

    for(int j=0;j<nB;j++)
        if(A[i]==B[j]) printf("%d ",A[i]);

for(int i=0;i<nA;i++) printf("%d ",A[i]);
for(int i=0;i<nB;i++) {
    int found=0;

for(int i=0;i<nA;i++) {
    int found=0;
    for(int j=0;j<nB;j++) if(A[i]==B[j]){found=1;break;}
    if(!found) printf("%d ",A[i]);
}
🌍 Analogy

Like comparing two class rosters: Intersection = students in BOTH classes. Union = ALL students across both. Unique = students in class A who are NOT in class B.

Section 10 — Memory Operations
Static vs Dynamic Allocation, Resize, Copy, Clone, Free
INTERMEDIATE
📖 Theory

Static allocation: size fixed at compile time (int arr[10]). Memory on the stack. Dynamic allocation: size decided at runtime using malloc(). Memory on the heap. You must manually free() it to avoid memory leaks. realloc() resizes a dynamic array. Copying requires looping element-by-element.

memory_ops.c
#include <stdlib.h>
#include <string.h>

// === Static Memory Allocation ===
int arr[10];          // 10 ints on the stack

// === Dynamic Memory Allocation ===
int n = 10;
int *arr2 = (int*)malloc(n * sizeof(int));
if(arr2 == NULL) { printf("Alloc failed!"); return 1; }

// === Expand Capacity (Resize) ===
int newSize = 20;
arr2 = (int*)realloc(arr2, newSize * sizeof(int));

// === Shrink Capacity ===
arr2 = (int*)realloc(arr2, 5 * sizeof(int));

// === Copy Array ===
int copy[10];
for(int i=0;i<n;i++) copy[i] = arr[i];
// OR: memcpy(copy, arr, n * sizeof(int));

// === Clone (dynamic copy) ===
int *clone = (int*)malloc(n * sizeof(int));
memcpy(clone, arr2, n * sizeof(int));

// === Free Memory (MUST do this!) ===
free(arr2);   // prevents memory leak
free(clone);
arr2 = NULL; // good practice: avoid dangling pointer
🌍 Analogy

Static = Booking a fixed hotel room size before your trip. Dynamic = Renting a flexible Airbnb where you can request more rooms as needed. Free = Checking out — don't forget or you're still paying for an empty room (memory leak)!

Section 11 — Conversion Operations
Convert Array → Linked List, Stack, Queue
ADVANCED
📖 Theory

Arrays can be converted to other data structures. Linked List: create a node for each element, link them. Stack: push array elements (last element becomes top). Queue: enqueue each element (first becomes front). These structures trade random access for flexible insertion/deletion.

conversions.c
#include <stdlib.h>
int arr[5] = {10,20,30,40,50};
int n = 5;

// === Convert to Linked List ===
struct Node { int data; struct Node* next; };

struct Node* arrayToLL(int arr[], int n) {
    struct Node *head=NULL, *tail=NULL;
    for(int i=0;i<n;i++) {
        struct Node *nd = malloc(sizeof(struct Node));
        nd->data = arr[i]; nd->next = NULL;
        if(!head) head=tail=nd;
        else { tail->next=nd; tail=nd; }
    }
    return head;
}

// === Convert to Stack (using array) ===
int stack[5]; int top=-1;
for(int i=0;i<n;i++) stack[++top]=arr[i]; // push all
// top of stack is arr[n-1]

// === Convert to Queue ===
int queue[5]; int front=0, rear=-1;
for(int i=0;i<n;i++) queue[++rear]=arr[i];
// front is arr[0], rear is arr[n-1]
🌍 Analogy

Stack = Pile of plates (add/remove from top). Queue = People in a line (join at back, leave from front). Linked List = Train cars — each car knows which is next.

Section 14 — Advanced Algorithms
Prefix Sum & Suffix Sum Arrays
ADVANCED
📖 Theory

A prefix sum array stores cumulative sums: prefix[i] = arr[0]+...+arr[i]. This allows range sum queries in O(1) instead of O(n). sum(l,r) = prefix[r] - prefix[l-1]. Suffix sum is the same but from right to left. Essential for competitive programming.

prefix_suffix.c
int arr[6] = {3,1,4,1,5,9};
int n = 6;
int prefix[6], suffix[6];

// === Build Prefix Sum ===
prefix[0] = arr[0];
for(int i=1;i<n;i++)
    prefix[i] = prefix[i-1] + arr[i];
// prefix = {3, 4, 8, 9, 14, 23}

// === Build Suffix Sum ===
suffix[n-1] = arr[n-1];
for(int i=n-2;i>=0;i--)
    suffix[i] = suffix[i+1] + arr[i];
// suffix = {23,20,19,15,14,9}

// === Range Sum Query O(1) ===
int l=1, r=4;  // sum of arr[1..4]
int rangeSum;
if(l == 0) rangeSum = prefix[r];
else       rangeSum = prefix[r] - prefix[l-1];
// = 14 - 3 = 11
🌍 Analogy

Like a running total in a ledger book. Instead of adding up sales between Jan–Jun every time someone asks, you precompute the cumulative total — then "Jun total minus Dec total" gives the sum in one step.

Sliding Window (Fixed & Dynamic)
ADVANCED
📖 Theory

The sliding window technique processes a contiguous subarray (window) by sliding it across the array. Instead of recomputing from scratch, you add the new element entering the window and subtract the element leaving. Converts O(n·k) to O(n). Fixed window: size k is constant. Dynamic window: size changes based on condition.

sliding_window.c
// === Fixed Window: Max sum of size k ===
int arr[8]={2,1,5,1,3,2,4,7}, n=8, k=3;
int winSum=0, maxSum;

// Build first window
for(int i=0;i<k;i++) winSum+=arr[i];
maxSum = winSum;

// Slide window
for(int i=k;i<n;i++) {
    winSum += arr[i] - arr[i-k]; // add new, remove old
    if(winSum > maxSum) maxSum = winSum;
}
printf("Max sum of %d elements: %d\n", k, maxSum);

// === Dynamic Window: longest subarray with sum ≤ target ===
int target=7, left=0, curSum=0, maxLen=0;
for(int right=0;right<n;right++) {
    curSum += arr[right];
    while(curSum > target) curSum -= arr[left++];
    int len = right - left + 1;
    if(len > maxLen) maxLen = len;
}
🌍 Analogy

Like a train window of fixed size moving through scenery. As the train moves, the leftmost view exits the window and new scenery enters from the right — you update what you see without scanning all previous scenery.

Kadane's Algorithm — Maximum Subarray Sum
ADVANCED
📖 Theory

Kadane's Algorithm finds the maximum sum of any contiguous subarray in O(n). Key insight: at each position, you either extend the current subarray or start a new one from here. maxEndingHere = max(arr[i], maxEndingHere + arr[i]). Works with negative numbers too! DP stored implicitly in two variables.

kadane.c
// Kadane's Algorithm — O(n), O(1) space
int maxSubarraySum(int arr[], int n) {
    int maxSum = arr[0];
    int curSum = arr[0];

    for(int i = 1; i < n; i++) {
        // extend or start fresh?
        curSum = arr[i] > curSum + arr[i] ? arr[i] : curSum + arr[i];
        if(curSum > maxSum) maxSum = curSum;
    }
    return maxSum;
}

// With subarray range tracking
void kadaneFull(int arr[], int n) {
    int maxSum=arr[0], curSum=arr[0];
    int start=0, end=0, tempStart=0;
    for(int i=1;i<n;i++) {
        if(arr[i] > curSum+arr[i]) { curSum=arr[i]; tempStart=i; }
        else curSum+=arr[i];
        if(curSum > maxSum) { maxSum=curSum; start=tempStart; end=i; }
    }
    printf("Max sum: %d, from [%d] to [%d]\n",maxSum,start,end);
}
// Example: arr={-2,1,-3,4,-1,2,1,-5,4}
// Max subarray: {4,-1,2,1} = sum 6
🌍 Analogy

Like planning a road trip for maximum profit. At each city you decide: "Is it worth continuing from here, or should I start a fresh trip?" You keep track of the best trip seen so far.

Two Pointer Technique & Frequency Counting
ADVANCED
📖 Theory

Two Pointer: Use two indices moving toward each other (or same direction) to solve problems in O(n) that would otherwise take O(n²). Great for: pair sum, removing duplicates, palindrome check. Frequency Counting: Use an auxiliary array to count occurrences — converts O(n²) search to O(n) with O(k) space (k = value range).

two_pointer.c
// === Two Pointer: Pair with target sum (sorted array) ===
int arr[7]={1,3,4,6,7,8,9}, n=7, target=10;
int l=0, r=n-1;
while(l < r) {
    int sum = arr[l] + arr[r];
    if(sum == target) {
        printf("Pair: %d + %d\n", arr[l], arr[r]);
        l++; r--;
    } else if(sum < target) l++;
    else r--;
}

// === Frequency Counting ===
int data[8] = {3,1,4,1,5,9,2,6};
int freq[10] = {0};   // assuming values 0-9
for(int i=0;i<8;i++) freq[data[i]]++;
for(int i=0;i<10;i++)
    if(freq[i]>0)
        printf("%d appears %d time(s)\n", i, freq[i]);

// === DP Table Storage ===
// For memoization in dynamic programming:
int dp[100]; // dp[i] stores subproblem result
for(int i=0;i<100;i++) dp[i]=-1; // -1 = not computed
// When computing: if(dp[n]!=-1) return dp[n];
🌍 Analogy

Two Pointer = Two people walking toward each other on a sorted number line looking for pairs that add up to a target. If their sum is too low, the left person steps right; too high, the right steps left. Frequency Count = Tally marks on a whiteboard — fast to update, instant to read.
