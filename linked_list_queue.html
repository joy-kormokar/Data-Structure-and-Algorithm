What is a Linked List Queue?

A Queue is a linear data structure that follows the FIFO (First In, First Out) principle — the first element inserted is the first one removed.


In a Linked List–based Queue, each element (called a node) is a dynamically allocated structure containing:

  • data — the actual value stored
  • next — a pointer to the next node

We maintain two special pointers: front (points to the first node — used for deletion) and rear (points to the last node — used for insertion).


Initialization means setting both front and rear to NULL, signalling an empty queue.

Real-Life Analogy

An Empty Store Counter

Think of opening a shop before any customers arrive. The front pointer is like your "next to serve" marker, and rear is the "end of the line" marker. Before any customer steps in, both markers point to nothing (NULL) — the queue is empty and ready to accept people.

C — Queue Initialization 
#include <stdio.h>
#include <stdlib.h>

/* Step 1: Define the Node structure */
struct Node {
    int data;           /* Stores the actual value */
    struct Node* next;  /* Points to the next node */
};

/* Step 2: Declare two global pointers */
struct Node* front = NULL;  /* Points to first node (for deletion) */
struct Node* rear  = NULL;  /* Points to last node  (for insertion) */

int main() {
    /* Both pointers are NULL — queue is initialized and empty */
    printf("Queue initialized.\n");
    printf("front = %p\n", (void*)front);  /* prints: (nil) */
    printf("rear  = %p\n", (void*)rear);   /* prints: (nil) */
    return 0;
}
02
Enqueue (Insert at Rear)
Insert O(1) Dynamic Memory
Theory

Adding Elements to the Queue

Enqueue inserts a new element at the rear of the queue. It always happens at the back — just like joining the end of a line.


  1. Create a new node using malloc()
  2. Assign the input data to newNode→data
  3. Set newNode→next = NULL (it's the last node)
  4. If the queue is empty, set both front and rear to newNode
  5. Otherwise, link: rear→next = newNode
  6. Update rear = newNode
⚠ Always check if malloc() returned NULL. If memory allocation fails, do not proceed with the insertion.

C — Enqueue Function 
void enqueue(int value) {
    /* Step 1: Allocate memory for a new node */
    struct Node* newNode = (struct Node*) malloc(sizeof(struct Node));

    /* Step 2: Check if memory allocation was successful */
    if (newNode == NULL) {
        printf("Memory allocation failed!\n");
        return;
    }

    /* Step 3: Store data and set next pointer to NULL */
    newNode->data = value;
    newNode->next = NULL;

    /* Step 4: Handle empty queue case */
    if (front == NULL) {
        front = newNode;   /* Both front and rear point to new node */
        rear  = newNode;
        printf("Enqueued %d (first element)\n", value);
        return;
    }

    /* Step 5: Link new node at the rear */
    rear->next = newNode;  /* Current last node now points to new node */
    rear = newNode;         /* Update rear to the new last node */

    printf("Enqueued: %d\n", value);
}
03
Dequeue (Delete from Front)
Delete O(1) Pointer Update
Theory

Removing Elements from Front

Dequeue removes the element at the front of the queue. Removal always happens from the front, maintaining the FIFO order.


  1. Check underflow: if front == NULL, queue is empty — cannot dequeue
  2. Save the front node's data to return it
  3. Move front pointer: front = front→next
  4. Free the old front node to prevent memory leaks
  5. If front == NULL now, also set rear = NULL (queue is now empty)
⚡ Critical: After dequeue makes queue empty, set rear = NULL too! Otherwise rear becomes a dangling pointer.

Single Node Edge Case

When the Last Node is Removed

When the queue has exactly one node, dequeuing it makes the queue empty:


Before: front → [5] → NULL, rear → [5]
After dequeue:
  front = front→next = NULL
  rear = NULL  ← must also update!
C — Dequeue Function 
int dequeue() {
    /* Step 1: Check underflow condition */
    if (front == NULL) {
        printf("Queue Underflow! Queue is empty.\n");
        return -1;  /* Return -1 to signal error */
    }

    /* Step 2: Save front data and get pointer to free */
    int removedData = front->data;
    struct Node* temp = front;  /* Save pointer to old front */

    /* Step 3: Move front pointer forward */
    front = front->next;

    /* Step 4: Free the deleted node's memory */
    free(temp);

    /* Step 5: If queue is now empty, update rear too! */
    if (front == NULL) {
        rear = NULL;  /* Prevent dangling pointer */
    }

    printf("Dequeued: %d\n", removedData);
    return removedData;
}
04
Peek / Front Element
Read-Only O(1)
Theory

Viewing Without Removing

Peek (or Front) lets you see the value at the front of the queue without removing it. The queue remains unchanged.


  1. Check if the queue is empty (front == NULL)
  2. If empty, report an underflow error
  3. Otherwise, return front→data — that's it!

ℹ Peek is a read-only operation. Never modify front or rear inside this function.

Rear Element Access
Read-Only O(1)
Theory

Accessing the Last Element

The rear element is the most recently inserted element — the one at the back of the queue. Since we maintain a rear pointer, accessing it is O(1).


  1. Check if rear == NULL (empty queue check)
  2. Return rear→data

💡 Without the rear pointer, you'd need O(n) traversal to find the last element. The rear pointer gives us O(1) access — that's why we maintain it!

IsEmpty Operation
Boolean Check O(1)
Theory

Checking If the Queue Has Elements

IsEmpty checks whether the queue contains any elements. It returns 1 (true) if empty, 0 (false) otherwise.


The condition is simple: a linked list queue is empty when front == NULL.


IsEmpty is used before every dequeue or peek operation to prevent errors. It's a safety guard function.

✓ Always call isEmpty() before dequeue() or peek() to avoid accessing a NULL pointer!

Queue Size Counting
Traversal O(n)
Theory

Counting All Nodes

Unlike an array queue, a linked list queue does not store a size variable by default. To count elements, we must traverse the entire list.


  1. Start a temporary pointer temp = front
  2. Initialize count = 0
  3. While temp != NULL: increment count, move temp = temp→next
  4. Return count

💡 Optimization tip: You can maintain a separate size variable (increment on enqueue, decrement on dequeue) to get O(1) size queries!

Counting People in a Line

To count how many people are waiting, you walk from the front to the back, counting each person. This is exactly what the size function does — a linear walk through all nodes.

C — Size Count Function 
int queueSize() {
    int count = 0;
    struct Node* temp = front;  /* Start from front, never modify front! */

    /* Traverse the entire linked list and count each node */
    while (temp != NULL) {
        count++;
        temp = temp->next;  /* Move to next node */
    }

    printf("Queue size: %d\n", count);
    return count;    /* O(n) time complexity */
}

Display Queue
Print / Output O(n)
Theory

Printing All Queue Elements

Display traverses the queue from front to rear and prints each node's data. This is a pure read operation — nothing is modified.


  1. Check if queue is empty; if so, print a message
  2. Use a temporary pointer temp = front
  3. Print temp→data at each step
  4. Move temp = temp→next until NULL
  5. Optionally show "FRONT →" and "→ NULL" for clarity

void displayQueue() {
    /* Handle empty queue */
    if (front == NULL) {
        printf("Queue is empty.\n");
        return;
    }

    printf("Queue: FRONT -> ");

    struct Node* temp = front;  /* Temp pointer for traversal */

    while (temp != NULL) {
        printf("[%d]", temp->data);
        if (temp->next != NULL)
            printf(" -> ");
        temp = temp->next;
    }

    printf(" -> NULL (REAR)\n");
}
09
Search Element
Linear Search O(n)
Theory

Finding a Value in the Queue

Search traverses the queue from front to rear, comparing each node's data with the target value. It returns the position (1-based index) if found, or -1 if not found.


  1. Start with temp = front, position = 1
  2. At each node, compare temp→data == target
  3. If match found, return position
  4. Move temp = temp→next, increment position
  5. If loop ends without finding, return -1
⚠ There is no random access in a linked list. We must always do linear traversal — O(n) in all cases.

Looking for a Person in a Line

To find someone specific in a queue of people, you start from the front and check each person one by one until you find them or reach the end. There's no shortcut — this is why search is O(n).

C — Search Function 
int searchQueue(int target) {
    struct Node* temp = front;
    int position = 1;  /* 1-based indexing */

    while (temp != NULL) {
        if (temp->data == target) {
            printf("Found %d at position %d\n", target, position);
            return position;  /* Return position (1-based) */
        }
        temp = temp->next;
        position++;
    }

    printf("%d not found in queue.\n", target);
    return -1;  /* Element not found */
}

Theory — Update by Key

Replacing an Old Value

Update changes the data in a specific node. You can update by key (find a value and replace it) or by position (go to the nth node).


Update by Key:

  1. Traverse from front, comparing each node's data with the target
  2. When found, replace: temp→data = newValue
  3. If not found, report error

Update by Position:

  1. Traverse to the nth node (1-based)
  2. Replace temp→data = newValue

C — Update Functions 
/* Update by key value */
int updateByKey(int oldVal, int newVal) {
    struct Node* temp = front;
    while (temp != NULL) {
        if (temp->data == oldVal) {
            temp->data = newVal;
            printf("Updated %d to %d\n", oldVal, newVal);
            return 1;  /* Success */
        }
        temp = temp->next;
    }
    printf("Value %d not found.\n", oldVal);
    return 0;  /* Not found */
}

/* Update by position (1-based) */
int updateByPosition(int pos, int newVal) {
    struct Node* temp = front;
    int i = 1;
    while (temp != NULL) {
        if (i == pos) {
            temp->data = newVal;
            printf("Updated position %d to %d\n", pos, newVal);
            return 1;
        }
        temp = temp->next;
        i++;
    }
    printf("Position %d out of range.\n", pos);
    return 0;
}

Clear / Destroy Queue
Free Memory O(n) Memory Leak Prevention
Theory

Removing All Nodes Safely

Clear/Destroy deletes every node and frees all allocated memory. This is critical in C to prevent memory leaks.


  1. Use a loop while front != NULL
  2. Save temp = front
  3. Advance front = front→next
  4. Call free(temp) to release that node
  5. After the loop, set rear = NULL
🚨 Memory Leak Warning: If you just do front = rear = NULL without freeing each node, those nodes still exist in memory but are unreachable — that's a memory leak!

Clearing a Restaurant at Closing Time

At closing time, each customer is individually asked to leave (freed) — one by one from front to back. You can't just declare "the restaurant is empty" without actually having everyone leave!

C — Clear/Destroy Queue 
void clearQueue() {
    struct Node* temp;

    /* Loop through every node and free it */
    while (front != NULL) {
        temp  = front;         /* Save pointer to current front */
        front = front->next;   /* Move front to next node */
        free(temp);            /* Free the saved node */
    }

    /* Reset both pointers */
    rear = NULL;

    printf("Queue cleared. All memory freed.\n");
}

/* BAD approach — causes memory leak! */
/* front = rear = NULL;  <-- Nodes still exist in heap! */

Traverse Queue
Iterative Recursive O(n)
Iterative Traversal

Sequential Node Visiting

Traversal means visiting every node in the queue sequentially from front to rear. This is the foundation for all O(n) operations like display, search, size, clear, and copy.


Key Rule: Always use a temp pointer — never move front itself, or you'll lose the queue!


temp = front
while (temp != NULL) {
  // visit temp→data
  temp = temp->next
}
Recursive Traversal

Using Function Recursion

Recursive traversal calls itself on the next node, building a call stack. It's elegant but uses O(n) stack space.


Base case: if node == NULL, return.

Recursive case: print node→data, then call with node→next.


⚠ For very large queues (thousands of nodes), recursive traversal risks stack overflow. Prefer iterative for production code.

C — Iterative Traversal 
void traverseIterative() {
    struct Node* temp = front; /* Use temp, never front! */
    int pos = 1;

    while (temp != NULL) {
        printf("[%d] %d\n", pos, temp->data);
        temp = temp->next;
        pos++;
    }
}
C — Recursive Traversal 
void traverseRecursive(struct Node* node) {
    if (node == NULL) {  /* Base case */
        printf("NULL\n");
        return;
    }
    printf("%d -> ", node->data);
    traverseRecursive(node->next); /* Recurse */
}
/* Call with: traverseRecursive(front); */

Copy Queue
Deep Copy O(n) Memory
Theory

Creating an Independent Duplicate

Copying a queue means creating a new, independent queue with the same values in the same order. This is called a deep copy.


⚡ Shallow Copy Danger: Simply doing front2 = front1 does NOT create a copy — both pointers point to the same nodes. Modifying one would affect the other!

  1. Initialize a new queue with front2 = rear2 = NULL
  2. Traverse the original queue using a temp pointer
  3. For each node, allocate a new node and assign the same data
  4. Enqueue into the new queue (using the new queue's pointers)
  5. Order is preserved because we traverse front to rear

Photocopying a Document

When you photocopy a form, you get a new, separate paper with the same content. You can write on the copy without affecting the original. That's a deep copy. If you just shared the original form with someone else (shallow copy), any change either makes would affect both people!

C — Copy Queue Function 
/* Copies source queue into dest (front2, rear2) */
void copyQueue(struct Node** front2, struct Node** rear2) {
    *front2 = *rear2 = NULL;  /* Initialize new queue */

    struct Node* temp = front;  /* Traverse original queue */

    while (temp != NULL) {
        /* Allocate new node */
        struct Node* newNode = (struct Node*) malloc(sizeof(struct Node));
        newNode->data = temp->data;  /* Copy data value */
        newNode->next = NULL;

        /* Insert into new queue */
        if (*front2 == NULL) {
            *front2 = *rear2 = newNode;
        } else {
            (*rear2)->next = newNode;
            *rear2 = newNode;
        }
        temp = temp->next;
    }
    printf("Queue copied successfully!\n");
}
14
Reverse Queue
Pointer Manipulation O(n) Stack Method
Method 1 — Using a Stack

Stack-Based Reversal

Push all elements from the queue into a stack (array/linked list), then dequeue all and re-enqueue from the stack. Stack's LIFO reverses the FIFO order.


  1. Dequeue all elements and push onto a temporary stack
  2. Pop all elements from the stack
  3. Enqueue them back into the queue
  4. The order is now reversed
Method 2 — Pointer Manipulation

In-Place Reversal

Directly manipulate the next pointers to reverse the linked list direction, then swap front and rear.


  1. Use three pointers: prev = NULL, curr = front, next
  2. At each step: save next = curr→next
  3. Reverse: curr→next = prev
  4. Move: prev = curr, curr = next
  5. Swap front and rear

Reversing a Train

Reversing a queue is like the last train carriage becoming the engine and the engine becoming the last carriage. The direction of travel reverses completely. Using a stack is like using a side track to rearrange the carriages.

C — Reverse Queue (Pointer Method) 
void reverseQueue() {
    if (front == NULL || front->next == NULL)
        return;  /* Nothing to reverse */

    struct Node* prev = NULL;
    struct Node* curr = front;
    struct Node* nextPtr;

    rear = front;  /* Old front becomes new rear */

    while (curr != NULL) {
        nextPtr      = curr->next;  /* Save next */
        curr->next   = prev;         /* Reverse pointer */
        prev         = curr;         /* Move prev forward */
        curr         = nextPtr;      /* Move curr forward */
    }

    front = prev;  /* Old rear becomes new front */
    printf("Queue reversed!\n");
}

Concatenate / Merge Queues
Join O(1) Pointer Link
Theory

Joining Two Queues into One

Concatenation appends the second queue to the end of the first, creating one combined queue. It maintains FIFO order for both original queues.


  1. Handle edge cases: if Q1 is empty, result = Q2; if Q2 is empty, Q1 unchanged
  2. Connect: rear1→next = front2 (link Q1's last node to Q2's first)
  3. Update: rear1 = rear2 (Q1's rear now points to Q2's rear)
  4. Optionally reset Q2: front2 = rear2 = NULL
⚡ Concatenation is O(1) because we just update two pointers — no traversal needed!

Merging Two Traffic Lanes

When two lanes of traffic merge into one, the first lane continues as-is, and the cars from the second lane line up directly behind the last car from the first lane. The order within each original lane is preserved. That's exactly what queue concatenation does — no one "cuts" into the middle.

C — Concatenate Queues 
/*
 * Merges queue2 into queue1 (appends Q2 at end of Q1)
 * After merge, Q2 is emptied (optional).
 * front1, rear1 = Q1 pointers
 * front2, rear2 = Q2 pointers
 */
void concatenateQueues(
    struct Node** front1, struct Node** rear1,
    struct Node** front2, struct Node** rear2
) {
    /* Case 1: Q2 is empty — nothing to do */
    if (*front2 == NULL) return;

    /* Case 2: Q1 is empty — Q1 becomes Q2 */
    if (*front1 == NULL) {
        *front1 = *front2;
        *rear1  = *rear2;
    } else {
        /* Case 3: Both non-empty — link rear of Q1 to front of Q2 */
        (*rear1)->next = *front2;   /* Connect Q1's tail to Q2's head */
        *rear1 = *rear2;             /* Update Q1's rear to Q2's rear */
    }

    /* Reset Q2 (optional — Q2 is now consumed) */
    *front2 = *rear2 = NULL;

    printf("Queues merged successfully!\n");
}
