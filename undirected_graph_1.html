Representation – Adjacency Matrix

// Foundation

An undirected graph is a set of vertices (nodes) connected by edges (links) where edges have no direction — if A is connected to B, then B is connected to A.

An adjacency matrix is a 2D square array of size V × V (V = number of vertices). If there is an edge between vertex i and vertex j, then matrix[i][j] = 1 AND matrix[j][i] = 1 (because the graph is undirected). Otherwise the cell holds 0. The diagonal is always 0 (no self-loops by default).
💡Real-life analogy: Think of a friendship table in school. Rows and columns are students. If Ali and Bob are friends, put 1 in row "Ali", column "Bob" — AND also row "Bob", column "Ali". If they are not friends, it's 0. The table represents every possible friendship pair at a glance.
Adjacency Matrix
→ Build a graph and see the matrix update live.
C adjacency_matrix.c — Structure definition
#include <stdio.h>

#define MAX 10  /* Maximum number of vertices */

/* Graph structure using adjacency matrix */
struct Graph {
    int adj[MAX][MAX];  /* The matrix — adj[i][j] = 1 means edge exists */
    int numVertices;    /* How many vertices the graph currently has */
};

/* Initialize: fill all cells with 0 */
void initGraph(struct Graph *g, int v) {
    g->numVertices = v;
    int i, j;
    for (i = 0; i < v; i++)
        for (j = 0; j < v; j++)
            g->adj[i][j] = 0;  /* No edges at start */
}

/* Print the matrix */
void printMatrix(struct Graph *g) {
    int i, j;
    printf("Adjacency Matrix:\n");
    for (i = 0; i < g->numVertices; i++) {
        for (j = 0; j < g->numVertices; j++)
            printf("%d ", g->adj[i][j]);
        printf("\n");
    }
}

int main() {
    struct Graph g;
    initGraph(&g, 4);     /* 4 vertices: 0,1,2,3 */
    g.adj[0][1] = 1;  g.adj[1][0] = 1;  /* Edge 0-1 */
    g.adj[1][2] = 1;  g.adj[2][1] = 1;  /* Edge 1-2 */
    g.adj[2][3] = 1;  g.adj[3][2] = 1;  /* Edge 2-3 */
    printMatrix(&g);
    return 0;
}

Space Complexity

Requires O(V²) memory. Works best for dense graphs (many edges). For sparse graphs, adjacency list is more efficient.

Symmetric Property

In undirected graphs, the matrix is always symmetric — the top-right triangle mirrors the bottom-left triangle exactly.

Edge Lookup

Checking if edge (i, j) exists is O(1) — just read adj[i][j]. This is the biggest advantage over adjacency lists.

Vertex Operations

// Insert · Delete · Search · Update

Insert Vertex
Adding a new vertex means adding a new row and column to the matrix. All new cells are initialized to 0 (no edges yet). The vertex count increases by 1. Since the matrix is pre-allocated, we just increment numVertices.
🏫Analogy: A new student joins the class. You add a new row and column to the friendship table with their name. Initially they know nobody, so all their cells are 0.
Vertices: 3
Matrix
→ Click "Insert Vertex" to add a new node.
C Insert Vertex — O(1)
/* Insert a new vertex into the graph */
void insertVertex(struct Graph *g) {
    if (g->numVertices >= MAX) {
        printf("Error: Graph is full!\n");
        return;
    }
    int v = g->numVertices;  /* New vertex index */
    int i;

    /* Set all new row entries to 0 */
    for (i = 0; i <= v; i++)
        g->adj[v][i] = 0;

    /* Set all new column entries to 0 */
    for (i = 0; i <= v; i++)
        g->adj[i][v] = 0;

    g->numVertices++;  /* Increment vertex count */
    printf("Vertex %d inserted.\n", v);
}
Delete Vertex
Deleting vertex k: shift all rows/columns after k one position left/up, filling the gap. All edges connected to k are automatically removed. The vertex count decreases by 1.
🚪Analogy: A student leaves the school. Remove their row and column from the friendship table. Re-number everyone who came after them.
C Delete Vertex — O(V²)
void deleteVertex(struct Graph *g, int k) {
    if (k < 0 || k >= g->numVertices) {
        printf("Vertex not found!\n");
        return;
    }
    int i, j;
    /* Shift rows up: remove row k */
    for (i = k; i < g->numVertices - 1; i++)
        for (j = 0; j < g->numVertices; j++)
            g->adj[i][j] = g->adj[i+1][j];

    /* Shift columns left: remove column k */
    for (j = k; j < g->numVertices - 1; j++)
        for (i = 0; i < g->numVertices - 1; i++)
            g->adj[i][j] = g->adj[i][j+1];

    g->numVertices--;  /* One fewer vertex */
    printf("Vertex %d deleted.\n", k);
}
Search Vertex
Searching for a vertex is simple — just check if the given index is within the valid range [0, numVertices). If vertices carry labels (like names), we scan a label array. Time: O(1) for index-based, O(V) for label-based.
C Search Vertex — O(1) or O(V)
/* Search vertex by index */
int searchVertexByIndex(struct Graph *g, int k) {
    if (k >= 0 && k < g->numVertices) {
        printf("Vertex %d EXISTS in the graph.\n", k);
        return 1;  /* Found */
    }
    printf("Vertex %d does NOT exist.\n", k);
    return 0;  /* Not found */
}

/* Search vertex by label (name) */
int searchVertexByLabel(char labels[][20], int n, char *name) {
    int i;
    for (i = 0; i < n; i++) {
        if (strcmp(labels[i], name) == 0)
            return i;  /* Return the index */
    }
    return -1;  /* Not found */
}
Update Vertex
Updating a vertex usually means changing its label/name (since the matrix stores indices, not labels). Edge data stays the same. If the graph has weighted edges, you can also update the vertex's associated data. Time: O(1).
C Update Vertex Label — O(1)
#include <string.h>

char labels[MAX][20];  /* Label for each vertex */

void updateVertexLabel(int k, char *newName, int numVertices) {
    if (k < 0 || k >= numVertices) {
        printf("Invalid vertex!\n");
        return;
    }
    strcpy(labels[k], newName);  /* Update the name */
    printf("Vertex %d renamed to %s\n", k, newName);
}

Edge Operations

// Insert · Delete · Search · Update

Insert Edge
Adding an edge between vertex i and vertex j: set adj[i][j] = 1 AND adj[j][i] = 1. Both must be set because the graph is undirected — the relationship goes both ways. Time: O(1).
🤝Analogy: Two cities open a road between them. Mark it in both directions on the map — you can drive from City A to City B, and also from City B to City A.
Matrix
→ Enter vertex indices and click Insert/Delete/Search.
C Insert Edge — O(1)
void insertEdge(struct Graph *g, int i, int j) {
    if (i < 0 || i >= g->numVertices ||
        j < 0 || j >= g->numVertices) {
        printf("Invalid vertices!\n");
        return;
    }
    if (i == j) {
        printf("Self-loops not allowed!\n");
        return;
    }
    /* Set BOTH sides — undirected graph! */
    g->adj[i][j] = 1;
    g->adj[j][i] = 1;
    printf("Edge (%d, %d) inserted.\n", i, j);
}
Delete Edge
Removing an edge between vertex i and j: set adj[i][j] = 0 AND adj[j][i] = 0. Both must be cleared. Time: O(1).
C Delete Edge — O(1)
void deleteEdge(struct Graph *g, int i, int j) {
    if (g->adj[i][j] == 0) {
        printf("Edge (%d, %d) does not exist!\n", i, j);
        return;
    }
    g->adj[i][j] = 0;  /* Remove both directions */
    g->adj[j][i] = 0;
    printf("Edge (%d, %d) deleted.\n", i, j);
}
Search Edge
Checking if an edge exists is the biggest advantage of the adjacency matrix — it's O(1). Just look up adj[i][j]. If it's 1, the edge exists; if 0, it doesn't.
C Search Edge — O(1) ⚡
int searchEdge(struct Graph *g, int i, int j) {
    if (g->adj[i][j] == 1) {
        printf("Edge (%d, %d) EXISTS.\n", i, j);
        return 1;
    }
    printf("Edge (%d, %d) does NOT exist.\n", i, j);
    return 0;
}
/* This O(1) lookup is the main reason to use adjacency matrix! */
Update Edge (Weighted Graph)
For weighted graphs, the matrix stores the edge weight instead of just 0/1. Updating an edge means changing the weight value in adj[i][j] and adj[j][i]. Time: O(1).
🛣️Analogy: Two cities upgrade their highway. The distance (weight) between them changes — update it in both cells of the distance table.
C Update Edge Weight — O(1)
/* For WEIGHTED graph, matrix stores weight (not just 0/1) */
int wAdj[MAX][MAX];  /* 0 means no edge, non-zero = weight */

void updateEdge(int i, int j, int newWeight, int n) {
    if (wAdj[i][j] == 0) {
        printf("Edge does not exist! Use insertEdge first.\n");
        return;
    }
    wAdj[i][j] = newWeight;
    wAdj[j][i] = newWeight;  /* Symmetric for undirected */
    printf("Edge (%d,%d) weight updated to %d\n", i, j, newWeight);
}

Traversal – BFS (Breadth-First Search)

// Level-by-level exploration

BFS starts from a given vertex and explores all its immediate neighbors first, then neighbors' neighbors, level by level. It uses a queue (FIFO) and a visited array to avoid revisiting. Think of it as throwing a stone in a pond — ripples spread outward in layers.

For each unvisited neighbor in the matrix row, we add it to the queue. Process until queue is empty.
📡Analogy: Spreading a rumor in school. You tell your 3 friends (Level 1). Each of them tells their friends (Level 2). Each of those tells more (Level 3). The rumor spreads in waves — that's BFS.
BFS Queue & Visited
→ Set start vertex and click Run BFS or Step through it.
C BFS Traversal — O(V²) with adjacency matrix
#include <stdio.h>
#define MAX 10

int queue[MAX], front = 0, rear = 0;  /* Simple queue */
int visited[MAX];

void enqueue(int v) { queue[rear++] = v; }
int  dequeue()      { return queue[front++]; }
int  isEmpty()      { return front == rear; }

void BFS(struct Graph *g, int start) {
    int i;
    /* Reset visited array */
    for (i = 0; i < g->numVertices; i++) visited[i] = 0;

    front = rear = 0;          /* Reset queue */
    visited[start] = 1;       /* Mark start as visited */
    enqueue(start);

    printf("BFS order: ");
    while (!isEmpty()) {
        int v = dequeue();
        printf("%d ", v);

        /* Check every possible neighbor */
        for (i = 0; i < g->numVertices; i++) {
            if (g->adj[v][i] == 1 && !visited[i]) {
                visited[i] = 1;  /* Mark before enqueue to avoid duplicates */
                enqueue(i);
            }
        }
    }
    printf("\n");
}

Traversal – DFS (Depth-First Search)

// Go deep before going wide

DFS starts from a vertex and goes as deep as possible along each branch before backtracking. It uses a stack (or recursion) and a visited array. At each vertex, pick one unvisited neighbor and go there. If stuck (all neighbors visited), backtrack.
🌲Analogy: Exploring a maze. You always try the first door you see. If it leads to a dead end, you come back and try the next door. You explore one full path before trying another.
DFS Stack & Visited
→ Set start vertex and run DFS.
C DFS — Recursive — O(V²)
int visited[MAX];

void DFS(struct Graph *g, int v) {
    visited[v] = 1;           /* Mark current vertex */
    printf("%d ", v);         /* Visit (print) it */

    int i;
    for (i = 0; i < g->numVertices; i++) {
        /* If neighbor exists and not visited, recurse */
        if (g->adj[v][i] == 1 && !visited[i])
            DFS(g, i);
    }
    /* Automatically backtracks when function returns */
}

void startDFS(struct Graph *g, int start) {
    int i;
    for (i = 0; i < g->numVertices; i++) visited[i] = 0;
    printf("DFS order: ");
    DFS(g, start);
    printf("\n");
}

BFS vs DFS

BFS uses a queue and finds shortest path (unweighted). DFS uses a stack/recursion and is better for cycle detection, path finding, and topological sort.

Time Complexity

Both BFS and DFS are O(V²) with adjacency matrix (must scan each row to find neighbors). With adjacency list they'd be O(V+E).

Path Operations

// Find Path · Shortest Path · All Paths

Find Path (Does a path exist?)
To check if a path exists between vertex src and vertex dest, run DFS/BFS from src and check if dest is reachable (visited). If visited[dest] == 1 after traversal, a path exists.
🗺️Analogy: Can you drive from city A to city B using existing roads? Start at A, keep driving, see if you ever reach B.
Path Trace
→ Enter source and destination, then click Find Path.
C Find Path using DFS — O(V²)
int visited[MAX];

int hasPath(struct Graph *g, int src, int dest) {
    if (src == dest) return 1;  /* Already there! */

    visited[src] = 1;
    int i;
    for (i = 0; i < g->numVertices; i++) {
        if (g->adj[src][i] == 1 && !visited[i]) {
            if (hasPath(g, i, dest))  /* Recurse deeper */
                return 1;
        }
    }
    return 0;  /* No path found from this vertex */
}

void checkPath(struct Graph *g, int src, int dest) {
    int i;
    for (i = 0; i < g->numVertices; i++) visited[i] = 0;
    if (hasPath(g, src, dest))
        printf("Path exists from %d to %d\n", src, dest);
    else
        printf("No path from %d to %d\n", src, dest);
}
Shortest Path (Unweighted – BFS)
For unweighted graphs, BFS naturally finds the shortest path (fewest edges). Track the parent of each visited vertex. When you reach the destination, trace back through parents to recover the path.
C Shortest Path via BFS — O(V²)
void shortestPath(struct Graph *g, int src, int dest) {
    int visited[MAX] = {0};
    int parent[MAX];   /* Track the path */
    int queue[MAX], front = 0, rear = 0;
    int i;

    for (i = 0; i < g->numVertices; i++) parent[i] = -1;

    visited[src] = 1;
    queue[rear++] = src;

    while (front < rear) {
        int v = queue[front++];
        if (v == dest) break;  /* Found destination! */

        for (i = 0; i < g->numVertices; i++) {
            if (g->adj[v][i] && !visited[i]) {
                visited[i] = 1;
                parent[i] = v;  /* Remember how we got here */
                queue[rear++] = i;
            }
        }
    }

    /* Trace back from dest to src using parent[] */
    if (!visited[dest]) {
        printf("No path!\n"); return;
    }
    int path[MAX], len = 0, cur = dest;
    while (cur != -1) { path[len++] = cur; cur = parent[cur]; }

    printf("Shortest path: ");
    for (i = len - 1; i >= 0; i--)
        printf("%d%s", path[i], i ? " -> " : "\n");
}
All Paths (DFS Backtracking)
Finding ALL paths between two vertices uses DFS with backtracking. We maintain a path array. At each step, add the current vertex. When destination is reached, print the path. Then backtrack (undo the last choice) and try other routes.
C All Paths via DFS Backtracking
int pathArr[MAX], pathLen = 0;
int vis[MAX];

void printPath() {
    int i;
    for (i = 0; i < pathLen; i++)
        printf("%d%s", pathArr[i], (i == pathLen-1) ? "\n" : " -> ");
}

void allPaths(struct Graph *g, int src, int dest) {
    vis[src] = 1;
    pathArr[pathLen++] = src;  /* Add to current path */

    if (src == dest) {
        printPath();  /* Found a complete path! */
    } else {
        int i;
        for (i = 0; i < g->numVertices; i++) {
            if (g->adj[src][i] && !vis[i])
                allPaths(g, i, dest);  /* Try this neighbor */
        }
    }
    /* BACKTRACK: undo the current choice */
    pathLen--;
    vis[src] = 0;
}

Connectivity

// Connected Check · Components · Isolation Check

Connected Check
A graph is connected if every vertex can reach every other vertex. To check: run BFS/DFS from any vertex and count visited vertices. If visitedCount == numVertices, the graph is connected.
🌐Analogy: Is every city in a country reachable by road from the capital? Start in the capital and see if you can drive to every city.
Components
→ Click Check Connectivity to analyze the graph.
C Connected Check — O(V²)
int isConnected(struct Graph *g) {
    int visited[MAX] = {0};
    int count = 0;
    int queue[MAX], front = 0, rear = 0;
    int i;

    visited[0] = 1;  /* Start BFS from vertex 0 */
    queue[rear++] = 0;

    while (front < rear) {
        int v = queue[front++];
        count++;
        for (i = 0; i < g->numVertices; i++)
            if (g->adj[v][i] && !visited[i]) {
                visited[i] = 1;
                queue[rear++] = i;
            }
    }
    return (count == g->numVertices);  /* All reached? */
}

void checkConnected(struct Graph *g) {
    if (isConnected(g))
        printf("Graph is CONNECTED\n");
    else
        printf("Graph is DISCONNECTED\n");
}
Connected Components
A connected component is a maximal set of vertices all connected to each other. If the graph is disconnected, it has multiple components. Run DFS/BFS from each unvisited vertex — each new start represents a new component.
C Find All Connected Components — O(V²)
void dfsComponent(struct Graph *g, int v, int visited[]) {
    visited[v] = 1;
    printf("%d ", v);
    int i;
    for (i = 0; i < g->numVertices; i++)
        if (g->adj[v][i] && !visited[i])
            dfsComponent(g, i, visited);
}

void findComponents(struct Graph *g) {
    int visited[MAX] = {0};
    int compNum = 1, i;

    for (i = 0; i < g->numVertices; i++) {
        if (!visited[i]) {
            printf("Component %d: ", compNum++);
            dfsComponent(g, i, visited);
            printf("\n");
        }
    }
    printf("Total components: %d\n", compNum - 1);
}
Isolation Check
An isolated vertex has degree 0 — no edges connecting to it. Check by summing the vertex's row in the matrix. If the sum equals 0, the vertex is isolated.
C Check Isolated Vertex — O(V)
int isIsolated(struct Graph *g, int v) {
    int i;
    for (i = 0; i < g->numVertices; i++) {
        if (g->adj[v][i] == 1)
            return 0;  /* Has at least one neighbor */
    }
    return 1;  /* No neighbors — isolated! */
}

void findAllIsolated(struct Graph *g) {
    int i;
    printf("Isolated vertices: ");
    for (i = 0; i < g->numVertices; i++)
        if (isIsolated(g, i))
            printf("%d ", i);
    printf("\n");
}

Cycle Operations – Detect Cycle

// DFS with parent tracking

A cycle exists in a graph if you can start at a vertex and return to it without reusing any edge. In DFS: if we visit a neighbor that's already visited AND is not our direct parent (the vertex we came from), then a cycle exists.
🔁Analogy: You're hiking and following a map. If you find yourself back at a junction you've already passed — without retracing your last step — you've gone in a circle. That's a cycle!
DFS Trace
→ Click Detect Cycle. Toggle Edge 2-4 to create/remove a cycle.
C Cycle Detection via DFS — O(V²)
int visited[MAX];

/* parent = the vertex we came from. Avoids false cycle on back-edge */
int dfsCycle(struct Graph *g, int v, int parent) {
    visited[v] = 1;
    int i;
    for (i = 0; i < g->numVertices; i++) {
        if (g->adj[v][i]) {          /* If neighbor exists */
            if (!visited[i]) {
                if (dfsCycle(g, i, v))
                    return 1;       /* Cycle found deeper */
            } else if (i != parent) {
                return 1;           /* Already visited & not parent → CYCLE! */
            }
        }
    }
    return 0;
}

void detectCycle(struct Graph *g) {
    int i;
    for (i = 0; i < g->numVertices; i++) visited[i] = 0;

    for (i = 0; i < g->numVertices; i++) {
        if (!visited[i]) {
            if (dfsCycle(g, i, -1)) {  /* -1 = no parent */
                printf("Cycle DETECTED!\n");
                return;
            }
        }
    }
    printf("No cycle found — graph is acyclic.\n");
}

Degree Operations – Degree Analysis

// Count edges per vertex

The degree of a vertex is the number of edges connected to it. In an adjacency matrix, the degree of vertex v is simply the sum of row v (count of 1s in that row). Key facts:
• Min degree = 0 (isolated vertex)
• Max degree = V-1 (connected to all others)
• Sum of all degrees = 2 × E (handshaking lemma — each edge contributes to 2 vertices)
📱Analogy: In a phone network, your "degree" is how many contacts you have. A central hub (router) has high degree; a phone with no contacts has degree 0.
Degree Report
→ Click Analyze Degrees for the complete degree report.
C Degree Analysis — O(V²)
/* Get degree of a single vertex */
int getDegree(struct Graph *g, int v) {
    int degree = 0, i;
    for (i = 0; i < g->numVertices; i++)
        degree += g->adj[v][i];  /* Sum of row = degree */
    return degree;
}

/* Full degree analysis of the graph */
void degreeAnalysis(struct Graph *g) {
    int i, totalEdges = 0;
    int minDeg = g->numVertices, maxDeg = 0;

    printf("%-8s %-8s\n", "Vertex", "Degree");
    printf("----------------\n");

    for (i = 0; i < g->numVertices; i++) {
        int d = getDegree(g, i);
        printf("%-8d %-8d\n", i, d);
        totalEdges += d;
        if (d < minDeg) minDeg = d;
        if (d > maxDeg) maxDeg = d;
    }
    printf("Total edges = %d (sum/2)\n", totalEdges / 2);
    printf("Min degree  = %d\n", minDeg);
    printf("Max degree  = %d\n", maxDeg);
}

Advanced – MST (Minimum Spanning Tree)

// Prim's Algorithm

A Spanning Tree connects all V vertices with exactly V-1 edges and no cycles. A Minimum Spanning Tree (MST) does this with the minimum total edge weight.

Prim's Algorithm (greedy approach): Start from any vertex. Repeatedly pick the cheapest edge that connects a visited vertex to an unvisited vertex, until all vertices are included. Works perfectly with an adjacency matrix.
📡Analogy: You need to lay cables to connect 5 buildings using the least total cable length. You always extend from your current network to the nearest unconnected building — that's Prim's algorithm.
MST Edges
→ Click Run Prim's MST to find the minimum spanning tree.
C Prim's MST — O(V²)
#define INF 99999

void primMST(int w[MAX][MAX], int n) {
    int inMST[MAX];    /* Is vertex in MST? */
    int key[MAX];      /* Minimum weight to reach vertex */
    int parent[MAX];   /* Parent in MST */
    int i, count, v, u;

    for (i = 0; i < n; i++) {
        key[i] = INF;    /* Start: all keys are infinity */
        inMST[i] = 0;
        parent[i] = -1;
    }
    key[0] = 0;  /* Start from vertex 0 */

    for (count = 0; count < n; count++) {
        /* Pick vertex with minimum key NOT yet in MST */
        int minKey = INF;
        for (v = 0; v < n; v++)
            if (!inMST[v] && key[v] < minKey) {
                minKey = key[v]; u = v;
            }

        inMST[u] = 1;  /* Add u to MST */

        /* Update keys of neighbors */
        for (v = 0; v < n; v++) {
            if (w[u][v] && !inMST[v] && w[u][v] < key[v]) {
                key[v] = w[u][v];
                parent[v] = u;
            }
        }
    }
    /* Print MST edges */
    int total = 0;
    printf("MST Edges:\n");
    for (i = 1; i < n; i++) {
        printf("%d -- %d  weight %d\n", parent[i], i, w[parent[i]][i]);
        total += w[parent[i]][i];
    }
    printf("Total MST weight: %d\n", total);
}

Advanced – Graph Coloring

// Greedy coloring algorithm

Graph coloring assigns colors to vertices such that no two adjacent vertices share the same color. The minimum number of colors needed is the chromatic number χ(G).

Greedy coloring: For each vertex in order, assign the lowest-numbered color not used by any of its already-colored neighbors. Not always optimal but fast.
🗺️Analogy: Coloring a map — neighboring countries must have different colors. The classic 4-color theorem says any planar map can be colored with just 4 colors. Same idea here!
Color Assignment
→ Click Color Graph to apply greedy coloring.
C Greedy Graph Coloring — O(V²)
void greedyColor(struct Graph *g) {
    int color[MAX];       /* color[i] = assigned color */
    int available[MAX];   /* available[c] = 1 means color c is usable */
    int i, j;

    color[0] = 0;  /* Assign first color to first vertex */
    for (i = 1; i < g->numVertices; i++) color[i] = -1;  /* Uncolored */

    for (i = 0; i < g->numVertices; i++) available[i] = 1;  /* All available */

    for (i = 1; i < g->numVertices; i++) {
        /* Mark colors used by neighbors as unavailable */
        for (j = 0; j < g->numVertices; j++) {
            if (g->adj[i][j] && color[j] != -1)
                available[color[j]] = 0;  /* Neighbor's color is taken */
        }

        /* Assign the lowest available color */
        int c;
        for (c = 0; c < g->numVertices; c++)
            if (available[c]) { color[i] = c; break; }

        /* Reset available array for next vertex */
        for (j = 0; j < g->numVertices; j++) available[j] = 1;
    }

    /* Find chromatic number (max color + 1) */
    int maxColor = 0;
    printf("Vertex → Color:\n");
    for (i = 0; i < g->numVertices; i++) {
        printf("  Vertex %d → Color %d\n", i, color[i]);
        if (color[i] > maxColor) maxColor = color[i];
    }
    printf("Chromatic number: %d\n", maxColor + 1);
}

Advanced – Bipartite Check

// 2-coloring with BFS

A graph is bipartite if we can split all vertices into two groups (sets) such that every edge connects a vertex in Group A to a vertex in Group B — never two vertices in the same group.

How to check: Try to 2-color the graph using BFS. Start with color 0, assign color 1 to all neighbors, color 0 to their neighbors, etc. If at any point a vertex's neighbor has the same color, the graph is NOT bipartite.
🏆Analogy: Scheduling matches in a sports tournament. Teams are vertices, and two teams "linked" means they've never played each other. A bipartite graph means you can split teams into two groups where all games are between groups — never within.

→ Click Check Bipartite. Toggle Edge 1-3 to make it non-bipartite.
C Bipartite Check using BFS 2-coloring — O(V²)
int isBipartite(struct Graph *g) {
    int color[MAX];    /* -1 = uncolored, 0 or 1 = two groups */
    int queue[MAX], front = 0, rear = 0;
    int i;

    for (i = 0; i < g->numVertices; i++) color[i] = -1;

    /* Handle disconnected graphs: try from each vertex */
    for (i = 0; i < g->numVertices; i++) {
        if (color[i] != -1) continue;

        color[i] = 0;          /* Start: group 0 */
        queue[rear++] = i;
        front = 0;

        while (front < rear) {
            int v = queue[front++];
            int j;
            for (j = 0; j < g->numVertices; j++) {
                if (g->adj[v][j]) {     /* Neighbor exists */
                    if (color[j] == -1) {
                        color[j] = 1 - color[v]; /* Opposite color */
                        queue[rear++] = j;
                    } else if (color[j] == color[v]) {
                        return 0;  /* Same color! NOT bipartite */
                    }
                }
            }
        }
    }
    return 1;  /* Successfully 2-colored → bipartite! */
}

void checkBipartite(struct Graph *g) {
    if (isBipartite(g))
        printf("Graph IS bipartite (2-colorable)\n");
    else
        printf("Graph is NOT bipartite\n");
}

Key Insight

A graph is bipartite if and only if it contains no odd-length cycles. Any graph with a triangle (3-cycle) is NOT bipartite.

Real Applications

Job matching (workers ↔ jobs), recommendation systems (users ↔ products), scheduling, network flow problems, and matching algorithms all use bipartite graphs.
