Complete Graph using Adjacency Matrix

A complete visual + code guide — from beginner fundamentals to advanced algorithms — using C programming.

What is a Complete Graph? Beginner
Theory
A Complete Graph (denoted Kₙ) is a graph where every vertex is directly connected to every other vertex. If there are n vertices, the total number of undirected edges is:
Total Edges = n × (n - 1) / 2
For example, K4 (4 vertices) has 4×3/2 = 6 edges. Each vertex has degree (n-1) because it connects to every other vertex exactly once.
● INTERACTIVE VISUALIZATION
Real-life Analogy
Think of a round-table meeting — every person shakes hands with every other person exactly once. 5 people = 10 handshakes. That's K5!
Adjacency Matrix Beginner
Theory
An Adjacency Matrix is a 2D array where matrix[i][j] = 1 means an edge exists from vertex i to vertex j, and 0 means no edge.

For a complete graph: every cell except the diagonal is 1. The diagonal (vertex to itself) is always 0.

Properties: The matrix is symmetric (undirected graph), has n² cells, uses O(n²) space, and allows O(1) edge lookup.

C Code — Define & Initialize
#define MAX 10

int adj[MAX][MAX];   /* Adjacency matrix */
int n = 0;          /* Number of vertices */

/* Initialize all entries to 0 */
void initGraph() {
    int i, j;
    for (i = 0; i < MAX; i++)
        for (j = 0; j < MAX; j++)
            adj[i][j] = 0;
}

/* Display adjacency matrix */
void displayMatrix() {
    int i, j;
    printf("Adjacency Matrix:\n");
    for (i = 0; i < n; i++) {
        for (j = 0; j < n; j++)
            printf("%d ", adj[i][j]);
        printf("\n");
    }
}
Real-life Analogy
Imagine a flight route map. Rows and columns are cities. A "1" means there's a direct flight between two cities. In a complete graph, you can fly directly between any two cities!
Vertex Operations Beginner
Insert Vertex
Theory
Adding a new vertex means increasing n by 1 and initializing all its row/column entries to 0 in the matrix. In a complete graph, after insertion, we also add edges from the new vertex to all existing vertices.

C Code
char vertices[MAX][20];  /* Vertex labels */

void insertVertex(char *label) {
    if (n >= MAX) {
        printf("Graph is full!\n");
        return;
    }
    /* Add label */
    strcpy(vertices[n], label);
    /* Initialize new row and column to 0 */
    int i;
    for (i = 0; i <= n; i++) {
        adj[n][i] = 0;
        adj[i][n] = 0;
    }
    n++;  /* Increase vertex count */
    printf("Vertex '%s' inserted. Total: %d\n", label, n);
}
Delete Vertex
Theory
To delete vertex v, we shift all rows and columns after index v one step back in the matrix. This fills the "hole" left by the deleted vertex and reduces n by 1. All edges connected to the deleted vertex are automatically removed.
C Code
void deleteVertex(int v) {
    if (v < 0 || v >= n) {
        printf("Invalid vertex!\n");
        return;
    }
    int i, j;
    /* Shift rows and columns left/up */
    for (i = v; i < n - 1; i++) {
        /* Shift vertex labels */
        strcpy(vertices[i], vertices[i+1]);
        for (j = 0; j < n; j++)
            adj[i][j] = adj[i+1][j];
    }
    for (j = v; j < n - 1; j++)
        for (i = 0; i < n - 1; i++)
            adj[i][j] = adj[i][j+1];
    n--;  /* Reduce count */
    printf("Vertex %d deleted.\n", v);
}
Real-life Analogy
Removing a person from the round-table: everyone reconnects with the remaining people. All handshakes with the removed person are automatically cancelled.
Search Vertex
Theory
Searching for a vertex means scanning the vertices[] array for a matching label. This is a linear search with O(n) time complexity. Returns the index if found, or -1 if not found.

int searchVertex(char *label) {

        if (strcmp(vertices[i], label) == 0)
            return i;  /* Found at index i */
    }
    return -1;  /* Not found */
}

Edge Operations Beginner
Insert Edge
Theory
An edge between vertex u and vertex v is added by setting adj[u][v] = 1 and adj[v][u] = 1 (for undirected graph). No duplicate check needed since we're writing to specific indices. Self-loops (u == v) are not allowed in a simple graph.

Count Edges & Auto-Connect New Vertex
Theory
Counting edges: scan the upper triangle of the matrix (since it's symmetric), count all 1s. Auto-connecting a new vertex means inserting it and immediately setting all edges to existing vertices — making the graph complete again.
C Code
int countEdges() {
    int i, j, count = 0;
    for (i = 0; i < n; i++)
        for (j = i + 1; j < n; j++)  /* Upper triangle */
            if (adj[i][j] == 1) count++;
    printf("Total edges: %d\n", count);
    return count;
}

/* Auto-connect new vertex to all existing */
void autoConnectVertex(char *label) {
    insertVertex(label);  /* Add vertex first */
    int i, newIdx = n - 1;
    for (i = 0; i < newIdx; i++) {
        adj[newIdx][i] = 1;
        adj[i][newIdx] = 1;
    }
    printf("Vertex '%s' connected to all %d vertices.\n", label, newIdx);
}

A new employee joins a company and immediately exchanges phone numbers with every single existing employee. That's auto-connect!
Traversal Operations Intermediate
Breadth First Search (BFS)
Theory
BFS explores vertices level by level. Starting from a source vertex, it visits all neighbours first (1 hop), then their neighbours (2 hops), and so on.

It uses a Queue (FIFO). Steps:
1. Enqueue start vertex, mark it visited
2. Dequeue a vertex, process it
3. Enqueue all unvisited adjacent vertices
4. Repeat until queue is empty

In a complete graph, BFS visits ALL vertices in exactly 2 steps — start vertex at level 0, all others at level 1!

C Code — BFS with Queue
int queue[MAX], front = -1, rear = -1;
int visited[MAX];

void enqueue(int v) { queue[++rear] = v; }
int  dequeue()      { return queue[++front]; }
int  isEmpty()      { return front == rear; }

void BFS(int start) {
    int i;
    /* Reset */
    front = rear = -1;
    for (i = 0; i < n; i++) visited[i] = 0;

    printf("BFS: ");
    visited[start] = 1;
    enqueue(start);

    while (!isEmpty()) {
        int v = dequeue();
        printf("%d ", v);
        for (i = 0; i < n; i++) {
            if (adj[v][i] == 1 && !visited[i]) {
                visited[i] = 1;
                enqueue(i);
            }
        }
    }
    printf("\n");
}
Real-life Analogy
BFS is like spreading news in a group chat. You tell your direct friends first (level 1), then they tell their friends (level 2), rippling outward.
Depth First Search (DFS)
Theory
DFS explores as deep as possible before backtracking. It uses recursion (or an explicit stack).

Steps:
1. Visit current vertex, mark visited
2. Recursively visit each unvisited neighbour
3. Backtrack when no unvisited neighbours remain

In a complete graph, DFS visits all n vertices in one continuous chain without backtracking (since every vertex is adjacent to every other).

C Code — Recursive & Stack DFS
int visited[MAX];

/* Recursive DFS */
void DFS_Recursive(int v) {
    visited[v] = 1;
    printf("%d ", v);
    int i;
    for (i = 0; i < n; i++)
        if (adj[v][i] == 1 && !visited[i])
            DFS_Recursive(i);  /* Backtracking */
}

/* Stack-based DFS */
void DFS_Stack(int start) {
    int stack[MAX], top = -1, i;
    for (i = 0; i < n; i++) visited[i] = 0;
    stack[++top] = start;
    while (top != -1) {
        int v = stack[top--];
        if (!visited[v]) {
            visited[v] = 1;
            printf("%d ", v);
            for (i = n - 1; i >= 0; i--)
                if (adj[v][i] && !visited[i])
                    stack[++top] = i;
        }
    }
}
Real-life Analogy
DFS is like exploring a maze — go as deep as you can, and only come back when you hit a dead end. Then try another path from where you last branched.
Completeness Operations Intermediate
Check Complete Graph & Validate Full Connectivity
Theory
A graph is complete if every pair of vertices has an edge. We verify by checking:
1. Formula check: actual edges == n×(n-1)/2
2. Row check: every row has exactly (n-1) ones
3. Diagonal check: all diagonal entries are 0
4. Missing edge detection: find any (i,j) where adj[i][j] == 0 (for i≠j)
● COMPLETENESS CHECKER
Click "Check Completeness" to verify...
C Code
int isCompleteGraph() {
    int i, j;
    int required = n * (n - 1) / 2;
    int actual = countEdges();
    if (actual != required) return 0;
    /* Also verify no self-loops */
    for (i = 0; i < n; i++)
        if (adj[i][i] != 0) return 0;
    return 1;  /* Is complete! */
}

void detectMissingEdges() {
    int i, j, found = 0;
    for (i = 0; i < n; i++)
        for (j = i + 1; j < n; j++)
            if (adj[i][j] == 0) {
                printf("Missing: (%d-%d)\n", i, j);
                found++;
            }
    if (!found) printf("No missing edges!\n");
}
Auto-Complete Graph
Theory
Auto-completing a graph means automatically adding all missing edges so that the graph becomes complete (Kₙ). This is done in O(n²) time.
C Code
void autoCompleteGraph() {
    int i, j, added = 0;
    for (i = 0; i < n; i++) {
        for (j = 0; j < n; j++) {
            if (i != j && adj[i][j] == 0) {
                adj[i][j] = 1;  /* Add missing edge */
                added++;
            }
        }
    }
    printf("Added %d edges. Graph is now complete!\n", added/2);
}
Real-life Analogy
Like a social network suggesting "People You May Know" and automatically connecting everyone in a group — until everyone knows everyone.
Path Operations Intermediate
Find Path, Shortest Path, All Paths, Reachability, Hamiltonian Path
Theory
Find Path: DFS/BFS from source to destination. In a complete graph, a path always exists.

Shortest Path: In an unweighted complete graph, the shortest path between any two vertices is always 1 hop (direct edge).

All Paths: Use recursive backtracking DFS to enumerate every simple path.

Reachability: In Kₙ, every vertex can reach every other vertex (always reachable).

Hamiltonian Path: A path that visits every vertex exactly once. In Kₙ, many Hamiltonian paths exist — there are (n-1)!/2 Hamiltonian cycles.

C Code — All Paths (Backtracking)
int path[MAX], pathLen;
int visited[MAX];

void findAllPaths(int src, int dst) {
    visited[src] = 1;
    path[pathLen++] = src;
    if (src == dst) {
        int i;
        for (i = 0; i < pathLen; i++)
            printf("%d ", path[i]);
        printf("\n");
    } else {
        int i;
        for (i = 0; i < n; i++)
            if (adj[src][i] && !visited[i])
                findAllPaths(i, dst);
    }
    /* Backtrack */
    visited[src] = 0;
    pathLen--;
}

/* Hamiltonian Path check */
int hamiltonianPath(int pos) {
    if (pos == n) return 1;  /* All visited */
    int v;
    for (v = 0; v < n; v++) {
        if (adj[path[pos-1]][v] && !visited[v]) {
            visited[v] = 1;
            path[pos] = v;
            if (hamiltonianPath(pos+1)) return 1;
            visited[v] = 0;  /* Backtrack */
        }
    }
    return 0;
}
Connectivity Operations Intermediate
Check Connected, Reachable Vertices, Strong Connectivity
Theory
Connected Graph: A graph where every vertex can reach every other vertex.

Complete Graph = Always Connected. In fact, it is the "most connected" possible graph.

Strongly Connected: In a directed version of the complete graph, every vertex can reach every other vertex via directed paths — checked via DFS from each vertex.

Reachable Vertices: From any vertex in Kₙ, all n-1 other vertices are directly reachable (1-hop).
C Code
/* Check if graph is connected using BFS */
int isConnected() {
    int i, count = 0;
    for (i = 0; i < n; i++) visited[i] = 0;
    BFS(0);  /* BFS from vertex 0 */
    for (i = 0; i < n; i++)
        if (visited[i]) count++;
    return count == n;  /* All vertices reached? */
}

/* Count reachable vertices from src */
int reachableVertices(int src) {
    int i, count = 0;
    for (i = 0; i < n; i++) visited[i] = 0;
    DFS_Recursive(src);
    for (i = 0; i < n; i++)
        if (visited[i]) count++;
    printf("Reachable from %d: %d vertices\n", src, count);
    return count;
}

/* Strongly connected: every vertex reaches every other */
int isStronglyConnected() {
    int i;
    for (i = 0; i < n; i++) {
        int r = reachableVertices(i);
        if (r != n) return 0;
    }
    return 1;
}
Cycle Operations Intermediate
Detect Cycle, Count Cycles, Hamiltonian & Euler Cycle
Theory
Cycle Detection: A cycle exists if DFS finds a visited vertex that is not the parent of the current vertex. Complete graphs always have cycles (for n ≥ 3).

Hamiltonian Cycle: A cycle that visits every vertex exactly once and returns to start. Kₙ has (n-1)!/2 distinct Hamiltonian cycles.

Euler Cycle: A cycle that traverses every edge exactly once. A graph has an Euler cycle if and only if every vertex has an even degree. In Kₙ, each vertex has degree (n-1), so Euler cycle exists only when (n-1) is even, i.e., n is odd.
● CYCLE CHECKER
C Code
/* Detect cycle using DFS */
int detectCycleDFS(int v, int parent) {
    visited[v] = 1;
    int i;
    for (i = 0; i < n; i++) {
        if (adj[v][i]) {
            if (!visited[i]) {
                if (detectCycleDFS(i, v)) return 1;
            } else if (i != parent) {
                printf("Cycle found!\n");
                return 1;  /* Back edge = cycle */
            }
        }
    }
    return 0;
}

/* Euler Cycle exists if all vertices have even degree */
int hasEulerCycle() {
    int i;
    for (i = 0; i < n; i++) {
        int deg = findDegree(i);
        if (deg % 2 != 0) return 0;
    }
    return isConnected();  /* Also must be connected */
}

/* Hamiltonian cycle count: (n-1)!/2 */
int hamiltonianCycleCount(int nv) {
    int i, fact = 1;
    for (i = 2; i <= nv - 1; i++) fact *= i;
    return fact / 2;
}
Real-life Analogy
Euler cycle = a postman who must deliver to every street (edge) exactly once and return home. Hamiltonian cycle = a salesman who must visit every city (vertex) exactly once and return home. The Traveling Salesman Problem is about finding the shortest Hamiltonian cycle!
Degree Operations Intermediate
Find Degree, In/Out Degree, Max/Min Degree, Validation
Theory
The degree of a vertex is the number of edges connected to it.

In an undirected complete graph Kₙ, every vertex has degree exactly (n-1).

In a directed complete graph: each vertex has in-degree = n-1 and out-degree = n-1.

Handshaking Lemma: Sum of all degrees = 2 × (number of edges).
● DEGREE CALCULATOR
C Code
/* Degree = count of 1s in row v */
int findDegree(int v) {
    int i, deg = 0;
    for (i = 0; i < n; i++)
        deg += adj[v][i];
    printf("Degree of %d: %d\n", v, deg);
    return deg;
}

/* In-degree (directed): count col v */
int inDegree(int v) {
    int i, deg = 0;
    for (i = 0; i < n; i++)
        deg += adj[i][v];
    return deg;
}

/* Out-degree (directed): count row v */
int outDegree(int v) {
    int i, deg = 0;
    for (i = 0; i < n; i++)
        deg += adj[v][i];
    return deg;
}

/* Max and min degree */
void degreeStats() {
    int i, maxD = 0, minD = n;
    for (i = 0; i < n; i++) {
        int d = findDegree(i);
        if (d > maxD) maxD = d;
        if (d < minD) minD = d;
    }
    printf("Max: %d  Min: %d\n", maxD, minD);
}

/* Degree validation for complete graph */
int validateDegree() {
    int i;
    for (i = 0; i < n; i++)
        if (findDegree(i) != n - 1) return 0;

}
Structural Operations Advanced
Size, Order, Density, Clique, Symmetry, Dense Graph
Theory
Graph Order: Number of vertices = n.
Graph Size: Number of edges = n(n-1)/2 for complete graph.

Density: Ratio of actual edges to maximum possible edges.
Density = (2 × |E|) / (n × (n-1))  →  Complete graph density = 1.0
Clique: A subset of vertices where every pair is connected — a complete graph IS a clique by definition. Finding the maximum clique in an arbitrary graph is NP-Hard.

Symmetry Check: For undirected graphs, adj[i][j] must equal adj[j][i] for all i,j.
● STRUCTURAL ANALYSIS
C Code
int   graphOrder()  { return n; }
int   graphSize()   { return countEdges(); }

float graphDensity() {
    int maxEdges = n * (n - 1) / 2;
    if (maxEdges == 0) return 0;
    return (float)countEdges() / maxEdges;
}

/* Symmetry check (undirected validation) */
int isSymmetric() {
    int i, j;
    for (i = 0; i < n; i++)
        for (j = 0; j < n; j++)
            if (adj[i][j] != adj[j][i]) return 0;
    return 1;
}

/* Dense graph: density > 0.5 */
int isDenseGraph() {
    return graphDensity() > 0.5f;
}

/* Clique detection: check subset of size k */
int isClique(int *subset, int k) {
    int i, j;
    for (i = 0; i < k; i++)
        for (j = i+1; j < k; j++)
            if (!adj[subset[i]][subset[j]]) return 0;
    return 1;
}
Advanced Algorithms Advanced
Hamiltonian Algorithms, Clique Algorithms, Graph Coloring, TSP, Complete Subgraph
Theory — Hamiltonian Algorithms
In Kₙ, a Hamiltonian cycle visits all n vertices exactly once and returns to start. There are (n-1)!/2 distinct Hamiltonian cycles. For K5, that's 4!/2 = 12 distinct cycles!

We use backtracking to find one: try each unvisited vertex as the next step, backtrack if stuck.
C Code — Hamiltonian Cycle (Backtracking)
int ham[MAX];  /* Stores the cycle path */

int isSafe(int v, int pos) {
    if (!adj[ham[pos-1]][v]) return 0;
    int i;
    for (i = 0; i < pos; i++)
        if (ham[i] == v) return 0;
    return 1;
}

int hamiltonUtil(int pos) {
    if (pos == n) {
        /* Check if last vertex connects to first */
        return adj[ham[pos-1]][ham[0]] ? 1 : 0;
    }
    int v;
    for (v = 1; v < n; v++) {
        if (isSafe(v, pos)) {
            ham[pos] = v;
            if (hamiltonUtil(pos+1)) return 1;
            ham[pos] = -1;  /* Backtrack */
        }
    }
    return 0;
}

void findHamiltonianCycle() {
    int i;
    for (i = 0; i < n; i++) ham[i] = -1;
    ham[0] = 0;  /* Start at vertex 0 */
    if (hamiltonUtil(1)) {
        printf("Hamiltonian Cycle: ");
        for (i = 0; i < n; i++) printf("%d ", ham[i]);
        printf("%d\n", ham[0]);
    }
}
Theory — Graph Coloring
Graph Coloring assigns colors to vertices such that no two adjacent vertices share the same color. The minimum colors needed = chromatic number χ(G).

For a complete graph Kₙ: χ(Kₙ) = n — since every vertex is adjacent to every other, all n vertices must have different colors.

This is why complete graphs are the "hardest" to color!
C Code — Greedy Graph Coloring
int color[MAX];

void greedyColoring() {
    int i, j;
    int available[MAX];
    for (i = 0; i < n; i++) color[i] = -1;

    color[0] = 0;  /* First vertex gets color 0 */

    for (i = 1; i < n; i++) {
        /* Mark colors used by neighbours */
        for (j = 0; j < n; j++) available[j] = 1;
        for (j = 0; j < n; j++)
            if (adj[i][j] && color[j] != -1)
                available[color[j]] = 0;
        /* Assign first available color */
        for (j = 0; j < n; j++)
            if (available[j]) { color[i] = j; break; }
    }
    /* In Kn, each vertex gets a unique color */
    for (i = 0; i < n; i++)
        printf("Vertex %d → Color %d\n", i, color[i]);
}
Theory — Traveling Salesman Problem (TSP) Basics
The TSP asks: given a weighted complete graph, find the shortest Hamiltonian cycle.

This is one of the most famous NP-Hard problems. For small n, we can try all permutations (O(n!)) or use dynamic programming (Held-Karp: O(2ⁿ·n²)).

In practice, heuristics and approximation algorithms are used for large graphs.
C Code — TSP Brute Force (small n)
int weight[MAX][MAX];  /* Edge weights */
int minCost = 99999;
int bestPath[MAX];

void tspUtil(int v, int count, int cost, int *path) {
    if (count == n && weight[v][0]) {
        int total = cost + weight[v][0];
        if (total < minCost) {
            minCost = total;
            /* Save best path */
        }
        return;
    }
    int i;
    for (i = 0; i < n; i++) {
        if (!visited[i] && weight[v][i]) {
            visited[i] = 1;
            tspUtil(i, count+1, cost+weight[v][i], path);
            visited[i] = 0;  /* Backtrack */
        }
    }
}
/* O(n!) — only practical for n <= 12 */
Theory — Complete Subgraph (Clique) Detection
A clique is a subset of vertices that form a complete subgraph. Finding the maximum clique is NP-Hard in general graphs.

In Kₙ, the graph itself is the maximum clique. But finding max cliques in subgraphs uses the Bron-Kerbosch algorithm.

C Code — All Utility Functions
/* Check if graph has no vertices */
int isEmpty() {
    return n == 0;
}

/* Remove all vertices and edges */
void clearGraph() {
    int i, j;
    n = 0;
    for (i = 0; i < MAX; i++)
        for (j = 0; j < MAX; j++)
            adj[i][j] = 0;
    printf("Graph cleared!\n");
}

/* Copy graph into another array */
void copyGraph(int dest[MAX][MAX]) {
    int i, j;
    for (i = 0; i < n; i++)
        for (j = 0; j < n; j++)
            dest[i][j] = adj[i][j];
    printf("Graph copied!\n");
}

/* Display full adjacency matrix and vertices */
void displayGraph() {
    int i, j;
    printf("Vertices: %d | Edges: %d\n", n, countEdges());
    printf("   ");
    for (i = 0; i < n; i++) printf("%s ", vertices[i]);
    printf("\n");
    for (i = 0; i < n; i++) {
        printf("%s ", vertices[i]);
        for (j = 0; j < n; j++)
            printf("%d  ", adj[i][j]);
        printf("\n");
    }
}

/* Reset visited array before traversal */
void resetVisited() {
    int i;
    for (i = 0; i < n; i++)
        visited[i] = 0;
}
