01

Representation — Adjacency List

What is an Undirected Graph?

A graph is a collection of vertices (nodes) connected by edges. In an undirected graph, every edge goes both ways — if node A connects to B, then B also connects to A.


An Adjacency List stores the graph as an array of linked lists. Each index represents a vertex, and the linked list at that index stores all neighbors of that vertex.

Memory Layout
1
Array of size V — one slot per vertex.
2
Each slot holds a pointer to a linked list of neighbors.
3
Each node in the list has: neighbor's index + pointer to next.
4
Undirected: edge(u,v) adds v to u's list AND u to v's list.
OperationTimeSpace
Add vertexO(1)—
Add edgeO(1)—
Remove edgeO(E)—
Find neighborsO(deg)—
Check edgeO(deg)—
Total space—O(V+E)
// Real-Life Analogy

Think of a phone contact list. Each person (vertex) has a list of friends they can call (neighbors). If Alice has Bob's number, Bob also has Alice's — that's undirected. The adjacency list is exactly this: a phonebook per person, listing who they're connected to.

Hover over a vertex to highlight its adjacency list.
C
/* ─── Undirected Graph using Adjacency List ─── */

#include <stdio.h>
#include <stdlib.h>

#define MAX_VERTICES 100

/* A single node in the linked list (one neighbor) */
struct Node {
    int vertex;          /* neighbor's index */
    struct Node* next;   /* pointer to next neighbor */
};

/* The graph: an array of linked list heads */
struct Graph {
    int numVertices;
    struct Node* adjList[MAX_VERTICES]; /* array of lists */
};

/* Create a new graph with V vertices */
struct Graph* createGraph(int V) {
    struct Graph* g = (struct Graph*)malloc(sizeof(struct Graph));
    g->numVertices = V;
    for (int i = 0; i < V; i++)
        g->adjList[i] = NULL; /* start with empty lists */
    return g;
}

/* Print the entire adjacency list */
void printGraph(struct Graph* g) {
    for (int i = 0; i < g->numVertices; i++) {
        printf("Vertex %d: ", i);
        struct Node* cur = g->adjList[i];
        while (cur) {
            printf("%d → ", cur->vertex);
            cur = cur->next;
        }
        printf("NULL\n");
    }
}

Vertex Operations — Insert Vertex

Theory

Inserting a vertex means adding a new slot to our adjacency list array. The new vertex starts with no neighbors — its list is empty (NULL). We simply increment the vertex count and initialize the new list to NULL.

// Real-Life Analogy

A new student joins school. The teacher adds their name to the class register with no friends yet. The student exists but hasn't made connections.

Key Points
  • Time complexity: O(1)
  • New vertex index = current numVertices
  • adjList[new] = NULL initially
  • Must not exceed MAX_VERTICES
// insert vertex visualization
Click "+ Insert Vertex" to add a new vertex to the graph.
C
/* Insert a new vertex into the graph */
int insertVertex(struct Graph* g) {
    if (g->numVertices >= MAX_VERTICES) {
        printf("Graph is full!\n");
        return -1;
    }
    int newIndex = g->numVertices;   /* next available index */
    g->adjList[newIndex] = NULL;    /* no neighbors yet */
    g->numVertices++;                /* update count */
    printf("Vertex %d inserted.\n", newIndex);
    return newIndex;
}

/* Usage */
int main() {
    struct Graph* g = createGraph(0); /* start empty */
    insertVertex(g); /* vertex 0 */
    insertVertex(g); /* vertex 1 */
    insertVertex(g); /* vertex 2 */
    printGraph(g);   /* each has NULL list */
    return 0;
}
03

Vertex Operations — Delete Vertex

Theory

Deleting a vertex requires two steps: (1) Free the adjacency list of the vertex itself. (2) Scan all other vertices' lists and remove any reference to the deleted vertex. This ensures no dangling pointers remain.

// Real-Life Analogy

A student leaves school. The teacher removes them from the register AND removes their name from every other student's "friends" list — otherwise classmates would reference someone who no longer exists.

Key Points
  • Time complexity: O(V + E)
  • Must remove from ALL other lists
  • Free all malloc'd nodes
  • In simple arrays, "shift" or "mark deleted"
// delete vertex visualization
Choose a vertex and click Delete. Watch all references disappear.
C
/* Helper: free a linked list */
void freeList(struct Node* head) {
    while (head) {
        struct Node* tmp = head;
        head = head->next;
        free(tmp);
    }
}

/* Helper: remove a specific value from a linked list */
struct Node* removeFromList(struct Node* head, int val) {
    struct Node dummy; dummy.next = head;
    struct Node* prev = &dummy;
    while (prev->next) {
        if (prev->next->vertex == val) {
            struct Node* del = prev->next;
            prev->next = del->next;
            free(del);
        } else prev = prev->next;
    }
    return dummy.next;
}

/* Delete vertex v from the graph */
void deleteVertex(struct Graph* g, int v) {
    if (v < 0 || v >= g->numVertices) { printf("Invalid!\n"); return; }
    
    /* Step 1: Free vertex v's own list */
    freeList(g->adjList[v]);
    g->adjList[v] = NULL;

    /* Step 2: Remove v from every other vertex's list */
    for (int i = 0; i < g->numVertices; i++) {
        if (i != v)
            g->adjList[i] = removeFromList(g->adjList[i], v);
    }
    printf("Vertex %d deleted.\n", v);
}

Vertex Operations — Search Vertex

Theory

Since our graph stores vertices in a simple array (0 to numVertices-1), searching for a vertex is just checking if the given index is within the valid range. We can also retrieve its neighbor list in O(1).

// Real-Life Analogy

Looking up a student by their roll number. If roll number 5 exists (between 1 and class size), they're found instantly — and you can then read their contacts list.

Key Points
  • Time: O(1) for existence check
  • O(deg v) to list all neighbors
  • Vertex is valid if 0 ≤ v < numVertices
  • Also check if list is NULL (isolated)
// search vertex
Enter a vertex index and search. Found vertex highlights in orange.
C
/* Check if vertex v exists */
int searchVertex(struct Graph* g, int v) {
    if (v >= 0 && v < g->numVertices) {
        printf("Vertex %d found! Neighbors: ", v);
        struct Node* cur = g->adjList[v];
        if (!cur) printf("(none — isolated)");
        while (cur) {
            printf("%d ", cur->vertex);
            cur = cur->next;
        }
        printf("\n");
        return 1; /* found */
    }
    printf("Vertex %d does not exist.\n", v);
    return 0; /* not found */
}

Vertex Operations — Update Vertex

Theory

In a pure index-based graph, vertices don't have a "value" — they are identified by index. To support labels/data, we maintain a separate label[] array alongside the adjacency list. Updating a vertex means changing this label array.

// Real-Life Analogy

A student changes their name. The roll number stays the same (index), but the name displayed next to roll #3 is updated. All connections remain intact.

C
/* Extended graph with labels */
struct Graph {

06

Edge Operations — Insert Edge

Theory

An edge connects two vertices. In an undirected graph, inserting edge (u, v) means prepending v to u's list AND u to v's list. We prepend (add to front) because it's O(1) — no need to traverse to the end.

// Real-Life Analogy

Two cities build a road between them. You add the other city to each city's "reachable cities" list. Since roads go both ways, both cities get updated simultaneously.

Key Points
  • Time: O(1) — prepend to both lists
  • Creates two new Node structs
  • Self-loops: u == v (usually disallowed)
  • May create duplicate edges — add a check if needed
// insert edge
Enter two vertex indices and add an edge between them.
C
/* Create a new list node */
struct Node* createNode(int v) {
    struct Node* n = (struct Node*)malloc(sizeof(struct Node));
    n->vertex = v;
    n->next = NULL;
    return n;
}

/* Insert undirected edge between u and v */
void insertEdge(struct Graph* g, int u, int v) {
    if (u < 0 || v < 0 || u >= g->numVertices || v >= g->numVertices) {
        printf("Invalid vertices!\n");
        return;
    }

    /* Add v to u's list (prepend) */
    struct Node* nodeV = createNode(v);
    nodeV->next = g->adjList[u];
    g->adjList[u] = nodeV;

    /* Add u to v's list (prepend) — undirected! */
    struct Node* nodeU = createNode(u);
    nodeU->next = g->adjList[v];
    g->adjList[v] = nodeU;

    printf("Edge (%d-%d) inserted.\n", u, v);
}

Edge Operations — Delete Edge

Theory

Deleting edge (u, v): remove v from u's adjacency list AND remove u from v's adjacency list. We must traverse both lists to find and unlink the target nodes, then free them.

// Real-Life Analogy

Two cities close the road between them. Each city removes the other from its "connected cities" list. The road is gone in both directions.

Key Points
  • Time: O(deg u + deg v)
  • Must remove from BOTH lists
  • Use a prev pointer to unlink node
  • Check edge exists before deleting
C
/* Remove a single node with given value from list */
struct Node* removeNode(struct Node* head, int val) {
    struct Node dummy; dummy.next = head;
    struct Node* prev = &dummy;
    while (prev->next) {
        if (prev->next->vertex == val) {
            struct Node* del = prev->next;
            prev->next = del->next;
            free(del);
            break; /* remove only first occurrence */
        }
        prev = prev->next;
    }
    return dummy.next;
}

/* Delete undirected edge (u, v) */
void deleteEdge(struct Graph* g, int u, int v) {
    g->adjList[u] = removeNode(g->adjList[u], v);
    g->adjList[v] = removeNode(g->adjList[v], u);
    printf("Edge (%d-%d) deleted.\n", u, v);
}

Edge Operations — Search Edge

Theory

To check if edge (u, v) exists: traverse u's adjacency list and look for v. If found, the edge exists. In undirected graphs, we only need to check one side — if v is in u's list, u is in v's list too.

// Real-Life Analogy

To check if there's a direct road from City A to City B, look at City A's "roads" list. If City B is in it — yes, there's a direct road.

Key Points
  • Time: O(deg u)
  • Traverse u's list, look for v
  • Return 1 (found) or 0 (not found)
  • Only need to check one direction
C
/* Check if edge (u, v) exists */
int searchEdge(struct Graph* g, int u, int v) {
    struct Node* cur = g->adjList[u]; /* start at u's list */
    while (cur != NULL) {
        if (cur->vertex == v) {
            printf("Edge (%d-%d) EXISTS.\n", u, v);
            return 1;
        }
        cur = cur->next;
    }
    printf("Edge (%d-%d) does NOT exist.\n", u, v);
    return 0;
}

Edge Operations — Update Edge

Theory

In a simple graph (no weights), edges have no data to update. In a weighted graph, each edge has a weight. We extend our Node struct to include a weight field. Updating an edge means finding the node and modifying its weight.

// Real-Life Analogy

The flight distance between two cities changes due to route optimization. You update the "distance" attached to that route in both cities' flight lists.

Key Points
  • Time: O(deg u + deg v)
  • Add weight field to Node struct
  • Update both sides (undirected)
  • Check edge exists first
C
/* Weighted node */
struct Node {
    int vertex;
    int weight;          /* edge weight */
    struct Node* next;
};

/* Update weight of edge (u, v) */
void updateEdge(struct Graph* g, int u, int v, int newWeight) {
    /* Update weight in u's list */
    struct Node* cur = g->adjList[u];
    while (cur) {
        if (cur->vertex == v) { cur->weight = newWeight; break; }
        cur = cur->next;
    }
    /* Update weight in v's list (undirected) */
    cur = g->adjList[v];
    while (cur) {
        if (cur->vertex == u) { cur->weight = newWeight; break; }
        cur = cur->next;
    }
    printf("Edge (%d-%d) weight updated to %d.\n", u, v, newWeight);
}

Traversal — BFS (Breadth First Search)

Theory

BFS explores the graph level by level, starting from a source vertex. It uses a queue: visit a node, enqueue all its unvisited neighbors, then process them one by one. This guarantees we visit nearest nodes first.


1
Enqueue start vertex, mark visited
2
Dequeue front, process it
3
Enqueue all unvisited neighbors
4
Repeat until queue empty
Characteristics
  • Time: O(V + E)
  • Space: O(V) — for queue + visited array
  • Finds shortest path (unweighted)
  • Best for: level-by-level exploration
  • Uses queue (FIFO)
// Real-Life Analogy

Spreading news at a party: you tell your friends (level 1), they tell their friends (level 2), and so on. Everyone at the same "distance" from you hears the news at the same time.

// BFS traversal animation
Press "Run BFS" to animate breadth-first search.
C
/* BFS using a simple integer queue */
void BFS(struct Graph* g, int start) {
    int visited[MAX_VERTICES] = {0}; /* all unvisited */
    int queue[MAX_VERTICES];
    int front = 0, rear = 0;

    /* Enqueue start */
    visited[start] = 1;
    queue[rear++] = start;

    printf("BFS order: ");

    while (front < rear) {             /* while queue not empty */
        int v = queue[front++];        /* dequeue front */
        printf("%d ", v);

        struct Node* cur = g->adjList[v];
        while (cur) {
            if (!visited[cur->vertex]) {
                visited[cur->vertex] = 1;
                queue[rear++] = cur->vertex; /* enqueue neighbor */
            }
            cur = cur->next;
        }
    }
    printf("\n");
}

Traversal — DFS (Depth First Search)

Theory

DFS explores as deep as possible along one branch before backtracking. It can be implemented recursively (using the call stack) or iteratively (using an explicit stack). DFS is excellent for detecting cycles, finding paths, and topological sorting.

1
Visit current vertex, mark visited
2
Recursively visit each unvisited neighbor
3
Backtrack when no unvisited neighbor
BFS vs DFS

Cycle detectionYesYes
MemoryMoreLess
// Real-Life Analogy

Exploring a maze: always take the first available turn, go as deep as you can, then backtrack to the last junction and try the next turn. You fully explore one path before trying another.

// DFS traversal animation
Press "Run DFS" to animate depth-first search.
C
/* Recursive DFS helper */
void DFSHelper(struct Graph* g, int v, int* visited) {
    visited[v] = 1;
    printf("%d ", v); /* visit */

    struct Node* cur = g->adjList[v];
    while (cur) {
        if (!visited[cur->vertex])
            DFSHelper(g, cur->vertex, visited); /* recurse deep */
        cur = cur->next;
    }
}

/* DFS starting from vertex 'start' */
void DFS(struct Graph* g, int start) {
    int visited[MAX_VERTICES] = {0};
    printf("DFS order: ");
    DFSHelper(g, start, visited);
    printf("\n");
}

Path Operations — Find Path

Theory

Find whether any path exists between source and destination. We use DFS: start at source, recursively explore neighbors. Keep a path[] array to record the route. If we reach the destination, print and return true.

// Real-Life Analogy

Can you drive from City A to City Z using any combination of roads? Follow roads one by one, backtracking dead ends, until you find a route or confirm none exists.

Key Points
  • Time: O(V + E)
  • DFS-based exploration
  • Track path in an array
  • Mark visited to avoid loops
  • Returns first found path (not necessarily shortest)
C
int findPathDFS(struct Graph* g, int cur, int dest,
                 int* visited, int* path, int pathLen) {
    visited[cur] = 1;
    path[pathLen++] = cur;

    if (cur == dest) {              /* destination reached! */
        printf("Path: ");
        for (int i = 0; i < pathLen; i++)
            printf("%d%s", path[i], i+1<pathLen?"→":"\n");
        return 1;
    }

    struct Node* neighbor = g->adjList[cur];
    while (neighbor) {
        if (!visited[neighbor->vertex]) {
            if (findPathDFS(g, neighbor->vertex, dest, visited, path, pathLen))
                return 1;
        }
        neighbor = neighbor->next;
    }
    return 0; /* backtrack */
}

void findPath(struct Graph* g, int src, int dest) {
    int visited[MAX_VERTICES]={0}, path[MAX_VERTICES];
    if (!findPathDFS(g, src, dest, visited, path, 0))
        printf("No path from %d to %d.\n", src, dest);
}

Path Operations — Shortest Path (BFS)

Theory

In an unweighted graph, BFS naturally gives the shortest path (fewest edges). We track a parent[] array: when we visit a new vertex, we record which vertex led us there. After BFS, we trace back from destination to source using parent pointers.

// Real-Life Analogy

Finding the quickest route using the fewest train stops. BFS explores all 1-stop routes first, then 2-stop, etc. The first time you reach the destination, that's the minimum stops.

// shortest path visualization
Find the shortest path between two vertices using BFS.
C
void shortestPath(struct Graph* g, int src, int dest) {
    int visited[MAX_VERTICES] = {0};
    int parent[MAX_VERTICES];
    int queue[MAX_VERTICES];
    int front = 0, rear = 0;

    for (int i=0; i<g->numVertices; i++) parent[i] = -1;

    visited[src] = 1;
    queue[rear++] = src;

    while (front < rear) {
        int v = queue[front++];
        if (v == dest) break; /* found! */
        struct Node* cur = g->adjList[v];
        while (cur) {
            if (!visited[cur->vertex]) {
                visited[cur->vertex] = 1;
                parent[cur->vertex] = v; /* record parent */
                queue[rear++] = cur->vertex;
            }
            cur = cur->next;
        }
    }

    /* Trace path from dest to src using parent[] */
    int path[MAX_VERTICES], pLen = 0;
    for (int v = dest; v != -1; v = parent[v])
        path[pLen++] = v;
    printf("Shortest path: ");
    for (int i = pLen-1; i >= 0; i--)
        printf("%d%s", path[i], i?"→":"\n");
}

Path Operations — All Paths

Theory

Find ALL possible paths from source to destination using recursive DFS with backtracking. At each step: mark current as visited, explore neighbors, when reaching destination print path, then unmark (backtrack) to allow the vertex to be used in other paths.

// Real-Life Analogy

Finding all possible driving routes from home to airport. You exhaustively try every combination of turns, noting each complete route that reaches the airport.

C
void allPathsDFS(struct Graph* g, int cur, int dest,
                  int* visited, int* path, int pLen) {
    visited[cur] = 1;
    path[pLen++] = cur;

    if (cur == dest) {           /* found one complete path */
        for (int i=0; i<pLen; i++)
            printf("%d%s", path[i], i+1<pLen?"→":"\n");
    } else {
        struct Node* cur2 = g->adjList[cur];
        while (cur2) {
            if (!visited[cur2->vertex])
                allPathsDFS(g, cur2->vertex, dest, visited, path, pLen);
            cur2 = cur2->next;
        }
    }

    visited[cur] = 0; /* BACKTRACK: unmark so others can use */
}

void allPaths(struct Graph* g, int src, int dest) {
    int visited[MAX_VERTICES]={0}, path[MAX_VERTICES];
    printf("All paths from %d to %d:\n", src, dest);
    allPathsDFS(g, src, dest, visited, path, 0);
}

Connectivity — Connected Check

Theory

A graph is connected if there exists a path between every pair of vertices. The simplest check: start BFS/DFS from vertex 0, then verify that every vertex was visited. If any vertex remains unvisited, the graph is disconnected.

// Real-Life Analogy

Can every city reach every other city by road? Start from one city, travel all reachable roads. If some cities remain unreachable, the road network is disconnected.

Key Points
  • Time: O(V + E)
  • One DFS/BFS from vertex 0
  • Check all visited[] == 1
  • For directed: need strongly/weakly connected check
// connectivity check
Build a graph, then run the connectivity check.
C
void dfsVisit(struct Graph* g, int v, int* visited) {
    visited[v] = 1;
    struct Node* cur = g->adjList[v];
    while (cur) {
        if (!visited[cur->vertex])
            dfsVisit(g, cur->vertex, visited);
        cur = cur->next;
    }
}

int isConnected(struct Graph* g) {
    if (g->numVertices == 0) return 1;
    int visited[MAX_VERTICES] = {0};
    dfsVisit(g, 0, visited);      /* start from 0 */
    for (int i = 0; i < g->numVertices; i++)
        if (!visited[i]) return 0; /* some unreachable */
    return 1; /* all visited → connected */
}

/* Usage */
if (isConnected(g))
    printf("Graph is CONNECTED\n");
else
    printf("Graph is DISCONNECTED\n");

Connectivity — Find Components

Theory

A connected component is a maximal set of vertices where every pair is connected. To find all components: iterate over all vertices; when you find an unvisited vertex, start a DFS — all vertices visited in that DFS form one component.

// Real-Life Analogy

Islands connected by bridges: each island group forms a component. From any island in a group you can reach all others. Different island groups are separate components.

Key Points
  • Time: O(V + E)
  • Number of DFS calls = number of components
  • Use a component[] label array
  • Used in network analysis, social graphs
C
void dfsComponent(struct Graph* g, int v, int* visited, int compID) {
    visited[v] = compID;  /* label with component ID */
    struct Node* cur = g->adjList[v];
    while (cur) {
        if (!visited[cur->vertex])
            dfsComponent(g, cur->vertex, visited, compID);
        cur = cur->next;
    }
}

int findComponents(struct Graph* g) {
    int visited[MAX_VERTICES] = {0};
    int compCount = 0;
    for (int i = 0; i < g->numVertices; i++) {
        if (!visited[i]) {
            compCount++;
            dfsComponent(g, i, visited, compCount);
            printf("Component %d: vertex %d and its connected set\n", compCount, i);
        }
    }
    printf("Total components: %d\n", compCount);
    return compCount;
}

Connectivity — Isolation Check

Theory

An isolated vertex is one with degree 0 — it has no edges. Check: if adjList[v] == NULL, the vertex is isolated. We can find all isolated vertices in O(V) by scanning the adjacency list heads.

// Real-Life Analogy

A new city with no roads connecting it to any other city. It exists on the map but is completely cut off — no one can reach it and it can't reach anyone.

Key Points
  • Time: O(V)
  • adjList[v] == NULL → isolated
  • Isolated ≠ disconnected graph
  • Degree = 0 for isolated vertices
C
/* Check if a specific vertex is isolated */
int isIsolated(struct Graph* g, int v) {
    return g->adjList[v] == NULL; /* no neighbors = isolated */
}

/* Find ALL isolated vertices */
void findIsolatedVertices(struct Graph* g) {
    printf("Isolated vertices: ");
    int found = 0;
    for (int i = 0; i < g->numVertices; i++) {
        if (isIsolated(g, i)) {
            printf("%d ", i);
            found = 1;
        }
    }
    if (!found) printf("(none)");
    printf("\n");
}

Cycle Operations — Detect Cycle

Theory

A cycle exists if during DFS we encounter a visited vertex that is NOT the parent of the current vertex. In an undirected graph, we always go back to parent — that's not a cycle. But reaching a visited non-parent vertex means a cycle exists.

1
DFS from any vertex, track parent
2
If neighbor is visited AND not parent → cycle!
3
Run for all unvisited vertices (for disconnected graphs)
Key Points
  • Time: O(V + E)
  • Track parent to avoid false positives
  • Cycle detection used in: deadlock detection, network analysis
  • Tree has no cycle (V-1 edges for V vertices)
// Real-Life Analogy

Walking city roads: if you can return to a city you've already visited without retracing the exact road you came from — there's a loop (cycle) in the road network.

// cycle detection
Choose a graph type and run cycle detection.
C
int dfsCycle(struct Graph* g, int v, int parent, int* visited) {
    visited[v] = 1;
    struct Node* cur = g->adjList[v];
    while (cur) {
        if (!visited[cur->vertex]) {
            if (dfsCycle(g, cur->vertex, v, visited))
                return 1; /* cycle found deeper */
        } else if (cur->vertex != parent) {
            return 1; /* visited & not parent → CYCLE! */
        }
        cur = cur->next;
    }
    return 0;
}

int hasCycle(struct Graph* g) {
    int visited[MAX_VERTICES] = {0};
    for (int i = 0; i < g->numVertices; i++)
        if (!visited[i] && dfsCycle(g, i, -1, visited))
            return 1;
    return 0;
}

Degree Operations — Degree Analysis

Theory

The degree of a vertex is the number of edges incident to it — i.e., the length of its adjacency list. In undirected graphs: sum of all degrees = 2 × E (each edge counted twice). This is the Handshaking Lemma.


Degree analysis helps identify: hubs (high degree), leaves (degree 1), isolated (degree 0).

A social network: a celebrity (hub) may have a million followers (degree=1M). An introvert has 2-3 friends (degree=2-3). The sum rule means if you add up everyone's friend count, it's double the total friendships.

/* Get degree of vertex v */
int degree(struct Graph* g, int v) {
    int count = 0;
    struct Node* cur = g->adjList[v];
    while (cur) { count++; cur = cur->next; }
    return count;
}

/* Full degree analysis of the graph */
void degreeAnalysis(struct Graph* g) {
    int maxDeg = 0, minDeg = INT_MAX, sumDeg = 0;
    printf("Degree of each vertex:\n");
    for (int i = 0; i < g->numVertices; i++) {
        int d = degree(g, i);
        printf("  Vertex %d: degree %d\n", i, d);
        if (d > maxDeg) maxDeg = d;
        if (d < minDeg) minDeg = d;
        sumDeg += d;
    }
    printf("Max degree: %d\n", maxDeg);
    printf("Min degree: %d\n", minDeg);
    printf("Total edges: %d (sum/2 = %d)\n", sumDeg, sumDeg/2);
}

Advanced — Minimum Spanning Tree

Theory

A Minimum Spanning Tree (MST) connects all vertices with the minimum total edge weight and no cycles. Kruskal's Algorithm: sort all edges by weight, then greedily add the cheapest edge that doesn't form a cycle. We use a Union-Find data structure to detect cycles efficiently.

1
Sort all edges by weight (ascending)
2
For each edge: check if u and v are in different components
3
If yes: add edge to MST, merge components
4
Stop when MST has V-1 edges
Characteristics
  • Time: O(E log E) for sort
  • MST has exactly V-1 edges
  • MST is unique if all weights distinct
  • Applications: network design, circuit layout, clustering
// Real-Life Analogy

Laying electrical cable to connect all houses at minimum cost. You pick the cheapest cable runs that eventually connect everyone, never creating unnecessary loops (redundant connections).

/* Union-Find (Disjoint Set) for cycle detection */
int parent[MAX_VERTICES];

int find(int x) {
    if (parent[x] != x) parent[x] = find(parent[x]); /* path compress */
    return parent[x];
}

void unionSets(int a, int b) { parent[find(a)] = find(b); }

/* Edge struct for Kruskal */
struct Edge { int u, v, w; };

/* Compare function for sorting edges */
int cmpEdge(const void* a, const void* b) {
    return ((struct Edge*)a)->w - ((struct Edge*)b)->w;
}

void kruskalMST(struct Edge* edges, int E, int V) {
    /* Initialize Union-Find */
    for (int i=0; i<V; i++) parent[i] = i;

    /* Sort edges by weight */
    qsort(edges, E, sizeof(struct Edge), cmpEdge);

    printf("MST Edges:\n");
    int mstCost = 0, edgesUsed = 0;

    for (int i = 0; i < E && edgesUsed < V-1; i++) {
        int pu = find(edges[i].u), pv = find(edges[i].v);
        if (pu != pv) {              /* different components: no cycle */
            printf("  %d--%d (w=%d)\n", edges[i].u, edges[i].v, edges[i].w);
            unionSets(edges[i].u, edges[i].v);
            mstCost += edges[i].w;
            edgesUsed++;
        }
    }
    printf("Total MST cost: %d\n", mstCost);
}

Advanced — Graph Coloring

Theory

Graph coloring assigns colors to vertices so no two adjacent vertices share the same color. The greedy coloring algorithm assigns the smallest available color not used by any neighbor. The minimum colors needed is called the chromatic number.

1
For vertex 0, assign color 1
2
For each next vertex, find colors used by neighbors
3
Assign smallest color NOT used by any neighbor
Key Points
  • Greedy: not always optimal but fast — O(V + E)
  • Optimal coloring is NP-Hard in general
  • Bipartite graphs need only 2 colors
  • Applications: scheduling, register allocation, map coloring
// Real-Life Analogy

Exam scheduling: if two subjects share a student, they can't be in the same time slot. Colors = time slots. You want the fewest slots (colors) with no conflicts (adjacent vertices same color).

void greedyColor(struct Graph* g) {
    int color[MAX_VERTICES];
    int available[MAX_VERTICES]; /* tracks which colors are used by neighbors */
    int V = g->numVertices;

    for (int i=0; i<V; i++) { color[i] = -1; available[i] = 0; }

    color[0] = 0; /* first vertex gets color 0 */

    for (int v = 1; v < V; v++) {
        /* Mark colors used by neighbors as unavailable */
        struct Node* cur = g->adjList[v];
        while (cur) {
            if (color[cur->vertex] != -1)
                available[color[cur->vertex]] = 1; /* mark unavailable */
            cur = cur->next;
        }
        /* Find smallest available color */
        int c;
        for (c = 0; available[c]; c++);
        color[v] = c;

        /* Reset for next vertex */
        cur = g->adjList[v];
        while (cur) {
            if (color[cur->vertex] != -1)
                available[color[cur->vertex]] = 0; /* reset */
            cur = cur->next;
        }
    }

    for (int i=0; i<V; i++)
        printf("Vertex %d → Color %d\n", i, color[i]);
}

Advanced — Bipartite Check

Theory

A graph is bipartite if its vertices can be split into two groups (sets) such that every edge goes between groups — no edge stays within one group. This is equivalent to asking: can we 2-color the graph?


Algorithm: BFS/DFS, alternately color vertices 0 and 1. If we ever need to color a vertex the same as its neighbor → NOT bipartite (contains an odd cycle).

1
Color start vertex with color 0
2
Color each unvisited neighbor with opposite color
3
If neighbor has same color as current → not bipartite
Key Facts
  • Time: O(V + E)
  • Bipartite ↔ no odd-length cycles
  • Trees are always bipartite
  • Applications: matching problems, scheduling, recommendation systems
// Real-Life Analogy

A dating app: users in Group A (left side) connect only to users in Group B (right side). No two users in the same group directly match. This is a bipartite graph — like a matching problem.

// bipartite check
Build a graph and check if it's bipartite. Colors show the two groups.
C
int isBipartite(struct Graph* g) {
    int color[MAX_VERTICES];
    int queue[MAX_VERTICES];
    for (int i=0; i<g->numVertices; i++) color[i] = -1;

    for (int start = 0; start < g->numVertices; start++) {
        if (color[start] != -1) continue; /* already colored */

        /* BFS from 'start' */
        int front = 0, rear = 0;
        color[start] = 0;
        queue[rear++] = start;

        while (front < rear) {
            int v = queue[front++];
            struct Node* cur = g->adjList[v];
            while (cur) {
                if (color[cur->vertex] == -1) {
                    color[cur->vertex] = 1 - color[v]; /* flip color */
                    queue[rear++] = cur->vertex;
                } else if (color[cur->vertex] == color[v]) {
                    printf("NOT Bipartite! Conflict at %d--%d\n", v, cur->vertex);
                    return 0; /* same color neighbors → not bipartite */
                }
                cur = cur->next;
            }
        }
    }
    printf("Graph IS Bipartite!\n");
    return 1;
}
