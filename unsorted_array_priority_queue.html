Introduction to Priority QueuesWhat is it, why use it, and how does the array version work?

❓ What is a Priority Queue?

A Priority Queue is a special type of queue where every element has a priority value attached to it. Elements are served based on their priority, not just their insertion order.


  • Higher priority elements are removed first
  • If two elements have the same priority, insertion order applies
  • Combines features of a queue and a sorted structure

📋 Unsorted Array Approach

In the unsorted array version, elements are inserted in any order — no sorting on insert. Instead, finding the highest/lowest priority requires scanning the whole array.


  • Insert: O(1) — just append
  • Delete/Peek max: O(n) — scan entire array
  • Simple to implement, great for learning
🏥
Real-Life Analogy — Hospital Emergency Room

Think of a hospital ER where patients arrive and are given a triage priority (1 = critical, 5 = minor). Patients arrive in any order (unsorted), but when a doctor is free, they scan the waiting room to pick the most critical patient. This scanning is exactly the O(n) find we do in an unsorted priority queue.

01
Priority Queue InitializationCreating the structure, defining capacity, allocating memory

📐 Theory

Before we can use a priority queue, we need to:

  • Define a struct to hold each element's data and priority
  • Declare an array of that struct with a fixed MAX_SIZE
  • Initialize a count variable to 0 (empty)
  • Optional: allocate memory dynamically with malloc

🔑 Key Concepts

  • MAX_SIZE — maximum elements the queue can hold
  • count — current number of elements (starts at 0)
  • data — the actual value stored
  • priority — integer; lower number = higher priority (common convention)
📦
Analogy — An Empty Inbox Tray

Initialization is like placing an empty inbox tray on your desk. The tray has a fixed capacity (MAX_SIZE), it's currently empty (count = 0), and each slot can hold a document (data) with a sticky note showing its urgency (priority).

C — Initialization 
/* ── Step 1: Define the structure ── */
#include <stdio.h>
#define MAX_SIZE 8   /* Maximum capacity of the queue */

/* Each element stores a value and a priority */
typedef struct {
    int data;       /* The actual value (e.g., patient ID) */
    int priority;   /* Priority (lower number = higher priority) */
} Element;

/* The priority queue itself */
typedef struct {
    Element arr[MAX_SIZE];  /* Unsorted array of elements */
    int     count;           /* Current number of elements */
} PriorityQueue;

/* ── Step 2: Initialize the queue ── */
void initQueue(PriorityQueue *pq) {
    pq->count = 0;   /* Queue starts empty */
    /* No need to zero out arr[] — count tracks valid elements */
    printf("Queue initialized. Capacity: %d\n", MAX_SIZE);
}

/* ── Main entry point ── */
int main() {
    PriorityQueue pq;   /* Declare the queue on the stack */
    initQueue(&pq);      /* Initialize it — count = 0 */
    printf("Count: %d\n", pq.count); /* Output: 0 */
    return 0;
}
02
Insert / Enqueue OperationAdding elements with priority — O(1) time complexity

➕ Theory

Inserting into an unsorted priority queue is simply appending to the end of the array — no sorting required! Steps:

  1. Check if queue is full (overflow check)
  2. Place element at index arr[count]
  3. Increment count by 1

⏱ Time Complexity


Best CaseO(1)
Average CaseO(1)
Worst CaseO(1)
SpaceO(1) extra

Insert is the fastest operation — just append. Sorting is deferred until deletion.

📬
Analogy — Mail Slot

Inserting is like dropping a letter into a mail slot — you don't sort them as they arrive; you just toss them in. When someone wants the most urgent letter, they'll search through the pile (that's the O(n) delete we'll see next).

C — Insert / Enqueue 
/* ── Enqueue: Insert element with priority ── */
void enqueue(PriorityQueue *pq, int data, int priority) {

    /* Step 1: Overflow check */
    if (pq->count == MAX_SIZE) {
        printf("Queue is FULL! Cannot insert %d.\n", data);
        return;
    }

    /* Step 2: Insert at the current end */
    pq->arr[pq->count].data     = data;
    pq->arr[pq->count].priority = priority;

    /* Step 3: Increment count */
    pq->count++;

    printf("Inserted: data=%d, priority=%d\n", data, priority);
}

/* Usage example */
int main() {
    PriorityQueue pq;
    initQueue(&pq);

    enqueue(&pq, 42, 2);  /* data=42, priority=2 */
    enqueue(&pq, 10, 1);  /* data=10, priority=1 (highest!) */
    enqueue(&pq, 99, 5);  /* data=99, priority=5 (lowest)  */
    /* Array is UNSORTED: [42,2] [10,1] [99,5] */
    return 0;
}
03
Delete Highest Priority ElementFind minimum priority value, remove it, shift array — O(n)

🗑 Theory

Since the array is unsorted, we must scan all elements to find the highest priority (lowest priority number). Steps:

  1. Check for underflow (empty queue)
  2. Scan array to find element with minimum priority value
  3. Remove that element
  4. Shift remaining elements left to fill the gap
  5. Decrement count

⏱ Time Complexity


Find max priorityO(n)
Shift elementsO(n)
OverallO(n)

This is the most expensive operation in an unsorted priority queue. We pay the O(n) cost here so insert is O(1).

🚨
Analogy — Picking the Most Urgent Patient

The nurse scans the entire waiting room (O(n) scan), finds the most critical patient, removes them from the queue, and everyone else shifts up one seat. That shifting is why deletion is O(n).

C — Delete Highest Priority 
/* ── Delete element with highest priority (lowest priority number) ── */
void deleteHighest(PriorityQueue *pq) {
    int i, highIdx;

    /* Step 1: Underflow check */
    if (pq->count == 0) {
        printf("Queue is EMPTY! Nothing to delete.\n");
        return;
    }

    /* Step 2: Find index of highest priority element */
    highIdx = 0;   /* Assume first element is highest */
    for (i = 1; i < pq->count; i++) {
        /* Lower priority number = higher priority */
        if (pq->arr[i].priority < pq->arr[highIdx].priority) {
            highIdx = i;
        }
    }

    printf("Deleting: data=%d, priority=%d\n",
           pq->arr[highIdx].data, pq->arr[highIdx].priority);

    /* Step 3: Shift elements left to fill gap */
    for (i = highIdx; i < pq->count - 1; i++) {
        pq->arr[i] = pq->arr[i + 1];
    }

    /* Step 4: Decrement count */
    pq->count--;
}
04
Delete Lowest Priority ElementFind maximum priority value, remove it — O(n)

🗑 Theory

Same idea as deleting the highest, but now we search for the element with the maximum priority number (which means it has the lowest urgency).

  1. Check for underflow
  2. Scan array to find element with maximum priority value
  3. Remove that element from the array
  4. Shift remaining elements left
  5. Decrement count

💡 Use Case

Deleting the lowest priority element is useful when:

  • Queue is nearly full — need to make space by evicting least important item
  • Cache eviction — removing the least recently used or least important entry
  • Task scheduling — cancelling the least urgent background job
🗂
Analogy — Clearing Your Desk

If your desk is getting too full (queue is almost full), you might remove the least important document (lowest priority) to make space. You scan all items, find the least urgent, discard it, and slide everything else together.

C — Delete Lowest Priority 
/* ── Delete element with lowest priority (highest priority number) ── */
void deleteLowest(PriorityQueue *pq) {
    int i, lowIdx;

    if (pq->count == 0) {
        printf("Queue is EMPTY!\n");
        return;
    }

    /* Find index of element with MAXIMUM priority number */
    lowIdx = 0;
    for (i = 1; i < pq->count; i++) {
        if (pq->arr[i].priority > pq->arr[lowIdx].priority) {
            lowIdx = i;   /* Found one with higher priority number */
        }
    }

    printf("Removing lowest: data=%d, priority=%d\n",
           pq->arr[lowIdx].data, pq->arr[lowIdx].priority);

    /* Shift elements left */
    for (i = lowIdx; i < pq->count - 1; i++) {
        pq->arr[i] = pq->arr[i + 1];
    }
    pq->count--;
}
05
Peek Highest Priority ElementRead without removing — O(n) scan

👁 Theory

Peek is a read-only operation — we find and return the highest priority element without removing it from the queue.

  • Scan the array for the minimum priority number
  • Return its data value
  • Array is not modified
  • Useful for checking what's next without committing to removal

🏷 Peek vs Delete


OperationModifies Queue?Complexity
Peek HighestNoO(n)
Delete HighestYesO(n)

Both scan the entire array, but peek leaves the queue intact.

🔍
Analogy — Reading the Top Ticket

Peek is like a nurse reading the chart of the most critical patient without actually calling them in yet. She scans all patients (O(n)), identifies who is most critical, and announces their name — but they're still in the waiting room.

C — Peek Highest Priority 
/* ── Peek: View highest priority element WITHOUT removing it ── */
int peekHighest(PriorityQueue *pq) {
    int i, highIdx;

    /* Check if queue is empty */
    if (pq->count == 0) {
        printf("Queue is EMPTY!\n");
        return -1;   /* Sentinel value for error */
    }

    /* Scan entire array for minimum priority number */
    highIdx = 0;
    for (i = 1; i < pq->count; i++) {
        if (pq->arr[i].priority < pq->arr[highIdx].priority) {
            highIdx = i;
        }
    }

    /* Return data — queue is NOT modified */
    printf("Peek: data=%d, priority=%d\n",
           pq->arr[highIdx].data, pq->arr[highIdx].priority);

    return pq->arr[highIdx].data;
}

07
IsEmpty OperationCheck if the queue has zero elements — O(1)

❔ Theory

isEmpty simply checks whether count == 0. This is an O(1) constant time operation — we don't scan anything.

  • Returns 1 (true) if queue is empty
  • Returns 0 (false) if queue has elements
  • Always call this before Delete/Peek to prevent underflow errors

🛡 Guard Pattern

All destructive operations should begin with:


if (isEmpty(&pq)) {
  printf("Error!\n");
  return;
}
🪣
Analogy — Checking an Empty Bucket

Before you try to pour water from a bucket, you glance at it to see if there's anything inside. If it's empty, you don't tip it. That's isEmpty — a quick O(1) glance before any action.

/* ── isEmpty: Returns 1 if queue has no elements ── */
int isEmpty(PriorityQueue *pq) {
    return (pq->count == 0);   /* 1 = true, 0 = false */
}

/* ── isFull: Returns 1 if queue is at max capacity ── */
int isFull(PriorityQueue *pq) {
    return (pq->count == MAX_SIZE);
}

/* Usage */
int main() {
    PriorityQueue pq;
    initQueue(&pq);

    printf("Empty? %s\n", isEmpty(&pq) ? "YES" : "NO");  /* YES */
    enqueue(&pq, 10, 1);
    printf("Empty? %s\n", isEmpty(&pq) ? "YES" : "NO");  /* NO  */
    printf("Full?  %s\n", isFull(&pq) ?  "YES" : "NO");  /* NO  */
    return 0;
}
08
IsFull OperationCheck if the queue has reached maximum capacity — O(1)

🔴 Theory

isFull checks whether count == MAX_SIZE. Another O(1) operation — just one comparison. Always call this before Insert to avoid overflow (writing past the end of the array).

  • Returns 1 (true) → queue at capacity
  • Returns 0 (false) → space available
  • Critical safety guard for all enqueue operations

⚠ Overflow vs Underflow


Error TypeTriggered WhenGuard
OverflowInsert on full queueisFull()
UnderflowDelete on empty queueisEmpty()
🚗
Analogy — Parking Lot Barrier

A parking lot with 50 spaces has a sensor that shows "FULL" when all spots are taken. The barrier arm won't lift (insertion blocked). isFull() is that sensor — it checks capacity before allowing a new car (element) in.

C — IsFull with Safe Enqueue 
/* ── isFull with safe enqueue pattern ── */
int isFull(PriorityQueue *pq) {
    return (pq->count == MAX_SIZE);
}

void safeEnqueue(PriorityQueue *pq, int data, int priority) {
    /* Guard: block overflow */
    if (isFull(pq)) {
        printf("OVERFLOW: Cannot insert %d — queue is FULL (%d/%d)\n",
               data, pq->count, MAX_SIZE);
        return;
    }
    pq->arr[pq->count].data     = data;
    pq->arr[pq->count].priority = priority;
    pq->count++;
    printf("Inserted %d (pri=%d). Count: %d/%d\n",
           data, priority, pq->count, MAX_SIZE);
}

10
Traverse Priority QueueSequential, forward, and reverse traversal — O(n)

↔ Types of Traversal

  • Forward: Index 0 → count-1 (left to right)
  • Reverse: Index count-1 → 0 (right to left)
  • Both are O(n) — visit every element once
  • Read-only — no modification to queue

🔄 Why Reverse Traverse?

  • Process elements in reverse insertion order
  • Useful in undo operations
  • Required when processing lowest priority first
  • Common in stack-like access patterns
🚶
Analogy — Walking Down a Row of Seats

Forward traversal is like walking from seat 1 to the last seat, checking each person. Reverse traversal is walking from the last seat back to seat 1. Either way, you visit everyone exactly once.

/* ── Forward Traversal: index 0 → count-1 ── */
void traverseForward(PriorityQueue *pq) {
    int i;
    printf("Forward: ");
    for (i = 0; i < pq->count; i++) {
        printf("[%d|p%d] ",
               pq->arr[i].data, pq->arr[i].priority);
    }
    printf("\n");

/* ── Reverse Traversal: count-1 → 0 ── */
void traverseReverse(PriorityQueue *pq) {
    int i;
    printf("Reverse: ");
    for (i = pq->count - 1; i >= 0; i--) {
        printf("[%d|p%d] ",
               pq->arr[i].data, pq->arr[i].priority);
    }
    printf("\n");
}
11
Search ElementLinear search by value or priority — O(n)

🔎 Theory

Since the array is unsorted, the only search strategy is linear search — scan every element comparing the target value.

  1. Start at index 0
  2. Compare each element's data with target
  3. If found, return its index and priority
  4. If not found after full scan, return -1

⏱ Why Can't We Use Binary Search?

Binary search requires a sorted array. Since our array is unsorted (insertion order), we can't use it.


Linear SearchO(n)
Binary SearchO(log n) — not applicable here
🕵️
Analogy — Finding a Specific Patient

A receptionist is asked "Is patient John here?" She checks every name on the waiting list from top to bottom until she finds "John" or reaches the end. That linear check-by-check is exactly linear search.

/* ── Search: Find element by data value ── */
int searchElement(PriorityQueue *pq, int target) {
    int i;

    for (i = 0; i < pq->count; i++) {
        if (pq->arr[i].data == target) {
            printf("Found %d at index %d (priority=%d)\n",
                   target, i, pq->arr[i].priority);
            return i;   /* Return the index */
        }
    }

    printf("%d not found in the queue.\n", target);
    return -1;   /* Not found */
}

/* Track comparisons for performance analysis */
int searchWithCount(PriorityQueue *pq, int target) {
    int i, comparisons = 0;
    for (i = 0; i < pq->count; i++) {
        comparisons++;
        if (pq->arr[i].data == target) {
            printf("Found after %d comparisons.\n", comparisons);
            return i;
        }
    }
    printf("Not found after %d comparisons.\n", comparisons);
    return -1;
}
12
Update Element PrioritySearch and change priority value — O(n)

✏ Theory

Updating a priority requires first finding the element (linear search O(n)), then changing its priority field in-place. The array remains unsorted — no reshuffling needed.

  1. Search for element with target data value
  2. If found, overwrite priority with new value
  3. Queue structure maintained (still unsorted)

💡 Use Case

  • Task escalation — a task becomes more urgent
  • Dynamic scheduling — OS boosting starvation-prone processes
  • Network QoS — upgrading a packet's service class mid-queue

In-place update is an advantage of the unsorted array — no need to re-insert or re-sort.

🏷
Analogy — Changing a Ticket Priority

A support ticket logged as "minor" gets escalated to "critical" by a manager. The ticket stays where it is in the list — only its priority label changes. Next time the team picks the highest priority ticket, this one might now be chosen.

C — Update Priority 
/* ── Update: Change priority of an element ── */
int updatePriority(PriorityQueue *pq, int target, int newPriority) {
    int i;

    /* Step 1: Search for the element */
    for (i = 0; i < pq->count; i++) {
        if (pq->arr[i].data == target) {

            /* Step 2: Update priority in-place */
            int oldPri = pq->arr[i].priority;
            pq->arr[i].priority = newPriority;

            printf("Updated data=%d: priority %d → %d\n",
                   target, oldPri, newPriority);
            return 1;   /* Success */
        }
    }

    printf("Element %d not found.\n", target);
    return 0;   /* Not found */
}
13
Count / Size OperationGet the number of elements currently in the queue — O(1)

🔢 Theory

Since we maintain a count variable at all times, getting the size is an O(1) operation — just return count.

  • count is incremented on every successful insert
  • count is decremented on every successful delete
  • We never need to scan the array to count elements

📈 Dynamic Tracking

The count variable is our single source of truth for queue size. It enables:

  • isEmpty() — is count == 0?
  • isFull() — is count == MAX_SIZE?
  • getSize() — return count directly
  • Loop bounds in all traversals
🧮
Analogy — Counting Visitors with a Clicker

A bouncer holds a click counter — he clicks it up when someone enters and down when someone leaves. Asking "how many people are inside?" is instant — he just reads the counter. No need to count heads. That's O(1) size tracking.

C — Count / Size 
/* ── getSize: Return number of elements in O(1) ── */
int getSize(PriorityQueue *pq) {
    return pq->count;   /* O(1) — no scanning needed */
}

/* ── Detailed size report ── */
void sizeReport(PriorityQueue *pq) {
    printf("Size: %d / %d (%.0f%% full)\n",
           pq->count,
           MAX_SIZE,
           (pq->count * 100.0) / MAX_SIZE);
}

/* Usage */

    initQueue(&pq);
    enqueue(&pq, 10, 1);

}
14
Clear / Destroy Priority QueueRemove all elements and reset the queue — O(1) or O(n)

🗑 Theory

Clearing the queue means making it appear empty. For a static array implementation, we just reset count = 0. The old data remains in memory but becomes inaccessible (overwritten on next insert).

  • Soft clear: set count = 0 — O(1)
  • Hard clear: zero out all array cells — O(n)
  • For malloc-based queues: call free() to release memory

⚠ Memory Leak Warning

If your queue elements contain pointers to dynamically allocated data (e.g., char *name), you must free() each element's data before clearing — otherwise you create a memory leak.

⚠ Always free dynamically allocated members before clearing!
🧹
Analogy — Erasing a Whiteboard

Soft clear is like erasing the number count on a scoreboard — the old scores are invisible but the board still exists. Hard clear is wiping every cell clean. Freeing memory is like returning the whiteboard itself to the stockroom.

Interactive — Clear Queue

/* ── Soft Clear: Reset count — O(1) ── */
void clearQueue(PriorityQueue *pq) {
    pq->count = 0;   /* All elements become "invisible" */
    printf("Queue cleared. Count reset to 0.\n");
}

/* ── Hard Clear: Zero out all cells — O(n) ── */
void hardClear(PriorityQueue *pq) {
    int i;
    for (i = 0; i < MAX_SIZE; i++) {
        pq->arr[i].data     = 0;
        pq->arr[i].priority = 0;
    }
    pq->count = 0;
    printf("Queue fully zeroed out.\n");
}

/* ── For dynamic (malloc) queues ── */
void destroyQueue(PriorityQueue *pq) {
    /* If arr was malloc'd: free(pq->arr); */
    pq->count = 0;
    /* If pq itself was malloc'd: free(pq); */
    printf("Queue destroyed.\n");
}
15
Sort Priority QueueAscending and descending sort by priority — O(n²) bubble sort

↕ Theory

Sorting rearranges the internal array by priority value. We use Bubble Sort for clarity:

  • Ascending: priority 1, 2, 3… (highest urgency first)
  • Descending: priority 5, 4, 3… (lowest urgency first)
  • After sorting, the queue looks like a sorted array priority queue
  • O(n²) time — acceptable for learning

⚡ Bubble Sort Review

Bubble sort compares adjacent pairs and swaps them if they're in the wrong order. After each full pass, the largest (or smallest) element "bubbles" to the end.


Best CaseO(n)
AverageO(n²)
Worst CaseO(n²)
📊
Analogy — Sorting a Hand of Cards

You have a hand of shuffled playing cards (unsorted queue). To sort them ascending, you compare each adjacent pair and swap if needed, repeatedly until sorted — exactly like bubble sort. This makes future "pick the best card" instant.

/* ── Sort Ascending by priority (1, 2, 3...) ── */
void sortAscending(PriorityQueue *pq) {
    int i, j;
    Element temp;

    for (i = 0; i < pq->count - 1; i++) {
        for (j = 0; j < pq->count - i - 1; j++) {
            /* Swap if current priority is greater than next */
            if (pq->arr[j].priority > pq->arr[j+1].priority) {
                temp            = pq->arr[j];
                pq->arr[j]     = pq->arr[j+1];
                pq->arr[j+1] = temp;
            }
        }
    }
    printf("Sorted ascending by priority.\n");
}

/* ── Sort Descending by priority (5, 4, 3...) ── */
void sortDescending(PriorityQueue *pq) {
    int i, j;
    Element temp;

    for (i = 0; i < pq->count - 1; i++) {
        for (j = 0; j < pq->count - i - 1; j++) {
            /* Swap if current priority is LESS than next */
            if (pq->arr[j].priority < pq->arr[j+1].priority) {
                temp            = pq->arr[j];
                pq->arr[j]     = pq->arr[j+1];
                pq->arr[j+1] = temp;
            }
        }
    }
    printf("Sorted descending by priority.\n");
}

Merge / Concatenate Priority QueuesCombine two queues into one — O(n+m)

🔗 Theory

Merging combines elements from Queue A and Queue B into a new Queue C. Steps:

  1. Check if combined size exceeds MAX_SIZE
  2. Copy all elements from Queue A into Queue C
  3. Append all elements from Queue B into Queue C
  4. Optionally sort the merged queue
  5. Result: one unified, unsorted array queue

⏱ Complexity


Copy Queue A (n elements)O(n)
Append Queue B (m elements)O(m)
Optional re-sortO((n+m)²)
Total (no sort)O(n+m)
🏥

Two departments merge their patient waiting lists into one combined ER queue. Every patient from both lists gets added (O(n+m)), then the triage team might re-sort the combined list to know who to see first.

/* ── Merge: Combine two queues into a third ── */
int mergeQueues(PriorityQueue *a, PriorityQueue *b,
                PriorityQueue *result) {
    int i;

    /* Step 1: Overflow check */
    if (a->count + b->count > MAX_SIZE) {
        printf("Merge would exceed capacity!\n");
        return 0;
    }

    /* Step 2: Initialize result */
    initQueue(result);

    /* Step 3: Copy all elements from Queue A */
    for (i = 0; i < a->count; i++) {
        result->arr[result->count++] = a->arr[i];
    }

    /* Step 4: Append all elements from Queue B */
    for (i = 0; i < b->count; i++) {
        result->arr[result->count++] = b->arr[i];
    }

    printf("Merged! New size: %d\n", result->count);

    /* Step 5: Optionally sort the result */
    sortAscending(result);   /* Re-sort merged queue */
    return 1;
}
17
Copy Priority QueueDeep copy all elements to a new queue — O(n)

📋 Theory

Copying creates an independent duplicate of the queue. With primitive data types (int), a simple element-by-element copy is a deep copy — changes to the copy don't affect the original.

  • Copy each element's data and priority
  • Copy count to the new queue
  • Both queues are now fully independent

⚠ Shallow vs Deep Copy


TypeWhat it copiesIndependent?
Deep CopyAll values recursively✔ Yes
Shallow CopyPointer references only✘ No

For int data, element copy is always deep. For pointer data, you'd need to copy what the pointer points to.

📄
Analogy — Photocopying Documents

Deep copying a queue is like photocopying a stack of documents. You get an identical set of papers — editing the copy doesn't change the originals. Shallow copy would be like handing out the same physical papers to multiple people; a change affects everyone.

/* ── Deep Copy: Duplicate queue independently ── */
void copyQueue(PriorityQueue *src, PriorityQueue *dst) {
    int i;

    /* Copy each element (data + priority) */
    for (i = 0; i < src->count; i++) {
        dst->arr[i].data     = src->arr[i].data;
        dst->arr[i].priority = src->arr[i].priority;
    }

    /* Copy the count */
    dst->count = src->count;

    printf("Copied %d elements. Queues are now independent.\n",
           src->count);
}

/* Verify independence */
int main() {
    PriorityQueue original, copy;
    initQueue(&original);
    enqueue(&original, 10, 1);
    enqueue(&original, 20, 3);

    copyQueue(&original, &copy);

    /* Modifying copy won't affect original */
    copy.arr[0].priority = 99;
    printf("Original[0] priority: %d\n", original.arr[0].priority); /* Still 1 */
    return 0;
}

Reverse Priority QueueReverse the array order using two-pointer swap — O(n)

🔄 Theory

Reversing swaps elements from both ends working toward the middle using the two-pointer technique.

  1. Set left = 0, right = count - 1
  2. Swap arr[left] with arr[right]
  3. Move left forward, right backward
  4. Repeat until left >= right

⏱ Complexity


TimeO(n/2) = O(n)
SpaceO(1) extra
Swaps performed⌊n/2⌋

In-place reversal — no extra array needed. Only one temporary variable for swapping.

🪞
Analogy — Reversing a Line of People

Imagine a line of people holding numbered cards. You ask the first and last person to swap places, then the second and second-to-last, and so on. After n/2 swaps, the entire line is reversed. That's the two-pointer reversal technique.

/* ── Reverse: Swap elements from both ends inward ── */
void reverseQueue(PriorityQueue *pq) {
    int left, right;
    Element temp;

    if (pq->count <= 1) {
        printf("Nothing to reverse.\n");
        return;
    }

    left  = 0;
    right = pq->count - 1;

    while (left < right) {
        /* Swap elements at left and right pointers */
        temp           = pq->arr[left];
        pq->arr[left]  = pq->arr[right];
        pq->arr[right] = temp;

        left++;    /* Move left pointer right */
        right--;   /* Move right pointer left */
    }

    printf("Queue reversed successfully.\n");
}

/* Usage */
int main() {
    PriorityQueue pq;
    initQueue(&pq);
    enqueue(&pq, 10, 1);
    enqueue(&pq, 20, 3);
    enqueue(&pq, 30, 2);
    /* Before: [10,p1] [20,p3] [30,p2] */
    reverseQueue(&pq);
    /* After:  [30,p2] [20,p3] [10,p1] */
    return 0;
}
