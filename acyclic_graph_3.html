Chapter 1 · Foundations

What is an Acyclic Graph?

Definition

A graph is a collection of vertices (nodes) connected by edges (lines). An acyclic graph is one where you can never start at a vertex, follow edges, and return to the same vertex. There are no loops!

Think of it as a one-way road network with no roundabouts — once you leave a city, you can never come back to it by following the roads.

Types of Acyclic Graphs
  • Tree — Connected acyclic graph
  • Forest — Disconnected acyclic graph
  • DAG — Directed Acyclic Graph (edges have direction)
Key Properties
  • No cycle exists in the graph
  • If directed: no directed cycle
  • A tree with N nodes has exactly N-1 edges
  • Supports topological ordering

Chapter 1.2 · Representation

Incidence Matrix

What is an Incidence Matrix?

An Incidence Matrix is a 2D array (table) of size V × E where V = number of vertices, E = number of edges.

Each cell M[v][e] tells us the relationship between vertex v and edge e:

  • +1 — Vertex is the tail (source) of the edge (edge goes OUT)
  • -1 — Vertex is the head (destination) of the edge (edge comes IN)
  • 0 — Vertex is not connected to this edge
📬 Analogy: Post Office Ledger

Imagine a ledger where rows are cities and columns are delivery routes. +1 means "a package leaves this city on this route". -1 means "a package arrives at this city on this route". 0 means "this route doesn't touch this city".

C — Incidence Matrix Structure 
#include <stdio.h>
#include <stdlib.h>
#include <string.h>

#define MAX_V 10   /* max vertices */
#define MAX_E 20   /* max edges    */

/* Incidence Matrix structure */
typedef struct {
    int mat[MAX_V][MAX_E]; /* +1 tail, -1 head, 0 none */
    int numV;              /* number of vertices */
    int numE;              /* number of edges    */
} Graph;

/* Initialize empty graph */
void initGraph(Graph *g) {
    g->numV = 0;
    g->numE = 0;
    memset(g->mat, 0, sizeof(g->mat));
}

int main() {
    Graph g;
    initGraph(&g);
    printf("Empty graph initialized.\n");
    return 0;
}

Vertex Operations · 1

Insert Vertex

Theory

Adding a vertex means adding a new row to the incidence matrix. The new row is all zeros (no edges connected yet). We also increment the vertex counter.

Time Complexity: O(E) — we must zero out the new row across all edge columns.

🏙️ Analogy

Like adding a new city to a map. The city exists but has no roads to/from it yet — its row in the ledger is blank.

// Insert Vertex Demo
Start with 0 vertices. Click "Add Vertex" to insert.
C — insertVertex() 
/* Add a new vertex to the graph */
int insertVertex(Graph *g) {
    if (g->numV >= MAX_V) {
        printf("Error: max vertices reached!\n");
        return -1;
    }
    int v = g->numV;       /* new vertex id = current count */
    for (int e = 0; e < g->numE; e++) {
        g->mat[v][e] = 0;  /* new row: all zeros */
    }
    g->numV++;             /* increment vertex count */
    printf("Vertex %d inserted.\n", v);
    return v;
}

Vertex Operations · 2

Delete Vertex

Theory

Deleting vertex v means: first remove all edges incident to v, then shift all rows above v down by one position. Update the vertex count.

Time Complexity: O(V × E) due to matrix row shifting.

🗺️ Analogy

Removing a city from the map. First cancel all delivery routes to/from that city, then renumber all remaining cities.

// Delete Vertex Demo
Graph initialized with 4 vertices and 3 edges.
C — deleteVertex() 
/* Delete vertex v from graph */
void deleteVertex(Graph *g, int v) {
    if (v < 0 || v >= g->numV) {
        printf("Vertex %d not found!\n", v);
        return;
    }
    /* Step 1: Remove all edges incident to v */
    int e = 0;
    while (e < g->numE) {
        if (g->mat[v][e] != 0) {
            deleteEdge(g, e); /* remove this edge */
        } else { e++; }
    }
    /* Step 2: Shift rows down */
    for (int i = v; i < g->numV - 1; i++) {
        for (int j = 0; j < g->numE; j++) {
            g->mat[i][j] = g->mat[i+1][j];
        }
    }
    g->numV--;
    printf("Vertex %d deleted.\n", v);
}

Edge Operations · 1

Insert Edge

Theory

Adding a directed edge from vertex u to vertex v adds a new column to the matrix. In that column: set mat[u][newEdge] = +1 (source) and mat[v][newEdge] = -1 (destination). All others stay 0.

Before inserting, we must verify the edge won't create a cycle (cycle detection).

Time Complexity: O(V) to set up the new column.

🛣️ Analogy

Building a one-way road from City A to City B. In our ledger we add a new column for this road, mark +1 for City A (starting point) and -1 for City B (ending point).

// Insert Edge Demo
Graph has 4 vertices. Add directed edges.
C — insertEdge() 
/* Insert directed edge from u to v */
int insertEdge(Graph *g, int u, int v) {
    if (u < 0 || u >= g->numV || v < 0 || v >= g->numV) {
        printf("Invalid vertices!\n");
        return -1;
    }
    if (g->numE >= MAX_E) {
        printf("Max edges reached!\n");
        return -1;
    }
    /* (In real DAG: check for cycles first!) */
    int e = g->numE;        /* new edge index */
    for (int i = 0; i < g->numV; i++)
        g->mat[i][e] = 0;   /* zero the new column */
    g->mat[u][e] = +1;    /* source: +1 */
    g->mat[v][e] = -1;    /* dest:   -1 */
    g->numE++;
    printf("Edge e%d: %d -> %d inserted.\n", e, u, v);
    return e;
}

Delete Edge

Theory

Deleting edge e means removing the column e from the matrix. We shift all columns to the right of e one position left. Then decrement edge count.

Time Complexity: O(V × E) due to column shifting.

// Delete Edge Demo
Graph initialized with 4 vertices and 4 edges.
C — deleteEdge() 
/* Delete edge at index e */
void deleteEdge(Graph *g, int e) {
    if (e < 0 || e >= g->numE) {
        printf("Edge e%d not found!\n", e);
        return;
    }
    /* Shift columns left starting at e */
    for (int j = e; j < g->numE - 1; j++) {
        for (int i = 0; i < g->numV; i++) {
            g->mat[i][j] = g->mat[i][j+1];
        }
    }
    g->numE--;
    printf("Edge e%d deleted.\n", e);
}

Edge Operations · 3

Search Edge

Theory

To find if edge (u→v) exists, scan all columns. For each column e, check if mat[u][e] == +1 AND mat[v][e] == -1. If both true, the edge exists.

Time Complexity: O(E) — scan all edge columns.

// Search Edge Demo
Edges: 0→1, 1→2, 2→3, 0→3. Search for any edge.
C — searchEdge() 
/* Search for directed edge u->v, returns edge index or -1 */
int searchEdge(Graph *g, int u, int v) {
    for (int e = 0; e < g->numE; e++) {
        if (g->mat[u][e] == +1 && g->mat[v][e] == -1) {
            printf("Edge %d->%d found at index e%d\n", u, v, e);
            return e;
        }
    }
    printf("Edge %d->%d NOT found.\n", u, v);
    return -1;
}

Edge Operations · 4

Update Edge

Theory

Updating an edge means changing its endpoints. This is equivalent to: (1) delete the old edge column, (2) insert a new edge column with the updated source/destination vertices.

We can also store edge weights in a separate array weight[MAX_E] and update those.

C — updateEdge() 
/* Update edge e to connect newU -> newV */
void updateEdge(Graph *g, int e, int newU, int newV) {
    if (e < 0 || e >= g->numE) {
        printf("Edge e%d not found!\n", e);
        return;
    }
    /* Zero out the column first */
    for (int i = 0; i < g->numV; i++)
        g->mat[i][e] = 0;
    /* Set new endpoints */
    g->mat[newU][e] = +1;
    g->mat[newV][e] = -1;
    printf("Edge e%d updated to %d -> %d\n", e, newU, newV);
}

/* For weighted graphs: */
typedef struct {
    int mat[MAX_V][MAX_E];
    int weight[MAX_E];     /* edge weights */
    int numV, numE;
} WGraph;

void updateWeight(WGraph *g, int e, int w) {
    if (e < 0 || e >= g->numE) return;
    g->weight[e] = w;
    printf("Edge e%d weight updated to %d\n", e, w);
}

Traversal Operations · 1

Breadth First Search (BFS)

Theory

BFS visits all vertices level by level — first the starting vertex, then its neighbors, then their neighbors, and so on. It uses a Queue (first-in, first-out).

Steps: Initialize queue → Visit start node → Mark visited → Add neighbors to queue → Repeat until queue empty.

Time Complexity: O(V × E) with incidence matrix.

🪨 Analogy: Ripples in Water

Drop a stone in a pond. The ripples expand outward one ring at a time — that's BFS. All nodes at distance 1, then distance 2, then distance 3...

// BFS Interactive Visualization

VISITED ORDER:

Set start vertex and click Run BFS.
C — BFS with Incidence Matrix 
#define MAX_Q 100

/* Get neighbors of vertex v using incidence matrix */
/* For directed graph: look for columns where mat[v][e]=+1 */
void bfs(Graph *g, int start) {
    int visited[MAX_V] = {0};
    int queue[MAX_Q];
    int front = 0, back = 0;

    /* Step 1: Queue initialization + visit start */
    visited[start] = 1;
    queue[back++] = start;
    printf("BFS order: ");

    while (front < back) {
        int v = queue[front++];  /* dequeue */
        printf("%d ", v);

        /* Traverse adjacent nodes via incidence matrix */
        for (int e = 0; e < g->numE; e++) {
            if (g->mat[v][e] == +1) {  /* v is source of edge e */
                for (int u = 0; u < g->numV; u++) {
                    if (g->mat[u][e] == -1 && !visited[u]) {
                        visited[u] = 1;
                        queue[back++] = u; /* enqueue */
                    }
                }
            }
        }
    }
    printf("\n");
}

Traversal Operations · 2

Depth First Search (DFS)

Theory

DFS dives as deep as possible along each branch before backtracking. It uses a Stack (or recursion). Two flavors: Recursive DFS (uses call stack) and Iterative DFS (uses explicit stack).

Backtracking: When no unvisited neighbors exist, we go back to the previous vertex.

🗺️ Analogy: Maze Explorer

In a maze, DFS means: always take the first unexplored passage. When stuck at a dead end, backtrack to the last junction and try the next passage.

// DFS Interactive Visualization

DFS VISIT ORDER:

Set start vertex and click Run DFS.
C — Recursive DFS & Iterative DFS 
int visited[MAX_V];

/* RECURSIVE DFS */
void dfsRecursive(Graph *g, int v) {
    visited[v] = 1;
    printf("%d ", v);
    for (int e = 0; e < g->numE; e++) {
        if (g->mat[v][e] == +1) {        /* v is source */
            for (int u = 0; u < g->numV; u++) {
                if (g->mat[u][e] == -1 && !visited[u])
                    dfsRecursive(g, u); /* recurse */
            }
        }
    }
}

/* ITERATIVE DFS (Stack-based) */
void dfsStack(Graph *g, int start) {
    int stack[MAX_V], top = -1;
    int vis[MAX_V] = {0};
    stack[++top] = start;
    while (top >= 0) {
        int v = stack[top--];   /* pop */
        if (vis[v]) continue;
        vis[v] = 1;
        printf("%d ", v);
        for (int e = 0; e < g->numE; e++) {
            if (g->mat[v][e] == +1) {
                for (int u = 0; u < g->numV; u++) {
                    if (g->mat[u][e] == -1 && !vis[u])
                        stack[++top] = u; /* push */
                }
            }
        }
    }
}

Path Operations · 1

Find Path & All Paths

Find Path (Source to Destination)

Use DFS: start at source, try each neighbor. If we reach destination, we found a path. Track the path using a parent array or recursive path array.

All Paths

A variant of DFS that doesn't stop at first path found — it backtracks and finds every possible route from source to destination.

// Find Path Demo
Graph: 0→1→2→4, 0→3→4, 0→1→3→4
C — findPath() & allPaths() 
/* Find ONE path from src to dst using DFS */
int path[MAX_V], pathLen;

int findPath(Graph *g, int v, int dst, int *vis) {
    vis[v] = 1;
    path[pathLen++] = v;
    if (v == dst) return 1;  /* found! */
    for (int e = 0; e < g->numE; e++) {
        if (g->mat[v][e] == +1) {
            for (int u = 0; u < g->numV; u++) {
                if (g->mat[u][e] == -1 && !vis[u])
                    if (findPath(g, u, dst, vis))
                        return 1;
            }
        }
    }
    pathLen--;  /* backtrack */
    return 0;
}

/* Find ALL paths from src to dst */
void allPaths(Graph *g, int v, int dst,
              int *vis, int *path, int pLen) {
    vis[v] = 1;
    path[pLen++] = v;
    if (v == dst) {
        for (int i = 0; i < pLen; i++)
            printf("%d%s", path[i], i+1<pLen ? "->" : "\n");
    } else {
        for (int e = 0; e < g->numE; e++) {
            if (g->mat[v][e] == +1) {
                for (int u = 0; u < g->numV; u++) {
                    if (g->mat[u][e] == -1 && !vis[u])
                        allPaths(g, u, dst, vis, path, pLen);
                }
            }
        }
    }
    vis[v] = 0;  /* unmark for other paths */
}

Path Operations · 2

Shortest Path

Unweighted Shortest Path (BFS-based)

For unweighted DAGs, BFS automatically finds the shortest path (fewest edges). The first time BFS reaches the destination, it has found the shortest route.

Use a parent[] array to reconstruct the path backward from destination to source.

🚇 Analogy: Metro Route

Finding the minimum number of metro stops between two stations. BFS explores all stations reachable in 1 stop, then 2 stops, etc. The first time you reach your destination = shortest path.

// Shortest Path BFS Demo
Click Find Shortest Path to visualize BFS.
C — shortestPath() using BFS 
/* Shortest path (unweighted) using BFS + parent array */
void shortestPath(Graph *g, int src, int dst) {
    int visited[MAX_V] = {0};
    int parent[MAX_V];
    int dist[MAX_V];
    int queue[MAX_Q];
    int front = 0, back = 0;

    for (int i = 0; i < g->numV; i++) { parent[i] = -1; dist[i] = -1; }
    visited[src] = 1;
    dist[src] = 0;
    queue[back++] = src;

    while (front < back) {
        int v = queue[front++];
        if (v == dst) break;
        for (int e = 0; e < g->numE; e++) {
            if (g->mat[v][e] == +1) {
                for (int u = 0; u < g->numV; u++) {
                    if (g->mat[u][e] == -1 && !visited[u]) {
                        visited[u] = 1;
                        parent[u] = v;
                        dist[u] = dist[v] + 1;
                        queue[back++] = u;
                    }
                }
            }
        }
    }
    /* Reconstruct path backward */
    if (dist[dst] == -1) { printf("No path!\n"); return; }
    printf("Shortest path length: %d\n", dist[dst]);
}

Reachability Check & Path Length

Reachability Check

Run DFS or BFS from vertex u. If vertex v gets visited, then v is reachable from u.

Path Length Count

The number of edges in a path. For BFS, the dist[] array gives us path length. For any path, count edges traversed.

// Reachability Check
Enter two vertices to check if one can reach the other.
C — isReachable() & pathLength() 
/* Check if dst is reachable from src */
int isReachable(Graph *g, int src, int dst) {
    int vis[MAX_V] = {0};
    int stack[MAX_V], top = -1;
    stack[++top] = src;
    while (top >= 0) {
        int v = stack[top--];
        if (v == dst) return 1;
        if (vis[v]) continue;
        vis[v] = 1;
        for (int e = 0; e < g->numE; e++) {
            if (g->mat[v][e] == +1) {
                for (int u = 0; u < g->numV; u++) {
                    if (g->mat[u][e]==-1 && !vis[u])
                        stack[++top] = u;
                }
            }
        }
    }
    return 0;  /* not reachable */
}

/* Count path length = number of edges in BFS shortest path */
int pathLength(Graph *g, int src, int dst) {
    /* Same as shortestPath dist[dst] */
    /* Returns -1 if not reachable */
    int dist[MAX_V]; /* filled by BFS */
    /* ... BFS ... */
    return dist[dst];
}

Acyclic Operations · 1

Detect Cycle Absence & Edge Safety Check

Detect Cycle Absence (DFS Color Method)

Use 3-color DFS: WHITE (unvisited) → GRAY (in current path) → BLACK (done). If we encounter a GRAY node during DFS, we found a back edge → cycle exists!

If no GRAY node is ever encountered, the graph is acyclic.

Edge Safety Check

Before inserting a new edge (u→v), check if v can reach u via existing edges. If yes, adding u→v would create a cycle — unsafe!

🚦 Analogy

Like a traffic system ensuring no route forms a roundabout. Before building a new road from A to B, check if B already has a road back to A — that would create a loop!

// Cycle Detection Demo
Graph is a DAG: 0→1→2→3. Click to check.
C — hasCycle() with DFS 3-Color 
#define WHITE 0
#define GRAY  1
#define BLACK 2

int color[MAX_V];

int dfsHasCycle(Graph *g, int v) {
    color[v] = GRAY;  /* mark: currently visiting */
    for (int e = 0; e < g->numE; e++) {
        if (g->mat[v][e] == +1) {
            for (int u = 0; u < g->numV; u++) {
                if (g->mat[u][e] == -1) {
                    if (color[u] == GRAY)
                        return 1; /* back edge = cycle! */
                    if (color[u] == WHITE)
                        if (dfsHasCycle(g, u)) return 1;
                }
            }
        }
    }
    color[v] = BLACK;  /* fully processed */
    return 0;
}

int detectCycleAbsence(Graph *g) {
    for (int i = 0; i < g->numV; i++) color[i] = WHITE;
    for (int v = 0; v < g->numV; v++) {
        if (color[v] == WHITE)
            if (dfsHasCycle(g, v)) {
                printf("Cycle detected!\n");
                return 0;  /* not acyclic */
            }
    }
    printf("Graph is acyclic!\n");
    return 1;
}

Acyclic Operations · 2

Tree & Forest Check

Tree Structure

A tree is a connected acyclic undirected graph. For N vertices, a tree has exactly N-1 edges. All vertices are reachable from any other vertex.

Forest Structure

A forest is a collection of trees — acyclic but possibly disconnected. If the graph has C connected components and N-C edges, it's a forest.

🌳 Analogy

A tree is one big family where everyone can trace their lineage to a common root. A forest is multiple separate family trees that don't share ancestry.

C — isTree() & isForest() 
/* Count connected components using DFS */
int countComponents(Graph *g) {
    int vis[MAX_V] = {0};
    int comp = 0;
    for (int v = 0; v < g->numV; v++) {
        if (!vis[v]) {
            dfsRecursive(g, v);  /* uses global visited */
            comp++;
        }
    }
    return comp;
}

/* A tree: connected + acyclic + numE == numV-1 */
int isTree(Graph *g) {
    if (hasCycle(g)) return 0;
    if (g->numE != g->numV - 1) return 0;
    if (countComponents(g) != 1) return 0;
    return 1;
}

/* A forest: acyclic (possibly disconnected) */
int isForest(Graph *g) {
    if (hasCycle(g)) return 0;
    /* numE == numV - number_of_components */
    int c = countComponents(g);
    return (g->numE == g->numV - c) ? 1 : 0;
}

DAG Validation

What makes a valid DAG?

A Directed Acyclic Graph (DAG) must satisfy:

  • All edges have a direction (directed)
  • No directed cycles exist (acyclic)
  • A topological ordering must be possible

The simplest test: try to perform topological sort (Kahn's algorithm). If all vertices get ordered, it's a valid DAG.

// DAG Validation Demo
Click a button to validate.
C — validateDAG() 
/* Validate that graph is a DAG using Kahn's approach */
int validateDAG(Graph *g) {
    /* Compute in-degrees */
    int inDeg[MAX_V] = {0};
    for (int e = 0; e < g->numE; e++) {
        for (int v = 0; v < g->numV; v++) {
            if (g->mat[v][e] == -1) inDeg[v]++;
        }
    }
    /* Queue zero in-degree vertices */
    int queue[MAX_Q], front=0, back=0, cnt=0;
    for (int v=0; v<g->numV; v++)
        if (inDeg[v]==0) queue[back++]=v;
    while (front < back) {
        int v = queue[front++]; cnt++;
        for (int e=0; e<g->numE; e++) {
            if (g->mat[v][e]==+1) {
                for (int u=0; u<g->numV; u++) {
                    if (g->mat[u][e]==-1)
                        if (--inDeg[u]==0) queue[back++]=u;
                }
            }
        }
    }
    return (cnt == g->numV) ? 1 : 0; /* all processed = DAG */
}

Topological Operations · 1

Topological Sort & Ordering Vertices

What is Topological Sort?

Topological sort orders vertices so that for every directed edge (u→v), u comes before v in the ordering. It's only possible on DAGs.

Two main methods: DFS-based (push to stack after processing) and Kahn's Algorithm (use in-degrees).

📚 Analogy: Course Prerequisites

Topological sort gives you a valid order to complete courses where some courses depend on others. You'll never be asked to take "Advanced Algorithms" before "Programming Basics".

// Topological Sort Demo

TOPOLOGICAL ORDER:

DAG: 5→2, 5→0, 4→0, 4→1, 2→3, 3→1
C — DFS Topological Sort 
int topoStack[MAX_V], sTop;

void topoHelper(Graph *g, int v, int *vis) {
    vis[v] = 1;
    for (int e = 0; e < g->numE; e++) {
        if (g->mat[v][e] == +1) {
            for (int u = 0; u < g->numV; u++) {
                if (g->mat[u][e] == -1 && !vis[u])
                    topoHelper(g, u, vis);
            }
        }
    }
    topoStack[sTop++] = v;  /* push AFTER processing */
}

void topologicalSort(Graph *g) {
    int vis[MAX_V] = {0};
    sTop = 0;
    for (int v = 0; v < g->numV; v++)
        if (!vis[v]) topoHelper(g, v, vis);
    printf("Topological order: ");
    for (int i = sTop-1; i >= 0; i--)  /* print stack reverse */
        printf("%d ", topoStack[i]);
    printf("\n");
}

void taskSchedule(Graph *g) {

        if (inDeg[v]==0) q[b++]=v;

}

Kahn's Algorithm

How Kahn's Algorithm Works

Kahn's is a BFS-based topological sort. The key insight: in a DAG, at least one vertex has in-degree 0 (no prerequisites). We repeatedly remove these vertices and update in-degrees.

Compute in-degrees Count -1 entries per row in incidence matrix.
Enqueue zeros Add all vertices with in-degree 0 to queue.
Process queue Dequeue vertex, add to result, reduce neighbors' in-degrees.
Repeat When a neighbor reaches in-degree 0, enqueue it.
Validate If result has all V vertices → valid DAG. Otherwise → cycle exists.
// Kahn's Algorithm Step-by-Step

QUEUE:

RESULT ORDER:

DAG: 5→2, 5→0, 4→0, 4→1, 2→3, 3→1
C — Kahn's Algorithm 
void kahnsAlgorithm(Graph *g) {
    int inDeg[MAX_V] = {0};
    int result[MAX_V], rCnt = 0;
    int queue[MAX_Q], f = 0, b = 0;

    /* Step 1: Compute in-degrees */
    for (int e = 0; e < g->numE; e++)
        for (int v = 0; v < g->numV; v++)
            if (g->mat[v][e] == -1) inDeg[v]++;

    /* Step 2: Enqueue all zero in-degree vertices */
    for (int v = 0; v < g->numV; v++)
        if (inDeg[v] == 0) queue[b++] = v;

    /* Step 3: Process */
    while (f < b) {
        int v = queue[f++];
        result[rCnt++] = v;
        for (int e = 0; e < g->numE; e++) {
            if (g->mat[v][e] == +1) {
                for (int u = 0; u < g->numV; u++) {
                    if (g->mat[u][e] == -1) {
                        inDeg[u]--;
                        if (inDeg[u] == 0) queue[b++] = u;
                    }
                }
            }
        }
    }
    /* Step 4: Validate */
    if (rCnt != g->numV) printf("Cycle detected!\n");
    else {
        printf("Topo order: ");
        for (int i=0; i<rCnt; i++) printf("%d ", result[i]);
    }
}

Longest Path in DAG

Algorithm

The longest path in a DAG can be found in O(V+E) using topological sort + dynamic programming. This is unique to DAGs — in graphs with cycles, longest path is NP-hard!

Idea: Process vertices in topological order. For each vertex, update the longest path to each neighbor: dist[neighbor] = max(dist[neighbor], dist[v] + weight)

🎯 Real Use: Critical Path Method (CPM)

In project management, the longest path through a task dependency DAG is the critical path — it determines the minimum project duration. Delay any task on this path and the whole project is delayed.

// Longest Path Visualization
Click to find the longest path through this DAG.
C — Longest Path in DAG 
/* Longest path from any source in DAG */
void longestPath(Graph *g) {
    int topoOrder[MAX_V], cnt = 0;
    kahnsGetOrder(g, topoOrder, &cnt); /* get topo order */

    int dist[MAX_V];
    for (int i = 0; i < g->numV; i++) dist[i] = 0;

    /* Process vertices in topological order */
    for (int i = 0; i < cnt; i++) {
        int v = topoOrder[i];
        for (int e = 0; e < g->numE; e++) {
            if (g->mat[v][e] == +1) {
                for (int u = 0; u < g->numV; u++) {
                    if (g->mat[u][e] == -1) {
                        if (dist[v] + 1 > dist[u])
                            dist[u] = dist[v] + 1; /* DP update */
                    }
                }
            }
        }
    }
    int maxLen = 0;
    for (int v = 0; v < g->numV; v++)
        if (dist[v] > maxLen) maxLen = dist[v];
    printf("Longest path length: %d\n", maxLen);
}

Dynamic Programming on DAG

Why DP works perfectly on DAGs

DAGs have no cycles, so if we process vertices in topological order, we guarantee all predecessors of a vertex are processed before the vertex itself. This is exactly the property DP needs — solve subproblems before the main problem!

Examples: longest path, shortest path, number of paths, minimum cost path.

Count Paths: DP Example

To count all paths from source S to all vertices:

  • paths[S] = 1 (one way to reach start)
  • For each vertex v in topo order, for each neighbor u: paths[u] += paths[v]
// DP: Count Paths from Source
Click to count paths from source vertex.
C — DP Count Paths on DAG 
/* Count number of paths from src to each vertex */
void countPaths(Graph *g, int src) {
    int topoOrder[MAX_V], cnt = 0;
    kahnsGetOrder(g, topoOrder, &cnt);

    int paths[MAX_V] = {0};
    paths[src] = 1;  /* one way to be at source */

    for (int i = 0; i < cnt; i++) {
        int v = topoOrder[i];
        if (paths[v] == 0) continue; /* unreachable */
        for (int e = 0; e < g->numE; e++) {
            if (g->mat[v][e] == +1) {
                for (int u = 0; u < g->numV; u++) {
                    if (g->mat[u][e] == -1) {
                        paths[u] += paths[v]; /* DP recurrence */
                    }
                }
            }
        }
    }
    for (int v = 0; v < g->numV; v++)
        printf("Paths to %d: %d\n", v, paths[v]);
}
