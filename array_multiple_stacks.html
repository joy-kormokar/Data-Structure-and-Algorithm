1. Push Operation

Adding an element to the top of a stack — the fundamental write operation

Theory

A Multiple Stack is a clever way to store two or more stacks inside a single array to save memory. The most common approach is the Two-Stack Method:

• Stack 1 starts at index 0 and grows rightward (toward higher indices). Its top pointer starts at -1.

• Stack 2 starts at the last index (MAX-1) and grows leftward (toward lower indices). Its top pointer starts at MAX.

Both stacks share the entire array, so unused space in either stack can be used by the other — highly efficient!

Push Rule: Before pushing, always check for overflow. Overflow happens when Stack 1's top pointer and Stack 2's top pointer are adjacent: top1 + 1 == top2

Beginner-Friendly C Code
push_operation.cC
#include <stdio.h>

#define MAX 10          /* Total array size */

int arr[MAX];           /* Single array for BOTH stacks */
int top1 = -1;          /* Stack 1: starts from LEFT, -1 means empty */
int top2 = MAX;         /* Stack 2: starts from RIGHT, MAX means empty */

/* ── Push in Stack 1 (grows left → right) ── */
void push1(int x) {
    /* Overflow check: are the two tops adjacent? */
    if (top1 + 1 == top2) {
        printf("OVERFLOW! No space left.\n");
        return;
    }
    top1++;             /* Move top1 one step right */
    arr[top1] = x;      /* Place element */
    printf("Pushed %d to Stack 1 (index %d)\n", x, top1);
}

/* ── Push in Stack 2 (grows right → left) ── */
void push2(int x) {
    /* Same overflow check applies */
    if (top1 + 1 == top2) {
        printf("OVERFLOW! No space left.\n");
        return;
    }
    top2--;             /* Move top2 one step left */
    arr[top2] = x;      /* Place element */
    printf("Pushed %d to Stack 2 (index %d)\n", x, top2);
}

/* ── Push in kth Stack (k stacks in one array) ── */
/* Uses a "next free slot" array — explained in Topic 9 */
void pushK(int arr[], int top[], int nxt[], int *freeTop,
           int stackNo, int x) {
    if (*freeTop == -1) {
        printf("OVERFLOW! No free slots.\n");
        return;
    }
    int i = *freeTop;       /* Get first free slot index */
    *freeTop = nxt[i];      /* Update free list */
    nxt[i] = top[stackNo]; /* Link to old top */
    top[stackNo] = i;       /* Update this stack's top */
    arr[i] = x;             /* Store the value */
    printf("Pushed %d to Stack %d\n", x, stackNo);
}

int main() {
    push1(10);   /* arr[0] = 10, top1 = 0 */
    push1(20);   /* arr[1] = 20, top1 = 1 */
    push2(90);   /* arr[9] = 90, top2 = 9 */
    push2(80);   /* arr[8] = 80, top2 = 8 */
    return 0;
}
Real-Life Analogy
🚇

Think of a double-ended subway car. Passengers boarding from the front door pack inward from the left. Passengers boarding from the back door pack inward from the right. Both groups share the same car. Passengers are pushed in one at a time — exactly like a push operation!

Use Case: Undo/Redo systems — Undo stack (Stack 1) and Redo stack (Stack 2) share memory so the combined history can be as large as possible.

2. Pop Operation

Removing and returning the top element — the fundamental delete operation

Theory

The Pop operation removes the topmost element from a stack and returns it. The top pointer is updated to point to the new top after removal.

Before popping, always check for underflow — trying to pop from an empty stack. For Stack 1, underflow occurs when top1 == -1. For Stack 2, when top2 == MAX.

Underflow: Attempting to pop from an empty stack causes undefined behaviour. Always validate before popping!

Beginner-Friendly C Code
pop_operation.cC
#include <stdio.h>

#define MAX 10
int arr[MAX];
int top1 = -1;
int top2 = MAX;

/* ── Pop from Stack 1 ── */
int pop1() {
    /* Underflow check for Stack 1 */
    if (top1 == -1) {
        printf("UNDERFLOW! Stack 1 is empty.\n");
        return -1;          /* Return sentinel value */
    }
    int val = arr[top1];    /* Save the top value */
    arr[top1] = 0;          /* Clear the slot (optional) */
    top1--;                 /* Move top1 one step left */
    printf("Popped %d from Stack 1\n", val);
    return val;
}

/* ── Pop from Stack 2 ── */
int pop2() {
    /* Underflow check for Stack 2 */
    if (top2 == MAX) {
        printf("UNDERFLOW! Stack 2 is empty.\n");
        return -1;
    }
    int val = arr[top2];    /* Save the top value */
    arr[top2] = 0;          /* Clear the slot (optional) */
    top2++;                 /* Move top2 one step right */
    printf("Popped %d from Stack 2\n", val);
    return val;
}

/* ── Pop from kth Stack ── */
int popK(int arr[], int top[], int nxt[], int *freeTop, int sn) {
    if (top[sn] == -1) {
        printf("UNDERFLOW! Stack %d is empty.\n", sn);
        return -1;
    }
    int i = top[sn];         /* Get index of current top */
    top[sn] = nxt[i];        /* Update stack top to next */
    nxt[i] = *freeTop;       /* Return slot to free list */
    *freeTop = i;
    return arr[i];
}

int main() {
    /* Assume stacks are pre-filled */
    pop1();     /* Removes top of Stack 1 */
    pop2();     /* Removes top of Stack 2 */
    pop1();     /* Underflow if Stack 1 empty */
    return 0;
}
Real-Life Analogy
🍽️

Imagine a cafeteria with two spring-loaded plate dispensers sharing the same cabinet space. You can only take the top plate from each dispenser. If a dispenser is empty and you try to take a plate — that's an underflow! In our array, both dispensers share the same shelf efficiently.

Use Case: Browser back/forward navigation — clicking Back pops from the history stack, clicking Forward pops from the future stack.

3. Peek / Top Operation

Reading the top element WITHOUT removing it from the stack

Theory

Peek (also called Top) lets you look at the element on top of the stack without removing it. The top pointer is not changed — it's a read-only view.

For Stack 1: simply return arr[top1]. For Stack 2: return arr[top2]. Always check for underflow (empty stack) before peeking.

Key Difference from Pop: Pop removes the element and changes the top pointer. Peek only reads — the stack stays exactly the same.

Beginner-Friendly C Code
peek_operation.cC
#include <stdio.h>

#define MAX 10
int arr[MAX];
int top1 = -1;
int top2 = MAX;

/* ── Peek Stack 1 ── */
int peek1() {
    if (top1 == -1) {
        printf("Stack 1 is empty! Nothing to peek.\n");
        return -1;
    }
    /* Just return the top value — no pointer change */
    printf("Stack 1 top element: %d (at index %d)\n",
           arr[top1], top1);
    return arr[top1];
}

/* ── Peek Stack 2 ── */
int peek2() {
    if (top2 == MAX) {
        printf("Stack 2 is empty! Nothing to peek.\n");
        return -1;
    }
    printf("Stack 2 top element: %d (at index %d)\n",
           arr[top2], top2);
    return arr[top2];
}

/* ── Peek kth Stack ── */
int peekK(int arr[], int top[], int stackNo) {
    if (top[stackNo] == -1) {
        printf("Stack %d is empty!\n", stackNo);
        return -1;
    }
    /* top[stackNo] holds the index of the top element */
    return arr[top[stackNo]];
}

int main() {
    /* Assume elements have been pushed */
    int t1 = peek1();   /* Reads top of Stack 1 */
    int t2 = peek2();   /* Reads top of Stack 2 */
    /* Stack remains UNCHANGED after peek */
    printf("After peek — top1=%d, top2=%d\n", top1, top2);
    return 0;
}
Real-Life Analogy
📬

Think of looking at the top envelope in a mail tray — you read the address without opening or removing it. The pile stays exactly as it was. That's exactly what Peek does: no modification, just a quick look at what's on top!

Use Case: Parsing expressions (e.g., checking if the top of an operator stack has a higher precedence before deciding whether to push or pop).

4. IsEmpty Operation

Checking whether a stack has any elements — a safety guard before pop/peek

Theory

IsEmpty checks if a stack contains no elements. This is done purely by comparing the top pointer to its initial value.

• Stack 1 is empty when top1 == -1 (it was initialized to -1)

• Stack 2 is empty when top2 == MAX (it was initialized to MAX)

• kth Stack is empty when top[k] == -1

Best Practice: Always call isEmpty before pop() or peek() to prevent underflow errors.

Beginner-Friendly C Code
isempty_operation.cC
#include <stdio.h>

#define MAX 10
int arr[MAX];
int top1 = -1;    /* -1 means Stack 1 is empty */
int top2 = MAX;   /* MAX means Stack 2 is empty */

/* ── Is Stack 1 Empty? ── */
int isEmpty1() {
    /* top1 == -1 means no elements in Stack 1 */
    return (top1 == -1);   /* Returns 1 (true) or 0 (false) */
}

/* ── Is Stack 2 Empty? ── */
int isEmpty2() {
    /* top2 == MAX means no elements in Stack 2 */
    return (top2 == MAX);
}

/* ── Is kth Stack Empty? ── */
int isEmptyK(int top[], int stackNo) {
    /* top[stackNo] == -1 means that stack has no elements */
    return (top[stackNo] == -1);
}

/* ── Safe usage example ── */
int safePop1() {
    if (isEmpty1()) {               /* Always check first! */
        printf("Cannot pop: Stack 1 is empty.\n");
        return -1;
    }
    int val = arr[top1--];
    return val;
}

int main() {
    printf("Stack 1 empty? %s\n", isEmpty1() ? "YES" : "NO");
    printf("Stack 2 empty? %s\n", isEmpty2() ? "YES" : "NO");
    /* Push something and recheck */
    arr[++top1] = 42;
    printf("After push: Stack 1 empty? %s\n",
           isEmpty1() ? "YES" : "NO");
    return 0;
}
Real-Life Analogy
🧺

Before reaching into a laundry basket, you glance inside to see if there's anything there. If it's empty, you don't bother. That's isEmpty() — a quick check so your program doesn't crash trying to retrieve something that isn't there.

Use Case: Evaluating postfix expressions — check isEmpty before popping operands to avoid runtime crashes.

5. IsFull Operation

Detecting when the shared array has no more space for new elements

Theory

IsFull checks whether the shared array is completely occupied. In the two-stack model, the array is full when Stack 1 and Stack 2 are about to collide — i.e., when there is no empty slot between them.

This happens when: top1 + 1 == top2

At this moment, pushing to either stack would overwrite data from the other — so we must stop and report overflow.

Important: In this model, neither stack has a fixed size limit. Stack 1 can use 9 slots while Stack 2 uses 1 slot, or any other combination — as long as the total ≤ MAX.

Beginner-Friendly C Code
isfull_operation.cC
#include <stdio.h>

#define MAX 10
int arr[MAX];
int top1 = -1;
int top2 = MAX;

/* ── Global Full Check ── */
int isFull() {
    /* When top1 and top2 are adjacent — no gap between stacks */
    return (top1 + 1 == top2);
}

/* ── Space Available Check ── */
int spaceAvailable() {
    /* Number of empty slots between the two tops */
    return top2 - top1 - 1;
}

/* ── Collision Detection (used inside push) ── */
void pushSafe(int stackNo, int value) {
    /* Detect collision BEFORE pushing */
    if (top1 + 1 == top2) {
        printf("OVERFLOW! Collision detected. "
               "top1=%d, top2=%d\n", top1, top2);
        return;  /* Stop — do not push */
    }
    if (stackNo == 1) {
        arr[++top1] = value;
        printf("Pushed %d to Stack 1\n", value);
    } else {
        arr[--top2] = value;
        printf("Pushed %d to Stack 2\n", value);
    }
}

int main() {
    printf("Full? %s | Space: %d\n",
           isFull() ? "YES" : "NO", spaceAvailable());
    pushSafe(1, 10);
    pushSafe(2, 90);
    printf("Full? %s | Space: %d\n",
           isFull() ? "YES" : "NO", spaceAvailable());
    return 0;
}
Real-Life Analogy
🅿️

Think of a parking lot with two entrances — cars enter from the left and right and park inward. When the cars from both sides nearly touch in the middle, the lot is full! isFull() detects this moment before another car tries to enter and causes a crash.

Use Case: Memory management systems — detecting when memory regions are about to overlap (heap growing up, stack growing down).

6. Size Operation

Counting how many elements are currently stored in each stack

Theory

The Size operation returns the number of elements currently stored in a given stack. It uses only the top pointer — no looping required, making it O(1).

Size of Stack 1 = top1 + 1 Size of Stack 2 = MAX − top2 Size of kth Stack = count of nodes in its linked list

For Stack 1: since top1 starts at -1 (empty), adding 1 gives the count of elements. For Stack 2: since it grows from the right, MAX - top2 gives the count.

Beginner-Friendly C Code
size_operation.cC
#include <stdio.h>

#define MAX 10
int arr[MAX];
int top1 = -1;
int top2 = MAX;

/* ── Size of Stack 1 ── */
int size1() {
    /* top1 starts at -1 (empty), so +1 gives element count */
    return top1 + 1;
}

/* ── Size of Stack 2 ── */
int size2() {
    /* Stack 2 grows from right; MAX - top2 = number of elements */
    return MAX - top2;
}

/* ── Size of kth Stack (using linked list next[] array) ── */
int sizeK(int top[], int nxt[], int stackNo) {
    int count = 0;
    int i = top[stackNo];     /* Start from this stack's top */
    while (i != -1) {         /* Traverse the linked list */
        count++;
        i = nxt[i];           /* Follow the next pointer */
    }
    return count;
}

int main() {
    /* Push some elements */
    arr[++top1] = 10;   /* Stack 1: size = 1 */
    arr[++top1] = 20;   /* Stack 1: size = 2 */
    arr[--top2] = 90;   /* Stack 2: size = 1 */

    printf("Size of Stack 1: %d\n", size1());   /* Output: 2 */
    printf("Size of Stack 2: %d\n", size2());   /* Output: 1 */
    printf("Total elements: %d / %d\n",
           size1() + size2(), MAX);
    return 0;
}
Real-Life Analogy
📦

Imagine two piles of boxes growing from opposite ends of a warehouse. To know how many boxes are in each pile, you just look at the height of the top box — no need to count every box from the bottom! That's size() — an O(1) count using just the top index.

Use Case: Memory allocators tracking how much of each memory region is used so they can report usage statistics to the OS.

7. Capacity Check

Calculating how much shared space is still available between the two stacks

Theory

The Capacity Check tells us how much empty space remains in the shared array — available for either stack to use. This is the "free zone" between top1 and top2.

Remaining Space = top2 − top1 − 1 Shared Space = MAX − (Size1 + Size2)

This shared model is efficient because unused space is never wasted. If one stack grows large and the other stays small, the large stack can borrow that unused space naturally.

Beginner-Friendly C Code
capacity_check.cC
#include <stdio.h>

#define MAX 10
int arr[MAX];
int top1 = -1;
int top2 = MAX;

/* ── Remaining Space (free slots between two tops) ── */
int remainingSpace() {
    /* The gap between top1 and top2 — shared free zone */
    return top2 - top1 - 1;
}

/* ── Shared Space Calculation ── */
void showCapacity() {
    int used1 = top1 + 1;          /* Elements in Stack 1 */
    int used2 = MAX - top2;        /* Elements in Stack 2 */
    int free   = remainingSpace(); /* Available slots */

    printf("=== Capacity Report ===\n");
    printf("Total array size  : %d\n", MAX);
    printf("Stack 1 uses      : %d slots\n", used1);
    printf("Stack 2 uses      : %d slots\n", used2);
    printf("Shared free space : %d slots\n", free);
    printf("Fill percentage   : %.1f%%\n",
           (used1 + used2) * 100.0 / MAX);
}

int main() {
    /* Push 3 to Stack 1 and 2 to Stack 2 */
    arr[++top1]=10; arr[++top1]=20; arr[++top1]=30;
    arr[--top2]=90; arr[--top2]=80;

    showCapacity();
    /* Output:
       Stack 1 uses: 3  Stack 2 uses: 2  Free: 5  */
    return 0;
}
Real-Life Analogy
💾

Like a USB drive showing how much space is used and how much is free — two folders can grow independently, and the remaining space is available to both. The drive doesn't care which folder takes the space; it just tracks the total remaining.

Use Case: Dual-buffer systems — two streaming buffers fill from opposite ends of a buffer pool, and remaining capacity determines whether new data can be received.

8. Display / Traverse Operation

Visiting and printing every element in a stack from top to bottom

Theory

Display (or Traverse) visits every element in a stack and prints it. In stacks, we typically display from top to bottom (most recently added first). For Stack 1, we loop from top1 down to 0. For Stack 2, we loop from top2 up to MAX-1.

Top-to-Bottom Traversal: Since stacks are LIFO (Last In First Out), displaying top-to-bottom shows the most recently added elements first — which is the natural order for a stack.

display_operation.cC
#include <stdio.h>

#define MAX 10
int arr[MAX];
int top1 = -1;
int top2 = MAX;

/* ── Display Stack 1 (top to bottom) ── */
void display1() {
    if (top1 == -1) {
        printf("Stack 1 is empty.\n");
        return;
    }
    printf("Stack 1 (top to bottom): ");
    /* Start from top1, go down to index 0 */
    for (int i = top1; i >= 0; i--) {
        printf("%d ", arr[i]);
    }
    printf("\n");
}

/* ── Display Stack 2 (top to bottom) ── */
void display2() {
    if (top2 == MAX) {
        printf("Stack 2 is empty.\n");
        return;
    }
    printf("Stack 2 (top to bottom): ");
    /* Start from top2, go up to MAX-1 */
    for (int i = top2; i < MAX; i++) {
        printf("%d ", arr[i]);
    }
    printf("\n");
}

/* ── Display kth Stack (follows linked list) ── */
void displayK(int arr[], int top[], int nxt[], int stackNo) {
    printf("Stack %d: ", stackNo);
    int i = top[stackNo];     /* Start at this stack's top */
    while (i != -1) {         /* Follow next pointers */
        printf("%d ", arr[i]);
        i = nxt[i];
    }
    printf("\n");
}

int main() {
    /* Fill some data */
    arr[++top1]=10; arr[++top1]=20; arr[++top1]=30;
    arr[--top2]=90; arr[--top2]=80;
    display1();   /* Output: 30 20 10 */
    display2();   /* Output: 80 90   */
    return 0;
}

🔍

Think of reading a stack of papers — you look at the top sheet first, then the one under it, and so on, until you reach the bottom. Traversal lets you see every element in order without permanently removing them.

Use Case: Debugging — printing the call stack (stack trace) in an IDE to see the sequence of function calls that led to an error.

9. Initialize Multiple Stack

Setting up the data structures before any stack operation can be performed

Theory

Initialization is the first step — setting up all pointers and arrays to their starting values before any push/pop can happen. Without proper initialization, the stacks will behave unpredictably.

Two-Stack Initialization:
Set top1 = -1 (Stack 1 empty) and top2 = MAX (Stack 2 empty). That's all — simple and elegant!

K-Stack Initialization:
Requires a linked free list. Each slot in the array links to the next free slot via a next[] array. Each stack's top is set to -1.

Beginner-Friendly C Code
initialize_stacks.cC
#include <stdio.h>

/* ── Two-Stack Initialization ── */
#define MAX 10
int arr[MAX];
int top1, top2;

void initTwoStacks() {
    top1 = -1;     /* Stack 1 pointer: -1 = empty */
    top2 = MAX;    /* Stack 2 pointer: MAX = empty */
    printf("Two stacks initialized.\n");
    printf("top1 = %d (Stack 1 empty)\n", top1);
    printf("top2 = %d (Stack 2 empty)\n", top2);
}

/* ── K-Stack Initialization ── */
#define MAXK 9
#define K    3

int kArr[MAXK];     /* Shared array for K stacks */
int top[K];         /* Top pointer for each stack */
int nxt[MAXK];      /* Linked free list */
int freeTop;        /* Index of first free slot */

void initKStacks() {
    /* All stacks start empty */
    for (int i = 0; i < K; i++) {
        top[i] = -1;    /* -1 means empty */
    }
    /* Build the free list: each slot points to the next */
    for (int i = 0; i < MAXK - 1; i++) {
        nxt[i] = i + 1;  /* slot i → slot i+1 */
    }
    nxt[MAXK - 1] = -1;  /* Last slot has no next */
    freeTop = 0;          /* Free list starts at index 0 */

    printf("K=%d stacks initialized over %d slots.\n", K, MAXK);
    printf("Free list: 0→1→2→3→...→%d→-1\n", MAXK-1);
}

int main() {
    initTwoStacks();
    initKStacks();
    return 0;
}
Real-Life Analogy
🏗️

Initialization is like setting up numbered lockers before a school starts. Each locker gets a number (index), an "empty" sign (top = -1), and a list of which locker is free next (the next[] array). Before any student (data) arrives, the structure must be prepared.

Use Case: Operating system startup — initializing process stacks, call stacks, and interrupt stacks before the first process runs.

10. Dynamic Space Sharing

Flexible allocation — stacks borrow free memory from a shared pool

Theory

The linked-list approach for k-stacks enables true dynamic space sharing. Instead of dividing the array into fixed k partitions, every slot is part of a global free list.

When any stack needs to push, it grabs a slot from the free list. When it pops, it returns the slot back to the free list. This means no stack ever runs out of its own space as long as any free slot exists anywhere.

Advantage over fixed division: If you divide MAX=9 into K=3 stacks, each gets only 3 slots. Stack 0 might need 7 slots while Stack 1 and 2 need only 1 each — fixed division wastes 4 slots! Dynamic sharing uses all 9 for Stack 0 if needed.

Beginner-Friendly C Code
dynamic_space_sharing.cC
#include <stdio.h>

#define MAXK 9     /* Total shared slots */
#define K    3     /* Number of stacks */

int kArr[MAXK];    /* Stores actual values */
int top[K];        /* Top of each stack */
int nxt[MAXK];     /* Linked list: next free or next in stack */
int freeTop;       /* Head of free-slot list */

void init() {
    for (int i = 0; i < K; i++) top[i] = -1;
    for (int i = 0; i < MAXK-1; i++) nxt[i] = i+1;
    nxt[MAXK-1] = -1;
    freeTop = 0;
}

/* ── Flexible Space Allocation (Push) ── */
void flexPush(int stackNo, int x) {
    if (freeTop == -1) {
        printf("All %d slots used — no free space!\n", MAXK);
        return;
    }
    /* Step 1: Pick the first free slot */
    int i = freeTop;
    /* Step 2: Advance the free list head */
    freeTop = nxt[i];
    /* Step 3: Link this slot to old top of this stack */
    nxt[i] = top[stackNo];
    /* Step 4: This slot is now the new top */
    top[stackNo] = i;
    /* Step 5: Store the value */
    kArr[i] = x;
    printf("Pushed %d to Stack %d at slot %d\n",
           x, stackNo, i);
}

/* ── Shared Memory Management (Pop) ── */
int flexPop(int stackNo) {
    if (top[stackNo] == -1) {
        printf("Stack %d is empty!\n", stackNo);
        return -1;
    }
    /* Step 1: Get the top slot index */
    int i = top[stackNo];
    int val = kArr[i];
    /* Step 2: Update stack top to next in chain */
    top[stackNo] = nxt[i];
    /* Step 3: Return slot to free list */
    nxt[i] = freeTop;
    freeTop = i;
    printf("Popped %d from Stack %d (slot %d returned)\n",
           val, stackNo, i);
    return val;
}

int main() {
    init();
    /* All 3 stacks can freely use any of the 9 slots */
    flexPush(0, 10); flexPush(0, 20); flexPush(0, 30);
    flexPush(1, 50);
    flexPush(2, 70); flexPush(2, 80);
    /* Stack 0 used 3 slots, Stack 1 used 1, Stack 2 used 2 */
    /* 3 slots remain in free pool */
    flexPop(0);   /* Returns slot to free pool */
    return 0;
}

When they leave, the room goes back to the pool. No group has a fixed quota; all share the hotel dynamically!

11. Overflow Collision Handling

Detecting and responding when the two stacks run out of shared space

Theory

Overflow Collision is the moment when Stack 1's right boundary meets Stack 2's left boundary — there is literally no empty slot between them. Any further push to either stack would corrupt the other stack's data.

Detection: top1 + 1 == top2 — this single condition catches collision for the two-stack model.

Boundary Adjustment (Advanced): Some systems handle this by dynamically resizing the underlying array (doubling capacity) and copying elements — but this requires heap allocation.

Never ignore overflow! Pushing past the collision point overwrites valid data in the other stack — causing silent data corruption that is very hard to debug.

#include <stdio.h>
#include <stdlib.h>

#define MAX 10
int arr[MAX];
int top1 = -1;
int top2 = MAX;

/* ── Collision Detection ── */
int detectCollision() {
    /* Adjacent tops = collision = no space between stacks */
    if (top1 + 1 == top2) {
        printf("COLLISION DETECTED! top1=%d, top2=%d\n",
               top1, top2);
        printf("top1+1 (%d) == top2 (%d) → TRUE\n",
               top1+1, top2);
        return 1;   /* Collision found */
    }
    printf("No collision. Gap = %d slots\n",
           top2 - top1 - 1);
    return 0;   /* Safe */
}

/* ── Boundary Adjustment (resize approach) ── */
/* In static arrays this is done by refusing the push.
   With dynamic memory you could do: */
int* resizeAndAdjust(int *oldArr, int *oldMax) {
    int newMax = (*oldMax) * 2;
    int *newArr = (int*)malloc(newMax * sizeof(int));

    /* Copy Stack 1 to the left of new array */
    for (int i = 0; i <= top1; i++)
        newArr[i] = oldArr[i];

    /* Copy Stack 2 to the right of new array */
    int offset = newMax - *oldMax;
    for (int i = top2; i < *oldMax; i++)
        newArr[i + offset] = oldArr[i];

    top2 = top2 + offset;  /* Adjust Stack 2's top */
    *oldMax = newMax;
    printf("Array resized to %d. top2 adjusted to %d\n",
           newMax, top2);
    return newArr;
}

int main() {
    /* Fill array until collision */
    while (!detectCollision()) {
        if (top1 % 2 == 0) arr[++top1] = top1 * 10;
        else arr[--top2] = top2 * 5;
    }
    return 0;
}

12. Clear Stack

Emptying an entire stack in one operation by resetting its top pointer

Theory

Clear Stack empties an entire stack in O(1) time by simply resetting its top pointer — no need to pop each element individually! The data cells may still hold old values but they will be overwritten on future pushes, so they are considered "cleared".

• Clear Stack 1: top1 = -1

• Clear Stack 2: top2 = MAX

If you want to truly zero out the memory (for security), loop through and zero each cell before resetting the pointer.

void secureClear1() {
    /* Zero out all cells Stack 1 owns */
    for (int i = 0; i <= top1; i++) {
        arr[i] = 0;     /* Overwrite with 0 */
    }

}

13. Search Operation

Finding a specific value within a stack using linear traversal

Theory

Stacks do not support direct random access. To find an element, you must perform a Linear Search — visiting each element from the top down until the value is found or the stack is exhausted.

• Time Complexity: O(n) — worst case visits all n elements

• Space Complexity: O(1) — no extra memory needed

For the k-stack (linked list approach), the search follows the next[] chain of a specific stack from its top pointer.

Note: Since the search is O(n) and doesn't use the LIFO property, using a different data structure (like a hash table) is recommended if search is a primary operation.

search_operation.cC
#include <stdio.h>

#define MAX 10
int arr[MAX];
int top1 = -1;
int top2 = MAX;

/* ── Linear Search in Stack 1 ── */
int search1(int key) {
    printf("Searching for %d in Stack 1...\n", key);
    /* Start from top (most recent) and go to bottom */
    for (int i = top1; i >= 0; i--) {
        printf("  Checking index %d: arr[%d] = %d\n",
               i, i, arr[i]);
        if (arr[i] == key) {
            printf("Found %d at index %d! Position from top: %d\n",
                   key, i, top1 - i);
            return i;   /* Return the index where found */
        }
    }
    printf("%d not found in Stack 1.\n", key);
    return -1;          /* Return -1 if not found */
}

/* ── Linear Search in Stack 2 ── */
int search2(int key) {
    printf("Searching for %d in Stack 2...\n", key);
    for (int i = top2; i < MAX; i++) {
        printf("  Checking index %d: arr[%d] = %d\n",
               i, i, arr[i]);
        if (arr[i] == key) {
            printf("Found %d at index %d!\n", key, i);
            return i;
        }
    }
    printf("%d not found in Stack 2.\n", key);
    return -1;
}

/* ── Search in kth Stack (follows linked list) ── */
int searchK(int arr[], int top[], int nxt[], int sn, int key) {
    int i = top[sn];          /* Start at this stack's top */
    int pos = 0;              /* Position from top */
    while (i != -1) {         /* Traverse the chain */
        if (arr[i] == key) {
            printf("Found %d in Stack %d at slot %d "
                   "(pos %d from top)\n", key, sn, i, pos);
            return i;
        }
        i = nxt[i];           /* Follow next pointer */
        pos++;
    }
    return -1;
}

int main() {
    /* Pre-fill stacks */
    arr[++top1]=10; arr[++top1]=20; arr[++top1]=30;
    arr[--top2]=90; arr[--top2]=80;

    search1(20);   /* Found at index 1 */
    search1(99);   /* Not found */
    search2(90);   /* Found at index 9 */
    return 0;
}

📚

Searching a stack is like looking for a specific book in a pile. You start with the top book (most recently placed), check its title, and if it's not what you want, you lift it aside and check the next one. You continue until you find the book or reach the bottom of the pile.
