What is a Dense Graph?
Foundation — understand the core idea before coding

Graph Basics

A graph is a collection of vertices (nodes) connected by edges (links). Think of a map of cities — each city is a vertex and each road connecting two cities is an edge.

A dense graph is a graph that has many edges — close to the maximum possible. If a graph has V vertices, the maximum number of edges is V×(V-1)/2 for undirected, or V×(V-1) for directed. A graph is considered dense when its edge count is close to these maximums.

Real-life analogy: Imagine a group of 5 friends where almost everyone knows everyone else. Each person is a vertex; each friendship is an edge. If nearly all possible friendships exist, that's a dense graph. Compare this to a sparse graph — like a social network where each user follows only 3-4 others out of thousands.

What is an Adjacency List?

An adjacency list stores the graph as an array of linked lists. Each index in the array represents a vertex. The linked list at that index contains all vertices that vertex is connected to (its neighbors).

For example, if vertex 0 is connected to vertices 1, 2, and 3 — then adjList[0] points to a list: 1 → 2 → 3 → NULL.

Analogy: Think of a school attendance register. Each row is a student (vertex). Next to each student's name, you list all their friends in that class (neighbors). That list of friends is the adjacency list for that student.

C code — adjacency list structure
/* Node in the linked list */
struct Node {
    int vertex;           /* which vertex this neighbor is */
    struct Node* next;    /* pointer to next neighbor */
};

/* Graph structure */
struct Graph {
    int numVertices;      /* total number of vertices */
    struct Node** adjList; /* array of linked list heads */
};

/* Create a new graph with V vertices */
struct Graph* createGraph(int V) {
    struct Graph* g = malloc(sizeof(struct Graph));
    g->numVertices = V;
    g->adjList = malloc(V * sizeof(struct Node*));
    for (int i = 0; i < V; i++)
        g->adjList[i] = NULL; /* start with empty lists */
    return g;
}

Understanding Edge Operations

In an adjacency list, inserting an edge from u to v means creating a new node with value v and prepending it to adjList[u]. For undirected graphs, we also add u to adjList[v].

Analogy: Think of a phone contact list. Adding an edge is like adding a new contact to someone's phone — and in an undirected friendship, both people add each other's numbers.

/* Helper: create a new node */
struct Node* newNode(int v) {
    struct Node* n = malloc(sizeof(struct Node));
    n->vertex = v;
    n->next = NULL;
    return n;
}

/* Insert edge u -> v (directed) */
void insertEdge(struct Graph* g, int u, int v) {
    struct Node* n = newNode(v);
    n->next = g->adjList[u];   /* prepend to list */
    g->adjList[u] = n;
    /* For undirected: also add u to adjList[v] */
}

/* Search edge u -> v */
int searchEdge(struct Graph* g, int u, int v) {
    struct Node* cur = g->adjList[u];
    while (cur != NULL) {
        if (cur->vertex == v) return 1; /* found */
        cur = cur->next;
    }
    return 0; /* not found */
}

/* Count all edges */
int countEdges(struct Graph* g) {
    int count = 0;
    for (int i = 0; i < g->numVertices; i++) {
        struct Node* cur = g->adjList[i];
        while (cur) { count++; cur = cur->next; }
    }
    return count; /* divide by 2 for undirected */
}

/* Bulk edge insertion from array */
void bulkInsert(struct Graph* g, int edges[][2], int n) {
    for (int i = 0; i < n; i++)
        insertEdge(g, edges[i][0], edges[i][1]);
}
Breadth First Search (BFS)
Queue Initialization · Visit Node · Mark Visited · Traverse Adjacent Nodes

How BFS Works

BFS explores a graph level by level. Start at a source node, visit all its direct neighbors first, then visit their neighbors, and so on. It uses a Queue (FIFO) to track the next nodes to visit.

Step-by-step: (1) Enqueue the start node, mark it visited. (2) Dequeue a node, process it. (3) Enqueue all unvisited neighbors. (4) Repeat until queue is empty.

Analogy: Imagine you drop a stone in water — ripples spread outward layer by layer. BFS works exactly like that: it visits all nodes at distance 1 first, then distance 2, then distance 3, and so on.

C code — BFS with queue
#define MAX 100

void BFS(struct Graph* g, int start) {
    int visited[MAX] = {0};  /* 0 = not visited */
    int queue[MAX], front = 0, rear = 0;

    /* Step 1: Enqueue start, mark visited */
    visited[start] = 1;
    queue[rear++] = start;

    while (front < rear) {  /* while queue not empty */
        int v = queue[front++];   /* dequeue */
        printf("%d ", v);          /* visit node */

        /* Traverse all adjacent nodes */
        struct Node* cur = g->adjList[v];
        while (cur != NULL) {
            if (!visited[cur->vertex]) {
                visited[cur->vertex] = 1;
                queue[rear++] = cur->vertex;
            }
            cur = cur->next;
        }
    }
}
Depth First Search (DFS)
Recursive DFS · Stack DFS · Visit Node · Backtracking

How DFS Works

DFS explores as deep as possible before backtracking. It goes down one path until it hits a dead end, then backtracks and tries another path. It can be implemented recursively (using the call stack) or iteratively (using an explicit stack).

Analogy: Imagine exploring a maze — you keep going forward through corridors until you hit a wall, then you backtrack to the last intersection and try another corridor. That's exactly DFS!

Recursive DFS
void DFS(struct Graph* g, int v,
         int* visited) {
    visited[v] = 1;
    printf("%d ", v);
    struct Node* cur = g->adjList[v];
    while (cur != NULL) {
        if (!visited[cur->vertex])
            DFS(g, cur->vertex, visited);
        cur = cur->next; /* backtrack */
    }
}
Iterative DFS (Stack)
void DFS_stack(struct Graph* g, int start) {
    int stack[MAX], top = -1;
    int visited[MAX] = {0};
    stack[++top] = start;
    while (top >= 0) {
        int v = stack[top--]; /* pop */
        if (!visited[v]) {
            visited[v] = 1;
            printf("%d ", v);
            struct Node* c = g->adjList[v];
            while(c) {
                if(!visited[c->vertex])
                    stack[++top] = c->vertex;
                c = c->next;
            }
        }
    }
}
Dense Analysis Operations

What Makes a Graph Dense?

Graph density = actual edges / maximum possible edges. A graph with density > 0.5 (or 50%) is typically called dense. For a directed graph: density = E / (V × (V-1)). For undirected: density = 2E / (V × (V-1)).

/* Calculate density */
float graphDensity(int V, int E, int directed) {
    int maxEdges = directed ? V*(V-1) : V*(V-1)/2;
    if (maxEdges == 0) return 0;
    return (float)E / maxEdges;
}

/* Check if graph is dense (density > 0.5) */
int isDense(int V, int E, int directed) {
    return graphDensity(V, E, directed) > 0.5f;
}

/* Edge saturation analysis */
void edgeSaturation(int V, int E) {
    int max = V*(V-1)/2;
    int missing = max - E;
    printf("Current: %d, Max: %d, Missing: %d\n",
            E, max, missing);
}

/* Detect high connectivity (all degrees >= V/2) */
int highConnectivity(struct Graph* g) {
    int V = g->numVertices;
    for (int i = 0; i < V; i++) {
        int deg = 0;
        struct Node* c = g->adjList[i];
        while (c) { deg++; c = c->next; }
        if (deg < V/2) return 0;
    }
    return 1;
}
Connectivity Operations
Check Connected · Strong Connectivity · Reachability · Connected Components

Types of Connectivity

Connected Graph: Every vertex can reach every other vertex (ignoring edge direction). Use BFS/DFS from any vertex — if all vertices get visited, the graph is connected.

Strongly Connected: In a directed graph, every vertex can reach every other via directed paths. Check using Kosaraju's or Tarjan's algorithm — or simply run DFS twice.

Analogy: A connected graph is like a road network where you can drive from any city to any other city. A strongly connected directed graph means you can also drive back — there are no one-way-street dead ends that trap you.

C code — connectivity check
/* Check if graph is connected using DFS */
int isConnected(struct Graph* g) {
    int visited[MAX] = {0};
    DFS(g, 0, visited);  /* DFS from vertex 0 */

    for (int i = 0; i < g->numVertices; i++)
        if (!visited[i]) return 0; /* found unreachable */
    return 1; /* all vertices reachable */
}

/* Count connected components */
int connectedComponents(struct Graph* g) {
    int visited[MAX] = {0}, count = 0;
    for (int i = 0; i < g->numVertices; i++) {
        if (!visited[i]) {
            DFS(g, i, visited);
            count++;  /* new component found */
        }
    }
    return count;
}

/* Check reachability: can we get from src to dst? */
int canReach(struct Graph* g, int src, int dst) {
    int visited[MAX] = {0};
    DFS(g, src, visited);
    return visited[dst];
}
Path Operations
Find Path · Shortest Path · All Paths · Reachability · Path Length

Paths in a Graph

A path is a sequence of vertices where each consecutive pair is connected by an edge. The shortest path has the minimum number of edges (in unweighted graphs). BFS naturally finds the shortest path; DFS finds all possible paths.

Analogy: GPS navigation — a path from home to school might have multiple routes. The shortest path is the one with fewest turns. BFS is how a GPS finds the shortest route level by level.

C code — BFS shortest path
/* BFS shortest path — returns path length, fills path[] */
int shortestPath(struct Graph* g, int src, int dst,
                 int* path, int* pathLen) {
    int visited[MAX]={0}, parent[MAX], q[MAX];
    int front=0, rear=0;

    for(int i=0;i<MAX;i++) parent[i]=-1;
    visited[src]=1; q[rear++]=src;

    while(front<rear) {
        int v=q[front++];
        if(v==dst) break; /* found destination */
        struct Node* c=g->adjList[v];
        while(c) {
            if(!visited[c->vertex]) {
                visited[c->vertex]=1;
                parent[c->vertex]=v;
                q[rear++]=c->vertex;
            }
            c=c->next;
        }
    }
    /* Trace back path */
    *pathLen=0;
    for(int v=dst; v!=-1; v=parent[v])
        path[(*pathLen)++]=v;
    /* reverse path[] for correct order */
    return visited[dst];
}
Cycle Operations
Detect Cycle · Count Cycles · Hamiltonian Cycle · Euler Cycle

Types of Cycles

Basic cycle: A path that starts and ends at the same vertex. Detected with DFS — if we visit a node that's already visited (and not the parent), a cycle exists.

Hamiltonian cycle: Visits every vertex exactly once and returns to start. Very hard to find (NP-complete).

Euler cycle: Traverses every edge exactly once. Exists if and only if every vertex has even degree.

Analogy: A basic cycle is like a circular road. A Hamiltonian cycle is a road trip that visits every city exactly once and returns home — extremely hard to plan optimally. An Euler cycle visits every road exactly once — like a mail carrier covering every street on their route.

C code — cycle detection & Euler check
/* Detect cycle in undirected graph using DFS */
int detectCycleDFS(struct Graph* g, int v,
                   int* visited, int parent) {
    visited[v] = 1;
    struct Node* cur = g->adjList[v];
    while (cur != NULL) {
        int u = cur->vertex;
        if (!visited[u]) {
            if (detectCycleDFS(g, u, visited, v))
                return 1; /* cycle found */
        } else if (u != parent) {
            return 1; /* back edge = cycle! */
        }
        cur = cur->next;
    }
    return 0;
}

/* Euler cycle: all vertices must have even degree */
int hasEulerCycle(struct Graph* g) {
    for (int i = 0; i < g->numVertices; i++) {
        int deg = 0;
        struct Node* c = g->adjList[i];
        while (c) { deg++; c = c->next; }
        if (deg % 2 != 0) return 0; /* odd degree */
    }
    return 1;
}

What is Vertex Degree?

The degree of a vertex is the number of edges connected to it (its neighbor count). In dense graphs, most vertices have high degrees. The degree sequence describes how degrees are distributed across all vertices.

/* Find degree of a single vertex */
int findDegree(struct Graph* g, int v) {
    int deg = 0;

/* Maximum degree in graph */
int maxDegree(struct Graph* g) {
    int max = 0;

        int d = findDegree(g, i);
        if (d > max) max = d;

float avgDegree(struct Graph* g) {
    int total = 0;

        total += findDegree(g, i);
    return (float)total / g->numVertices;

Structural Operations
Graph Size · Order · Density Check · Sparse/Dense Classification · Complete Graph · Clique Detection

Graph Size vs Order

Order = number of vertices (V). Size = number of edges (E). These are the two fundamental metrics of a graph.

Complete Graph

A complete graph Kn has an edge between every pair of vertices. It has exactly n(n-1)/2 edges — the maximum possible. Every complete graph is dense.

Clique Detection

A clique is a subset of vertices that are all connected to each other (a complete subgraph). Finding the maximum clique is NP-complete but important in social network analysis.

Sparse vs Dense

Rule of thumb: if E is O(V²), it's dense. If E is O(V), it's sparse. Use adjacency matrix for dense graphs (O(1) edge lookup) and adjacency list for sparse (saves memory).

Analogy: A clique in social network terms is a group of friends where everyone knows everyone else — like a tight-knit team. Finding the largest such group is extremely hard computationally.

Floyd-Warshall Algorithm — All Pairs Shortest Path

Floyd-Warshall finds the shortest path between every pair of vertices at once. It uses dynamic programming: for each intermediate vertex k, update the distance between every pair (i, j) if going through k is shorter. Complexity: O(V³). Ideal for dense graphs.

Analogy: You have a table of direct flight prices between cities. Floyd-Warshall finds the cheapest route between every pair of cities, even if it means taking connecting flights through other cities.

C code — Floyd-Warshall
#define INF 99999

/* Floyd-Warshall: dist[i][j] = shortest path i to j */
void floydWarshall(int dist[][MAX], int V) {
    /* Try every vertex k as intermediate */
    for (int k = 0; k < V; k++)
        for (int i = 0; i < V; i++)
            for (int j = 0; j < V; j++)
                if (dist[i][k] + dist[k][j] < dist[i][j])
                    dist[i][j] = dist[i][k] + dist[k][j];
}

/* Initialize distance matrix from adjacency matrix */
void initDist(int adj[][MAX], int dist[][MAX], int V) {
    for (int i = 0; i < V; i++)
        for (int j = 0; j < V; j++)
            dist[i][j] = (i == j) ? 0 :
                         (adj[i][j] ? adj[i][j] : INF);
}

Prim's Algorithm — Minimum Spanning Tree (MST)

Prim's builds a Minimum Spanning Tree — a subset of edges connecting all vertices with minimum total weight and no cycles. It greedily picks the cheapest edge that extends the current tree. Works well on dense graphs. Complexity: O(V²) with adjacency matrix, O(E log V) with priority queue.

Analogy: You want to connect all houses in a neighborhood with cables at minimum total cost. Prim's starts from one house and always extends the network by adding the cheapest new connection.

C code — Prim's algorithm
/* Prim's MST using adjacency matrix */
void primMST(int graph[][MAX], int V) {
    int key[MAX], parent[MAX], inMST[MAX];

    for(int i=0;i<V;i++) {
        key[i]=INF; inMST[i]=0;
    }
    key[0]=0; parent[0]=-1;

    for(int count=0; count<V-1; count++) {
        /* Pick minimum key vertex not in MST */
        int u=-1;
        for(int v=0;v<V;v++)
            if(!inMST[v] && (u==-1 || key[v]<key[u]))
                u=v;

        inMST[u]=1;
        /* Update keys of adjacent vertices */
        for(int v=0;v<V;v++)
            if(graph[u][v] && !inMST[v] &&
               graph[u][v]<key[v]) {
                key[v]=graph[u][v];
                parent[v]=u;
            }
    }
    /* Print MST */
    for(int i=1;i<V;i++)
        printf("%d-%d: weight %d\n", parent[i], i, key[i]);
}

Graph Coloring

Assign colors to vertices so that no two adjacent vertices share the same color. The minimum number of colors needed is the chromatic number. The greedy approach assigns the smallest available color to each vertex in order.

C code — greedy graph coloring
/* Greedy graph coloring */
void greedyColor(struct Graph* g) {
    int V = g->numVertices;
    int color[MAX], available[MAX];

    for(int i=0;i<V;i++) color[i]=-1;

    color[0]=0; /* assign first color to first vertex */

    for(int u=1;u<V;u++) {
        for(int i=0;i<V;i++) available[i]=1;

        /* Mark colors used by neighbors */
        struct Node* c = g->adjList[u];
        while(c) {
            if(color[c->vertex] != -1)
                available[color[c->vertex]] = 0;
            c = c->next;
        }
        /* Assign smallest available color */
        for(int i=0;i<V;i++) {
            if(available[i]) { color[u]=i; break; }
        }
    }
    for(int i=0;i<V;i++)
        printf("Vertex %d -> Color %d\n", i, color[i]);
}
Utility Operations
Is Empty · Clear Graph · Copy Graph · Display Graph · Reset Visited Array

isEmpty

Returns true if the graph has no vertices (numVertices == 0). Useful as a guard before performing any graph operations.

clearGraph

Frees all linked list nodes for every vertex, then resets all adjacency list pointers to NULL. The graph exists but has no edges.

copyGraph

Creates a new graph with the same vertices and edges. Must deep-copy the linked lists — not just copy pointers, or changes to one graph affect the other.

resetVisited

After a BFS/DFS traversal, the visited array must be reset (all zeros) before performing another traversal. A common bug is forgetting this step.

Analogy: copyGraph is like photocopying a document — the copy is independent. But if you only copy the cover page reference (pointer copy), changing one changes both. Always do a deep copy!

/* Clear all edges from graph */
void clearGraph(struct Graph* g) {
    for (int i = 0; i < g->numVertices; i++) {
        struct Node* cur = g->adjList[i];
        while (cur) {
            struct Node* tmp = cur;
            cur = cur->next;
            free(tmp);
        }
        g->adjList[i] = NULL;
    }
}

/* Deep copy a graph */
struct Graph* copyGraph(struct Graph* src) {
    struct Graph* dst = createGraph(src->numVertices);
    for (int i = 0; i < src->numVertices; i++) {
        struct Node* cur = src->adjList[i];
        while (cur) {
            insertEdge(dst, i, cur->vertex);
            cur = cur->next;
        }
    }
    return dst;
}

/* Display full adjacency list */

    for (int i = 0; i < g->numVertices; i++) {
        printf("%d: ", i);
        struct Node* cur = g->adjList[i];
        while (cur) {
            printf("%d -> ", cur->vertex);
            cur = cur->next;
        }
        printf("NULL\n");
    }
}

/* Reset visited array */
void resetVisited(int* visited, int V) {
    for (int i = 0; i < V; i++) visited[i] = 0;
}
