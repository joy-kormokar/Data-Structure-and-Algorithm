Connected Graph — Incidence Matrix

Foundation
What is a Connected Graph?
A graph where every vertex is reachable from every other vertex — no node is left isolated. Connected graphs are the backbone of networks, maps, and relationships in computer science.
Core Concepts
  • 1Vertex (Node): A point in the graph. Represents an entity (city, user, router).
  • 2Edge: A connection between two vertices. Represents a relationship.
  • 3Connected: A path exists between every pair of vertices.
  • 4Directed vs Undirected: Edges can be one-way (arrows) or two-way (lines).
  • 5Weighted: Edges can carry a numerical value (distance, cost).
Key Terminology
V = Vertices E = Edges Degree = edge count Path = sequence of vertices Cycle = closed path Adjacency = direct connection Incidence = vertex-edge relation Component = isolated subgraph
🎨 Interactive Connected Graph
🌍 Real-Life Analogy
Think of a road map. Cities are vertices. Roads are edges. A connected graph means you can drive from any city to any other city — there are no isolated towns!
Representation
Incidence Matrix
A 2D array of size V×E where rows represent vertices and columns represent edges. Cell [i][e] = 1 if vertex i is an endpoint of edge e (for undirected), or +1 (tail) / -1 (head) for directed graphs.
📊 Live Matrix Builder
Rules of Incidence Matrix
  • 1Rows = Vertices (V rows)
  • 2Columns = Edges (E columns)
  • 3Cell [i][e]=1 if vertex i touches edge e (undirected)
  • 4Each undirected edge column has exactly TWO 1s
  • 5Directed: +1 = tail (source), -1 = head (destination)
  • 6Sum of column = 2 (undirected), 0 (directed)
  • 7Sum of row = degree of that vertex
C — Structure
#define MAX_V 10 #define MAX_E 20 typedef struct { int matrix[MAX_V][MAX_E]; /* incidence matrix */ int vertices; /* number of vertices */ int edges; /* number of edges */ char label[MAX_V][20]; /* vertex labels */ } Graph; /* Initialize an empty graph */ void initGraph(Graph *g) { g->vertices = 0; g->edges = 0; for(int i=0; i<MAX_V; i++) for(int e=0; e<MAX_E; e++) g->matrix[i][e] = 0; }
🎭 Real-Life Analogy
Imagine a school attendance sheet. Rows = students (vertices), Columns = classes (edges). Mark 1 if student attends that class. The incidence matrix works the same — it tracks which vertices "attend" (are endpoints of) each edge.
Vertex Operations
Insert Vertex
Adding a new vertex means expanding the incidence matrix by adding a new row of zeros (the new vertex has no edges yet).
How it works
  • 1Check if max vertices not exceeded
  • 2Add a new row (all zeros) to the matrix
  • 3Store the vertex label/name
  • 4Increment vertex count (V++)
Time: O(E) Space: O(1) new row

C — Insert Vertex
/* Insert a new vertex with given label */ int insertVertex(Graph *g, char *label) { /* Check capacity */ if (g->vertices >= MAX_V) { printf("Graph is full!\n"); return -1; } int v = g->vertices; /* new vertex index */ /* New row = all zeros (no edges yet) */ for (int e = 0; e < g->edges; e++) g->matrix[v][e] = 0; /* Store label */ strcpy(g->label[v], label); /* Increase vertex count */ g->vertices++; printf("Vertex '%s' inserted at index %d\n", label, v); return v; }
🏙 Real-Life Analogy
Like adding a new city to a map. The city exists (new row added), but it has no roads yet (all zeros). Roads are added separately via edge operations.
Vertex Operations
Delete Vertex
Removing a vertex requires deleting its row from the matrix AND removing all edges (columns) that are incident to it.
Deletion Steps
  • 1Find all edges (columns) incident to the vertex
  • 2Remove those edge columns (shift left)
  • 3Remove the vertex row (shift rows up)
  • 4Decrement V and E counts accordingly
Time: O(V×E) Careful: cascading deletions
C — Delete Vertex
void deleteVertex(Graph *g, int v) { if (v < 0 || v >= g->vertices) { printf("Invalid vertex!\n"); return; } /* Step 1: Remove edges incident to v */ int e = 0; while (e < g->edges) { if (g->matrix[v][e] != 0) { /* Shift columns left */ for (int i=0; i<g->vertices; i++) for (int j=e; j<g->edges-1; j++) g->matrix[i][j] = g->matrix[i][j+1]; g->edges--; } else e++; } /* Step 2: Remove vertex row (shift up) */ for (int i=v; i<g->vertices-1; i++) { for (int j=0; j<g->edges; j++) g->matrix[i][j] = g->matrix[i+1][j]; strcpy(g->label[i], g->label[i+1]); } g->vertices--; printf("Vertex %d deleted.\n", v); }
🏗 Real-Life Analogy
Like demolishing a city. All roads (edges) connected to it are also demolished — you can't have a road to nowhere!
Vertex Operations
Search Vertex
Find a vertex by its label or index by scanning the vertex labels array.
C — Search Vertex
/* Search by label, return index or -1 */ int searchVertex(Graph *g, char *label) { for (int i = 0; i < g->vertices; i++) { if (strcmp(g->label[i], label) == 0) { printf("Found '%s' at index %d\n", label, i); return i; } } printf("Vertex '%s' not found.\n", label); return -1; }
Search Details
Linear search through the label array. Returns the row index, which you use to read that vertex's row in the incidence matrix.
Time: O(V) Uses: strcmp()

Vertex Operations
Update Vertex
Change the label of an existing vertex. The matrix structure stays identical — only the label array entry changes.
C — Update Vertex
int updateVertex(Graph *g, int v, char *newLabel) { if (v < 0 || v >= g->vertices) { printf("Invalid index!\n"); return -1; } printf("Updating '%s' to '%s'\n", g->label[v], newLabel); strcpy(g->label[v], newLabel); return 0; }
Notes
Only label changes. All incidence matrix values remain the same because edge relationships don't change — you're just renaming the vertex.

Time complexity: O(1) once index is known.
🏷 Real-Life Analogy
Like renaming a city. Bombay → Mumbai. All roads still connect the same place — only the name tag changes.
Vertex Operations
Count Vertices
Simply returns the current number of vertices stored in the graph structure. This is the "order" of the graph.
C — Count Vertices
/* Returns number of vertices (graph order) */ int countVertices(Graph *g) { printf("Total vertices: %d\n", g->vertices); return g->vertices; } /* Or simply read the field directly: */ /* int V = g->vertices; */
Key Points
• The count is always maintained as an integer field
• O(1) operation — no traversal needed
• Also called the order of the graph
• The matrix has exactly this many rows
Time: O(1) = Graph Order
Vertex Operations
Display Vertices
Print all vertex labels and their corresponding row index in the incidence matrix.
C — Display Vertices
void displayVertices(Graph *g) { printf("\n=== Vertices (%d total) ===\n", g->vertices); for (int i = 0; i < g->vertices; i++) { printf(" [%d] %s", i, g->label[i]); /* Show degree = sum of row i */ int degree = 0; for (int e = 0; e < g->edges; e++) if (g->matrix[i][e] != 0) degree++; printf(" (degree = %d)\n", degree); } }
Edge Operations
Insert Edge
Adding an edge means adding a new column to the incidence matrix. For an undirected edge between vertices u and v, set matrix[u][e]=1 and matrix[v][e]=1.
Steps
  • 1Check max edges not exceeded
  • 2Validate vertex indices u and v
  • 3Create new column (all zeros)
  • 4Set matrix[u][e] = 1, matrix[v][e] = 1
  • 5Increment edge count (E++)
Time: O(1) Undirected: two 1s per column Directed: +1 tail, -1 head

C — Insert Edge (Undirected)
int insertEdge(Graph *g, int u, int v) { /* Bounds check */ if (g->edges >= MAX_E) { printf("Edge limit reached!\n"); return -1; } if (u < 0 || u >= g->vertices || v < 0 || v >= g->vertices) { printf("Invalid vertices!\n"); return -1; } int e = g->edges; /* new column index */ /* Zero out the new column */ for (int i = 0; i < g->vertices; i++) g->matrix[i][e] = 0; /* Mark both endpoints */ g->matrix[u][e] = 1; g->matrix[v][e] = 1; g->edges++; printf("Edge %d-%d added (col %d)\n",u,v,e); return e; } /* For DIRECTED graph: */ /* matrix[u][e] = +1; (tail/source) */ /* matrix[v][e] = -1; (head/dest) */
🛣 Real-Life Analogy
Building a new road between two cities. Both cities (vertices) now have an entry in the road's column — they are incident to that road.
Edge Operations
Delete Edge
Remove an edge by deleting its column from the incidence matrix, then shift all subsequent columns left to fill the gap.
C — Delete Edge
void deleteEdge(Graph *g, int edgeIndex) { if (edgeIndex < 0 || edgeIndex >= g->edges){ printf("Invalid edge index!\n"); return; } /* Shift columns left from edgeIndex */ for (int i = 0; i < g->vertices; i++) for (int j = edgeIndex; j < g->edges-1; j++) g->matrix[i][j] = g->matrix[i][j+1]; g->edges--; printf("Edge %d deleted.\n", edgeIndex); } /* Find edge between u and v first: */ int findEdge(Graph *g, int u, int v) { for (int e = 0; e < g->edges; e++) if (g->matrix[u][e] != 0 && g->matrix[v][e] != 0) return e; return -1; /* not found */ }
Key Points
• Delete = remove one column
• Must shift all columns right of deletion left
• Decrement E after removal
• To delete by endpoints (u,v), first find the edge column using findEdge()
Time: O(V×E) Column shift needed
Edge Operations
Search Edge
Find the column index of an edge that connects vertices u and v. Scan each column looking for the column where both matrix[u][e] and matrix[v][e] are non-zero.
C — Search Edge
int searchEdge(Graph *g, int u, int v) { for (int e = 0; e < g->edges; e++) { /* Undirected: both cells nonzero */ if (g->matrix[u][e] != 0 && g->matrix[v][e] != 0) { printf("Edge (%d,%d) at col %d\n", u, v, e); return e; } } printf("No edge between %d and %d\n",u,v); return -1; }
Search Logic
For each edge column e:
• If both matrix[u][e] AND matrix[v][e] are non-zero → this is the edge connecting u and v

For directed graphs, also check the sign (+1 vs -1) to confirm direction.
Time: O(E)
Edge Operations
Update Edge
Modify an existing edge — either reassign its endpoints or update its weight. Done by modifying the column values in the incidence matrix.
C — Update Edge
/* Update edge at column e to connect newU and newV */ int updateEdge(Graph *g, int e, int newU, int newV) { if (e < 0 || e >= g->edges) { printf("Invalid edge!\n"); return -1; } /* Clear the column first */ for (int i = 0; i < g->vertices; i++) g->matrix[i][e] = 0; /* Set new endpoints */ g->matrix[newU][e] = 1; g->matrix[newV][e] = 1; printf("Edge %d updated to (%d-%d)\n", e, newU, newV); return 0; }
Edge Operations
Count & Display Edges
Count returns the number of columns in the incidence matrix. Display iterates columns, finds the two non-zero rows (endpoints), and prints them.
C — Count Edges
/* Count = graph SIZE */ int countEdges(Graph *g) { printf("Total edges: %d\n", g->edges); return g->edges; }
C — Display Edges
void displayEdges(Graph *g) { printf("\n=== Edges (%d) ===\n", g->edges); for (int e = 0; e < g->edges; e++) { printf(" Edge e%d: ", e); for (int i = 0; i < g->vertices; i++) if (g->matrix[i][e] != 0) printf("%s(%+d) ", g->label[i], g->matrix[i][e]); printf("\n"); } }
Traversal
Breadth First Search (BFS)
BFS explores the graph level by level. Start at a node, visit all its neighbors first, then their neighbors, using a Queue. Guarantees the shortest path in unweighted graphs.
BFS Algorithm Steps
  • 1Queue Init: Create an empty queue, enqueue start vertex
  • 2Mark Visited: Mark start as visited
  • 3Dequeue: Remove front vertex, process it
  • 4Traverse Adjacent: Find all adjacent vertices via incidence matrix
  • 5Enqueue: Enqueue unvisited adjacent vertices, mark visited
  • 6Repeat steps 3-5 until queue empty

C — BFS with Incidence Matrix
#include <stdio.h> /* Simple queue using array */ int queue[100], front=0, rear=0; void enqueue(int x){ queue[rear++]=x; } int dequeue() { return queue[front++]; } int isEmpty() { return front==rear; } /* Find adjacent vertices using incidence matrix */ void getAdjacent(Graph *g, int u, int adj[], int *count) { *count = 0; for(int e=0; e<g->edges; e++) { if(g->matrix[u][e] != 0) { /* find the other endpoint */ for(int v=0; v<g->vertices; v++) if(v!=u && g->matrix[v][e]!=0) adj[(*count)++] = v; } } } void BFS(Graph *g, int start) { int visited[MAX_V] = {0}; front = rear = 0; enqueue(start); visited[start] = 1; printf("BFS: "); while(!isEmpty()) { int u = dequeue(); printf("%s ", g->label[u]); int adj[MAX_V], cnt; getAdjacent(g, u, adj, &cnt); for(int i=0; i<cnt; i++) { if(!visited[adj[i]]) { visited[adj[i]] = 1; enqueue(adj[i]); } } } printf("\n"); }
🌊 Real-Life Analogy
Like ripples in a pond. Drop a stone at the start node — the water spreads outward level by level. All nodes at distance 1 are visited before nodes at distance 2.
Traversal
Depth First Search (DFS)
DFS goes as deep as possible down one branch before backtracking. Can be implemented recursively (call stack) or iteratively (explicit stack).
DFS vs BFS
DFS
Uses Stack / Recursion
Goes deep first
Good for: cycles, paths
May not find shortest path
BFS
Uses Queue
Goes wide first
Good for: shortest path
Level-by-level exploration

C — Recursive DFS
int visited[MAX_V]; void DFSUtil(Graph *g, int u) { visited[u] = 1; printf("%s ", g->label[u]); int adj[MAX_V], cnt; getAdjacent(g, u, adj, &cnt); for(int i=0; i<cnt; i++) if(!visited[adj[i]]) DFSUtil(g, adj[i]); /* recurse */ } void DFS(Graph *g, int start) { for(int i=0;i<g->vertices;i++) visited[i]=0; printf("DFS: "); DFSUtil(g, start); printf("\n"); }
C — Stack DFS (Iterative)
void DFS_Stack(Graph *g, int start) { int stack[MAX_V], top=-1; int vis[MAX_V]={0}; stack[++top] = start; while(top >= 0) { int u = stack[top--]; /* pop */ if(vis[u]) continue; vis[u] = 1; printf("%s ", g->label[u]); int adj[MAX_V], cnt; getAdjacent(g, u, adj, &cnt); /* push neighbors (reverse order) */ for(int i=cnt-1; i>=0; i--) if(!vis[adj[i]]) stack[++top] = adj[i]; } }
🏔 Real-Life Analogy
Like exploring a maze. You go down one corridor as far as possible. Hit a dead end? Backtrack to the last junction and try a different direction.
Connectivity
Check Connected Graph
A graph is connected if running BFS/DFS from any vertex visits ALL vertices. If any vertex remains unvisited, the graph is disconnected.
C — Is Connected?
int isConnected(Graph *g) { if(g->vertices == 0) return 1; int vis[MAX_V] = {0}; /* DFS from vertex 0 */ dfsVisit(g, 0, vis); /* Check all vertices visited */ for(int i=0; i<g->vertices; i++) { if(!vis[i]) { printf("NOT connected: vertex" " %d unreachable\n",i); return 0; } } printf("Graph IS connected!\n"); return 1; }
Visual Test
Run DFS from vertex 0 → if visited count = V → connected. Otherwise, isolated components exist.
Time: O(V+E) Uses DFS internally

Connectivity
Strong Connectivity
In a directed graph, strongly connected means: for every pair (u, v), there is a directed path from u→v AND from v→u. Tested with Kosaraju's or Tarjan's algorithm.
C — Strong Connectivity Check
/* Kosaraju's idea (simplified): */ /* 1. DFS on original graph */ /* 2. DFS on transposed graph */ /* If all reachable in both → strongly */ /* connected */ void dfsVisit(Graph *g, int u, int vis[]) { vis[u] = 1; int adj[MAX_V], cnt; getAdjacent(g, u, adj, &cnt); for(int i=0; i<cnt; i++) if(!vis[adj[i]]) dfsVisit(g, adj[i], vis); } int isStronglyConnected(Graph *g) { int vis[MAX_V]={0}; dfsVisit(g, 0, vis); for(int i=0;i<g->vertices;i++) if(!vis[i]) return 0; /* Transpose: reverse all edge signs */ Graph trans = *g; for(int i=0;i<trans.vertices;i++) for(int e=0;e<trans.edges;e++) trans.matrix[i][e] *= -1; int vis2[MAX_V]={0}; dfsVisit(&trans, 0, vis2); for(int i=0;i<g->vertices;i++) if(!vis2[i]) return 0; return 1; }
Strong vs Weak Connectivity
Strongly Connected: Every vertex can reach every other vertex following directed edges. A→B AND B→A paths both exist.
Weakly Connected: Ignoring edge directions, the graph would be connected. A→B path exists but not necessarily B→A.
Connectivity
Weak Connectivity
A directed graph is weakly connected if treating all directed edges as undirected produces a connected graph.
C — Weak Connectivity
/* Treat all edges as undirected */ /* Use absolute value of matrix cells */ void dfsWeak(Graph *g, int u, int vis[]) { vis[u] = 1; for(int e=0; e<g->edges; e++) { if(g->matrix[u][e] != 0) { for(int v=0;v<g->vertices;v++) { /* ignore direction (abs value) */ if(v!=u && g->matrix[v][e] != 0 && !vis[v]) dfsWeak(g, v, vis); } } } } int isWeaklyConnected(Graph *g) { int vis[MAX_V]={0}; dfsWeak(g, 0, vis); for(int i=0;i<g->vertices;i++) if(!vis[i]) return 0; return 1; }
Connectivity Hierarchy
Disconnected → Weakly Connected → Strongly Connected

• Undirected graph: connected or disconnected
• Directed graph: weakly or strongly connected
• Strongly connected ⊆ Weakly connected

Find Reachable Vertices
Given a starting vertex, find all vertices reachable from it via DFS/BFS. Returns the set of vertices you can reach following edges.

void findReachable(Graph *g, int src) { int vis[MAX_V] = {0}; dfsVisit(g, src, vis); printf("Reachable from %s: ", g->label[src]); int count = 0; for(int i=0; i<g->vertices; i++) { if(vis[i] && i != src) { printf("%s ", g->label[i]); count++; } } printf("\nTotal reachable: %d\n", count); }
Connectivity
Count Connected Components
Count how many disconnected subgraphs exist. Start DFS from each unvisited vertex — each new DFS start means a new component.
C — Count Components
int countComponents(Graph *g) { int vis[MAX_V] = {0}; int components = 0; for(int i=0; i<g->vertices; i++) { if(!vis[i]) { /* New component found! */ printf("Component %d: ", components+1); dfsVisit(g, i, vis); printf("\n"); components++; } } printf("Total components: %d\n", components); return components; }
How Components Work
A connected graph has exactly 1 component. Each disconnected subgraph adds to the count.

A graph with V vertices and no edges has V components.

Connectivity
Detect Isolated Vertex
An isolated vertex has degree 0 — its entire row in the incidence matrix is all zeros. It is not connected to any other vertex.
C — Detect Isolated Vertices
int detectIsolated(Graph *g) { int count = 0; for(int i=0; i<g->vertices; i++) { int degree = 0; /* Sum of non-zero in row i */ for(int e=0;e<g->edges;e++) if(g->matrix[i][e] != 0) degree++; if(degree == 0) { printf("Isolated: %s\n", g->label[i]); count++; } } if(count == 0) printf("No isolated vertices.\n"); return count; }
🏝 Real-Life Analogy
Like an island with no bridges. You can't reach it from the mainland no matter which route you take.
Connectivity
Connectivity Restoration
If a graph is disconnected, add edges to connect all components into one connected graph. A minimum spanning approach: connect one vertex from each component to another.
C — Restore Connectivity
/* Find one representative from each component */ /* and connect them to make graph connected. */ void restoreConnectivity(Graph *g) { int vis[MAX_V]={0}; int rep[MAX_V]; /* one vertex per component */ int compCount = 0; for(int i=0; i<g->vertices; i++) { if(!vis[i]) { rep[compCount++] = i; dfsVisit(g, i, vis); } } if(compCount == 1) { printf("Already connected!\n"); return; } /* Connect consecutive component reps */ for(int c=0; c<compCount-1; c++) { printf("Adding edge: %s -- %s\n", g->label[rep[c]], g->label[rep[c+1]]); insertEdge(g, rep[c], rep[c+1]); } }
🌉 Real-Life Analogy
Like building bridges between islands. You find disconnected landmasses and build the minimum number of bridges to connect everything.
Path Operations
Find Path
Find any path between source and destination using DFS with backtracking. Returns one valid sequence of vertices from source to destination.
C — Find Path (DFS)
int path[MAX_V], pathLen; int findPathDFS(Graph *g, int u, int dest, int vis[]) { vis[u] = 1; path[pathLen++] = u; if(u == dest) return 1; /* found! */ int adj[MAX_V], cnt; getAdjacent(g, u, adj, &cnt); for(int i=0; i<cnt; i++) { if(!vis[adj[i]]) { if(findPathDFS(g, adj[i], dest, vis)) return 1; } } /* Backtrack: remove current from path */ pathLen--; return 0; } void printPath(Graph *g) { for(int i=0; i<pathLen; i++) { printf("%s", g->label[path[i]]); if(i < pathLen-1) printf(" → "); } printf("\n"); }
Path Types
Simple Path: No vertex repeated
Trail: No edge repeated
Walk: Anything goes (can repeat)

DFS with a visited array gives a simple path. Remove the visited check for walks.

Path Operations
Shortest Path (BFS)
For unweighted graphs, BFS finds the shortest path — the path with the fewest edges. Track parent pointers to reconstruct the path.
C — Shortest Path via BFS
void shortestPath(Graph *g, int src, int dest) { int vis[MAX_V]={0}; int parent[MAX_V]; int dist[MAX_V]; for(int i=0;i<g->vertices;i++) { parent[i] = -1; dist[i] = 999999; } front = rear = 0; enqueue(src); vis[src]=1; dist[src]=0; while(!isEmpty()) { int u = dequeue(); int adj[MAX_V], cnt; getAdjacent(g, u, adj, &cnt); for(int i=0;i<cnt;i++) { if(!vis[adj[i]]) { vis[adj[i]]=1; dist[adj[i]]=dist[u]+1; parent[adj[i]]=u; enqueue(adj[i]); } } } /* Reconstruct path via parent array */ if(dist[dest] == 999999) { printf("No path found!\n"); return; } printf("Shortest dist: %d\n",dist[dest]); /* Trace back via parent[] */ int stack[MAX_V],top=0,cur=dest; while(cur != -1) { stack[top++]=cur; cur=parent[cur]; } while(top>0) printf("%s ",g->label[stack[--top]]); }
🗺 Real-Life Analogy
Like Google Maps finding the route with fewest turns. BFS explores all routes at the same distance simultaneously, guaranteeing the shortest one is found first.
Path Operations
All Paths
Find every possible simple path from source to destination using DFS with backtracking. Can be exponential in the worst case.
C — Find All Paths
void allPaths(Graph *g, int u, int dest, int vis[], int path[], int pLen) { vis[u] = 1; path[pLen++] = u; if(u == dest) { /* Print this path */ for(int i=0;i<pLen;i++) { printf("%s",g->label[path[i]]); if(i<pLen-1) printf("→"); } printf("\n"); } else { int adj[MAX_V], cnt; getAdjacent(g, u, adj, &cnt); for(int i=0;i<cnt;i++) if(!vis[adj[i]]) allPaths(g, adj[i], dest, vis, path, pLen); } /* Backtrack */ vis[u] = 0; pLen--; }
Path Operations
Reachability Check
Check if vertex destination is reachable from source. Simply run DFS from source and check if destination was visited.
C — Reachability Check
int isReachable(Graph *g, int src, int dest) { int vis[MAX_V]={0}; dfsVisit(g, src, vis); if(vis[dest]) { printf("%s IS reachable from %s\n", g->label[dest], g->label[src]); return 1; } printf("%s NOT reachable from %s\n", g->label[dest], g->label[src]); return 0; }
Time: O(V+E) Returns: 1 = reachable, 0 = not
Path Operations
Path Length Count
The length of a path is the number of edges in it (for unweighted graphs) or the sum of edge weights (for weighted). BFS distance array gives path lengths.
C — Path Length via BFS
int pathLength(Graph *g, int src, int dest) { int dist[MAX_V], vis[MAX_V]; for(int i=0;i<g->vertices;i++) { dist[i]=-1; vis[i]=0; } front=rear=0; enqueue(src); vis[src]=1; dist[src]=0; while(!isEmpty()) { int u=dequeue(); int adj[MAX_V],cnt; getAdjacent(g,u,adj,&cnt); for(int i=0;i<cnt;i++) { if(!vis[adj[i]]) { vis[adj[i]]=1; dist[adj[i]]=dist[u]+1; enqueue(adj[i]); } } } printf("Length from %s to %s: %d\n", g->label[src],g->label[dest], dist[dest]); return dist[dest]; /* -1 if unreachable */ }
Cycle Operations
Detect Cycle
A cycle exists if during DFS we encounter an already-visited node that is not the parent. For undirected graphs, use DFS; for directed, use recursion stack coloring.
C — Detect Cycle (Undirected)
int hasCycleDFS(Graph *g, int u, int parent, int vis[]) { vis[u] = 1; int adj[MAX_V], cnt; getAdjacent(g, u, adj, &cnt); for(int i=0; i<cnt; i++) { int v = adj[i]; if(!vis[v]) { if(hasCycleDFS(g,v,u,vis)) return 1; } else if(v != parent) { /* Back edge = cycle found! */ printf("Cycle at: %s\n", g->label[v]); return 1; } } return 0; } int detectCycle(Graph *g) { int vis[MAX_V]={0}; for(int i=0;i<g->vertices;i++) if(!vis[i]) if(hasCycleDFS(g,i,-1,vis)) return 1; return 0; }
Cycle Detection Key Idea
During DFS: if we reach a vertex that is already visited AND it's not our direct parent → there's a back edge → there's a cycle!

For directed graphs, use a "recursion stack" (color coding: white=unvisited, gray=in stack, black=done).

Cycle Operations
Count Cycles, Check Cyclic Structure & Remove Cycle
Count total number of independent cycles; check if entire graph is cyclic; remove an edge to break a cycle.
C — Count Cycles (Circuit Rank)
/* Circuit rank (cyclomatic complexity) */ /* = E - V + C (C = connected components) */ int countCycles(Graph *g) { int C = countComponents(g); int rank = g->edges - g->vertices + C; printf("E=%d V=%d C=%d → Cycles=%d\n", g->edges, g->vertices, C, rank); return rank; } /* Check if graph is fully cyclic (no tree edges) */ int isCyclic(Graph *g) { return detectCycle(g); }
C — Remove Back Edge (Break Cycle)
/* Find back edge and delete it */ void removeCycleEdge(Graph *g) { int vis[MAX_V]={0}; /* Find back edge using DFS */ for(int u=0;u<g->vertices;u++) { if(vis[u]) continue; int adj[MAX_V],cnt; getAdjacent(g,u,adj,&cnt); vis[u]=1; for(int i=0;i<cnt;i++) { if(vis[adj[i]]) { /* Found back edge, delete it */ int e=findEdge(g,u,adj[i]); if(e>=0) { deleteEdge(g,e); printf("Cycle edge removed!\n"); return; } } } } }
Cyclomatic Complexity = E - V + 2P (for programs) Tree has 0 cycles: E = V - 1
Degree Operations
Find Degree, In-Degree & Out-Degree
The degree of a vertex = number of edges incident to it = count of non-zero entries in its row of the incidence matrix. For directed graphs, in-degree counts incoming edges (-1 entries), out-degree counts outgoing (+1 entries).
C — Degree Calculations
/* Undirected degree = row sum of nonzeros */ int getDegree(Graph *g, int v) { int deg = 0; for(int e=0;e<g->edges;e++) if(g->matrix[v][e] != 0) deg++; printf("degree(%s) = %d\n", g->label[v], deg); return deg; } /* In-degree (directed): count -1 entries */ int inDegree(Graph *g, int v) { int deg = 0; for(int e=0;e<g->edges;e++) if(g->matrix[v][e] == -1) deg++; return deg; } /* Out-degree (directed): count +1 entries */ int outDegree(Graph *g, int v) { int deg = 0; for(int e=0;e<g->edges;e++) if(g->matrix[v][e] == 1) deg++; return deg; }
Degree Summary
  • ∑Sum of all degrees = 2 × E (Handshaking Lemma)
  • +Out-degree: edges going OUT (matrix[v][e] = +1)
  • −In-degree: edges coming IN (matrix[v][e] = -1)
  • 0Degree 0 = Isolated vertex
  • 1Degree 1 = Leaf node (pendant vertex)

Degree Operations
Maximum & Minimum Degree
Scan all vertices, compute each degree, track the max and min. The vertex with highest degree is the most connected hub.
C — Max and Min Degree
void maxMinDegree(Graph *g) { int maxD=0, minD=999999; int maxV=0, minV=0; for(int i=0; i<g->vertices; i++) { int d = getDegree(g, i); if(d > maxD) { maxD=d; maxV=i; } if(d < minD) { minD=d; minV=i; } } printf("Max degree: %s = %d\n", g->label[maxV], maxD); printf("Min degree: %s = %d\n", g->label[minV], minD); }
✈️ Real-Life Analogy
In an airline network, the airport with the most flights (highest degree) is the main hub. The one with fewest flights is a regional airport.
Structural Operations
Graph Size & Order
Order = number of vertices (V). Size = number of edges (E). These two numbers define the scale of a graph.
C — Size and Order
/* Order = V (number of rows in matrix) */ int graphOrder(Graph *g) { printf("Order (V) = %d\n",g->vertices); return g->vertices; } /* Size = E (number of columns) */ int graphSize(Graph *g) { printf("Size (E) = %d\n",g->edges); return g->edges; }
Important Formulas
• Max edges in undirected: V(V-1)/2
• Max edges in directed: V(V-1)
• Tree: E = V - 1
• Density = E / max_E
Order = |V| Size = |E|
Structural Operations
Density, Sparse & Dense Check
Density = actual edges / max possible edges. Dense graphs have density close to 1.0; sparse graphs have density close to 0. For incidence matrix, sparse graphs waste a lot of memory.
C — Graph Density
float graphDensity(Graph *g) { int V = g->vertices; int E = g->edges; if(V <= 1) return 0.0; /* Max edges for undirected graph */ float maxE = (float)(V*(V-1)) / 2.0; float density = E / maxE; printf("Density = %.4f\n", density); if(density < 0.3) printf("→ SPARSE graph\n"); else printf("→ DENSE graph\n"); return density; }
Sparse vs Dense
Sparse: Few edges relative to V². Social networks, road maps.

Dense: Many edges relative to V². Fully connected networks, neural layers.

Tip: For sparse graphs, adjacency lists are more efficient than incidence matrix!

Structural Operations
Complete Graph Check
A complete graph Kₙ has every vertex connected to every other vertex. E = V(V-1)/2 for undirected. Every vertex has degree V-1.
C — Is Complete Graph?
int isCompleteGraph(Graph *g) { int V = g->vertices; int expectedE = V*(V-1)/2; if(g->edges != expectedE) { printf("Not complete: E=%d, need %d\n", g->edges, expectedE); return 0; } /* Also verify each degree = V-1 */ for(int i=0;i<V;i++) { if(getDegree(g,i) != V-1) { printf("Vertex %d degree wrong!\n",i); return 0; } } printf("Graph is COMPLETE (K%d)!\n", V); return 1; }
Complete Graph Facts
K₁ = 1 vertex, 0 edges
K₂ = 2 vertices, 1 edge
K₃ = 3 vertices, 3 edges (triangle)
K₄ = 4 vertices, 6 edges
K₅ = 5 vertices, 10 edges

Formula: E = V(V-1)/2
Structural Operations
Tree Validation
A graph is a tree if it is connected AND acyclic (no cycles). Equivalently: connected with exactly V-1 edges.
C — Is Tree?
int isTree(Graph *g) { /* Condition 1: E = V - 1 */ if(g->edges != g->vertices - 1) { printf("Not a tree: E(%d) != V-1(%d)\n", g->edges, g->vertices-1); return 0; } /* Condition 2: Graph is connected */ if(!isConnected(g)) { printf("Not a tree: disconnected!\n"); return 0; } /* Condition 3 (implicit): No cycles */ /* Already guaranteed by 1 + 2 */ printf("Graph IS a valid tree!\n"); return 1; }
🌲 Real-Life Analogy
A family tree. No circular family relationships (no cycles), and every family member is connected through ancestry (connected). Exactly N-1 parent-child links for N people.
Advanced Algorithms
Minimum Spanning Tree
An MST is a subset of edges that connects all vertices with minimum total weight and no cycles. Two famous algorithms: Kruskal's (sort edges) and Prim's (grow tree).
MST Properties
  • 1Contains exactly V-1 edges
  • 2Connects ALL vertices
  • 3NO cycles
  • 4Minimum total weight among all spanning trees
  • 5Not necessarily unique (ties in weight)
Kruskal: O(E log E) Prim: O(V²) or O(E log V)

Advanced Algorithms
Kruskal's Algorithm
Sort all edges by weight. Add each edge greedily if it doesn't create a cycle (using Union-Find). Stop when V-1 edges are added.
C — Kruskal's Algorithm
/* Union-Find (Disjoint Set) helpers */ int parent[MAX_V]; int find(int x) { if(parent[x] != x) parent[x] = find(parent[x]); return parent[x]; } void unite(int a, int b) { parent[find(a)] = find(b); } /* Edge list extracted from incidence matrix */ typedef struct { int u,v,w; } Edge; void kruskal(Graph *g, int weights[]) { Edge edges[MAX_E]; int n=0; /* Extract edges from incidence matrix */ for(int e=0;e<g->edges;e++) { int u=-1,v=-1; for(int i=0;i<g->vertices;i++) if(g->matrix[i][e]==1) { if(u==-1) u=i; else v=i; } edges[n++] = (Edge){u,v,weights[e]}; } /* Sort edges by weight (bubble sort) */ for(int i=0;i<n-1;i++) for(int j=0;j<n-i-1;j++) if(edges[j].w>edges[j+1].w) { Edge tmp=edges[j]; edges[j]=edges[j+1]; edges[j+1]=tmp; } /* Init Union-Find */ for(int i=0;i<g->vertices;i++) parent[i]=i; printf("MST (Kruskal):\n"); int total=0, count=0; for(int i=0;i<n;i++) { int pu=find(edges[i].u); int pv=find(edges[i].v); if(pu != pv) { /* no cycle */ unite(edges[i].u, edges[i].v); printf(" %s - %s (w=%d)\n", g->label[edges[i].u], g->label[edges[i].v], edges[i].w); total += edges[i].w; if(++count == g->vertices-1) break; } } printf("Total MST weight: %d\n", total); }
Advanced Algorithms
Prim's Algorithm
Start from one vertex. Repeatedly add the cheapest edge that connects the current MST to a new vertex. Uses a visited array and minimum edge tracking.
C — Prim's Algorithm
void prim(Graph *g, int weights[]) { int inMST[MAX_V]={0}; int minW[MAX_V], parent2[MAX_V]; for(int i=0;i<g->vertices;i++) { minW[i]=999999; parent2[i]=-1; } minW[0] = 0; /* Start from vertex 0 */ for(int iter=0;iter<g->vertices;iter++){ /* Pick min weight vertex not in MST */ int u=-1; for(int i=0;i<g->vertices;i++) if(!inMST[i]&&(u==-1|| minW[i]<minW[u])) u=i; inMST[u] = 1; /* Update adjacent vertices */ for(int e=0;e<g->edges;e++) { if(g->matrix[u][e]!=0) { for(int v=0;v<g->vertices;v++){ if(v!=u&&g->matrix[v][e]!=0&& !inMST[v]&& weights[e]<minW[v]) { minW[v]=weights[e]; parent2[v]=u; } } } } } printf("MST (Prim):\n"); int total=0; for(int i=1;i<g->vertices;i++) { printf(" %s - %s (w=%d)\n", g->label[parent2[i]], g->label[i], minW[i]); total += minW[i]; } printf("Total: %d\n", total); }
Advanced Algorithms
Strongly Connected Components (Kosaraju)
Find all maximal subgraphs where every vertex can reach every other vertex. Uses Kosaraju's two-pass DFS algorithm.
C — Kosaraju's SCC Algorithm
/* Pass 1: DFS and record finish order */ int finishStack[MAX_V], fTop=0; void dfsFinish(Graph *g, int u, int vis[]) { vis[u]=1; int adj[MAX_V],cnt; getAdjacent(g,u,adj,&cnt); for(int i=0;i<cnt;i++) if(!vis[adj[i]]) dfsFinish(g,adj[i],vis); finishStack[fTop++]=u; /* push on finish */ } /* Pass 2: DFS on transposed graph */ void dfsTranspose(Graph *g, int u, int vis[], int comp) { vis[u]=1; printf("%s ",g->label[u]); /* Traverse INCOMING edges (reversed) */ for(int e=0;e<g->edges;e++) { if(g->matrix[u][e]==-1) { /* was head */ for(int v=0;v<g->vertices;v++) if(v!=u&&g->matrix[v][e]==1 &&!vis[v]) dfsTranspose(g,v,vis,comp); } } } void kosarajuSCC(Graph *g) { int vis[MAX_V]={0}; fTop=0; /* Pass 1 */ for(int i=0;i<g->vertices;i++) if(!vis[i]) dfsFinish(g,i,vis); /* Pass 2: process in reverse finish order */ int vis2[MAX_V]={0}, scc=0; while(fTop>0) { int u=finishStack[--fTop]; if(!vis2[u]) { printf("SCC %d: ",++scc); dfsTranspose(g,u,vis2,scc); printf("\n"); } } }
🌐 Real-Life Analogy
In a social network with follow relationships (directed), an SCC is a group where everyone follows everyone else directly or indirectly — a tight-knit community.
Advanced Algorithms
Graph Coloring
Assign colors to vertices so no two adjacent vertices share the same color. The minimum number of colors needed is the chromatic number χ(G). Greedy coloring gives a good (not always minimum) result.
C — Greedy Graph Coloring
void greedyColoring(Graph *g) { int color[MAX_V]; for(int i=0;i<g->vertices;i++) color[i] = -1; /* uncolored */ color[0] = 0; /* First vertex = color 0 */ int avail[MAX_V]; for(int u=1; u<g->vertices; u++) { /* Mark neighbor colors as unavailable */ for(int c=0;c<g->vertices;c++) avail[c]=1; int adj[MAX_V],cnt; getAdjacent(g,u,adj,&cnt); for(int i=0;i<cnt;i++) if(color[adj[i]] != -1) avail[color[adj[i]]]=0; /* Pick first available color */ for(int c=0;c<g->vertices;c++) { if(avail[c]) { color[u]=c; break; } } } /* Print coloring result */ int maxColor=0; for(int i=0;i<g->vertices;i++) { printf("%s → Color %d\n", g->label[i], color[i]); if(color[i]>maxColor) maxColor=color[i]; } printf("Colors used: %d\n",maxColor+1); }
Coloring Applications
• Exam scheduling: Students = vertices, shared students = edges. Minimize time slots (colors).
• Register allocation: Variables = vertices, simultaneous = edges. Minimize CPU registers.
• Map coloring: Countries = vertices, borders = edges. 4 colors always suffice (4-Color Theorem)!
