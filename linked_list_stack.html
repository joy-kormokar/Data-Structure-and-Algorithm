What is a Stack?

📚 Theory

A Stack is a linear data structure that follows the LIFO (Last In, First Out) principle. The last element added is the first one to be removed — just like a stack of plates.


In a linked-list based stack, each element is a node containing data and a pointer to the next node. The top pointer always points to the most recently added node.


→No fixed size — grows dynamically
→Each node stores: data + next pointer
→top = NULL means stack is empty
→No overflow (unlike arrays)
🏗️ Node Structure in C
node.cC
/* Every node holds data + pointer to next */
struct Node {
    int  data;
    struct Node *next;
};

/* top points to top of stack */
struct Node *top = NULL;
🍽️
Real Life: Think of a stack of dinner plates. You can only add or remove a plate from the top. You can't take one from the middle — that's exactly how a stack works!

Push — Add to Top

📖 Theory

Push adds a new element at the top of the stack. In a linked list, we:

1Create a new node with the given data
2Set new node's next = current top
3Move top pointer to new node

Time Complexity: O(1) — always constant time!

📥
Analogy: Like placing a new tray on the TOP of a cafeteria tray stack. The new tray becomes the first one you'd pick up next.
push.cC
struct Node *createNode(int val) {
    struct Node *n = (struct Node*)
        malloc(sizeof(struct Node));
    n->data = val;
    n->next = NULL;
    return n;
}

void push(int val) {
    struct Node *n = createNode(val);
    n->next = top;   /* link to old top */
    top    = n;      /* new top = new node */
    printf("Pushed: %d\n", val);
}

02

Pop — Remove Top

📖 Theory

Pop removes the top element from the stack. In a linked list:

1Check if stack is empty (top == NULL)
2Save the top node in a temp pointer
3Move top = top->next
4Free the old top node's memory

Time Complexity: O(1)

📤
Analogy: Like removing the topmost book from a pile. You can only take the one on top — not from the middle!
pop.cC
void pop() {
    if (top == NULL) {
        printf("Stack Underflow!\n");
        return;
    }
    struct Node *temp = top;
    int val = temp->data;
    top = top->next;  /* move top down */
    free(temp);       /* release memory */
    printf("Popped: %d\n", val);
}

Peek / Top — View Without Remove

📖 Theory

Peek returns the top element without removing it. It simply reads top->data without changing the stack structure.


1Check if stack is empty
2Return top->data (no modification)

Time Complexity: O(1)

👀
Analogy: Looking at the top card of a deck without drawing it. You see it, but it stays in place.
peek.cC
int peek() {
    if (top == NULL) {
        printf("Stack is empty!\n");
        return -1;
    }
    return top->data; /* no removal */
}

/* Usage */
printf("Top = %d\n", peek());

isEmpty — Check if Stack Empty

📖 Theory

isEmpty checks whether the stack has any elements. In a linked-list stack, the stack is empty when top == NULL.


This is used as a guard before pop and peek operations to prevent underflow errors.


Time Complexity: O(1)

📭
Analogy: Checking if your email inbox is empty before reading. If it's empty, you don't open it expecting a message!

int isEmpty() {
    return (top == NULL); /* 1=empty, 0=not */
}

/* Usage */
if (isEmpty()) {

isFull — Check Stack Capacity

📖 Theory

In a linked-list based stack, isFull checks if system memory is exhausted (malloc returns NULL). Unlike arrays, there's no fixed capacity limit.


In practice, the stack is "full" only when the system runs out of heap memory — which is very rare.


Array Stack → Fixed Size
Linked List Stack → Dynamic
💾
Analogy: A linked-list stack is like a notepad that grows with more pages. It's only "full" when you've literally used all the paper in the world!
isfull.cC
/* Linked list: check malloc success */
int isFull() {
    struct Node *test =
        (struct Node*)malloc(
            sizeof(struct Node));
    if (test == NULL) {
        return 1; /* memory exhausted */
    }
    free(test);
    return 0; /* can still push */
}

Size / Count — Number of Elements

📖 Theory

Size traverses the linked list from top to NULL, counting each node.


1Start at top, count = 0
2While node != NULL, count++
3Move to next node
4Return count

Time Complexity: O(n) — must traverse all nodes.

size.cC
int size() {
    int count = 0;
    struct Node *curr = top;
    while (curr != NULL) {
        count++;
        curr = curr->next;
    }
    return count;
}

printf("Stack size: %d\n", size());

Traverse / Display — Print All Elements

📖 Theory

Traverse visits every node from top to the bottom and prints each element. The stack is NOT modified.


Order: Top → Bottom (opposite of insertion order)


Time: O(n)  |  Space: O(1)

📋
Analogy: Like reading the titles of books stacked on a table — from the topmost book down to the bottom, without moving any.
traverse.cC
void display() {
    if (top == NULL) {
        printf("Stack empty!\n");
        return;
    }
    struct Node *curr = top;
    printf("top → ");
    while (curr != NULL) {
        printf("%d → ", curr->data);
        curr = curr->next;
    }
    printf("NULL\n");
}

Search — Find an Element

📖 Theory

Search linearly scans from the top of the stack downward, comparing each node's data with the target value.


1Start at top, position = 1
2Compare current node data with target
3If found → return position
4Move next, position++
5Return -1 if not found

Time: O(n)

search.cC
int search(int target) {
    struct Node *curr = top;
    int pos = 1;
    while (curr != NULL) {
        if (curr->data == target) {
            printf("Found at pos %d\n", pos);
            return pos;
        }
        curr = curr->next;
        pos++;
    }
    printf("Not found!\n");
    return -1;
}

Clear — Remove All Elements

📖 Theory

Clear removes ALL elements from the stack by repeatedly popping until empty. Frees all allocated memory to prevent memory leaks.


Time: O(n) — must free each node individually.


⚠️ Always free memory in C!
🗑️
Analogy: Emptying your entire to-do list at once — every item goes, the board is wiped clean.
clear.cC
void clear() {
    while (top != NULL) {
        struct Node *temp = top;
        top = top->next;
        free(temp); /* free each node */
    }
    printf("Stack cleared!\n");
    /* top is now NULL */
}

Copy / Clone — Duplicate the Stack

📖 Theory

Cloning creates a completely new stack with the same elements in the same order. Since stacks reverse on push, we need an auxiliary stack.

1→Pop all from original into temp stack
2→Push from temp into both original and copy

Time: O(n)  |  Space: O(n)

copy.cC
struct Node* copyStack(struct Node *src) {
    struct Node *temp = NULL;
    struct Node *copy = NULL;
    struct Node *cur  = src;
    /* step 1: reverse into temp */
    while (cur) {
        struct Node *n = createNode(cur->data);
        n->next = temp; temp = n;
        cur = cur->next;
    }
    /* step 2: restore src, build copy */
    while (temp) {
        struct Node *n1 = createNode(temp->data);
        n1->next = src; src = n1;
        struct Node *n2 = createNode(temp->data);
        n2->next = copy; copy = n2;
        temp = temp->next;
    }
    return copy;
}

Reverse Stack — Flip the Order

📖 Theory

Reversing a stack flips the order so the bottom becomes the top. Method: pop all into a temp stack, then pop temp back to original (this double-reversal gives correct order).


Time: O(n)  |  Space: O(n)

🔄
Analogy: Like rewinding a video — the last frame becomes the first. Or reversing a queue of people: the last person now goes first.
reverse.cC
void reverseStack() {
    struct Node *temp = NULL;
    /* pop all into temp (reverses) */
    while (top) {
        struct Node *n = top;
        top = top->next;
        n->next = temp;
        temp = n;
    }
    top = temp; /* now reversed */
}

Swap Top Elements — Exchange Top Two

📖 Theory

Swap exchanges the values (or positions) of the top two elements without modifying the rest of the stack.

1Check stack has ≥ 2 elements
2Store top->data in a temp variable
3top->data = top->next->data
4top->next->data = temp

Time: O(1)

swap.cC
void swapTop() {
    if (!top || !top->next) {
        printf("Need >= 2 elements!\n");
        return;
    }
    int temp    = top->data;
    top->data   = top->next->data;
    top->next->data = temp;
    printf("Top two swapped!\n");
}

Sort Stack — Arrange in Order

📖 Theory

Sort a stack using only stack operations (no arrays). The approach: pop element, then insert it in the correct sorted position using a helper function.


Time: O(n²)  |  Space: O(n) recursive stack


Ascending: smallest on top
Descending: largest on top
🔢
Analogy: Sorting a stack of numbered plates by placing each one in the right spot among those already placed.
sort.cC
/* Insert val in sorted position */
void sortedInsert(int val) {
    if (!top || top->data <= val) {
        push(val); return;
    }
    int tmp = top->data;
    pop();
    sortedInsert(val);
    push(tmp);
}

/* Sort entire stack */
void sortStack() {
    if (!top) return;
    int tmp = top->data;
    pop();
    sortStack();
    sortedInsert(tmp);
}

Merge Stacks — Combine Two Stacks

📖 Theory

Merging combines two stacks into one. The simplest method appends stack2 on top of stack1 by finding the bottom of stack2 and linking it to top of stack1.


Time: O(n) where n = size of stack2

🤝
Analogy: Combining two piles of papers into one. The second pile goes on top of the first pile.
merge.cC
/* Append s2 on top of s1 */
struct Node* mergeStacks(
    struct Node *s1,
    struct Node *s2)
{
    if (!s2) return s1;
    /* find bottom of s2 */
    struct Node *cur = s2;
    while (cur->next) cur = cur->next;
    cur->next = s1; /* link bottom to s1 */
    return s2;      /* s2 is new top */
}

Split Stack — Divide Into Parts

📖 Theory

Split divides a stack at a given position into two separate stacks. The top half goes to stack1, the bottom half to stack2.

1Traverse to the split position
2Cut the link at that node
3The two halves are now separate stacks

Time: O(k) where k = split position

split.cC
/* Split at position k (1-indexed from top) */
void splitStack(
    struct Node *src, int k,
    struct Node **s1,
    struct Node **s2)
{
    *s1 = src;
    struct Node *cur = src;
    int i;
    for (i=1; i<k && cur; i++)
        cur = cur->next;
    if (cur) {
        *s2 = cur->next;
        cur->next = NULL; /* cut link */
    }
}

Recursive Stack Operations — Reverse & Sort with Recursion

📖 Theory — Recursive Reverse

We can reverse a stack recursively by:

1Pop top element, store it
2Recursively reverse the remaining stack
3Insert saved element at the bottom

The call stack itself acts as auxiliary storage — no extra data structure needed!


Time: O(n²)  |  Space: O(n) call stack

recursive.cC
/* Insert at bottom of stack */
void insertBottom(int val) {
    if (!top) { push(val); return; }
    int tmp = top->data;
    pop();
    insertBottom(val); /* recurse */
    push(tmp);
}

/* Reverse using recursion */
void reverseRecursive() {
    if (!top) return;
    int tmp = top->data;
    pop();
    reverseRecursive(); /* recurse */
    insertBottom(tmp); /* insert at base */
}

Evaluate Expressions — Postfix / Prefix Evaluation

📖 Theory — Postfix Evaluation

Postfix (Reverse Polish) notation has operators AFTER operands. Stack makes this simple:

1Scan left to right
2If operand → push onto stack
3If operator → pop 2 values, apply operator, push result
4Final stack top = answer

Example: 5 3 + 2 * → (5+3)*2 = 16

postfix_eval.cC
int evalPostfix(char *expr) {
    int i;
    for (i=0; expr[i]; i++) {
        if (isdigit(expr[i])) {
            push(expr[i] - '0');
        } else if (expr[i] != ' ') {
            int b = pop(), a = pop();
            switch (expr[i]) {
              case '+': push(a+b); break;
              case '-': push(a-b); break;
              case '*': push(a*b); break;
              case '/': push(a/b); break;
            }
        }
    }
    return pop();
}

infix_to_postfix.cC
int prec(char op) {
    if(op=='^') return 3;
    if(op=='*'||op=='/') return 2;
    if(op=='+'||op=='-') return 1;
    return 0;
}
void infixToPostfix(char *e, char *out){
    int i=0,j=0; char stk[100]; int t=-1;
    for(;e[i];i++){
        if(isalnum(e[i])) out[j++]=e[i];
        else if(e[i]=='(') stk[++t]=e[i];
        else if(e[i]==')'){
            while(t>=0&&stk[t]!='(')
                out[j++]=stk[t--];
            t--; /* pop '(' */
        } else {
            while(t>=0&&prec(stk[t])>=prec(e[i]))
                out[j++]=stk[t--];
            stk[++t]=e[i];
        }
    }
    while(t>=0) out[j++]=stk[t--];
    out[j]='\0';
}

Balanced Parentheses — Validate Brackets

📖 Theory

Check if every opening bracket has a matching closing bracket in the correct order.

1Scan each character left to right
2Opening bracket ( [ { → push
3Closing bracket → pop and check match
4At end: empty stack = balanced ✓

Time: O(n)  |  Space: O(n)

🔑
Analogy: Every door you open (open bracket) must be properly closed (close bracket). If you leave a door open or close a door you didn't open — it's invalid!
balanced.cC
int isMatch(char o, char c) {
    return (o=='('&&c==')') ||
           (o=='['&&c==']') ||
           (o=='{'&&c=='}');
}
int isBalanced(char *s) {
    char stk[100]; int t=-1, i;
    for(i=0;s[i];i++) {
        if(s[i]=='('||s[i]=='['||s[i]=='{')
            stk[++t]=s[i];
        else if(s[i]==')'||s[i]==']'||s[i]=='}') {
            if(t<0||!isMatch(stk[t],s[i]))
                return 0; /* unbalanced */
            t--;
        }
    }
    return t==-1;
}

Undo / Redo — Two-Stack State Management

📖 Theory

Undo/Redo uses two stacks:


↩Undo Stack: stores history of states. Undo = pop from undo, push to redo.
↪Redo Stack: stores undone states. Redo = pop from redo, push to undo.
✏️New action: push to undo, clear redo stack.

Time: O(1) per operation

📝
Analogy: Ctrl+Z and Ctrl+Y in your text editor. Every keystroke is saved in an undo stack. Ctrl+Z pops and moves it to the redo stack!
undo_redo.cC
char *undoStack[100], *redoStack[100];
int  undoTop=-1, redoTop=-1;

void doAction(char *state) {
    undoStack[++undoTop] = state;
    redoTop = -1; /* clear redo */
}
char* undo() {
    if(undoTop<0) return NULL;
    redoStack[++redoTop]=undoStack[undoTop--];
    return undoTop>=0?undoStack[undoTop]:"";
}
char* redo() {
    if(redoTop<0) return NULL;
    undoStack[++undoTop]=redoStack[redoTop--];
    return undoStack[undoTop];
}

Backtracking — DFS & Maze Solving

📖 Theory — DFS with Stack

Depth-First Search (DFS) uses a stack to explore graph nodes as deep as possible before backtracking.

1Push start node onto stack
2Pop a node, mark as visited
3Push all unvisited neighbors
4Repeat until stack empty

Maze solving uses the same idea — stack keeps track of path. Backtrack when stuck!

dfs.cC
/* DFS using explicit stack */
void DFS(int graph[][6], int V, int start) {
    int visited[6]={0};
    int stk[100], top=-1;
    stk[++top] = start;
    while (top >= 0) {
        int v = stk[top--];
        if (!visited[v]) {
            visited[v] = 1;
            printf("%d ", v);
            int i;
            for(i=V-1; i>=0; i--)
                if(graph[v][i]&&!visited[i])
                    stk[++top]=i;
        }
    }
}
