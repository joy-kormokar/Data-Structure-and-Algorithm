Core Concepts

Terminology every binary tree programmer must know

🧠

Binary Tree Terminology

Theory▶
A Binary Tree is a hierarchical data structure where each node has at most 2 children — called left child and right child.

In a linked-list implementation, each node is a struct containing data and two pointers (left, right). Unlike arrays, nodes are scattered in memory and linked via pointers.
  • Root: The top-most node (no parent)
  • Leaf: Node with no children
  • Internal Node: Node with at least one child
  • Height: Longest path from root to a leaf
  • Depth: Distance from root to a specific node
  • Level: Depth + 1 (root is at level 1)
  • Degree: Number of children (0, 1, or 2)

OPERATION GROUP 1

Basic Operations

Create Node · Initialize Tree · Check Empty · Allocate Memory · Init Pointers

📖

Theory: Create Tree, Create Node, Initialize

Theory▶
Before using a binary tree, we must set up its structure in memory.
  • Create Node: Dynamically allocate a new node using malloc(), set its data, and set both children to NULL.
  • Initialize Tree: The tree is represented by a single pointer to its root. Initially root = NULL.
  • Check Empty: A tree is empty if root == NULL.
  • Create Root Node: The very first node inserted becomes the root.
  • Allocate Memory: Done with malloc(sizeof(struct Node)).
  • Initialize Left/Right Pointer: Always set to NULL to prevent garbage pointer bugs.
⚠️ Rule #1: Always initialize left and right pointers to NULL after allocation. Uninitialized pointers cause segmentation faults!

// basic_operations.c
#include <stdio.h>
#include <stdlib.h>

/* ── Step 1: Define the Node structure ── */
struct Node {
    int data;
    struct Node* left;
    struct Node* right;
};

/* ── Step 2: Create a new node ── */
struct Node* createNode(int value) {
    /* Allocate memory on the heap */
    struct Node* newNode = (struct Node*) malloc(sizeof(struct Node));
    
    /* Always check if malloc succeeded */
    if (newNode == NULL) {
        printf("Memory allocation failed!\n");
        exit(1);
    }
    
    /* Initialize the node */
    newNode->data  = value;   /* set data */
    newNode->left  = NULL;    /* init left pointer  */
    newNode->right = NULL;    /* init right pointer */
    
    return newNode;
}

/* ── Step 3: Initialize an empty tree ── */
void initTree(struct Node** root) {
    *root = NULL;   /* empty tree = NULL root */
}

/* ── Step 4: Check if tree is empty ── */
int isEmpty(struct Node* root) {
    return (root == NULL);   /* 1=empty, 0=not empty */
}

/* ── Step 5: Create root node ── */
struct Node* createRoot(int value) {
    return createNode(value);  /* root is just the first node */
}

/* ── main: putting it all together ── */
int main() {
    struct Node* root = NULL;
    initTree(&root);
    
    printf("Is empty? %d\n", isEmpty(root)); /* 1 */
    
    root = createRoot(10);
    root->left  = createNode(5);
    root->right = createNode(15);
    
    printf("Root: %d\n", root->data);       /* 10 */
    printf("Left: %d\n", root->left->data); /* 5  */
    printf("Right: %d\n", root->right->data);/* 15 */
    
    return 0;
}

Creating a node is like buying a new house lot: malloc() purchases the land (heap memory), setting data is writing the house number, and setting left/right to NULL means the lot has no neighboring houses connected yet.

OPERATION GROUP 2

Insertion Operations

Insert Root · Left Child · Right Child · Specific Position · Recursive · Iterative · Level Order

📖

Theory: All Insertion Types Explained

Theory▶
Inserting means adding a new node to the tree. Several strategies exist:
  • Insert Root Node: If tree is empty, the new node becomes root.
  • Insert Left Child: Place node as the left child of a target node.
  • Insert Right Child: Place node as the right child of a target node.
  • Level Order Insertion: Insert at the first available position level by level (BFS). This keeps the tree complete. Most beginner-friendly.
  • Recursive Insertion: Use recursion to find the right place.
  • Iterative Insertion: Use a queue (for level-order) without recursion.
💡 Level Order Insertion is the most common approach for a general binary tree (not BST). It fills nodes left-to-right, level by level, keeping the tree as complete as possible.

// insertion_operations.c
#include <stdio.h>
#include <stdlib.h>

struct Node { int data; struct Node *left, *right; };

struct Node* createNode(int v){
    struct Node* n=(struct Node*)malloc(sizeof(struct Node));
    n->data=v; n->left=n->right=NULL; return n;
}

/* ── Method 1: Insert as left child of a node ── */
void insertLeft(struct Node* parent, int val) {
    if (parent->left != NULL) {
        printf("Left child already exists!\n");
        return;
    }
    parent->left = createNode(val);
}

/* ── Method 2: Insert as right child of a node ── */
void insertRight(struct Node* parent, int val) {
    if (parent->right != NULL) {
        printf("Right child already exists!\n");
        return;
    }
    parent->right = createNode(val);
}

/* ── Method 3: Level Order Insertion (BFS) ─────
   Inserts at the first empty spot, level by level.
   Uses a simple queue (array-based). ── */
struct Node* insertLevelOrder(struct Node* root, int val) {
    struct Node* newNode = createNode(val);
    
    if (root == NULL) return newNode; /* first node = root */
    
    /* BFS queue */
    struct Node* queue[100];
    int front = 0, rear = 0;
    queue[rear++] = root;
    
    while (front < rear) {
        struct Node* curr = queue[front++];
        
        if (curr->left == NULL) {   /* found empty left spot */
            curr->left = newNode;
            return root;
        } else {
            queue[rear++] = curr->left;
        }
        
        if (curr->right == NULL) {  /* found empty right spot */
            curr->right = newNode;
            return root;
        } else {
            queue[rear++] = curr->right;
        }
    }
    return root;
}

int main() {
    struct Node* root = NULL;
    
    /* Build: 1 → 2 → 3 → 4 → 5 (level order) */
    for (int i = 1; i <= 5; i++)
        root = insertLevelOrder(root, i);
    
    printf("Root=%d Left=%d Right=%d\n",
           root->data, root->left->data, root->right->data);
    return 0;
}
🌍 Real-Life Analogy
Level Order Insertion is like filling seats in a cinema row by row — you don't skip to the back row until the front is full. Every level is completely filled left-to-right before going deeper.

OPERATION GROUP 3

Deletion Operations

Delete Leaf · Internal Node · Root · Deepest Node · Replace Deepest · Recursive/Iterative Delete

📖

Theory: How Deletion Works

Theory▶
Deletion in a general binary tree follows a clever strategy:
  1. Find the node to delete
  2. Find the deepest rightmost node in the tree
  3. Replace the target node's data with the deepest node's data
  4. Delete the deepest rightmost node (which is now a leaf)
This keeps the tree's structure intact.
  • Delete Leaf Node: Simply set parent's pointer to NULL and free memory.
  • Delete Internal Node: Use the replace-deepest strategy above.
  • Delete Root Node: Same as internal node — replace with deepest, then free deepest.
  • Delete Tree: Use postorder traversal to free all nodes.
  • Delete Deepest Node: BFS to find & remove the last-level rightmost node.
⚠️ Always free() the deleted node to prevent memory leaks!

// deletion_operations.c
#include <stdio.h>
#include <stdlib.h>

struct Node { int data; struct Node *left, *right; };

/* ── Find deepest rightmost node ── */
void findDeepest(struct Node* root,
                  struct Node** deepNode,
                  struct Node** deepParent) {
    struct Node* queue[100];
    struct Node* parent[100];
    int front=0, rear=0;
    queue[rear]=root; parent[rear++]=NULL;
    
    while (front < rear) {
        *deepNode   = queue[front];
        *deepParent = parent[front++];
        if ((*deepNode)->left) { queue[rear]=(*deepNode)->left; parent[rear++]=*deepNode; }
        if ((*deepNode)->right){ queue[rear]=(*deepNode)->right;parent[rear++]=*deepNode; }
    }
}

/* ── Delete a node by value (general binary tree) ── */
struct Node* deleteNode(struct Node* root, int val) {
    if (root == NULL) return NULL;
    if (root->left == NULL && root->right == NULL) {
        if (root->data == val) { free(root); return NULL; }
        return root;
    }
    
    struct Node *deepNode=NULL, *deepParent=NULL, *targetNode=NULL;
    
    /* BFS to find both the target and deepest node */
    struct Node* queue[100]; int f=0,r=0;
    queue[r++]=root;
    while(f<r){
        struct Node* c=queue[f++];
        if(c->data==val) targetNode=c;
        if(c->left)  queue[r++]=c->left;
        if(c->right) queue[r++]=c->right;
        deepNode=c;
    }
    
    if(targetNode) {
        /* Replace target's data with deepest node's data */
        targetNode->data = deepNode->data;
        /* Now delete the deepest node */
        findDeepest(root, &deepNode, &deepParent);
        if(deepParent->right == deepNode) deepParent->right=NULL;
        else                              deepParent->left =NULL;
        free(deepNode);
    }
    return root;
}

/* ── Delete entire tree (postorder) ── */
void deleteTree(struct Node* root) {
    if (root == NULL) return;
    deleteTree(root->left);
    deleteTree(root->right);
    free(root);   /* free AFTER freeing children */
}
🌍 Real-Life Analogy
Deleting a node is like removing a player from an organizational chart. Instead of reorganizing everyone, you copy the name from the last-added (deepest) employee into the deleted spot, then remove the duplicate at the bottom.

OPERATION GROUP 4

Searching Operations

DFS Search · BFS Search · Recursive Search · Iterative Search · Exact Match

📖

Theory: DFS vs BFS Search

Theory▶
Two main ways to search a binary tree:
  • DFS (Depth-First Search): Go deep first. Uses a stack (or recursion). Visits one full branch before backtracking. Good for finding deeply nested nodes.
  • BFS (Breadth-First Search): Go wide first. Uses a queue. Visits all nodes level by level. Good for finding nodes close to root.
  • Recursive Search: Check root, then recursively search left and right subtrees.
  • Iterative Search: Use an explicit stack (DFS) or queue (BFS) instead of recursion.
  • Exact Match Search: Returns true/false — does this value exist?
💡 For a general binary tree (not BST), you must check every node in the worst case — O(n) time complexity.

// searching_operations.c
/* ── Recursive DFS Search ── */
int searchRecursive(struct Node* root, int key) {
    if (root == NULL)        return 0; /* not found */
    if (root->data == key)   return 1; /* found! */
    
    /* Search left subtree, then right subtree */
    return searchRecursive(root->left,  key) ||
           searchRecursive(root->right, key);
}

/* ── Iterative BFS Search (uses queue) ── */
int searchBFS(struct Node* root, int key) {
    if (root == NULL) return 0;
    
    struct Node* queue[100];
    int front = 0, rear = 0;
    queue[rear++] = root;
    
    while (front < rear) {
        struct Node* curr = queue[front++];
        
        if (curr->data == key) return 1; /* found! */
        
        if (curr->left)  queue[rear++] = curr->left;
        if (curr->right) queue[rear++] = curr->right;
    }
    return 0; /* not found */
}

/* ── Iterative DFS Search (uses stack) ── */
int searchDFS(struct Node* root, int key) {
    if (root == NULL) return 0;
    
    struct Node* stack[100];
    int top = -1;
    stack[++top] = root;
    
    while (top >= 0) {
        struct Node* curr = stack[top--];
        
        if (curr->data == key) return 1;
        
        if (curr->right) stack[++top] = curr->right;
        if (curr->left)  stack[++top] = curr->left;
    }
    return 0;
}

/* ── Return the node pointer (exact match) ── */
struct Node* findNode(struct Node* root, int key) {
    if (root == NULL)      return NULL;
    if(root->data == key)  return root;
    struct Node* res = findNode(root->left, key);
    return res ? res : findNode(root->right, key);
}
🌍 Real-Life Analogy
BFS is like searching a building floor by floor. DFS is like exploring one elevator shaft all the way down before checking the next one. Both find the person — the order just differs.

OPERATION GROUP 5

Traversal Operations

Inorder · Preorder · Postorder · Level Order · Recursive & Iterative · Queue/Stack Based

📖

Theory: The Four Traversals Explained

Theory▶
Traversal means visiting every node exactly once in a specific order.
  • Inorder (L → Root → R): Visit left subtree, then root, then right subtree. Gives sorted output for BSTs.
  • Preorder (Root → L → R): Visit root first, then children. Used to copy/serialize a tree.
  • Postorder (L → R → Root): Visit children before root. Used to delete or evaluate expression trees.
  • Level Order: Visit nodes level by level using a queue. Also called BFS traversal.
  • Recursive Traversal: Elegant code — uses the call stack.
  • Iterative Traversal: Uses explicit stack/queue — no recursion.
TraversalOrderKey Use Case
InorderL → N → RSorted output (BST)
PreorderN → L → RCopy, serialize
PostorderL → R → NDelete, eval expressions
Level OrderLevel by levelBFS, shortest path

// traversal_operations.c
/* ── 1. Inorder: Left → Root → Right ── */
void inorder(struct Node* root) {
    if (root == NULL) return;
    inorder(root->left);        /* visit left  */
    printf("%d ", root->data);  /* visit node  */
    inorder(root->right);       /* visit right */
}

/* ── 2. Preorder: Root → Left → Right ── */
void preorder(struct Node* root) {
    if (root == NULL) return;
    printf("%d ", root->data);
    preorder(root->left);
    preorder(root->right);
}

/* ── 3. Postorder: Left → Right → Root ── */
void postorder(struct Node* root) {
    if (root == NULL) return;
    postorder(root->left);
    postorder(root->right);
    printf("%d ", root->data); /* root LAST */
}

/* ── 4. Level Order (BFS) using queue ── */
void levelOrder(struct Node* root) {
    if(!root) return;
    struct Node* q[100]; int f=0,r=0;
    q[r++]=root;
    while(f<r){
        struct Node* c=q[f++];
        printf("%d ",c->data);
        if(c->left)  q[r++]=c->left;
        if(c->right) q[r++]=c->right;
    }
}

/* ── 5. Iterative Inorder (uses stack) ── */
void inorderIterative(struct Node* root) {
    struct Node* stack[100]; int top=-1;
    struct Node* curr = root;
    
    while (curr != NULL || top >= 0) {
        /* Go as left as possible */
        while (curr != NULL) { stack[++top]=curr; curr=curr->left; }
        curr = stack[top--];
        printf("%d ", curr->data); /* visit */
        curr = curr->right;          /* go right */
    }
}

/* ── 6. Iterative Preorder (uses stack) ── */
void preorderIterative(struct Node* root) {
    if(!root) return;
    struct Node* stack[100]; int top=-1;
    stack[++top]=root;
    while(top>=0){
        struct Node* c=stack[top--];
        printf("%d ",c->data);
        if(c->right) stack[++top]=c->right; /* push right first */
        if(c->left)  stack[++top]=c->left;
    }
}
🌍 Real-Life Analogy
Traversal is like reading a book: Preorder = read the chapter title first. Inorder = read content between chapters. Postorder = read the summary last. Level Order = scan headings page by page before reading content.

Queue & Stack Based Traversal

Stack-Based Traversal · Queue-Based Traversal

💻

C Code: Stack-Based vs Queue-Based Traversal

Code▶
Stack-based traversal uses LIFO (Last In First Out) — perfect for DFS (Preorder, Inorder, Postorder).
Queue-based traversal uses FIFO (First In First Out) — perfect for BFS (Level Order).
// stack_queue_traversal.c
/* Simple array-based stack for DFS */
struct Stack { struct Node* arr[100]; int top; };

void push(struct Stack* s, struct Node* n){ s->arr[++s->top]=n; }
struct Node* pop(struct Stack* s){ return s->arr[s->top--]; }
int stackEmpty(struct Stack* s){ return s->top==-1; }

/* Stack-Based DFS Preorder */
void stackBasedPreorder(struct Node* root) {
    if(!root) return;
    struct Stack s; s.top = -1;
    push(&s, root);
    while (!stackEmpty(&s)) {
        struct Node* node = pop(&s);
        printf("%d ", node->data);
        if (node->right) push(&s, node->right);
        if (node->left)  push(&s, node->left);
    }
}

/* Simple array-based queue for BFS */
struct Queue { struct Node* arr[100]; int front, rear; };

void enqueue(struct Queue* q, struct Node* n){ q->arr[q->rear++]=n; }
struct Node* dequeue(struct Queue* q){ return q->arr[q->front++]; }
int queueEmpty(struct Queue* q){ return q->front==q->rear; }

/* Queue-Based Level Order */
void queueBasedLevelOrder(struct Node* root) {
    if(!root) return;
    struct Queue q; q.front=q.rear=0;
    enqueue(&q, root);
    while (!queueEmpty(&q)) {
        struct Node* node = dequeue(&q);
        printf("%d ", node->data);
        if(node->left)  enqueue(&q, node->left);
        if(node->right) enqueue(&q, node->right);
    }
}

OPERATION GROUP 6A

Count Operations

Total Nodes · Leaf Nodes · Internal Nodes · Full Nodes · Half Nodes

📖

Theory: Node Types & Counting

Theory▶
Different node types have distinct properties:
  • Total Nodes: Count all nodes recursively. Base case: NULL = 0.
  • Leaf Nodes: Nodes with left==NULL && right==NULL.
  • Internal Nodes: Nodes with at least one child.
  • Full Nodes: Nodes with exactly two children.
  • Half Nodes: Nodes with exactly one child (left XOR right is non-NULL).
📊 For a perfect binary tree of height h: Total = 2ʰ⁺¹ - 1 | Leaves = 2ʰ | Internal = 2ʰ - 1

C Code: Count Operations

// count_operations.c
/* ── Count ALL nodes ── */
int countTotal(struct Node* root) {
    if (root == NULL) return 0;
    return 1 + countTotal(root->left) + countTotal(root->right);
}

/* ── Count LEAF nodes (no children) ── */
int countLeaves(struct Node* root) {
    if (root == NULL) return 0;
    if (root->left == NULL && root->right == NULL)
        return 1;  /* it's a leaf! */
    return countLeaves(root->left) + countLeaves(root->right);
}

/* ── Count INTERNAL nodes (at least 1 child) ── */
int countInternal(struct Node* root) {
    if (root == NULL) return 0;
    if (root->left == NULL && root->right == NULL)
        return 0;  /* leaf = not internal */
    return 1 + countInternal(root->left) + countInternal(root->right);
}

/* ── Count FULL nodes (exactly 2 children) ── */
int countFull(struct Node* root) {
    if (root == NULL) return 0;
    int curr = (root->left != NULL && root->right != NULL) ? 1 : 0;
    return curr + countFull(root->left) + countFull(root->right);
}

/* ── Count HALF nodes (exactly 1 child) ── */
int countHalf(struct Node* root) {
    if (root == NULL) return 0;
    int hasLeft  = (root->left  != NULL);
    int hasRight = (root->right != NULL);
    int curr = (hasLeft != hasRight) ? 1 : 0; /* XOR */
    return curr + countHalf(root->left) + countHalf(root->right);
}
🌍 Real-Life Analogy
In a company org chart: Leaf nodes = employees with no one reporting to them. Internal nodes = managers. Full nodes = managers with exactly two direct reports. Half nodes = managers with only one report.

OPERATION GROUP 6B

Height, Depth & Level

Find Height · Find Depth · Find Level · Find Maximum Depth

📖

Theory: Height vs Depth vs Level

Theory▶
These three terms are often confused:
  • Height of a node: Length of the longest path from that node down to any leaf. Leaf height = 0. Height of tree = height of root.
  • Depth of a node: Length of the path from the root down to that node. Root depth = 0.
  • Level of a node: Depth + 1. Root is at Level 1.
  • Maximum Depth: The height of the tree (longest path root to leaf).
📐 Height = max(height(left), height(right)) + 1
💻

C Code: Height, Depth, Level

Code▶
// height_depth_level.c
/* ── Find HEIGHT of tree (recursive) ── */
int height(struct Node* root) {
    if (root == NULL) return -1; /* -1 for empty, 0 for leaf */
    int leftH  = height(root->left);
    int rightH = height(root->right);
    return (leftH > rightH ? leftH : rightH) + 1;
}

/* ── Find DEPTH of a specific node ── */
int depth(struct Node* root, int key, int level) {
    if (root == NULL) return -1;
    if (root->data == key) return level;
    
    int d = depth(root->left, key, level + 1);
    if (d != -1) return d;
    return depth(root->right, key, level + 1);
}
/* Usage: depth(root, key, 0) */

/* ── Find LEVEL of a node (Level = Depth + 1) ── */
int findLevel(struct Node* root, int key) {
    int d = depth(root, key, 0);
    return (d == -1) ? -1 : d + 1;
}

/* ── Find MAXIMUM DEPTH (same as height) ── */
int maxDepth(struct Node* root) {
    return height(root); /* max depth = height of tree */
}

int main(){
    /* Build: root=1, left=2, right=3, 2.left=4 */
    struct Node* root = createNode(1);
    root->left = createNode(2);
    root->right = createNode(3);
    root->left->left = createNode(4);

    printf("Height = %d\n", height(root));         /* 2 */
    printf("Depth of 4 = %d\n", depth(root,4,0)); /* 2 */
    printf("Level of 4 = %d\n", findLevel(root,4));/* 3 */
}
🌍 Real-Life Analogy

OPERATION GROUP 6C

Relationship Operations

Find Parent · Find Sibling · Find Ancestor · Find Descendant

💻

C Code: Parent, Sibling, Ancestor, Descendant

Code▶
Parent: The node directly above a given node.
Sibling: Nodes sharing the same parent.
Ancestor: Any node on the path from root to the node.
Descendant: Any node reachable going down from a given node.
// relationships.c
/* ── Find PARENT of a node ── */
struct Node* findParent(struct Node* root, int key) {
    if (root == NULL || root->data == key) return NULL;
    
    /* Check if either child matches */
    if ((root->left  && root->left->data  == key) ||
        (root->right && root->right->data == key))
        return root;
    
    struct Node* res = findParent(root->left, key);
    return res ? res : findParent(root->right, key);
}

/* ── Find SIBLING of a node ── */
struct Node* findSibling(struct Node* root, int key) {
    struct Node* parent = findParent(root, key);
    if (!parent) return NULL; /* root has no sibling */
    
    if (parent->left && parent->left->data == key)
        return parent->right; /* sibling = other child */
    return parent->left;
}

/* ── Check if node A is ANCESTOR of node B ── */
int isAncestor(struct Node* root, int ancestor, int descendant) {
    if (!root) return 0;
    if (root->data == ancestor)
        return searchRecursive(root, descendant);
    return isAncestor(root->left, ancestor, descendant) ||
           isAncestor(root->right, ancestor, descendant);
}

/* ── Print all ANCESTORS of a node ── */
int printAncestors(struct Node* root, int key) {
    if (!root) return 0;
    if (root->data == key) return 1;
    if (printAncestors(root->left, key) ||
        printAncestors(root->right, key)) {
        printf("%d ", root->data); /* this node is an ancestor */
        return 1;
    }
    return 0;
}

/* ── Check if node B is DESCENDANT of A ── */
int isDescendant(struct Node* root, int ancestor, int key) {
    return isAncestor(root, ancestor, key); /* symmetric */
}

OPERATION GROUP 7

Property Checking

Full · Complete · Perfect · Balanced · Identical · Symmetric · Mirror

📖

Theory: Types of Binary Trees

Theory▶
  • Full Binary Tree: Every node has 0 or 2 children (no node has exactly 1 child).
  • Complete Binary Tree: All levels completely filled except possibly the last, which is filled left-to-right.
  • Perfect Binary Tree: All internal nodes have 2 children AND all leaves are at the same level.
  • Balanced Tree: Height difference between left and right subtrees of any node is ≤ 1.
  • Identical Trees: Two trees have same structure and same node values.
  • Symmetric/Mirror Tree: Tree is a mirror image of itself (left side mirrors right side).

// property_checks.c
/* ── Check FULL Binary Tree ── */
int isFullBT(struct Node* root) {
    if (!root || (!root->left && !root->right)) return 1;
    if (root->left && root->right)
        return isFullBT(root->left) && isFullBT(root->right);
    return 0; /* exactly 1 child = not full */
}

/* ── Check COMPLETE Binary Tree ── */
int isCompleteBT(struct Node* root) {
    if (!root) return 1;
    struct Node* q[100]; int f=0,r=0;
    int foundNull = 0;
    q[r++] = root;
    while (f < r) {
        struct Node* c = q[f++];
        if (c->left) {
            if (foundNull) return 0;
            q[r++] = c->left;
        } else foundNull = 1;
        if (c->right) {
            if (foundNull) return 0;
            q[r++] = c->right;
        } else foundNull = 1;
    }
    return 1;
}

/* ── Check PERFECT Binary Tree ── */
int countNodes(struct Node* r){ return !r?0:1+countNodes(r->left)+countNodes(r->right); }
int isPerfectBT(struct Node* root) {
    int h = height(root);
    int n = countNodes(root);
    /* Perfect BT: n = 2^(h+1) - 1 */
    int expected = (1 << (h + 1)) - 1;
    return n == expected;
}

/* ── Check BALANCED Tree (AVL condition) ── */
int isBalanced(struct Node* root) {
    if (!root) return 1;
    int lh = height(root->left);
    int rh = height(root->right);
    int diff = lh - rh;
    if (diff < -1 || diff > 1) return 0;
    return isBalanced(root->left) && isBalanced(root->right);
}

/* ── Check IDENTICAL Trees ── */
int isIdentical(struct Node* a, struct Node* b) {
    if (!a && !b) return 1;
    if (!a || !b) return 0;
    return (a->data == b->data) &&
           isIdentical(a->left,  b->left) &&
           isIdentical(a->right, b->right);
}

/* ── Check SYMMETRIC / MIRROR Tree ── */
int isMirrorHelper(struct Node* l, struct Node* r) {
    if (!l && !r) return 1;
    if (!l || !r) return 0;
    return (l->data == r->data) &&
           isMirrorHelper(l->left,  r->right) &&
           isMirrorHelper(l->right, r->left);
}
int isSymmetric(struct Node* root) {
    return !root || isMirrorHelper(root->left, root->right);
}
int isMirrorTree(struct Node* a, struct Node* b) {
    return isMirrorHelper(a, b);
}

OPERATION GROUP 8

Min/Max Operations

Find Minimum · Find Maximum · Find Leftmost · Find Rightmost Node

📖

Theory + Code: Min, Max, Leftmost, Rightmost

Theory▶
In a general binary tree (not BST), min/max values can be anywhere — we must traverse all nodes.
  • Find Minimum: Traverse all nodes, track smallest value seen.
  • Find Maximum: Same — track largest value.
  • Find Leftmost: Keep going left until no more left child.
  • Find Rightmost: Keep going right until no more right child.
// minmax_operations.c
#include <limits.h>  /* for INT_MAX, INT_MIN */

/* ── Find MINIMUM value in tree ── */
int findMin(struct Node* root) {
    if (!root) return INT_MAX;
    int leftMin  = findMin(root->left);
    int rightMin = findMin(root->right);
    int smallest = root->data;
    if (leftMin  < smallest) smallest = leftMin;
    if (rightMin < smallest) smallest = rightMin;
    return smallest;
}

/* ── Find MAXIMUM value in tree ── */
int findMax(struct Node* root) {
    if (!root) return INT_MIN;
    int leftMax  = findMax(root->left);
    int rightMax = findMax(root->right);
    int largest = root->data;
    if (leftMax  > largest) largest = leftMax;
    if (rightMax > largest) largest = rightMax;
    return largest;
}

/* ── Find LEFTMOST node (go left as far as possible) ── */
struct Node* findLeftmost(struct Node* root) {
    if (!root) return NULL;
    while (root->left != NULL)
        root = root->left;  /* keep going left */
    return root;
}

/* ── Find RIGHTMOST node (go right as far as possible) ── */
struct Node* findRightmost(struct Node* root) {
    if (!root) return NULL;
    while (root->right != NULL)
        root = root->right; /* keep going right */
    return root;
}

int main() {
    /* Build: 5 → (3, 8) → (1, 4, _, 9) */
    struct Node* root = createNode(5);
    root->left = createNode(3);
    root->right = createNode(8);
    root->left->left = createNode(1);
    root->left->right = createNode(4);
    root->right->right = createNode(9);
    
    printf("Min = %d\n", findMin(root));                      /* 1 */
    printf("Max = %d\n", findMax(root));                      /* 9 */
    printf("Leftmost = %d\n", findLeftmost(root)->data);   /* 1 */
    printf("Rightmost = %d\n", findRightmost(root)->data); /* 9 */
}

Finding min/max in a general binary tree is like finding the shortest/tallest person in a crowd — you can't take shortcuts; you must look at everyone. But finding the leftmost node is like always taking the leftmost lane on a highway — you just keep turning left until you can't anymore.

// update_operations.c
/* ── Update (modify) a node's value ── */
int updateNode(struct Node* root, int oldVal, int newVal) {

    if (root->data == oldVal) {
        root->data = newVal;  /* just update the data field */

    if (!root) return;

    replaceAll(root->right, oldVal, newVal);
}

int swapNodeValues(struct Node* root, int val1, int val2) {
    struct Node* node1 = findNode(root, val1);
    struct Node* node2 = findNode(root, val2);
    
    if (!node1 || !node2) return 0; /* not found */
    
    /* Classic swap with temp variable */
    int temp  = node1->data;
    node1->data = node2->data;
    node2->data = temp;
    return 1;
}

void modifyNodeData(struct Node* node, int newData) {

OPERATION GROUP 10

Utility Operations

Copy · Clone · Mirror · Invert · Clear · Destroy · Deep Copy · Recursive Copy/Clear

📖

Theory: Copy vs Clone vs Mirror vs Invert

Theory▶
  • Copy / Clone / Deep Copy: Create a completely new tree with the same structure and values. Both trees exist independently in memory (changing one doesn't affect the other).
  • Mirror Tree: Create a new tree where left and right subtrees are swapped at every node.
  • Invert Tree: Same as mirror but done in-place (modifies the original tree).
  • Clear Tree: Remove all nodes but keep the root pointer (set it to NULL).
  • Destroy Tree: Free all memory and set root to NULL. Same as deleteTree.
💡 Copy = new tree, same data. Mirror = new flipped tree. Invert = flip in place.
💻

C Code: All Utility Operations

Code▶
// utility_operations.c
/* ── Deep Copy / Clone Tree ── */
struct Node* copyTree(struct Node* root) {
    if (!root) return NULL;
    
    /* Create a new node with same data */
    struct Node* newNode = createNode(root->data);
    
    /* Recursively copy both subtrees */
    newNode->left  = copyTree(root->left);
    newNode->right = copyTree(root->right);
    
    return newNode;  /* returns root of new copy */
}
/* cloneTree = same as copyTree */
struct Node* cloneTree(struct Node* r){ return copyTree(r); }

/* ── Invert/Mirror Tree IN PLACE ── */
void invertTree(struct Node* root) {
    if (!root) return;
    
    /* Swap left and right children */
    struct Node* temp = root->left;
    root->left  = root->right;
    root->right = temp;
    
    /* Recursively invert subtrees */
    invertTree(root->left);
    invertTree(root->right);
}
/* mirrorTree = same as invertTree (in-place) */
void mirrorTree(struct Node* r){ invertTree(r); }

/* ── Create a MIRROR (new tree, don't modify original) ── */
struct Node* createMirror(struct Node* root) {
    if (!root) return NULL;
    struct Node* newNode = createNode(root->data);
    /* NOTE: left ↔ right are swapped! */
    newNode->left  = createMirror(root->right);
    newNode->right = createMirror(root->left);
    return newNode;
}

/* ── Clear Tree (free all, set root to NULL) ── */
void clearTree(struct Node** root) {
    destroyTree(*root);  /* free all nodes */
    *root = NULL;         /* reset the pointer */
}

/* ── Destroy Tree (recursive postorder free) ── */
void destroyTree(struct Node* root) {
    if (!root) return;
    destroyTree(root->left);   /* free left first  */
    destroyTree(root->right);  /* free right next  */
    free(root);                 /* free this node   */
}
🌍 Real-Life Analogy
Clone = photocopying a document (two independent copies). Invert = flipping the document upside down (same document, modified). Mirror = scanning and reflecting it (a new mirrored copy). Destroy = shredding all copies — memory returned to system.

OPERATION GROUP 11

Path Operations

Find Path to Node · Print Root-to-Leaf Paths · LCA · Distance Between Nodes

📖

Theory: Paths & LCA Explained

Theory▶
  • Find Path: Find the sequence of nodes from root to a target node. Uses backtracking.
  • Root-to-Leaf Paths: Print all paths from root down to every leaf node.
  • Lowest Common Ancestor (LCA): The deepest node that is an ancestor of both node A and node B. Used in network routing, biology, etc.
  • Distance Between Nodes: Calculated as: dist(A,B) = depth(A) + depth(B) - 2 × depth(LCA)
🏆 LCA is one of the most important advanced tree algorithms. It's used in competitive programming, databases, and network topology.
💻

C Code: Path, LCA, and Distance

Code▶
// path_operations.c
/* ── Find PATH from root to target node ── */
int findPath(struct Node* root, int key, int* path, int idx) {
    if (!root) return 0;
    
    path[idx] = root->data;  /* add current to path */
    
    if (root->data == key) {   /* found the target! */
        for (int i = 0; i <= idx; i++)
            printf("%d ", path[i]);
        printf("\n");
        return 1;
    }
    
    /* Search left and right */
    if (findPath(root->left,  key, path, idx+1)) return 1;
    if (findPath(root->right, key, path, idx+1)) return 1;
    return 0; /* backtrack */
}

/* ── Print ALL root-to-leaf paths ── */
void printAllPaths(struct Node* root, int* path, int idx) {
    if (!root) return;
    
    path[idx] = root->data;
    
    if (!root->left && !root->right) {  /* leaf! print path */
        for (int i = 0; i <= idx; i++)
            printf("%d ", path[i]);
        printf("\n");
        return;
    }
    
    printAllPaths(root->left,  path, idx+1);
    printAllPaths(root->right, path, idx+1);
}

/* ── Lowest Common Ancestor (LCA) ── */
struct Node* findLCA(struct Node* root, int a, int b) {
    if (!root) return NULL;
    
    /* If current node is a or b, this is the LCA */
    if (root->data == a || root->data == b) return root;
    
    struct Node* leftLCA  = findLCA(root->left,  a, b);
    struct Node* rightLCA = findLCA(root->right, a, b);
    
    /* If both sides returned non-NULL, this node is LCA */
    if (leftLCA && rightLCA) return root;
    
    return leftLCA ? leftLCA : rightLCA;
}

/* ── Distance between two nodes ── */
int distFromRoot(struct Node* r, int k, int d){
    if(!r) return -1;
    if(r->data==k) return d;
    int l=distFromRoot(r->left,k,d+1);
    return l!=-1?l:distFromRoot(r->right,k,d+1);
}

int distanceBetween(struct Node* root, int a, int b) {
    struct Node* lca = findLCA(root, a, b);
    int dA = distFromRoot(lca, a, 0);
    int dB = distFromRoot(lca, b, 0);
    return dA + dB;  /* distance via LCA */
}
🌍 Real-Life Analogy
LCA is like finding the common ancestor in a family tree — if Alice and Bob are cousins, their LCA is the grandparent they share. Distance is how many "handshakes apart" two people are at a family reunion.

OPERATION GROUP 12

Recursive Operations — Summary

Recursive Insert · Delete · Search · Traversal · Height · Count · Copy

📖

Theory: How Recursion Works on Trees

Theory▶
Recursion and trees are a perfect match because trees are naturally recursive structures — a tree is a root node with left and right subtrees, which are themselves trees!

Every recursive tree function has the same pattern:
  1. Base case: Handle NULL (empty tree/subtree) — return 0, NULL, or empty.
  2. Recursive case: Call the same function on left and right children.
  3. Combine: Merge/use the results from left and right to get the answer for the current node.
🧠 The call stack automatically acts as the "memory" for recursion — no need to manually manage a stack!

Count nodes1 + L + RO(n)O(h)
Height1 + max(L,R)O(n)O(h)
Searchcheck || L || RO(n)O(h)
CopynewNode, copy(L), copy(R)O(n)O(h)

OPERATION GROUP 13

Iterative Operations

Iterative Insert · Delete · Search · Traversal · Queue & Stack Based Iteration

📖

Theory: Recursive vs Iterative — When to Use Which

Theory▶
Recursive approaches are clean and elegant but use the call stack — which can overflow for very deep trees (stack overflow for n > ~10,000 levels on most systems).

Iterative approaches use an explicit data structure (stack for DFS, queue for BFS) that lives in heap memory — much larger capacity.
AspectRecursiveIterative
Code sizeShort, elegantMore verbose
Stack usageCall stack (limited)Heap (large)
SpeedSame O(n)Same O(n)
RiskStack overflowNone for depth
Best forSmall trees, learningLarge/deep trees

// iterative_operations.c
/* ── Iterative INSERT (level order, BFS) ── */
struct Node* iterativeInsert(struct Node* root, int val) {
    struct Node* newNode = createNode(val);
    if(!root) return newNode;
    struct Node* q[100]; int f=0,r=0;
    q[r++]=root;
    while(f<r){
        struct Node* c=q[f++];
        if(!c->left){  c->left=newNode;  return root; }
        q[r++]=c->left;
        if(!c->right){ c->right=newNode; return root; }
        q[r++]=c->right;
    }
    return root;
}

int iterativeSearch(struct Node* root, int key) {
    if(!root) return 0;

    q[r++]=root;
    while(f<r){
        struct Node* c=q[f++];
        if(c->data==key) return 1;
        if(c->left)  q[r++]=c->left;
        if(c->right) q[r++]=c->right;
    }
    return 0;
}

/* ── Iterative INORDER Traversal (stack-based) ── */
void iterativeInorder(struct Node* root){
    struct Node* s[100]; int top=-1;
    struct Node* curr=root;
    while(curr||top>=0){
        while(curr){ s[++top]=curr; curr=curr->left; }
        curr=s[top--];
        printf("%d ",curr->data);
        curr=curr->right;
    }
}

/* ── Iterative POSTORDER Traversal (two-stack method) ── */
void iterativePostorder(struct Node* root){
    if(!root) return;
    struct Node* s1[100], *s2[100]; int t1=-1,t2=-1;
    s1[++t1]=root;
    while(t1>=0){
        struct Node* c=s1[t1--];
        s2[++t2]=c;
        if(c->left)  s1[++t1]=c->left;
        if(c->right) s1[++t1]=c->right;
    }
    while(t2>=0) printf("%d ",s2[t2--]->data);
}

/* ── Queue-Based Iteration (Level Order) ── */
void queueBasedIteration(struct Node* root){
    if(!root) return;
    struct Node* q[100]; int f=0,r=0;
    q[r++]=root;
    while(f<r){
        struct Node* c=q[f++];
        printf("%d ",c->data);
        if(c->left)  q[r++]=c->left;
        if(c->right) q[r++]=c->right;
    }
}

/* ── Stack-Based Iteration (Preorder DFS) ── */
void stackBasedIteration(struct Node* root){
    if(!root) return;
    struct Node* s[100]; int top=-1;
    s[++top]=root;
    while(top>=0){
        struct Node* c=s[top--];
        printf("%d ",c->data);
        if(c->right) s[++top]=c->right;
        if(c->left)  s[++top]=c->left;
    }
}

Iterative with a stack = you carry a physical to-do list on paper (the stack) as you walk through the building. Iterative with a queue = you have a numbered ticket system — process ticket #1 before #2. In both cases, you never "teleport" back (no function call overhead).
