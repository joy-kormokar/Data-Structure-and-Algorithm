Introduction to Undirected Graphs

Beginner

What is an Undirected Graph?

A graph is a collection of vertices (nodes) connected by edges (links). In an undirected graph, every edge is bidirectional — if A is connected to B, then B is also connected to A. There is no "direction" to the connection.

  • Vertices (V): The objects/entities (e.g., cities, computers)
  • Edges (E): The connections between them (e.g., roads, cables)
  • Edge between A and B is written as {A, B} or (A, B)
  • Degree of a vertex = number of edges touching it
🏙️

Real-Life Analogy

Think of a city road map. Each intersection is a vertex. Each two-way road is an edge. If you can drive from intersection A to B on a road, you can also drive from B to A on the same road — that's undirected!

Other examples: Facebook friends (if you're my friend, I'm yours), electrical circuits, and computer networks.

Key Terminology

Adjacent
Two vertices connected by an edge are called adjacent (neighbors).

Path
A sequence of vertices where each consecutive pair is connected by an edge.

Cycle
A path that starts and ends at the same vertex without repeating edges.

Representation — Incidence Matrix

Beginner

What is an Incidence Matrix?

The Incidence Matrix is a 2D array of size V × E (vertices × edges). Each cell [i][j] is 1 if vertex i is an endpoint of edge j, and 0 otherwise.

  • Rows represent vertices
  • Columns represent edges
  • Each column has exactly two 1s (since each edge connects two vertices)
  • Row sum = degree of that vertex
📋

Analogy: Club Membership List

Imagine a school where students (vertices) join clubs (edges). Each club has exactly 2 members. The incidence matrix is a sign-in sheet: rows = students, columns = clubs. A "1" means that student is in that club.

Looking at a row tells you which clubs a student is in (their degree). Looking at a column tells you which two students share that club (the edge).

Example: 4 Vertices, 4 Edges

Graph: V0-V1 (e0), V1-V2 (e1), V2-V3 (e2), V0-V3 (e3)

Comparison: Why Incidence Matrix?

Incidence Matrix
✅ Shows edge–vertex relationships clearly
✅ Easy to list all edges
❌ More memory (V×E)
❌ Slower adjacency check

Adjacency Matrix
✅ Fast O(1) adjacency check
❌ V² space always
❌ Hard to find all edges fast

Adjacency List
✅ Space-efficient
✅ Fast neighbor iteration
❌ Slower edge lookup
❌ Less intuitive for some ops

graph.c — Structure Definition 
#include <stdio.h>
#include <stdlib.h>
#include <string.h>

#define MAX_V 10   /* Maximum number of vertices */
#define MAX_E 20   /* Maximum number of edges    */

/* Graph structure using Incidence Matrix */
typedef struct {
    int matrix[MAX_V][MAX_E]; /* incMatrix[vertex][edge] = 0 or 1 */
    int numVertices;          /* current number of vertices        */
    int numEdges;             /* current number of edges           */
    char vertexLabel[MAX_V][20]; /* optional: label for each vertex */
} Graph;

/* Initialize an empty graph */
void initGraph(Graph *g) {
    g->numVertices = 0;
    g->numEdges    = 0;
    /* Set entire matrix to 0 */
    memset(g->matrix, 0, sizeof(g->matrix));
}

Insert Vertex

Beginner

Theory

Inserting a vertex adds a new row to the incidence matrix. The new row is initially all zeros because the new vertex has no edges yet.

1

Check if max vertices limit is reached.

2

Assign the next available row index to the new vertex.

3

Fill that row with zeros (no connections yet).

4

Increment numVertices by 1.

Time Complexity: O(E) to zero out the new row  |  Space: One new row added.

🏙️

Analogy

Adding a new intersection to a city map. Initially it has no roads (edges). You mark it on the map but draw no roads from it yet.

insert_vertex.c
/* Insert a new vertex into the graph */
int insertVertex(Graph *g, const char *label) {
    if (g->numVertices >= MAX_V) {
        printf("Error: Maximum vertex limit reached!\n");
        return -1;
    }
    int v = g->numVertices;  /* new vertex index */

    /* Zero out the new row (no edges yet) */
    for (int e = 0; e < g->numEdges; e++) {
        g->matrix[v][e] = 0;
    }

    /* Store optional label */
    if (label != NULL) {
        strncpy(g->vertexLabel[v], label, 19);
        g->vertexLabel[v][19] = '\0';
    }

    g->numVertices++;
    printf("Vertex %s inserted at index %d\n", label, v);
    return v;  /* return the new vertex index */
}

/* Usage Example */
int main() {
    Graph g;
    initGraph(&g);
    insertVertex(&g, "V0");
    insertVertex(&g, "V1");
    insertVertex(&g, "V2");
    return 0;
}

Delete Vertex

Beginner

Theory

Deleting a vertex is more complex. We must:

1

Find all edges that connect to this vertex (where matrix[v][e] == 1).

2

Delete all those edges first (remove their columns).

3

Remove the vertex's row by shifting all subsequent rows up by one.

4

Decrement numVertices.

Time: O(V × E) — must scan and shift the matrix.

🏗️

Analogy

Demolishing a city intersection. Before you can remove the intersection, all roads connecting to it must be closed (edges deleted). Then you renumber remaining intersections.

delete_vertex.c
/* Remove all edges incident to vertex v */
void removeIncidentEdges(Graph *g, int v) {
    int e = 0;
    while (e < g->numEdges) {
        if (g->matrix[v][e] == 1) {
            /* Shift columns left to remove edge e */
            for (int i = 0; i < g->numVertices; i++) {
                for (int j = e; j < g->numEdges - 1; j++) {
                    g->matrix[i][j] = g->matrix[i][j + 1];
                }
            }
            g->numEdges--;
            /* Don't increment e — re-check same column */
        } else {
            e++;
        }
    }
}

int deleteVertex(Graph *g, int v) {
    if (v < 0 || v >= g->numVertices) {
        printf("Error: Invalid vertex index!\n");
        return -1;
    }
    /* Step 1: Remove all edges connected to v */
    removeIncidentEdges(g, v);

    /* Step 2: Shift rows up to fill the gap */
    for (int i = v; i < g->numVertices - 1; i++) {
        for (int e = 0; e < g->numEdges; e++) {
            g->matrix[i][e] = g->matrix[i + 1][e];
        }
        strcpy(g->vertexLabel[i], g->vertexLabel[i + 1]);
    }
    g->numVertices--;
    printf("Vertex %d deleted successfully.\n", v);
    return 0;
}

Search Vertex

Beginner

Theory

Searching for a vertex by label means scanning the vertexLabel[] array. We can also retrieve all edges connected to a vertex by scanning its row in the incidence matrix.

Time: O(V) for label search, O(E) to list edges.

🔍

Analogy

Looking up a person in a directory, then checking which clubs (edges) they belong to by scanning their row in the membership sheet.

/* Search vertex by label; returns index or -1 */
int searchVertex(Graph *g, const char *label) {
    for (int v = 0; v < g->numVertices; v++) {
        if (strcmp(g->vertexLabel[v], label) == 0) {
            printf("Found '%s' at index %d\n", label, v);
            return v;
        }
    }
    printf("Vertex '%s' not found.\n", label);
    return -1;
}

    printf("Edges of vertex %s:\n", g->vertexLabel[v]);
    for (int e = 0; e < g->numEdges; e++) {
        if (g->matrix[v][e] == 1) {
            printf("  Edge e%d\n", e);
        }
    }
}

Insert Edge

Beginner

Theory

Inserting an edge adds a new column to the incidence matrix. In that column, exactly two cells are set to 1 — the rows corresponding to the two connected vertices.

1

Check that max edges limit is not exceeded.

2

Verify both vertex indices are valid.

3

Add a new column (index = numEdges).

4

Set matrix[u][newEdge] = 1 and matrix[v][newEdge] = 1.

5

Increment numEdges.

Time: O(V) to initialize new column | Self-loops: If u == v, only one row gets set to 1.

🛣️

Analogy

Building a new two-way road (edge) between two city intersections (vertices). The membership sheet gets a new club column, and exactly the two members who join that club are marked.

Interactive: Build Graph with Edges

Add vertices first (V0..V4 auto-loaded). Insert edges above.
insert_edge.c
/* Insert edge between vertices u and v */
int insertEdge(Graph *g, int u, int v) {
    if (g->numEdges >= MAX_E) {
        printf("Error: Maximum edge limit reached!\n");
        return -1;
    }
    if (u < 0 || u >= g->numVertices ||
        v < 0 || v >= g->numVertices) {
        printf("Error: Invalid vertex index!\n");
        return -1;
    }
    int e = g->numEdges;  /* new edge column index */

    /* Initialize the column to 0 */
    for (int i = 0; i < g->numVertices; i++) {
        g->matrix[i][e] = 0;
    }

    /* Mark the two endpoints */
    g->matrix[u][e] = 1;
    g->matrix[v][e] = 1;  /* same row if self-loop (u==v) */

    g->numEdges++;
    printf("Edge e%d inserted: V%d -- V%d\n", e, u, v);
    return e;
}

Delete Edge

Beginner

Theory

Deleting an edge removes a column from the incidence matrix. All subsequent columns shift one position to the left to fill the gap.

Time: O(V × E) due to column shifting.

🚧

Analogy

Permanently closing a road between two intersections. The road is removed from the map and all subsequent roads are re-indexed.

delete_edge.c
/* Delete edge at column index e */
int deleteEdge(Graph *g, int e) {
    if (e < 0 || e >= g->numEdges) {
        printf("Error: Invalid edge index!\n");
        return -1;
    }
    /* Shift all columns after e to the left */
    for (int j = e; j < g->numEdges - 1; j++) {
        for (int i = 0; i < g->numVertices; i++) {
            g->matrix[i][j] = g->matrix[i][j + 1];
        }
    }
    g->numEdges--;
    printf("Edge e%d deleted.\n", e);
    return 0;
}

/* Find edge between two vertices (returns edge index or -1) */
int findEdgeIndex(Graph *g, int u, int v) {
    for (int e = 0; e < g->numEdges; e++) {
        if (g->matrix[u][e] == 1 && g->matrix[v][e] == 1) {
            return e;  /* found! */
        }
    }
    return -1;  /* not found */
}

Search Edge

Beginner

Theory

To check if an edge exists between vertices u and v, scan all columns and look for one where both matrix[u][e] == 1 and matrix[v][e] == 1.

Time: O(E) — scan all edge columns.

🗺️

Analogy

Checking if a direct road exists between two cities — scan through all roads and check if both cities appear as endpoints of that road.

search_edge.c
/* Check if edge exists between u and v */
int searchEdge(Graph *g, int u, int v) {
    for (int e = 0; e < g->numEdges; e++) {
        /* Both u and v must be endpoints of edge e */
        if (g->matrix[u][e] == 1 && g->matrix[v][e] == 1) {
            printf("Edge found: V%d -- V%d (edge e%d)\n", u, v, e);
            return e;
        }
    }
    printf("No edge between V%d and V%d.\n", u, v);
    return -1;
}

/* Print all edges in the graph */
void printAllEdges(Graph *g) {
    printf("All edges:\n");
    for (int e = 0; e < g->numEdges; e++) {
        printf("  e%d: ", e);
        int count = 0;
        for (int v = 0; v < g->numVertices; v++) {
            if (g->matrix[v][e] == 1) {
                if (count > 0) printf(" -- ");
                printf("V%d", v);
                count++;
            }
        }
        printf("\n");
    }
}

Update Edge

Beginner

Theory

Updating an edge means changing its endpoints. This is equivalent to deleting the old edge and inserting a new one. In the incidence matrix, we find the column, zero it out, and set new rows to 1.

Use case: Rerouting a connection from one node to another.

🔄

Analogy

Rerouting a road — instead of connecting city A and B, you change it to connect city A and C. The road number (edge index) stays the same.

update_edge.c
/* Update edge e to now connect newU and newV */
int updateEdge(Graph *g, int e, int newU, int newV) {
    if (e < 0 || e >= g->numEdges) {
        printf("Error: Invalid edge index!\n");
        return -1;
    }
    if (newU < 0 || newU >= g->numVertices ||
        newV < 0 || newV >= g->numVertices) {
        printf("Error: Invalid vertex index!\n");
        return -1;
    }
    /* Clear the entire column for this edge */
    for (int i = 0; i < g->numVertices; i++) {
        g->matrix[i][e] = 0;
    }
    /* Set the new endpoints */
    g->matrix[newU][e] = 1;
    g->matrix[newV][e] = 1;
    printf("Edge e%d updated: now V%d -- V%d\n", e, newU, newV);
    return 0;
}

Breadth-First Search (BFS)

Intermediate

Theory

BFS explores the graph level by level. Starting from a source vertex, it first visits all direct neighbors (distance 1), then their neighbors (distance 2), and so on.

  • Uses a Queue (FIFO) data structure
  • Marks vertices as visited to avoid revisiting
  • Finds shortest path in unweighted graphs
  • Time: O(V + E)  |  Space: O(V)

In the incidence matrix, to find neighbors of vertex v: scan each edge column — if matrix[v][e] == 1, find the other endpoint in that column.

🌊

Analogy

Imagine dropping a stone in water. The ripples spread outward in concentric circles — first the immediate neighbors, then neighbors of neighbors. BFS is exactly this wave propagation.

Real use: Finding the shortest route on Google Maps (unweighted), social network friend suggestions, web crawlers.

bfs.c
#define MAX_V 10

/* Get neighbors of vertex v using incidence matrix */
int getNeighbors(Graph *g, int v, int *nbrs) {
    int count = 0;
    for (int e = 0; e < g->numEdges; e++) {
        if (g->matrix[v][e] == 1) {
            /* Find the other endpoint of this edge */
            for (int u = 0; u < g->numVertices; u++) {
                if (u != v && g->matrix[u][e] == 1) {
                    nbrs[count++] = u;
                }
            }
        }
    }
    return count;
}

/* BFS traversal from start vertex */
void bfs(Graph *g, int start) {
    int visited[MAX_V] = {0};  /* 0 = not visited */
    int queue[MAX_V];            /* simple array queue */
    int front = 0, rear = 0;
    int nbrs[MAX_V];

    /* Enqueue starting vertex */
    queue[rear++] = start;
    visited[start] = 1;
    printf("BFS order: ");

    while (front < rear) {
        int v = queue[front++];  /* Dequeue */
        printf("V%d ", v);

        int numNbrs = getNeighbors(g, v, nbrs);
        for (int i = 0; i < numNbrs; i++) {
            if (!visited[nbrs[i]]) {
                visited[nbrs[i]] = 1;
                queue[rear++] = nbrs[i];
            }
        }
    }
    printf("\n");
}

Depth-First Search (DFS)

Intermediate

Theory

DFS explores as deep as possible along each branch before backtracking. It uses a Stack (or recursion, which uses the call stack implicitly).

  • Goes deep into one path before exploring others
  • Uses visited[] array to avoid cycles
  • Used for: cycle detection, topological sort, finding connected components
  • Time: O(V + E)  |  Space: O(V) recursion stack
🌀

Analogy

Exploring a maze: you always keep going forward (deeper) until you hit a dead end, then backtrack to the last junction and try a different path. You mark every room you visit.

Real use: Solving puzzles/mazes, detecting cycles, checking if a graph is connected, finding strongly connected components.

dfs.c
int visited[MAX_V];  /* global visited array */

void dfsHelper(Graph *g, int v) {
    visited[v] = 1;
    printf("V%d ", v);  /* process this vertex */

    int nbrs[MAX_V];
    int numNbrs = getNeighbors(g, v, nbrs);

    for (int i = 0; i < numNbrs; i++) {
        if (!visited[nbrs[i]]) {
            dfsHelper(g, nbrs[i]);  /* recurse deeper */
        }
    }
    /* Backtrack automatically when function returns */
}

void dfs(Graph *g, int start) {
    memset(visited, 0, sizeof(visited));
    printf("DFS order: ");
    dfsHelper(g, start);
    printf("\n");
}

Find Path

Intermediate

Theory

Finding whether a path exists between two vertices uses DFS (or BFS). We recursively explore from the source; if we reach the destination, a path exists.

  • Track the path using a parent[] array
  • Reconstruct path by backtracking from destination to source
  • Time: O(V + E)
🗺️

Analogy

GPS navigation finding if there's any route from your home to a destination — not the shortest, just any route. You follow roads, marking visited intersections, until you reach the destination.

find_path.c
int parent[MAX_V];   /* to reconstruct path */
int pathVisited[MAX_V];

int dfsPath(Graph *g, int cur, int dest) {
    pathVisited[cur] = 1;
    if (cur == dest) return 1;  /* Found! */

    int nbrs[MAX_V];
    int count = getNeighbors(g, cur, nbrs);

    for (int i = 0; i < count; i++) {
        if (!pathVisited[nbrs[i]]) {
            parent[nbrs[i]] = cur;
            if (dfsPath(g, nbrs[i], dest)) return 1;
        }
    }
    return 0;  /* not found from this branch */
}

void findPath(Graph *g, int src, int dest) {
    memset(pathVisited, 0, sizeof(pathVisited));
    memset(parent, -1, sizeof(parent));
    parent[src] = src;

    if (!dfsPath(g, src, dest)) {
        printf("No path from V%d to V%d\n", src, dest);
        return;
    }
    /* Reconstruct path from dest back to src */
    int path[MAX_V], len = 0, cur = dest;
    while (cur != src) { path[len++] = cur; cur = parent[cur]; }
    path[len++] = src;
    printf("Path: ");
    for (int i = len - 1; i >= 0; i--)
        printf("V%d%s", path[i], (i > 0 ? " -> " : "\n"));
}

Shortest Path (BFS-based)

Intermediate

Theory

For unweighted graphs, BFS naturally finds the shortest path. Each level of BFS represents one "hop". The first time BFS reaches the destination, it has taken the minimum number of edges.

  • Use a dist[] array initialized to -1 (unvisited)
  • dist[src] = 0, then each neighbor gets dist[v] + 1
  • Track parent[] to reconstruct the path
  • Time: O(V + E)
⚡

Analogy

Finding the fewest bus stops between two stations. BFS expands outward hop by hop — the first time you reach your destination, you've used the minimum number of stops (edges).

Real use: Shortest route in a simple network, minimum hops in routing protocols, friend-of-friend distance in social networks.

shortest_path.c
void shortestPath(Graph *g, int src, int dest) {
    int dist[MAX_V], par[MAX_V];
    int queue[MAX_V];
    int front = 0, rear = 0;

    /* Initialize all distances to -1 (unvisited) */
    for (int i = 0; i < g->numVertices; i++) {
        dist[i] = -1;
        par[i]  = -1;
    }

    dist[src] = 0;
    queue[rear++] = src;

    while (front < rear) {
        int v = queue[front++];
        int nbrs[MAX_V];
        int cnt = getNeighbors(g, v, nbrs);

        for (int i = 0; i < cnt; i++) {
            if (dist[nbrs[i]] == -1) {  /* not visited */
                dist[nbrs[i]] = dist[v] + 1;
                par[nbrs[i]]  = v;
                queue[rear++] = nbrs[i];
            }
        }
    }

    if (dist[dest] == -1) {
        printf("No path from V%d to V%d\n", src, dest);
        return;
    }
    printf("Shortest distance: %d hops\n", dist[dest]);

    /* Reconstruct path */
    int path[MAX_V], len = 0, cur = dest;
    while (cur != -1) { path[len++] = cur; cur = par[cur]; }
    printf("Path: ");
    for (int i = len-1; i >= 0; i--)
        printf("V%d%s", path[i], (i > 0 ? " -> " : "\n"));
}

All Paths Between Two Vertices

Intermediate

Theory

Finding all paths requires backtracking DFS. We must un-mark visited nodes after backtracking so other paths can use them.

  • Maintain a path[] array and pathLen
  • When we reach destination, print current path
  • Mark as visited before recursing, un-mark after (backtrack)
  • Time: O(V!) in worst case — exponential
🔀

Analogy

Finding all possible driving routes between two cities — you try every possible combination of roads. Used in network analysis to find all routes for redundancy planning.

all_paths.c
int apVisited[MAX_V];
int apPath[MAX_V];
int apPathLen = 0;

void printPath(int len) {
    for (int i = 0; i < len; i++)
        printf("V%d%s", apPath[i], (i < len-1 ? " -> " : "\n"));
}

void allPaths(Graph *g, int cur, int dest) {
    apVisited[cur] = 1;
    apPath[apPathLen++] = cur;

    if (cur == dest) {
        printPath(apPathLen);  /* found a complete path */
    } else {
        int nbrs[MAX_V];
        int cnt = getNeighbors(g, cur, nbrs);
        for (int i = 0; i < cnt; i++) {
            if (!apVisited[nbrs[i]]) {
                allPaths(g, nbrs[i], dest);
            }
        }
    }
    /* BACKTRACK: un-mark so other paths can use this node */
    apPathLen--;
    apVisited[cur] = 0;
}

/* Call like this: */
void findAllPaths(Graph *g, int src, int dest) {
    memset(apVisited, 0, sizeof(apVisited));
    apPathLen = 0;
    printf("All paths from V%d to V%d:\n", src, dest);
    allPaths(g, src, dest);
}

Connectivity Check

Intermediate

Theory

A graph is connected if there is a path between every pair of vertices. To check: run BFS/DFS from any vertex. If all vertices get visited, the graph is connected.

  • Run DFS/BFS from vertex 0
  • Count how many vertices were visited
  • If count == numVertices → connected
  • Time: O(V + E)
🔗

Analogy

Checking if every city in a country can be reached from the capital. If you can drive from the capital to every other city following roads, the road network is connected.

connectivity.c
int isConnected(Graph *g) {
    if (g->numVertices == 0) return 1;  /* empty is trivially connected */

    int vis[MAX_V] = {0};
    int queue[MAX_V];
    int front = 0, rear = 0;
    int count = 0;

    queue[rear++] = 0;  /* start BFS from vertex 0 */
    vis[0] = 1;

    while (front < rear) {
        int v = queue[front++];
        count++;
        int nbrs[MAX_V];
        int cnt = getNeighbors(g, v, nbrs);
        for (int i = 0; i < cnt; i++) {
            if (!vis[nbrs[i]]) {
                vis[nbrs[i]] = 1;
                queue[rear++] = nbrs[i];
            }
        }
    }
    return (count == g->numVertices);
}

/* Usage */
if (isConnected(&g))
    printf("Graph is CONNECTED\n");
else
    printf("Graph is DISCONNECTED\n");

Connected Components

Intermediate

Theory

A connected component is a maximal subgraph where every vertex is reachable from every other vertex. To find all components:

1

Start from vertex 0. Run DFS/BFS. Mark all reachable vertices as component 0.

2

Find the next unvisited vertex. Run DFS/BFS again — this is component 1.

3

Repeat until all vertices are visited. Count = number of components.

🧩

Analogy

Islands in an ocean. Each island (component) is a group of cities connected by roads. There are no roads between islands. The number of islands = number of components.

components.c
void dfsComponent(Graph *g, int v, int *visited, int compId) {
    visited[v] = compId;
    int nbrs[MAX_V];
    int cnt = getNeighbors(g, v, nbrs);
    for (int i = 0; i < cnt; i++) {
        if (visited[nbrs[i]] == -1)
            dfsComponent(g, nbrs[i], visited, compId);
    }
}

int findComponents(Graph *g) {
    int visited[MAX_V];
    memset(visited, -1, sizeof(visited));  /* -1 = unvisited */
    int numComponents = 0;

    for (int v = 0; v < g->numVertices; v++) {
        if (visited[v] == -1) {
            dfsComponent(g, v, visited, numComponents);
            numComponents++;
        }
    }

    printf("Number of components: %d\n", numComponents);
    for (int c = 0; c < numComponents; c++) {
        printf("Component %d: ", c);
        for (int v = 0; v < g->numVertices; v++) {
            if (visited[v] == c) printf("V%d ", v);
        }
        printf("\n");
    }
    return numComponents;
}

Isolation Check

Beginner

Theory

An isolated vertex is a vertex with degree 0 — it has no edges connecting it to any other vertex. In the incidence matrix, its entire row is all zeros.

Time: O(E) per vertex — scan the row for any 1.

🏝️

Analogy

A city with no roads at all — no connections to any other city. It exists on the map but is completely unreachable from anywhere else.

isolation.c
/* Check if vertex v is isolated (degree = 0) */
int isIsolated(Graph *g, int v) {
    for (int e = 0; e < g->numEdges; e++) {
        if (g->matrix[v][e] == 1) return 0;  /* has an edge */
    }
    return 1;  /* no edges found — isolated! */
}

/* Print all isolated vertices */
void findIsolatedVertices(Graph *g) {
    printf("Isolated vertices: ");
    int found = 0;
    for (int v = 0; v < g->numVertices; v++) {
        if (isIsolated(g, v)) {
            printf("V%d ", v);
            found = 1;
        }
    }
    if (!found) printf("none");
    printf("\n");
}

Detect Cycle

Intermediate

Theory

A cycle exists if DFS encounters an already-visited vertex that is not the parent of the current vertex. In undirected graphs, we track the parent to avoid false positives.

  • Run DFS tracking parent[]
  • If we see a visited neighbor that's NOT our parent → cycle found
  • Time: O(V + E)

Note: A graph with V vertices and ≥ V edges always contains a cycle.

🔁

Analogy

Walking on roads: if you can start from a city, keep walking, and return to where you started without going back on any road — you've found a cycle!

Real use: Detecting deadlocks in operating systems, finding circular dependencies in build systems.

detect_cycle.c
int dfsCycle(Graph *g, int v, int par, int *visited) {
    visited[v] = 1;
    int nbrs[MAX_V];
    int cnt = getNeighbors(g, v, nbrs);

    for (int i = 0; i < cnt; i++) {
        int nb = nbrs[i];
        if (!visited[nb]) {
            if (dfsCycle(g, nb, v, visited)) return 1;
        } else if (nb != par) {
            /* visited neighbor that's not our parent = CYCLE! */
            return 1;
        }
    }
    return 0;
}

int hasCycle(Graph *g) {
    int visited[MAX_V] = {0};
    for (int v = 0; v < g->numVertices; v++) {
        if (!visited[v]) {
            if (dfsCycle(g, v, -1, visited)) {
                printf("Cycle DETECTED in graph!\n");
                return 1;
            }
        }
    }
    printf("No cycle found.\n");
    return 0;
}

Degree Analysis

Beginner

Theory

The degree of a vertex is the number of edges connected to it. In the incidence matrix:

  • Degree of vertex v = sum of row v = count of 1s in that row
  • Handshaking Lemma: Sum of all degrees = 2 × number of edges
  • Vertex with degree 0 → isolated
  • Vertex with degree 1 → leaf (pendant)
  • Time: O(E) per vertex, O(V×E) total
📐

Analogy

The degree of a city intersection is the number of roads meeting at it. A 3-way stop has degree 3. A dead-end has degree 1. A city with no roads has degree 0.

Real use: Network analysis (finding hubs), social network influence measurement (who has the most friends?), protein interaction networks.

/* Get degree of vertex v = sum of its row */
int getDegree(Graph *g, int v) {
    int degree = 0;
    for (int e = 0; e < g->numEdges; e++) {
        degree += g->matrix[v][e];  /* add 1 for each edge */
    }
    return degree;
}

/* Full degree analysis of graph */
void degreeAnalysis(Graph *g) {
    int totalDeg = 0;
    int maxDeg = 0, minDeg = MAX_E;

    printf("Degree of each vertex:\n");
    for (int v = 0; v < g->numVertices; v++) {
        int d = getDegree(g, v);
        printf("  V%d (%s): degree = %d\n",
               v, g->vertexLabel[v], d);
        totalDeg += d;
        if (d > maxDeg) maxDeg = d;
        if (d < minDeg) minDeg = d;
    }
    printf("Sum of all degrees = %d (= 2 × %d edges)\n",
           totalDeg, g->numEdges);
    printf("Max degree = %d, Min degree = %d\n", maxDeg, minDeg);
}

Minimum Spanning Tree (MST)

Advanced

Theory

A Spanning Tree connects all vertices with V-1 edges and no cycles. The Minimum Spanning Tree has the smallest total edge weight.

Kruskal's Algorithm (used here):

1

Sort all edges by weight (ascending).

2

Pick the smallest edge. Add it if it doesn't form a cycle (use Union-Find).

3

Repeat until V-1 edges are selected.

Time: O(E log E) for sorting + O(E α(V)) for union-find.

🌲

Analogy

A telecom company must connect all cities with cables at minimum cost. They sort all possible cables by cost, add the cheapest ones that don't create redundant loops, until all cities are connected.

Real use: Laying electrical cables/fiber optic lines, designing road networks, clustering algorithms.

mst_kruskal.c
/* Extend graph to store edge weights */
int edgeWeight[MAX_E];  /* weight of each edge column */

/* Union-Find: find root of vertex v */
int find(int *parent, int v) {
    if (parent[v] != v)
        parent[v] = find(parent, parent[v]);  /* path compression */
    return parent[v];
}

/* Union-Find: merge two sets */
void unionSets(int *parent, int *rank, int a, int b) {
    int ra = find(parent, a);
    int rb = find(parent, b);
    if (rank[ra] < rank[rb]) parent[ra] = rb;
    else if (rank[ra] > rank[rb]) parent[rb] = ra;
    else { parent[rb] = ra; rank[ra]++; }
}

void kruskalMST(Graph *g) {
    /* Sort edges by weight using simple bubble sort */
    int order[MAX_E];
    for (int i = 0; i < g->numEdges; i++) order[i] = i;
    for (int i = 0; i < g->numEdges - 1; i++) {
        for (int j = 0; j < g->numEdges - i - 1; j++) {
            if (edgeWeight[order[j]] > edgeWeight[order[j+1]]) {
                int tmp = order[j]; order[j] = order[j+1]; order[j+1] = tmp;
            }
        }
    }

    int parent[MAX_V], rnk[MAX_V];
    for (int i = 0; i < g->numVertices; i++) {
        parent[i] = i; rnk[i] = 0;
    }

    int mstWeight = 0, edgesUsed = 0;
    printf("MST edges:\n");
    for (int k = 0; k < g->numEdges && edgesUsed < g->numVertices-1; k++) {
        int e = order[k];
        /* Find the two endpoints of edge e */
        int u = -1, v = -1;
        for (int i = 0; i < g->numVertices; i++) {
            if (g->matrix[i][e] == 1) {
                if (u == -1) u = i; else v = i;
            }
        }
        if (find(parent, u) != find(parent, v)) {
            unionSets(parent, rnk, u, v);
            printf("  V%d -- V%d  weight=%d\n", u, v, edgeWeight[e]);
            mstWeight += edgeWeight[e];
            edgesUsed++;
        }
    }
    printf("Total MST weight = %d\n", mstWeight);
}

Graph Coloring

Advanced

Theory

Graph Coloring assigns a color to each vertex such that no two adjacent vertices share the same color. The minimum number of colors needed is the chromatic number χ(G).

Greedy Coloring Algorithm:

1

Assign color 0 to the first vertex.

2

For each remaining vertex, find the smallest color not used by any of its neighbors.

3

Assign that color.

Note: Greedy doesn't always give the minimum, but is efficient: O(V + E).

🎨

Analogy

Coloring a map so no two neighboring countries share the same color. The famous "Four Color Theorem" states any planar map needs at most 4 colors.

Real use: Register allocation in compilers (variables ↔ colors, interference ↔ edges), timetable scheduling, frequency assignment in wireless networks.

void greedyColoring(Graph *g) {
    int color[MAX_V];
    memset(color, -1, sizeof(color));  /* -1 = uncolored */

    color[0] = 0;  /* assign first color to first vertex */

    for (int v = 1; v < g->numVertices; v++) {
        /* Track which colors are used by neighbors */
        int usedColors[MAX_V] = {0};  /* usedColors[c]=1 if color c is taken */

        int nbrs[MAX_V];
        int cnt = getNeighbors(g, v, nbrs);
        for (int i = 0; i < cnt; i++) {
            if (color[nbrs[i]] != -1) {
                usedColors[color[nbrs[i]]] = 1;
            }
        }

        /* Assign the smallest available color */
        int c = 0;
        while (usedColors[c]) c++;
        color[v] = c;
    }

    /* Find chromatic number (max color used + 1) */
    int maxColor = 0;
    printf("Coloring result:\n");
    for (int v = 0; v < g->numVertices; v++) {
        printf("  V%d → color %d\n", v, color[v]);
        if (color[v] > maxColor) maxColor = color[v];
    }
    printf("Chromatic number (greedy) = %d\n", maxColor + 1);
}

Bipartite Check

Advanced

Theory

A graph is bipartite if its vertices can be split into two groups (sets A and B) such that every edge connects a vertex in A to one in B — no edges within the same group.

Key fact: A graph is bipartite if and only if it has no odd-length cycles.

Algorithm: BFS 2-coloring — color vertices alternately 0 and 1. If any neighbor has the same color as current → not bipartite.

Time: O(V + E)

⚖️

Analogy

A dating app where only men can match with women (and vice versa). Men = set A, Women = set B. Matches (edges) only go between sets. If two men somehow match each other, it's not bipartite.

Real use: Matching problems (job assignments, student-project pairing), checking if a network can be 2-colored (frequency assignment), stable marriage problem.

bipartite.c
int isBipartite(Graph *g) {
    int color[MAX_V];
    memset(color, -1, sizeof(color));  /* -1 = unvisited */
    int queue[MAX_V];
    int front, rear;

    for (int start = 0; start < g->numVertices; start++) {
        if (color[start] != -1) continue;  /* already colored */

        front = rear = 0;
        color[start] = 0;  /* assign first color */
        queue[rear++] = start;

        while (front < rear) {
            int v = queue[front++];
            int nbrs[MAX_V];
            int cnt = getNeighbors(g, v, nbrs);

            for (int i = 0; i < cnt; i++) {
                int nb = nbrs[i];
                if (color[nb] == -1) {
                    color[nb] = 1 - color[v]; /* flip color: 0→1, 1→0 */
                    queue[rear++] = nb;
                } else if (color[nb] == color[v]) {
                    /* Same color as current = NOT bipartite */
                    printf("NOT bipartite: V%d and V%d "
                           "same color and adjacent!\n", v, nb);
                    return 0;
                }
            }
        }
    }
    printf("Graph IS bipartite!\n");
    printf("Set A: vertices with color 0\n");
    printf("Set B: vertices with color 1\n");
    return 1;
}
