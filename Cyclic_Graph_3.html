Incidence Matrix
The foundation — representing a cyclic graph as a 2D array

An Incidence Matrix is a 2D array (matrix) where rows represent vertices and columns represent edges. Each cell tells you whether a vertex is connected to an edge.

For an undirected graph: cell = 1 if vertex is an endpoint of that edge, else 0.
For a directed graph: +1 = tail (start), -1 = head (end), 0 = not connected.
A self-loop puts 2 in the cell.

🌍 Real-life Analogy

Imagine a bus route map. Rows are bus stops (vertices), columns are routes (edges). A "1" means that stop is on that route. The incidence matrix is exactly this attendance sheet — which stops appear on which routes.

📐 Matrix Structure
Matrix size = V × E  |  V = vertices, E = edges

Undirected graph — values: 0 or 1

Directed graph — values: -1, 0, +1

C — Incidence Matrix Definition
/* Cyclic Graph using Incidence Matrix in C
   Rows = Vertices, Columns = Edges
   For directed: +1 = start, -1 = end, 0 = not connected */

#include <stdio.h>
#include <string.h>

#define MAX_V  10   /* max vertices */
#define MAX_E  20   /* max edges    */

/* The graph structure */
struct Graph {
    int incMatrix[MAX_V][MAX_E]; /* incidence matrix */
    int vertices;                /* total vertices   */
    int edges;                   /* total edges      */
};

/* Initialize an empty graph */
void initGraph(struct Graph *g) {
    g->vertices = 0;
    g->edges = 0;
    /* Set all cells to 0 */
    memset(g->incMatrix, 0, sizeof(g->incMatrix));
}

int main() {
    struct Graph g;
    initGraph(&g);
    printf("Graph ready with %d vertices, %d edges\n",
           g.vertices, g.edges);
    return 0;
}

Insert Vertex
Adding a new node to the cyclic graph

Inserting a vertex means adding a new row to the incidence matrix. All cells in the new row are initialized to 0, because the new vertex has no edges yet. The vertex counter increases by 1.

🌍 Real-life Analogy

Think of a metro map. Adding a new station = inserting a vertex. The new station exists on the map but has no train lines (edges) yet — all its cells in the route table are 0.

▶ Insert Vertex Demo

C — Insert Vertex
/* Insert a new vertex into the graph */
int insertVertex(struct Graph *g) {
    if (g->vertices >= MAX_V) {
        printf("Graph is full! Cannot add vertex.\n");
        return -1;
    }
    int newRow = g->vertices; /* row index for new vertex */

    /* Set all edge cells to 0 for this vertex */
    for (int e = 0; e < g->edges; e++) {
        g->incMatrix[newRow][e] = 0;
    }

    g->vertices++; /* increase vertex count */
    printf("Vertex V%d inserted. Total: %d\n",
           newRow, g->vertices);
    return newRow; /* return index of new vertex */
}

Delete Vertex
Removing a node and all its connected edges

Deleting a vertex requires: (1) Remove all edges connected to it (delete columns where row = 1). (2) Remove that row from the matrix. (3) Shift remaining rows up. The vertex and edge counters update accordingly.

🌍 Real-life Analogy

Closing a bus stop — when you remove a stop, all bus routes that only passed through it must also be rechecked. The entire attendance row for that stop is deleted.

C — Delete Vertex
/* Delete vertex at given index */
void deleteVertex(struct Graph *g, int v) {
    if (v < 0 || v >= g->vertices) {
        printf("Invalid vertex index!\n"); return;
    }

    /* Step 1: Find and delete edges touching this vertex */
    for (int e = g->edges - 1; e >= 0; e--) {
        if (g->incMatrix[v][e] != 0) {
            /* Shift columns left to remove this edge */
            for (int i = 0; i < g->vertices; i++)
                for (int j = e; j < g->edges - 1; j++)
                    g->incMatrix[i][j] = g->incMatrix[i][j+1];
            g->edges--;
        }
    }

    /* Step 2: Shift rows up to remove this vertex */
    for (int i = v; i < g->vertices - 1; i++)
        for (int e = 0; e < g->edges; e++)
            g->incMatrix[i][e] = g->incMatrix[i+1][e];

    g->vertices--;
    printf("Vertex %d deleted.\n", v);
}

Search Vertex
Checking if a vertex exists in the graph

Searching for a vertex in a graph represented by an incidence matrix simply means checking if the index is within range (0 to vertices-1). Since vertices are numbered, a search checks: 0 ≤ v < vertices. You can also check if any edge column has a non-zero value for that row.

🌍 Real-life Analogy

It's like checking if a city exists on the bus network map. You look at the row labels — if the city name is there, it exists.

C — Search Vertex
/* Returns 1 if vertex v exists, 0 otherwise */
int searchVertex(struct Graph *g, int v) {
    if (v >= 0 && v < g->vertices) {
        printf("Vertex V%d FOUND!\n", v);
        return 1;
    }
    printf("Vertex V%d NOT found.\n", v);
    return 0;
}

Insert Edge
Adding a new connection between two vertices

Inserting an edge adds a new column to the incidence matrix. For an undirected edge (u, v): set matrix[u][new_col] = 1 and matrix[v][new_col] = 1. For a directed edge: matrix[u][new_col] = +1 (start) and matrix[v][new_col] = -1 (end).

🌍 Real-life Analogy

Adding a new bus route between two stops. The new route gets its own column. The two stops it connects get marked with "1" in that column.

C — Insert Edge (Undirected)
/* Insert undirected edge between u and v */
int insertEdge(struct Graph *g, int u, int v) {
    if (g->edges >= MAX_E) {
        printf("Max edges reached!\n"); return -1;
    }
    if (u < 0 || u >= g->vertices ||
        v < 0 || v >= g->vertices) {
        printf("Invalid vertices!\n"); return -1;
    }

    int col = g->edges; /* new column index */

    /* Mark both endpoints */
    g->incMatrix[u][col] = 1;
    g->incMatrix[v][col] = 1;

    g->edges++;
    printf("Edge E%d: V%d -- V%d added.\n", col, u, v);
    return col;
}

/* Insert DIRECTED edge u -> v */
int insertDirectedEdge(struct Graph *g, int u, int v) {
    int col = g->edges;
    g->incMatrix[u][col] = +1; /* tail: source */
    g->incMatrix[v][col] = -1; /* head: destination */
    g->edges++;
    printf("Directed edge E%d: V%d -> V%d added.\n", col, u, v);
    return col;
}

Delete Edge
Removing a connection from the graph

Deleting an edge means removing a column from the incidence matrix. All columns to the right of the deleted one shift one position left. The edge count decreases by 1.

C — Delete Edge
/* Delete edge at column index e */
void deleteEdge(struct Graph *g, int e) {
    if (e < 0 || e >= g->edges) {
        printf("Invalid edge index!\n"); return;
    }
    /* Shift columns left */
    for (int v = 0; v < g->vertices; v++)
        for (int j = e; j < g->edges - 1; j++)
            g->incMatrix[v][j] = g->incMatrix[v][j+1];

    g->edges--;
    printf("Edge E%d deleted.\n", e);
}

BEGINNER
Search Edge
Finding if an edge between two vertices exists

To search for an edge between vertices u and v, scan all columns of the matrix. A column represents an edge (u,v) if matrix[u][col] ≠ 0 AND matrix[v][col] ≠ 0. Time complexity: O(E).

C — Search Edge
/* Find edge between u and v; returns column index or -1 */
int searchEdge(struct Graph *g, int u, int v) {
    for (int e = 0; e < g->edges; e++) {
        /* Both vertices must appear in this column */
        if (g->incMatrix[u][e] != 0 &&
            g->incMatrix[v][e] != 0) {
            printf("Edge V%d-V%d found at E%d\n", u, v, e);
            return e;
        }
    }
    printf("No edge between V%d and V%d\n", u, v);
    return -1;
}

Update Edge
Changing an edge's endpoints

Updating an edge means changing which two vertices it connects. We first clear the existing column (set all to 0), then set the new endpoints. This is equivalent to delete + insert but reuses the same column index.

C — Update Edge
/* Update edge e to connect newU and newV */
void updateEdge(struct Graph *g, int e, int newU, int newV) {
    if (e < 0 || e >= g->edges) { printf("Invalid edge!\n"); return; }

    /* Clear all rows for this edge column */
    for (int v = 0; v < g->vertices; v++)
        g->incMatrix[v][e] = 0;

    /* Set new endpoints */
    g->incMatrix[newU][e] = 1;
    g->incMatrix[newV][e] = 1;
    printf("Edge E%d updated: V%d -- V%d\n", e, newU, newV);
}

Breadth First Search (BFS)
Level-by-level graph traversal using a queue

BFS visits vertices level by level — first all neighbors at distance 1, then distance 2, and so on. It uses a queue (FIFO) data structure. In incidence matrix BFS: for each vertex, scan all columns to find its neighbors (where matrix[v][e] ≠ 0, find other v' where matrix[v'][e] ≠ 0).

🌍 Real-life Analogy

Spreading news in a social network — person A tells all their friends (level 1), then each of them tells their friends (level 2), and so on. BFS mirrors exactly this wave-like spreading.

▶ BFS Animated Traversal

C — BFS on Incidence Matrix
#include <stdio.h>

/* Simple queue for BFS */
int queue[MAX_V], qFront = 0, qRear = 0;
void enqueue(int x) { queue[qRear++] = x; }
int  dequeue()      { return queue[qFront++]; }
int  isQEmpty()     { return qFront == qRear; }

void BFS(struct Graph *g, int start) {
    int visited[MAX_V] = {0};

    /* Step 1: Initialize queue */
    qFront = qRear = 0;
    enqueue(start);
    visited[start] = 1;

    printf("BFS order: ");
    while (!isQEmpty()) {
        int curr = dequeue();
        printf("V%d ", curr); /* Step 3: Visit Node */

        /* Step 4: Traverse Adjacent Nodes */
        for (int e = 0; e < g->edges; e++) {
            if (g->incMatrix[curr][e] == 0) continue;
            /* Find the neighbor on this edge */
            for (int nb = 0; nb < g->vertices; nb++) {
                if (nb != curr &&
                    g->incMatrix[nb][e] != 0 &&
                    !visited[nb]) {
                    visited[nb] = 1;  /* Step 2: Mark Visited */
                    enqueue(nb);
                }
            }
        }
    }
    printf("\n");
}

Depth First Search (DFS)
Deep-dive traversal using recursion or stack

DFS explores as far as possible before backtracking. It uses either a call stack (recursion) or an explicit stack. DFS is essential for cycle detection, topological sort, and finding connected components.

🌍 Real-life Analogy

Exploring a maze — you pick one path and follow it to the end (or a dead end), then backtrack and try another path. DFS is exactly this strategy.

▶ DFS Animated Traversal
Click "Run DFS" to start traversal from vertex 0.
C — DFS (Recursive + Stack versions)
int visited[MAX_V];

/* ── Recursive DFS ── */
void DFS_recursive(struct Graph *g, int v) {
    visited[v] = 1;
    printf("V%d ", v);
    for (int e = 0; e < g->edges; e++) {
        if (g->incMatrix[v][e] == 0) continue;
        for (int nb = 0; nb < g->vertices; nb++)
            if (nb != v && g->incMatrix[nb][e] != 0 && !visited[nb])
                DFS_recursive(g, nb); /* Backtracking handled by recursion */
    }
}

/* ── Stack-based DFS ── */
void DFS_stack(struct Graph *g, int start) {
    int stack[MAX_V], top = -1;
    int vis[MAX_V] = {0};

    stack[++top] = start;
    printf("DFS (stack): ");
    while (top >= 0) {
        int v = stack[top--];
        if (vis[v]) continue;
        vis[v] = 1;
        printf("V%d ", v);
        for (int e = 0; e < g->edges; e++) {
            if (g->incMatrix[v][e] == 0) continue;
            for (int nb = 0; nb < g->vertices; nb++)
                if (nb != v && g->incMatrix[nb][e] != 0 && !vis[nb])
                    stack[++top] = nb;
        }
    }
    printf("\n");
}

Detect Cycle
Determining if the graph contains any cycle

A cycle is a path that starts and ends at the same vertex. In an incidence matrix, we detect cycles using DFS: if we reach a vertex that is already visited (and it's not the parent in the DFS tree), a cycle exists. This is called finding a back edge.

🌍 Real-life Analogy

A roundabout road — if you start driving from a point and can return to the same point without reversing, there's a cycle.

▶ Cycle Detection Visualization
Graph is a tree (no cycle). Add an edge or click Detect.
C — Detect Cycle (Undirected Graph)
int visited_cd[MAX_V];

/* DFS helper: returns 1 if cycle found */
int dfs_cycle(struct Graph *g, int v, int parent) {
    visited_cd[v] = 1;
    for (int e = 0; e < g->edges; e++) {
        if (g->incMatrix[v][e] == 0) continue;
        for (int nb = 0; nb < g->vertices; nb++) {
            if (nb == v) continue;
            if (g->incMatrix[nb][e] == 0) continue;
            if (!visited_cd[nb]) {
                if (dfs_cycle(g, nb, v)) return 1;
            } else if (nb != parent) {
                return 1; /* Back edge → cycle! */
            }
        }
    }
    return 0;
}

int detectCycle(struct Graph *g) {
    memset(visited_cd, 0, sizeof(visited_cd));
    for (int v = 0; v < g->vertices; v++)
        if (!visited_cd[v])
            if (dfs_cycle(g, v, -1)) {
                printf("Cycle DETECTED!\n");
                return 1;
            }
    printf("No cycle found.\n");
    return 0;
}

Count Cycles
Finding the total number of independent cycles

The number of independent cycles in a graph is called the cycle rank or circuit rank. Formula: C = E − V + connected_components. For a connected graph: C = E − V + 1. Each extra edge beyond a spanning tree adds exactly one cycle.

Cycle Count = E − V + C  |  C = number of connected components
C — Count Cycles (Cycle Rank)
/* Count cycles using cycle rank formula */
int countCycles(struct Graph *g) {
    int components = findConnectedComponents(g);
    int cycles = g->edges - g->vertices + components;
    if (cycles < 0) cycles = 0;
    printf("Cycle rank (independent cycles): %d\n", cycles);
    return cycles;
}

INTERMEDIATE
Find Cycle Path
Printing the actual sequence of vertices forming a cycle

To find the actual cycle path, we use DFS with a parent tracking array. When we find a back edge to an already-visited vertex, we trace back through parents from that vertex to reconstruct the cycle path.

C — Find Cycle Path
int parent_fp[MAX_V];
int vis_fp[MAX_V];

void printCycle(int start, int end) {
    int path[MAX_V], len = 0, cur = end;
    while (cur != start) { path[len++] = cur; cur = parent_fp[cur]; }
    path[len++] = start;
    printf("Cycle: ");
    for (int i = len-1; i >= 0; i--)
        printf("V%d%s", path[i], i > 0 ? " → " : "");
    printf(" → V%d\n", start);
}

int dfs_path(struct Graph *g, int v, int par) {
    vis_fp[v] = 1;
    for (int e = 0; e < g->edges; e++) {
        if (!g->incMatrix[v][e]) continue;
        for (int nb = 0; nb < g->vertices; nb++) {
            if (nb == v || !g->incMatrix[nb][e]) continue;
            if (!vis_fp[nb]) {
                parent_fp[nb] = v;
                if (dfs_path(g, nb, v)) return 1;
            } else if (nb != par) {
                parent_fp[nb] = v; printCycle(nb, v); return 1;
            }
        }
    }
    return 0;
}

Check Self Loop
An edge connecting a vertex to itself

A self-loop is an edge where both endpoints are the same vertex. In the incidence matrix, a self-loop edge column has value 2 for that vertex (because the vertex appears as both endpoints). All other rows = 0.

Self-loop: incMatrix[v][e] == 2
C — Check Self Loop
/* Returns 1 if vertex v has a self-loop */
int checkSelfLoop(struct Graph *g, int v) {
    for (int e = 0; e < g->edges; e++) {
        if (g->incMatrix[v][e] == 2) {
            printf("V%d has a self-loop at E%d\n", v, e);
            return 1;
        }
    }
    printf("V%d has no self-loop.\n", v);
    return 0;
}

/* Insert a self-loop on vertex v */
void insertSelfLoop(struct Graph *g, int v) {
    int col = g->edges;
    g->incMatrix[v][col] = 2; /* self-loop convention */
    g->edges++;
}

INTERMEDIATE
Check Back Edge
The key indicator of a cycle in DFS

A back edge is an edge that points from a vertex to one of its ancestors in the DFS tree. Back edges are the hallmark of cycles. We use a color array: WHITE (unvisited), GRAY (in DFS stack), BLACK (done). A back edge exists if we reach a GRAY vertex.

C — Back Edge Detection
#define WHITE 0
#define GRAY  1
#define BLACK 2

int color[MAX_V];

int dfs_back(struct Graph *g, int v) {
    color[v] = GRAY; /* visiting */
    for (int e = 0; e < g->edges; e++) {
        if (g->incMatrix[v][e] != 1) continue; /* directed: tail */
        for (int nb = 0; nb < g->vertices; nb++) {
            if (nb == v || g->incMatrix[nb][e] != -1) continue;
            if (color[nb] == GRAY) {
                printf("Back edge: V%d → V%d (cycle!)\n", v, nb);
                return 1;
            }
            if (color[nb] == WHITE)
                if (dfs_back(g, nb)) return 1;
        }
    }
    color[v] = BLACK; /* done */
    return 0;
}

INTERMEDIATE
Detect Directed Cycle
Cycle detection in a directed (digraph) graph

In a directed graph, edge columns have +1 (tail) and -1 (head). A directed cycle means we can follow directions and return to the start. We use the color (WHITE/GRAY/BLACK) DFS method. GRAY means "currently in DFS call stack" — reaching GRAY = back edge = directed cycle.

C — Directed Cycle Detection
/* Directed cycle: uses color states */
int detectDirectedCycle(struct Graph *g) {
    for (int i = 0; i < g->vertices; i++) color[i] = WHITE;
    for (int v = 0; v < g->vertices; v++) {
        if (color[v] == WHITE)
            if (dfs_back(g, v)) {
                printf("Directed cycle DETECTED!\n");
                return 1;
            }
    }
    printf("No directed cycle found (DAG).\n");
    return 0;
}

Detect Undirected Cycle
Cycle detection in undirected graphs

For an undirected graph, every edge is bidirectional. A cycle exists if DFS finds a visited neighbor that is not the parent of the current vertex. We must track the parent to avoid treating the reverse edge as a back edge.

C — Undirected Cycle Detection
int detectUndirectedCycle(struct Graph *g) {
    int vis[MAX_V] = {0};
    for (int v = 0; v < g->vertices; v++)
        if (!vis[v])
            if (dfs_cycle(g, v, -1)) {
                printf("Undirected cycle FOUND!\n");
                return 1;
            }
    printf("No cycle in undirected graph.\n");
    return 0;
}

INTERMEDIATE
Remove Cycle
Breaking the cycle by removing back edges

To remove a cycle, we identify the back edge that creates it, then delete that edge using the delete edge operation. After removal, the graph becomes a DAG (directed acyclic graph) or a tree. This is essentially computing a spanning tree.

C — Remove Cycle (Remove Back Edge)
/* Remove the first back edge found (breaking the cycle) */
int backEdge_re = -1;

int dfs_find_back(struct Graph *g, int v, int par) {
    int vis[MAX_V] = {0}; vis[v] = 1;
    for (int e = 0; e < g->edges; e++) {
        if (!g->incMatrix[v][e]) continue;
        for (int nb = 0; nb < g->vertices; nb++) {
            if (nb != v && g->incMatrix[nb][e] != 0) {
                if (vis[nb] && nb != par) {
                    backEdge_re = e; return 1;
                }
            }
        }
    }
    return 0;
}

void removeCycle(struct Graph *g) {
    backEdge_re = -1;
    if (dfs_find_back(g, 0, -1) && backEdge_re != -1) {
        printf("Removing back edge E%d\n", backEdge_re);
        deleteEdge(g, backEdge_re);
    } else
        printf("No cycle to remove.\n");
}

C — Find Path (DFS + Backtracking)
int path_arr[MAX_V], pathLen;

int dfs_find_path(struct Graph *g, int v, int dest) {
    vis_p[v] = 1;
    path_arr[pathLen++] = v;
    if (v == dest) return 1; /* found! */

    for (int e = 0; e < g->edges; e++) {
        if (!g->incMatrix[v][e]) continue;
        for (int nb = 0; nb < g->vertices; nb++) {

                if (dfs_find_path(g, nb, dest)) return 1;
        }
    }
    pathLen--; /* backtrack */
    return 0;
}

    memset(vis_p, 0, sizeof(vis_p)); pathLen = 0;
    if (dfs_find_path(g, src, dst)) {
        printf("Path: ");
        for (int i = 0; i < pathLen; i++)
            printf("V%d%s", path_arr[i], i<pathLen-1?" → ":"\n");
    } else printf("No path found.\n");
}

Shortest Path
Finding the minimum-hop path using BFS

For unweighted graphs, BFS naturally finds the shortest path (fewest edges). We track the parent of each vertex during BFS. Once the destination is reached, we trace back through parents to reconstruct the path.

🌍 Real-life Analogy

GPS navigation — finding the route with fewest turns (hops) from your house to a restaurant. BFS explores all "1-hop" streets first, then "2-hop" streets, guaranteed to find the shortest.

C — Shortest Path (BFS)
void shortestPath(struct Graph *g, int src, int dst) {
    int vis[MAX_V]={0}, par[MAX_V], dist[MAX_V];
    int q[MAX_V], front=0, rear=0;
    for(int i=0;i<MAX_V;i++){par[i]=-1;dist[i]=-1;}

    vis[src]=1; dist[src]=0; q[rear++]=src;

    while(front<rear){
        int v=q[front++];
        for(int e=0;e<g->edges;e++){
            if(!g->incMatrix[v][e]) continue;
            for(int nb=0;nb<g->vertices;nb++){
                if(nb!=v && g->incMatrix[nb][e] && !vis[nb]){
                    vis[nb]=1; par[nb]=v;
                    dist[nb]=dist[v]+1;
                    q[rear++]=nb;
                }
            }
        }
    }
    if(dist[dst]==-1){ printf("No path\n"); return; }
    printf("Shortest dist: %d hops\n",dist[dst]);
    /* Trace path back */
    int path[MAX_V],len=0,cur=dst;
    while(cur!=-1){path[len++]=cur;cur=par[cur];}
    for(int i=len-1;i>=0;i--)
        printf("V%d%s",path[i],i?" → ":"\n");
}

        if(!vis[v]) return 0;
    return 1;
}

Tarjan's Algorithm
Find all Strongly Connected Components in O(V+E)

Tarjan's algorithm finds all Strongly Connected Components (SCCs) using a single DFS pass. It assigns each vertex a disc (discovery time) and low (lowest reachable disc). A vertex v is the root of an SCC when low[v] == disc[v].

🌍 Real-life Analogy

Finding groups of cities where you can travel between any two in both directions (bidirectional reachability). Each SCC is one such self-contained travel bubble.

▶ Tarjan SCC Visualization

C — Tarjan's SCC Algorithm
int disc_t[MAX_V], low_t[MAX_V], onStack[MAX_V];
int stk[MAX_V], stkTop=-1, timer_t=0;
int vis_t[MAX_V];

void tarjan_dfs(struct Graph *g, int v){
    disc_t[v] = low_t[v] = timer_t++;
    stk[++stkTop] = v; onStack[v] = 1;
    vis_t[v] = 1;

    for(int e=0;e<g->edges;e++){
        if(g->incMatrix[v][e]!=1) continue; /* directed: tail */
        for(int nb=0;nb<g->vertices;nb++){
            if(g->incMatrix[nb][e]!=-1) continue;
            if(!vis_t[nb]){
                tarjan_dfs(g,nb);
                if(low_t[nb]<low_t[v]) low_t[v]=low_t[nb];
            } else if(onStack[nb])
                if(disc_t[nb]<low_t[v]) low_t[v]=disc_t[nb];
        }
    }

    /* Root of SCC: pop the stack */
    if(low_t[v] == disc_t[v]){
        printf("SCC: { ");
        while(stk[stkTop] != v){
            int u=stk[stkTop--];
            onStack[u]=0; printf("V%d ",u);
        }
        onStack[stk[stkTop]]=0;
        printf("V%d }\n", stk[stkTop--]);
    }
}

void tarjanSCC(struct Graph *g){
    memset(vis_t,0,sizeof(vis_t));
    memset(onStack,0,sizeof(onStack));
    stkTop=-1; timer_t=0;
    for(int v=0;v<g->vertices;v++)
        if(!vis_t[v]) tarjan_dfs(g,v);
}

Kosaraju's Algorithm
Two-pass DFS for SCCs

Kosaraju's uses two DFS passes:
Pass 1: Run DFS, push vertices onto a stack in finish order.
Pass 2: Transpose (reverse) the graph, then run DFS in stack order — each DFS tree is one SCC.

C — Kosaraju's Algorithm
int finStk[MAX_V], finTop=-1;
int vis_k[MAX_V];

void dfs1(struct Graph *g, int v){
    vis_k[v]=1;
    for(int e=0;e<g->edges;e++){
        if(g->incMatrix[v][e]!=1) continue;
        for(int nb=0;nb<g->vertices;nb++)
            if(g->incMatrix[nb][e]==-1&&!vis_k[nb])
                dfs1(g,nb);
    }
    finStk[++finTop]=v; /* push at finish */
}

void dfs2(struct Graph *g, int v){
    /* On transposed graph (swap +1/-1) */
    vis_k[v]=1; printf("V%d ",v);
    for(int e=0;e<g->edges;e++){
        if(g->incMatrix[v][e]!=-1) continue; /* reversed */
        for(int nb=0;nb<g->vertices;nb++)
            if(g->incMatrix[nb][e]==1&&!vis_k[nb])
                dfs2(g,nb);
    }
}

void kosarajuSCC(struct Graph *g){
    /* Pass 1 */
    memset(vis_k,0,sizeof(vis_k)); finTop=-1;
    for(int v=0;v<g->vertices;v++)
        if(!vis_k[v]) dfs1(g,v);
    /* Pass 2 on transposed graph */
    memset(vis_k,0,sizeof(vis_k));
    while(finTop>=0){
        int v=finStk[finTop--];
        if(!vis_k[v]){
            printf("SCC: { ");
            dfs2(g,v);
            printf("}\n");
        }
    }
}

Floyd's Cycle Detection
Tortoise and Hare algorithm for cycle detection

Floyd's algorithm uses two pointers: a slow one (tortoise, moves 1 step) and a fast one (hare, moves 2 steps). If they meet, a cycle exists. Originally for linked lists, but applicable to graph traversal sequences. When applied to a path sequence in a graph, if slow == fast, there's a cycle.

🌍 Real-life Analogy

Two runners on a circular track — one fast, one slow. If the track is circular (cyclic), the fast runner will eventually lap and catch the slow runner. If it's a straight track (no cycle), the fast runner just finishes first.

C — Floyd's Cycle Detection (Path Sequence)
/* Get the "next" vertex in a deterministic walk
   (follow first available edge from v) */
int nextVertex(struct Graph *g, int v){
    for(int e=0;e<g->edges;e++){
        if(!g->incMatrix[v][e]) continue;
        for(int nb=0;nb<g->vertices;nb++)
            if(nb!=v&&g->incMatrix[nb][e])
                return nb;
    }
    return -1; /* no next (dead end) */
}

int floydCycleDetect(struct Graph *g){
    int slow=0, fast=0;
    do {
        slow = nextVertex(g, slow);          /* 1 step */
        int t = nextVertex(g, fast);
        fast = (t!=-1) ? nextVertex(g,t) : -1; /* 2 steps */
        if(slow==-1||fast==-1){
            printf("No cycle (dead end reached)\n");
            return 0;
        }
        printf("Tortoise@V%d  Hare@V%d\n",slow,fast);
    } while(slow != fast);
    printf("Cycle DETECTED at V%d!\n",slow);
    return 1;
}

int adjacent(struct Graph *g, int u, int v){

        if(g->incMatrix[u][e]&&g->incMatrix[v][e]) return 1;

void graphColoring(struct Graph *g){

    for(int i=0;i<MAX_V;i++) color[i]=-1;
    color[0]=0;

    for(int v=1;v<g->vertices;v++){
        int used[MAX_V]={0};
        /* Mark colors used by neighbors */
        for(int nb=0;nb<g->vertices;nb++)
            if(adjacent(g,v,nb)&&color[nb]!=-1)
                used[color[nb]]=1;
        /* Assign first available color */
        int c=0;
        while(used[c]) c++;
        color[v]=c;
    }

    printf("Graph Coloring:\n");
    int maxColor=0;
    for(int v=0;v<g->vertices;v++){
        printf("  V%d → Color %d\n",v,color[v]);
        if(color[v]>maxColor) maxColor=color[v];
    }
    printf("Chromatic number ≤ %d\n",maxColor+1);
}

    printf("Graph is NOT empty.\n"); return 0;
}

/* Clear graph */
void clearGraph(struct Graph *g){
    g->vertices=0; g->edges=0;
    memset(g->incMatrix,0,sizeof(g->incMatrix));
    printf("Graph cleared.\n");
}

/* Copy graph */
void copyGraph(struct Graph *src, struct Graph *dst){
    dst->vertices=src->vertices;
    dst->edges=src->edges;
    for(int v=0;v<src->vertices;v++)
        for(int e=0;e<src->edges;e++)
            dst->incMatrix[v][e]=src->incMatrix[v][e];
    printf("Graph copied successfully.\n");
}

/* Display full incidence matrix */
void displayGraph(struct Graph *g){
    printf("\nIncidence Matrix (%dV x %dE):\n",g->vertices,g->edges);
    printf("     ");
    for(int e=0;e<g->edges;e++) printf("E%-3d",e);
    printf("\n");
    for(int v=0;v<g->vertices;v++){
        printf("V%-3d |",v);
        for(int e=0;e<g->edges;e++)
            printf("%-4d",g->incMatrix[v][e]);
        printf("\n");
    }
}

/* Reset visited array */
void resetVisited(int *vis, int n){
    for(int i=0;i<n;i++) vis[i]=0;
    printf("Visited array reset.\n");
}
