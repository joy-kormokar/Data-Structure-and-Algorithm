What is a Fenwick Tree?

Fenwick Tree, also known as a Binary Indexed Tree (BIT), is a data structure invented by Peter Fenwick in 1994. It is a clever array-based structure that allows you to efficiently compute prefix sums and perform point updates — both in O(log n) time.

The key idea: instead of storing values directly, each position stores the sum of a range of elements. The size of that range is determined by the lowest set bit in the binary representation of the index.
💡 Real-Life Analogy
Imagine a bank ledger system. Instead of reading every transaction from page 1 to get your balance, the accountant maintains subtotals at strategic checkpoints — every 2 transactions, every 4, every 8, etc. To get your total, you only add 2–3 checkpoints, not all 1000 entries. That's the Fenwick Tree!
Update: O(log n) Query: O(log n) Space: O(n) Build: O(n log n)

01

Create Operations

🌱
Create Fenwick Tree
Creating a Fenwick Tree means declaring an integer array BIT[] of size n+1 (1-indexed). All cells start at 0. The tree "lives inside" this flat array — no pointers, no nodes!
🏗️ Analogy
Like buying a fresh notebook — you label pages 1 to n, all pages start blank (zero). You'll fill them as you add data.
C
/* Create a Fenwick Tree of size n */
#include <stdio.h>
#include <stdlib.h>

#define MAXN 1005

int BIT[MAXN]; /* The Fenwick Tree array */
int n;         /* Size of the tree */

void createTree(int size) {
    n = size;
    /* Set all elements to zero */
    for (int i = 0; i <= n; i++) {
        BIT[i] = 0;
    }
    printf("Fenwick Tree created with size %d\n", n);
}

int main() {
    createTree(8);
    return 0;
}
// VISUALIZATION — Empty Tree
📋
Create Array / Allocate Memory
The source array holds original values. The BIT array holds prefix sums. Memory can be static (fixed size at compile time) or dynamic (allocated at runtime using malloc).
📦 Analogy
Static = buying a fixed-size box. Dynamic = asking the warehouse for exactly as many boxes as you need.
C
/* Static allocation */
int arr[MAXN];  /* source array */
int BIT[MAXN];  /* Fenwick Tree */

/* Dynamic allocation */
int createDynamic(int size) {
    int *BIT_dyn = (int*)malloc((size + 1) * sizeof(int));
    if (BIT_dyn == NULL) {
        printf("Memory allocation failed!\n");
        return -1;
    }
    for (int i = 0; i <= size; i++)
        BIT_dyn[i] = 0;
    printf("Allocated %d integers\n", size + 1);
    free(BIT_dyn); /* free when done */
    return 0;
}
⚠️ Key Rule
Always use 1-based indexing. BIT[0] is never used because the lowest-set-bit trick requires index ≥ 1.
⚙️
Initialize Tree
Initialization sets every BIT cell to 0. This prepares the tree to receive updates. Think of it as clearing the slate before writing.
C
void initTree(int *BIT, int n) {
    for (int i = 0; i <= n; i++)
        BIT[i] = 0;
}

/* Using memset (faster) */
#include <string.h>
void initFast(int *BIT, int n) {
    memset(BIT, 0, (n + 1) * sizeof(int));
}

Build Operations

Building fills the BIT with actual data. The lowbit(i) = i & (-i) trick is at the heart of how the tree stores range sums. Each index i stores the sum of elements from i - lowbit(i) + 1 to i.

🔨
Build from Array
The simplest build: iterate through each element and call update(i, arr[i]). This takes O(n log n) time.
C
int BIT[MAXN], n;

/* Core update: add 'val' at index i */
void update(int i, int val) {
    for (; i <= n; i += i & (-i))
        BIT[i] += val;
}

/* Build BIT from array arr[] (1-indexed) */
void buildFromArray(int arr[], int size) {
    n = size;
    for (int i = 0; i <= n; i++)
        BIT[i] = 0;
    for (int i = 1; i <= n; i++)
        update(i, arr[i]);
}

int main() {
    int arr[] = {0,3,2,-1,6,5,4,-3,3};
    buildFromArray(arr, 8);
    return 0;
}
💡 The Magic Formula
i & (-i) extracts the lowest set bit of i.
Example: i=6 → binary 110 → lowest set bit = 010 = 2
So BIT[6] stores sum of arr[5..6] (2 elements)
⚡
Incremental / Prefix-Based Build
Incremental build: insert elements one at a time as they arrive — useful for streaming data.

Prefix-based build: an O(n) optimization — use prefix sums to fill BIT directly without calling update for each element.
C
/* Incremental build — O(n log n) */
void incrementalBuild(int arr[], int size) {
    n = size;
    for (int i = 1; i <= n; i++) {
        update(i, arr[i]); /* one by one */
    }
}

/* O(n) build using prefix sum trick */
void fastBuild(int arr[], int size) {
    n = size;
    for (int i = 1; i <= n; i++) {
        BIT[i] += arr[i];
        int j = i + (i & -i); /* parent */
        if (j <= n)
            BIT[j] += BIT[i];
    }
}

03

Traversal Operations

Traversal in a BIT is different from trees with left/right pointers. We navigate using index arithmetic — moving to parent or child by adding/removing the lowest set bit.
➡️
Forward Traversal
Moving from index i to its next update node: i += i & (-i). This is used during update operations to propagate changes upward.
// Forward path from index 3
// i=3 → 3+lowbit(3)=3+1=4 → 4+4=8 → done
C
/* Forward traversal (for update) */
void traverseForward(int i) {
    printf("Forward path from %d: ", i);
    while (i <= n) {
        printf("%d ", i);
        i += i & (-i); /* move to parent */
    }
    printf("\n");
}
⬅️
Backward Traversal / Prefix
Moving from index i toward the root: i -= i & (-i). This is used during query operations to accumulate prefix sums by stripping the lowest set bit.
// Backward path from index 7
// i=7 → 7-lowbit(7)=7-1=6 → 6-2=4 → 4-4=0 → done
C
/* Backward traversal (for query) */
void traverseBackward(int i) {
    printf("Prefix path from %d: ", i);
    while (i > 0) {
        printf("%d ", i);
        i -= i & (-i); /* strip lowest bit */
    }
    printf("\n");
}

Access Operations

🏠
Root, Parent, Child, Current Node
In a BIT, root = index 1 (the first element). Parent of i = i + lowbit(i). Child of i = i - lowbit(i) (conceptually). Current node = BIT[i] value.
C
/* Access operations */
int accessRoot() {
    return BIT[1]; /* root is at index 1 */
}

int accessParent(int i) {
    /* In update path, parent is at i + lowbit(i) */
    int parent = i + (i & -i);
    if (parent > n) return -1; /* no parent */
    return BIT[parent];
}

int accessChild(int i) {
    /* In query path, child is at i - lowbit(i) */
    int child = i - (i & -i);
    if (child <= 0) return -1; /* no child */
    return BIT[child];
}

int accessCurrentNode(int i) {
    return BIT[i]; /* direct array access */
}

Access Prefix Value
The prefix value at index i is the sum of all elements from index 1 to i. We compute it by walking backward from i, accumulating BIT values.
C
/* Get prefix sum from 1 to i */
int prefixSum(int i) {
    int sum = 0;
    while (i > 0) {
        sum += BIT[i];
        i -= i & (-i); /* go to previous range */
    }
    return sum;
}

Σ
Prefix Sum Query
Sum from index 1 to i. Walk from i backward, subtracting the lowest set bit each time, accumulating BIT values. Time: O(log n)
C
/* Prefix sum: sum of arr[1..i] */
int query(int i) {
    int sum = 0;
    while (i > 0) {
        sum += BIT[i];
        i -= i & (-i);
    }
    return sum;
}
/* Example: query(4) for arr=[3,2,-1,6] */
/* = BIT[4] = 10 */
↔️
Range Sum Query
Sum from index l to r. Use two prefix queries: query(r) - query(l-1). Time: O(log n)
C
/* Range sum: sum of arr[l..r] */
int rangeQuery(int l, int r) {
    return query(r) - query(l - 1);
}

/* Example usage */
int main() {
    /* arr = [3,2,-1,6,5,4,-3,3] */
    int r = rangeQuery(2, 6); /* 2+(-1)+6+5+4 = 16 */
    printf("Range sum [2,6] = %d\n", r);
    return 0;
}
📍
Point Query
Get the value of a single element at index i: query(i) - query(i-1). This works when only point updates are done (not range updates).
C
/* Point query: get value at index i */
int pointQuery(int i) {
    return query(i) - query(i - 1);
}

/* OR if you maintain original array: */
int arr[MAXN]; /* keep original */
int pointQueryDirect(int i) {
    return arr[i]; /* O(1) if arr maintained */
}
📈
Cumulative / Partial Sum Query
Cumulative frequency query: how many elements ≤ x? Useful in frequency counting.
Partial sum query: sum of a specific portion, combining prefix queries.
C
/* Cumulative frequency up to value x */
/* BIT stores frequencies of each value */
int freq[MAXN]; /* freq[v] = count of v */

void addValue(int v) {
    update(v, 1); /* increment frequency */
}

int cumulativeFreq(int x) {
    return query(x); /* count of vals <= x */
}

/* Partial sum: specific window */
int partialSum(int l, int r) {
    return query(r) - query(l - 1);
}

Update Operations

All updates use the forward traversal pattern: start at index i, add delta to BIT[i], then move to i + lowbit(i), repeat until i > n. This propagates the change to all responsible ranges.

🔺
Point / Increment / Decrement Update
C
/* Point update: add delta to arr[i] */
void update(int i, int delta) {
    for (; i <= n; i += i & (-i))
        BIT[i] += delta;
}

/* Increment: arr[i] += 1 */
void increment(int i) {
    update(i, 1);
}

/* Decrement: arr[i] -= 1 */
void decrement(int i) {
    update(i, -1);
}

/* Add value v to arr[i] */
void addVal(int i, int v) {
    update(i, v);
}

/* Subtract value v from arr[i] */
void subtractVal(int i, int v) {
    update(i, -v);
}
🔄
Replace Value
To replace arr[i] with a new value, we need the delta: update(i, newVal - arr[i]). Always keep a copy of the original array to compute the difference.
C
int arr[MAXN]; /* keep original values */

/* Replace arr[i] with newVal */
void replaceValue(int i, int newVal) {
    int delta = newVal - arr[i];
    arr[i] = newVal;       /* update source array */
    update(i, delta);     /* update BIT with difference */
}

/* Example: replace arr[3]=-1 with 10 */
/* delta = 10-(-1) = 11, update BIT by +11 */
📌 Why the difference?
The BIT stores sums, not direct values. To change arr[3] from -1 to 10, we add 11 to all BIT nodes responsible for index 3.

Search Operations

🔎
Search Element / Search Prefix Sum
Search element: linear scan — BIT doesn't support direct element search efficiently.
Search prefix sum: binary search on the BIT to find the index where prefix sum = target.
C
/* Search for element (linear O(n)) */
int searchElement(int arr[], int val) {
    for (int i = 1; i <= n; i++)
        if (arr[i] == val) return i;
    return -1; /* not found */
}

/* Find smallest i where prefix sum >= target */
int searchPrefixSum(int target) {
    int pos = 0, bitMask;
    for (bitMask = 1<<20; bitMask; bitMask >>= 1) {
        int next = pos + bitMask;
        if (next <= n && BIT[next] < target) {
            pos = next;
            target -= BIT[pos];
        }
    }
    return pos + 1;
}
🏆
Find K-th Element
Given a frequency BIT, find the k-th smallest element. Walk the BIT in O(log n) using binary lifting — the most elegant BIT trick!
C
/* Find k-th smallest element — O(log n) */
/* BIT stores frequencies: BIT[v] = freq of v */
int findKth(int k) {
    int pos = 0;
    int logN = 0;
    
    /* Find highest power of 2 <= n */
    while ((1 << (logN + 1)) <= n) logN++;
    
    for (int i = logN; i >= 0; i--) {
        int next = pos + (1 << i);
        if (next <= n && BIT[next] < k) {
            k -= BIT[next];
            pos = next;
        }
    }
    return pos + 1; /* k-th element */
}
🏅 Use Case
Used in competitive programming to find median or specific rank in a dynamically changing sorted list.
⬆️
Binary Lifting Search
Binary Lifting is a technique that halves the search space at each step by testing powers of 2. It allows finding positions in O(log n) without a separate binary search.
C
/* Binary lifting on BIT */
int binaryLift(int target) {
    int idx = 0;
    for (int pw = (1<<17); pw; pw >>= 1) {
        if (idx + pw <= n &&
            BIT[idx + pw] < target) {
            idx += pw;
            target -= BIT[idx];
        }
    }
    return idx + 1;
}
⬇️⬆️
Lower / Upper Bound Search
Lower bound: first index where prefix_sum ≥ target.
Upper bound: first index where prefix_sum > target.
C
/* Lower bound: first i with prefix >= target */
int lowerBound(int target) {
    int pos = 0;
    for (int pw = 1<<20; pw; pw >>= 1) {
        if (pos+pw <= n && BIT[pos+pw] < target) {
            pos += pw;
            target -= BIT[pos];
        }
    }
    return pos + 1;
}

/* Upper bound: first i with prefix > target */
int upperBound(int target) {
    return lowerBound(target + 1);
}

Range Operations

Range update (add a value to all elements in [l,r]) requires a difference array trick. We maintain a BIT of differences so that range updates and point/range queries both stay O(log n).
💡 Real-Life Analogy
A teacher gives +5 bonus marks to all students from seat 3 to seat 7. Instead of updating each student's record individually, she notes "+5 starts at seat 3" and "−5 starts at seat 8". The bookkeeper reconstructs everyone's score in one pass.
🔃
Range Update + Point Query
Use a difference BIT: update(l, +v) and update(r+1, -v). Then a point query at i = prefix_sum(i).
C
int BIT_diff[MAXN]; /* difference BIT */

void rangeUpdate(int l, int r, int v) {
    update_bit(BIT_diff, l, v);
    update_bit(BIT_diff, r + 1, -v);
}

int pointQueryAfterRangeUpdate(int i) {
    return prefix_query(BIT_diff, i);
}

📐
Range Update + Range Query
Uses two BITs: BIT1 and BIT2. For range sum after range updates, a formula involving both BITs gives the correct result.
C
int BIT1[MAXN], BIT2[MAXN];

void add(int i, int v) {
    for (int x=i; x<=n; x+=x&-x) {
        BIT1[x] += v;
        BIT2[x] += v * (i - 1);
    }
}

void rangeAdd(int l, int r, int v) {
    add(l, v);
    add(r + 1, -v);
}

int prefQ(int i) {
    int s = 0;
    for (int x=i; x>0; x-=x&-x)
        s += BIT1[x] * i - BIT2[x];
    return s;
}

int rangeSum(int l, int r) {
    return prefQ(r) - prefQ(l - 1);
}

Bit Manipulation Operations

Fenwick Trees are built entirely on bitwise operations. Understanding i & (-i) is the key to understanding everything. Let's demystify it!
// INTERACTIVE — Lowbit Calculator

⚡
Get Lowest Set Bit
i & (-i) extracts the rightmost set bit. This tells us how many consecutive elements BIT[i] is responsible for.
C
/* Get lowest set bit of i */
int lowbit(int i) {
    return i & (-i);
}
/*
  How it works:
  i  =  6 → binary: 0110
  -i = -6 → 2's complement: 1010
  i&(-i)  = 0010 = 2 ✓

  i  =  12 → binary: 1100
  -i = -12 → 2's comp: 0100
  i&(-i)   = 0100 = 4 ✓
*/
👆
Parent / Next Index Calculation
Parent (update path): i + lowbit(i) — move upward to next node responsible for i.
Prev (query path): i - lowbit(i) — move backward to previous range.
C
/* Next index during update */
int nextIndex(int i) {
    return i + (i & -i);
}

/* Previous index during query */
int prevIndex(int i) {
    return i - (i & -i);
}

/* Index jump: skip entire subtree */
int indexJump(int i, int steps) {
    while (steps--) i = nextIndex(i);
    return i;
}

Validation Operations

✅
Check Empty / Validate Index / Range
Always validate inputs before operations to prevent buffer overflow or undefined behavior. A BIT is "empty" when all values are 0.
C
/* Check if BIT is empty */
int isEmpty() {
    for (int i = 1; i <= n; i++)
        if (BIT[i] != 0) return 0;
    return 1;
}

/* Validate single index */
int validateIndex(int i) {
    return (i >= 1 && i <= n);
}

/* Validate range [l, r] */
int validateRange(int l, int r) {
    return (l >= 1 && r <= n && l <= r);
}

/* Boundary check with error message */
void safeUpdate(int i, int v) {
    if (!validateIndex(i)) {
        printf("Error: index %d out of bounds!\n", i);
        return;
    }
    update(i, v);
}

int safeQuery(int l, int r) {
    if (!validateRange(l, r)) {
        printf("Error: invalid range [%d,%d]!\n",l,r);
        return -1;
    }
    return rangeQuery(l, r);
}
🛡️ Best Practice
Always check bounds in competitive programming too — even one wrong index causes subtle wrong answers that are hard to debug.

    memset(BIT, 0, (n+1)*sizeof(int));

    clearTree();
    n = 0;

void deleteTree() {
    free(BIT_dyn);
    BIT_dyn = NULL;
    n = 0;
}
♻️ Memory Best Practices
Always free dynamically allocated memory. Use memset (not a loop) for clearing — it's much faster and maps directly to CPU instructions.

Advanced Operations

🔲
2D Fenwick Tree
Extend to 2D: BIT[i][j] stores cumulative sum over a rectangle. Update and query both run in O(log n × log m).
🗺️ Use Case
Count the number of points in a 2D grid rectangle. Used in computational geometry and image processing.
C
int BIT2D[105][105];
int N, M; /* rows and columns */

void update2D(int x, int y, int v) {
    for (int i=x; i<=N; i+=i&-i)
        for (int j=y; j<=M; j+=j&-j)
            BIT2D[i][j] += v;
}

int query2D(int x, int y) {
    int sum = 0;
    for (int i=x; i>0; i-=i&-i)
        for (int j=y; j>0; j-=j&-j)
            sum += BIT2D[i][j];
    return sum;
}

/* Rectangle sum: (r1,c1) to (r2,c2) */
int rectSum(int r1,int c1,int r2,int c2) {
    return query2D(r2,c2)
          -query2D(r1-1,c2)
          -query2D(r2,c1-1)
          +query2D(r1-1,c1-1);
}
// 2D BIT Grid (5×5)
🗜️
Compressed Fenwick Tree
When values are large (e.g., up to 10⁹) but sparse (only N distinct values), we compress coordinates to 1..N first, then use a BIT of size N.
C
#include <stdlib.h>

int vals[MAXN]; /* large values */
int sorted[MAXN];

int cmp(const void* a, const void* b) {
    return (*(int*)a) - (*(int*)b);
}

/* Compress values to 1..n */
void compress(int arr[], int n) {
    memcpy(sorted, arr, n * sizeof(int));
    qsort(sorted, n, sizeof(int), cmp);
    for (int i = 0; i < n; i++) {
        /* binary search for rank */
        int lo=0, hi=n-1, rank=-1;
        while(lo<=hi) {
            int mid=(lo+hi)/2;
            if(sorted[mid]==arr[i]) {rank=mid+1;break;}
            else if(sorted[mid]<arr[i]) lo=mid+1;
            else hi=mid-1;
        }
        arr[i] = rank;
    }
}

/* Frequency BIT for values 1..MAXV */

🔀
Inversion Counting
An inversion is a pair (i, j) where i < j but arr[i] > arr[j]. BIT counts inversions in O(n log n) — essential in sorting algorithm analysis.
🃏 Analogy
In a deck of cards, inversions = how "shuffled" it is. A sorted deck has 0 inversions. A reversed deck has n(n-1)/2 inversions.
C
/* Count inversions in arr[] — O(n log n) */
int countInversions(int arr[], int n) {
    int inversions = 0;
    int maxVal = 0;
    
    /* Find max for BIT size */
    for (int i=0; i<n; i++)
        if (arr[i] > maxVal) maxVal = arr[i];
    
    /* Reset BIT to maxVal size */
    memset(BIT, 0, (maxVal+1)*sizeof(int));
    
    for (int i = n-1; i >= 0; i--) {
        /* Count elements < arr[i] seen so far */
        inversions += query(arr[i] - 1);
        update_bit(arr[i], 1);
    }
    return inversions;
}

Optimization Operations

Fast Update / Fast Query
Micro-optimizations to squeeze out every nanosecond in competitive programming:
  • Use register int for loop variables
  • Avoid function call overhead by inlining
  • Prefer i += i & -i over computing lowbit separately
C
/* Optimized update — inline, no extra vars */
inline void fastUpdate(int i, int v) {
    for (; i <= n; i += i & -i)
        BIT[i] += v;
}

/* Optimized query — no function call */
inline int fastQuery(int i) {
    int s = 0;
    for (; i > 0; i -= i & -i)
        s += BIT[i];
    return s;
}

/* Prefix optimization: precompute all prefix sums */
void prefixOptimize(int prefix[]) {
    for (int i = 1; i <= n; i++)
        prefix[i] = fastQuery(i);
}

/* Binary Indexed Optimization: O(n) build */
void binaryIndexedOptimize(int arr[]) {
    for (int i = 1; i <= n; i++) {
        BIT[i] += arr[i];
        int j = i + (i & -i);
        if (j <= n) BIT[j] += BIT[i];
    }
}

Competitive Programming Operations

🗜️
Coordinate Compression
When values span [1, 10⁹] but there are only N ≤ 10⁵ queries, map values to [1, N] first. This reduces BIT size drastically.
C
#include <stdlib.h>

int vals[MAXN], comp[MAXN], m;

int cmpInt(const void* a, const void* b) {
    return (*(int*)a) - (*(int*)b);
}

void coordinateCompress(int n) {
    memcpy(comp, vals, n*sizeof(int));
    qsort(comp, n, sizeof(int), cmpInt);
    /* remove duplicates */
    m = 0;
    for (int i=0; i<n; i++)
        if (!m || comp[m-1] != comp[i])
            comp[m++] = comp[i];
    /* map vals[i] to rank in [1..m] */
    for (int i=0; i<n; i++) {
        int lo=0, hi=m-1;
        while(lo<hi) {
            int mid=(lo+hi)/2;
            if(comp[mid]<vals[i]) lo=mid+1;
            else hi=mid;
        }
        vals[i] = lo + 1;
    }
}
📦
Offline Query Processing
Offline queries: read all queries first, sort/group them, then answer together — often faster than online processing.
C
typedef struct {
    int l, r, idx;
} Query;

Query queries[MAXN];
int ans[MAXN];

int cmpQuery(const void* a, const void* b) {
    return ((Query*)a)->r - ((Query*)b)->r;
}

void offlineProcess(int arr[], int q) {
    /* Sort queries by right endpoint */
    qsort(queries, q, sizeof(Query), cmpQuery);
    
    int qi = 0;
    for (int i = 1; i <= n; i++) {
        update(i, arr[i]);
        while (qi < q && queries[qi].r == i) {
            ans[queries[qi].idx] =
                rangeQuery(queries[qi].l, i);
            qi++;
        }
    }
}

Complete BIT Template

The complete, competition-ready Fenwick Tree implementation in C with all essential operations in one place.
C — Complete Template
#include <stdio.h>
#include <string.h>

#define MAXN 100005

int BIT[MAXN], arr[MAXN], n;

/* ── CORE ─────────────────────────────── */
void update(int i, int v) {
    for (; i <= n; i += i&-i) BIT[i] += v;
}

int query(int i) {
    int s = 0;
    for (; i > 0; i -= i&-i) s += BIT[i];
    return s;
}

int rangeQuery(int l, int r) {
    return query(r) - query(l-1);
}

/* ── BUILD ────────────────────────────── */
void build(int a[], int sz) {
    n = sz;
    memset(BIT, 0, (n+1)*sizeof(int));
    for (int i=1; i<=n; i++) {
        BIT[i] += a[i];
        int j = i+(i&-i);
        if (j<=n) BIT[j] += BIT[i];
    }
}

/* ── REPLACE ──────────────────────────── */
void replace(int i, int v) {
    update(i, v - arr[i]);
    arr[i] = v;
}

/* ── K-TH ─────────────────────────────── */
int kth(int k) {
    int pos = 0;
    for (int pw=(1<<17); pw; pw>>=1)
        if (pos+pw<=n && BIT[pos+pw]<k) {
            k -= BIT[pos+=pw];
        }
    return pos+1;
}

/* ── MAIN ─────────────────────────────── */
int main() {
    int a[] = {0,3,2,-1,6,5,4,-3,3};
    memcpy(arr, a, sizeof(a));
    build(a, 8);

    printf("Prefix sum [1..5] = %d\n", query(5));
    printf("Range sum [2..6]  = %d\n", rangeQuery(2,6));

    replace(3, 10);
    printf("After update: [1..3] = %d\n", query(3));

    return 0;
}
