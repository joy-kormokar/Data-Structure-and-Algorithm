What is a Doubly Linked List?
Beginner
Understanding the Concept

A Doubly Linked List (DLL) is a sequence of elements called nodes. Each node has three parts:

  • PREV pointer — points to the previous node (or NULL if it's the first)
  • DATA — the actual value stored in the node
  • NEXT pointer — points to the next node (or NULL if it's the last)

Unlike a Singly Linked List which can only go forward, a DLL lets you go both forward and backward through the list — making many operations faster and easier.

🚂 Real-Life Analogy

Think of a train! Each carriage (node) has a front coupling connecting to the next carriage (NEXT), a back coupling connecting to the previous carriage (PREV), and passengers inside (DATA). You can walk from front to back or back to front through the couplings!

1. Creation Operations
Beginner
📦 Create Node
1.1 — Create Node
Theory

Creating a node means allocating memory for one node structure using malloc() and filling in its data, prev (NULL initially), and next (NULL initially) fields. This is the atomic unit of a DLL.

🏠 Real-Life Analogy

Creating a node is like building an empty room with two doors — one door facing the previous room and one facing the next room. Initially both doors open to a dead end (NULL). You place your furniture (data) inside, then later connect rooms via the doors (pointers).

C — Create Node 
/* Step 1: Define the Node structure */
struct Node {
    int data;            // stores the value
    struct Node* prev;   // pointer to previous node
    struct Node* next;   // pointer to next node
};

/* Step 2: Function to create a new node */
struct Node* createNode(int value) {
    /* malloc allocates memory of node's size */
    struct Node* newNode = (struct Node*) malloc(sizeof(struct Node));

    if (newNode == NULL) {           // always check malloc
        printf("Memory error!\n");
        return NULL;
    }

    newNode->data = value;    // store the value
    newNode->prev = NULL;    // no previous node yet
    newNode->next = NULL;    // no next node yet

    return newNode;           // return the node's address
}

🔧 Initialize List
1.2 — Initialize List
Theory

Initializing the list means setting the head pointer to NULL. The head pointer is a global pointer to the first node of the list. An empty list has head = NULL. This is your starting point before inserting any nodes.

C — Initialize List 
/* The head pointer keeps track of the start of the list */
struct Node* head = NULL;  // empty list: head points to nothing

/* Optional: function-based initialization */
void initList(struct Node** headRef) {
    *headRef = NULL;    // dereference and set to NULL
    printf("List initialized (empty)\n");
}

/* Usage */
int main() {
    struct Node* head = NULL;   // start with empty list
    initList(&head);            // or use function
    return 0;
}

2. Insertion Operations
Beginner
💡 Insertion in a DLL requires careful pointer manipulation. You must update both the PREV and NEXT pointers of neighboring nodes. Always handle edge cases like empty list and single-node list.
▸ Interactive DLL — Insert Operations
List is empty. Add some nodes!
⬅️ Insert at Beginning
2.1 — Insert at Beginning
Theory

To insert at the beginning: create a new node, set its next to the current head, set its prev to NULL, update the old head's prev to the new node (if head wasn't NULL), then make head point to the new node. Time complexity: O(1).

🚶 Real-Life Analogy

Imagine a queue of people. Inserting at beginning is like cutting to the front of the line. You attach yourself in front of the first person — the person who was first now has you behind them (they update their "prev"), and you point forward to them.

C — Insert at Beginning
void insertBegin(struct Node** head, int value) {
    struct Node* newNode = createNode(value);

    /* Step 1: new node's next points to current head */
    newNode->next = *head;

    /* Step 2: new node's prev is NULL (it's the first) */
    newNode->prev = NULL;

    /* Step 3: if list wasn't empty, update old head's prev */
    if (*head != NULL) {
        (*head)->prev = newNode;
    }

    /* Step 4: move head to point to new node */
    *head = newNode;

    printf("Inserted %d at beginning\n", value);
}
➡️ Insert at End
2.2 — Insert at End
Theory

Traverse to the last node (where next == NULL), then link the new node after it. The new node's next = NULL, the last node's next = new node, and new node's prev = last node. Time: O(n) unless you maintain a tail pointer (then O(1)).

C — Insert at End
void insertEnd(struct Node** head, int value) {
    struct Node* newNode = createNode(value);

    /* Case 1: list is empty */
    if (*head == NULL) {
        *head = newNode;
        return;
    }

    /* Case 2: traverse to last node */
    struct Node* temp = *head;
    while (temp->next != NULL) {
        temp = temp->next;   // move to next node
    }

    /* Now temp is the last node */
    temp->next = newNode;    // last node points to new
    newNode->prev = temp;    // new node points back to last
    newNode->next = NULL;   // new node is the last now

    printf("Inserted %d at end\n", value);
}
📍 Insert at Specific Position
2.3 — Insert at Position
Theory

Traverse to the node at (position - 1), then insert the new node between that node and its next. Update 4 pointers: newNode->prev, newNode->next, prevNode->next, and nextNode->prev. Time: O(n).

C — Insert at Position
void insertAtPos(struct Node** head, int value, int pos) {
    if (pos == 1) {               // position 1 = beginning
        insertBegin(head, value);
        return;
    }

    struct Node* newNode = createNode(value);
    struct Node* temp = *head;

    /* Traverse to node just BEFORE the position */
    for (int i = 1; i < pos - 1 && temp != NULL; i++) {
        temp = temp->next;
    }

    if (temp == NULL) {
        printf("Position out of range!\n");
        return;
    }

    /* Insert between temp and temp->next */
    newNode->next = temp->next;   // new -> what was after temp
    newNode->prev = temp;         // new <- temp

    if (temp->next != NULL) {     // if there's a node after
        temp->next->prev = newNode;  // that node's prev = new
    }

    temp->next = newNode;         // temp -> new
}

⬅️ Delete from Beginning
3.1 — Delete from Beginning
Theory

Move head to the second node, set the new head's prev to NULL, then free the old head node. Time: O(1). Always free memory with free() to avoid memory leaks.

C — Delete from Beginning
void deleteBegin(struct Node** head) {
    if (*head == NULL) {
        printf("List is empty!\n");
        return;
    }

    struct Node* temp = *head;   // save old head

    *head = temp->next;           // head moves to next node

    if (*head != NULL) {
        (*head)->prev = NULL;   // new head has no previous
    }

    printf("Deleted %d\n", temp->data);
    free(temp);                   // release memory
}
➡️ Delete from End
3.2 — Delete from End
C — Delete from End
void deleteEnd(struct Node** head) {
    if (*head == NULL) { printf("Empty!\n"); return; }

    struct Node* temp = *head;

    /* If only one node exists */
    if (temp->next == NULL) {
        *head = NULL;
        free(temp);
        return;
    }

    /* Traverse to the last node */
    while (temp->next != NULL) {
        temp = temp->next;
    }

    /* temp->prev is now second-to-last node */
    temp->prev->next = NULL;   // cut the last node off
    printf("Deleted %d\n", temp->data);
    free(temp);
}
📍 Delete at Specific Position
3.3 — Delete at Position
C — Delete at Position
void deleteAtPos(struct Node** head, int pos) {
    if (*head == NULL) return;
    if (pos == 1) { deleteBegin(head); return; }

    struct Node* temp = *head;

    /* Move to the node at given position */
    for (int i = 1; i < pos && temp != NULL; i++) {
        temp = temp->next;
    }

    if (temp == NULL) { printf("Position out of range!\n"); return; }

    /* Connect previous and next nodes to each other */
    if (temp->prev) temp->prev->next = temp->next;
    if (temp->next) temp->next->prev = temp->prev;

    free(temp);
}

/* Delete by Value */
void deleteByValue(struct Node** head, int value) {
    struct Node* temp = *head;

    while (temp != NULL && temp->data != value) {
        temp = temp->next;
    }

    if (temp == NULL) { printf("Value not found!\n"); return; }

    if (temp == *head) { deleteBegin(head); return; }
    if (temp->prev) temp->prev->next = temp->next;
    if (temp->next) temp->next->prev = temp->prev;

    free(temp);
}

Beginner
▸ Interactive Traversal Visualization
▶ Forward Traversal
4.1 — Forward Traversal
Theory

Start at the head node. Print the data, then follow the next pointer to the next node. Repeat until you reach NULL. Time: O(n).

📖 Real-Life Analogy

Reading a book page by page from front to back. Each page (node) has an arrow pointing to the next page. You start at page 1 (head) and follow the arrows until there are no more pages (NULL).

C — Forward & Backward Traversal
/* Forward: head → NULL */
void forwardTraversal(struct Node* head) {
    if (head == NULL) { printf("List is empty!\n"); return; }

    struct Node* temp = head;   // start at head

    printf("Forward: NULL ← ");
    while (temp != NULL) {
        printf("%d ↔ ", temp->data);
        temp = temp->next;          // move to next
    }
    printf("NULL\n");
}

/* Backward: tail → head (using prev pointer) */
void backwardTraversal(struct Node* head) {
    if (head == NULL) return;

    /* First: reach the last node */
    struct Node* temp = head;
    while (temp->next != NULL) {
        temp = temp->next;
    }

    /* Now traverse backward using prev */
    printf("Backward: NULL ← ");
    while (temp != NULL) {
        printf("%d ↔ ", temp->data);
        temp = temp->prev;          // move to previous!
    }
    printf("NULL\n");
}

int searchByValue(struct Node* head, int val) {

6. Update Node Data
Beginner
▸ Update Visualization
Theory

Search for the node with the target value (or at a given position), then simply assign the new value to its data field. No pointer changes needed — only the data changes. Time: O(n) for search + O(1) for update.

📝 Real-Life Analogy

Like updating a contact's phone number in your address book. You find the contact's card (search), then erase the old number and write the new one (update data). The card's position in the book doesn't change.

C — Update Node Data
void updateNode(struct Node* head, int oldVal, int newVal) {
    struct Node* temp = head;

    while (temp != NULL) {
        if (temp->data == oldVal) {
            temp->data = newVal;     // just update the value!
            printf("Updated %d → %d\n", oldVal, newVal);
            return;
        }
        temp = temp->next;
    }
    printf("Value %d not found!\n", oldVal);
}

7. Counting Operations
Beginner
Theory — Count Total Nodes

Traverse the entire list from head to NULL, incrementing a counter for each node visited. This gives the total length of the list. Time: O(n). Tip: maintain a size variable updated at each insertion/deletion for O(1) counting.

🧮 Real-Life Analogy

Like counting cars in a parking lot by walking through each row. You start at the entrance (head) and count each car (node) until you reach the end of the lot (NULL).

C — Count Total Nodes
int countNodes(struct Node* head) {
    int count = 0;
    struct Node* temp = head;

    while (temp != NULL) {
        count++;              // increment for each node
        temp = temp->next;    // move to next
    }

    printf("Total nodes: %d\n", count);
    return count;
}

/* Optimized: maintain a size variable */
int size = 0;   // increment on insert, decrement on delete

8. Sorting Operations
Intermediate
▸ Sort Visualization (Bubble Sort on DLL)
Theory — Bubble Sort on DLL

We use Bubble Sort because it's easiest to understand and implement on linked lists. Repeatedly compare adjacent nodes and swap their data (not the nodes themselves — swapping data is simpler!). After each pass, the largest unsorted element "bubbles" to the end. Time: O(n²).

🫧 Real-Life Analogy

Like arranging students by height in a line. You walk from left to right comparing neighboring students — if a taller student is to the left of a shorter one, they swap. You repeat this until the line is sorted. Heaviest "bubbles" to the end each time!

C — Ascending & Descending Sort
/* Ascending Sort (Bubble Sort) */
void sortAscending(struct Node* head) {
    if (head == NULL) return;

    int swapped;
    struct Node* ptr;

    do {
        swapped = 0;
        ptr = head;

        while (ptr->next != NULL) {
            /* Compare adjacent nodes */
            if (ptr->data > ptr->next->data) {
                /* Swap the data values */
                int temp = ptr->data;
                ptr->data = ptr->next->data;
                ptr->next->data = temp;
                swapped = 1;    // mark that a swap happened
            }
            ptr = ptr->next;
        }
    } while (swapped);   // repeat until no swaps
}

/* Descending Sort — just flip the comparison */
void sortDescending(struct Node* head) {
    int swapped; struct Node* ptr;
    do {
        swapped = 0; ptr = head;
        while (ptr->next != NULL) {
            if (ptr->data < ptr->next->data) {  // flip !
                int temp = ptr->data;
                ptr->data = ptr->next->data;
                ptr->next->data = temp;
                swapped = 1;
            }
            ptr = ptr->next;
        }
    } while (swapped);
}

9. Reversing Operations
Intermediate
▸ Reverse Visualization
Theory — Reverse Entire List

In a DLL, reversing is elegant: for every node, swap its PREV and NEXT pointers. Then move to what used to be the previous node (now stored in next). Finally, set head to the last node we visited. Time: O(n).

🚗 Real-Life Analogy

Like a train reversing direction. Each carriage's front coupling becomes the back coupling and vice versa. The engine (head) becomes the caboose, and the caboose becomes the new engine!

C — Reverse Entire List
void reverseList(struct Node** head) {
    if (*head == NULL || (*head)->next == NULL) return;

    struct Node* current = *head;
    struct Node* temp = NULL;

    while (current != NULL) {
        /* Swap the prev and next pointers */
        temp = current->prev;
        current->prev = current->next;
        current->next = temp;

        /* Move to what was the previous node */
        current = current->prev;   // prev now holds old next
    }

    /* temp now points to old head's prev = new head */
    if (temp != NULL) {
        *head = temp->prev;        // update head
    }
}

/* Simpler understanding: after swap loop,
   the last node visited before NULL becomes the new head */

🔀 Merge Two Lists
10.1 — Merge Two Lists
Theory

Connect the tail of list1 to the head of list2. Update the tail's next to list2's head, and list2's head's prev to the tail. Time: O(n) to find tail.

C — Merge Two Lists
struct Node* mergeLists(
    struct Node* h1,
    struct Node* h2) {

    if (h1 == NULL) return h2;
    if (h2 == NULL) return h1;

    /* Find tail of list1 */
    struct Node* tail = h1;
    while (tail->next != NULL)
        tail = tail->next;

    /* Connect tail → h2 */
    tail->next = h2;
    h2->prev = tail;

    return h1;
}
✅ Merge Sorted Lists
10.2 — Merge Sorted Lists
Theory

Merge two already-sorted lists into one sorted list. Compare head nodes, pick the smaller, advance that list's pointer. Build the result list using both prev and next pointers. Time: O(n+m).

C — Merge Sorted Lists
struct Node* mergeSorted(
    struct Node* h1,
    struct Node* h2) {

    if (!h1) return h2;
    if (!h2) return h1;

    struct Node* result = NULL;
    struct Node* tail = NULL;

    while (h1 && h2) {
        struct Node* chosen;
        if (h1->data <= h2->data) {
            chosen = h1; h1 = h1->next;
        } else {
            chosen = h2; h2 = h2->next;
        }
        chosen->prev = tail;
        chosen->next = NULL;
        if (!result) result = chosen;
        else tail->next = chosen;
        tail = chosen;
    }
    /* Attach remaining nodes */
    if (h1) { tail->next=h1; h1->prev=tail; }
    if (h2) { tail->next=h2; h2->prev=tail; }
    return result;
}

11. Concatenation
Intermediate
Theory — Concatenate Two Lists

Concatenation is similar to merging — you attach the end of List 1 to the beginning of List 2. The key difference from merge is that the original lists no longer exist as separate entities. The second list's head's prev is updated to point to list1's tail, forming one continuous list. Time: O(n).

📎 Real-Life Analogy

Like connecting two paper chains together. You clip the last ring of the first chain to the first ring of the second chain — now they're one long chain. The joint is permanent.

C — Concatenate Two Lists
void concatenate(struct Node** head1, struct Node* head2) {
    if (*head1 == NULL) {  // list1 is empty
        *head1 = head2;
        return;
    }
    if (head2 == NULL) return;  // list2 is empty

    /* Traverse to end of list1 */
    struct Node* tail = *head1;
    while (tail->next != NULL) {
        tail = tail->next;
    }

    /* Join: tail of list1 ↔ head of list2 */
    tail->next = head2;     // list1's tail → list2's head
    head2->prev = tail;     // list2's head ← list1's tail

    printf("Lists concatenated!\n");
    /* head1 now points to combined list */
}

Theory — Split into Two Halves

Use the Fast-Slow pointer technique: slow moves one step at a time, fast moves two steps. When fast reaches the end, slow is at the midpoint. Then cut the list at slow's position by setting slow->next = NULL and adjusting the second half's head's prev to NULL. Time: O(n).

C — Split into Two Halves
void splitList(struct Node* head,
              struct Node** first,
              struct Node** second) {
    if (head == NULL || head->next == NULL) {
        *first = head;
        *second = NULL;
        return;
    }

    struct Node* slow = head;   // moves 1 step
    struct Node* fast = head;   // moves 2 steps

    /* Fast-Slow pointer: find middle */
    while (fast->next != NULL && fast->next->next != NULL) {
        slow = slow->next;          // slow: 1 step
        fast = fast->next->next;    // fast: 2 steps
    }

    /* slow is now at the middle node */
    *first = head;              // first half starts at head
    *second = slow->next;       // second half starts at slow+1

    /* Cut the list at midpoint */
    slow->next = NULL;
    if (*second != NULL) {
        (*second)->prev = NULL;   // second half has no prev now
    }
}

13. Copying Operations
Advanced
🔗 Shallow Copy
13.1 — Shallow Copy
Theory

A shallow copy creates a new pointer that points to the same nodes. Changes to the copy affect the original — they share the same memory. In C, this is as simple as assigning the head pointer: copy = original. Both variables point to the same list.

📸 Analogy

Like giving someone a copy of a library's card catalog. Both catalogs point to the SAME books. If someone updates a book, both catalogs reflect the change.

C — Shallow Copy
/* Shallow copy - same memory */
struct Node* shallowCopy(struct Node* head) {
    /* Just return the same head pointer */
    return head;
    /* WARNING: both pointers now share
       the same nodes in memory!
       Modifying one changes the other. */
}

/* Usage: */
struct Node* copy = head;  // shallow!
🧬 Deep Copy
13.2 — Deep Copy
Theory

A deep copy creates brand new nodes with the same data values. The copy is completely independent — changes to one list don't affect the other. You must allocate new memory for each node. Time: O(n).

📸 Analogy

Like printing an entirely new copy of every page in a book. Both books have the same content, but are physically separate. Writing in one doesn't affect the other.

C — Deep Copy
struct Node* deepCopy(struct Node* head) {
    if (head == NULL) return NULL;

    struct Node* newHead = NULL;
    struct Node* tail = NULL;
    struct Node* curr = head;

    while (curr != NULL) {
        /* Allocate a brand new node */
        struct Node* newNode =
            createNode(curr->data);

        if (newHead == NULL) {
            newHead = newNode;
        } else {
            tail->next = newNode;
            newNode->prev = tail;
        }
        tail = newNode;
        curr = curr->next;
    }
    return newHead;  // independent copy!
}

15. Duplicate Handling
Intermediate
▸ Remove Duplicates Visualization
Theory — Remove Duplicates

If the list is sorted, traverse once: if a node's data equals its neighbor's data, delete the neighbor. If unsorted, use a hash set to track seen values, and delete any node whose value was seen before. Time: O(n) sorted, O(n²) unsorted without extra space.

📚 Real-Life Analogy

Like removing duplicate books from a sorted library shelf. As you walk along, if you see the same title twice in a row, you pull the extra copy off the shelf. Easy when sorted — hard when books are scattered randomly.

C — Remove Duplicates (Sorted List)
void removeDuplicates(struct Node** head) {
    struct Node* curr = *head;

    while (curr != NULL && curr->next != NULL) {

        if (curr->data == curr->next->data) {
            /* Found duplicate! Remove curr->next */
            struct Node* dup = curr->next;

            curr->next = dup->next;        // skip the duplicate

            if (dup->next != NULL) {
                dup->next->prev = curr;    // update prev link
            }

            free(dup);   // release duplicate's memory
            /* Don't advance curr, check next pair again */

        } else {
            curr = curr->next;   // move forward only if no dup
        }
    }
}

16. Rotation Operations
Advanced
▸ Rotation Visualization
Theory — Left & Right Rotation

Left rotation by k: Move the first k nodes to the end. The (k+1)th node becomes the new head. Right rotation by k: Move the last k nodes to the front. The (n-k)th node's next becomes the new head. The key trick: find the rotation point, then rearrange the head and tail connections.

⚙️ Real-Life Analogy

Like rotating a carousel of slides. Left rotation means the slides at the front get moved to the back. Right rotation means the slides at the back jump to the front. The content doesn't change — just their positions in the sequence.

C — Left & Right Rotation
/* Left Rotation by k positions */
void leftRotate(struct Node** head, int k) {
    if (*head == NULL || k == 0) return;

    /* Step 1: Find the kth node (new tail of first part) */
    struct Node* kth = *head;
    for (int i = 1; i < k && kth->next; i++)
        kth = kth->next;

    if (kth->next == NULL) return;  // k >= length

    /* Step 2: Find the tail of the whole list */
    struct Node* tail = kth;
    while (tail->next) tail = tail->next;

    /* Step 3: Rearrange connections */
    tail->next = *head;          // tail → old head
    (*head)->prev = tail;        // old head ← tail
    *head = kth->next;           // new head = kth+1
    (*head)->prev = NULL;        // new head has no prev
    kth->next = NULL;            // kth is now the new tail
}

/* Right Rotation by k: left rotate by (n-k) */
void rightRotate(struct Node** head, int k) {
    int n = countNodes(*head);
    if (n == 0) return;
    k = k % n;             // handle k >= n
    if (k == 0) return;
    leftRotate(head, n - k);  // right = left by (n-k)
}

▸ Find Middle Node & Palindrome Check
🎯 Find Middle Node
17.1 — Find Middle Node
Theory

Use the Fast-Slow pointer technique: slow moves 1 step, fast moves 2 steps. When fast reaches the end, slow is at the middle. For even-length lists, slow lands at the first of the two middle nodes. Time: O(n).

🐢🐇 Analogy

Two runners race on a track: the hare runs twice as fast as the tortoise. When the hare finishes, the tortoise is at the halfway point — that's your middle node!

C — Find Middle Node
struct Node* findMiddle(struct Node* head) {
    if (head == NULL) return NULL;

    struct Node* slow = head;   // 1 step at a time
    struct Node* fast = head;   // 2 steps at a time

    while (fast != NULL && fast->next != NULL) {
        slow = slow->next;
        fast = fast->next->next;
    }

    /* slow is now at middle node */
    printf("Middle node: %d\n", slow->data);
    return slow;
}
🔄 Palindrome Check
17.2 — Palindrome Check
Theory

A DLL makes palindrome checking elegant! Use two pointers: front starting at head and back starting at tail. Compare their data — if they match, move front forward and back backward. If any mismatch, not a palindrome. Time: O(n). The DLL's prev pointer makes backward traversal trivial!

C — Palindrome Check
int isPalindrome(struct Node* head) {
    if (head == NULL) return 1;

    /* Find the tail (last node) */
    struct Node* tail = head;
    while (tail->next != NULL)
        tail = tail->next;

    struct Node* front = head;
    struct Node* back = tail;

    /* Compare from both ends towards center */
    while (front != back && back->next != front) {
        if (front->data != back->data) {
            printf("NOT a palindrome!\n");
            return 0;   // mismatch found
        }
        front = front->next;   // move forward
        back = back->prev;     // move backward
    }

    printf("IS a palindrome!\n");
    return 1;
}

18. Memory Management Operations
Advanced
⚠️ Critical: Every node you create with malloc() must be freed with free() when no longer needed. Memory leaks in C are bugs that waste system memory and can crash programs over time. Always clean up your linked lists!
💥 Destroy List
18.1 — Destroy List
Theory

Destroy List frees every single node in the list AND sets head to NULL. The list is completely gone from memory. This is a permanent operation — use it when you're done with the list forever.

💣 Analogy

Like demolishing an entire building. Every room (node) is torn down and the land (memory) is returned to be used for something else. The address (head) is zeroed out — nothing remains.

C — Destroy List
void destroyList(struct Node** head) {
    struct Node* curr = *head;
    struct Node* nextNode;

    while (curr != NULL) {
        nextNode = curr->next; // save next before free
        free(curr);            // release this node
        curr = nextNode;       // move to saved next
    }

    *head = NULL;   // mark list as empty
    printf("List destroyed. All memory freed.\n");
}
🧹 Clear List
18.2 — Clear List
Theory

Clear List also frees all nodes, but the difference is conceptual: after clearing, the list structure still exists (head variable remains) but points to NULL — it's an empty list ready to be reused, not destroyed.

🗑️ Analogy

Like emptying a warehouse. All the goods (data/nodes) are removed and the space freed, but the warehouse building (list structure) still stands — ready to receive new goods. The address (head) exists but points to an empty space.

C — Clear List
void clearList(struct Node** head) {
    struct Node* curr = *head;

    while (curr != NULL) {
        struct Node* next = curr->next;
        free(curr);    // free each node
        curr = next;
    }

    *head = NULL;   // list is now empty but reusable
    printf("List cleared. Ready for new data.\n");
}

/* Difference:
   destroyList: list is gone forever
   clearList  : list is empty but can be refilled */
🎓 Complete DLL Program
All operations in one file

Here is a complete, beginner-friendly C program that puts all the core DLL operations together — create, insert, delete, search, traverse, and destroy. Copy and run this in any C compiler (GCC, Code::Blocks, Dev-C++).

C — Complete DLL Program
/* ===================================
   Complete Doubly Linked List in C
   =================================== */
#include <stdio.h>
#include <stdlib.h>

/* Node structure */
struct Node {
    int data;
    struct Node* prev;
    struct Node* next;
};

/* Create a new node */
struct Node* createNode(int val) {
    struct Node* n = malloc(sizeof(struct Node));
    n->data = val; n->prev = n->next = NULL;
    return n;
}

/* Insert at beginning */
void insertBegin(struct Node** h, int v) {
    struct Node* n = createNode(v);
    n->next = *h;
    if (*h) (*h)->prev = n;
    *h = n;
}

/* Insert at end */
void insertEnd(struct Node** h, int v) {
    struct Node* n = createNode(v);
    if (!*h) { *h = n; return; }
    struct Node* t = *h;
    while (t->next) t = t->next;
    t->next = n; n->prev = t;
}

/* Print forward */
void print(struct Node* h) {
    printf("List: NULL ↔ ");
    while (h) { printf("%d ↔ ", h->data); h=h->next; }
    printf("NULL\n");
}

/* Destroy entire list */
void destroy(struct Node** h) {
    struct Node* c = *h, *nx;
    while (c) { nx=c->next; free(c); c=nx; }
    *h = NULL;
}

int main() {
    struct Node* head = NULL;  // initialize empty list

    insertEnd(&head, 10);
    insertEnd(&head, 20);
    insertEnd(&head, 30);
    insertBegin(&head, 5);

    print(head);   // NULL ↔ 5 ↔ 10 ↔ 20 ↔ 30 ↔ NULL

    destroy(&head);
    printf("After destroy: head = %p\n", head);  // NULL

    return 0;
}
