01 — FOUNDATIONS

Directed Graph

A complete guide from beginner to advanced using Adjacency Matrix in C

🧭What is a Directed Graph?

A Directed Graph (Digraph) is a set of vertices (nodes) connected by edges that have a direction. Unlike undirected graphs, an edge from vertex A to vertex B does NOT imply an edge from B to A.

// KEY TERMS

Vertex (Node): An entity in the graph (e.g., a city, a person, a task).
Directed Edge (Arc): A one-way connection from source to destination.
Adjacency Matrix: A 2D grid where cell [i][j] = 1 (or weight) if there's an edge from i → j, else 0.

// REAL-LIFE ANALOGY

Think of a one-way road system. You can drive from A → B, but not necessarily from B → A. Twitter followers work the same way — you can follow someone without them following you back.

02 — REPRESENTATION

Adjacency Matrix

The core data structure used throughout this guide

📊Matrix Representation

An Adjacency Matrix for a directed graph with V vertices is a V×V 2D array where:

  • matrix[i][j] = 1 (or weight) → edge exists from vertex i to vertex j
  • matrix[i][j] = 0 → no edge from i to j
// IMPORTANT NOTE

In a directed graph, matrix[i][j] and matrix[j][i] can be different. The matrix is NOT always symmetric.

OperationTime Complexity
Check edge (u,v)O(1)
Add / Remove edgeO(1)
Get neighbors of vO(V)
SpaceO(V²)
// ANALOGY

Imagine a spreadsheet where rows are senders and columns are receivers. A checkmark in cell (A, B) means "A sends a message to B."

C — GRAPH STRUCTURE
#include <stdio.h>
#include <string.h>

#define MAX 10   /* maximum number of vertices */

/* Graph structure using adjacency matrix */
struct Graph {
    int  matrix[MAX][MAX]; /* 0 = no edge, weight otherwise */
    char vertex[MAX];      /* vertex labels: A, B, C, ...   */
    int  count;            /* number of vertices            */
};

/* Initialize an empty graph */
void initGraph(struct Graph *g) {
    g->count = 0;
    memset(g->matrix, 0, sizeof(g->matrix));
}

int main() {
    struct Graph g;
    initGraph(&g);
    printf("Graph initialized with %d vertices\n", g.count);
    return 0;
}
03 — VERTEX OPERATIONS

Insert Vertex

Add a new node to the graph

➕Insert Vertex

Inserting a vertex adds a new row and a new column to the adjacency matrix. All new entries are initialized to 0 (no edges yet).

  • Check if the graph is already full (count == MAX)
  • Add the vertex label to the vertex array
  • Initialize the new row and column to all zeros
  • Increment the vertex count
// ANALOGY

Like adding a new city to a road map. The city exists but has no roads yet — those come when you add edges.

// INSERT VERTEX DEMO
Label:
Graph has 3 vertices: A, B, C
C — INSERT VERTEX
void insertVertex(struct Graph *g, char label) {
    if (g->count >= MAX) {
        printf("Graph is full!\n");
        return;
    }
    int i = g->count;  /* index of the new vertex */

    g->vertex[i] = label;

    /* initialize the new row (outgoing edges) */
    for (int j = 0; j <= i; j++)
        g->matrix[i][j] = 0;

    /* initialize the new column (incoming edges) */
    for (int j = 0; j < i; j++)
        g->matrix[j][i] = 0;

    g->count++;
    printf("Vertex '%c' inserted at index %d\n", label, i);
}
🗑️Delete Vertex

Deleting a vertex requires removing its row and column from the matrix and shifting remaining rows/columns. All edges involving the vertex are automatically removed.

  • Find the index of the vertex to delete
  • Shift all rows after the deleted row upward
  • Shift all columns after the deleted column leftward
  • Remove the vertex label and decrement count
// ANALOGY

Like demolishing a city — all roads leading to and from it must also be removed from the map.

C — DELETE VERTEX
void deleteVertex(struct Graph *g, char label) {
    int idx = -1;
    for (int i = 0; i < g->count; i++)
        if (g->vertex[i] == label) { idx = i; break; }

    if (idx == -1) { printf("Vertex not found!\n"); return; }

    /* shift rows up */
    for (int i = idx; i < g->count - 1; i++) {
        g->vertex[i] = g->vertex[i + 1];
        for (int j = 0; j < g->count; j++)
            g->matrix[i][j] = g->matrix[i+1][j];
    }
    /* shift columns left */
    for (int j = idx; j < g->count - 1; j++)
        for (int i = 0; i < g->count - 1; i++)
            g->matrix[i][j] = g->matrix[i][j+1];

    g->count--;
    printf("Vertex '%c' deleted.\n", label);
}
🔍Search Vertex

Searching for a vertex means scanning the vertex labels array to find the index corresponding to a given label. Returns -1 if not found.

// ANALOGY

Like looking up a city in an index before querying its connections on the map.

C — SEARCH VERTEX
int searchVertex(struct Graph *g, char label) {
    for (int i = 0; i < g->count; i++)
        if (g->vertex[i] == label)
            return i;  /* found at index i */
    return -1;           /* not found       */
}

/* usage */
int idx = searchVertex(&g, 'B');
if (idx != -1)
    printf("Vertex 'B' found at index %d\n", idx);
else
    printf("Vertex 'B' not found.\n");
✏️Update Vertex

Updating a vertex simply renames its label in the vertex array. The matrix structure remains unchanged because it uses indices, not labels, internally.

C — UPDATE VERTEX
void updateVertex(struct Graph *g,
                   char oldLabel, char newLabel) {
    int idx = searchVertex(g, oldLabel);
    if (idx == -1) {
        printf("Vertex '%c' not found!\n", oldLabel);
        return;
    }
    g->vertex[idx] = newLabel;
    printf("Vertex '%c' renamed to '%c'\n",
           oldLabel, newLabel);
}

04 — EDGE OPERATIONS

Insert Directed Edge

Add a one-way connection between two vertices

➡️Insert Directed Edge

To insert a directed edge from vertex u → v with weight w, simply set matrix[u][v] = w. Only this single cell changes — matrix[v][u] remains unchanged (directional!).

// KEY POINT

Setting matrix[u][v] = 1 does NOT automatically set matrix[v][u] = 1. That's what makes it directed.

// ANALOGY

Adding a one-way street from City A to City B. Cars can go A→B but not B→A unless you explicitly add that road too.

// INSERT DIRECTED EDGE (Weighted)
From: To: Weight:
Enter from/to indices (0-3) and click Insert Edge.
C — INSERT DIRECTED EDGE
void insertEdge(struct Graph *g,
               char src, char dst, int weight) {
    int u = searchVertex(g, src);
    int v = searchVertex(g, dst);

    if (u == -1 || v == -1) {
        printf("One or both vertices not found!\n");
        return;
    }
    g->matrix[u][v] = weight;  /* only one cell! */
    printf("Edge %c->%c (weight=%d) added.\n",
           src, dst, weight);
}
✂️Delete Directed Edge

To delete a directed edge u→v, set matrix[u][v] = 0. The reverse direction is unaffected.

// ANALOGY

Closing a one-way road. The road in the opposite direction (if it existed) stays open.

C — DELETE DIRECTED EDGE
void deleteEdge(struct Graph *g,
               char src, char dst) {
    int u = searchVertex(g, src);
    int v = searchVertex(g, dst);

    if (u == -1 || v == -1) {
        printf("Vertices not found!\n");
        return;
    }
    if (g->matrix[u][v] == 0) {
        printf("Edge %c->%c does not exist!\n", src, dst);
        return;
    }
    g->matrix[u][v] = 0;
    printf("Edge %c->%c deleted.\n", src, dst);
}
🔎Search Edge

Checking if an edge u→v exists is a O(1) operation — just check matrix[u][v] != 0. This is the biggest advantage of adjacency matrix over adjacency list.

C — SEARCH EDGE
int searchEdge(struct Graph *g,
              char src, char dst) {
    int u = searchVertex(g, src);
    int v = searchVertex(g, dst);
    if (u == -1 || v == -1) return -1;
    return g->matrix[u][v];  /* 0 = no edge, else weight */
}

/* usage */
int w = searchEdge(&g, 'A', 'C');
if (w) printf("Edge A->C exists, weight = %d\n", w);
else   printf("No edge from A to C.\n");
⚖️Update Edge Weight

Updating the weight of a directed edge u→v replaces the existing non-zero value in matrix[u][v] with a new weight. If the edge doesn't exist, report an error.

// ANALOGY

Changing the speed limit on a one-way road — the road direction stays the same, only the cost to travel changes.

C — UPDATE EDGE WEIGHT
void updateEdgeWeight(struct Graph *g,
                      char src, char dst, int newWeight) {
    int u = searchVertex(g, src);
    int v = searchVertex(g, dst);
    if (u == -1 || v == -1) {
        printf("Vertices not found!\n"); return;
    }
    if (g->matrix[u][v] == 0) {
        printf("Edge does not exist!\n"); return;
    }
    g->matrix[u][v] = newWeight;
    printf("Edge %c->%c weight updated to %d\n",
           src, dst, newWeight);
}
📏Count Edges

Count all non-zero entries in the full V×V matrix. Each non-zero entry represents one directed edge. Time complexity: O(V²).

C — COUNT EDGES
int countEdges(struct Graph *g) {
    int count = 0;
    for (int i = 0; i < g->count; i++)
        for (int j = 0; j < g->count; j++)
            if (g->matrix[i][j] != 0)
                count++;
    return count;
}
printf("Total directed edges: %d\n", countEdges(&g));
🗺️Display Edges

Iterate over the matrix. For every non-zero entry at [i][j], print the directed edge vertex[i] → vertex[j] and its weight.

C — DISPLAY EDGES
void displayEdges(struct Graph *g) {
    printf("Directed Edges (src -> dst : weight):\n");
    for (int i = 0; i < g->count; i++)
        for (int j = 0; j < g->count; j++)
            if (g->matrix[i][j] != 0)
                printf("  %c -> %c : %d\n",
                       g->vertex[i],
                       g->vertex[j],
                       g->matrix[i][j]);
}
05 — TRAVERSAL

Breadth First Search

Explore the graph level by level using a queue

🌊BFS — Queue-based Traversal

BFS visits all neighbors of the current vertex before moving deeper. It uses a queue (FIFO) to track which vertex to visit next.

  • Queue Initialization: Enqueue the start vertex. Mark it visited.
  • Dequeue the front vertex. Visit (print) it.
  • Traverse Neighbors: For each neighbor j where matrix[current][j] != 0 and j is not visited — enqueue j and mark visited.
  • Repeat until queue is empty.
// COMPLEXITY

Time: O(V²) with adjacency matrix  |  Space: O(V) for queue and visited array

// ANALOGY

Imagine dropping a stone in water — the ripples spread outward in rings. BFS visits nodes ring by ring (level by level) from the starting point.

// BFS ANIMATION — CLICK START TO ANIMATE
Press Start to begin BFS traversal...
C — BFS WITH QUEUE
void BFS(struct Graph *g, char startLabel) {
    int start = searchVertex(g, startLabel);
    if (start == -1) { printf("Vertex not found!\n"); return; }

    int visited[MAX] = {0};
    int queue[MAX], front = 0, rear = 0;

    /* enqueue start */
    queue[rear++] = start;
    visited[start] = 1;

    printf("BFS: ");
    while (front < rear) {
        int curr = queue[front++];  /* dequeue */
        printf("%c ", g->vertex[curr]);

        /* visit all unvisited neighbors */
        for (int j = 0; j < g->count; j++) {
            if (g->matrix[curr][j] && !visited[j]) {
                visited[j] = 1;
                queue[rear++] = j;
            }
        }
    }
    printf("\n");
}
🏔️Depth First Search — Recursive & Stack

DFS dives as deep as possible along one path before backtracking. It uses a stack (implicit via recursion, or explicit).

  • Recursive DFS: Visit node, mark visited, recurse on each unvisited neighbor.
  • Stack DFS: Push start node. While stack not empty: pop, visit if unvisited, push unvisited neighbors.
  • Backtracking: When no unvisited neighbor exists, return (backtrack) to the previous node.
// COMPLEXITY

Time: O(V²) with adjacency matrix  |  Space: O(V) for recursion stack and visited array

// ANALOGY

Like exploring a maze — go as far down one corridor as possible. If you hit a dead end, backtrack to the last junction and try another path.

// DFS ANIMATION
Press Start to begin DFS traversal...
C — DFS RECURSIVE + STACK
/* --- Recursive DFS --- */
void dfsHelper(struct Graph *g, int v, int visited[]) {
    visited[v] = 1;
    printf("%c ", g->vertex[v]);
    for (int j = 0; j < g->count; j++)
        if (g->matrix[v][j] && !visited[j])
            dfsHelper(g, j, visited);  /* recurse (backtrack automatic) */
}

void DFS_Recursive(struct Graph *g, char startLabel) {
    int start = searchVertex(g, startLabel);
    int visited[MAX] = {0};
    printf("DFS: ");
    dfsHelper(g, start, visited);
    printf("\n");
}

/* --- Stack-based DFS --- */
void DFS_Stack(struct Graph *g, char startLabel) {
    int start = searchVertex(g, startLabel);
    int visited[MAX] = {0};
    int stack[MAX], top = -1;

    stack[++top] = start;
    printf("DFS (Stack): ");
    while (top >= 0) {
        int v = stack[top--];
        if (visited[v]) continue;
        visited[v] = 1;
        printf("%c ", g->vertex[v]);
        for (int j = g->count - 1; j >= 0; j--)
            if (g->matrix[v][j] && !visited[j])
                stack[++top] = j;
    }
    printf("\n");
}
06 — DEGREE OPERATIONS

In / Out / Total Degree

Count how many edges connect to each vertex

📐Degree Calculations

In-degree of vertex v = number of edges coming into v = count of non-zeros in column v.

Out-degree of vertex v = number of edges going out from v = count of non-zeros in row v.

Total Degree = In-degree + Out-degree.

// THEOREM

Sum of all out-degrees = Sum of all in-degrees = Total number of directed edges in the graph.

// ANALOGY

In a social network: in-degree = number of followers, out-degree = number of people you follow, total degree = your total connections.

// DEGREE CALCULATOR
C — DEGREE OPERATIONS
/* In-degree: count non-zeros in column v */
int inDegree(struct Graph *g, int v) {
    int deg = 0;
    for (int i = 0; i < g->count; i++)
        if (g->matrix[i][v]) deg++;
    return deg;
}

/* Out-degree: count non-zeros in row v */
int outDegree(struct Graph *g, int v) {
    int deg = 0;
    for (int j = 0; j < g->count; j++)
        if (g->matrix[v][j]) deg++;
    return deg;
}

/* Total degree */
int totalDegree(struct Graph *g, int v) {
    return inDegree(g, v) + outDegree(g, v);
}

/* Display degrees for all vertices */
void displayDegrees(struct Graph *g) {
    printf("%-6s %-8s %-10s %-12s\n",
           "Vertex", "In-Deg", "Out-Deg", "Total-Deg");
    for (int i = 0; i < g->count; i++)
        printf("  %-6c %-8d %-10d %-12d\n",
               g->vertex[i],
               inDegree(g, i),
               outDegree(g, i),
               totalDegree(g, i));
}
📊Max & Min Degree

Iterate through all vertices, compute each vertex's out-degree (or in-degree), track the maximum and minimum seen so far.

// ANALOGY

Finding the most followed person (max in-degree hub) and the least followed person (min in-degree) in a social network.

C — MAX & MIN DEGREE
void maxMinDegree(struct Graph *g) {
    int maxD = -1, minD = MAX*2+1;
    int maxV = 0, minV = 0;

    for (int i = 0; i < g->count; i++) {
        int d = totalDegree(g, i);
        if (d > maxD) { maxD = d; maxV = i; }
        if (d < minD) { minD = d; minV = i; }
    }
    printf("Max degree: %c (%d)\n", g->vertex[maxV], maxD);
    printf("Min degree: %c (%d)\n", g->vertex[minV], minD);
}
07 — PATH OPERATIONS

Find Path & All Paths

Discover routes between vertices in the directed graph

🛣️Find Path & All Paths

Find Path: Use DFS to find any one path between source and destination. Returns true if a path exists.

All Paths: Use DFS with backtracking to enumerate every possible path from source to destination. Store the current path in an array, add to solutions when destination is reached.

// KEY CONCEPT

For All Paths, after visiting a neighbor, unmark it as visited (backtrack) so other paths can use it.

// ANALOGY

Like finding all possible routes from home to work on a city map — some use highways, some use side streets, but all reach the same destination.

// PATH FINDER — 5 NODE GRAPH
From: To:
Select start and destination, then click Find All Paths.
C — FIND PATH & ALL PATHS
/* ---- Find ONE path using DFS ---- */
int findPath(struct Graph *g, int src,
             int dst, int visited[]) {
    if (src == dst) { printf("%c\n", g->vertex[dst]); return 1; }
    visited[src] = 1;
    for (int j = 0; j < g->count; j++) {
        if (g->matrix[src][j] && !visited[j]) {
            printf("%c -> ", g->vertex[src]);
            if (findPath(g, j, dst, visited)) return 1;
        }
    }
    return 0;  /* no path from src */
}

/* ---- Find ALL paths (backtracking) ---- */
void allPathsHelper(struct Graph *g, int curr,
                    int dst, int visited[],
                    int path[], int pLen) {
    path[pLen] = curr;
    if (curr == dst) {
        for (int i = 0; i <= pLen; i++)
            printf("%c%s", g->vertex[path[i]],
                   i < pLen ? " -> " : "\n");
        return;
    }
    visited[curr] = 1;
    for (int j = 0; j < g->count; j++)
        if (g->matrix[curr][j] && !visited[j])
            allPathsHelper(g, j, dst, visited, path, pLen+1);
    visited[curr] = 0;  /* BACKTRACK */
}

void allPaths(struct Graph *g, char src, char dst) {
    int s = searchVertex(g, src);
    int d = searchVertex(g, dst);
    int visited[MAX] = {0}, path[MAX];
    printf("All paths from %c to %c:\n", src, dst);
    allPathsHelper(g, s, d, visited, path, 0);
}
🎯Reachability Check

A vertex v is reachable from u if there exists any directed path from u → v. Use BFS or DFS from u; if v is visited, it is reachable. This is the basis of the Warshall transitive closure algorithm.

// ANALOGY

Can you drive from City A to City B following only one-way roads? Reachability answers yes or no.

C — REACHABILITY CHECK
int isReachable(struct Graph *g,
               char src, char dst) {
    int s = searchVertex(g, src);
    int d = searchVertex(g, dst);
    if (s == -1 || d == -1) return 0;

    int visited[MAX] = {0};
    int queue[MAX], front=0, rear=0;
    queue[rear++] = s;
    visited[s] = 1;

    while (front < rear) {
        int curr = queue[front++];
        if (curr == d) return 1;  /* found! */
        for (int j = 0; j < g->count; j++)
            if (g->matrix[curr][j] && !visited[j]) {
                visited[j] = 1;
                queue[rear++] = j;
            }
    }
    return 0;  /* not reachable */
}
💰Path Cost Calculation

For a weighted directed graph, the cost of a path is the sum of edge weights along that path. Given a specific path (sequence of vertices), accumulate matrix[path[i]][path[i+1]] for each consecutive pair.

// ANALOGY

Planning a road trip — each road segment has a toll/distance. Total cost = sum of all tolls along your chosen route.

C — PATH COST
/* path[] = array of vertex indices, len = number of vertices */
int pathCost(struct Graph *g,
             int path[], int len) {
    int cost = 0;
    for (int i = 0; i < len - 1; i++) {
        int u = path[i], v = path[i+1];
        if (g->matrix[u][v] == 0) {
            printf("No edge %c->%c! Invalid path.\n",
                   g->vertex[u], g->vertex[v]);
            return -1;
        }
        cost += g->matrix[u][v];
    }
    return cost;
}

/* usage: path A->C->D */
int p[] = {0, 2, 3};
printf("Path cost = %d\n", pathCost(&g, p, 3));
08 — CONNECTIVITY

Strong & Weak Connectivity

How well-connected is your directed graph?

🔗Strong & Weak Connectivity Check

Strongly Connected: Every vertex can reach every other vertex following directed edges. Checked by running DFS from any node, then DFS on the transposed graph — if both cover all vertices, it's strongly connected.

Weakly Connected: The graph would be connected if all edges were made undirected (bidirectional). Less strict than strong connectivity.

// ALGORITHM IDEA

1. Run DFS from vertex 0. If all nodes visited → check 1.
2. Transpose the graph (reverse all edges).
3. Run DFS on transposed graph from vertex 0. If all visited → Strongly Connected.

// ANALOGY

Strong: Everyone in a city can reach everyone else via one-way roads.
Weak: Everyone could reach everyone if roads were two-way.

C — STRONG / WEAK CONNECTIVITY
void dfs(struct Graph *g, int v, int visited[]) {
    visited[v] = 1;
    for (int j = 0; j < g->count; j++)
        if (g->matrix[v][j] && !visited[j])
            dfs(g, j, visited);
}

int isStronglyConnected(struct Graph *g) {
    int visited[MAX] = {0};
    dfs(g, 0, visited);
    for (int i = 0; i < g->count; i++)
        if (!visited[i]) return 0;

    /* Transpose the graph */
    struct Graph trans = *g;
    for (int i = 0; i < g->count; i++)
        for (int j = 0; j < g->count; j++)
            trans.matrix[i][j] = g->matrix[j][i];

    memset(visited, 0, sizeof(visited));
    dfs(&trans, 0, visited);
    for (int i = 0; i < g->count; i++)
        if (!visited[i]) return 0;

    return 1;  /* strongly connected */
}

/* Weak connectivity: treat edges as undirected */
int isWeaklyConnected(struct Graph *g) {
    struct Graph undirected = *g;
    for (int i = 0; i < g->count; i++)
        for (int j = 0; j < g->count; j++)
            if (g->matrix[i][j])
                undirected.matrix[j][i] = 1;

    int visited[MAX] = {0};
    dfs(&undirected, 0, visited);
    for (int i = 0; i < g->count; i++)
        if (!visited[i]) return 0;
    return 1;
}
🧩Strongly Connected Components (Kosaraju's)

A Strongly Connected Component (SCC) is a maximal set of vertices such that every vertex is reachable from every other vertex within the set.

Kosaraju's Algorithm (3 steps):

  • Do DFS on the original graph. Push each vertex to a stack in finish-time order.
  • Transpose the graph (reverse all edges).
  • Pop vertices from stack. Do DFS on the transposed graph in that order. Each DFS tree is one SCC.
// COMPLEXITY

Time: O(V²) with adjacency matrix  |  Space: O(V)

// ANALOGY

Islands of mutual reachability in a network. Within each island, everyone can talk to everyone. Between islands, traffic only goes one way.

// SCC VISUALIZATION
Click 'Find SCCs' to run Kosaraju's algorithm.
C — KOSARAJU'S SCC
int stk[MAX], top = -1;

void dfs1(struct Graph *g, int v, int vis[]) {
    vis[v] = 1;
    for (int j = 0; j < g->count; j++)
        if (g->matrix[v][j] && !vis[j])
            dfs1(g, j, vis);
    stk[++top] = v;  /* push in finish order */
}

void dfs2(struct Graph *t, int v, int vis[]) {
    vis[v] = 1;
    printf("%c ", t->vertex[v]);
    for (int j = 0; j < t->count; j++)
        if (t->matrix[v][j] && !vis[j])
            dfs2(t, j, vis);
}

void findSCC(struct Graph *g) {
    int vis[MAX] = {0};

    /* Step 1: DFS, fill stack */
    for (int i = 0; i < g->count; i++)
        if (!vis[i]) dfs1(g, i, vis);

    /* Step 2: Transpose */
    struct Graph trans = *g;
    for (int i = 0; i < g->count; i++)
        for (int j = 0; j < g->count; j++)
            trans.matrix[i][j] = g->matrix[j][i];

    /* Step 3: DFS on transposed in stack order */
    memset(vis, 0, sizeof(vis));
    int sccNum = 1;
    while (top >= 0) {
        int v = stk[top--];
        if (!vis[v]) {
            printf("SCC %d: ", sccNum++);
            dfs2(&trans, v, vis);
            printf("\n");
        }
    }

Cycle Detection & Count

Find loops in the directed graph

🔄Detect Cycle & Directed Cycle Detection

A directed cycle exists when there is a path from a vertex back to itself following directed edges. We use DFS with a recursion stack (gray set):

  • White (0): Not visited
  • Gray (1): Currently in DFS call stack (being processed)
  • Black (2): Fully processed
// KEY RULE

If during DFS we encounter a gray vertex, we've found a back edge → cycle exists.

// ANALOGY

Detecting circular dependencies in software packages: if Package A needs B, B needs C, and C needs A — that's a directed cycle and will cause infinite loops.

C — DIRECTED CYCLE DETECTION (DFS COLOR)
/* color: 0=white, 1=gray, 2=black */
int dfsHasCycle(struct Graph *g, int v, int color[]) {
    color[v] = 1;  /* mark gray */
    for (int j = 0; j < g->count; j++) {
        if (!g->matrix[v][j]) continue;
        if (color[j] == 1)            /* back edge! */
            return 1;
        if (color[j] == 0)
            if (dfsHasCycle(g, j, color)) return 1;
    }
    color[v] = 2;  /* mark black */
    return 0;
}

int detectCycle(struct Graph *g) {
    int color[MAX] = {0};
    for (int i = 0; i < g->count; i++)
        if (color[i] == 0)
            if (dfsHasCycle(g, i, color))
                return 1;  /* cycle found */
    return 0;               /* no cycle (DAG) */
}
10 — TOPOLOGICAL

Topological Sort

Order vertices such that every edge u→v has u before v

📑Topological Sort (Kahn's BFS Algorithm)

Topological sort only works on DAGs (Directed Acyclic Graphs). It produces a linear ordering of vertices such that for every directed edge u → v, vertex u comes before v.

Kahn's Algorithm (BFS-based):

  • Compute in-degree of all vertices.
  • Enqueue all vertices with in-degree 0 (no prerequisites).
  • While queue is not empty: dequeue a vertex, add to result, reduce in-degree of all its neighbors.
  • If a neighbor's in-degree drops to 0, enqueue it.
  • If result has fewer than V vertices → cycle exists, no topological sort possible.
// ANALOGY

Course prerequisites: you must take Calculus before Physics, and Physics before Engineering. Topological sort gives you a valid study order.

// TOPOLOGICAL SORT (DAG)

C — TOPOLOGICAL SORT (KAHN'S)
void topologicalSort(struct Graph *g) {
    int inDeg[MAX] = {0};
    int queue[MAX], front=0, rear=0;
    int result[MAX], rCount=0;

    /* Step 1: compute in-degrees */
    for (int i = 0; i < g->count; i++)
        for (int j = 0; j < g->count; j++)
            if (g->matrix[j][i]) inDeg[i]++;

    /* Step 2: enqueue in-degree 0 */
    for (int i = 0; i < g->count; i++)
        if (inDeg[i] == 0) queue[rear++] = i;

    /* Step 3: process queue */
    while (front < rear) {
        int u = queue[front++];
        result[rCount++] = u;
        for (int v = 0; v < g->count; v++) {
            if (g->matrix[u][v]) {
                inDeg[v]--;
                if (inDeg[v] == 0) queue[rear++] = v;
            }
        }
    }

    if (rCount != g->count)
        printf("Cycle detected! No topological sort.\n");
    else {
        printf("Topological Order: ");
        for (int i = 0; i < rCount; i++)
            printf("%c ", g->vertex[result[i]]);
        printf("\n");
    }
}

    printf("Execution order:\n");

    while (f < r) {

11 — SHORTEST PATH

Dijkstra's Algorithm

Find shortest paths from a single source (non-negative weights)

🗺️Dijkstra — Single Source Shortest Path

Dijkstra's algorithm finds the shortest path from a single source to all other vertices. It requires non-negative edge weights.

  • Initialize all distances to ∞, source distance to 0. Mark all vertices unvisited.
  • Pick the unvisited vertex with the smallest distance (greedy choice).
  • For each unvisited neighbor: if dist[u] + weight(u,v) < dist[v], update dist[v].
  • Mark u as visited. Repeat until all visited.
// COMPLEXITY

Time: O(V²) with adjacency matrix (O(E log V) with priority queue)  |  Space: O(V)

// ANALOGY

GPS navigation: starting from your location, always explore the closest unexplored intersection until the destination is found. The route you found is guaranteed to be the shortest.

// DIJKSTRA STEP-BY-STEP
Click 'Run from A' to start Dijkstra.
C — DIJKSTRA'S ALGORITHM
#define INF 99999

int minDist(int dist[], int visited[], int n) {
    int min = INF, minIdx = -1;
    for (int i = 0; i < n; i++)
        if (!visited[i] && dist[i] <= min)
            { min = dist[i]; minIdx = i; }
    return minIdx;
}

void dijkstra(struct Graph *g, char srcLabel) {
    int src = searchVertex(g, srcLabel);
    int dist[MAX], visited[MAX]={0};
    int parent[MAX];

    for (int i = 0; i < g->count; i++)
        dist[i] = INF, parent[i] = -1;
    dist[src] = 0;

    for (int iter = 0; iter < g->count; iter++) {
        int u = minDist(dist, visited, g->count);
        if (u == -1) break;
        visited[u] = 1;

        for (int v = 0; v < g->count; v++) {
            int w = g->matrix[u][v];
            if (w && !visited[v] && dist[u]+w < dist[v]) {
                dist[v] = dist[u] + w;
                parent[v] = u;
            }
        }
    }

    printf("Shortest distances from %c:\n", srcLabel);
    for (int i = 0; i < g->count; i++)
        printf("  to %c: %d\n", g->vertex[i],
               dist[i] == INF ? -1 : dist[i]);
}
⚡Bellman-Ford Algorithm

Bellman-Ford handles negative edge weights (unlike Dijkstra) and can detect negative cycles. It relaxes all edges V-1 times.

  • Initialize dist[src]=0, all others = ∞.
  • Repeat V-1 times: for every edge (u,v,w), if dist[u] + w < dist[v], update dist[v].
  • V-th relaxation: if any update occurs → negative cycle detected.
// COMPLEXITY

Time: O(V³) with adjacency matrix (iterate all V² edges, V-1 times)

// ANALOGY

Currency arbitrage detection: negative cycles in a forex graph mean you can make infinite money by looping through currency conversions — Bellman-Ford catches this!

C — BELLMAN-FORD
void bellmanFord(struct Graph *g, char srcLabel) {
    int src = searchVertex(g, srcLabel);
    int dist[MAX], n = g->count;

    for (int i = 0; i < n; i++) dist[i] = INF;
    dist[src] = 0;

    /* Relax all edges V-1 times */
    for (int iter = 0; iter < n - 1; iter++) {
        for (int u = 0; u < n; u++) {
            if (dist[u] == INF) continue;
            for (int v = 0; v < n; v++) {
                if (g->matrix[u][v] &&
                    dist[u] + g->matrix[u][v] < dist[v])
                    dist[v] = dist[u] + g->matrix[u][v];
            }
        }
    }

    /* Check for negative cycles (V-th relaxation) */
    for (int u = 0; u < n; u++)
        for (int v = 0; v < n; v++)
            if (g->matrix[u][v] &&
                dist[u] + g->matrix[u][v] < dist[v]) {
                printf("Negative cycle detected!\n");
                return;
            }

    printf("Bellman-Ford from %c:\n", srcLabel);
    for (int i = 0; i < n; i++)
        printf("  to %c: %d\n", g->vertex[i],
               dist[i]==INF ? -1 : dist[i]);
}
🌐Floyd-Warshall — All-Pairs Shortest Path

Floyd-Warshall computes shortest paths between ALL pairs of vertices. It uses dynamic programming: consider whether routing through each intermediate vertex k reduces the path cost.

// RECURRENCE

dist[i][j] = min(dist[i][j], dist[i][k] + dist[k][j])
for every intermediate vertex k

// COMPLEXITY

Time: O(V³)  |  Space: O(V²) — the whole matrix is the answer!

// ANALOGY

Finding the cheapest flight between every pair of cities, considering all possible stopover cities. We check: "Is going via London cheaper than a direct flight from Paris to Tokyo?"

// FLOYD-WARSHALL DISTANCE MATRIX
Click 'Compute All-Pairs' to run Floyd-Warshall.
C — FLOYD-WARSHALL
void floydWarshall(struct Graph *g) {
    int n = g->count;
    int dist[MAX][MAX];

    /* Initialize: copy matrix, INF where no edge */
    for (int i = 0; i < n; i++)
        for (int j = 0; j < n; j++) {
            if (i == j)           dist[i][j] = 0;
            else if (g->matrix[i][j]) dist[i][j] = g->matrix[i][j];
            else                  dist[i][j] = INF;
        }

    /* Try every vertex k as intermediate */
    for (int k = 0; k < n; k++)
        for (int i = 0; i < n; i++)
            for (int j = 0; j < n; j++)
                if (dist[i][k] != INF &&
                    dist[k][j] != INF &&
                    dist[i][k]+dist[k][j] < dist[i][j])
                    dist[i][j] = dist[i][k]+dist[k][j];

    /* Print result matrix */
    printf("All-Pairs Shortest Distances:\n");
    for (int i = 0; i < n; i++) {
        for (int j = 0; j < n; j++)
            dist[i][j]==INF ? printf("  INF") :
                               printf("%5d", dist[i][j]);
        printf("\n");
    }
}

🔁Transpose & Reverse Graph

The transpose (or reverse) of a directed graph G is a graph Gᵀ with the same vertices but all edge directions flipped: if G has edge u→v, Gᵀ has edge v→u.

In matrix terms: transpose[i][j] = original[j][i] — simply reflect the matrix across its main diagonal.

Transposing is a key step in Kosaraju's SCC algorithm. It's also used to find vertices that can reach a given vertex (reverse reachability).

// ANALOGY

Reversing all one-way roads in a city. If you could reach B from A before, now you can reach A from B on the reversed map.

struct Graph transposeGraph(struct Graph *g) {
    struct Graph t;
    t.count = g->count;

    /* copy vertex labels */
    for (int i = 0; i < g->count; i++)
        t.vertex[i] = g->vertex[i];

    for (int i = 0; i < g->count; i++)

void reverseGraph(struct Graph *g) {
    for (int i = 0; i < g->count; i++)
        for (int j = i+1; j < g->count; j++) {
            int tmp = g->matrix[i][j];
            g->matrix[i][j] = g->matrix[j][i];
            g->matrix[j][i] = tmp;
        }
}
🎨Graph Coloring

Graph coloring assigns a color (number) to each vertex such that no two adjacent vertices share the same color. For directed graphs, "adjacent" typically means connected by an edge in either direction.

The greedy coloring algorithm assigns the smallest available color to each vertex in order.

// CHROMATIC NUMBER

The minimum number of colors needed is the chromatic number χ(G). Greedy may not always find the minimum, but it gives a valid coloring.

// ANALOGY

Scheduling exams: students taking multiple courses must not have two exams at the same time. Each time slot is a color; connected courses (shared students) need different colors.

// GREEDY GRAPH COLORING

void graphColoring(struct Graph *g) {
    int color[MAX], n = g->count;
    int avail[MAX];

    for (int i = 0; i < n; i++) color[i] = -1;

    color[0] = 0;  /* assign color 0 to first vertex */

    for (int u = 1; u < n; u++) {
        /* mark colors of adjacent vertices unavailable */
        for (int c = 0; c < n; c++) avail[c] = 1;
        for (int v = 0; v < n; v++) {
            if ((g->matrix[u][v] || g->matrix[v][u])
                && color[v] != -1)
                avail[color[v]] = 0;
        }
        /* assign smallest available color */
        for (int c = 0; c < n; c++) {
            if (avail[c]) { color[u] = c; break; }
        }
    }

    printf("Vertex Coloring:\n");
    for (int i = 0; i < n; i++)
        printf("  %c -> Color %d\n", g->vertex[i], color[i]);
}
⚖️Bipartite Check

A graph is bipartite if its vertices can be split into two groups (sets) such that every edge connects a vertex from one group to the other — no edges within the same group. Equivalently, a graph is bipartite if and only if it contains no odd-length cycles.

Check using BFS: try to 2-color the graph. If any adjacent vertices have the same color → not bipartite.

// ANALOGY

A bipartite graph is like a set of students and courses — edges only go between students and courses, never student-to-student or course-to-course.

C — BIPARTITE CHECK (BFS)
int isBipartite(struct Graph *g) {
    int color[MAX], n = g->count;
    for (int i = 0; i < n; i++) color[i] = -1;

    int queue[MAX], front, rear;

    for (int src = 0; src < n; src++) {
        if (color[src] != -1) continue;
        front = rear = 0;
        color[src] = 0;
        queue[rear++] = src;

        while (front < rear) {
            int u = queue[front++];
            for (int v = 0; v < n; v++) {
                if (!g->matrix[u][v] && !g->matrix[v][u]) continue;
                if (color[v] == -1) {
                    color[v] = 1 - color[u];
                    queue[rear++] = v;
                } else if (color[v] == color[u])
                    return 0;  /* NOT bipartite */
            }
        }
    }
    return 1;  /* bipartite */
}
🌊Network Flow Basics (Ford-Fulkerson Concept)

A flow network is a directed graph where each edge has a capacity (maximum flow it can carry). The goal is to find the maximum flow from a source s to a sink t.

Key concepts:

  • Capacity: matrix[u][v] = max flow on edge u→v
  • Flow: actual flow on each edge (≤ capacity)
  • Residual Graph: remaining capacity after sending flow
  • Augmenting Path: path from s to t in residual graph with capacity > 0
// MAX FLOW MIN CUT THEOREM

Maximum flow from s to t = Minimum cut capacity separating s and t. This fundamental theorem connects flow and cuts.

// ANALOGY

Water flowing through pipes: each pipe has a maximum capacity. Ford-Fulkerson finds the maximum water flow from a reservoir (source) to a city (sink).

C — FORD-FULKERSON (BFS AUGMENTATION)
/* Find augmenting path using BFS */
int bfsAugment(int rg[][MAX], int s, int t,
              int parent[], int n) {
    int visited[MAX]={0}, q[MAX], f=0, r=0;
    visited[s]=1; parent[s]=-1; q[r++]=s;
    while (fint u=q[f++];
        for (int v=0; vif (!visited[v] && rg[u][v]>0) {
                parent[v]=u; visited[v]=1; q[r++]=v;
                if (v==t) return 1;
            }
    }
    return 0;
}

    int rg[MAX][MAX], parent[MAX], n=g->count;
    for (int i=0; ifor (int j=0; jmatrix[i][j]; /* residual = capacity */

    int flow = 0;
    while (bfsAugment(rg, s, t, parent, n)) {
        /* find min residual along the path */
        int pathFlow = INF;
        for (int v=t; v!=s; v=parent[v]) {
            int u=parent[v];
            if (rg[u][v] < pathFlow) pathFlow=rg[u][v];
        }
        /* update residual capacities */
        for (int v=t; v!=s; v=parent[v]) {
            int u=parent[v];

            rg[v][u] += pathFlow;  /* reverse edge */
        }
        flow += pathFlow;
    }
    return flow;
}
