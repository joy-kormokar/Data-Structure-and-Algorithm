// SECTION 01
Basic Operations
Foundation of Array-Based Circular Linked List — creating, initializing, checking empty/full states. Every advanced concept builds on these fundamentals.
Create List Initialize Check Empty Check Full
📖 Theory Explanation
An Array-Based Circular Linked List stores elements in a fixed-size array where the last position wraps around to the first position, forming a logical circle.

Think of it as a regular array enhanced with circular indexing — when you reach the end, you go back to index 0 automatically. This is managed by two pointers: FRONT (where elements come out) and REAR (where elements go in).
📌 Key Concept: The array has a fixed MAX capacity. The circular nature is created mathematically using the modulo operator (%): index = (index + 1) % MAX. This is not a pointer-based list — all data lives in a plain C array.
Storage Fixed-size array
Wrapping Modulo arithmetic
Front ptr Head of list
Rear ptr Tail of list
Empty front == -1
Full (rear+1)%MAX == front

💻 C Code
C LANGUAGE
#include <stdio.h>

#define MAX 6   // maximum capacity of the list

/* The structure representing our circular list */
struct CircularList {
    int data[MAX];   // array to store elements
    int front;       // index of first element
    int rear;        // index of last element
};

/* CREATE: Declare and prepare the list */
struct CircularList createList() {
    struct CircularList cl;
    cl.front = -1;   // -1 means list is empty
    cl.rear  = -1;
    return cl;
}

/* CHECK EMPTY: Is the list empty? */
int isEmpty(struct CircularList *cl) {
    return (cl->front == -1);  // 1 = yes empty, 0 = no
}

/* CHECK FULL: Is the list full? */
int isFull(struct CircularList *cl) {
    // Next position after rear equals front → full!
    return ((cl->rear + 1) % MAX == cl->front);
}

int main() {
    struct CircularList cl = createList();

    if (isEmpty(&cl))
        printf("List is EMPTY\n");

    if (!isFull(&cl))
        printf("List is NOT FULL — ready to insert\n");

    return 0;
}
🎡
Real-Life Analogy
Imagine a Ferris Wheel with exactly 6 cabins. FRONT is the cabin currently at the boarding platform. REAR is the last cabin that got a passenger. When all 6 cabins are filled (FULL), no one else can board. When all cabins are empty, it's an empty list. The wheel always rotates in one direction — just like our circular array!
// SECTION 02
Insertion Operations
Adding elements at the front, rear, or any specific position in the circular list. The circular wrap-around insert handles boundary crossing elegantly.
📖 Theory Explanation
Insertion in a circular array requires careful handling of the wrap-around. There are three main types:

Insert at Rear — most common. Move REAR forward by 1 (wrapping if needed), place value there.
Insert at Front — move FRONT backward by 1 (wrapping if needed), place value there.
Insert at Position — shift elements to make room, then insert.
Wrap-Around Insert — when REAR reaches the last index and wraps back to 0.
⚠️ Always check isFull() before inserting! If the list is full and you try to insert, you'll overwrite existing data. This is called overflow.
Insert Rear Formula:
rear = (rear + 1) % MAX
data[rear] = value
Insert Front Formula:
front = (front - 1 + MAX) % MAX
data[front] = value

/* INSERT AT REAR — most common insertion */
void insertRear(struct CircularList *cl, int val) {
    if (isFull(cl)) {
        printf("List is FULL! Cannot insert.\n");
        return;
    }
    if (isEmpty(cl)) {
        cl->front = 0;   // first element — set front to 0
        cl->rear  = 0;
    } else {
        cl->rear = (cl->rear + 1) % MAX;  // wrap around!
    }
    cl->data[cl->rear] = val;
    printf("Inserted %d at rear (index %d)\n", val, cl->rear);
}

/* INSERT AT FRONT */
void insertFront(struct CircularList *cl, int val) {
    if (isFull(cl)) {
        printf("List is FULL!\n");
        return;
    }
    if (isEmpty(cl)) {
        cl->front = 0;
        cl->rear  = 0;
    } else {
        // Move front backward; +MAX prevents negative result
        cl->front = (cl->front - 1 + MAX) % MAX;
    }
    cl->data[cl->front] = val;
    printf("Inserted %d at front (index %d)\n", val, cl->front);
}

/* INSERT AT SPECIFIC POSITION */
void insertAtPos(struct CircularList *cl, int val, int pos) {
    int size = getSize(cl);
    if (isFull(cl) || pos < 0 || pos > size) {
        printf("Invalid position or list is full!\n");
        return;
    }
    if (pos == 0) { insertFront(cl, val); return; }
    if (pos == size) { insertRear(cl, val); return; }
    /* Shift elements right from rear to pos, then insert */
    cl->rear = (cl->rear + 1) % MAX;
    for (int i = size; i > pos; i--) {
        int curr = (cl->front + i) % MAX;
        int prev = (cl->front + i - 1) % MAX;
        cl->data[curr] = cl->data[prev];
    }
    cl->data[(cl->front + pos) % MAX] = val;
    printf("Inserted %d at position %d\n", val, pos);
}
🎟️
Real-Life Analogy
Insert Rear = A person joins the back of a circular queue at a cinema. Insert Front = A VIP guest jumps to the front. Insert at Position = Someone with a reserved seat cuts in at a specific spot, pushing others back. Wrap-around = The queue wraps around the corner — the last person stands right next to the first!
// SECTION 03
Deletion Operations
Removing elements from the front, rear, or any position. Must handle wrap-around carefully and reset front/rear to -1 when the list becomes empty.
📖 Theory Explanation
Deletion always starts with an isEmpty() check. Deleting from an empty list causes underflow.

Delete Front — Remove the element at FRONT, then advance FRONT forward.
Delete Rear — Remove element at REAR, then move REAR backward.
Delete at Position — Remove element, shift remaining elements to fill the gap.
Wrap-Around Delete — When FRONT wraps around from the end back to the beginning.
🔑 Reset to empty: After deletion, if front == rear (only one element was left), set both front = rear = -1 to signal the list is now empty.

💻 C Code — Deletion Operations
C LANGUAGE
/* DELETE FRONT ELEMENT */
int deleteFront(struct CircularList *cl) {
    if (isEmpty(cl)) {
        printf("Underflow! List is empty.\n");
        return -1;
    }
    int val = cl->data[cl->front];
    if (cl->front == cl->rear) {   // last element!
        cl->front = -1;
        cl->rear  = -1;           // list is now empty
    } else {
        cl->front = (cl->front + 1) % MAX;  // wrap around
    }
    printf("Deleted front: %d\n", val);
    return val;
}

/* DELETE REAR ELEMENT */
int deleteRear(struct CircularList *cl) {
    if (isEmpty(cl)) {
        printf("Underflow! List is empty.\n");
        return -1;
    }
    int val = cl->data[cl->rear];
    if (cl->front == cl->rear) {
        cl->front = -1;
        cl->rear  = -1;
    } else {
        // Move rear backward (+MAX avoids negative modulo)
        cl->rear = (cl->rear - 1 + MAX) % MAX;
    }
    printf("Deleted rear: %d\n", val);
    return val;
}

/* DELETE AT SPECIFIC POSITION */
int deleteAtPos(struct CircularList *cl, int pos) {
    int size = getSize(cl);
    if (isEmpty(cl) || pos < 0 || pos >= size) {
        printf("Invalid position!\n");
        return -1;
    }
    int realIdx = (cl->front + pos) % MAX;
    int val = cl->data[realIdx];
    /* Shift elements left to fill the gap */
    for (int i = pos; i < size - 1; i++) {
        int curr = (cl->front + i) % MAX;
        int next = (cl->front + i + 1) % MAX;
        cl->data[curr] = cl->data[next];
    }
    cl->rear = (cl->rear - 1 + MAX) % MAX;
    if (size == 1) { cl->front = -1; cl->rear = -1; }
    printf("Deleted %d from position %d\n", val, pos);
    return val;
}
🚌
Real-Life Analogy
Think of a circular bus route with 6 stops. Delete Front = The bus picks up the passenger at stop 1 (front) and drives them away — now stop 2 becomes the new front. Delete Rear = The last passenger gets off at the last stop, and the previous stop becomes the new rear. Wrap-around delete = When the front stop wraps from stop 6 back to stop 1!

Traversal Operations
Visiting every element in the list — forward, circular, and wrap-around traversal. The key challenge is knowing when to stop in a circular structure.
📖 Theory Explanation
Traversal means visiting every element exactly once. In a circular array, we start at FRONT and visit elements one by one until we reach REAR.

Forward Traversal — Go from FRONT to REAR, using modulo at each step.
Circular Traversal — Keep going in circles (infinite, until manually stopped).
Wrap-Around Traversal — Handle the case where REAR < FRONT in the array.
Iterative Traversal — Use a loop counter (count from 0 to size-1).
🔑 Stop condition for traversal: Use a count variable starting at 0. Increment until count equals the list size. This avoids infinite loops!
Alternatively: traverse while current != rear, then process rear separately.

💻 C Code — Traversal Operations
C LANGUAGE
/* FORWARD TRAVERSAL — visit each element once */
void forwardTraversal(struct CircularList *cl) {
    if (isEmpty(cl)) {
        printf("List is empty!\n");
        return;
    }
    printf("List: ");
    int i = cl->front;   // start from front
    int count = 0;
    int size = getSize(cl);
    while (count < size) {
        printf("%d ", cl->data[i]);
        i = (i + 1) % MAX;   // wrap around the array
        count++;
    }
    printf("\n");
}

/* CIRCULAR TRAVERSAL — loops forever (use with care!) */
void circularTraversal(struct CircularList *cl, int rounds) {
    if (isEmpty(cl)) return;
    int size = getSize(cl);
    int total = size * rounds;  // how many times to visit
    int i = cl->front;
    for (int t = 0; t < total; t++) {
        printf("%d → ", cl->data[i]);
        i = (i + 1) % MAX;
        if (i == cl->front) printf("[WRAP] ");
    }
    printf("\n");
}

/* ITERATIVE TRAVERSAL — position-based */
void iterativeTraversal(struct CircularList *cl) {
    int size = getSize(cl);
    for (int pos = 0; pos < size; pos++) {
        int realIdx = (cl->front + pos) % MAX;
        printf("Position[%d] = %d (array[%d])\n",
               pos, cl->data[realIdx], realIdx);
    }
}
🎠
Real-Life Analogy
Forward traversal = A ticket inspector on a merry-go-round checks each rider exactly once, starting from one horse and going all the way around until back to start. Circular traversal = The inspector keeps going around and around for multiple rounds. Wrap-around = When they pass the "start" marker and continue — the count tells them when to stop!

Searching Operations
Finding elements by value or position. Linear and circular search strategies with circular index arithmetic.
📖 Theory Explanation
Searching scans the list to find a matching element.

Linear Search — Start from FRONT, check each element one by one until found or all checked.
Circular Search — Search continuously from any starting point, wrapping around.
Search by Position — Directly compute the array index using: (front + pos) % MAX.
Search by Value — Compare each element to the target value.
Search by Value:
Returns the logical position (0-indexed from FRONT) if found, or -1 if not found.
Search by Position:
Directly access: data[(front + pos) % MAX]. O(1) time!
Search TypeTime ComplexitySpace
Linear Search by ValueO(n)O(1)
Search by PositionO(1)O(1)
Circular SearchO(n)O(1)

💻 C Code — Search Operations
C LANGUAGE
/* LINEAR SEARCH — find by value */
int linearSearch(struct CircularList *cl, int target) {
    if (isEmpty(cl)) return -1;
    int size = getSize(cl);
    for (int pos = 0; pos < size; pos++) {
        int realIdx = (cl->front + pos) % MAX;
        if (cl->data[realIdx] == target) {
            printf("Found %d at position %d (index %d)\n",
                   target, pos, realIdx);
            return pos;   // logical position
        }
    }
    printf("%d not found in list\n", target);
    return -1;
}

/* SEARCH BY POSITION — direct O(1) access */
int searchByPos(struct CircularList *cl, int pos) {
    int size = getSize(cl);
    if (pos < 0 || pos >= size) {
        printf("Invalid position!\n");
        return -1;
    }
    int realIdx = (cl->front + pos) % MAX;
    printf("Position %d → value %d (array[%d])\n",
           pos, cl->data[realIdx], realIdx);
    return cl->data[realIdx];
}

/* CIRCULAR SEARCH — search from any starting point */
int circularSearch(struct CircularList *cl, int target, int startPos) {
    int size = getSize(cl);
    for (int i = 0; i < size; i++) {
        int pos = (startPos + i) % size;
        int realIdx = (cl->front + pos) % MAX;
        if (cl->data[realIdx] == target) {
            printf("Found at circular pos %d\n", pos);
            return pos;
        }
    }
    return -1;
}
🔦
Real-Life Analogy
Linear search = A security guard walking through a circular parking lot, checking each car's number plate one by one until they find the target car. Search by position = Knowing parking spot #3, going directly there — instant! Circular search = Starting from the guard's current position and sweeping in one direction all the way around.

Update Operations
Modifying existing elements — replacing values, changing data at a specific position, and swapping two elements.
📖 Theory Explanation
Update operations modify data in-place without changing the structure.

Replace Value — Search for old value, replace with new value.
Modify Position Data — Directly update data at a given logical position.
Swap Elements — Exchange data between two positions using a temporary variable.
✅ Update operations are generally O(1) when you know the position (direct index access), or O(n) when searching by value first.

💻 C Code — Update Operations
C LANGUAGE
/* REPLACE VALUE — find and replace */
void replaceValue(struct CircularList *cl, int oldVal, int newVal) {
    int pos = linearSearch(cl, oldVal);
    if (pos == -1) {
        printf("Value %d not found!\n", oldVal);
        return;
    }
    int realIdx = (cl->front + pos) % MAX;
    cl->data[realIdx] = newVal;
    printf("Replaced %d with %d\n", oldVal, newVal);
}

/* MODIFY POSITION DATA — direct update by position */
void modifyAt(struct CircularList *cl, int pos, int newVal) {
    int size = getSize(cl);
    if (pos < 0 || pos >= size) {
        printf("Invalid position!\n");
        return;
    }
    int realIdx = (cl->front + pos) % MAX;
    int old = cl->data[realIdx];
    cl->data[realIdx] = newVal;
    printf("Position %d: %d → %d\n", pos, old, newVal);
}

/* SWAP ELEMENTS — exchange two positions */
void swapElements(struct CircularList *cl, int pos1, int pos2) {
    int size = getSize(cl);
    if (pos1 < 0 || pos1 >= size || pos2 < 0 || pos2 >= size) {
        printf("Invalid positions!\n");
        return;
    }
    int idx1 = (cl->front + pos1) % MAX;
    int idx2 = (cl->front + pos2) % MAX;
    int temp = cl->data[idx1];   // temp variable holds first
    cl->data[idx1] = cl->data[idx2];
    cl->data[idx2] = temp;
    printf("Swapped pos %d ↔ pos %d\n", pos1, pos2);
}
✏️
Real-Life Analogy
Replace value = A teacher erases a wrong answer on a student's paper and writes the correct one. Modify at position = A librarian directly goes to shelf #3 and relabels that book. Swap elements = Two students swap seats — neither leaves the room, they just change positions with each other.

Access Operations
Reading elements without removing them — peek at front, peek at rear, or get any element by its logical position.
📖 Theory Explanation
Access operations read data without modifying the list structure.

Peek Front — Return data[front] without removing it.
Peek Rear — Return data[rear] without removing it.
Get by Position — Return data[(front + pos) % MAX].
💡 All three access operations run in O(1) time — they directly compute the index and return the value. No searching, no shifting!

// SECTION 08
Structural Operations
Counting elements, finding the current size, and checking remaining capacity. Essential for managing the list's health.
📖 Theory Explanation
Count Elements / Find Size — The number of elements currently in the list.
When FRONT > REAR (wrap-around happened): size = MAX - front + rear + 1
When FRONT ≤ REAR (no wrap): size = rear - front + 1

A simpler formula that always works: size = (rear - front + MAX) % MAX + 1

Check Capacity — Remaining slots = MAX - size
📐 These are O(1) operations — they just compute from front and rear indices without looping.
💻 C Code — Structural Operations
C LANGUAGE
/* COUNT / GET SIZE — how many elements? */
int getSize(struct CircularList *cl) {
    if (isEmpty(cl)) return 0;
    /* This formula handles both wrap and no-wrap cases */
    return (cl->rear - cl->front + MAX) % MAX + 1;
}

/* COUNT ELEMENTS — same as getSize, just renamed */
int countElements(struct CircularList *cl) {
    return getSize(cl);
}

/* CHECK CAPACITY — remaining free slots */
int checkCapacity(struct CircularList *cl) {
    int used = getSize(cl);
    int remaining = MAX - used;
    printf("Used: %d / MAX: %d / Free: %d\n",
           used, MAX, remaining);
    return remaining;
}

/* EXAMPLE USAGE */
int main() {
    struct CircularList cl = createList();
    insertRear(&cl, 10);
    insertRear(&cl, 20);
    insertRear(&cl, 30);
    printf("Size: %d\n", getSize(&cl));        // 3
    checkCapacity(&cl);                          // Used: 3, Free: 3
    return 0;
}
📦
Real-Life Analogy
Think of a circular conveyor belt in a factory. Count Elements = Count how many boxes are on the belt right now. Find Size = Same answer. Check Capacity = How many more boxes can fit before the belt is completely packed? The belt has a fixed number of slots (MAX), so you subtract used from total.
// SECTION 09
Circular Management Operations
The mathematical heart of circular lists — increment, decrement, and modulo index management. These functions power every other operation.
📖 Theory Explanation
Circular management ensures indices never go out of bounds — they wrap around automatically.

Circular Increment — Move an index forward: (index + 1) % MAX. When index = MAX-1, it wraps to 0.
Circular Decrement — Move backward: (index - 1 + MAX) % MAX. When index = 0, it wraps to MAX-1.
Modulo Index Update — General: (index + steps) % MAX. Allows jumping by any number of steps.
Why +MAX in decrement?
In C, (-1) % 6 = -1 (negative!). Adding MAX first prevents this: (-1 + 6) % 6 = 5 ✅
Modulo magic:
% MAX always returns a value in [0, MAX-1]. It's the mathematical key to circular behavior.

💻 C Code — Circular Management
C LANGUAGE
/* CIRCULAR INCREMENT — move index forward by 1 */
int circularIncrement(int index) {
    return (index + 1) % MAX;
    /* Example: index=5 → (5+1)%6 = 0 (wraps!) */
}

/* CIRCULAR DECREMENT — move index backward by 1 */
int circularDecrement(int index) {
    return (index - 1 + MAX) % MAX;
    /* Example: index=0 → (0-1+6)%6 = 5 (wraps!) */
}

/* MODULO INDEX UPDATE — jump N steps forward */
int moduloUpdate(int index, int steps) {
    return (index + steps) % MAX;
    /* Example: index=4, steps=3 → (4+3)%6 = 1 */
}

/* DEMONSTRATION */
int main() {
    int idx = 0;
    printf("Circular Increment sequence:\n");
    for (int i = 0; i < 8; i++) {
        printf("%d → ", idx);
        idx = circularIncrement(idx);
    }
    printf("\nOutput: 0→1→2→3→4→5→0→1→ ...\n");
    return 0;
}
🕐
Real-Life Analogy
A clock is the perfect analogy! After 12 comes 1 (not 13). After 12 o'clock, incrementing gives 1. Going back from 1 gives 12. The modulo operator is exactly the math behind how a clock works — it resets after reaching its maximum value. In our case, MAX is the number on the clock's face!
// SECTION 10
Utility Operations
Housekeeping operations — clearing the list, copying it to another, and cloning (deep copy). Essential for maintenance and reuse.
📖 Theory Explanation
Clear List — Reset FRONT and REAR to -1. Existing data stays in the array but is logically erased (overwritten on next insert).

Copy List — Create a destination list and copy all elements one by one from source to destination.

Clone List — Same as copy but preserves the internal array structure exactly (same front/rear indices, same layout).
⚠️ Clear vs Reset: Clear() just resets pointers — old data still sits in memory but is "logically gone." For security-sensitive applications, explicitly overwrite with zeros first!
💻 C Code — Utility Operations
C LANGUAGE
/* CLEAR LIST — logically empty the list */
void clearList(struct CircularList *cl) {
    cl->front = -1;
    cl->rear  = -1;
    printf("List cleared!\n");
    /* Optional: zero out memory for security */
    for (int i = 0; i < MAX; i++) cl->data[i] = 0;
}

/* COPY LIST — create a new list with same elements */
struct CircularList copyList(struct CircularList *src) {
    struct CircularList dest = createList();
    int size = getSize(src);
    for (int i = 0; i < size; i++) {
        int val = getAt(src, i);
        insertRear(&dest, val);
    }
    printf("Copied %d elements to new list\n", size);
    return dest;
}

/* CLONE LIST — exact structural copy (same layout) */
struct CircularList cloneList(struct CircularList *src) {
    struct CircularList clone;
    clone.front = src->front;
    clone.rear  = src->rear;
    for (int i = 0; i < MAX; i++) {
        clone.data[i] = src->data[i];  // exact copy
    }
    printf("List cloned (exact copy)\n");
    return clone;
}
🗑️
Real-Life Analogy
Clear = Erasing a whiteboard — the data is gone, the board is ready for new content. Copy = Transcribing a shopping list onto a fresh piece of paper, item by item. Clone = Photocopying the list — it's an exact duplicate, including any doodles and corrections on the sides!

Sorting Operations
Arranging elements in ascending or descending order using Bubble Sort — implemented with circular index arithmetic.
📖 Theory Explanation
Sorting in a circular array uses the same algorithms as regular arrays, but indices must go through the circular increment formula.

Bubble Sort — Repeatedly compare adjacent elements and swap if they're out of order. After each pass, the largest element "bubbles" to the end. Requires n-1 passes for n elements.

Ascending Sort — Smaller values at front, larger at rear.
Descending Sort — Larger values at front, smaller at rear.
⏱️ Bubble Sort has O(n²) time complexity — fine for small lists (n ≤ 100), but use QuickSort or MergeSort for large data!

💻 C Code — Sorting Operations
C LANGUAGE
/* BUBBLE SORT ASCENDING */
void sortAscending(struct CircularList *cl) {
    int size = getSize(cl);
    for (int pass = 0; pass < size - 1; pass++) {
        int swapped = 0;
        for (int i = 0; i < size - 1 - pass; i++) {
            int idx1 = (cl->front + i) % MAX;
            int idx2 = (cl->front + i + 1) % MAX;
            if (cl->data[idx1] > cl->data[idx2]) {
                /* Swap */
                int temp = cl->data[idx1];
                cl->data[idx1] = cl->data[idx2];
                cl->data[idx2] = temp;
                swapped = 1;
            }
        }
        if (!swapped) break;  // already sorted!
    }
    printf("List sorted ascending\n");
}

/* BUBBLE SORT DESCENDING */
void sortDescending(struct CircularList *cl) {
    int size = getSize(cl);
    for (int pass = 0; pass < size - 1; pass++) {
        int swapped = 0;
        for (int i = 0; i < size - 1 - pass; i++) {
            int idx1 = (cl->front + i) % MAX;
            int idx2 = (cl->front + i + 1) % MAX;
            if (cl->data[idx1] < cl->data[idx2]) {  // reversed!
                int temp = cl->data[idx1];
                cl->data[idx1] = cl->data[idx2];
                cl->data[idx2] = temp;
                swapped = 1;
            }
        }
        if (!swapped) break;
    }
    printf("List sorted descending\n");
}
🫧
Real-Life Analogy
Imagine students standing in a circle, each holding a number. In each round, neighboring students compare numbers and swap if out of order. After each round, the largest number has "bubbled" to its final position. After enough rounds, everyone stands in order! This is exactly Bubble Sort — just with modulo arithmetic to handle the circular arrangement.
// SECTION 12
Merge & Split Operations
Combining two circular lists into one, splitting one into two, and concatenating lists end-to-end.
📖 Theory Explanation
Merge — Combine two lists into one. All elements from both lists must fit into a single MAX-capacity array.

Split — Divide one list into two halves. The first half stays in the original, the second goes to a new list.

Concatenate — Append all elements of list B to the end of list A (same as merge but order-preserving).
⚠️ When merging, ensure the combined size doesn't exceed MAX: size1 + size2 ≤ MAX. Otherwise, create a bigger list!
💻 C Code — Merge & Split
C LANGUAGE
/* MERGE — combine two circular lists into one */
struct CircularList mergeLists(
    struct CircularList *a,
    struct CircularList *b)
{
    struct CircularList merged = createList();
    int sizeA = getSize(a);
    int sizeB = getSize(b);
    if (sizeA + sizeB > MAX) {
        printf("Cannot merge: combined size exceeds MAX!\n");
        return merged;
    }
    /* Insert all elements from A */
    for (int i = 0; i < sizeA; i++)
        insertRear(&merged, getAt(a, i));
    /* Insert all elements from B */
    for (int i = 0; i < sizeB; i++)
        insertRear(&merged, getAt(b, i));
    printf("Merged %d + %d = %d elements\n",
           sizeA, sizeB, sizeA + sizeB);
    return merged;
}

/* SPLIT — split list into two halves */
void splitList(struct CircularList *src,
               struct CircularList *first,
               struct CircularList *second)
{
    int size = getSize(src);
    int half = size / 2;
    *first  = createList();
    *second = createList();
    for (int i = 0; i < half; i++)
        insertRear(first, getAt(src, i));
    for (int i = half; i < size; i++)
        insertRear(second, getAt(src, i));
    printf("Split into %d + %d elements\n",
           half, size - half);
}
🔗
Real-Life Analogy
Merge = Two train lines joining into one combined route. Split = One bus line splitting at a junction into two separate routes. Concatenate = Two queues merging, with one line appended after the other. All three keep the circular structure — elements still wrap around as usual!
// SECTION 13
Recursive Operations
Solving traversal, search, and counting problems using recursive function calls. Each function calls itself with a smaller sub-problem.
📖 Theory Explanation
Recursion means a function calls itself to solve a smaller version of the same problem. Every recursive function has two parts:

1️⃣ Base Case — When to stop (e.g., count == 0, or we've visited all elements).
2️⃣ Recursive Case — Solve a smaller sub-problem and call self again.

For circular arrays, the key is passing a count parameter that decreases with each call. When count = 0, we stop (this prevents infinite loops!).
🌀 Important: Without a counter, a circular structure would cause infinite recursion (the list never "ends"). Always use a count or size parameter as the base case!
📚 Call Stack: Each recursive call uses memory on the call stack. For very large lists, recursion may cause a stack overflow. Iterative solutions are safer for large n.

💻 C Code — Recursive Operations
C LANGUAGE
/* RECURSIVE TRAVERSAL */
void recursiveTraversal(struct CircularList *cl,
                        int idx, int count) {
    /* BASE CASE: visited all elements → stop */
    if (count == 0) return;

    /* PROCESS current element */
    printf("%d ", cl->data[idx]);

    /* RECURSIVE CASE: move to next index, count-1 */
    recursiveTraversal(cl,
        (idx + 1) % MAX,   // next circular index
        count - 1);        // one less element to visit
}

/* Call it: recursiveTraversal(&cl, cl.front, getSize(&cl)); */


/* RECURSIVE SEARCH */
int recursiveSearch(struct CircularList *cl,
                    int idx, int target, int count) {
    if (count == 0) return -1;  // not found
    if (cl->data[idx] == target) {
        printf("Found at index %d\n", idx);
        return idx;
    }
    return recursiveSearch(cl,
        (idx + 1) % MAX,
        target,
        count - 1);
}


/* RECURSIVE COUNT */
int recursiveCount(struct CircularList *cl,
                   int idx, int remaining) {
    if (remaining == 0) return 0;
    return 1 + recursiveCount(cl,
        (idx + 1) % MAX,
        remaining - 1);
}
/* Returns: total count of elements from idx */
🪞
Real-Life Analogy
Recursion is like Russian Matryoshka dolls. Each doll (function call) contains a smaller doll (the recursive call). You keep opening dolls until you hit the tiniest one (base case). Then you go back up. For circular traversal, the "count" tells you how many more dolls to open — once it reaches 0, stop opening!
// SECTION 14
Iterative Operations
Loop-based traversal, search, and all operations using explicit loops instead of recursion. Generally preferred for production code — safer and faster.
📖 Theory Explanation
Iterative operations use loops (for/while) instead of recursive calls.

Advantages over recursion:
• No risk of stack overflow for large lists
• Generally faster (no function call overhead)
• Easier to debug and trace

Iterative Traversal — Use a for loop with a counter from 0 to size-1.
Iterative Search — Loop through elements comparing each to the target.
Loop-based Traversal — Use a while loop advancing a circular index.
For Loop Pattern:
for(int i=0; i<size; i++)
idx = (front+i)%MAX;
While Loop Pattern:
int idx = front, cnt = 0;
while(cnt++ < size)
idx = (idx+1)%MAX;

💻 C Code — Iterative Operations
C LANGUAGE
/* ITERATIVE TRAVERSAL — using for loop */
void iterativeTraversalFor(struct CircularList *cl) {
    if (isEmpty(cl)) { printf("Empty!\n"); return; }
    int size = getSize(cl);
    printf("Elements: ");
    for (int i = 0; i < size; i++) {
        int realIdx = (cl->front + i) % MAX;
        printf("%d ", cl->data[realIdx]);
    }
    printf("\n");
}

/* LOOP-BASED TRAVERSAL — using while loop */
void iterativeTraversalWhile(struct CircularList *cl) {
    if (isEmpty(cl)) return;
    int size = getSize(cl);
    int idx = cl->front;
    int count = 0;
    printf("Elements: ");
    while (count < size) {
        printf("%d ", cl->data[idx]);
        idx = (idx + 1) % MAX;   // circular advance
        count++;
    }
    printf("\n");
}

/* ITERATIVE SEARCH */
int iterativeSearch(struct CircularList *cl, int target) {
    int size = getSize(cl);
    int idx = cl->front;
    for (int i = 0; i < size; i++) {
        if (cl->data[idx] == target) {
            printf("Found %d at logical pos %d (array[%d])\n",
                   target, i, idx);
            return i;
        }
        idx = (idx + 1) % MAX;
    }
    printf("%d not found!\n", target);
    return -1;
}
