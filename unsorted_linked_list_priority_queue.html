01
Priority Queue Initialization
Setting up the structure — head pointer, node definition, memory
Theory

A Priority Queue is a special type of queue where every element has an associated priority value. Elements are served based on priority, not just the order they arrive. In a linked-list-based implementation, we use nodes connected via pointers.

Each node holds three things:

  • data — the actual value stored
  • priority — an integer that determines importance
  • next — a pointer to the next node
Key Idea: In an unsorted priority queue, we insert anywhere (usually at the head for O(1)) and scan the whole list when we need the highest/lowest priority element.
Insert: O(1) Delete/Peek: O(n)

Click "Show Initialization" to see the structure built step by step.
C Code
C LANGUAGE 
// ── Node structure for Unsorted Linked List Priority Queue ──
#include <stdio.h>
#include <stdlib.h>

// Define the Node structure
struct Node {
    int data;        // Value stored
    int priority;    // Lower number = Higher priority (like 1 is urgent)
    struct Node* next; // Pointer to next node
};

// Head pointer - always starts as NULL (empty list)
struct Node* head = NULL;

// Function to create a new node dynamically
struct Node* createNode(int data, int priority) {
    struct Node* newNode = (struct Node*) malloc(sizeof(struct Node));
    newNode->data     = data;
    newNode->priority = priority;
    newNode->next     = NULL;
    return newNode;
}

int main() {
    // At start, head is NULL = empty priority queue
    printf("Priority Queue Initialized. head = NULL\n");
    return 0;
}

02
Insert / Enqueue Operation
Adding new elements to the priority queue — O(1) at head
Theory

In an unsorted priority queue, we insert new nodes at the beginning (head) of the linked list. This keeps insertion very fast — O(1) — because we don't need to scan or sort anything.

1Allocate a new node using malloc()
2Set its data and priority values
3Point the new node's next to current head
4Update head to point to the new node
Time: O(1) Space: O(1)

C Code
C LANGUAGE
// Insert at the beginning — O(1)
void enqueue(struct Node** head, int data, int priority) {
    // Step 1: Create new node
    struct Node* newNode = (struct Node*) malloc(sizeof(struct Node));
    newNode->data     = data;
    newNode->priority = priority;

    // Step 2: Point new node to current head
    newNode->next = *head;

    // Step 3: Update head to new node
    *head = newNode;

    printf("Inserted: data=%d, priority=%d\n", data, priority);
}

int main() {
    struct Node* head = NULL;
    enqueue(&head, 10, 3);  // data=10, priority=3
    enqueue(&head, 20, 1);  // data=20, priority=1 (highest)
    enqueue(&head, 30, 2);  // data=30, priority=2
    return 0;
}

📋
Adding to a Task Board
Think of pinning task cards on a board randomly — you just stick each new task card at the top (beginning) without arranging them. You'll sort out which is most urgent only when you need to pick the next task. This is exactly "insert at head, sort later" — the unsorted approach!
03
Delete Highest Priority Element
Traverse, find, track prev pointer, unlink, free
Theory

Since the list is unsorted, we must traverse all nodes to find the one with the highest priority (lowest numeric value if priority 1 = most urgent). We also track the previous node so we can properly unlink the target node.

1Traverse the entire list, track node with minimum priority number
2Keep track of the node before the target (prev pointer)
3If target is head, update head to head->next
4Otherwise: prev->next = target->next
5Free the target node's memory
Time: O(n) Space: O(1)

C Code

// Delete highest priority (lowest priority number) — O(n)
void deleteHighest(struct Node** head) {
    if (*head == NULL) {
        printf("Queue is empty!\n");
        return;
    }

    struct Node* current  = *head;
    struct Node* maxNode   = *head; // tracks highest priority node
    struct Node* prevMax   = NULL;  // tracks node before maxNode
    struct Node* prev      = NULL;

    // Traverse to find highest priority (smallest number)
    while (current != NULL) {
        if (current->priority < maxNode->priority) {
            maxNode = current;
            prevMax = prev;
        }
        prev    = current;
        current = current->next;
    }

    // Remove the node
    if (prevMax == NULL)
        *head = maxNode->next;      // It was the head
    else
        prevMax->next = maxNode->next; // Bypass the node

    printf("Deleted: data=%d, priority=%d\n",
           maxNode->data, maxNode->priority);
    free(maxNode); // Free memory
}

Delete Lowest Priority Element
Find and remove the node with the lowest urgency
Theory

Similar to deleting the highest priority, but now we look for the node with the largest priority number (least urgent). This is useful in a max-priority scenario or when cleaning up the least-urgent items first.

The logic is identical — just flip the comparison: find the node with the maximum priority number instead of minimum.

// Delete lowest priority (highest priority number) — O(n)
void deleteLowest(struct Node** head) {
    if (*head == NULL) {
        printf("Queue is empty!\n");
        return;
    }

    struct Node* current = *head;
    struct Node* minNode = *head; // node with lowest priority
    struct Node* prevMin = NULL;
    struct Node* prev    = NULL;

    while (current != NULL) {
        if (current->priority > minNode->priority) { // flip to >
            minNode = current;
            prevMin = prev;
        }
        prev    = current;
        current = current->next;
    }

    if (prevMin == NULL)
        *head = minNode->next;
    else
        prevMin->next = minNode->next;

    printf("Deleted lowest: data=%d\n", minNode->data);
    free(minNode);
}

05
Peek Highest Priority Element
Read the top element without removing it
Theory

Peek means "look at" without removing. We traverse the list once and return the data of the node with the smallest priority number (highest urgency). The list remains unchanged.

Peek is a read-only operation — it does NOT modify the linked list in any way.
Time: O(n)Space: O(1)

// Peek highest priority — read-only, O(n)
int peekHighest(struct Node* head) {
    if (head == NULL) {
        printf("Queue is empty!\n");
        return -1; // sentinel value
    }

    struct Node* current = head;
    struct Node* best    = head; // tracks best node found

    while (current != NULL) {
        if (current->priority < best->priority)
            best = current;
        current = current->next;
    }

    // List NOT modified — only returning data
    printf("Highest priority: data=%d, priority=%d\n",
           best->data, best->priority);
    return best->data;
}

IsEmpty Operation
Check if the priority queue has any elements
Theory

The simplest operation: if head == NULL, the queue is empty. This is used as a guard check before performing deletions or peeks to avoid crashes (null pointer dereference).

Time: O(1)Space: O(1)

C Code
C LANGUAGE
// Check if priority queue is empty — O(1)
int isEmpty(struct Node* head) {
    return (head == NULL); // returns 1 (true) if empty, 0 (false) otherwise
}

int main() {
    struct Node* head = NULL;

    if (isEmpty(head))
        printf("Queue is EMPTY\n");

    enqueue(&head, 10, 1);  // add an element

    if (!isEmpty(head))
        printf("Queue is NOT empty\n");

    return 0;
}

Checking the Mailbox

// Display entire priority queue — O(n)
void display(struct Node* head) {
    if (head == NULL) {
        printf("Queue is empty!\n");
        return;
    }

    struct Node* current = head;
    printf("Priority Queue:\n");
    printf("HEAD");

    while (current != NULL) {
        printf(" -> [data=%d | pri=%d]",

        current = current->next;
    }

}

09
Traverse Priority Queue
Node-by-node sequential access with animated pointer
Theory

Traversal means visiting each node exactly once, starting from head and following next pointers until we reach NULL. It's the foundation of all O(n) operations.

Pattern: current = head; while(current != NULL) { ...process... current = current->next; }
Time: O(n)
Animated Traversal
Ready to traverse...
C Code
C LANGUAGE
// Traverse the priority queue — O(n)
void traverse(struct Node* head) {
    struct Node* current = head; // start at head
    int pos = 0;

    while (current != NULL) {        // stop at NULL
        printf("Position %d: data=%d, priority=%d\n",
               pos, current->data, current->priority);

        current = current->next; // move to next node
        pos++;
    }
}

🚶

Search Element
Linear search through the queue by data value
Theory

Since the list is unsorted, we can only use linear search — scan every node until we find one whose data matches our target. We also track the position (index) for reference.

Time: O(n)Space: O(1)

// Search element by data value — O(n)
int search(struct Node* head, int target) {
    struct Node* current = head;
    int pos = 0;

    while (current != NULL) {
        if (current->data == target) {
            printf("Found %d at position %d (priority=%d)\n",
                   target, pos, current->priority);
            return pos; // return position
        }
        current = current->next;
        pos++;
    }

    printf("%d not found in queue.\n", target);
    return -1; // not found
}

void updatePriority(struct Node* head, int target, int newPriority) {
    struct Node* current = head;

    while (current != NULL) {

            printf("Updated data=%d: priority %d -> %d\n",

13
Clear / Destroy Priority Queue
Delete all nodes, free all memory, reset head to NULL
Theory

We traverse the list, freeing each node one by one. We use a temporary pointer to hold the next node's address before freeing the current one, to avoid losing the chain.

Always free memory you allocate! Memory leaks can crash long-running programs.
Time: O(n)Space: O(1)
Live Destroy Visualization
NULL
C Code
C LANGUAGE
// Clear/Destroy entire priority queue — O(n)
void clear(struct Node** head) {
    struct Node* current = *head;
    struct Node* temp;

    while (current != NULL) {
        temp    = current->next; // save next BEFORE freeing
        free(current);            // free current node
        current = temp;           // move to saved next
    }

    *head = NULL; // reset head to NULL
    printf("Queue destroyed. Memory freed.\n");
}
Real-Life Analogy
🧹
Clearing the Task Board at End of Day
At end of shift, the nurse removes every card from the board, tears each one up (frees memory), and leaves the board blank (head = NULL). If she forgot to pick up a card before tearing up the next one, she'd lose track of that card — hence the temp pointer!
14
Reverse Priority Queue
Flip all pointer directions, update head to old tail
Theory

Reversing a linked list means making each node's next point backward to its previous node. We use three pointers: prev, current, and next_node.

1prev = NULL, current = head
2Save next: next_node = current->next
3Flip pointer: current->next = prev
4Advance: prev = current, current = next_node
5Repeat; when done, head = prev
Time: O(n)Space: O(1)
Live Reverse Visualization
NULL
C Code
C LANGUAGE
// Reverse the linked list — O(n) with 3 pointers
void reverse(struct Node** head) {
    struct Node* prev      = NULL;
    struct Node* current   = *head;
    struct Node* next_node = NULL;

    while (current != NULL) {
        next_node       = current->next; // 1. save next
        current->next   = prev;          // 2. flip pointer
        prev            = current;       // 3. advance prev
        current         = next_node;     // 4. advance current
    }

    *head = prev; // prev is now the new head
    printf("List reversed!\n");
}
Real-Life Analogy
🔄
Reversing a Train
Imagine a train where each car has an arrow pointing forward. Reversing means making every arrow point backward. You go car by car, flip the arrow, and once done, the last car becomes the locomotive (new head). The three pointers track "where I came from," "where I am," and "where I'm going."

Connect two linked lists end-to-end
Theory

To merge two queues, we reach the tail of the first list (the node whose next == NULL) and point its next to the head of the second list. The first list is extended by the entire second list.

// Merge two priority queues — O(n) to find tail of first
struct Node* merge(struct Node* head1, struct Node* head2) {
    if (head1 == NULL) return head2; // if list1 empty, return list2
    if (head2 == NULL) return head1; // if list2 empty, return list1

    // Traverse to find tail of first list
    struct Node* tail = head1;
    while (tail->next != NULL)
        tail = tail->next;

    // Connect tail of first to head of second
    tail->next = head2;

    printf("Lists merged successfully!\n");
    return head1; // head of merged list
}

16
Copy Priority Queue
Deep copy — create independent duplicate nodes
Theory

A deep copy creates entirely new nodes with the same data and priority values. The copy is independent — modifying the copy doesn't affect the original (unlike a shallow copy which just copies the pointer).

Shallow copy: copies pointer → same memory → dangerous!
Deep copy: allocates new nodes → separate memory → safe!
Time: O(n)Space: O(n)

// Deep copy of priority queue — O(n)
struct Node* copyQueue(struct Node* head) {
    if (head == NULL) return NULL;

    struct Node* newHead = NULL;
    struct Node* tail    = NULL;
    struct Node* current = head;

    while (current != NULL) {
        // Create brand new node (new memory address)
        struct Node* newNode = createNode(current->data,
                                              current->priority);
        if (newHead == NULL) {
            newHead = newNode;
            tail    = newNode;
        } else {
            tail->next = newNode; // append to copy
            tail        = newNode;
        }
        current = current->next;
    }

    printf("Deep copy created!\n");
    return newHead;
}
Real-Life Analogy
🖨
Photocopying Each Card
Photocopying a stack of task cards: each card gets a brand new physical copy. If you scribble on the photocopy, the original is unaffected — they're independent. Just taking a photo of the pile (shallow copy) would mean there's still only one set of real cards!

Find Maximum Priority Node
Traverse and return the node with highest urgency
Theory

Traverse the entire list and track the node with the smallest priority number (highest urgency). Return a pointer to that node. This is like peek but returns the full node reference for further use.

Time: O(n)Space: O(1)

// Find node with maximum priority (lowest priority number) — O(n)
struct Node* findMaxPriority(struct Node* head) {
    if (head == NULL) {
        printf("Queue is empty!\n");
        return NULL;
    }

    struct Node* current = head;
    struct Node* maxNode = head;

    while (current != NULL) {
        if (current->priority < maxNode->priority)
            maxNode = current; // found better candidate
        current = current->next;
    }

    printf("Max priority node: data=%d, priority=%d\n",
           maxNode->data, maxNode->priority);
    return maxNode; // returns pointer to the node
}
