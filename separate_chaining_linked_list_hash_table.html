MODULE 00
Introduction & Core Concept
Understand what a hash table is, what separate chaining solves, and how the entire structure works before we dive into operations.
🏫
Real-life Analogy

Imagine a school with numbered lockers (1–10). Each student's locker is determined by: locker# = last digit of student ID. Two students with IDs 23 and 43 both map to locker 3 — so they share a locker using a chain (linked list) attached to slot 3. That's separate chaining!

What is a Hash Table?
Key → Index Mapping

A hash table is a data structure that maps keys to values using a hash function. The hash function converts a key (e.g., a name or number) into an array index (bucket number) in O(1) average time.

What is Separate Chaining?
Collision Resolution via Linked Lists

When two keys hash to the same index (collision), separate chaining stores both in a linked list attached to that bucket. Each bucket is the head of its own linked list chain.

Time Complexity
Average Case O(1)

Insert, Delete, Search all average O(1) when load factor is low. Worst case O(n) when all keys collide into one chain.

Space Complexity
O(n + m)

n = number of stored elements, m = number of buckets. Extra pointers per node for linking.

Hash Function (Simple)
index = key % tableSize

Modulo operation maps any key to a valid bucket index. Good hash functions distribute keys uniformly.

C — Data Structures
/* Node of a linked list (one element in the chain) */
struct Node {
    int    key;        /* the key we store */
    int    value;      /* data associated with the key */
    struct Node *next; /* pointer to next node in chain */
};

/* Hash Table = array of Node pointers (bucket array) */
struct HashTable {
    struct Node **buckets; /* array of linked list heads */
    int          size;     /* number of buckets */
    int          count;    /* total elements stored */
};

/* Hash Function: maps key → bucket index */
int hashFunction(int key, int size) {
    return key % size; /* simple modulo hash */
}

C — Initialization
/* Step 1: Create and Initialize a Hash Table */
struct HashTable* createHashTable(int size) {

    /* Allocate memory for the hash table struct */
    struct HashTable *ht = (struct HashTable*) malloc(sizeof(struct HashTable));

    if (ht == NULL) {
        printf("Memory allocation failed!\n");
        return NULL;
    }

    /* Set table size */
    ht->size  = size;
    ht->count = 0;

    /* Allocate the bucket array (array of Node pointers) */
    ht->buckets = (struct Node**) calloc(size, sizeof(struct Node*));

    /* calloc already sets all buckets to NULL (zero-initialized) */
    if (ht->buckets == NULL) {
        free(ht);
        return NULL;
    }

    printf("Hash table of size %d created successfully!\n", size);
    return ht;
}

/* Usage */
int main() {
    struct HashTable *ht = createHashTable(7);
    /* Now ht->buckets[0..6] are all NULL — empty chains */
    return 0;
}
MODULE 02
Insertion Operations
Inserting a key-value pair involves hashing the key, navigating to the correct bucket, checking for duplicates, and adding a new node to the linked list chain.
📬
Real-life Analogy

Inserting is like a postal worker delivering mail. They compute the street number (hash), walk to that building, check if the tenant already exists (duplicate), then add the letter to the mailbox chain. If the box is full they link it to a side-chain.

Compute Hash Index: index = key % tableSize — find which bucket this key belongs to
Traverse Bucket List: Walk the existing linked list in that bucket to find the end or a duplicate
Check Duplicate Key: If key already exists → update its value (no duplicate nodes)
Insert at Head / Tail: Create new node; insert at head (O(1)) or tail (O(n))
Handle Collision: If another key already occupies the bucket, chain the new node behind it
Increase Element Count: Increment ht→count after successful insert

Insert at Head — O(1)
Fastest — New node becomes new head

Simply point new node's next to current head, then update bucket to point to new node. No traversal needed.

Insert at Tail — O(n)
Maintains order — traverses to end

Walk the entire chain to find the last node, then append. Useful when insertion order matters.

C — Insert at Head
/* Create a brand new node */
struct Node* createNode(int key, int value) {
    struct Node *node = (struct Node*) malloc(sizeof(struct Node));
    node->key   = key;
    node->value = value;
    node->next  = NULL;
    return node;
}

/* Insert key-value pair (insert at head for O(1)) */
void insert(struct HashTable *ht, int key, int value) {

    /* Step 1: Compute bucket index */
    int index = key % ht->size;

    /* Step 2: Check for duplicate key — update if found */
    struct Node *curr = ht->buckets[index];
    while (curr != NULL) {
        if (curr->key == key) {
            curr->value = value; /* update existing */
            printf("Key %d updated to %d\n", key, value);
            return;
        }
        curr = curr->next;
    }

    /* Step 3: Create new node */
    struct Node *newNode = createNode(key, value);

    /* Step 4: Insert at HEAD of bucket chain — O(1) */
    newNode->next        = ht->buckets[index]; /* new→next = old head */
    ht->buckets[index]   = newNode;            /* bucket = new node */

    /* Step 5: Increase element count */
    ht->count++;
    printf("Key %d inserted at bucket %d\n", key, index);
}
MODULE 03
Deletion Operations
Deleting a node requires finding it in the chain while maintaining the links of surrounding nodes, then freeing the memory.
🔗
Real-life Analogy

Imagine a chain of paperclips. To remove one from the middle, you unclip it and directly link the clip before it to the clip after it. The chain stays intact — just one clip shorter.

🔍 Find Target Node
Traverse the chain keeping track of previous node pointer
🗑️ Delete Head Node
Special case: bucket pointer updated to head's next node
✂️ Delete Middle Node
prev→next = curr→next, bypassing the target node
🔚 Delete Tail Node
prev→next = NULL, removing last element of chain
🔄 Update Links
Ensure chain integrity after removal — no broken pointers
🆓 Free Node Memory
Call free(node) to prevent memory leaks

C — Deletion (All Cases)
void delete(struct HashTable *ht, int key) {

    /* Step 1: Compute bucket index */
    int index = key % ht->size;

    struct Node *curr = ht->buckets[index];
    struct Node *prev = NULL;  /* track previous node */

    /* Step 2: Traverse the chain to find target */
    while (curr != NULL && curr->key != key) {
        prev = curr;
        curr = curr->next;
    }

    /* Key not found */
    if (curr == NULL) {
        printf("Key %d not found!\n", key);
        return;
    }

    /* Step 3: Delete head node (special case) */
    if (prev == NULL) {
        ht->buckets[index] = curr->next; /* move head forward */
    }
    /* Step 4: Delete middle or tail node */
    else {
        prev->next = curr->next; /* bypass current node */
    }

    /* Step 5: Free memory and update count */
    free(curr);
    ht->count--;
    printf("Key %d deleted from bucket %d\n", key, index);
}

Search Operations
Search is the most common hash table operation. We hash the key to find the bucket, then traverse the chain comparing keys.
🔎
Real-life Analogy

Like a library catalog — instead of searching every shelf, you look up the call number (hash) to jump directly to the right aisle, then scan only that aisle for your book.

🔍 Interactive — Search Visualization
Add sample data first, then search for a key.
Best Case — O(1)
Key is the only element in bucket

Hash directly to bucket, first node matches. No traversal needed.

Worst Case — O(n)
All keys in same bucket (poor hash)

Must traverse entire chain of n nodes. Happens with bad hash functions or clustered keys.

C — Search Operation
/* Search for a key and return its value (-1 if not found) */
int search(struct HashTable *ht, int key) {

    /* Step 1: Compute bucket index */
    int index = key % ht->size;

    /* Step 2: Traverse the bucket's linked list */
    struct Node *curr = ht->buckets[index];

    while (curr != NULL) {

        /* Step 3: Compare keys */
        if (curr->key == key) {
            printf("Found! Key=%d, Value=%d\n", key, curr->value);
            return curr->value; /* Step 4: Return value */
        }
        curr = curr->next;
    }

    /* Step 5: Detect missing key */
    printf("Key %d not found in table.\n", key);
    return -1;
}

C — Update Operation
int update(struct HashTable *ht, int key, int newValue) {

    /* Step 1: Compute bucket index */
    int index = key % ht->size;

    /* Step 2: Search for the target key */
    struct Node *curr = ht->buckets[index];

    while (curr != NULL) {
        if (curr->key == key) {

            /* Step 3: Key found — replace old value */
            int oldValue = curr->value;
            curr->value  = newValue;
            printf("Key %d: %d → %d\n", key, oldValue, newValue);
            return 1; /* success */
        }
        curr = curr->next;
    }

    /* Step 4: Key not found */
    printf("Error: Key %d does not exist!\n", key);
    return 0; /* failure */
}

C — Display All Key-Value Pairs
void display(struct HashTable *ht) {
    printf("\n=== Hash Table Contents ===\n");

    /* Step 1: Visit each bucket (outer loop) */
    for (int i = 0; i < ht->size; i++) {
        printf("Bucket[%d]: ", i);

        struct Node *curr = ht->buckets[i];

        /* Step 2: Traverse linked list at this bucket */
        if (curr == NULL) {
            printf("[EMPTY]");
        }
        while (curr != NULL) {

            /* Step 3 & 4: Display key and value */
            printf("[%d:%d]", curr->key, curr->value);

            if (curr->next != NULL)
                printf(" → ");

            curr = curr->next;
        }
        printf("\n");
    }
}
MODULE 07
Collision Handling Operations
A collision happens when two different keys produce the same hash index. Separate chaining handles this elegantly by building linked list chains.
🚗
Real-life Analogy

Think of a parking lot where spaces are numbered 0–6. Two cars arrive and both are assigned space 3 (same hash). Instead of turning one away, we park them in a chain — car A, then car B linked behind it. Any more collisions just extend the chain.

⛓ Create Chain
First collision: bucket gets its first node, chain is born
➕ Insert into Existing Chain
Subsequent collision: add node to existing chain (head or tail)
🚶 Traverse Collision Chain
Walk the chain node-by-node to find/delete/update
🔗 Maintain Chain Links
Keep next pointers intact during all modifications
🚫 Resolve Duplicate Entry
Same key collision → update value, not create a new node

C — Collision Handling in Insertion
/*
 * Separate chaining handles collision AUTOMATICALLY
 * in the insert function — no extra logic needed.
 * When bucket[index] already has a node, we just
 * chain the new node at the head.
 */

void insertWithCollisionHandling(struct HashTable *ht,
                                    int key, int value) {
    int index = key % ht->size;

    /* Check for same-key collision (duplicate) */
    struct Node *curr = ht->buckets[index];
    while (curr) {
        if (curr->key == key) {
            printf("Duplicate key %d — updating value\n", key);
            curr->value = value;
            return;
        }
        curr = curr->next;
    }

    /* Different-key collision: just chain it at head */
    struct Node *newNode  = createNode(key, value);
    newNode->next         = ht->buckets[index]; /* chain to existing */
    ht->buckets[index]    = newNode;            /* new head */
    ht->count++;

    if (newNode->next != NULL)
        printf("Collision at bucket %d — chain extended!\n", index);
}

C — Bucket Management
/* Detect empty bucket */
int isBucketEmpty(struct HashTable *ht, int index) {
    return ht->buckets[index] == NULL;
}

/* Count chain length at a given bucket */
int chainLength(struct HashTable *ht, int index) {
    int count = 0;
    struct Node *curr = ht->buckets[index];
    while (curr != NULL) {
        count++;
        curr = curr->next;
    }
    return count;
}

/* Find the bucket with the longest chain */
int findLongestChain(struct HashTable *ht) {
    int maxLen = 0, maxIdx = -1;
    for (int i = 0; i < ht->size; i++) {
        int len = chainLength(ht, i);
        if (len > maxLen) {
            maxLen = len;
            maxIdx = i;
        }
    }
    printf("Longest chain: bucket %d (length %d)\n", maxIdx, maxLen);
    return maxIdx;
}

/* Find shortest non-empty chain */
int findShortestChain(struct HashTable *ht) {
    int minLen = ht->count + 1, minIdx = -1;
    for (int i = 0; i < ht->size; i++) {
        int len = chainLength(ht, i);
        if (len > 0 && len < minLen) {
            minLen = len;
            minIdx = i;
        }
    }
    return minIdx;
}
MODULE 09
Load Factor Operations
The load factor measures how "full" the hash table is. It determines when to resize (rehash) to maintain O(1) performance.
⚖️
Real-life Analogy

Load factor is like a hotel occupancy rate. At 30% occupancy (load=0.3), guests easily find rooms. At 95% (load=0.95), finding a room takes much longer — staff has to check many floors. When occupancy is too high, the hotel builds a new wing (rehash)!

C — Load Factor Operations
/* Count total elements */
int countElements(struct HashTable *ht) {
    return ht->count; /* maintained by insert/delete */
}

/* Calculate load factor: λ = n / m */
float loadFactor(struct HashTable *ht) {
    return (float) ht->count / (float) ht->size;
}

/* Check if load factor exceeds threshold (0.75) */
int needsRehash(struct HashTable *ht) {
    float lf = loadFactor(ht);
    if (lf > 0.75f) {
        printf("Load factor %.2f > 0.75 — Rehashing needed!\n", lf);
        return 1; /* true */
    }
    printf("Load factor %.2f — OK\n", lf);
    return 0; /* false */
}
MODULE 10
Rehashing Operations
When load factor is too high, we create a bigger table (usually 2× size), recompute all hash indices, and reinsert all existing elements.
🏗️
Real-life Analogy

Like moving from a small apartment to a bigger house. You pack all your belongings (existing nodes), move to the new larger house (new table with 2× buckets), and replace each item in its new correct spot based on the new layout. Then demolish the old apartment.

Create New Table: Allocate new bucket array with 2× the original size
Recompute Hash Index: For every existing node, compute new_index = key % newSize
Reinsert Existing Nodes: Insert each node into the new table at its new index
Transfer Linked Lists: All chains are redistributed into the new table
Delete Old Table: Free old bucket array memory to prevent leaks

C — Rehashing
void rehash(struct HashTable *ht) {
    int oldSize      = ht->size;
    int newSize      = oldSize * 2; /* double the size */

    /* Step 1: Create new bucket array */
    struct Node **newBuckets = (struct Node**)
        calloc(newSize, sizeof(struct Node*));

    /* Step 2: Traverse all old buckets */
    for (int i = 0; i < oldSize; i++) {
        struct Node *curr = ht->buckets[i];

        /* Step 3: Reinsert each node into new table */
        while (curr != NULL) {
            struct Node *next  = curr->next; /* save next */

            /* Step 4: Recompute hash index for new size */
            int newIndex = curr->key % newSize;

            /* Insert at head of new bucket */
            curr->next         = newBuckets[newIndex];
            newBuckets[newIndex] = curr;

            curr = next; /* move to next old node */
        }
    }

    /* Step 5: Delete old bucket array */
    free(ht->buckets);

    /* Update hash table */
    ht->buckets = newBuckets;
    ht->size    = newSize;
    printf("Rehashed: size %d → %d\n", oldSize, newSize);
}

/* Clear all elements from the table (keep structure) */
void clearTable(struct HashTable *ht) {
    for (int i = 0; i < ht->size; i++) {
        struct Node *curr = ht->buckets[i];
        while (curr != NULL) {
            struct Node *temp = curr;
            curr = curr->next;
            free(temp); /* free each node */
        }
        ht->buckets[i] = NULL; /* reset bucket */
    }
    ht->count = 0;
    printf("Table cleared.\n");
}
MODULE 12
Memory Management Operations
C programs manage memory manually. Every malloc() must have a corresponding free(). Poor memory management causes leaks and crashes.
🏛️
Real-life Analogy

Memory management is like renting hotel rooms. When you check in (malloc), a room is reserved. When you check out (free), the room is released for others. If you never check out, the room is unavailable forever — that's a memory leak!

🆕 Create Node
malloc(sizeof(struct Node)) — allocate one node on heap
🆓 Free Single Node
free(node) — release one node's memory back to OS
🗑️ Free Entire Chain
Loop through chain, saving next before freeing each node
⚠️ Prevent Memory Leak
Always free before losing pointer reference; set to NULL after free

C — Memory Management
/* Create (allocate) a new node */
struct Node* createNode(int key, int value) {
    struct Node *node = (struct Node*) malloc(sizeof(struct Node));
    if (!node) { perror("malloc failed"); exit(1); }
    node->key   = key;
    node->value = value;
    node->next  = NULL;
    return node;
}

/* Free an entire linked list chain */
void freeChain(struct Node *head) {
    while (head != NULL) {
        struct Node *temp = head;   /* save current */
        head = head->next;           /* advance BEFORE free! */
        free(temp);                  /* release memory */
        temp = NULL;                 /* avoid dangling pointer */
    }
}

/* Completely free the entire hash table */
void freeHashTable(struct HashTable *ht) {
    for (int i = 0; i < ht->size; i++)
        freeChain(ht->buckets[i]); /* free each chain */
    free(ht->buckets);              /* free bucket array */
    free(ht);                       /* free struct itself */
}

/* Count total collisions */
int countCollisions(struct HashTable *ht) {
    int cols = 0;
    for (int i = 0; i < ht->size; i++) {
        int len = chainLength(ht, i);
        if (len > 1) cols += (len - 1); /* first is non-collision */
    }
    return cols;
}

/* Measure collision rate = collisions / total elements */
float collisionRate(struct HashTable *ht) {
    if (ht->count == 0) return 0.0;
    return (float) countCollisions(ht) / ht->count;
}

/* Calculate average chain length (non-empty buckets) */
float avgChainLength(struct HashTable *ht) {
    int nonEmpty = 0, total = 0;
    for (int i = 0; i < ht->size; i++) {
        int len = chainLength(ht, i);
        if (len > 0) { nonEmpty++; total += len; }
    }
    return nonEmpty ? (float) total / nonEmpty : 0.0;
}

/* Evaluate hash function efficiency (distribution uniformity) */
void evaluateHashFunction(struct HashTable *ht) {
    printf("=== Hash Function Evaluation ===\n");
    printf("Load Factor:      %.2f\n", loadFactor(ht));
    printf("Collision Rate:   %.2f%%\n", collisionRate(ht) * 100);
    printf("Avg Chain Length: %.2f\n", avgChainLength(ht));
    printf("Longest Chain:    bucket %d\n", findLongestChain(ht));
}
MODULE 14
Iterator Operations
An iterator allows sequential access to all elements without exposing the internal structure. It maintains state between calls using bucket index and node pointer.
🎬
Real-life Analogy

An iterator is like a TV remote's "Next Channel" button. Each press moves you to the next channel (element) — you don't need to know which frequency it's on internally. The remote tracks where you are so you can resume anytime.

➡️ Move to Next Bucket
When current chain is exhausted, advance to next non-empty bucket
⬇️ Move to Next Node
Follow node→next to advance within the current chain
👁️ Access Current Element
Return key and value of the node the iterator currently points to

C — Iterator Pattern
/* Iterator structure maintains traversal state */
struct Iterator {
    struct HashTable *ht;          /* reference to table */
    int              bucketIndex;  /* current bucket */
    struct Node     *currentNode;  /* current node in chain */
};

/* Initialize iterator at first element */
void iterInit(struct Iterator *it, struct HashTable *ht) {
    it->ht          = ht;
    it->bucketIndex = 0;
    it->currentNode = NULL;

    /* Advance to first non-empty bucket */
    while (it->bucketIndex < ht->size &&
           ht->buckets[it->bucketIndex] == NULL)
        it->bucketIndex++;

    if (it->bucketIndex < ht->size)
        it->currentNode = ht->buckets[it->bucketIndex];
}

/* Move to next element, return 1 if valid, 0 if done */
int iterNext(struct Iterator *it) {
    if (it->currentNode == NULL) return 0; /* done */

    /* Access current element */
    printf("[%d → %d] ", it->currentNode->key, it->currentNode->value);

    /* Move to next node in chain */
    it->currentNode = it->currentNode->next;

    /* If chain exhausted, move to next non-empty bucket */
    while (it->currentNode == NULL &&
           ++it->bucketIndex < it->ht->size)
        it->currentNode = it->ht->buckets[it->bucketIndex];

    return it->currentNode != NULL || it->bucketIndex < it->ht->size;
}

Merge Hash Tables
Union of Two Tables

Traverse all elements of table B and insert them into table A. On key conflicts, typically B's value overrides A's (or vice versa depending on policy).

Copy Hash Table
Deep Clone

Create a new table of same size, then insert every key-value pair from original. Must allocate new nodes — not just copy pointers (that would be shallow copy).

Compare Hash Tables
Check Equality

Two tables are equal if: same number of elements, and for every key in table A, the same key exists in table B with the same value. Search-based comparison — O(n) average.

/* Merge table B into table A (B's values override on conflict) */
void mergeTables(struct HashTable *a, struct HashTable *b) {
    for (int i = 0; i < b->size; i++) {
        struct Node *curr = b->buckets[i];
        while (curr != NULL) {
            insert(a, curr->key, curr->value); /* reuse insert */
            curr = curr->next;
        }
    }
}

/* Deep copy of hash table */
struct HashTable* copyTable(struct HashTable *src) {
    struct HashTable *dest = createHashTable(src->size);
    for (int i = 0; i < src->size; i++) {
        struct Node *curr = src->buckets[i];
        while (curr != NULL) {
            insert(dest, curr->key, curr->value);
            curr = curr->next;
        }
    }
    return dest;
}

/* Compare two hash tables (1 = equal, 0 = different) */
int compareTables(struct HashTable *a, struct HashTable *b) {
    if (a->count != b->count) return 0;
    for (int i = 0; i < a->size; i++) {
        struct Node *curr = a->buckets[i];
        while (curr != NULL) {
            int val = search(b, curr->key);
            if (val != curr->value) return 0; /* mismatch */
            curr = curr->next;
        }
    }
    return 1; /* equal */
}
