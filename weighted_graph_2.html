01
What is a Weighted Graph?
Core concept & terminology
📌 Theory

A graph is a collection of vertices (nodes) connected by edges. In a weighted graph, every edge carries a number called its weight (or cost). That weight can represent distance, time, money, bandwidth — anything measurable.

Key terms: Vertex a point/node. Edge a connection between two vertices. Weight the cost of traversing an edge. Directed edges have direction. Undirected edges go both ways.

🌍 Real-Life Analogy

Think of Google Maps. Cities are vertices. Roads connecting them are edges. The distance (or travel time) on each road is the weight. Finding the shortest route = finding the path with minimum total weight.

02
Weighted Adjacency List
Memory-efficient representation
📌 Theory

An adjacency list stores each vertex's neighbors in a linked list. For weighted graphs, each node in that list also stores the edge weight. This is more memory-efficient than an adjacency matrix for sparse graphs.

Structure: an array of size V (vertices). Each index stores a pointer to a linked list of (neighbor, weight) pairs.

Adjacency List

Space: O(V + E) — only stores existing edges.

Best for: Sparse graphs (few edges). Most real-world networks.

Adjacency Matrix

Space: O(V²) — stores even non-existent edges as 0.

Best for: Dense graphs where almost all pairs are connected.

Structure Visualization — Adjacency List in Memory
C — Structure Definition
/* Node in the adjacency list (neighbor + weight) */
struct AdjNode {
    int dest;           /* destination vertex */
    int weight;         /* edge weight       */
    struct AdjNode *next; /* pointer to next node */
};

/* Adjacency list head for one vertex */
struct AdjList {
    struct AdjNode *head;
};

/* The graph: array of adjacency lists */
struct Graph {
    int numVertices;
    struct AdjList *array; /* size = numVertices */
};

/* Create a new graph with V vertices */
struct Graph* createGraph(int V) {
    struct Graph *g = (struct Graph*) malloc(sizeof(struct Graph));
    g->numVertices = V;
    g->array = (struct AdjList*) malloc(V * sizeof(struct AdjList));
    for (int i = 0; i < V; i++)
        g->array[i].head = NULL;
    return g;
}

Insert Vertex
Vertex Operations
📌 Theory

Adding a vertex means growing the adjacency list array by one slot and initializing its head pointer to NULL. The new vertex starts with no edges.

🌍 Analogy

Like adding a new city to a map. The city exists but has no roads yet — those come later when you add edges.

C — Insert Vertex
/* Add a new vertex — resize the adjacency list array */
void insertVertex(struct Graph *g) {
    int V = g->numVertices;
    g->array = (struct AdjList*) realloc(
        g->array, (V + 1) * sizeof(struct AdjList)
    );
    g->array[V].head = NULL; /* no edges yet */
    g->numVertices++;
    printf("Vertex %d inserted.\n", V);
}

04
Delete Vertex
Vertex Operations
📌 Theory

Deleting vertex v requires: (1) free its entire adjacency list, (2) scan every other vertex's list and remove any edge pointing to v, (3) shift the array and update indices. This is the most complex vertex operation.

🌍 Analogy

Closing an airport. You must cancel not just the flights from that airport but also all flights from other airports that landed there.

C — Delete Vertex
/* Free an adjacency list */
void freeList(struct AdjNode *head) {
    struct AdjNode *tmp;
    while (head) { tmp = head; head = head->next; free(tmp); }
}

void deleteVertex(struct Graph *g, int v) {
    int V = g->numVertices;
    if (v < 0 || v >= V) { printf("Invalid vertex\n"); return; }

    /* Step 1: free vertex v's list */
    freeList(g->array[v].head);

    /* Step 2: remove edges pointing TO v in all other lists */
    for (int i = 0; i < V; i++) {
        if (i == v) continue;
        struct AdjNode *prev = NULL, *cur = g->array[i].head;
        while (cur) {
            if (cur->dest == v) {
                if (prev) prev->next = cur->next;
                else g->array[i].head = cur->next;
                free(cur); cur = (prev ? prev->next : g->array[i].head);
            } else { prev = cur; cur = cur->next; }
        }
        /* Decrement destinations > v (re-index) */
        for (AdjNode *n = g->array[i].head; n; n = n->next)
            if (n->dest > v) n->dest--;
    }
    /* Step 3: shift array */
    for (int i = v; i < V - 1; i++)
        g->array[i] = g->array[i + 1];
    g->numVertices--;
    printf("Vertex %d deleted.\n", v);
}
Time:O(V + E)
Space:O(1)

Search Vertex
Vertex Operations
📌 Theory

Searching for a vertex simply checks whether the given vertex ID is within the valid range [0, numVertices-1]. Since vertices are indexed, this is an O(1) check. Optionally, you can also display its neighbor list.

🌍 Analogy

Checking if city #5 exists on your map — just look at the index. If the map has 10 cities (0–9), city 5 exists.

C — Search Vertex
int searchVertex(struct Graph *g, int v) {
    if (v >= 0 && v < g->numVertices) {
        printf("Vertex %d found. Neighbors: ", v);
        struct AdjNode *n = g->array[v].head;
        if (!n) printf("(none)");
        while (n) {
            printf("%d(w=%d) ", n->dest, n->weight);
            n = n->next;
        }
        printf("\n");
        return 1; /* found */
    }
    printf("Vertex %d not found.\n", v);
    return 0; /* not found */
}
Time:O(1)

Insert Weighted Edge
Edge Operations
📌 Theory

To add edge (src → dest, weight): create a new AdjNode with dest and weight, then prepend it to array[src].head. For undirected graphs, also add (dest → src, weight).

🌍 Analogy

Building a road of known length between two cities. You record the road in both cities' travel guides (undirected), noting the exact distance.

C — Insert Weighted Edge
/* Create a new adjacency node */
struct AdjNode* newNode(int dest, int weight) {
    struct AdjNode *node = (struct AdjNode*) malloc(sizeof(struct AdjNode));
    node->dest   = dest;
    node->weight = weight;
    node->next   = NULL;
    return node;
}

/* Add undirected weighted edge (src -- dest, weight) */
void addEdge(struct Graph *g, int src, int dest, int weight) {
    /* src -> dest */
    struct AdjNode *node = newNode(dest, weight);
    node->next = g->array[src].head;
    g->array[src].head = node;

    /* dest -> src (undirected) */
    node = newNode(src, weight);
    node->next = g->array[dest].head;
    g->array[dest].head = node;
}
Time:O(1)
Space:O(1)
08
Delete Edge
Edge Operations
📌 Theory

Walk the adjacency list of src, find the node with dest, remove and free it. For undirected graphs, repeat for dest's list. Use a previous-pointer technique to safely unlink the node.

C — Delete Edge
void removeFromList(struct AdjList *list, int target) {
    struct AdjNode *prev = NULL, *cur = list->head;
    while (cur && cur->dest != target) {
        prev = cur; cur = cur->next;
    }
    if (!cur) { printf("Edge not found.\n"); return; }
    if (prev) prev->next = cur->next;
    else      list->head = cur->next;
    free(cur);
}

void deleteEdge(struct Graph *g, int src, int dest) {
    removeFromList(&g->array[src],  dest);
    removeFromList(&g->array[dest], src);  /* undirected */
    printf("Edge %d -- %d deleted.\n", src, dest);
}
Time:O(degree(src))

Breadth-First Search (BFS)
Traversal Operations
📌 Theory

BFS explores neighbors level by level. It uses a queue. Start at source, mark visited, enqueue it. Dequeue a vertex, visit all its unvisited neighbors, enqueue them. BFS finds the fewest-hop path (ignoring weights). In weighted graphs, BFS is used for structure analysis, not shortest weighted paths.

🌍 Analogy

Ripples in a pond — all nodes at distance 1 are visited before nodes at distance 2, before nodes at distance 3, and so on.

BFS Traversal Animation
Click "Run BFS" to animate traversal.
C — BFS
void BFS(struct Graph *g, int start) {
    int visited[100] = {0};   /* visited flags */
    int queue[100], front = 0, rear = 0;

    visited[start] = 1;
    queue[rear++] = start;

    printf("BFS: ");
    while (front < rear) {
        int v = queue[front++];
        printf("%d ", v);

        struct AdjNode *n = g->array[v].head;
        while (n) {
            if (!visited[n->dest]) {
                visited[n->dest] = 1;
                queue[rear++] = n->dest;
            }
            n = n->next;
        }
    }
    printf("\n");
}
Time:O(V + E)
Space:O(V)

Depth-First Search (DFS)
Traversal Operations
📌 Theory

DFS dives as deep as possible along each branch before backtracking. It uses a stack (or recursion). DFS is used for cycle detection, topological sort, and finding connected components. In weighted graphs it helps detect all reachable nodes.

🌍 Analogy

Exploring a maze — always go forward until you hit a dead end, then backtrack and try another route.

C — DFS (Recursive)
void DFSUtil(struct Graph *g, int v, int visited[]) {
    visited[v] = 1;
    printf("%d ", v);
    struct AdjNode *n = g->array[v].head;
    while (n) {
        if (!visited[n->dest])
            DFSUtil(g, n->dest, visited);
        n = n->next;
    }
}

void DFS(struct Graph *g, int start) {
    int visited[100] = {0};
    printf("DFS: ");
    DFSUtil(g, start, visited);
    printf("\n");
}
Time:O(V + E)
Space:O(V)

Weighted Traversal Analysis
Traversal Operations
📌 Theory

During a BFS or DFS traversal of a weighted graph, we can accumulate the total weight of edges visited, find the maximum/minimum edge encountered, or analyze cost per depth level. This is useful for network capacity planning and route cost estimation.

C — DFS with Accumulated Weight
void DFSWeighted(struct Graph *g, int v,
                  int visited[], int *totalCost) {
    visited[v] = 1;
    struct AdjNode *n = g->array[v].head;
    while (n) {
        if (!visited[n->dest]) {
            *totalCost += n->weight;
            printf("  Edge %d->%d  weight=%d  running total=%d\n",
                   v, n->dest, n->weight, *totalCost);
            DFSWeighted(g, n->dest, visited, totalCost);
        }
        n = n->next;
    }
}

void weightedTraversal(struct Graph *g, int src) {
    int visited[100] = {0};
    int total = 0;
    printf("Weighted DFS from %d:\n", src);
    DFSWeighted(g, src, visited, &total);
    printf("Total traversal cost = %d\n", total);
}
15
Find Path
Path Operations
📌 Theory

Find any path from source to destination using DFS. We keep a path[] array to track the current route and a visited[] array to avoid revisiting. When we reach the destination, we print the path and its total weight.

🌍 Analogy

Finding ANY road from city A to city B — not necessarily the fastest, just a valid connected route.

C — Find Path (DFS-based)
int findPathUtil(struct Graph *g, int src, int dest,
                  int visited[], int path[], int pathIdx) {
    visited[src] = 1;
    path[pathIdx] = src;

    if (src == dest) {
        printf("Path: ");
        for (int i = 0; i <= pathIdx; i++)
            printf("%d%s", path[i], i < pathIdx ? "->" : "\n");
        return 1;
    }
    struct AdjNode *n = g->array[src].head;
    while (n) {
        if (!visited[n->dest])
            if (findPathUtil(g, n->dest, dest, visited, path, pathIdx + 1))
                return 1;
        n = n->next;
    }
    return 0;
}

void findPath(struct Graph *g, int src, int dest) {
    int visited[100] = {0};
    int path[100];
    if (!findPathUtil(g, src, dest, visited, path, 0))
        printf("No path from %d to %d.\n", src, dest);
}
16
Shortest Path (Basic)
Path Operations — Greedy DFS
📌 Theory

A simple approach to find the path with minimum total weight is exhaustive DFS — explore all paths and track the minimum cost. This is O(V!) in the worst case. For large graphs, use Dijkstra's algorithm (covered later).

C — Shortest Path via Exhaustive DFS
void shortestDFS(struct Graph *g, int cur, int dest,
                  int visited[], int curCost, int *minCost) {
    if (cur == dest) {
        if (curCost < *minCost) *minCost = curCost;
        return;
    }
    visited[cur] = 1;
    struct AdjNode *n = g->array[cur].head;
    while (n) {
        if (!visited[n->dest])
            shortestDFS(g, n->dest, dest, visited,
                         curCost + n->weight, minCost);
        n = n->next;
    }
    visited[cur] = 0; /* backtrack */
}

void findShortestPath(struct Graph *g, int src, int dest) {
    int visited[100] = {0};
    int minCost = 99999;
    shortestDFS(g, src, dest, visited, 0, &minCost);
    if (minCost == 99999) printf("No path.\n");
    else printf("Min cost from %d to %d = %d\n", src, dest, minCost);
}

Longest Path (Specific Cases)
Path Operations
📌 Theory

The longest path problem is NP-hard for general graphs. However, in DAGs (Directed Acyclic Graphs) it can be solved in O(V+E) using topological sort + dynamic programming. For weighted DAGs, we negate weights and run Bellman-Ford, or use DP directly.

🌍 Analogy

Finding the longest scenic route from A to B that visits the most beautiful (high-weight) roads — useful in pipeline scheduling where we want the critical path.

C — Longest Path (Exhaustive DFS)
void longestDFS(struct Graph *g, int cur, int dest,
                 int visited[], int curCost, int *maxCost) {
    if (cur == dest) {
        if (curCost > *maxCost) *maxCost = curCost;
        return;
    }
    visited[cur] = 1;
    struct AdjNode *n = g->array[cur].head;
    while (n) {
        if (!visited[n->dest])
            longestDFS(g, n->dest, dest, visited,
                        curCost + n->weight, maxCost);
        n = n->next;
    }
    visited[cur] = 0; /* backtrack */
}

Minimum Cost Path
Path Operations
📌 Theory

The minimum cost path in an unweighted sense means fewest hops (BFS). With weights, it means minimum total weight (Dijkstra). Here we use the exhaustive approach to also track and print the actual path, not just the cost.

C — Min Cost Path with Route Tracking
int bestPath[100], bestLen = 0, bestCost = 99999;

void minCostDFS(struct Graph *g, int cur, int dest,
               int visited[], int path[], int idx, int cost) {
    visited[cur] = 1;
    path[idx++] = cur;
    if (cur == dest && cost < bestCost) {
        bestCost = cost; bestLen = idx;
        for (int i = 0; i < idx; i++) bestPath[i] = path[i];
    }
    struct AdjNode *n = g->array[cur].head;
    while (n) {
        if (!visited[n->dest])
            minCostDFS(g, n->dest, dest, visited, path,
                        idx, cost + n->weight);
        n = n->next;
    }
    visited[cur] = 0;
}

Dijkstra's Algorithm
Shortest Path Algorithms
📌 Theory

Dijkstra's finds the shortest path from a source to all other vertices in a weighted graph with non-negative weights. It uses a greedy approach: always pick the unvisited vertex with the smallest known distance, then relax its neighbors.

Steps: (1) Set dist[src]=0, all others=∞. (2) Pick unvisited vertex u with min dist. (3) For each neighbor v: if dist[u]+w(u,v) < dist[v], update dist[v]. (4) Mark u visited. Repeat.

🌍 Analogy

Spreading water from a source — it always flows along the cheapest path first, gradually reaching all nodes in order of increasing distance.

Click "Next Step" to run Dijkstra from node A.
C — Dijkstra's Algorithm
#define INF 99999

int minDistance(int dist[], int visited[], int V) {
    int min = INF, minIdx = -1;
    for (int v = 0; v < V; v++)
        if (!visited[v] && dist[v] < min)
            { min = dist[v]; minIdx = v; }
    return minIdx;
}

void dijkstra(struct Graph *g, int src) {
    int V = g->numVertices;
    int dist[100], visited[100], parent[100];

    for (int i = 0; i < V; i++)
        { dist[i] = INF; visited[i] = 0; parent[i] = -1; }
    dist[src] = 0;

    for (int count = 0; count < V - 1; count++) {
        int u = minDistance(dist, visited, V);
        if (u == -1) break;
        visited[u] = 1;

        struct AdjNode *n = g->array[u].head;
        while (n) {
            int v = n->dest, w = n->weight;
            if (!visited[v] && dist[u] + w < dist[v]) {
                dist[v]   = dist[u] + w;
                parent[v] = u;
            }
            n = n->next;
        }
    }
    /* Print results */
    printf("Vertex  Distance  Parent\n");
    for (int i = 0; i < V; i++)
        printf("%6d  %8d  %6d\n", i, dist[i], parent[i]);
}
Time:O(V²)
With min-heap:O((V+E) log V)
Limitation:No negative weights

Bellman-Ford Algorithm
Shortest Path Algorithms
📌 Theory

Bellman-Ford handles negative weight edges (but not negative cycles). It relaxes ALL edges V-1 times. On the V-th relaxation, if any distance still improves, a negative cycle exists.

Key difference from Dijkstra: slower (O(V·E)) but supports negative weights. Used in currency arbitrage detection.

🌍 Analogy

A rumor spreading through a network — it takes multiple "rounds" before everyone has the most accurate (cheapest) information, especially if some routes have discounts (negative weights).

C — Bellman-Ford Algorithm
struct Edge { int src, dest, weight; };

void bellmanFord(struct Edge edges[], int E, int V, int src) {
    int dist[100];
    for (int i = 0; i < V; i++) dist[i] = INF;
    dist[src] = 0;

    /* Relax all edges V-1 times */
    for (int i = 1; i <= V - 1; i++) {
        for (int j = 0; j < E; j++) {
            int u = edges[j].src, v = edges[j].dest, w = edges[j].weight;
            if (dist[u] != INF && dist[u] + w < dist[v])
                dist[v] = dist[u] + w;
        }
    }
    /* Check for negative-weight cycles */
    for (int j = 0; j < E; j++) {
        int u = edges[j].src, v = edges[j].dest, w = edges[j].weight;
        if (dist[u] != INF && dist[u] + w < dist[v])
            { printf("Negative cycle detected!\n"); return; }
    }
    for (int i = 0; i < V; i++)
        printf("dist[%d] = %d\n", i, dist[i]);
}
Time:O(V · E)
Supports:Negative weights
22
Floyd-Warshall Algorithm
Shortest Path Algorithms — All Pairs
📌 Theory

Floyd-Warshall finds shortest paths between ALL pairs of vertices. It uses dynamic programming: for each intermediate vertex k, check if going through k is shorter than the direct path. Uses an adjacency matrix internally.

🌍 Analogy

A complete travel guide for all city pairs — for every possible origin and destination, it tells you the cheapest route, potentially via intermediate hubs.

C — Floyd-Warshall
void floydWarshall(int dist[][100], int V) {
    /* dist[][] initialized from adjacency matrix
       dist[i][i]=0, dist[i][j]=weight or INF */
    for (int k = 0; k < V; k++)       /* intermediate vertex */
        for (int i = 0; i < V; i++)   /* source */
            for (int j = 0; j < V; j++) /* dest */
                if (dist[i][k] != INF && dist[k][j] != INF &&
                    dist[i][k] + dist[k][j] < dist[i][j])
                    dist[i][j] = dist[i][k] + dist[k][j];

    /* Print shortest distances */
    printf("All-Pairs Shortest Paths:\n");
    for (int i = 0; i < V; i++) {
        for (int j = 0; j < V; j++)
            if (dist[i][j] == INF) printf("%5s", "INF");
            else printf("%5d", dist[i][j]);
        printf("\n");
    }
}
Time:O(V³)
Space:O(V²)
Finds:All pairs
23
A* Search (Advanced)
Shortest Path Algorithms — Heuristic
📌 Theory

A* extends Dijkstra with a heuristic function h(n) that estimates the cost from node n to the goal. It selects nodes based on f(n) = g(n) + h(n) where g(n) is actual cost from start and h(n) is estimated cost to goal. If h is admissible (never overestimates), A* is guaranteed optimal.

Common heuristics: Manhattan distance for grids, Euclidean distance for geometric graphs. A* is used in GPS navigation, game AI pathfinding, and robotics.

🌍 Analogy

Navigation that "looks ahead" — instead of exploring equally in all directions like Dijkstra, A* uses your compass to preferentially explore roads going toward your destination.

C — A* Search (Simplified)
/* heuristic[]: pre-computed h(n) estimates for each vertex */
void aStar(struct Graph *g, int src, int dest, int heuristic[]) {
    int V = g->numVertices;
    int gCost[100], fCost[100], visited[100], parent[100];
    for (int i = 0; i < V; i++) {
        gCost[i] = INF; fCost[i] = INF;
        visited[i] = 0;  parent[i] = -1;
    }
    gCost[src] = 0;
    fCost[src] = heuristic[src];

    while (1) {
        /* Pick unvisited node with smallest fCost */
        int u = -1, minF = INF;
        for (int i = 0; i < V; i++)
            if (!visited[i] && fCost[i] < minF)
                { minF = fCost[i]; u = i; }
        if (u == -1 || u == dest) break;
        visited[u] = 1;

        struct AdjNode *n = g->array[u].head;
        while (n) {
            int v = n->dest;
            int tentG = gCost[u] + n->weight;
            if (tentG < gCost[v]) {
                gCost[v]  = tentG;
                fCost[v]  = tentG + heuristic[v];
                parent[v] = u;
            }
            n = n->next;
        }
    }
    printf("A* cost to %d = %d\n", dest,
           gCost[dest] == INF ? -1 : gCost[dest]);
}

Detect Cycle
Cycle Operations
📌 Theory

For undirected graphs: use DFS. If you visit a neighbor that is already visited and is NOT the parent, a cycle exists. For directed graphs: use a recursion stack — if a node is visited and on the current DFS stack, there's a cycle.

🌍 Analogy

Walking roads and realizing you've returned to a city you already visited by a different route — that's a cycle in the road network.

C — Cycle Detection (Undirected)
int cycleUtil(struct Graph *g, int v, int visited[], int parent) {
    visited[v] = 1;
    struct AdjNode *n = g->array[v].head;
    while (n) {
        if (!visited[n->dest]) {
            if (cycleUtil(g, n->dest, visited, v)) return 1;
        } else if (n->dest != parent) {
            return 1; /* back edge = cycle */
        }
        n = n->next;
    }
    return 0;
}

int hasCycle(struct Graph *g) {
    int visited[100] = {0};
    for (int i = 0; i < g->numVertices; i++)
        if (!visited[i])
            if (cycleUtil(g, i, visited, -1)) {
                printf("Cycle DETECTED!\n"); return 1;
            }
    printf("No cycle found.\n"); return 0;
}

Weighted Cycle Cost Analysis
Cycle Operations
📌 Theory

Once a cycle is detected, we can compute its total weight by summing the weights of all edges in the cycle. This is critical in finance (negative-weight cycles = arbitrage) and network routing (routing loops can waste bandwidth proportional to the loop's weight).

🌍 Analogy

A delivery driver going in a circular route — the cycle cost tells you exactly how much fuel is wasted by the unnecessary loop.

C — Compute Cycle Weight
/* Detect cycle AND compute its cost */
int cycleWeightUtil(struct Graph *g, int v,
                    int visited[], int parent,
                    int path[], int idx, int *cycleCost) {
    visited[v] = 1;
    path[idx++] = v;
    struct AdjNode *n = g->array[v].head;
    while (n) {
        if (!visited[n->dest]) {
            if (cycleWeightUtil(g, n->dest, visited, v, path, idx, cycleCost))
                return 1;
        } else if (n->dest != parent) {
            *cycleCost += n->weight;
            return 1;
        }
        n = n->next;
    }
    return 0;
}

Prim's Algorithm (MST)
Minimum Spanning Tree
📌 Theory

A Minimum Spanning Tree (MST) is a subset of edges that connects all vertices with minimum total weight and no cycles. Prim's algorithm grows the MST greedily: start with any vertex, always add the cheapest edge that connects a new vertex to the tree.

🌍 Analogy

Laying the cheapest possible network of pipes to supply water to all houses — add the cheapest pipe that connects a new unserved house each time.

Click "Next MST Edge" to build the tree.
C — Prim's Algorithm
void primsAlgo(struct Graph *g) {
    int V = g->numVertices;
    int parent[100], key[100], inMST[100];

    for (int i = 0; i < V; i++)
        { key[i] = INF; inMST[i] = 0; parent[i] = -1; }
    key[0] = 0;

    for (int cnt = 0; cnt < V; cnt++) {
        /* Pick vertex not in MST with minimum key */
        int u = -1, minKey = INF;
        for (int v = 0; v < V; v++)
            if (!inMST[v] && key[v] < minKey)
                { minKey = key[v]; u = v; }
        if (u == -1) break;
        inMST[u] = 1;

        struct AdjNode *n = g->array[u].head;
        while (n) {
            if (!inMST[n->dest] && n->weight < key[n->dest]) {
                key[n->dest]    = n->weight;
                parent[n->dest] = u;
            }
            n = n->next;
        }
    }
    printf("MST edges (Prim):\n");
    int total = 0;
    for (int i = 1; i < V; i++) {
        printf("  %d -- %d  weight=%d\n", parent[i], i, key[i]);
        total += key[i];
    }
    printf("Total MST cost = %d\n", total);
}
Time:O(V²)
With heap:O(E log V)

Kruskal's Algorithm (MST)
Minimum Spanning Tree
📌 Theory

Kruskal's sorts all edges by weight, then adds each edge to the MST if it doesn't create a cycle (checked using Union-Find/Disjoint Set). It works edge-by-edge regardless of vertex connectivity.

Key data structure: Union-Find (Disjoint Set Union) — efficiently tracks which vertices are in the same component to detect cycles in O(α(n)) ≈ O(1).

🌍 Analogy

Sorting all possible roads by cost and building cheapest first — skip any road that would create a loop between already-connected cities.

C — Kruskal's Algorithm with Union-Find
int parent_uf[100], rank_uf[100];

int find(int x) {
    if (parent_uf[x] != x) parent_uf[x] = find(parent_uf[x]);
    return parent_uf[x];
}
void unionSets(int a, int b) {
    int ra = find(a), rb = find(b);
    if (rank_uf[ra] < rank_uf[rb]) parent_uf[ra] = rb;
    else if (rank_uf[ra] > rank_uf[rb]) parent_uf[rb] = ra;
    else { parent_uf[rb] = ra; rank_uf[ra]++; }
}
/* Comparison for qsort */
int cmpEdge(const void *a, const void *b) {
    return ((struct Edge*)a)->weight - ((struct Edge*)b)->weight;
}

void kruskal(struct Edge edges[], int E, int V) {
    qsort(edges, E, sizeof(struct Edge), cmpEdge);
    for (int i = 0; i < V; i++)
        { parent_uf[i] = i; rank_uf[i] = 0; }

    printf("MST edges (Kruskal):\n");
    int total = 0, edgesCnt = 0;
    for (int i = 0; i < E && edgesCnt < V - 1; i++) {
        int u = edges[i].src, v = edges[i].dest;
        if (find(u) != find(v)) {
            unionSets(u, v);
            printf("  %d -- %d  w=%d\n", u, v, edges[i].weight);
            total += edges[i].weight;
            edgesCnt++;
        }
    }
    printf("Total MST cost = %d\n", total);
}
Time:O(E log E)
Union-Find:O(α(V)) ≈ O(1)

Optimization Operations

Bipartite Check
Advanced Operations
📌 Theory

A graph is bipartite if its vertices can be split into two sets such that every edge connects a vertex in one set to one in the other. Equivalently, a graph is bipartite if and only if it has no odd-length cycles. Check using BFS 2-coloring.

🌍 Analogy

A job market graph — workers on one side, jobs on the other. Edges represent "this worker can do this job." No worker-worker or job-job edges. This is bipartite by design.

C — Bipartite Check (BFS)
int isBipartite(struct Graph *g) {
    int color[100];
    for(int i=0;inumVertices;i++) color[i]=-1;
    int queue[100],f=0,r=0;

    for(int start=0;startnumVertices;start++){
        if(color[start]!=-1) continue;
        color[start]=0; queue[r++]=start;
        while(fint u=queue[f++];
            struct AdjNode*n=g->array[u].head;
            while(n){
                if(color[n->dest]==-1){
                    color[n->dest]=1-color[u];
                    queue[r++]=n->dest;
                } else if(color[n->dest]==color[u]){
                    printf("NOT bipartite.\n"); return 0;
                }
                n=n->next;
            }
        }
    }
    printf("Graph IS bipartite.\n"); return 1;
}

Network Flow (Ford-Fulkerson)
Advanced Operations
📌 Theory

Network flow finds the maximum flow from a source to a sink in a directed weighted graph where edge weights represent capacities. Ford-Fulkerson uses BFS to find augmenting paths in the residual graph, adding flow until no more paths exist.

Key concepts: Capacity max flow on an edge. Residual graph remaining capacity. Augmenting path path from source to sink with spare capacity. Max-flow min-cut theorem max flow = min cut.

🌍 Analogy

Water flowing through pipes of different diameters — the maximum water that can flow from reservoir to city, limited by the narrowest pipes (bottleneck).

Strongly Connected Components
Advanced Operations — Kosaraju's
📌 Theory

In a directed graph, a Strongly Connected Component (SCC) is a maximal subgraph where every vertex is reachable from every other. Kosaraju's algorithm: (1) DFS on original graph recording finish times, (2) transpose the graph (reverse all edges), (3) DFS in decreasing finish order on transposed graph — each DFS tree is one SCC.

🌍 Analogy

Social circles in a directed "who follows whom" network — an SCC is a group where everyone can reach everyone else by following the follow chains.

C — Kosaraju's SCC Algorithm
int finishStack[100], top_fs = 0;

void dfsFinish(struct Graph *g, int v, int vis[]) {
    vis[v] = 1;
    struct AdjNode*n=g->array[v].head;
    while(n){ if(!vis[n->dest]) dfsFinish(g,n->dest,vis); n=n->next;}
    finishStack[top_fs++] = v; /* push on finish */
}

struct Graph* transpose(struct Graph *g) {
    struct Graph *t = createGraph(g->numVertices);
    for(int v=0;vnumVertices;v++){
        struct AdjNode*n=g->array[v].head;
        while(n){ addEdge(t,n->dest,v,n->weight); n=n->next;}
    }
    return t;
}

void kosarajuSCC(struct Graph *g) {
    int vis[100]={0};
    for(int i=0;inumVertices;i++)
        if(!vis[i]) dfsFinish(g,i,vis);

    struct Graph*gt = transpose(g);
    int vis2[100]={0};
    int sccNum=0;
    while(top_fs>0){
        int v=finishStack[--top_fs];
        if(!vis2[v]){
            sccNum++;
            printf("SCC %d: ",sccNum);
            DFSUtil(gt,v,vis2);
            printf("\n");
        }
    }
}
Time:O(V + E)
