01

Circular Queue Initialization

Foundation

📖Theory

A Circular Queue using a linked list is a dynamic data structure where the last node's next pointer loops back to the first node — forming a circle. Unlike arrays, it has no fixed size limit.

Key Idea: Two global pointers manage the queue:
• front → points to the first (oldest) element
• rear → points to the last (newest) element
• rear→next = front always (circular link)
  • Each node stores data and a next pointer
  • Empty state: both front and rear are NULL
  • The circular structure is established on the very first insertion
  • Memory is allocated dynamically using malloc()
  • No wasted space — grows and shrinks as needed
Create empty queue front = NULL rear = NULL Circular structure setup

💻C Code — Initialization

C LANGUAGE
#include <stdio.h>
#include <stdlib.h>

/* ── Node Structure ── */
struct Node {
    int          data;   /* The actual value stored */
    struct Node* next;   /* Pointer to next node   */
};

/* ── Global Queue Pointers ── */
struct Node* front = NULL;  /* Points to first element */
struct Node* rear  = NULL;  /* Points to last element  */

/* ── Initialize: just set both pointers to NULL ── */
void initQueue() {
    front = NULL;
    rear  = NULL;
    printf("Queue initialized. front=NULL, rear=NULL\n");
}

int main() {
    initQueue();

    /* Check if empty — NULL means no elements */
    if (front == NULL) {
        printf("Queue is empty and ready to use!\n");
    }

    return 0;
}

🎠
The merry-go-round is the circular queue structure itself — circular by design.
🚫
No children seated yet = front == NULL and rear == NULL. Nothing to process.
🔗
The track loops back on itself = the circular pointer (rear→next = front) that makes it a circle.
02

Enqueue — Insert at Rear

Foundation

📖Theory

Enqueue adds a new element at the rear end of the queue. New elements always go to the back — just like a ticket counter line where new arrivals join at the end.

Step-by-Step:
1. Create a new node with malloc()
2. If queue is empty: new node becomes both front and rear, and it points to itself
3. If not empty: link rear's next to new node, move rear forward, then connect rear→next back to front
  • Always maintain: rear→next = front after insertion
  • First node is both front AND rear simultaneously
  • Time complexity: O(1) — no traversal needed
  • Space: O(1) per operation
Create new node malloc() Insert first node Update rear pointer Circular connection O(1) time
TimeO(1)
SpaceO(1)

💻C Code — Enqueue

C LANGUAGE
#include <stdio.h>
#include <stdlib.h>

struct Node { int data; struct Node* next; };
struct Node* front = NULL;
struct Node* rear  = NULL;

void enqueue(int value) {

    /* Step 1: Allocate memory for a new node */
    struct Node* newNode = (struct Node*)malloc(sizeof(struct Node));

    if (newNode == NULL) {           /* Always check malloc result! */
        printf("Memory allocation failed!\n");
        return;
    }

    newNode->data = value;           /* Store the value */
    newNode->next = NULL;           /* Initialize next pointer */

    /* Step 2: Handle empty queue (first insertion) */
    if (front == NULL) {
        front         = newNode;     /* New node is the front */
        rear          = newNode;     /* New node is also rear */
        rear->next    = front;       /* Points to itself → circular! */
    }

    /* Step 3: Queue already has elements */
    else {
        rear->next = newNode;        /* Old rear links to new node */
        rear       = newNode;        /* Rear moves to new node    */
        rear->next = front;          /* Maintain circular link     */
    }

    printf("Enqueued: %d\n", value);
}

int main() {
    enqueue(10);   /* First node: front=rear=10, 10→10 */
    enqueue(20);   /* front=10, rear=20,  10→20→10     */
    enqueue(30);   /* front=10, rear=30,  10→20→30→10  */
    return 0;
}

🎯Real-Life Analogy

"Think of people queuing for a roller coaster. Each new person joins at the back of the line. Once they board and the ride completes a loop, the last person is right next to where the first person started — forming a circle."
🎢
A new person joining = calling enqueue(value) — they always go to the rear.
1️⃣
First person in line = they are simultaneously front AND rear until a second person joins.
🔁
Circular track = rear's next always points back to front, keeping the circular structure intact.
03

Dequeue — Delete from Front

Foundation

📖Theory

Dequeue removes an element from the front. This follows the FIFO principle — first in, first out. The oldest element leaves first.

Step-by-Step:
1. Check if queue is empty (underflow check!)
2. Save the front's data to return it
3. If only one node: set front = rear = NULL
4. Otherwise: advance front, update rear→next = front, free old node
  • Underflow: dequeuing from empty queue — must be checked first
  • Single-node case: after removal, both pointers become NULL
  • Always call free() on the removed node to prevent memory leaks
  • Time complexity: O(1)
Queue underflow check Remove front node Move front pointer Single-node queue Update rear link free() node O(1) time
TimeO(1)
SpaceO(1)

💻C Code — Dequeue

C LANGUAGE
int dequeue() {

    /* Step 1: Underflow check — queue must not be empty */
    if (front == NULL) {
        printf("Queue Underflow! Queue is empty.\n");
        return -1;              /* Return sentinel value */
    }

    /* Step 2: Save the data we are about to remove */
    int removedData = front->data;
    struct Node* temp = front;  /* Keep a pointer to free later */

    /* Step 3: Special case — only ONE node in the queue */
    if (front == rear) {
        front = NULL;
        rear  = NULL;
    }

    /* Step 4: General case — two or more nodes */
    else {
        front      = front->next; /* Advance front to next node */
        rear->next = front;       /* Relink rear back to new front */
    }

    /* Step 5: Free memory of the removed node */
    free(temp);
    printf("Dequeued: %d\n", removedData);
    return removedData;
}

int main() {
    enqueue(10);  enqueue(20);  enqueue(30);
    dequeue();   /* Removes 10 → front becomes 20 */
    dequeue();   /* Removes 20 → front becomes 30 */
    dequeue();   /* Removes 30 → queue is now empty */
    dequeue();   /* Underflow! Queue is empty.    */
    return 0;
}

🎯Real-Life Analogy

"At a busy cafeteria, the tray that was placed on the stack first is used first. In a queue (not a stack!), the first customer to arrive at the counter is the first to be served and leave — FIFO: First In, First Out."
🍽️
Customer at the front = the node pointed to by front — first to be served and removed.
⚠️
No customers in line = underflow condition — you can't serve someone who isn't there.
🔄
Last customer = first in new loop = rear→next = front is updated so the circle remains intact.

Peek / Front Element

Core Access

📖Theory

Peek lets you look at the front element without removing it. It's a read-only operation — the queue structure stays completely unchanged.

Why is this useful?
Sometimes you need to check what's next before deciding to dequeue. For example, a scheduler might peek at the next task's priority before actually running it.
  • Returns front→data without modifying the queue
  • Must check for empty queue first (just like dequeue)
  • Read-only operation — no pointer updates needed
  • Time complexity: O(1)
Access front node data Empty queue check Read-only operation

💻C Code — Peek

C LANGUAGE
/* Peek: Returns front element without removing it */
int peek() {

    /* First, check if the queue is empty */
    if (front == NULL) {
        printf("Queue is empty! Nothing to peek at.\n");
        return -1;     /* Sentinel: no valid element */
    }

    /* Just read the data — no pointer changes! */
    printf("Front element is: %d\n", front->data);
    return front->data;
}

int main() {
    enqueue(10);  enqueue(20);  enqueue(30);

    peek();       /* Output: Front element is: 10 */
    peek();       /* Output: Front element is: 10 (unchanged!) */

    /* Compare: dequeue actually removes it */
    dequeue();    /* Removes 10 */
    peek();       /* Output: Front element is: 20 (now 20 is front) */
    return 0;
}

🎯Real-Life Analogy

"Peeking is like looking at the name on the first envelope in your mailbox without opening or removing it. You know what's there, but you haven't actually done anything with it yet."
📬
Looking at the first envelope = peek() reads front→data without removing the node.
👀
No mail is taken out = the queue is completely unchanged after peek — all pointers remain the same.

Rear Element Access

Core Access

📖Theory

Just as we can peek at the front, we can also read the rear element — the most recently added item. This too is a read-only operation.

Access pattern:
Since we maintain a rear pointer, accessing the rear element is just rear→data — no traversal needed. This is a major advantage of the linked-list circular queue.
  • Returns rear→data without modifying anything
  • Must check for empty queue (rear could be NULL)
  • Extremely fast — O(1) because we track rear directly
  • Useful for monitoring what was added most recently
Access rear node value Empty queue check Read-only operation

/* Get the rear (most recently added) element */
int getRear() {

    if (rear == NULL) {

    }

    printf("Rear element is: %d\n", rear->data);
    return rear->data;   /* Read-only, no changes */

    enqueue(10);
    getRear();   /* Output: Rear element is: 10 */

    enqueue(20);
    getRear();   /* Output: Rear element is: 20 */

    enqueue(30);
    peek();      /* Output: Front element is: 10 (front unchanged) */
    getRear();   /* Output: Rear element is:  30 (rear updated)    */
    return 0;
}

"In a print queue, the last document you sent to print is at the rear. You can check which document is newest without canceling anything — that's exactly what getRear() does."

👁️

IsEmpty Operation

Core Access

📖Theory

IsEmpty is a guard function — it checks if the queue has any elements. It should be called before every dequeue or peek to prevent errors.

The Rule:
The queue is empty if and only if front == NULL. Since we set both to NULL on initialization and after the last element is removed, this single check is sufficient.
  • Returns 1 (true) if empty, 0 (false) otherwise
  • Check: front == NULL is all you need
  • Time complexity: O(1) — instant check
  • Use before dequeue, peek, getRear to avoid crashes
Check front == NULL Return boolean status Use before dequeue/peek

👁Interactive Demo

► Try adding and removing to see when isEmpty triggers

💻C Code — IsEmpty

C LANGUAGE
/* isEmpty: Check if the circular queue has no elements */
int isEmpty() {
    if (front == NULL) {
        printf("Queue is EMPTY.\n");
        return 1;       /* 1 = true = empty */
    }
    printf("Queue is NOT empty.\n");
    return 0;           /* 0 = false = has elements */
}

/* Safe dequeue using isEmpty guard */
void safeDequeue() {
    if (isEmpty()) {     /* Always check before dequeuing! */
        printf("Cannot dequeue — queue is empty!\n");
        return;
    }
    dequeue();
}

int main() {
    isEmpty();   /* Output: Queue is EMPTY (nothing added yet)    */
    enqueue(42);
    isEmpty();   /* Output: Queue is NOT empty                    */
    dequeue();   /* Removes 42                                    */
    isEmpty();   /* Output: Queue is EMPTY (back to initial state) */
    return 0;
}

🎯Real-Life Analogy

"Before a cashier calls 'Next customer please!', they look to see if anyone is standing in line. Calling isEmpty() is that exact look — it's a safety check before taking action."
🔍
Cashier checking the line = isEmpty() checking if front == NULL.
🛡️
Preventing error = calling out to nobody causes confusion — dequeuing from empty queue causes a crash.

IsFull Operation

Core Access

📖Theory

Unlike array-based queues, a linked-list circular queue has no fixed maximum size. It's "full" only when the system runs out of memory (RAM).

How to check:
Try to allocate a new node using malloc(). If it returns NULL, memory is exhausted and the queue is effectively full. If successful, free that test node immediately.
  • No fixed capacity — limited only by available system memory
  • Check by attempting a test malloc()
  • If malloc() returns NULL → system memory is full
  • This is an "optional concept" — in practice rarely needed
  • In embedded systems with limited RAM, this check matters
Memory allocation failure Dynamic memory limitation Circular queue overflow

👁Visualization

► In linked-list queue, "full" means RAM exhausted

💻C Code — IsFull (Concept)

C LANGUAGE
/* isFull: Attempt a test allocation to check memory */
int isFull() {
    /* Try to allocate one node's worth of memory */
    struct Node* testNode = (struct Node*)malloc(sizeof(struct Node));

    if (testNode == NULL) {
        /* malloc returned NULL — no memory left! */
        printf("Queue is FULL (memory exhausted).\n");
        return 1;   /* true = full */
    }

    /* Memory is available — free the test node */
    free(testNode);
    printf("Queue can still grow (memory available).\n");
    return 0;       /* false = not full */
}

/* Safe enqueue using isFull guard */
void safeEnqueue(int value) {
    if (isFull()) {
        printf("Cannot enqueue %d — out of memory!\n", value);
        return;
    }
    enqueue(value);
}

🎯Real-Life Analogy

"A linked-list queue is like a parking lot with no designated spaces — cars park wherever there's open asphalt. It's 'full' only when every square inch of pavement is occupied. An array queue is like a lot with numbered, fixed spaces."
🅿️
Open asphalt parking = linked list, grows dynamically until system RAM is exhausted.
📐
Numbered spots = array-based queue, fixed capacity, isFull is straightforward.

Display Circular Queue

Traversal

📖Theory

Display prints all elements of the queue from front to rear. The key challenge: because the list is circular, we need a stopping condition to avoid an infinite loop.

The Trick:
Use a do-while loop starting from front. Continue until the current node's next equals front again — meaning we've gone all the way around.
  • Start from front, move through next pointers
  • Stop when you return to front — one complete circle
  • Use do-while so the first node is always printed
  • Always check for empty queue before traversing
  • Time complexity: O(n)
Circular traversal Stop condition Print all nodes Empty queue display Avoid infinite loop

👁Interactive Demo

► Add elements then hit Display — watch the traversal!

💻C Code — Display

C LANGUAGE
void display() {

    /* Step 1: Check for empty queue */
    if (front == NULL) {
        printf("Queue is empty! Nothing to display.\n");
        return;
    }

    printf("Queue [front → rear]: ");

    /* Step 2: Use a temporary pointer, start at front */
    struct Node* temp = front;

    /* Step 3: do-while ensures front is visited at least once */
    do {
        printf("%d", temp->data);
        temp = temp->next;        /* Move to next node */

        if (temp != front) {
            printf(" → ");
        }
    } while (temp != front);  /* Stop when we reach front again */

    printf(" ↩ (circular)\n");
}

int main() {
    enqueue(10); enqueue(20); enqueue(30);
    display();
    /* Output: Queue [front → rear]: 10 → 20 → 30 ↩ (circular) */
    return 0;
}

🎯Real-Life Analogy

"Listing items in a circular queue is like reading every seat number on a rotating Ferris wheel. You start at the bottom (front), read each seat as it passes, and stop when you're back at the bottom — you've done exactly one full revolution."
🎡
One full revolution = the do-while (temp != front) stopping condition — exactly one loop.
⚠️
Not stopping = the Ferris wheel spinning forever — an infinite loop in code without the stop condition.

Traverse Circular Queue

Traversal

📖Theory

Traversal is more deliberate than display — you visit each node individually, typically to process or examine data at each step. Same circular logic applies.

Two Methods:
1. do-while loop — most common, clean, handles single-node case naturally.
2. Count-based — first count nodes with size(), then loop N times.
  • Use a temp pointer — never move front or rear
  • Track position with a counter variable
  • Full-cycle traversal: visit every node exactly once
  • Basis for search, count, update, and other operations
  • Time complexity: O(n)
Temporary pointer Circular traversal logic do-while method Full-cycle traversal

👁Interactive Demo

► Watch the traversal highlight each node one by one

💻C Code — Traverse

C LANGUAGE
/* Method 1: do-while traversal (recommended) */
void traverse() {
    if (front == NULL) {
        printf("Nothing to traverse.\n");
        return;
    }

    struct Node* temp = front; /* Use temp, NEVER move front! */
    int position = 0;

    do {
        /* Process node at current position */
        printf("Position %d: data = %d\n", position, temp->data);
        position++;
        temp = temp->next;
    } while (temp != front);

    printf("Traversal complete. Total nodes: %d\n", position);
}

/* Method 2: Count-based traversal */
void traverseByCount() {
    int n = size();          /* Count total nodes first */
    struct Node* temp = front;

    for (int i = 0; i < n; i++) {
        printf("Node[%d] = %d\n", i, temp->data);
        temp = temp->next;
    }
}

🎯Real-Life Analogy

"Traversing a circular queue is like a teacher calling roll on students seated in a circle. She starts with the first student, moves clockwise, and stops when she's back at the first student — everyone was visited exactly once."
📋
Roll call = visiting each node to read or process its data without modifying the list.
🔄
Stopping at start = the do-while (temp != front) condition — one clean full pass.

Queue Size Counting

Traversal

📖Theory

Unlike arrays, there's no built-in .length in a linked list. We count by traversing the entire circle and incrementing a counter for each node.

Algorithm:
Start count = 0 at front. Use do-while: increment count, advance temp, stop when back at front. Return count.
  • Returns 0 for an empty queue (special case)
  • Traverses every node exactly once
  • Time complexity: O(n) — must visit all nodes
  • If size is needed frequently, consider tracking it as a variable
Count circularly Full-circle termination Dynamic size calculation O(n) complexity
TimeO(n)
OptimizationTrack size variable

👁Interactive Demo

► Watch the counter increment as each node is visited

💻C Code — Size

C LANGUAGE
/* size: Count all nodes in the circular queue */
int size() {

    /* Empty queue has zero nodes */
    if (front == NULL) {
        return 0;
    }

    int count = 0;
    struct Node* temp = front;

    do {
        count++;              /* Count this node */
        temp = temp->next;    /* Move to next    */
    } while (temp != front); /* Until full circle */

    printf("Queue size: %d\n", count);
    return count;
}

/* Optimized: Track size as a global variable */
int queueSize = 0;  /* Increment on enqueue, decrement on dequeue */

int main() {
    enqueue(10);  enqueue(20);  enqueue(30);
    size();        /* Output: Queue size: 3 */
    dequeue();
    size();        /* Output: Queue size: 2 */
    return 0;
}

🎯Real-Life Analogy

"Counting the size is like counting guests seated at a circular banquet table. You pick any seat as your start, count each person as you go around, and stop when you're back at the beginning — that's your headcount."
🍽️
Counting guests at a round table = traversing the circular linked list and incrementing a counter.
📝
Keeping a separate guest list = the optimization: track queueSize as a variable for O(1) access.

Search Element

Utility

📖Theory

Search traverses the queue looking for a specific value. It's a linear search — it checks each node one by one until found or the full circle is completed.

Strategy:
Start at front, compare each node's data to the key. Track the position. Return the position if found, or -1 if not found after a full traversal.
  • Circular linear search — checks every node at most once
  • Returns the 0-based position of the first match
  • Returns -1 if the key is not found
  • No binary search possible — no sorting guaranteed
  • Time complexity: O(n)
Circular linear search Compare node data Return found/not found Position tracking

► Add elements, enter a value, then search

💻C Code — Search

C LANGUAGE
/* search: Find the position of a key in the circular queue */
int search(int key) {

    if (front == NULL) {
        printf("Queue is empty!\n");
        return -1;
    }

    struct Node* temp = front;
    int position = 0;

    do {
        if (temp->data == key) {       /* Found the key! */
            printf("Found %d at position %d\n", key, position);
            return position;
        }
        position++;
        temp = temp->next;
    } while (temp != front);

    /* If we reach here, key was not found */
    printf("%d not found in queue.\n", key);
    return -1;
}

int main() {
    enqueue(10); enqueue(25); enqueue(30); enqueue(15);

    search(25);   /* Output: Found 25 at position 1 */
    search(99);   /* Output: 99 not found in queue.  */
    return 0;
}

🎯Real-Life Analogy

"Searching a circular queue is like looking for a specific person in a circular seating arrangement. You start at one chair, check each person's name badge, and work your way around. If you reach your starting chair without finding them — they're not there."
🏷️
Checking name badges = comparing temp→data == key at each node.
❌
Back at starting chair = the full traversal complete — return -1 (not found).

Update Node Value

Utility

📖Theory

Update modifies the data of an existing node — either by searching for a specific old value, or by targeting a specific position (index).

Two Approaches:
By value: Search for the node with oldVal, then change temp→data = newVal.
By position: Traverse to the target index, then update the data.
  • No structural changes — only the data field changes
  • Pointers remain entirely untouched
  • Works on a copy of the pointer (temp), not front/rear
  • Time complexity: O(n) for searching the node
Search target node Replace old data Update by position Update by key value

💻C Code — Update

C LANGUAGE
/* Update by value: Replace oldVal with newVal */
int updateByValue(int oldVal, int newVal) {
    if (front == NULL) { printf("Queue is empty!\n"); return 0; }

    struct Node* temp = front;
    do {
        if (temp->data == oldVal) {
            temp->data = newVal;     /* Only data changes! */
            printf("Updated %d → %d\n", oldVal, newVal);
            return 1;              /* Success */
        }
        temp = temp->next;
    } while (temp != front);

    printf("%d not found!\n", oldVal);
    return 0;   /* Not found */
}

/* Update by position (0-indexed) */
int updateByPosition(int pos, int newVal) {
    if (front == NULL) return 0;

    struct Node* temp = front;
    int n = size();
    if (pos < 0 || pos >= n) { printf("Invalid position!\n"); return 0; }

    for (int i = 0; i < pos; i++) temp = temp->next;
    printf("Position %d: %d → %d\n", pos, temp->data, newVal);
    temp->data = newVal;
    return 1;
}

int main() {
    enqueue(10); enqueue(20); enqueue(30);
    updateByValue(20, 99);   /* 20 → 99 */
    updateByPosition(0, 55); /* front (10) → 55 */
    display();  /* Queue: 55 → 99 → 30 */
    return 0;
}

🎯Real-Life Analogy

"Updating a node is like correcting a typo on a boarding pass without issuing a new ticket. The passenger's seat in the queue remains the same — only the printed name is changed."
✏️
Correcting the name = temp→data = newVal — only the data field changes, structure intact.
🔍
Finding the right ticket = linear search through nodes to find the one with the old value.

Clear / Destroy Circular Queue

Utility

📖Theory

Clear / Destroy removes ALL nodes and frees all allocated memory. This is critical to prevent memory leaks in C programs.

The Challenge:
The circular link means the last node points BACK to front. We must break this circular link first (set rear→next = NULL), then traverse and free each node in sequence.
  • Break circular link first: rear→next = NULL
  • Traverse linearly and free each node
  • Reset both front and rear to NULL when done
  • Critical for memory management — always destroy before program ends
  • Time complexity: O(n)
Delete all nodes Break circular links Free dynamic memory Reset front and rear Prevent memory leak

👁Interactive Demo

► Add elements then watch them all get freed

💻C Code — Clear Queue

C LANGUAGE
void clearQueue() {

    if (front == NULL) {
        printf("Queue is already empty.\n");
        return;
    }

    /* Step 1: CRITICAL — break the circular link */
    /* Without this, the while loop below would run forever! */
    rear->next = NULL;    /* Now it's a regular linear list */

    /* Step 2: Traverse and free every node */
    struct Node* temp = front;
    int count = 0;

    while (temp != NULL) {
        struct Node* next = temp->next; /* Save next before freeing */
        printf("Freeing node with data: %d\n", temp->data);
        free(temp);                      /* Release memory */
        temp = next;
        count++;
    }

    /* Step 3: Reset pointers to NULL (empty state) */
    front = NULL;
    rear  = NULL;
    printf("Queue cleared! %d nodes freed.\n", count);
}

int main() {
    enqueue(10); enqueue(20); enqueue(30);
    clearQueue();  /* Frees all 3 nodes, resets front and rear */
    isEmpty();     /* Output: Queue is EMPTY */
    return 0;
}

🎯Real-Life Analogy

"Clearing a circular queue is like dismantling a merry-go-round — you first disconnect the circular track (break the loop), then remove each seat one by one and return them to storage. Finally, the ride area is completely empty and ready for a new setup."
🔗
Disconnecting the circular track = rear→next = NULL — MUST be done first.
🗑️
Returning seats to storage = calling free(temp) on each node — prevents memory leaks.

Reverse Circular Queue

Advanced

📖Theory

Reversing a circular queue means flipping all the next pointer directions — so what was the rear becomes the front, and vice versa.

Key Steps:
1. Use three pointers: prev, curr, next
2. Traverse the full circle, reversing each link
3. After reversal: swap front and rear
4. Restore circular link: rear→next = front
  • Same algorithm as reversing a singly linked list
  • No new nodes created — only pointers are changed
  • Old front becomes new rear, old rear becomes new front
  • Time complexity: O(n)
Reverse node links Maintain circular structure Update front and rear Pointer manipulation Recursive reversal

💻C Code — Reverse

C LANGUAGE
void reverseQueue() {

    /* Nothing to reverse for 0 or 1 nodes */
    if (front == NULL || front == rear) return;

    struct Node* prev     = rear;    /* Start prev at rear */
    struct Node* curr     = front;   /* Start curr at front */
    struct Node* nextNode = NULL;
    struct Node* oldFront = front;   /* Remember original front */

    /* Traverse the full circle and reverse each link */
    do {
        nextNode   = curr->next; /* Save next before overwriting */
        curr->next = prev;       /* Reverse the link            */
        prev       = curr;       /* Advance prev                */
        curr       = nextNode;   /* Advance curr                */
    } while (curr != oldFront);

    /* Update front and rear to the reversed positions */
    front      = rear;       /* Old rear is now front */
    rear       = oldFront;   /* Old front is now rear */
    rear->next = front;      /* Restore circular link  */

    printf("Queue reversed!\n");
}

int main() {
    enqueue(1); enqueue(2); enqueue(3); enqueue(4);
    display();        /* 1 → 2 → 3 → 4 */
    reverseQueue();
    display();        /* 4 → 3 → 2 → 1 */
    return 0;
}

🎯Real-Life Analogy

"Reversing a circular queue is like reversing the direction of a one-way traffic roundabout. Every car's direction arrow gets flipped — what was 'next' is now 'previous' — and the entry/exit points swap roles."
🔄
Flipping car direction arrows = reversing each node's next pointer with three-pointer technique.
🚗
Entry becomes exit = what was front becomes rear and vice versa after reversal.

Concatenate / Merge Circular Queues

Advanced

📖Theory

Merging joins two circular queues into one. Queue 2 is appended at the rear of Queue 1, forming a single unified circular queue.

Key Steps:
1. Handle empty queue edge cases
2. Connect rear1→next = front2 (link Q1's end to Q2's start)
3. Connect rear2→next = front1 (close the new circle)
4. Update rear1 = rear2 (Q1's rear is now Q2's rear)
  • Only pointer manipulation — no node creation or copying
  • O(1) operation — just reconnect 2 pointers
  • Handle cases: Q1 empty, Q2 empty, both empty
  • After merge, Q2 is absorbed into Q1 — Q2 becomes invalid
Join two queues Connect rear nodes Maintain circular property Update rear pointer Handle empty queues
TimeO(1)
SpaceO(1)

/* Merge queue2 into queue1 (queue2 becomes invalid after this) */
void merge(struct Node** front1, struct Node** rear1,
           struct Node*  front2, struct Node*  rear2) {

    /* Edge case: Q2 is empty — nothing to merge */
    if (front2 == NULL) {
        printf("Queue 2 is empty, nothing to merge.\n");
        return;
    }

    /* Edge case: Q1 is empty — Q1 simply becomes Q2 */
    if (*front1 == NULL) {
        *front1 = front2;
        *rear1  = rear2;
        return;
    }

    /* General case: Both queues have elements */
    /* Step 1: Connect rear of Q1 to front of Q2 */
    (*rear1)->next = front2;

    /* Step 2: Connect rear of Q2 back to front of Q1 (new circle!) */
    rear2->next = *front1;

    /* Step 3: Update Q1's rear to Q2's rear */
    *rear1 = rear2;

    printf("Queues merged successfully!\n");
}

int main() {
    /* Queue 1: 10 → 20 → 30 */
    enqueue(10); enqueue(20); enqueue(30);

    /* Queue 2: 40 → 50 */
    struct Node* f2 = NULL, *r2 = NULL;
    /* (enqueue into f2, r2 separately) */

    merge(&front, &rear, f2, r2);
    /* Result: 10 → 20 → 30 → 40 → 50 (circular) */
    display();
    return 0;
}

🔗
Connecting the roads = rear1→next = front2 and rear2→next = front1 — just two pointer changes.
⚡

Split Circular Queue

Advanced

📖Theory

Split divides one circular queue into two halves. The first half contains the front elements, the second half contains the rear elements.

Algorithm:
1. Count total nodes with size()
2. Find the midpoint node (traverse n/2 - 1 steps)
3. The midpoint node becomes rear1
4. The midpoint's next becomes front2
5. Fix both circular links: rear1→next = front1 and rear2→next = front2
  • Handles both even and odd-length queues
  • For odd length: first half gets extra element
  • Adjust circular links carefully — easy to create dangling pointers
  • Time complexity: O(n) for traversal
Divide into two parts Find midpoint node Adjust circular links First half queue Second half queue Maintain circular structure

💻C Code — Split

void splitQueue(struct Node** front1, struct Node** rear1,
               struct Node** front2, struct Node** rear2) {

    if (front == NULL || front == rear) {
        printf("Need at least 2 nodes to split!\n");
        return;
    }

    int n = size();              /* Total number of nodes */
    int mid = n / 2;            /* Midpoint index         */

    /* Step 1: Find the midpoint node (mid-1 steps from front) */
    struct Node* midNode = front;
    for (int i = 0; i < mid - 1; i++) {
        midNode = midNode->next;
    }

    /* Step 2: Set up Q1 (front...midNode) */
    *front1  = front;           /* Q1 starts at original front */
    *rear1   = midNode;         /* Q1 ends at midpoint         */

    /* Step 3: Set up Q2 (midNode->next...rear) */
    *front2  = midNode->next;   /* Q2 starts right after mid   */
    *rear2   = rear;            /* Q2 ends at original rear    */

    /* Step 4: Restore circular links for both queues */
    (*rear1)->next = *front1;   /* Q1 circular: rear1 → front1 */
    (*rear2)->next = *front2;   /* Q2 circular: rear2 → front2 */

    /* Original front/rear are now invalid — use front1/rear1 and front2/rear2 */
    printf("Split complete! Q1 has %d nodes, Q2 has %d nodes.\n",
           mid, n - mid);
}

int main() {
    enqueue(10); enqueue(20); enqueue(30); enqueue(40);
    /* 10 → 20 → 30 → 40 → (back to 10) */

    struct Node *f1, *r1, *f2, *r2;
    splitQueue(&f1, &r1, &f2, &r2);
    /* Q1: 10 → 20 →(10)   Q2: 30 → 40 →(30) */
    return 0;
}

"Splitting a circular queue is like dividing one large round table into two smaller round tables. You find the middle seat, cut the table there, and curve each half into its own complete circle — now you have two separate circular arrangements."
✂️
Cutting the table = breaking the link at the midpoint and at the rear-to-front junction.
🔵🔵
Two new round tables = two independent circular queues, each with proper rear→next = front links.

Copy Circular Queue

Advanced

📖Theory

Copy creates a completely independent duplicate (deep copy) of a circular queue. The copy must preserve the same order and maintain its own circular structure.

Deep Copy vs Shallow Copy:
A shallow copy just copies pointers — both queues share the same nodes (dangerous!). A deep copy allocates NEW nodes for each element, so modifications to one don't affect the other.
  • Traverse original queue with do-while, enqueue each value into the copy
  • The copy gets its own front and rear pointers
  • All nodes are newly allocated — completely independent
  • Same order is preserved: front element is still front
  • Time complexity: O(n)
Duplicate queue creation Preserve circular structure Deep copy concept Copy node-by-node Maintain same order Update front and rear pointers

/* copyQueue: Create a deep copy of the circular queue */
void copyQueue(struct Node** newFront, struct Node** newRear) {

    *newFront = NULL;
    *newRear  = NULL;

    if (front == NULL) {
        printf("Source queue is empty!\n");
        return;
    }

    struct Node* temp = front;

    do {
        /* Step 1: Create a brand new node for each element */
        struct Node* newNode = (struct Node*)malloc(sizeof(struct Node));
        newNode->data = temp->data;  /* Copy the data value */
        newNode->next = NULL;

        /* Step 2: Enqueue into the new copy */
        if (*newFront == NULL) {     /* First node of copy */
            *newFront      = newNode;
            *newRear       = newNode;
            (*newRear)->next = *newFront;  /* Self-circular */
        } else {                     /* Subsequent nodes */
            (*newRear)->next = newNode;
            *newRear         = newNode;
            (*newRear)->next = *newFront;  /* Maintain circular link */
        }

        temp = temp->next;
    } while (temp != front);

    printf("Deep copy created with %d nodes.\n", size());
}

int main() {
    enqueue(10); enqueue(20); enqueue(30);

    struct Node *copyFront = NULL, *copyRear = NULL;
    copyQueue(&copyFront, &copyRear);

    /* Modify original — copy is NOT affected! */
    dequeue();           /* Original: 20 → 30       */
                          /* Copy still: 10 → 20 → 30 */
    return 0;
}

🎯Real-Life Analogy

"A deep copy is like photocopying a document — you get a completely new, independent copy. Scribbling on the photocopy doesn't change the original, and tearing the original doesn't affect your copy. Both are fully independent."
📄
Photocopy = deep copy — new malloc() calls create entirely new nodes in memory.
⚠️
Sharing the same paper = shallow copy (just copying pointers) — dangerous, changes affect both.
✅
Independence = modifying or freeing the original queue has zero effect on the deep copy.
