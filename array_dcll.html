Basic Operations

The foundation — understanding what an array-based doubly circular linked list is, how to set it up, and how to check its state.
🏗️

1.1 Create List

Theory
Visualize
C Code
Analogy

An Array-Based Doubly Circular Linked List (DCLL) stores nodes in a fixed-size array. Each node has three parts:

  • PREV — index of the previous node
  • DATA — the actual value stored
  • NEXT — index of the next node

What makes it special?

  • Doubly: You can move forward AND backward between nodes
  • Circular: The last node's NEXT points back to the first node; first node's PREV points to the last
  • Array-Based: All nodes live in a single fixed array — no dynamic memory (malloc)
💡 The list uses a head variable (index) to track where the list starts, and a count variable to track how many elements are in it.
Structure of the Array (MAX=5)
Click "Create List" to initialize the structure
create_list.c
#include <stdio.h>

#define MAX 10   /* Maximum nodes the list can hold */

/* Node structure stored in array */
struct Node {
    int prev;   /* index of previous node */
    int data;   /* actual data value */
    int next;   /* index of next node */
};

/* Global array and control variables */
struct Node list[MAX];
int head = -1;   /* -1 means empty */
int count = 0;  /* number of elements */

/* Create: just declare the array above — nothing else needed! */
int main() {
    printf("Array-based DCLL created.\n");
    printf("Capacity: %d nodes\n", MAX);
    printf("Head = %d (empty)\n", head);
    return 0;
}
🚂 Real-Life Analogy: Think of a train with a fixed number of seats (the array). The train is circular — the last car connects back to the first. Each seat has a tag pointing to the seat in front and behind. "Creating the list" is like building the empty train before any passengers board.
🔧

1.2 Initialize List

Theory
Visualize
C Code
Analogy

Initialization sets all node fields to a sentinel value (usually -1 for pointers and 0 for data) so we know which slots are free.

Steps:

  • Loop through all MAX positions in the array
  • Set each node's prev = -1, data = 0, next = -1
  • Set head = -1 and count = 0
💡 -1 is used as a NULL equivalent since we can't use pointers in an array-based list.
Before vs After Initialization
Before: random/garbage values → After: all slots = [-1 | 0 | -1]
init_list.c
void initList() {
    int i;
    for (i = 0; i < MAX; i++) {
        list[i].prev = -1;  /* no prev node */
        list[i].data = 0;    /* empty data */
        list[i].next = -1;  /* no next node */
    }
    head  = -1;  /* list is empty */
    count = 0;   /* zero elements */
    printf("List initialized. All slots cleared.\n");
}
🧹 Real-Life Analogy: Initializing is like cleaning a whiteboard before writing. Each slot on the whiteboard is wiped clean so you can clearly see what's empty and what has data later.

Insertion Operations

Adding nodes to the list at various positions while maintaining the doubly-linked and circular structure.
⬅️

2.1 Insert at Front

Theory
Visualize
C Code
Analogy

Inserting at the front means the new node becomes the new head. Steps:

  • Find a free slot in the array
  • Store data in that slot
  • Set new node's NEXT = old head
  • Set old head's PREV = new node index
  • Find the tail (head's PREV before update) and connect it to new node
  • Set new node's PREV = tail index
  • Update head = new node index
💡 For the very first node, it points to itself: PREV = NEXT = its own index.
Insert at Front
List is empty — insert first!
insert_front.c
void insertFront(int val) {
    int i, newNode = -1, tail;

    if (isFull()) { printf("List full!\n"); return; }

    /* Step 1: Find a free slot */
    for (i = 0; i < MAX; i++) {
        if (list[i].data == 0 && list[i].next == -1) {
            newNode = i; break;
        }
    }

    list[newNode].data = val;

    if (head == -1) {         /* Case 1: Empty list */
        list[newNode].prev = newNode; /* points to itself */
        list[newNode].next = newNode;
        head = newNode;
    } else {                  /* Case 2: Non-empty */
        tail = list[head].prev;
        list[newNode].next = head;
        list[newNode].prev = tail;
        list[head].prev    = newNode;
        list[tail].next    = newNode;
        head = newNode;
    }
    count++;
    printf("Inserted %d at front.\n", val);
}
🎪 Real-Life Analogy: Like a person joining a circular conga line at the front. They grab the shoulder of the current front person and the last person grabs their shoulder — the circle is maintained!
➡️

2.2 Insert at Rear

Theory
Visualize
C Code
Analogy

Inserting at the rear adds the new node just before the head (since it's circular, rear → head). The key difference from front insertion: head does NOT change.

  • Find free slot, store data
  • tail = list[head].prev (current last node)
  • newNode.prev = tail, newNode.next = head
  • tail.next = newNode, head.prev = newNode
  • head stays the same!
Insert at Rear
List is empty — insert first!
insert_rear.c
void insertRear(int val) {
    int i, newNode = -1, tail;

    if (isFull()) { printf("List full!\n"); return; }

    for (i = 0; i < MAX; i++) {
        if (list[i].data == 0 && list[i].next == -1) {
            newNode = i; break;
        }
    }
    list[newNode].data = val;

    if (head == -1) {
        list[newNode].prev = newNode;
        list[newNode].next = newNode;
        head = newNode;
    } else {
        tail = list[head].prev;  /* find tail */
        list[newNode].next = head;
        list[newNode].prev = tail;
        list[tail].next   = newNode;
        list[head].prev   = newNode;
        /* head does NOT change */
    }
    count++;
    printf("Inserted %d at rear.\n", val);
}
🎢 Real-Life Analogy: Like joining a circular queue at the back. The last person in line holds your hand, and you hold the hand of the first person — the circle never breaks!
↩️

2.3 Insert Before Position

Theory
C Code
Analogy

Insert a new node before the node at a given position (1-based). Traverse to position, then link the new node before it.

  • If pos==1 → same as Insert at Front
  • Otherwise traverse to node at (pos-1)
  • newNode.next = current; newNode.prev = current.prev
  • current.prev.next = newNode; current.prev = newNode
insert_before.c
void insertBefore(int pos, int val) {
    int i, newNode = -1, curr, prevNode, step;

    if (isFull() || isEmpty()) { printf("Error!\n"); return; }
    if (pos == 1) { insertFront(val); return; }
    if (pos < 1 || pos > count) { printf("Invalid pos!\n"); return; }

    for (i = 0; i < MAX; i++)
        if (list[i].data == 0 && list[i].next == -1) { newNode = i; break; }

    list[newNode].data = val;

    /* Traverse to the node at 'pos' */
    curr = head;
    for (step = 1; step < pos; step++)
        curr = list[curr].next;

    prevNode = list[curr].prev;
    list[newNode].next  = curr;
    list[newNode].prev  = prevNode;
    list[prevNode].next = newNode;
    list[curr].prev     = newNode;
    count++;
    printf("Inserted %d before position %d.\n", val, pos);
}
🎬 Real-Life Analogy: Like cutting in line before position 3. You tap person 3 on the shoulder, stand behind person 2 and in front of person 3, and everyone adjusts who they're holding hands with.
↪️

2.4 Insert After Position

Theory
C Code
Analogy

Insert a new node after the node at a given position. Similar to before, but links differently.

  • Traverse to node at position pos
  • newNode.prev = curr; newNode.next = curr.next
  • curr.next.prev = newNode; curr.next = newNode
  • If inserted after last node → same as Insert at Rear
insert_after.c
void insertAfter(int pos, int val) {
    int i, newNode = -1, curr, nextNode, step;

    if (isFull() || isEmpty()) { printf("Error!\n"); return; }
    if (pos < 1 || pos > count) { printf("Invalid pos!\n"); return; }

    for (i = 0; i < MAX; i++)
        if (list[i].data == 0 && list[i].next == -1) { newNode = i; break; }

    list[newNode].data = val;
    curr = head;
    for (step = 1; step < pos; step++)
        curr = list[curr].next;

    nextNode = list[curr].next;
    list[newNode].prev  = curr;
    list[newNode].next  = nextNode;
    list[curr].next     = newNode;
    list[nextNode].prev = newNode;
    count++;
    printf("Inserted %d after position %d.\n", val, pos);
}
🚶 Real-Life Analogy: Like inserting yourself into a circular dance line after position 2. You stand right after person 2, take their right hand, and offer your left hand to person 3. The circle is complete again.

Deletion Operations

Removing nodes from the list while properly relinking the remaining nodes to preserve the doubly-circular structure.
🗑️

3.1 Delete Front

Theory
Visualize
C Code

Remove the node at head. Steps:

  • If only 1 node: set head=-1
  • Else: tail = list[head].prev
  • new head = list[head].next
  • list[newHead].prev = tail; list[tail].next = newHead
  • Clear old head slot; head = newHead
Delete Front
Add nodes first
del_front.c
void deleteFront() {
    int tail, newHead;
    if (isEmpty()) { printf("Empty!\n"); return; }

    printf("Deleted: %d\n", list[head].data);

    if (count == 1) {
        list[head].data = 0;
        list[head].prev = list[head].next = -1;
        head = -1;
    } else {
        tail    = list[head].prev;
        newHead = list[head].next;
        list[newHead].prev = tail;
        list[tail].next    = newHead;
        list[head].data = 0;
        list[head].prev = list[head].next = -1;
        head = newHead;
    }
    count--;
}
🗑️

3.2 Delete Rear

Theory
Visualize
C Code

Remove the tail node (the node just before head).

  • tail = list[head].prev
  • newTail = list[tail].prev
  • list[newTail].next = head; list[head].prev = newTail
  • Clear tail slot; count--
Delete Rear
Add nodes first
del_rear.c
void deleteRear() {

    if (isEmpty()) { printf("Empty!\n"); return; }

    tail = list[head].prev;
    printf("Deleted: %d\n", list[tail].data);

    if (count == 1) {
        list[head].data = 0;
        list[head].prev = list[head].next = -1;
        head = -1;
    } else {
        newTail = list[tail].prev;
        list[newTail].next = head;
        list[head].prev    = newTail;
        list[tail].data = 0;
        list[tail].prev = list[tail].next = -1;
    }
    count--;
}
🎯

3.3 Delete by Value

Theory
C Code
Analogy

Search for a node containing a specific value and remove it. Since it's circular, traverse with a do-while loop starting from head.

  • Traverse until we find the value or wrap back to head
  • If found at head → call deleteFront()
  • Else: prevNode.next = curr.next; curr.next.prev = prevNode
  • Clear the slot
del_value.c
void deleteByValue(int val) {
    int curr, prevNode, nextNode;
    if (isEmpty()) { printf("Empty!\n"); return; }

    curr = head;
    do {
        if (list[curr].data == val) {
            if (curr == head) { deleteFront(); return; }
            prevNode = list[curr].prev;
            nextNode = list[curr].next;
            list[prevNode].next = nextNode;
            list[nextNode].prev = prevNode;
            list[curr].data = 0;
            list[curr].prev = list[curr].next = -1;
            count--;
            printf("Deleted value: %d\n", val);
            return;
        }
        curr = list[curr].next;
    } while (curr != head);

    printf("Value %d not found.\n", val);
}
🔍 Real-Life Analogy: Like finding a specific person by name in a circular conga line, removing them, and having the people on either side join hands with each other to keep the line intact.
📍

3.4 Delete Specific Position

Theory
C Code
Analogy

Remove the node at a specific 1-based position. Traverse to that position, then unlink and clear it.

  • pos==1 → deleteFront(); pos==count → deleteRear()
  • Else traverse to that position and unlink
del_pos.c
void deletePosition(int pos) {
    int curr, prevNode, nextNode, step;
    if (isEmpty()) { printf("Empty!\n"); return; }
    if (pos < 1 || pos > count) { printf("Invalid!\n"); return; }
    if (pos == 1) { deleteFront(); return; }
    if (pos == count) { deleteRear(); return; }

    curr = head;
    for (step = 1; step < pos; step++)
        curr = list[curr].next;

    prevNode = list[curr].prev;
    nextNode = list[curr].next;
    list[prevNode].next = nextNode;
    list[nextNode].prev = prevNode;
    printf("Deleted pos %d: %d\n", pos, list[curr].data);
    list[curr].data = 0;
    list[curr].prev = list[curr].next = -1;
    count--;
}
🎭 Real-Life Analogy: Like removing seat number 3 from a circular arrangement. The person at seat 2 and seat 4 now sit next to each other, filling the gap.

Traversal Operations

Visiting every node in the list in different directions. Traversal is the backbone of most data structure operations.
🎮

4.1 Forward Traversal

Theory
C Code

Start at head, follow NEXT pointers. Stop when we reach head again (circular). Use a do-while loop since we visit head first.

Order: HEAD → node2 → node3 → ... → tail → (back to HEAD)
forward_trav.c
void forwardTraversal() {
    int curr;
    if (isEmpty()) { printf("Empty!\n"); return; }
    curr = head;
    printf("Forward: ");
    do {
        printf("%d ", list[curr].data);
        curr = list[curr].next;
    } while (curr != head);
    printf("\n");
}
◀️

4.2 Backward Traversal

Theory
C Code

Start at tail (list[head].prev), follow PREV pointers back to head. This is only possible because of the doubly linkage.

Order: TAIL → ... → node2 → HEAD
backward_trav.c
void backwardTraversal() {
    int curr;
    if (isEmpty()) { printf("Empty!\n"); return; }
    curr = list[head].prev; /* start at tail */
    printf("Backward: ");
    do {
        printf("%d ", list[curr].data);
        curr = list[curr].prev;
    } while (curr != list[head].prev);
    printf("\n");
}
↻

4.3 Circular Traversal

Theory
C Code
Analogy

Continue traversing PAST the head back around — traverse the list for exactly N full rounds. Useful for round-robin scheduling, carousel display, etc.

💡 Use a loop counter: stop after (count × rounds) steps, OR after a time limit.
circular_trav.c
void circularTraversal(int rounds) {
    int curr, steps, total;
    if (isEmpty()) { printf("Empty!\n"); return; }
    total = count * rounds;
    curr = head;
    printf("Circular (%d rounds): ", rounds);
    for (steps = 0; steps < total; steps++) {
        printf("%d ", list[curr].data);
        curr = list[curr].next; /* wraps around */
    }
    printf("\n");
}
🎠 Real-Life Analogy: Like a merry-go-round — it keeps going in circles. Circular traversal lets you go around the list as many times as needed, which is perfect for scheduling tasks in a loop (OS round-robin CPU scheduling).

Searching Operations

Finding elements in the list using different search strategies.
🔍

5.1 Circular Search

Theory
Visualize
C Code

Start at head, traverse using NEXT pointers in a do-while loop. Returns the position (1-based) if found, -1 if not. Searches the entire circular list once.

Search Demo
Add nodes first
circular_search.c
int circularSearch(int val) {
    int curr, pos = 1;
    if (isEmpty()) return -1;
    curr = head;
    do {
        if (list[curr].data == val) {
            printf("Found %d at position %d\n", val, pos);
            return pos;
        }
        curr = list[curr].next;
        pos++;
    } while (curr != head);
    printf("%d not found.\n", val);
    return -1;
}
➡️🔍

5.2 Forward Search

Theory
C Code

Search from head toward tail using NEXT pointers. Returns position on first match. Identical to circular search but stops at tail.

fwd_search.c
int forwardSearch(int val) {
    int curr = head, pos = 1;
    if (isEmpty()) return -1;
    do {
        if (list[curr].data == val)
            return pos;
        curr = list[curr].next;
        pos++;
    } while (curr != head);
    return -1;  /* not found */
}
⬅️🔍

5.3 Backward Search

Theory
C Code

Search from tail toward head using PREV pointers. Useful when the target is more likely near the end of the list.

bwd_search.c
int backwardSearch(int val) {
    int curr, pos;
    if (isEmpty()) return -1;
    curr = list[head].prev; /* start at tail */
    pos  = count;
    do {
        if (list[curr].data == val) {
            printf("Found %d at pos %d\n", val, pos);
            return pos;
        }
        curr = list[curr].prev;
        pos--;
    } while (curr != list[head].prev);
    return -1;
}

Update Operations

Modifying the data within existing nodes without changing the structure of the list.
✏️

6.1 Replace Value

Theory
C Code

Find a node by its old value and replace it with a new value. Structure stays the same — only the data field changes.

replace.c
void replaceValue(int oldVal, int newVal) {
    int curr = head;
    if (isEmpty()) return;
    do {
        if (list[curr].data == oldVal) {
            list[curr].data = newVal;
            printf("Replaced %d with %d\n",
                   oldVal, newVal);
            return;
        }
        curr = list[curr].next;
    } while (curr != head);
    printf("%d not found.\n", oldVal);
}
🔄

6.2 Modify Data (by Position)

Theory
C Code

Update the data at a specific position directly without searching by value. Traverse to position, then overwrite data field.

modify.c
void modifyData(int pos, int newVal) {
    int curr = head, step;
    if (isEmpty() || pos < 1 || pos > count) {
        printf("Invalid!\n"); return;
    }
    for (step = 1; step < pos; step++)
        curr = list[curr].next;
    printf("Changed pos %d: %d → %d\n",
           pos, list[curr].data, newVal);
    list[curr].data = newVal;
}
🔀

6.3 Swap Nodes

Theory
C Code
Analogy

Swap the data of two nodes at given positions. The simplest approach: traverse to both positions, swap their .data fields. No need to change PREV/NEXT links!

💡 Swapping data is much simpler than swapping node positions in an array-based implementation.
swap.c
void swapNodes(int pos1, int pos2) {
    int n1 = head, n2 = head, step, temp;
    if (pos1 == pos2) return;
    if (pos1<1 || pos1>count || pos2<1 || pos2>count) {
        printf("Invalid position!\n"); return;
    }
    for (step=1; step<pos1; step++) n1=list[n1].next;
    for (step=1; step<pos2; step++) n2=list[n2].next;
    temp = list[n1].data;
    list[n1].data = list[n2].data;
    list[n2].data = temp;
    printf("Swapped pos %d and pos %d\n", pos1, pos2);
}
🎴 Real-Life Analogy: Like two students swapping name tags without moving their chairs. The seats (nodes) stay in place; only the names (data) change hands.

Access Operations

Reading data from specific positions without modifying the list structure.
👀

7.1 Peek Front

Theory
C Code

Return the data at the front (head) without removing it. Simply access list[head].data.

Always check isEmpty() before peeking!
peek.c
int peekFront() {
    if (isEmpty()) {
        printf("List is empty!\n");
        return -1;
    }
    printf("Front: %d\n", list[head].data);
    return list[head].data;
}

int peekRear() {
    int tail;
    if (isEmpty()) {
        printf("List is empty!\n");
        return -1;
    }
    tail = list[head].prev;
    printf("Rear: %d\n", list[tail].data);

📌

7.3 Get Element by Position

Theory
C Code

Traverse from head (count-1) times to reach position pos and return its data. Time complexity: O(n).

get_pos.c
int getElement(int pos) {
    int curr = head, step;
    if (isEmpty() || pos<1 || pos>count) {
        printf("Invalid!\n"); return -1;
    }
    for (step = 1; step < pos; step++)
        curr = list[curr].next;
    printf("Element at pos %d = %d\n",
           pos, list[curr].data);
    return list[curr].data;
}

Structural Operations

Operations that examine the structure and properties of the list itself.
🔢

8.1 Count Elements

Theory
C Code

Since we maintain a count variable, this is O(1). Alternatively, traverse and count manually — useful for validation.

count.c
/* O(1) - using maintained counter */
int countElements() {
    return count;
}

/* O(n) - manual count by traversal */
int countManual() {
    int curr, c = 0;
    if (isEmpty()) return 0;
    curr = head;
    do {
        c++;
        curr = list[curr].next;
    } while (curr != head);
    return c;
}
📏

8.2 Find Size / 8.3 Detect Circular Wrap

Theory
C Code

Find Size: Returns count (same as countElements).
Detect Circular Wrap: Verifies the circular property: does list[tail].next == head? And does list[head].prev == tail?

Use detectCircularWrap() as a sanity check after complex operations to ensure the circular structure is intact.
structural.c
int findSize() {
    return count;  /* same as countElements */
}

int detectCircularWrap() {
    int tail;
    if (isEmpty()) {
        printf("Empty — no wrap to detect.\n");
        return 1;
    }
    tail = list[head].prev;
    if (list[tail].next == head &&
        list[head].prev == tail) {
        printf("Circular wrap: INTACT ✓\n");
        return 1;
    }
    printf("Circular wrap: BROKEN ✗\n");
    return 0;
}

Link Management Operations

Managing the explicit circular connections between the head and tail of the list.
🔗

9.1 Connect Tail to Head

Theory
C Code

Ensures tail.next = head. Used to restore the circular forward link if it was broken during operations.

link_mgmt.c
/* Connect tail's NEXT back to head */
void connectTailToHead() {
    int tail;
    if (isEmpty()) return;
    tail = list[head].prev;
    list[tail].next = head;
    printf("Tail[%d].next → Head[%d]\n",
           tail, head);
}

/* Connect head's PREV back to tail */
void connectHeadToTail() {
    int tail;
    if (isEmpty()) return;
    tail = list[head].prev;
    list[head].prev = tail;
    printf("Head[%d].prev → Tail[%d]\n",
           head, tail);
}

/* Restore both circular links */
void restoreCircularLink() {
    connectTailToHead();
    connectHeadToTail();
    printf("Circular links restored.\n");
}

/* Move current pointer forward (wraps around) */
int circularIncrement(int curr) {
    int next = list[curr].next;
    if (next == head)
        printf("Wrap-around: tail → head\n");
    return next;

/* Move current pointer backward (wraps around) */

    int prev = list[curr].prev;
    if (curr == head)

    return prev;

        printf("Next step wraps to HEAD.\n");

    return 0;
}

11.3 Reverse List

Theory
Visualize
C Code
Analogy

Swap PREV and NEXT pointers for every node. After swapping, update head to the old tail.

Algorithm:

  • Traverse all nodes
  • For each node: temp = node.prev; node.prev = node.next; node.next = temp
  • head = old tail = list[head].next (which is now the old prev after swap)
Reverse List Demo
Add nodes first
reverse.c
void reverseList() {
    int curr, temp, oldTail;
    if (isEmpty() || count == 1) return;

    oldTail = list[head].prev; /* save before swap */
    curr = head;
    do {
        /* Swap prev and next for each node */
        temp         = list[curr].prev;
        list[curr].prev = list[curr].next;
        list[curr].next = temp;
        curr = list[curr].prev; /* move forward (was .next) */
    } while (curr != head);

    head = oldTail; /* old tail is new head */
    printf("List reversed.\n");
}
🎬 Real-Life Analogy: Like reversing a train — the engine becomes the caboose and the caboose becomes the engine. Each car's "facing" direction flips, and what was behind is now in front.

Sorting Operations

Arranging nodes in order using classic sorting algorithms adapted for the circular doubly linked list structure. Since it's array-based, we swap data values (not pointers).
🫧

12.1 Bubble Sort

Theory
Visualize
C Code
Analogy

Compare adjacent nodes and swap data if out of order. Repeat for (n-1) passes. Each pass "bubbles" the largest unsorted element to the tail.

Time Complexity: O(n²)  |  Space: O(1)

💡 Best for small lists or nearly-sorted data.
Bubble Sort Visualization
Add nodes then sort
bubble_sort.c
void bubbleSort() {
    int i, j, curr, nextNode, temp;
    if (isEmpty() || count == 1) return;

    for (i = 0; i < count - 1; i++) {
        curr = head;
        for (j = 0; j < count - 1 - i; j++) {
            nextNode = list[curr].next;
            if (list[curr].data > list[nextNode].data) {
                /* Swap data, not pointers */
                temp = list[curr].data;
                list[curr].data = list[nextNode].data;
                list[nextNode].data = temp;
            }
            curr = list[curr].next;
        }
    }
    printf("Bubble sort complete.\n");
}
🫧 Real-Life Analogy: Like sorting people in a line by height — walk the line repeatedly and if two adjacent people are out of order, they swap places. Heaviest person "bubbles" to the back each round.
🎯

12.2 Selection Sort

Theory
C Code

Find the minimum element in the unsorted portion and swap it to the front of that portion. Only n-1 swaps — fewer than bubble sort.

Time: O(n²)  |  Swaps: O(n)

selection_sort.c
void selectionSort() {
    int i, j, curr, minNode, scan, temp;
    if (isEmpty()) return;
    curr = head;
    for (i = 0; i < count - 1; i++) {
        minNode = curr;
        scan = list[curr].next;
        for (j = i+1; j < count; j++) {
            if (list[scan].data < list[minNode].data)
                minNode = scan;
            scan = list[scan].next;
        }
        if (minNode != curr) {
            temp = list[curr].data;
            list[curr].data = list[minNode].data;
            list[minNode].data = temp;
        }
        curr = list[curr].next;
    }
    printf("Selection sort done.\n");
}
🃏

12.3 Insertion Sort

Theory
C Code

Build the sorted list one element at a time. Take each element and insert it in its correct position among the already-sorted elements.

Best case: O(n) for nearly sorted  |  Worst: O(n²)

insertion_sort.c
void insertionSort() {
    int i, j, curr, prev, key;
    if (isEmpty()) return;
    curr = list[head].next; /* start from 2nd */
    for (i = 1; i < count; i++) {
        key  = list[curr].data;
        prev = list[curr].prev;
        while (prev != list[head].prev &&
               list[prev].data > key) {
            list[list[prev].next].data = list[prev].data;
            prev = list[prev].prev;
        }
        list[list[prev].next].data = key;
        curr = list[curr].next;
    }
    printf("Insertion sort done.\n");
}

Recursive Operations

Using recursion — functions that call themselves — to traverse, search, and count. The key challenge in circular lists is preventing infinite recursion by passing a termination condition.
♾️

13.1 Recursive Traversal

Theory
C Code
Analogy

Traverse recursively by calling the function with the next node. The base case is when we return to head — but since we'd call it first with head, we need a flag or a steps counter to know when to stop.

Strategy: Pass original head as parameter

  • Call: recursiveTraverse(head, head)
  • Base case: curr == start AND we've already printed head once
⚠️ Without a proper base case, circular recursion causes INFINITE calls and stack overflow!
recursive_trav.c
/* Helper: traverse n nodes starting from curr */
void recursiveTravHelper(int curr, int remaining) {
    if (remaining == 0) return; /* base case */
    printf("%d ", list[curr].data);
    recursiveTravHelper(list[curr].next, remaining - 1);
}

void recursiveTraversal() {
    if (isEmpty()) { printf("Empty!\n"); return; }
    printf("Recursive: ");
    recursiveTravHelper(head, count); /* visit count nodes */
    printf("\n");
}

/* Backward recursive traversal */
void recursiveTravBackward(int curr, int remaining) {
    if (remaining == 0) return;
    printf("%d ", list[curr].data);
    recursiveTravBackward(list[curr].prev, remaining - 1);
}
🪆 Real-Life Analogy: Like Russian nesting dolls — each doll opens to reveal a smaller one. Recursive traversal is each doll saying "print me, then open the next." The counter tells us when to stop so we don't keep opening forever.
🔎

13.2 Recursive Search

Theory
C Code

Search for a value recursively. Each call checks the current node; if not found, recurse to next. Base case: remaining==0 means not found.

recursive_search.c
int recursiveSearch(int curr, int val,
                          int remaining, int pos) {
    if (remaining == 0) return -1; /* not found */
    if (list[curr].data == val) {
        printf("Found %d at pos %d\n", val, pos);
        return pos;
    }
    return recursiveSearch(
        list[curr].next, val, remaining-1, pos+1);
}

/* Caller function */
int searchRecursive(int val) {
    if (isEmpty()) return -1;
    return recursiveSearch(head, val, count, 1);
}

Iterative Operations

Loop-based (non-recursive) alternatives — usually preferred for large lists since they avoid stack overflow risk.
🔁

14.1 Iterative Traversal / 14.2 Iterative Search / 14.3 Circular Iteration

Theory
Visualize
C Code
Analogy

Iterative Traversal: A do-while loop starting from head, moving forward node by node, stopping when we return to head.

Iterative Search: Same do-while, but checks each node's data against the target. Returns position or -1.

Circular Iteration: Combines iteration with circular wrap to repeat for N rounds or until a condition is met — useful for simulation, scheduling, or game loops.

iterative_ops.c
/* 14.1 Iterative Traversal */
void iterativeTraversal() {
    int curr;
    if (isEmpty()) return;
    curr = head;
    do {
        printf("%d ", list[curr].data);
        curr = list[curr].next;
    } while (curr != head);
    printf("\n");
}

/* 14.2 Iterative Search */
int iterativeSearch(int val) {
    int curr = head, pos = 1;
    if (isEmpty()) return -1;
    do {
        if (list[curr].data == val) return pos;
        curr = list[curr].next;
        pos++;
    } while (curr != head);
    return -1;
}

/* 14.3 Circular Iteration — run N rounds */
void circularIteration(int rounds) {
    int curr = head;
    int total = count * rounds;
    int step;
    printf("Circular (%d rounds):\n", rounds);
    for (step = 0; step < total; step++) {
        printf("  Step %d: node[%d]=%d\n",
               step+1, curr, list[curr].data);
        curr = list[curr].next; /* wraps at tail */
    }
}
🏃 Real-Life Analogy: Iterative traversal is like a security guard doing rounds. They walk the entire circular building once (do-while stops at start). Circular iteration is the same guard doing 3 rounds — they keep going past the entrance until they've completed all rounds.
