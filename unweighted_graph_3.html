01 · Introduction
What is a Graph?
The fundamental data structure for modelling relationships
📐 Core Concept

A graph is a collection of vertices (nodes) connected by edges. Think of cities connected by roads, people connected by friendships, or computers connected by cables.


An unweighted graph means all edges are equal — there's no "distance" or "cost". An edge either exists or it doesn't.

🔑 Key Terminology
TermMeaning
Vertex (V)A node/point in the graph
Edge (E)A connection between two vertices
DegreeNumber of edges at a vertex
PathSequence of vertices connected by edges
CycleA path that starts and ends at same vertex

02 · Representation
Incidence Matrix
Rows = Vertices · Columns = Edges
📊 What is an Incidence Matrix?

An incidence matrix is a 2D array where:


  • Each row represents a vertex
  • Each column represents an edge
  • Cell = 1 if the vertex is part of that edge
  • Cell = 0 if the vertex is NOT part of that edge
💡 vs Adjacency Matrix

#include <stdio.h>
#include <stdlib.h>
#include <string.h>

#define MAX_V 10   /* max vertices */
#define MAX_E 20   /* max edges    */

/* Graph structure using incidence matrix */
typedef struct {
    int  matrix[MAX_V][MAX_E]; /* 1 = vertex is part of edge */
    int  vertexCount;          /* current number of vertices */
    int  edgeCount;            /* current number of edges    */
    char vertexName[MAX_V][20];/* optional vertex labels     */
} Graph;

/* Initialize empty graph */
void initGraph(Graph *g) {
    g->vertexCount = 0;
    g->edgeCount   = 0;
    memset(g->matrix, 0, sizeof(g->matrix));
}

int main() {
    Graph g;
    initGraph(&g);
    printf("Graph initialized! Vertices: %d, Edges: %d\n",
           g.vertexCount, g.edgeCount);
    return 0;
}

The incidence matrix is like a movie cast list. Movies are the columns (edges) and actors are the rows (vertices). A cell is 1 if that actor appeared in that movie. You can instantly see: which movies share actors, how many movies an actor was in, etc.

03 · Vertex Operations
Insert Vertex
Adding a new node to the graph
➕ How It Works

Inserting a vertex means registering a new row in the incidence matrix. Since no edges connect it yet, the entire new row is filled with zeros. The vertexCount increments by 1.


  1. Check if MAX_V limit is reached
  2. Assign the next available index
  3. Optionally set a name/label for the vertex
  4. Fill the new row with 0s (already done by memset)
  5. Increment vertexCount

Time Complexity: O(E) — we must zero out the new row across all edge columns

C · insert_vertex.c
/* Insert a vertex with an optional name label */
int insertVertex(Graph *g, const char *name) {
    if (g->vertexCount >= MAX_V) {
        printf("Error: Maximum vertex limit (%d) reached!\n", MAX_V);
        return -1;  /* failure */
    }

    int idx = g->vertexCount;  /* new vertex index */

    /* Zero out the new row (no edges yet) */
    for (int e = 0; e < g->edgeCount; e++)
        g->matrix[idx][e] = 0;

    /* Store name if provided */
    if (name != NULL)
        strncpy(g->vertexName[idx], name, 19);
    else
        sprintf(g->vertexName[idx], "V%d", idx);

    g->vertexCount++;
    printf("Vertex '%s' inserted at index %d\n",
           g->vertexName[idx], idx);
    return idx;   /* return the new vertex index */
}
🏙️ Real-Life Analogy

Inserting a vertex is like building a new city on a map. The city exists but has no roads (edges) yet. It's registered on the map, and roads can be added later.

03 · Vertex Operations
Delete Vertex
Removing a node and all its edges
🗑️ How It Works

Deleting a vertex is trickier than inserting. We must:


  1. Find all edges that include this vertex
  2. Delete each of those edges (remove their columns)
  3. Remove the vertex's row from the matrix
  4. Shift all subsequent rows up by one
  5. Decrement vertexCount

Time Complexity: O(V × E) — shifting rows requires updating many cells

// Delete Vertex Visualizer
Graph has 5 vertices and 4 edges. Select a vertex to delete.
C · delete_vertex.c
/* Delete vertex at given index, along with all its edges */
void deleteVertex(Graph *g, int vIdx) {
    if (vIdx < 0 || vIdx >= g->vertexCount) {
        printf("Error: Invalid vertex index %d\n", vIdx);
        return;
    }

    /* Step 1: Remove all edges incident to this vertex */
    int e = 0;
    while (e < g->edgeCount) {
        if (g->matrix[vIdx][e] == 1) {
            /* shift columns left to remove this edge column */
            for (int v = 0; v < g->vertexCount; v++)
                for (int j = e; j < g->edgeCount - 1; j++)
                    g->matrix[v][j] = g->matrix[v][j + 1];
            g->edgeCount--;
        } else {
            e++;
        }
    }

    /* Step 2: Remove vertex row — shift rows up */
    for (int v = vIdx; v < g->vertexCount - 1; v++) {
        for (int j = 0; j < g->edgeCount; j++)
            g->matrix[v][j] = g->matrix[v + 1][j];
        strcpy(g->vertexName[v], g->vertexName[v + 1]);
    }
    g->vertexCount--;
    printf("Vertex %d deleted. Vertices now: %d\n", vIdx, g->vertexCount);
}
✈️ Real-Life Analogy

Deleting a vertex is like closing an airport. You don't just erase the airport — you also cancel all flight routes (edges) that passed through it, then update all flight charts accordingly.

04 · Edge Operations
Insert Edge
Adding a connection between two vertices
🔗 How It Works

Adding an edge means creating a new column in the incidence matrix. We set matrix[v1][edgeIdx] = 1 and matrix[v2][edgeIdx] = 1. All other rows in that column stay 0.


  1. Check both vertex indices are valid
  2. Check edge limit (MAX_E) not exceeded
  3. Check for duplicate edges (optional)
  4. Create new column: set 1 at v1 and v2
  5. Increment edgeCount

Time Complexity: O(V) — must initialize the new column for all V vertices

Graph ready — 5 vertices (0–4), 0 edges. Add an edge!
C · insert_edge.c
/* Insert edge between vertices v1 and v2 */
int insertEdge(Graph *g, int v1, int v2) {
    if (v1 < 0 || v1 >= g->vertexCount ||
        v2 < 0 || v2 >= g->vertexCount) {
        printf("Error: Invalid vertex indices.\n"); return -1;
    }
    if (g->edgeCount >= MAX_E) {
        printf("Error: Edge limit reached.\n"); return -1;
    }
    if (v1 == v2) {
        printf("Error: Self-loops not supported.\n"); return -1;
    }

    /* Check for duplicate edge */
    for (int e = 0; e < g->edgeCount; e++) {
        if (g->matrix[v1][e] == 1 && g->matrix[v2][e] == 1) {
            printf("Edge (%d,%d) already exists!\n", v1, v2);
            return -1;
        }
    }

    int eIdx = g->edgeCount;  /* new edge column index */

    /* Zero out the new column first */
    for (int v = 0; v < g->vertexCount; v++)
        g->matrix[v][eIdx] = 0;

    /* Mark the two vertices */
    g->matrix[v1][eIdx] = 1;
    g->matrix[v2][eIdx] = 1;
    g->edgeCount++;

    printf("Edge e%d: (%s — %s) inserted.\n",
           eIdx, g->vertexName[v1], g->vertexName[v2]);
    return eIdx;
}
🛤️ Real-Life Analogy

Adding an edge is like building a new railway line between two cities. The cities already exist — you're just connecting them. The railway line (column) now "touches" both cities (rows).

/* Delete edge between v1 and v2 */
void deleteEdge(Graph *g, int v1, int v2) {
    for (int e = 0; e < g->edgeCount; e++) {
        if (g->matrix[v1][e] == 1 && g->matrix[v2][e] == 1) {
            /* Shift all columns after e to the left */
            for (int v = 0; v < g->vertexCount; v++)
                for (int j = e; j < g->edgeCount - 1; j++)
                    g->matrix[v][j] = g->matrix[v][j + 1];
            g->edgeCount--;
            printf("Edge (%d, %d) deleted.\n", v1, v2);
            return;
        }
    }
    printf("Edge (%d, %d) not found.\n", v1, v2);
}

05 · Traversal Operations
Breadth First Search (BFS)
Explore level-by-level using a Queue
🌊 BFS Concept

BFS visits the starting vertex first, then all its neighbours, then their neighbours — spreading out like a wave. It uses a queue (FIFO) to track what to visit next.


  1. Enqueue the start vertex; mark it visited
  2. Dequeue front vertex — process it
  3. Find all adjacent unvisited vertices
  4. Enqueue each adjacent vertex; mark visited
  5. Repeat until queue is empty

Time Complexity: O(V × E) — finding adjacency in incidence matrix costs O(E) per vertex

C · bfs.c
/* BFS using incidence matrix */
void BFS(const Graph *g, int start) {
    int visited[MAX_V] = {0};   /* 0 = not visited */
    int queue[MAX_V];
    int front = 0, rear = 0;

    /* Enqueue start, mark visited */
    queue[rear++] = start;
    visited[start] = 1;

    printf("BFS from %s: ", g->vertexName[start]);

    while (front < rear) {              /* queue not empty */
        int curr = queue[front++];      /* dequeue */
        printf("%s ", g->vertexName[curr]);

        /* Find all vertices adjacent to curr via incidence matrix */
        for (int e = 0; e < g->edgeCount; e++) {
            if (g->matrix[curr][e] == 1) {  /* edge e is incident to curr */
                for (int v = 0; v < g->vertexCount; v++) {
                    if (v != curr && g->matrix[v][e] == 1 && !visited[v]) {
                        visited[v] = 1;
                        queue[rear++] = v;  /* enqueue neighbour */
                    }
                }
            }
        }
    }
    printf("\n");
}
💧 Real-Life Analogy

BFS is like water spreading in a pond. Drop a stone at the start vertex — the ripple visits all nodes 1 hop away first, then 2 hops, then 3. It guarantees the shortest path in an unweighted graph.

Depth First Search (DFS)
Explore as deep as possible before backtracking
🌲 DFS Concept

DFS goes as deep as possible down one path before backtracking. It can be implemented recursively (using the call stack) or iteratively (using an explicit stack).

C · dfs.c
/* Helper: get all neighbours of vertex v */
void getNeighbours(const Graph *g, int v, int *nbrs, int *cnt) {
    *cnt = 0;
    for (int e = 0; e < g->edgeCount; e++)
        if (g->matrix[v][e] == 1)
            for (int u = 0; u < g->vertexCount; u++)
                if (u != v && g->matrix[u][e] == 1)
                    nbrs[(*cnt)++] = u;
}

/* Recursive DFS */
void DFS_Recursive(const Graph *g, int v, int visited[]) {
    visited[v] = 1;
    printf("%s ", g->vertexName[v]);
    int nbrs[MAX_V], cnt;
    getNeighbours(g, v, nbrs, &cnt);
    for (int i = 0; i < cnt; i++)
        if (!visited[nbrs[i]])
            DFS_Recursive(g, nbrs[i], visited);  /* recurse deeper */
}

/* Iterative DFS using explicit stack */
void DFS_Stack(const Graph *g, int start) {
    int visited[MAX_V] = {0};
    int stk[MAX_V];
    int top = -1;
    stk[++top] = start;

    while (top >= 0) {
        int v = stk[top--];           /* pop */
        if (visited[v]) continue;
        visited[v] = 1;
        printf("%s ", g->vertexName[v]);
        int nbrs[MAX_V], cnt;
        getNeighbours(g, v, nbrs, &cnt);
        for (int i = cnt - 1; i >= 0; i--)
            if (!visited[nbrs[i]])
                stk[++top] = nbrs[i];  /* push neighbours */
    }
}
🌿 Real-Life Analogy

DFS is like exploring a maze. You always pick a new unexplored path and go as far as you can. When you hit a dead end, you backtrack to the last junction and try another path. Eventually you've explored every passage.

📍 Shortest Path

In an unweighted graph, BFS naturally finds the shortest path — minimum number of edges between two nodes.

O(V × E)

🗺️ All Paths

Use DFS with backtracking — explore all routes from source to destination, collecting each complete path.

O(V!) worst case

📡 Reachability Check

Run BFS/DFS from source. If destination is visited at the end, it is reachable. Otherwise it isn't.

O(V × E)

C · path_operations.c
/* Shortest path via BFS — stores path in 'path[]', returns length */
int shortestPath(const Graph *g, int src, int dst, int path[]) {
    int visited[MAX_V] = {0};
    int parent[MAX_V];
    for (int i = 0; i < MAX_V; i++) parent[i] = -1;

    int queue[MAX_V], front = 0, rear = 0;
    visited[src] = 1;
    queue[rear++] = src;

    while (front < rear) {
        int curr = queue[front++];
        if (curr == dst) break;
        for (int e = 0; e < g->edgeCount; e++)
            if (g->matrix[curr][e] == 1)
                for (int v = 0; v < g->vertexCount; v++)
                    if (v != curr && g->matrix[v][e] == 1 && !visited[v]) {
                        visited[v] = 1;
                        parent[v] = curr;
                        queue[rear++] = v;
                    }
    }
    if (!visited[dst]) return -1;  /* unreachable */

    /* Reconstruct path from parent array */
    int len = 0, tmp = dst;
    while (tmp != -1) { path[len++] = tmp; tmp = parent[tmp]; }
    /* Reverse path array */
    for (int i = 0, j = len-1; i < j; i++, j--) {
        int t = path[i]; path[i] = path[j]; path[j] = t;
    }
    return len;
}

/* All paths using DFS + backtracking */
void allPaths(const Graph *g, int curr, int dst,
              int visited[], int path[], int depth) {
    visited[curr] = 1;
    path[depth] = curr;
    if (curr == dst) {
        for (int i = 0; i <= depth; i++)
            printf("%d%s", path[i], i == depth ? "\n" : " → ");
    } else {
        int nbrs[MAX_V], cnt;
        getNeighbours(g, curr, nbrs, &cnt);
        for (int i = 0; i < cnt; i++)
            if (!visited[nbrs[i]])
                allPaths(g, nbrs[i], dst, visited, path, depth + 1);
    }
    visited[curr] = 0;  /* backtrack */
}

/* Reachability check — returns 1 if dst is reachable from src */
int isReachable(const Graph *g, int src, int dst) {
    int path[MAX_V], len = shortestPath(g, src, dst, path);
    return (len != -1);
}

/* Path Length Count — number of edges in path */
int pathLength(const Graph *g, int src, int dst) {
    int path[MAX_V];
    int len = shortestPath(g, src, dst, path);
    if (len == -1) return -1;
    return len - 1;  /* vertices - 1 = edges in path */
}
🗺️ Real-Life Analogy

Path operations are like Google Maps. Finding a path = any route exists. Shortest path = fewest turns/roads. All paths = every possible route from A to B. Reachability = can you even get there at all?

07 · Connectivity Operations
Connectivity Operations
Connected · Components · Isolated Vertices · Reachable Vertices
🔗 Check Connected Graph

A graph is connected if you can reach every vertex from any starting vertex. Run BFS/DFS from vertex 0 — if all vertices are visited, it's connected.

O(V × E)

🧩 Connected Components

A connected component is a maximal subgraph where all vertices are reachable from each other. A disconnected graph has 2+ components.

Algorithm: Repeat BFS/DFS from each unvisited vertex — each BFS tree = one component.

C · connectivity.c
/* Check if graph is connected */
int isConnected(const Graph *g) {
    if (g->vertexCount == 0) return 1;
    int visited[MAX_V] = {0};
    int queue[MAX_V], front = 0, rear = 0;
    visited[0] = 1; queue[rear++] = 0;
    while (front < rear) {
        int curr = queue[front++];
        for (int e = 0; e < g->edgeCount; e++)
            if (g->matrix[curr][e])
                for (int v = 0; v < g->vertexCount; v++)
                    if (v != curr && g->matrix[v][e] && !visited[v])
                        { visited[v] = 1; queue[rear++] = v; }
    }
    for (int v = 0; v < g->vertexCount; v++)
        if (!visited[v]) return 0;  /* some vertex unreached */
    return 1;
}

/* Find number of connected components */
int connectedComponents(const Graph *g, int comp[]) {
    int visited[MAX_V] = {0};
    int numComp = 0;
    for (int i = 0; i < g->vertexCount; i++) {
        if (!visited[i]) {
            int queue[MAX_V], front = 0, rear = 0;
            visited[i] = 1; comp[i] = numComp;
            queue[rear++] = i;
            while (front < rear) {
                int curr = queue[front++];
                for (int e = 0; e < g->edgeCount; e++)
                    if (g->matrix[curr][e])
                        for (int v = 0; v < g->vertexCount; v++)
                            if (v != curr && g->matrix[v][e] && !visited[v])
                                { visited[v]=1; comp[v]=numComp; queue[rear++]=v; }
            }
            numComp++;
        }
    }
    return numComp;
}

/* Detect isolated vertices (degree = 0) */
void detectIsolated(const Graph *g) {
    printf("Isolated vertices: ");
    for (int v = 0; v < g->vertexCount; v++) {
        int deg = 0;
        for (int e = 0; e < g->edgeCount; e++) deg += g->matrix[v][e];
        if (deg == 0) printf("%s ", g->vertexName[v]);
    }
    printf("\n");
}
🏝️ Real-Life Analogy

A connected component is like an island. Within the island, you can travel anywhere by road. But you cannot reach the mainland (another component) without a bridge. An isolated vertex is a tiny island with no bridges at all.

08 · Cycle Operations
Cycle Operations
Detect Cycle · Count Cycles · Check Cyclic Graph
🔄 What is a Cycle?

A cycle is a path that starts and ends at the same vertex without repeating any vertex or edge. A graph with at least one cycle is called a cyclic graph; one with no cycles is called a tree (acyclic).


Using DFS — if we find an already-visited neighbour that isn't the direct parent, a cycle exists.


Time: O(V × E)

// Cycle Detector
Graph loaded. Click "Detect Cycle" to check for cycles.
C · cycle_detection.c
/* DFS-based cycle detection for undirected graph */
int dfsHasCycle(const Graph *g, int v, int parent,
               int visited[]) {
    visited[v] = 1;
    int nbrs[MAX_V], cnt;
    getNeighbours(g, v, nbrs, &cnt);
    for (int i = 0; i < cnt; i++) {
        int u = nbrs[i];
        if (!visited[u]) {
            if (dfsHasCycle(g, u, v, visited))
                return 1;                  /* cycle found deeper */
        } else if (u != parent) {
            return 1;                      /* back-edge → cycle! */
        }
    }
    return 0;
}

/* Check if graph is cyclic */
int isCyclic(const Graph *g) {
    int visited[MAX_V] = {0};
    for (int v = 0; v < g->vertexCount; v++)
        if (!visited[v])
            if (dfsHasCycle(g, v, -1, visited))
                return 1;
    return 0;
}

/* Count minimum cycles (fundamental cycles): |E| - |V| + components */
int countCycles(const Graph *g) {
    int comp[MAX_V];
    int c = connectedComponents(g, comp);
    return g->edgeCount - g->vertexCount + c;  /* cycle rank formula */
}
🚗 Real-Life Analogy

A cycle in a road network means you can drive in a loop and return to where you started. A road with no cycles (a tree) would mean there's only one path between any two cities — like a branching tree of roads with no loop-backs.

    int maxD = 0, maxV = 0;
    for (int v = 0; v < g->vertexCount; v++) {
        int d = findDegree(g, v);

/* Average degree = 2|E| / |V|  (Handshaking Lemma) */
float avgDegree(const Graph *g) {
    if (g->vertexCount == 0) return 0.0;
    float avg = (float)(2 * g->edgeCount) / g->vertexCount;
    printf("Average degree = %.2f\n", avg);
    return avg;
}

/* Degree distribution: count how many vertices have each degree */

    int dist[MAX_E + 1] = {0};
    for (int v = 0; v < g->vertexCount; v++) {
        int d = findDegree(g, v);
        dist[d]++;
    }
    printf("Degree Distribution:\n");
    for (int d = 0; d <= g->edgeCount; d++)
        if (dist[d] > 0)
            printf("  Degree %d: %d vertices\n", d, dist[d]);
}

🎨 What is a Bipartite Graph?

A graph is bipartite if vertices can be split into two groups such that every edge connects a vertex from group 1 to group 2 — never two vertices in the same group.


Key fact: A graph is bipartite if and only if it contains no odd-length cycles.


Algorithm: BFS 2-coloring — try to colour every vertex alternately. If you ever need to colour a vertex a colour it already has, it's not bipartite.


O(V × E)

/* Bipartite check using BFS 2-coloring */
int isBipartite(const Graph *g, int color[]) {
    for (int i = 0; i < g->vertexCount; i++) color[i] = -1;

    for (int start = 0; start < g->vertexCount; start++) {
        if (color[start] != -1) continue;

        int queue[MAX_V], front = 0, rear = 0;
        color[start] = 0;          /* assign colour 0 */
        queue[rear++] = start;

        while (front < rear) {
            int v = queue[front++];
            int nbrs[MAX_V], cnt;
            getNeighbours(g, v, nbrs, &cnt);
            for (int i = 0; i < cnt; i++) {
                int u = nbrs[i];
                if (color[u] == -1) {
                    color[u] = 1 - color[v];  /* alternate colour */
                    queue[rear++] = u;
                } else if (color[u] == color[v]) {
                    printf("NOT bipartite! Conflict at vertices %d and %d\n", v, u);
                    return 0;              /* same colour conflict! */
                }
            }
        }
    }
    printf("Graph IS bipartite.\n");
    return 1;
}
⚽ Real-Life Analogy

A bipartite graph is like a sports match schedule. Group 1 = Team A players, Group 2 = Team B players. Every game (edge) must connect one player from each team — you'd never schedule Team A vs Team A.

/* Density = E / (V*(V-1)/2)  — between 0 and 1 */
float graphDensity(const Graph *g) {
    int V = g->vertexCount;
    if (V < 2) return 0.0;
    int maxEdges = V * (V - 1) / 2;
    float d = (float)g->edgeCount / maxEdges;
    printf("Density = %.4f (%s)\n", d, d > 0.5 ? "Dense" : "Sparse");
    return d;
}

/* Complete graph check: every pair of vertices must be connected */
int isComplete(const Graph *g) {
    int V = g->vertexCount;
    int needed = V * (V - 1) / 2;  /* max edges for undirected */
    if (g->edgeCount != needed) return 0;
    /* Also verify every pair has exactly one edge */
    for (int v1 = 0; v1 < V; v1++)
        for (int v2 = v1 + 1; v2 < V; v2++)
            if (searchEdge(g, v1, v2) == -1) return 0;
    printf("Graph is COMPLETE (K%d)\n", V);
    return 1;
}

A complete graph is like everyone in a room knowing everyone else — maximum connectivity. A sparse graph is like a sparse city bus network with few routes. A dense graph is like a city subway system with connections everywhere.

💡 Key Takeaway: Incidence Matrix Trade-offs

The incidence matrix is best when you need to enumerate edges per vertex or perform edge-centric analysis. Its main downside is that adjacency checks are O(E) instead of O(1) like in an adjacency matrix. For most general-purpose graph algorithms, adjacency lists are more space-efficient and faster. Use the incidence matrix when edges need explicit IDs or when hypergraph generalizations are needed.
