Introduction to Cyclic Graphs

Beginner

What is a Graph?

A graph is a collection of vertices (nodes) connected by edges (links). Think of cities (vertices) connected by roads (edges).

What Makes a Graph "Cyclic"?

A graph is cyclic if there exists at least one cycle — a path that starts and ends at the same vertex without repeating edges. Example: A→B→C→A is a cycle.

🌍 Real-life Analogy

Think of a road network in a city. You can drive from Home → School → Market → Home in a loop. That loop is a cycle! Any network where you can return to your starting point has cycles.

🔵 Directed Cyclic

Edges have direction. A→B is different from B→A. Common in dependency graphs, state machines.

⚪ Undirected Cyclic

Edges go both ways. A–B means you can go A→B or B→A. Common in social networks, road maps.

🔁 Self-Loop

A vertex has an edge pointing to itself. A→A. Common in automata theory and dependency graphs.

Adjacency Matrix Representation

Beginner

What is an Adjacency Matrix?

A 2D array where matrix[i][j] = 1 means there is an edge from vertex i to vertex j, and 0 means no edge.

Space Complexity: O(V²) — where V = number of vertices.
Edge Lookup: O(1) — check matrix[i][j] directly.
Best for: Dense graphs (many edges).

Graph: 0→1, 0→3, 1→2, 2→3, 3→0 (cycle!)

adjacency_matrix.c
#include <stdio.h>

#define MAX 10

int adj[MAX][MAX];   // adjacency matrix
int V = 0;           // number of vertices

/* Initialize the matrix to all zeros */
void initGraph(int vertices) {
    V = vertices;
    int i, j;
    for (i = 0; i < V; i++)
        for (j = 0; j < V; j++)
            adj[i][j] = 0;
}

/* Print the adjacency matrix */
void printMatrix() {
    int i, j;
    printf("Adjacency Matrix:\n");
    for (i = 0; i < V; i++) {
        for (j = 0; j < V; j++)
            printf("%d ", adj[i][j]);
        printf("\n");
    }
}

int main() {
    initGraph(4);
    /* Add edges manually */
    adj[0][1] = 1;  adj[0][3] = 1;
    adj[1][2] = 1;  adj[2][3] = 1;
    adj[3][0] = 1;  /* creates a cycle! */
    printMatrix();
    return 0;
}
🌍 Real-life Analogy

An adjacency matrix is like a friendship table in school. Rows and columns are students. Put a 1 where two students are friends. A cycle means: Alex → Bob → Carol → Alex are all friends in a circle!

Vertex Operations — Insert, Delete, Search

Beginner

Theory

Adding a vertex means increasing V by 1. The new row and column in the matrix are initialized to 0 (no edges yet). The matrix size grows from V×V to (V+1)×(V+1).

1

Increment V

Increase vertex count by 1.

2

Zero new row

Set adj[newV][j] = 0 for all j.

3

Zero new column

Set adj[i][newV] = 0 for all i.

insert_vertex.c
/* Insert a new vertex into the graph */
void insertVertex() {
    int i;
    /* Initialize new row to 0 */
    for (i = 0; i <= V; i++)
        adj[V][i] = 0;
    /* Initialize new column to 0 */
    for (i = 0; i < V; i++)
        adj[i][V] = 0;
    V++;   /* Increase vertex count */
    printf("Vertex %d inserted.\n", V-1);
}

🌍 Real-life Analogy

Adding a new city to a map. The city starts with no roads (0 connections). Later you can add roads (edges) to connect it.

Theory

Deleting vertex v means shifting all rows and columns after v one position left/up, then decrementing V. All edges involving v are removed.

⚠️ After deletion, vertex indices shift! Vertex 3 becomes 2 if vertex 2 is deleted.
delete_vertex.c
/* Delete vertex v from the graph */
void deleteVertex(int v) {
    int i, j;
    if (v >= V) {
        printf("Vertex not found!\n");
        return;
    }
    /* Shift rows up */
    for (i = v; i < V - 1; i++)
        for (j = 0; j < V; j++)
            adj[i][j] = adj[i+1][j];
    /* Shift columns left */
    for (j = v; j < V - 1; j++)
        for (i = 0; i < V - 1; i++)
            adj[i][j] = adj[i][j+1];
    V--;
    printf("Vertex %d deleted.\n", v);
}
🌍 Real-life Analogy

Removing a city from a map. All roads (edges) to/from that city are automatically removed too.

Theory

Searching for a vertex simply checks if vertex index v is within range [0, V-1]. If valid, the vertex exists. Time complexity: O(1).

Changing the distance (weight) on a road sign between two cities.

Count all adj[i][j] == 1 entries. For directed graphs, this is the full sum. For undirected, divide by 2 (each edge stored twice). Also called the size of the graph. Time complexity: O(V²).

int countEdges(int directed) {

            if (adj[i][j] != 0)
                count++;
    if (!directed) count /= 2; /* undirected */
    printf("Total edges: %d\n", count);
    return count;
}

Breadth First Search (BFS)

Intermediate

Theory

BFS visits all vertices at distance 1 first, then distance 2, and so on. Uses a Queue. It explores level by level — like ripples in water. Time complexity: O(V²) with adjacency matrix.

1

Queue Initialization

Create a queue. Mark start vertex as visited. Enqueue start.

2

Mark Visited

Keep a visited[] array. Set visited[v]=1 when enqueued.

3

Visit Node

Dequeue a vertex, process it (print it).

4

Traverse Adjacent Nodes

For each neighbor j: if adj[v][j]==1 and !visited[j], enqueue j.

🌍 Real-life Analogy

Spreading news in a city: You tell all your direct friends first (level 1), then they tell their friends (level 2), and so on — like social media sharing.

bfs.c
#define MAX 10
int queue[MAX], front = -1, rear = -1;
int visited[MAX];

void enqueue(int x) { queue[++rear] = x; }
int  dequeue()      { return queue[++front]; }
int  isEmpty()      { return front == rear; }

void BFS(int start) {
    int i, v;
    /* Initialize visited */
    for (i = 0; i < V; i++) visited[i] = 0;
    front = rear = -1;

    visited[start] = 1;
    enqueue(start);

    while (!isEmpty()) {
        v = dequeue();
        printf("%d ", v);   /* Visit node */

        /* Traverse adjacent nodes */
        for (i = 0; i < V; i++) {
            if (adj[v][i] == 1 && !visited[i]) {
                visited[i] = 1;
                enqueue(i);
            }
        }
    }
}

Depth First Search (DFS)

Intermediate

Theory

DFS goes as deep as possible before backtracking. Uses a Stack (or recursion). Like exploring a maze — go forward until stuck, then backtrack. Time: O(V²).

1

Recursive DFS

Call DFS(v): mark visited, recurse on unvisited neighbors.

2

Stack DFS (Iterative)

Push start. While stack not empty: pop, visit, push neighbors.

3

Visit Node

Process the current node (e.g., print it).

4

Backtracking

When no unvisited neighbors remain, return to the previous node.

🌍 Real-life Analogy

Exploring a cave system: keep going deeper into one tunnel until it dead-ends, then backtrack to try another tunnel.

dfs.c
int visited[MAX];

/* Recursive DFS */
void DFS(int v) {
    int i;
    visited[v] = 1;
    printf("%d ", v);   /* Visit node */
    for (i = 0; i < V; i++)
        if (adj[v][i] == 1 && !visited[i])
            DFS(i);   /* Recurse deep */
}

/* Iterative DFS using stack */
void DFS_Iterative(int start) {
    int stack[MAX], top = -1, i, v;
    int vis[MAX] = {0};

    stack[++top] = start;
    while (top != -1) {
        v = stack[top--];
        if (vis[v]) continue;
        vis[v] = 1;
        printf("%d ", v);
        for (i = V-1; i >= 0; i--)
            if (adj[v][i] && !vis[i])
                stack[++top] = i;
    }
}

Cycle Operations — Detect & Count Cycles

Intermediate

Theory

Use DFS with a recursion stack (recStack[]). If during DFS you reach a vertex that is currently in the recursion stack, a cycle exists. This is because reaching a node in the current call chain means we've looped back.

1

Start DFS from each unvisited vertex

Some vertices may not be reachable from others.

2

Track recursion stack

Mark vertex as "in stack" when DFS enters, unmark when it exits.

3

Check back edge

If adj[v][i]==1 and recStack[i]==1, cycle found!

detect_cycle_directed.c
int visited[MAX], recStack[MAX];

int isCyclicUtil(int v) {
    int i;
    visited[v] = 1;
    recStack[v] = 1;  /* add to stack */

    for (i = 0; i < V; i++) {
        if (adj[v][i] == 1) {
            if (!visited[i] && isCyclicUtil(i))
                return 1;
            else if (recStack[i])
                return 1; /* back edge! */
        }
    }
    recStack[v] = 0; /* remove from stack */
    return 0;
}

int detectCycle() {
    int i;
    for (i=0; i<V; i++) visited[i]=recStack[i]=0;
    for (i=0; i<V; i++)
        if(!visited[i] && isCyclicUtil(i))
            return 1;
    return 0;
}
🌍 Real-life Analogy

Checking if a set of tasks has a circular dependency — Task A needs B, B needs C, C needs A. This would cause an infinite loop!

Theory

Counting all distinct cycles in a graph is computationally expensive (NP-hard for the general case). A practical approach uses DFS and counts the number of back edges encountered. Each back edge in a DFS tree corresponds to at least one cycle.

count_cycles.c
int cycleCount = 0;

void countCyclesUtil(int v, int vis[], int rec[]) {
    int i;
    vis[v] = rec[v] = 1;
    for (i = 0; i < V; i++) {
        if (adj[v][i]) {
            if (!vis[i])
                countCyclesUtil(i, vis, rec);
            else if (rec[i])
                cycleCount++;  /* back edge = cycle */
        }
    }
    rec[v] = 0;
}
🌍 Real-life Analogy

Counting how many circular routes exist in a transport network (like counting metro loop lines).

Cycle Operations — Find Path, Self Loop

Intermediate

Theory

To find the actual vertices forming a cycle, maintain a path array during DFS. When a back edge is found, trace back from the current node to the ancestor to extract the cycle path.

find_cycle_path.c
int path[MAX], pathLen = 0;
int color[MAX]; /* 0=white,1=gray,2=black */

int findCycleUtil(int v) {
    int i;
    color[v] = 1;           /* gray: in stack */
    path[pathLen++] = v;

    for (i = 0; i < V; i++) {
        if (!adj[v][i]) continue;
        if (color[i] == 1) {
            /* Found cycle: print from i */
            int j = pathLen-1;
            printf("Cycle: ");
            while(path[j] != i) j--;
            for(int k=j; k<pathLen; k++)
                printf("%d->", path[k]);
            printf("%d\n", i);
            return 1;
        }
        if (!color[i] && findCycleUtil(i))
            return 1;
    }
    pathLen--;
    color[v] = 2;           /* black: done */
    return 0;
}
🌍 Real-life Analogy

Not just detecting a traffic loop, but telling drivers exactly which streets form the loop: "Turn left at Main St → Oak Ave → Park Rd → back to Main St."

Theory

A self-loop is an edge from a vertex to itself: adj[i][i] == 1. Simply check the main diagonal of the adjacency matrix. Time complexity: O(V).

Cycle Operations — Back Edge, Directed & Undirected Detection

Intermediate

Theory

In a DFS tree, a back edge is an edge from a vertex to one of its ancestors in the DFS tree. In directed graphs, back edges are the only edges that create cycles. Track using a "currently in recursion" (gray) marker.

back_edge.c
/* color: 0=unvisited, 1=in progress, 2=done */
int color[MAX];

int hasBackEdge(int v) {
    int i;
    color[v] = 1;   /* mark as "in progress" */
    for (i = 0; i < V; i++) {
        if (adj[v][i]) {
            if (color[i] == 1) {
                printf("Back edge: %d->%d\n",v,i);
                return 1;
            }
            if (!color[i] && hasBackEdge(i))
                return 1;
        }
    }
    color[v] = 2;   /* mark as "done" */
    return 0;
}
🌍 Real-life Analogy

In a family tree: a back edge would be if a child is somehow also an ancestor of their own parent — impossible in real life, but in a directed graph it means a cycle!

Theory

Already shown above using recStack. Key insight: for directed graphs, a cycle requires the back edge to go to a node currently in the active recursion stack (gray node), not just any visited node.

detect_directed_cycle.c
/* Same as isCyclicUtil above.
   Key: recStack[i]==1 means "in current path"
   This distinguishes directed from undirected. */

int detectDirectedCycle() {
    int i;
    int vis[MAX]={0}, rec[MAX]={0};
    for (i=0; i<V; i++)
        if (!vis[i] && isCyclicUtil(i))
            return 1;
    return 0;
}

Theory

For undirected graphs, a cycle exists if during DFS we reach an already-visited node that is not the parent of the current node. We track the parent to avoid false positives (since every undirected edge appears twice in the matrix).

detect_undirected_cycle.c
int undirCycleUtil(int v, int parent,
                   int vis[]) {
    int i;
    vis[v] = 1;
    for (i = 0; i < V; i++) {
        if (adj[v][i] == 1) {
            if (!vis[i]) {
                if (undirCycleUtil(i, v, vis))
                    return 1;
            }
            /* Not parent? Cycle! */
            else if (i != parent)
                return 1;
        }
    }
    return 0;
}
🌍 Real-life Analogy

Walking on an undirected road network: if you reach a city you've already visited without going backwards on the same road, you've found a loop!

Remove Cycle

Intermediate

Theory

To remove a cycle, find the back edge that creates it, then delete that edge. For the cycle A→B→C→A, removing any one edge (e.g., C→A) breaks the cycle. This is used in dependency resolution.

⚠️ Only ONE back edge needs to be removed to break a specific cycle. Multiple cycles may require multiple removals.
🌍 Real-life Analogy

Breaking a circular dependency in software packages: if A needs B, B needs C, C needs A — you remove one dependency to resolve the circular install problem.

remove_cycle.c
int removeCycleUtil(int v, int vis[], int rec[]) {
    int i;
    vis[v] = rec[v] = 1;
    for (i = 0; i < V; i++) {
        if (adj[v][i]) {
            if (!vis[i]) {
                if (removeCycleUtil(i, vis, rec))
                    return 1;
            } else if (rec[i]) {
                /* Remove back edge to break cycle */
                adj[v][i] = 0;
                printf("Removed edge %d->%d\n",v,i);
                return 1;
            }
        }
    }
    rec[v] = 0;
    return 0;
}

void removeCycle() {
    int vis[MAX]={0}, rec[MAX]={0}, i;
    for(i=0; i<V; i++)
        if(!vis[i]) removeCycleUtil(i, vis, rec);
}

Path Operations — Find Path, Shortest Path, All Paths

Intermediate

Theory

Use DFS to find any path from source to destination. Maintain a path array. When destination is reached, print the path. Backtrack if destination not reachable via current vertex.

find_path.c
int path[MAX], pathLen;
int vis[MAX];

int findPath(int src, int dest) {
    int i;
    vis[src] = 1;
    path[pathLen++] = src;

    if (src == dest) {
        for(i=0; i<pathLen; i++)
            printf("%d%s", path[i],
                   i<pathLen-1?"->":"\n");
        return 1;
    }
    for (i=0; i<V; i++)
        if (adj[src][i] && !vis[i])
            if (findPath(i, dest)) return 1;

    pathLen--;  /* backtrack */
    return 0;
}
🌍 Real-life Analogy

Finding any route from your home to a restaurant using Google Maps — not necessarily the shortest, just a valid path.

Theory

BFS naturally finds the shortest path in an unweighted graph — since it explores level by level, the first time it reaches the destination is via the fewest edges. Track parent[] to reconstruct the path.

shortest_path_bfs.c
void shortestPath(int src, int dest) {
    int vis[MAX]={0}, par[MAX], dist[MAX];
    int q[MAX], f=-1, r=-1, i, v;
    for(i=0;i<V;i++) par[i]=dist[i]=-1;

    vis[src]=1; dist[src]=0;
    q[++r]=src;
    while(f!=r) {
        v=q[++f];
        for(i=0;i<V;i++) {
            if(adj[v][i]&&!vis[i]){
                vis[i]=1;
                par[i]=v;
                dist[i]=dist[v]+1;
                q[++r]=i;
            }
        }
    }
    if(dist[dest]==-1){
        printf("No path!\n"); return;
    }
    /* Reconstruct path using parent array */
    int p[MAX], plen=0, cur=dest;
    while(cur!=-1){ p[plen++]=cur; cur=par[cur]; }
    for(i=plen-1;i>=0;i--)
        printf("%d%s",p[i],i?"->":"\n");
}
🌍 Real-life Analogy

GPS navigation finding the route with the fewest turns/roads between home and destination.

Theory

Use DFS with backtracking. Instead of stopping when destination is found, continue after printing the path. This explores all possible routes but can be exponential in complexity.

all_paths.c
void allPaths(int src, int dest,
             int vis[], int path[], int plen) {
    int i;
    vis[src] = 1;
    path[plen++] = src;

    if (src == dest) {
        for(i=0;i<plen;i++)
            printf("%d%s",path[i],
                   i<plen-1?"->":"\n");
    } else {
        for(i=0;i<V;i++)
            if(adj[src][i]&&!vis[i])
                allPaths(i,dest,vis,path,plen);
    }
    vis[src]=0; /* backtrack: unmark */
}
🌍 Real-life Analogy

Listing every possible route from home to work — useful for comparing options, but there could be many!

Path Operations — Reachability Check & Path Length

Intermediate

Theory

Can vertex dest be reached from src? Use BFS/DFS from src and check if dest gets visited. Time: O(V²) with adjacency matrix.

reachability.c
int isReachable(int src, int dest) {
    int vis[MAX]={0}, q[MAX], f=-1,r=-1, v, i;

Connectivity — Strong Connectivity & Reachable Vertices

Intermediate

Theory

A directed graph is strongly connected if every vertex can reach every other vertex. Method: (1) Run DFS from vertex 0. (2) Transpose the graph (reverse all edges). (3) Run DFS again from vertex 0. If all vertices visited both times → strongly connected.

strong_connectivity.c
void dfs(int v, int g[][MAX], int vis[]) {
    vis[v]=1;
    int i;
    for(i=0;i<V;i++)
        if(g[v][i]&&!vis[i]) dfs(i,g,vis);
}

int isStronglyConnected() {
    int vis[MAX]={0}, i, j;
    dfs(0, adj, vis);
    for(i=0;i<V;i++) if(!vis[i]) return 0;

    /* Transpose graph */
    int trans[MAX][MAX]={0};
    for(i=0;i<V;i++) for(j=0;j<V;j++)
        trans[j][i]=adj[i][j];

    int vis2[MAX]={0};
    dfs(0, trans, vis2);
    for(i=0;i<V;i++) if(!vis2[i]) return 0;
    return 1;
}

Tarjan's Algorithm

Advanced

Theory

Tarjan's algorithm finds all Strongly Connected Components (SCCs) in a single DFS pass. It uses two arrays: disc[] (discovery time) and low[] (lowest discovery reachable). A vertex is the root of an SCC when low[v] == disc[v].

1

DFS with timestamps

Assign each vertex a discovery time disc[v] and low[v].

2

Maintain a stack

Push vertices onto stack when visited. Track which are on stack.

3

Update low values

low[v] = min(low[v], low[neighbor]) for tree edges; min with disc for back edges.

4

Find SCC root

When low[v] == disc[v], pop stack until v to get one SCC.

🌍 Real-life Analogy

Finding clusters of cities where you can travel in a complete loop between any two cities in the cluster — these are Strongly Connected Components.

tarjan.c
int disc[MAX], low[MAX], onStack[MAX];
int stk[MAX], top=-1, timer=0;

void tarjanUtil(int v) {
    int i;
    disc[v] = low[v] = timer++;
    stk[++top] = v;
    onStack[v] = 1;

    for(i=0;i<V;i++) {
        if(!adj[v][i]) continue;
        if(disc[i]==-1) {      /* tree edge */
            tarjanUtil(i);
            low[v]=low[v]<low[i]?low[v]:low[i];
        } else if(onStack[i]) { /* back edge */
            low[v]=low[v]<disc[i]?low[v]:disc[i];
        }
    }
    /* Root of SCC? */
    if(low[v]==disc[v]) {
        printf("SCC: ");
        while(stk[top]!=v) {
            printf("%d ", stk[top]);
            onStack[stk[top--]]=0;
        }
        printf("%d\n", stk[top]);
        onStack[stk[top--]]=0;
    }
}

void tarjanSCC() {
    int i;
    for(i=0;i<V;i++) disc[i]=low[i]=-1;
    for(i=0;i<V;i++)
        if(disc[i]==-1) tarjanUtil(i);
}
Time Complexity: O(V + E) — but with adjacency matrix it's O(V²) due to row scanning. Space: O(V).
🔃

Kosaraju's Algorithm

Advanced

Theory

Kosaraju's algorithm also finds SCCs in two DFS passes: (1) Run DFS on original graph, push vertices in finish order to a stack. (2) Transpose the graph (reverse all edges). (3) Run DFS on transposed graph in stack pop order — each DFS call finds one SCC.

1

DFS + Finish Order

Run full DFS. Push each vertex onto a stack when finished.

2

Transpose Graph

Reverse all edges: if adj[i][j]=1, set trans[j][i]=1.

3

DFS on Transposed

Pop vertices from stack. For each unvisited: run DFS on transposed graph → that's one SCC.

🌍 Real-life Analogy

Finding groups of cities where you can drive forward and reverse your route. Two passes: once going forward, once going backward on the same roads.

kosaraju.c
int finStack[MAX], fTop=-1;

void dfs1(int v, int vis[]) {
    int i; vis[v]=1;
    for(i=0;i<V;i++)
        if(adj[v][i]&&!vis[i]) dfs1(i, vis);
    finStack[++fTop]=v;  /* push on finish */
}

void dfs2(int v, int vis[], int g[][MAX]) {
    int i; vis[v]=1;
    printf("%d ", v);
    for(i=0;i<V;i++)
        if(g[v][i]&&!vis[i]) dfs2(i, vis, g);
}

void kosarajuSCC() {
    int vis[MAX]={0}, i, j;
    for(i=0;i<V;i++)
        if(!vis[i]) dfs1(i, vis);

    /* Transpose */
    int tr[MAX][MAX]={0};
    for(i=0;i<V;i++) for(j=0;j<V;j++)
        tr[j][i]=adj[i][j];

    int vis2[MAX]={0};
    while(fTop!=-1) {
        int v=finStack[fTop--];
        if(!vis2[v]) {
            printf("SCC: ");
            dfs2(v, vis2, tr);
            printf("\n");
        }
    }
}
Difference from Tarjan: Kosaraju uses 2 DFS passes and graph transposition. Tarjan uses 1 DFS with a stack. Both find SCCs in O(V+E) — O(V²) with adjacency matrix.

Floyd's Cycle Detection (Tortoise & Hare)

Advanced

Theory

Floyd's algorithm detects cycles in a sequence/linked list using two pointers: a slow pointer (moves 1 step) and a fast pointer (moves 2 steps). If they ever meet, there's a cycle. In a graph, you simulate this on successor sequences. For the general graph, DFS is more appropriate — Floyd's is classically for linked structures.

1

Initialize

slow = start, fast = start.

2

Move pointers

slow = next(slow); fast = next(next(fast)).

3

Check meeting

If slow == fast: cycle detected.

4

Find cycle start

Reset slow to head. Move both 1 step. Meeting point = cycle start.

🌍 Real-life Analogy

Two runners on a circular track: a slow runner (1 lap/hr) and a fast runner (2 laps/hr). If the track is a loop, the fast runner will eventually lap the slow runner and they'll be at the same point!

floyd_cycle.c
/* Floyd's on graph: each vertex has one
   successor (first neighbor with edge).
   Works best on functional graphs.        */

int nextVertex(int v) {
    int i;
    for(i=0;i<V;i++)
        if(adj[v][i]) return i;
    return -1;  /* no successor */
}

int floydCycleDetect(int start) {
    int slow=start, fast=start;

    do {
        slow = nextVertex(slow);
        fast = nextVertex(nextVertex(fast));
        if(slow==-1||fast==-1) {
            printf("No cycle!\n");
            return 0;
        }
    } while(slow!=fast);

    printf("Cycle detected!\n");

    /* Find cycle start */
    slow=start;
    while(slow!=fast){
        slow=nextVertex(slow);
        fast=nextVertex(fast);
    }
    printf("Cycle start: %d\n", slow);
    return 1;
}

Graph Coloring

Advanced

Theory

Assign colors to vertices such that no two adjacent vertices share the same color. The minimum number of colors needed is the chromatic number χ(G). A greedy coloring assigns the smallest available color to each vertex. This may not always give the optimal solution.

Applications: Register allocation in compilers, scheduling problems, map coloring (4-color theorem), frequency assignment in wireless networks.

#define NO_COLOR -1

void greedyColoring() {
    int color[MAX], i, j;
    int usedColor[MAX];

    /* Initialize all vertices to no color */
    for(i=0;i<V;i++) color[i]=NO_COLOR;

    color[0] = 0;  /* First vertex gets color 0 */

    for(i=1;i<V;i++) {
        /* Mark colors used by neighbors */
        for(j=0;j<V;j++) usedColor[j]=0;
        for(j=0;j<V;j++)
            if(adj[i][j] && color[j]!=NO_COLOR)
                usedColor[color[j]]=1;

        /* Find first available color */
        int c=0;
        while(usedColor[c]) c++;
        color[i]=c;
    }

    printf("Vertex Colors:\n");
    for(i=0;i<V;i++)
        printf("Vertex %d -> Color %d\n",i,color[i]);
}
