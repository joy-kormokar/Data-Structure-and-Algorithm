What is an Unweighted Graph? Beginner

Understanding graphs, nodes, edges, and the concept of "unweighted" connections.

📖Theory

A graph is a collection of vertices (nodes) connected by edges (links). In an unweighted graph, every edge has the same "cost" — there is no weight or distance attached to any connection. You either have an edge between two vertices or you don't.

  • Vertex (Node) — A point/entity in the graph
  • Edge — A connection between two vertices
  • Undirected — Edge A↔B means you can go both ways
  • Directed — Edge A→B means one-way only
  • Unweighted — All edges are equal (no numbers on edges)

🏙️Real-Life Analogy

Think of Facebook Friends. Each person is a vertex. If two people are friends, there's an edge between them. You don't say "I'm 5-friends with you and 3-friends with someone else" — friendship is simply yes or no. That's exactly an unweighted, undirected graph.
Other examples: road maps (cities = vertices, roads = edges), internet (computers = vertices, cables = edges), social networks, city bus routes.

Adjacency Matrix Representation Beginner

Storing a graph as a 2D array — the foundation for all operations.

📖Theory

An adjacency matrix is a 2D array adj[N][N] where N is the number of vertices.

  • adj[i][j] = 1 → there is an edge from vertex i to vertex j
  • adj[i][j] = 0 → no edge between i and j
  • For undirected graphs: adj[i][j] == adj[j][i] (symmetric)
  • Diagonal is always 0 (no self-loops in simple graphs)
Space Complexity: O(V²) — uses V×V memory regardless of edges.

/* ─── Unweighted Graph using Adjacency Matrix ─── */
#include <stdio.h>
#include <string.h>

#define MAX 10   /* Maximum number of vertices */

/* ── Graph structure ── */
struct Graph {
    int adj[MAX][MAX];  /* adjacency matrix: 0 or 1 */
    int numVertices;    /* current vertex count     */
};

/* ── Initialize / create an empty graph ── */
void initGraph(struct Graph *g) {
    g->numVertices = 0;
    memset(g->adj, 0, sizeof(g->adj));  /* fill all with 0 */
}

int main() {
    struct Graph g;
    initGraph(&g);
    printf("Graph initialized. Vertices: %d\n", g.numVertices);
    return 0;
}

🏙️Real-Life Use Case

Campus LAN Network: Every computer in a campus lab is a vertex. Ethernet cables between computers are edges. An adjacency matrix tells the network administrator instantly: "Is computer #3 directly connected to computer #7?" — just look at adj[3][7]. If it's 1, yes. If 0, no cable exists.

Insert Vertex Beginner

Adding a new node to the graph.

📖Theory

Inserting a vertex means adding a new row and column to the adjacency matrix. Since the new vertex has no edges yet, all entries in its row and column are set to 0. We simply increment the vertex counter.

  • New vertex gets index = numVertices
  • Time Complexity: O(1)
  • All new entries initialized to 0 (no connections)

🎮Visualization

Click Insert Vertex to see the matrix expand.

💻C Code

insert_vertex.cC Language
/* Insert a new vertex into the graph */
void insertVertex(struct Graph *g) {
    if (g->numVertices >= MAX) {
        printf("Graph is full! Cannot insert.\n");
        return;
    }
    int v = g->numVertices;  /* new vertex index */

    /* Set entire new row to 0 */
    int i;
    for (i = 0; i < MAX; i++)
        g->adj[v][i] = 0;

    /* Set entire new column to 0 */
    for (i = 0; i < MAX; i++)
        g->adj[i][v] = 0;

    g->numVertices++;  /* increment count */
    printf("Vertex %d inserted.\n", v);
}

🏙️Analogy

Adding a new employee to a company network: A new person joins the company. They get their own row and column in the "who-knows-who" table. Initially, they know nobody — all zeros. Later we add their connections (edges).

Delete Vertex Intermediate

Removing a vertex and all its associated edges.

📖Theory

Deleting a vertex requires removing its row and column from the matrix. In our approach, we shift all rows and columns after the deleted vertex one position to the left/up, then decrement the vertex count.

  • All edges connected to this vertex are automatically removed
  • Remaining vertices get re-indexed
  • Time Complexity: O(V²) due to shifting

🎮Visualization

Graph has 4 vertices (0-3) with some edges. Try deleting vertex 1.

💻C Code

delete_vertex.cC Language
/* Delete vertex 'v' and all its edges */
void deleteVertex(struct Graph *g, int v) {
    if (v < 0 || v >= g->numVertices) {
        printf("Invalid vertex.\n");
        return;
    }
    int i, j;
    /* Shift rows up: remove row 'v' */
    for (i = v; i < g->numVertices - 1; i++)
        for (j = 0; j < g->numVertices; j++)
            g->adj[i][j] = g->adj[i+1][j];

    /* Shift columns left: remove column 'v' */
    for (j = v; j < g->numVertices - 1; j++)
        for (i = 0; i < g->numVertices; i++)
            g->adj[i][j] = g->adj[i][j+1];

    g->numVertices--;
    printf("Vertex %d deleted.\n", v);
}

🏙️Analogy

Removing a city from a map: If a city is destroyed, you remove it from the map and remove all roads that connected to it. All other cities get re-numbered. That's vertex deletion.

Search & Update Vertex Beginner

Finding a vertex and changing its properties.

📖Search Vertex — Theory

In an adjacency matrix, vertex indices are implicit (0 to N-1). "Searching" a vertex simply checks whether that index exists within the valid range.

  • Valid if: 0 ≤ v < numVertices
  • Time Complexity: O(1)

📖Update Vertex — Theory

Updating a vertex usually means updating its label (if stored separately) or redistributing all its edges. In a pure adjacency matrix, vertices don't carry data — only connections matter. We can store a separate label[] array.

  • Update label: label[v] = newName
  • Time Complexity: O(1)

💻C Code — Search & Update

search_update_vertex.cC Language
/* ── Search: check if vertex 'v' exists ── */
int searchVertex(struct Graph *g, int v) {
    if (v >= 0 && v < g->numVertices) {
        printf("Vertex %d exists.\n", v);
        return 1;  /* found */
    }
    printf("Vertex %d not found.\n", v);
    return 0;   /* not found */
}

/* ── Update: rename vertex label ── */
char labels[MAX][20];  /* global label array */

void updateVertex(struct Graph *g, int v, char *newLabel) {
    if (!searchVertex(g, v)) return;
    strcpy(labels[v], newLabel);
    printf("Vertex %d label updated to '%s'.\n", v, newLabel);
}

🏙️Analogy

Phone directory search: You want to find "Person #5" in a contact list. If the list has 10 people (0–9), person #5 definitely exists. If you ask for person #15 and there are only 10, it's not found. Updating is like correcting a typo in their name.

Insert Edge Beginner

Creating a connection between two vertices.

📖Theory

To add an edge between vertex u and vertex v:

  • Set adj[u][v] = 1
  • For undirected graphs, also set adj[v][u] = 1
  • For directed graphs, only adj[u][v] = 1
  • Time Complexity: O(1)
  • First validate that both u and v are valid vertex indices

🎮Visualization

4 vertices (0–3), no edges yet. Try inserting edge 0-2.

💻C Code

insert_edge.cC Language
/* Insert undirected edge between u and v */
void insertEdge(struct Graph *g, int u, int v) {
    if (u < 0 || u >= g->numVertices ||
        v < 0 || v >= g->numVertices) {
        printf("Invalid vertices.\n");
        return;
    }
    if (u == v) {
        printf("No self-loops allowed.\n");
        return;
    }
    g->adj[u][v] = 1;  /* u → v */
    g->adj[v][u] = 1;  /* v → u (undirected) */
    printf("Edge (%d, %d) inserted.\n", u, v);
}

/* For DIRECTED edge (one-way): */
void insertDirectedEdge(struct Graph *g, int u, int v) {
    g->adj[u][v] = 1;   /* only u → v */
    printf("Directed edge (%d → %d) inserted.\n", u, v);
}

🏙️Analogy

Building a bridge: Two islands (vertices) are connected by a new bridge (edge). You mark both sides: "Island A can reach Island B" and "Island B can reach Island A." That's an undirected edge insertion — two cells become 1.

Delete Edge Beginner

Removing a connection between two vertices.

📖Theory

To remove an edge between vertex u and vertex v:

  • Set adj[u][v] = 0
  • For undirected: also set adj[v][u] = 0
  • First check the edge actually exists
  • Time Complexity: O(1)

💻C Code

delete_edge.cC Language
/* Delete undirected edge between u and v */
void deleteEdge(struct Graph *g, int u, int v) {
    if (u < 0 || u >= g->numVertices ||
        v < 0 || v >= g->numVertices) {
        printf("Invalid vertices.\n");
        return;
    }
    if (g->adj[u][v] == 0) {
        printf("Edge (%d, %d) does not exist.\n", u, v);
        return;
    }
    g->adj[u][v] = 0;
    g->adj[v][u] = 0;
    printf("Edge (%d, %d) deleted.\n", u, v);
}

🏙️Analogy

Closing a road: When a road between two cities is closed due to repairs, you update your map: the connection no longer exists. Both directions become 0 in the matrix.

Search & Count Edges Beginner

Checking if an edge exists and counting all edges.

📖Search Edge

Check adj[u][v]. If it's 1, the edge exists. This is a direct O(1) lookup — the biggest advantage of adjacency matrix.

📖Count Edges

Loop through the upper triangle of the matrix and count cells with value 1. For undirected graphs, each edge appears twice (adj[u][v] and adj[v][u]), so divide by 2.

  • Time Complexity: O(V²)

💻C Code

search_count_edges.cC Language
/* ── Search edge ── */
int searchEdge(struct Graph *g, int u, int v) {
    if (u < 0 || u >= g->numVertices ||
        v < 0 || v >= g->numVertices)
        return -1;  /* invalid */

    if (g->adj[u][v] == 1) {
        printf("Edge (%d, %d) EXISTS.\n", u, v);
        return 1;
    }
    printf("Edge (%d, %d) does NOT exist.\n", u, v);
    return 0;
}

/* ── Count all edges (undirected) ── */
int countEdges(struct Graph *g) {
    int count = 0, i, j;
    for (i = 0; i < g->numVertices; i++)
        for (j = i + 1; j < g->numVertices; j++)
            if (g->adj[i][j] == 1)
                count++;
    printf("Total edges: %d\n", count);
    return count;
}

Breadth First Search (BFS) Intermediate

Exploring a graph layer by layer using a queue.

📖Theory

BFS starts at a source vertex and explores all neighbors before going deeper. It uses a Queue (FIFO) to track which vertex to visit next.

  • Queue Initialization — create empty queue, enqueue source
  • Visit Node — dequeue, print vertex
  • Mark Visited — visited[v] = 1 prevents re-visiting
  • Traverse Adjacent — enqueue all unvisited neighbors
  • Time: O(V²) with adjacency matrix
  • Finds shortest path in unweighted graphs

🏙️Analogy

Spreading News in a School: You tell your friends first (level 1). They each tell their friends (level 2). This ripple continues outward. BFS visits nodes exactly this way — level by level, nearest first.
💡 BFS is used in GPS navigation (shortest route), social networks (friend recommendations), and web crawlers.

💻C Code — BFS

bfs.cC Language
/* ─── BFS using a simple array-based queue ─── */
#define MAX 10

int queue[MAX], front = 0, rear = -1;
int visited[MAX];

void enqueue(int v)  { queue[++rear] = v; }
int  dequeue()       { return queue[front++]; }
int  isEmpty()       { return front > rear; }

void BFS(struct Graph *g, int start) {
    int i;
    /* Step 1: Initialize */
    for (i = 0; i < g->numVertices; i++)
        visited[i] = 0;
    front = 0; rear = -1;

    /* Step 2: Mark start and enqueue */
    visited[start] = 1;
    enqueue(start);

    printf("BFS from %d: ", start);

    while (!isEmpty()) {
        /* Step 3: Dequeue and visit */
        int v = dequeue();
        printf("%d  ", v);

        /* Step 4: Traverse adjacent nodes */
        for (i = 0; i < g->numVertices; i++) {
            if (g->adj[v][i] == 1 && !visited[i]) {
                visited[i] = 1;  /* mark visited */
                enqueue(i);
            }
        }
    }
    printf("\n");
}

Depth First Search (DFS) Intermediate

Exploring as deep as possible before backtracking, using recursion or a stack.

📖Theory

DFS dives as deep as possible along each branch before backtracking. It can be implemented recursively (using the call stack) or iteratively (using an explicit stack).

  • Recursive DFS — clean, uses system stack
  • Stack DFS — iterative version
  • Visit Node — mark and print on first visit
  • Backtracking — return when no unvisited neighbors remain
  • Time: O(V²) with adjacency matrix

🏙️Analogy

Exploring a Maze: You keep going deeper into one corridor until you hit a dead end, then you backtrack and try another path. DFS explores a graph the same way — deep first, backtrack when stuck.
💡 DFS is used in puzzle solving, topological sort, cycle detection, and finding connected components.

dfs.cC Language
int visited[MAX];

/* ── Recursive DFS ── */
void DFS_recursive(struct Graph *g, int v) {
    visited[v] = 1;
    printf("%d  ", v);  /* Visit node */
    int i;
    for (i = 0; i < g->numVertices; i++) {
        if (g->adj[v][i] == 1 && !visited[i])
            DFS_recursive(g, i);  /* go deeper */
    }
    /* Backtracking happens here automatically */
}

/* ── Iterative DFS using explicit stack ── */
void DFS_stack(struct Graph *g, int start) {
    int stack[MAX], top = -1;
    int i, v;
    for (i = 0; i < g->numVertices; i++) visited[i] = 0;

    stack[++top] = start;  /* push start */
    printf("DFS from %d: ", start);

    while (top >= 0) {
        v = stack[top--];  /* pop */
        if (visited[v]) continue;
        visited[v] = 1;
        printf("%d  ", v);
        /* Push unvisited neighbors (reverse order) */
        for (i = g->numVertices - 1; i >= 0; i--)
            if (g->adj[v][i] && !visited[i])
                stack[++top] = i;
    }
    printf("\n");
}

Path Operations Intermediate

Find Path · Shortest Path · All Paths · Reachability · Path Length

📖Find Path

Determine if any path exists from source s to destination d. Use DFS/BFS from s; if d gets visited, a path exists.

📖Shortest Path

Use BFS — BFS naturally finds the shortest path (fewest edges) in an unweighted graph. Track parent array to reconstruct the path.

📖All Paths

Use DFS with backtracking. Explore all routes from s to d, recording each path. Mark vertices as visited when entering, unmark when backtracking.

📖Reachability & Path Length

Reachability: From vertex s, can you reach vertex d? — BFS/DFS answer this.

Path Length: Count edges in the path. BFS gives the minimum; record level/distance of each vertex.

💻C Code — Shortest Path (BFS) & All Paths (DFS)

path_operations.cC Language
/* ─── Shortest Path using BFS ─── */
void shortestPath(struct Graph *g, int src, int dest) {
    int visited[MAX] = {0};
    int parent[MAX], dist[MAX];
    int queue[MAX], front = 0, rear = -1;
    int i, v;

    for (i = 0; i < MAX; i++) { parent[i] = -1; dist[i] = -1; }

    visited[src] = 1; dist[src] = 0;
    queue[++rear] = src;

    while (front <= rear) {
        v = queue[front++];
        for (i = 0; i < g->numVertices; i++) {
            if (g->adj[v][i] && !visited[i]) {
                visited[i] = 1;
                parent[i] = v;
                dist[i] = dist[v] + 1;
                queue[++rear] = i;
            }
        }
    }

    if (dist[dest] == -1) {
        printf("No path from %d to %d\n", src, dest);
        return;
    }
    /* Reconstruct path */
    int path[MAX], len = 0, cur = dest;
    while (cur != -1) { path[len++] = cur; cur = parent[cur]; }
    printf("Shortest path (%d edges): ", dist[dest]);
    for (i = len - 1; i >= 0; i--) printf("%d ", path[i]);
    printf("\n");
}

/* ─── All Paths using DFS + Backtracking ─── */
int pathArr[MAX], pathLen = 0;
int vis2[MAX];

void allPaths(struct Graph *g, int src, int dest) {
    vis2[src] = 1;
    pathArr[pathLen++] = src;

    if (src == dest) {
        int i;
        for (i = 0; i < pathLen; i++) printf("%d ", pathArr[i]);
        printf("\n");
    } else {
        int i;
        for (i = 0; i < g->numVertices; i++)
            if (g->adj[src][i] && !vis2[i])
                allPaths(g, i, dest);
    }
    /* Backtrack */
    pathLen--;
    vis2[src] = 0;
}

Connectivity Operations Intermediate

Connected graph · Connected components · Isolated vertices · Reachable vertices

📖Theory

  • Connected Graph — Every vertex can reach every other vertex. Run BFS/DFS from any vertex; if all are visited, it's connected.
  • Connected Components — In a disconnected graph, groups of vertices that can reach each other. Run DFS repeatedly from unvisited vertices.
  • Isolated Vertex — A vertex with degree 0 (no edges). Check if entire row is all zeros.
  • Reachable Vertices — From vertex s, which vertices can be reached? — BFS/DFS from s gives all reachable vertices.

connectivity.cC Language
/* ── Helper: DFS to mark reachable vertices ── */
void dfsUtil(struct Graph *g, int v, int vis[]) {
    vis[v] = 1;
    int i;
    for (i = 0; i < g->numVertices; i++)
        if (g->adj[v][i] && !vis[i])
            dfsUtil(g, i, vis);
}

/* ── Check if graph is connected ── */
int isConnected(struct Graph *g) {
    int vis[MAX] = {0}, i;
    dfsUtil(g, 0, vis);
    for (i = 0; i < g->numVertices; i++)
        if (!vis[i]) return 0;  /* some vertex unreachable */
    return 1;
}

/* ── Find all connected components ── */
void findComponents(struct Graph *g) {
    int vis[MAX] = {0}, i, comp = 0;
    for (i = 0; i < g->numVertices; i++) {
        if (!vis[i]) {
            comp++;
            printf("Component %d: ", comp);
            dfsUtil(g, i, vis);
            /* print visited in this run */
        }
    }
    printf("Total components: %d\n", comp);
}

/* ── Detect isolated vertex (degree 0) ── */
void detectIsolated(struct Graph *g) {
    int i, j, deg;
    for (i = 0; i < g->numVertices; i++) {
        deg = 0;
        for (j = 0; j < g->numVertices; j++)
            deg += g->adj[i][j];
        if (deg == 0)
            printf("Vertex %d is isolated.\n", i);
    }
}

/* ── Get all vertices reachable from src ── */
void reachableVertices(struct Graph *g, int src) {
    int vis[MAX] = {0}, i;
    dfsUtil(g, src, vis);
    printf("Reachable from %d: ", src);
    for (i = 0; i < g->numVertices; i++)
        if (vis[i]) printf("%d ", i);
    printf("\n");
}

🏙️Analogy

Island Groups in an Archipelago: Each island is a vertex, each ferry route is an edge. A connected component is a group of islands you can reach from one another. An isolated island has no ferry routes — degree 0. Checking connectivity is like asking: "Can I travel from any island to any other island?"

Cycle Operations Advanced

Detect Cycle · Count Cycles · Check Cyclic Graph

📖Theory

A cycle is a path that starts and ends at the same vertex without repeating any edge. In an undirected graph, a cycle exists if DFS encounters an already-visited node that isn't the parent of the current node.

  • Detect Cycle — DFS with parent tracking: O(V²)
  • Check Cyclic — same as detect; returns true/false
  • Count Cycles — more complex; requires tracking back-edges
  • A graph with no cycles is called a tree (if connected) or a forest

Click "Detect Cycle" to check the current graph.

💻C Code — Cycle Detection

cycle_detection.cC Language
/* ─── Cycle Detection using DFS (Undirected Graph) ─── */
int visited[MAX];

/* Returns 1 if cycle found, 0 otherwise */
int dfsCycle(struct Graph *g, int v, int parent) {
    visited[v] = 1;
    int i;
    for (i = 0; i < g->numVertices; i++) {
        if (g->adj[v][i]) {
            if (!visited[i]) {
                /* Go deeper */
                if (dfsCycle(g, i, v)) return 1;
            } else if (i != parent) {
                /* Visited AND not parent → cycle found! */
                return 1;
            }
        }
    }
    return 0;
}

int detectCycle(struct Graph *g) {
    int i;
    for (i = 0; i < g->numVertices; i++) visited[i] = 0;
    for (i = 0; i < g->numVertices; i++)
        if (!visited[i])
            if (dfsCycle(g, i, -1)) {
                printf("Cycle DETECTED.\n");
                return 1;
            }
    printf("No cycle found.\n");
    return 0;
}

/* ── Count cycles: count back-edges during DFS ── */
int cycleCount = 0;

void countCyclesDFS(struct Graph *g, int v, int parent) {
    visited[v] = 1;
    int i;
    for (i = 0; i < g->numVertices; i++) {
        if (g->adj[v][i]) {
            if (!visited[i])
                countCyclesDFS(g, i, v);
            else if (i != parent && i < v)
                cycleCount++;  /* each back-edge = 1 independent cycle */
        }
    }
}

🏙️Analogy

Circular Roads: In a city's road network, if you can leave point A, travel through several streets, and return to point A without using the same road twice — that's a cycle. Cycle detection helps in deadlock detection in operating systems and finding loops in circuits.

Degree Operations Intermediate

Find Degree · Max Degree · Min Degree · Average Degree · Degree Distribution

📖Theory

The degree of a vertex is the number of edges connected to it. In the adjacency matrix, the degree of vertex v is the sum of row v (or column v — they're equal for undirected graphs).

  • degree(v) = sum of adj[v][0..N-1]
  • Max degree — highest degree among all vertices
  • Min degree — lowest degree
  • Avg degree — 2 × edges / vertices
  • Degree distribution — frequency of each degree value
Handshaking Lemma: Sum of all degrees = 2 × (number of edges). Every edge contributes to exactly 2 vertices' degrees.

Bipartite Operations Advanced

Bipartite Check · Graph Coloring Basics

📖Theory — Bipartite Check

A graph is bipartite if its vertices can be divided into two groups (Set A and Set B) such that every edge connects a vertex from Set A to Set B — no edges within the same set.

Algorithm: Use BFS with 2-coloring. Assign color 0 to source; assign color 1 to all its neighbors; continue. If you ever try to assign a color to a vertex that already has the same color as its neighbor → not bipartite.

  • Time Complexity: O(V²)
  • A graph with an odd-length cycle is NOT bipartite

📖Graph Coloring Basics

Graph coloring assigns colors to vertices so that no two adjacent vertices share the same color. The minimum number of colors needed is the chromatic number.

  • Bipartite graphs need only 2 colors
  • A greedy coloring approach assigns the first available color to each vertex
  • Used in: scheduling, register allocation, map coloring

/* ─── Bipartite Check using BFS 2-coloring ─── */
int isBipartite(struct Graph *g) {
    int color[MAX], queue[MAX], front = 0, rear = -1;
    int i;
    for (i = 0; i < MAX; i++) color[i] = -1;  /* uncolored */

    for (i = 0; i < g->numVertices; i++) {
        if (color[i] != -1) continue;  /* already processed */
        color[i] = 0;                   /* color source 0 */
        queue[++rear] = i;

        while (front <= rear) {
            int v = queue[front++];
            int j;
            for (j = 0; j < g->numVertices; j++) {
                if (g->adj[v][j]) {
                    if (color[j] == -1) {
                        color[j] = 1 - color[v]; /* flip color */
                        queue[++rear] = j;
                    } else if (color[j] == color[v]) {
                        printf("NOT bipartite.\n");
                        return 0;  /* same color = odd cycle */
                    }
                }
            }
        }
    }
    printf("Graph IS bipartite.\n");
    return 1;
}

/* ─── Greedy Graph Coloring ─── */
void greedyColor(struct Graph *g) {
    int result[MAX], avail[MAX];
    int i, j;
    for (i = 0; i < MAX; i++) result[i] = -1;
    result[0] = 0;  /* first vertex gets color 0 */

    for (i = 1; i < g->numVertices; i++) {
        for (j = 0; j < g->numVertices; j++) avail[j] = 1;
        /* Mark colors used by neighbors as unavailable */
        for (j = 0; j < g->numVertices; j++)
            if (g->adj[i][j] && result[j] != -1)
                avail[result[j]] = 0;
        /* Assign first available color */
        for (j = 0; j < g->numVertices; j++)
            if (avail[j]) { result[i] = j; break; }
    }
    printf("Vertex coloring:\n");
    for (i = 0; i < g->numVertices; i++)
        printf("  Vertex %d → Color %d\n", i, result[i]);
}

🏙️Analogy

Boys vs Girls at a Dance: In a bipartite graph, every dance (edge) is between a boy and a girl — never two boys or two girls. If you can perfectly split the group into boys and girls (two colors) such that all dances cross between groups, it's bipartite. If two boys are dancing, it's not bipartite.

Structural Operations Advanced

Size · Order · Density · Sparse/Dense · Complete Graph

📖Definitions

  • Graph Order = number of vertices (V)
  • Graph Size = number of edges (E)
  • Density = E / (V×(V-1)/2) — ratio of actual edges to max possible
  • Sparse — density close to 0; few edges
  • Dense — density close to 1; many edges
  • Complete Graph (Kₙ) — every pair of vertices is connected; E = V(V-1)/2
Threshold: Density < 0.5 → sparse. Density ≥ 0.5 → dense.

int graphSize(struct Graph *g) {

/* ── Density ── */
float graphDensity(struct Graph *g) {
    int V = g->numVertices;
    int maxEdges = V * (V - 1) / 2;
    if (maxEdges == 0) return 0.0;
    return (float)graphSize(g) / maxEdges;
}

    printf("Density: %.2f → ", d);
    if (d < 0.5) printf("SPARSE\n");
    else        printf("DENSE\n");
}

/* ── Complete Graph Check ── */
int isCompleteGraph(struct Graph *g) {
    int V = g->numVertices;
    int maxEdges = V * (V - 1) / 2;
    if (graphSize(g) == maxEdges) {
        printf("Complete graph K%d.\n", V);
        return 1;
    }
    printf("Not a complete graph.\n");
    return 0;
}

🏙️Analogy

A party: Graph order = number of guests. Graph size = number of handshakes. A complete graph = everyone shook hands with everyone else. A sparse graph = most people don't know each other. A dense graph = most people are friends.
