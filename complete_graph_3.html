Introduction to Complete Graphs

Beginner Start here to understand what a complete graph is and why it matters.

What is a Graph?
A graph is a collection of vertices (nodes/points) connected by edges (lines/links). Think of it as a network — like cities connected by roads.

A Complete Graph (written Kn) is a special graph where every vertex is directly connected to every other vertex. If there are n vertices, there are exactly n(n-1)/2 edges.
Real-life analogy: Imagine a group of friends where everyone knows everyone else. No one is left out — every pair has a direct friendship link. That's a complete graph!

#define MAX_V 10  // maximum vertices
#define MAX_E 100 // maximum edges

// Incidence Matrix: rows=vertices, cols=edges
int incMat[MAX_V][MAX_E]; // 1 if vertex i is in edge j
int vertexCount = 0;
int edgeCount   = 0;
char vertices[MAX_V]; // vertex labels: A,B,C...

void initGraph() {
    int i, j;
    for(i = 0; i < MAX_V; i++)
        for(j = 0; j < MAX_E; j++)
            incMat[i][j] = 0; // fill with zeros
    vertexCount = 0;
    edgeCount   = 0;
}

Incidence Matrix Representation

Representation How a graph is stored as a 2D table of 0s and 1s.

What is an Incidence Matrix?
An incidence matrix is a 2D array where:
• Rows = Vertices (A, B, C...)
• Columns = Edges (e1, e2, e3...)
• Cell value = 1 if that vertex is an endpoint of that edge, else 0

For a complete graph K4 (4 vertices), there are 6 edges. The matrix is 4×6.
Analogy: Think of a bus route timetable. Rows are bus stops (vertices), columns are bus routes (edges). A 1 means "this stop is on this route".

C Code: Build Incidence Matrix for Kn
// Build complete graph incidence matrix
void buildCompleteGraph(int n) {
    int i, j, e = 0;
    vertexCount = n;
    for(i = 0; i < n; i++) vertices[i] = 'A' + i;

    // Connect every pair of vertices
    for(i = 0; i < n; i++) {
        for(j = i+1; j < n; j++) {
            incMat[i][e] = 1; // vertex i is in edge e
            incMat[j][e] = 1; // vertex j is in edge e
            e++;
        }
    }
    edgeCount = e; // = n*(n-1)/2
}

Insert Vertex

Vertex Operation Add a new vertex to the graph.

Theory
Inserting a vertex means adding a new row to the incidence matrix. The new vertex has no edges yet, so all new entries are 0. In a complete graph, you'll then need to add edges connecting it to all existing vertices.
Analogy: Like adding a new city on a map — at first, it has no roads. You'll need to build roads (edges) to connect it.
Try It
Vertices: A, B, C (3 vertices, 3 edges for K3)
C Code
void insertVertex() {
    if(vertexCount >= MAX_V) {
        printf("Graph full!\n");
        return;
    }
    // Add new row (all zeros — no edges yet)
    int i;
    for(i = 0; i < edgeCount; i++)
        incMat[vertexCount][i] = 0;

    vertices[vertexCount] = 'A' + vertexCount;
    vertexCount++;
    printf("Inserted vertex %c\n", vertices[vertexCount-1]);
}

Delete Vertex

Vertex Operation Remove a vertex and all its edges from the graph.

Theory
Deleting a vertex requires:
1. Finding all edges that include this vertex (column has a 1 in that row).
2. Removing those entire columns (edges).
3. Shifting remaining rows up to fill the gap.
This is expensive for large matrices.
Analogy: Removing a city from a map — you must also close all roads that went to or from it.
Try It
Select a vertex to delete.
C Code
void deleteVertex(int vIdx) {
    int i, j, newE = 0;
    int tempMat[MAX_V][MAX_E];
    // Copy columns that don't involve vIdx
    for(j = 0; j < edgeCount; j++) {
        if(incMat[vIdx][j] == 0) { // keep this edge
            for(i = 0; i < vertexCount; i++)
                tempMat[i][newE] = incMat[i][j];
            newE++;
        }
    }
    // Shift rows: remove row vIdx
    for(i = vIdx; i < vertexCount-1; i++) {
        for(j = 0; j < newE; j++)
            incMat[i][j] = tempMat[i+1][j];
        vertices[i] = vertices[i+1];
    }
    vertexCount--;
    edgeCount = newE;
}

Insert Edge

Edge Operation Add an edge between two vertices.

Theory
To insert edge (u, v):
1. Check it doesn't already exist.
2. Add a new column to the matrix.
3. Set incMat[u][newCol] = 1 and incMat[v][newCol] = 1.
4. Increment edgeCount.
Analogy: Building a new road between two cities. Once built, both cities are connected by that road.
Try It
Select two vertices to connect them.
C Code
int edgeExists(int u, int v) {
    int j;
    for(j = 0; j < edgeCount; j++)
        if(incMat[u][j] == 1 && incMat[v][j] == 1)
            return 1; // edge exists
    return 0;
}

void insertEdge(int u, int v) {
    if(edgeExists(u, v)) {
        printf("Edge already exists!\n");
        return;
    }
    incMat[u][edgeCount] = 1; // mark vertex u
    incMat[v][edgeCount] = 1; // mark vertex v
    edgeCount++;
    printf("Edge (%c-%c) added\n", vertices[u], vertices[v]);
}

Delete Edge

Edge Operation Remove an edge between two vertices.

Find the column where both vertices have a 1. Remove that column by shifting all columns after it one position left.
Analogy: Closing a road between two cities. The cities still exist, but the direct route is gone.
Try It
Select two connected vertices to remove their edge.
C Code
void deleteEdge(int u, int v) {
    int i, j, col = -1;
    for(j = 0; j < edgeCount; j++)
        if(incMat[u][j]==1 && incMat[v][j]==1)
            col = j;
    if(col == -1) { printf("Edge not found!\n"); return; }

    // Shift columns left from col
    for(j = col; j < edgeCount-1; j++)
        for(i = 0; i < vertexCount; i++)
            incMat[i][j] = incMat[i][j+1];
    edgeCount--;
    printf("Edge (%c-%c) deleted\n", vertices[u], vertices[v]);
}

Update Edge

Edge Operation Replace an existing edge with a new one.

Update = Delete old edge + Insert new edge. Find the column of the old edge, zero it out, then set the new vertex pair in that same column.
Analogy: Rerouting a road — close the old path, open a new one between different cities.
Select old edge and new edge to replace.
C Code
void updateEdge(int uOld, int vOld, int uNew, int vNew) {
    int i, j, col = -1;
    for(j = 0; j < edgeCount; j++)
        if(incMat[uOld][j]==1 && incMat[vOld][j]==1)
            col = j;
    if(col == -1) { printf("Old edge not found!\n"); return; }

    // Clear old edge in that column
    for(i = 0; i < vertexCount; i++) incMat[i][col] = 0;
    // Set new edge
    incMat[uNew][col] = 1;
    incMat[vNew][col] = 1;
    printf("Edge updated!\n");
}

Auto Connect New Vertex

Edge Operation When a new vertex is added, automatically connect it to all existing ones.

Theory
To keep a complete graph complete after adding a vertex, we must add edges from the new vertex to every existing vertex. If there are already k vertices, we add k new edges.
Analogy: A new person joins a group where everyone knows everyone. To maintain the "complete" social network, they immediately befriend every existing member.
Try It
Start with K3 (A,B,C). Click to add new vertex and auto-connect.
C Code
void autoConnectNewVertex() {
    int i, newV = vertexCount;
    // Add vertex (row of zeros)
    for(i = 0; i < edgeCount; i++) incMat[newV][i] = 0;
    vertices[newV] = 'A' + newV;
    vertexCount++;

    // Connect new vertex to ALL existing vertices
    for(i = 0; i < newV; i++) {
        incMat[i][edgeCount]   = 1; // existing vertex
        incMat[newV][edgeCount] = 1; // new vertex
        edgeCount++;
    }
    printf("Added %c and connected to all\n", vertices[newV-1]);
}

Breadth First Search (BFS)

Traversal Visit vertices level by level, like ripples in a pond.

Theory
BFS uses a queue. Start at a vertex, visit all its neighbors, then their neighbors, and so on. It explores wide before going deep.

In the incidence matrix, neighbors of vertex u are found by: for each edge column where incMat[u][j]==1, find the other vertex in that column.
Analogy: Spreading news in a town — you tell your direct friends first (level 1), then they tell their friends (level 2), and so on outward.
Animate BFS
Select start vertex and click Run.
C Code: BFS with Queue
void BFS(int start) {
    int visited[MAX_V] = {0};
    int queue[MAX_V], front = 0, rear = 0;
    int i, j, u, v;

    queue[rear++] = start; // enqueue start
    visited[start] = 1;

    printf("BFS: ");
    while(front != rear) {       // queue not empty
        u = queue[front++];       // dequeue
        printf("%c ", vertices[u]);

        // Find all neighbors via incidence matrix
        for(j = 0; j < edgeCount; j++) {
            if(incMat[u][j] == 1) {
                for(i = 0; i < vertexCount; i++) {
                    if(i != u && incMat[i][j]==1 && !visited[i]) {
                        visited[i] = 1;
                        queue[rear++] = i;
                    }
                }
            }
        }
    }
}

Depth First Search (DFS)

Traversal Go as deep as possible before backtracking.

Theory
DFS uses a stack (or recursion). Follow one path as deep as possible, then backtrack to try other paths. Two approaches: Recursive DFS and Stack-based DFS.
Analogy: Exploring a maze — always go forward until you hit a dead end, then backtrack and try another route.
Animate DFS
Select start and click Run.
C Code: Recursive DFS
int visited[MAX_V];

void DFS_recursive(int u) {
    int i, j;
    visited[u] = 1;
    printf("%c ", vertices[u]);

    for(j = 0; j < edgeCount; j++) {
        if(incMat[u][j] == 1) {
            for(i = 0; i < vertexCount; i++) {
                if(i != u && incMat[i][j]==1 && !visited[i])
                    DFS_recursive(i); // go deeper
            }
        }
    }
} // Backtracking happens automatically on return

// Stack-based DFS
void DFS_stack(int start) {
    int stack[MAX_V], top=-1;
    int vis[MAX_V]={0}, i, j, u;
    stack[++top] = start;
    while(top >= 0) {
        u = stack[top--];
        if(vis[u]) continue;
        vis[u] = 1;
        printf("%c ", vertices[u]);
        for(j=0; j<edgeCount; j++)
          if(incMat[u][j]==1)
            for(i=0; i<vertexCount; i++)
              if(i!=u && incMat[i][j]==1 && !vis[i])
                stack[++top]=i;
    }
}

Check Complete Graph

Completeness Verify and validate full connectivity of a graph.

Theory
A graph with n vertices is complete if and only if it has exactly n(n-1)/2 edges AND every pair of vertices is connected.

Steps: Count edges → Compare with formula → Detect any missing edges → Validate full connectivity.
Analogy: In a complete social network, every person is friends with every other person. We check: is anyone missing a friendship?
Try It
Set n and click Check.
C Code
int checkCompleteGraph() {
    int required = vertexCount*(vertexCount-1)/2;
    if(edgeCount != required) {
        printf("Not complete! Has %d edges, needs %d\n",
               edgeCount, required);
        return 0;
    }
    // Verify every pair has exactly one shared column
    int i, k, j, found;
    for(i = 0; i < vertexCount; i++) {
        for(k = i+1; k < vertexCount; k++) {
            found = 0;
            for(j = 0; j < edgeCount; j++)
                if(incMat[i][j]==1 && incMat[k][j]==1)
                    found = 1;
            if(!found) {
                printf("Missing edge: %c-%c\n",
                       vertices[i], vertices[k]);
                return 0;
            }
        }
    }
    printf("Graph is complete (K%d)!\n", vertexCount);
    return 1;
}

Auto Complete Graph

Completeness Automatically add missing edges to make a graph complete.

Scan every pair of vertices. If they don't share an edge, add one. After this, the graph is guaranteed to be Kn.
Analogy: A social app that automatically adds friendship requests for anyone who doesn't know everyone yet — making the network fully connected.
Try It
Click "Remove Random Edges", then "Auto Complete!" to see it work.
C Code
void autoCompleteGraph() {
    int i, k, j, found;
    for(i = 0; i < vertexCount; i++) {
        for(k = i+1; k < vertexCount; k++) {
            found = 0;
            for(j = 0; j < edgeCount; j++)
                if(incMat[i][j]==1 && incMat[k][j]==1)
                    found = 1;
            if(!found) { // missing edge — add it!
                incMat[i][edgeCount] = 1;
                incMat[k][edgeCount] = 1;
                edgeCount++;
                printf("Added edge %c-%c\n",
                       vertices[i], vertices[k]);
            }
        }
    }
    printf("Graph is now complete!\n");
}

Find Path & All Paths

Path Operations Find if a path exists and enumerate all paths.

Theory
A path from vertex u to v is a sequence of vertices where each consecutive pair is connected by an edge.

Find Path: Use DFS from u; stop when v is reached.
All Paths: Use backtracking DFS — explore all routes without revisiting vertices.
Analogy: Finding all driving routes between two cities — some direct, some through other cities.
Select source and destination, then click.
C Code
int path[MAX_V], pathLen = 0;

void findAllPaths(int u, int dest, int vis[]) {
    vis[u] = 1;
    path[pathLen++] = u;

    if(u == dest) {
        int k;
        for(k = 0; k < pathLen; k++)
            printf("%c ", vertices[path[k]]);
        printf("\n");
    } else {
        // Try all neighbors
        int i, j;
        for(j = 0; j < edgeCount; j++) {
            if(incMat[u][j] == 1) {
                for(i = 0; i < vertexCount; i++)
                    if(i!=u && incMat[i][j]==1 && !vis[i])
                        findAllPaths(i, dest, vis);
            }
        }
    }
    // Backtrack
    pathLen--;
    vis[u] = 0;
}

Shortest Path

Path Operations Find the path with fewest hops using BFS.

In an unweighted complete graph, the shortest path between any two vertices is always 1 hop (direct edge). But BFS-based shortest path works for any graph.

BFS guarantees shortest path in unweighted graphs — each level adds one more hop. Track the parent of each vertex to reconstruct the path.
Analogy: Finding the fewest number of flights (with stopovers) between two airports. BFS finds the route with fewest connections.
Select vertices and click.
C Code
void shortestPath(int src, int dest) {
    int dist[MAX_V], parent[MAX_V];
    int visited[MAX_V], queue[MAX_V];
    int front=0, rear=0, i, j, u;
    for(i=0;i<vertexCount;i++){dist[i]=-1;parent[i]=-1;visited[i]=0;}

    dist[src]=0; visited[src]=1;
    queue[rear++]=src;
    while(front!=rear){
        u=queue[front++];
        if(u==dest) break;
        for(j=0;j<edgeCount;j++){
            if(incMat[u][j]==1){
                for(i=0;i<vertexCount;i++){
                    if(i!=u&&incMat[i][j]==1&&!visited[i]){
                        visited[i]=1;
                        dist[i]=dist[u]+1;
                        parent[i]=u;
                        queue[rear++]=i;
                    }
                }
            }
        }
    }
    // Reconstruct path by following parent[]
}

Hamiltonian Path Check

Path Operations A path that visits every vertex exactly once.

Theory
A Hamiltonian path visits every vertex exactly once. In a complete graph Kn, a Hamiltonian path always exists (any permutation of vertices works!). Finding one uses backtracking.
Analogy: A delivery driver who visits every city exactly once (no repeats). In a complete road network, this is always possible.
Click to find a Hamiltonian Path.
C Code
int hamPath[MAX_V], hamVis[MAX_V];

int isAdjacentIM(int u, int v) {
    int j;
    for(j=0;j<edgeCount;j++)
        if(incMat[u][j]==1 && incMat[v][j]==1) return 1;
    return 0;
}

int hamiltonianPath(int pos) {
    if(pos == vertexCount) return 1; // all visited!
    int v;
    for(v = 0; v < vertexCount; v++) {
        if(!hamVis[v] && isAdjacentIM(hamPath[pos-1], v)) {
            hamPath[pos] = v;
            hamVis[v] = 1;
            if(hamiltonianPath(pos+1)) return 1;
            hamVis[v] = 0; // backtrack
        }
    }
    return 0;
}

Connectivity Operations

Connectivity Check if a graph is connected, find reachable vertices, validate strong connectivity.

Theory
Connected graph: Every vertex can reach every other vertex.
Reachable vertices: All vertices reachable from a start vertex via DFS/BFS.
Strong connectivity: For directed graphs — every vertex reaches every other. Complete graphs are always strongly connected.
Analogy: Can you travel from any city to any other city? If yes — connected network. If some city has no roads — disconnected.
Click buttons to check connectivity.
C Code
int isConnected() {
    int vis[MAX_V] = {0};
    // DFS from vertex 0
    DFS_visit(0, vis);
    int i;
    for(i=0; i < vertexCount; i++)
        if(!vis[i]) return 0; // unreachable vertex!
    return 1; // all reachable
}

void reachableFrom(int start) {
    int vis[MAX_V] = {0};
    DFS_visit(start, vis);
    printf("Reachable from %c: ", vertices[start]);
    int i;
    for(i=0; i < vertexCount; i++)
        if(vis[i]) printf("%c ", vertices[i]);
}

Detect & Count Cycles

Cycle Operations Find and count cycles in the graph.

Theory
A cycle is a path that starts and ends at the same vertex.

Detect: During DFS, if we reach a visited vertex (that's not our parent), a cycle exists.
Count: For Kn, the number of distinct cycles is very large. We count simple cycles using backtracking.
Analogy: A cycle in a road network means you can drive a loop — start at city A, go through others, and return to A without repeating any road.
Click to detect or count cycles.
C Code
int cycleVis[MAX_V];

int detectCycleDFS(int u, int parent) {
    cycleVis[u] = 1;
    int i, j;
    for(j=0; j < edgeCount; j++) {
        if(incMat[u][j] == 1) {
            for(i=0; i < vertexCount; i++) {
                if(i!=u && incMat[i][j]==1) {
                    if(!cycleVis[i]) {
                        if(detectCycleDFS(i, u)) return 1;
                    } else if(i != parent) {
                        return 1; // cycle found!
                    }
                }
            }
        }
    }
    return 0;
}

Euler & Hamiltonian Cycle

Cycle Operations Special cycles that cover all edges or all vertices.

Theory
Euler Cycle: Visits every edge exactly once and returns to start. Exists if every vertex has even degree. In Kn, this works when n is odd.

Hamiltonian Cycle: Visits every vertex exactly once and returns to start. Always exists in Kn for n ≥ 3.
Euler: A mailman delivering to every street exactly once and returning home.
Hamiltonian: A salesperson visiting every city exactly once and returning home.
Click to check.
C Code
// Euler Cycle: check all vertices have even degree
int hasEulerCycle() {
    int i, j, degree;
    for(i=0; i < vertexCount; i++) {
        degree = 0;
        for(j=0; j < edgeCount; j++)
            degree += incMat[i][j];
        if(degree % 2 != 0) return 0; // odd degree!
    }
    return 1;
}

// Hamiltonian Cycle: Hamiltonian path + edge back to start
int hamiltonianCycle(int pos) {
    if(pos == vertexCount)
        return isAdjacentIM(hamPath[pos-1], hamPath[0]);
    int v;
    for(v=0; v < vertexCount; v++) {
        if(!hamVis[v] && isAdjacentIM(hamPath[pos-1], v)) {
            hamPath[pos] = v; hamVis[v] = 1;
            if(hamiltonianCycle(pos+1)) return 1;
            hamVis[v] = 0;
        }
    }
    return 0;
}

Degree Operations

Degree Find the degree of vertices — how many edges connect to each one.

Theory
Degree of a vertex = number of edges connected to it = count of 1s in its row.

In Kn, every vertex has degree n-1 (connected to all others).

In-degree / Out-degree apply to directed graphs. For undirected complete graphs, in-degree = out-degree = n-1.

Degree validation: In Kn, if any vertex has degree ≠ n-1, the graph is not complete.
Click to calculate degrees.
C Code
int getDegree(int v) {
    int j, deg = 0;
    for(j=0; j < edgeCount; j++)
        deg += incMat[v][j]; // count 1s in row v
    return deg;
}

int getMaxDegree() {
    int i, max = 0;
    for(i=0; i < vertexCount; i++)
        if(getDegree(i) > max) max = getDegree(i);
    return max;
}

int getMinDegree() {
    int i, min = edgeCount;
    for(i=0; i < vertexCount; i++)
        if(getDegree(i) < min) min = getDegree(i);
    return min;
}

// Degree Validation for K_n
int validateDegrees() {
    int i, expected = vertexCount - 1;
    for(i=0; i < vertexCount; i++)
        if(getDegree(i) != expected) return 0;
    return 1;
}

Structural Operations

Structural Analyze graph size, order, density, cliques, symmetry, and dense connectivity.

Theory
Graph Order: Number of vertices (|V| = n).
Graph Size: Number of edges (|E| = n(n-1)/2 for Kn).
Density: Ratio of actual to maximum possible edges = |E| / (n(n-1)/2). For Kn, density = 1.0 (maximum!).
Clique: A complete subgraph. In Kn, every subset of k vertices forms a clique.
Symmetry: Kn is vertex-transitive and edge-transitive — highly symmetric.

    int i, k;

        for(k=i+1; k<sz; k++)

// Bron-Kerbosch Algorithm
void bronKerbosch(int R[], int rSz,
                  int P[], int pSz,
                  int X[], int xSz) {
    if(pSz == 0 && xSz == 0) {
        int k;
        printf("Maximal Clique: ");
        for(k=0;k<rSz;k++) printf("%c ", vertices[R[k]]);
        printf("\n");
        return;
    }
    int v, i;
    for(v=0; v < pSz; v++) {
        // R ∪ {v}, P ∩ N(v), X ∩ N(v)
        int newR[MAX_V], newP[MAX_V], newX[MAX_V];
        int nr=rSz, np=0, nx=0;
        for(k=0;k<rSz;k++) newR[k]=R[k]; newR[nr++]=P[v];
        for(i=0;i<pSz;i++) if(isAdjacentIM(P[v],P[i])) newP[np++]=P[i];
        for(i=0;i<xSz;i++) if(isAdjacentIM(P[v],X[i])) newX[nx++]=X[i];
        bronKerbosch(newR,nr,newP,np,newX,nx);
    }
}

Graph Coloring & Traveling Salesman Basics

Advanced Color vertices so no two adjacent vertices share a color; find TSP tour.

Graph Coloring
Graph Coloring: Assign colors to vertices so no two adjacent vertices share a color. The minimum colors needed is the chromatic number χ(G).

For Kn, every vertex is adjacent to every other — so you need exactly n colors (χ(Kn) = n).
Analogy: Coloring a map so no two neighboring countries share the same color. In a complete graph, every region borders every other — so each needs its own unique color.
Traveling Salesman Basics (TSP)
TSP: Find the shortest Hamiltonian cycle (visit all vertices, return to start, minimize total cost). For unweighted Kn, all tours have the same length. With weights, brute-force tries all (n-1)! permutations.

void greedyColoring() {
    int color[MAX_V], used[MAX_V];
    int i, j, v;
    for(i=0; i < vertexCount; i++) color[i] = -1;
    color[0] = 0; // first vertex gets color 0

    for(v=1; v < vertexCount; v++) {
        for(j=0; j < vertexCount; j++) used[j] = 0;
        // Mark colors used by neighbors
        for(j=0; j < edgeCount; j++) {
            if(incMat[v][j] == 1)
                for(i=0; i < vertexCount; i++)
                    if(i!=v && incMat[i][j]==1 && color[i]!=-1)
                        used[color[i]] = 1;
        }
        // Assign smallest unused color
        for(j=0; used[j]; j++);
        color[v] = j;
    }
    for(v=0; v < vertexCount; v++)
        printf("Vertex %c -> Color %d\n", vertices[v], color[v]);
}
