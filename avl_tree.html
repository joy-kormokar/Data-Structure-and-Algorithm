01

Basic Operations

📚
Theory — What Is an AVL Tree?

An AVL Tree (Adelson-Velsky and Landis, 1962) is a self-balancing Binary Search Tree. In a plain BST, inserting sorted data creates a chain — O(n) search. AVL prevents this by enforcing a balance rule at every node.

The Golden Rule: At every node, the heights of left and right subtrees may differ by at most 1.

  • Node: Stores data, left pointer, right pointer, and height.
  • Height: Longest path (edges) from this node down to any leaf.
  • Balance Factor (BF): height(left) − height(right). Always −1, 0, or +1.
⏱ All operations — insert, delete, search — are guaranteed O(log n) because the AVL tree always stays balanced.

C Code — Node Structure & Basic Operations
// ============================================================
// AVL TREE — NODE STRUCTURE & BASIC OPERATIONS
// ============================================================
#include <stdio.h>
#include <stdlib.h>

// ── Node Structure ───────────────────────────────────────────
struct Node {
    int          data;   // The value stored
    struct Node* left;   // Pointer to left child
    struct Node* right;  // Pointer to right child
    int          height; // Height of this node
};

// ── Create a New Node ────────────────────────────────────────
struct Node* createNode(int data) {
    struct Node* n = (struct Node*)malloc(sizeof(struct Node));
    if (!n) { printf("Memory error!\n"); return NULL; }
    n->data = data; n->left = n->right = NULL; n->height = 1;
    return n;
}

// ── Initialize Empty AVL Tree ────────────────────────────────
struct Node* initAVLTree() { return NULL; }

// ── Check If Empty ───────────────────────────────────────────
int isEmpty(struct Node* root) { return root == NULL; }

// ── Helper: Height ───────────────────────────────────────────
int height(struct Node* n) { return n ? n->height : 0; }

// ── Helper: Max of two ints ──────────────────────────────────
int maxVal(int a, int b) { return a > b ? a : b; }

// ── Update Height ────────────────────────────────────────────
void updateHeight(struct Node* n) {
    if (n) n->height = 1 + maxVal(height(n->left), height(n->right));
}

int main() {
    struct Node* root = initAVLTree();
    printf("Empty: %d\n", isEmpty(root)); // 1
    struct Node* n = createNode(42);
    printf("Data=%d h=%d\n", n->data, n->height); // 42, 1
    return 0;
}
💡
Real-Life Analogy
📚 Think of a perfectly organized library! A regular BST is like a library where books pile onto one shelf — eventually a long unbalanced stack. An AVL tree is like a self-organizing library where the librarian rebalances shelves after every addition or removal. The height rule means "no shelf can be more than 1 book taller than its neighbor." You find any book in O(log n) time, always.

Insertion Operations

Insert NodeRecursive InsertionIterative InsertionInsert with BalancingUpdate Height After InsertCalculate Balance Factor
📚
Theory — How AVL Insertion Works

Insertion follows 4 steps:

  • Step 1: Insert exactly like a BST — smaller → left, larger → right.
  • Step 2: On the way back up (recursion unwinds), update each ancestor's height.
  • Step 3: Check balance factor: BF = height(left) − height(right).
  • Step 4: If |BF| > 1, apply the appropriate rotation.
✅ Only ONE rotation is ever needed after a single insertion. After the rotation, the entire tree is balanced again.

C Code — Recursive Insertion with Balancing
// ── Get Balance Factor ───────────────────────────────────────
int getBalance(struct Node* n) {
    return n ? height(n->left) - height(n->right) : 0;
}

// ── Right Rotate (LL fix) ────────────────────────────────────
struct Node* rightRotate(struct Node* y) {
    struct Node* x = y->left, *T2 = x->right;
    x->right = y; y->left = T2;
    updateHeight(y); updateHeight(x);
    return x;
}

// ── Left Rotate (RR fix) ─────────────────────────────────────
struct Node* leftRotate(struct Node* x) {
    struct Node* y = x->right, *T2 = y->left;
    y->left = x; x->right = T2;
    updateHeight(x); updateHeight(y);
    return y;
}

// ── Recursive Insert with AVL Balancing ──────────────────────
struct Node* insert(struct Node* root, int data) {
    // Step 1: Standard BST insert
    if (!root) return createNode(data);
    if (data < root->data)      root->left  = insert(root->left,  data);
    else if (data > root->data) root->right = insert(root->right, data);
    else return root; // Duplicate ignored

    // Step 2: Update height
    updateHeight(root);

    // Step 3 & 4: Check and fix balance
    int bf = getBalance(root);

    // LL Case → Right Rotation
    if (bf > 1 && data < root->left->data)  return rightRotate(root);
    // RR Case → Left Rotation
    if (bf < -1 && data > root->right->data) return leftRotate(root);
    // LR Case → Left then Right
    if (bf > 1 && data > root->left->data) {
        root->left = leftRotate(root->left);
        return rightRotate(root);
    }
    // RL Case → Right then Left
    if (bf < -1 && data < root->right->data) {
        root->right = rightRotate(root->right);
        return leftRotate(root);
    }
    return root;
}
💡
Real-Life Analogy
📦 Like stacking books on a table: You add one book at a time (BST insert). After each addition, you check if the tower leans too much (balance factor). If it does, you rearrange a couple of books (rotation) to restore balance. The alphabetical order of books (BST property) is always preserved during the rearrangement.

Rotation Operations

Left Rotation (RR)Right Rotation (LL)Left-Right (LR)Right-Left (RL)
📚
Theory — The Heart of AVL Trees

When |BF| > 1, we perform a rotation. There are 4 cases:

  • LL: New node in left child's left subtree → single Right Rotation.
  • RR: New node in right child's right subtree → single Left Rotation.
  • LR: New node in left child's right subtree → Left then Right Rotation.
  • RL: New node in right child's left subtree → Right then Left Rotation.
⚠️ Rotations only rearrange pointers — the BST ordering (in-order sequence) is NEVER changed!

C Code — All 4 Rotation Functions
// ── Right Rotation (fixes LL) ─────────────────────────────────
//       y                 x
//      / \               / \
//     x   C   ──────►  A   y
//    / \                   / \
//   A   B                 B   C
struct Node* rightRotate(struct Node* y) {
    struct Node* x = y->left, *B = x->right;
    x->right = y; y->left = B;       // Rotate
    updateHeight(y); updateHeight(x); // y first! x becomes new root
    return x;
}

// ── Left Rotation (fixes RR) ──────────────────────────────────
//     x                   y
//    / \                 / \
//   A   y   ──────►    x   C
//      / \             / \
//     B   C           A   B
struct Node* leftRotate(struct Node* x) {
    struct Node* y = x->right, *B = y->left;
    y->left = x; x->right = B;
    updateHeight(x); updateHeight(y);
    return y;
}

Balancing Operations

Balance TreeRebalance NodeCalculate Balance FactorUpdate HeightDetect Unbalanced Node
📚
Theory — The Balancing Mechanism
  • Balance Factor: BF = height(left) − height(right). Tells which side is heavier.
  • Update Height: height = 1 + max(left height, right height). Done bottom-up.
  • Detect Imbalance: If |BF| > 1, rotation is needed.
  • Rebalance: Choose rotation based on BF and child's BF.
✅ One insertion → at most one rotation (single or double). Deletion → possibly multiple rotations up the path.

C Code — Rebalance Function
// ── Detect Imbalance ─────────────────────────────────────────
int isUnbalanced(struct Node* n) {
    int bf = getBalance(n);
    return bf > 1 || bf < -1;
}
int isLeftHeavy(struct Node* n)  { return getBalance(n) > 0; }
int isRightHeavy(struct Node* n) { return getBalance(n) < 0; }

// ── Universal Rebalance (called after insert/delete) ──────────
struct Node* rebalance(struct Node* node) {
    updateHeight(node);
    int bf = getBalance(node);

    if (bf > 1) {  // Left heavy
        if (getBalance(node->left) < 0)        // LR case
            node->left = leftRotate(node->left);
        return rightRotate(node);              // LL case
    }
    if (bf < -1) { // Right heavy
        if (getBalance(node->right) > 0)       // RL case
            node->right = rightRotate(node->right);
        return leftRotate(node);               // RR case
    }
    return node; // Already balanced
}

// ── Find First Unbalanced Node ───────────────────────────────
struct Node* findUnbalanced(struct Node* root) {
    if (!root) return NULL;
    struct Node* l = findUnbalanced(root->left);
    struct Node* r = findUnbalanced(root->right);
    if (l) return l;
    if (r) return r;
    return isUnbalanced(root) ? root : NULL;
}
💡
Real-Life Analogy
⚖️ Like a tightrope walker's balance pole: The BF is which side of the pole is heavier. If it tilts beyond ±1, the walker shifts weight (rotation). Height update is sensing total weight distribution. The walker (AVL tree) never stays unbalanced for long.

Deletion Operations

Delete NodeDelete LeafDelete One ChildDelete Two ChildrenRebalance After DeleteRotation After Delete
📚
Theory — Deletion in 3 Cases
  • Case 1 — Leaf: Node has no children → simply remove it (free memory).
  • Case 2 — One Child: Replace node with its only child.
  • Case 3 — Two Children: Find inorder successor (min of right subtree), copy its value here, then delete the successor recursively.

After deletion, rebalance going back up — may need multiple rotations.

⚠️ Deletion can require O(log n) rotations going up the path to root — unlike insertion which needs at most one.

C Code — Complete Deletion
// ── Find Minimum (for inorder successor) ─────────────────────
struct Node* findMin(struct Node* n) {
    while (n->left) n = n->left;
    return n;
}

// ── Delete Node (handles all 3 cases + rebalancing) ───────────
struct Node* deleteNode(struct Node* root, int data) {
    if (!root) return NULL;

    if      (data < root->data) root->left  = deleteNode(root->left,  data);
    else if (data > root->data) root->right = deleteNode(root->right, data);
    else {
        // NODE FOUND — handle 3 cases:
        if (!root->left || !root->right) {  // Case 1 or 2
            struct Node* child = root->left ? root->left : root->right;
            if (!child) { free(root); return NULL; }  // Case 1: leaf
            *root = *child; free(child);              // Case 2: one child
        } else {
            // Case 3: two children — use inorder successor
            struct Node* succ = findMin(root->right);
            root->data  = succ->data;
            root->right = deleteNode(root->right, succ->data);
        }
    }
    if (!root) return NULL;

    // Rebalance on the way back up
    updateHeight(root);
    int bf = getBalance(root);
    if (bf > 1  && getBalance(root->left)  >= 0)  return rightRotate(root);
    if (bf > 1  && getBalance(root->left)  < 0)
        { root->left  = leftRotate(root->left);  return rightRotate(root); }
    if (bf < -1 && getBalance(root->right) <= 0)  return leftRotate(root);
    if (bf < -1 && getBalance(root->right) > 0)
        { root->right = rightRotate(root->right); return leftRotate(root); }
    return root;
}
💡
Real-Life Analogy
🌲 Like removing a tree from a forest: Remove a sapling (leaf) — nothing else changes. Remove a one-branch tree — fill the gap with that branch. Remove a large two-branch tree — find the "next tree in the lineup" (successor), put it in the removed slot, then remove the simpler successor. After any removal, check if the remaining forest is still balanced.

Searching Operations

Search NodeRecursive SearchIterative SearchExact MatchSearch MinimumSearch Maximum
📚
Theory — Binary Search in O(log n)

Search exploits the BST property — at each node, all left values are smaller, all right values are larger. So we eliminate half the search space at every step.

  • target == node → Found!
  • target < node → go left
  • target > node → go right
  • reach NULL → not found
⏱ O(log n) time — the balanced AVL tree ensures height is always O(log n).

C Code — Search Variants
// ── Recursive Search ─────────────────────────────────────────
struct Node* searchRec(struct Node* root, int target) {
    if (!root || root->data == target) return root;
    if (target < root->data) return searchRec(root->left,  target);
    else                     return searchRec(root->right, target);
}

// ── Iterative Search (stack-safe for large trees) ─────────────
struct Node* searchIter(struct Node* root, int target) {
    while (root) {
        if      (target == root->data) return root;
        else if (target <  root->data) root = root->left;
        else                           root = root->right;
    }
    return NULL; // Not found
}

// ── Find Minimum (leftmost node) ─────────────────────────────
struct Node* findMinimum(struct Node* root) {
    if (!root) return NULL;
    while (root->left) root = root->left;
    return root;
}

// ── Find Maximum (rightmost node) ────────────────────────────
struct Node* findMaximum(struct Node* root) {
    if (!root) return NULL;
    while (root->right) root = root->right;
    return root;
}
💡
Real-Life Analogy
📖 Like searching a dictionary: You don't read every page. You open the middle — if your word comes before, search the left half; after, the right half. AVL tree search does the same thing at each node. Min is going straight to 'A', max is going straight to 'Z'.

Traversal Operations

Inorder (L→Root→R)Preorder (Root→L→R)Postorder (L→R→Root)Level Order (BFS)
📚
Theory — 4 Ways to Visit Every Node
  • Inorder (L→Root→R): Visits in ascending sorted order. Most useful BST property!
  • Preorder (Root→L→R): Root first. Useful for copying or printing structure.
  • Postorder (L→R→Root): Children before parent. Useful for deleting/freeing a tree.
  • Level Order (BFS): Level by level, left to right. Uses a queue.

// Inorder: Left → Root → Right (sorted output!)
void inorder(struct Node* root) {
    if (!root) return;
    inorder(root->left);
    printf("%d ", root->data);
    inorder(root->right);
}

// Preorder: Root → Left → Right (for copying)
void preorder(struct Node* root) {
    if (!root) return;
    printf("%d ", root->data);
    preorder(root->left);
    preorder(root->right);
}

// Postorder: Left → Right → Root (for deletion)
void postorder(struct Node* root) {
    if (!root) return;
    postorder(root->left);
    postorder(root->right);
    printf("%d ", root->data);
}

// Level Order (BFS) using array-based queue
void levelOrder(struct Node* root) {
    if (!root) return;
    struct Node* q[100]; int f = 0, r = 0;
    q[r++] = root;
    while (f < r) {
        struct Node* curr = q[f++];
        printf("%d ", curr->data);
        if (curr->left)  q[r++] = curr->left;
        if (curr->right) q[r++] = curr->right;
    }
}
💡
Real-Life Analogy
🏠 Like exploring a house: Preorder = photographer documents a room before exploring sub-rooms. Inorder = reading a bookshelf alphabetically left to right. Postorder = cleaner cleans inner rooms before the main hall. Level order = fire drill evacuating floor 1 before floor 2.

Height Operations

Find HeightUpdate HeightHeight DifferenceMaximum HeightSubtree Height
📚
Theory — Understanding Tree Height

Height of a node = longest downward path (in edges) to a leaf. NULL node = height 0, leaf = height 1.

  • Internal node height: 1 + max(height(left), height(right))
  • AVL max height: ≤ 1.44 × log₂(n+2) − 0.328 for n nodes
  • Height is cached in each node for O(1) access — updated after every modification.
Caching height prevents O(n) recalculation every time. It's stored and updated incrementally.

C Code — Height Operations
// ── Get Cached Height (O(1)) ─────────────────────────────────
int height(struct Node* n) { return n ? n->height : 0; }

// ── Compute Height by Traversal (O(n), for verification) ─────
int calcHeight(struct Node* root) {
    if (!root) return 0;
    return 1 + maxVal(calcHeight(root->left), calcHeight(root->right));
}

// ── Update Cached Height ─────────────────────────────────────
void updateHeight(struct Node* n) {
    if (n) n->height = 1 + maxVal(height(n->left), height(n->right));
}

// ── Height Difference (= Balance Factor) ─────────────────────
int heightDiff(struct Node* n) {
    return n ? height(n->left) - height(n->right) : 0;
}

int main() {
    struct Node* root = NULL;
    root = insert(root, 30); root = insert(root, 20); root = insert(root, 40);
    printf("Tree height: %d\n", height(root));  // 2
    printf("BF at root:  %d\n", heightDiff(root)); // 0
    return 0;
}
💡
Real-Life Analogy
🏢 Like floor count on a building: Height = number of floors from ground to top. A single room = 1 floor. A complex with wings = tallest wing + 1. We pre-store height (like a sign "12 floors") instead of counting from scratch each time. Updating height after adding a floor is like updating that sign.

Balance Factor Operations

Calculate BFCheck Left HeavyCheck Right HeavyDetect Imbalance
📚
Theory — The Balance Factor Rules

BF = height(left) − height(right). The AVL invariant: every node has |BF| ≤ 1.

  • BF = 0: Perfectly balanced
  • BF = +1: Left 1 taller — OK
  • BF = −1: Right 1 taller — OK
  • BF = +2: Left violation → LL or LR rotation
  • BF = −2: Right violation → RR or RL rotation
BF can never reach ±3 in a properly maintained AVL tree — we fix it immediately at ±2.
💻
C Code — Balance Factor Operations
int getBalance(struct Node* n) {
    return n ? height(n->left) - height(n->right) : 0;
}
int isLeftHeavy(struct Node* n)  { return getBalance(n) > 0; }
int isRightHeavy(struct Node* n) { return getBalance(n) < 0; }
int isImbalanced(struct Node* n) {
    int b = getBalance(n); return b > 1 || b < -1;
}

int main() {
    struct Node* root = NULL;
    root = insert(root, 20); root = insert(root, 10); root = insert(root, 30);
    printf("BF=%d lh=%d rh=%d imbal=%d\n",
        getBalance(root), isLeftHeavy(root),
        isRightHeavy(root), isImbalanced(root));
    return 0;
}
💡
Real-Life Analogy
⚖️ Like a balance scale reading: BF=0 = perfectly level. BF=±1 = slight tilt — acceptable. BF=±2 = too heavy on one side — you must move weight. An AVL tree ensures no node's scale ever tips more than "one unit."

Min / Max Operations

Find Minimum NodeFind Maximum NodeFind LeftmostFind Rightmost
📚
Theory — Finding Extremes
  • Minimum: Always the leftmost node — keep going left until NULL.
  • Maximum: Always the rightmost node — keep going right until NULL.

Both are O(log n) in AVL trees — much faster than O(n) in unsorted data.

findMin on the right subtree gives the inorder successor — used internally during deletion of two-child nodes.

C Code — Min/Max Operations
// ── Find Minimum (recursive) ─────────────────────────────────
struct Node* findMinNode(struct Node* root) {
    if (!root || !root->left) return root;
    return findMinNode(root->left);
}

// ── Find Minimum (iterative) ─────────────────────────────────
struct Node* findMinIter(struct Node* root) {
    if (!root) return NULL;
    while (root->left) root = root->left;
    return root;
}

// ── Find Maximum (iterative) ─────────────────────────────────
struct Node* findMaxIter(struct Node* root) {
    if (!root) return NULL;
    while (root->right) root = root->right;
    return root;
}

int main() {
    struct Node* root = NULL;
    int v[] = {50,30,70,10,40,60,80};
    for (int i=0; i<7; i++) root = insert(root, v[i]);
    printf("Min: %d\n", findMinIter(root)->data); // 10
    printf("Max: %d\n", findMaxIter(root)->data); // 80
    return 0;
}
💡
Real-Life Analogy
📏 Like finding the shortest/tallest in a sorted lineup: Students sorted by height in a V-shape. Shortest is far left end, tallest is far right end. No need to check everyone — walk straight to the end! Same in AVL: min = leftmost, max = rightmost.

Successor & Predecessor Operations

Find Inorder SuccessorFind Inorder PredecessorNext Greater ElementNext Smaller Element
📚
Theory — Next and Previous in Sorted Order
  • Successor (next greater): Has right subtree? → min of right subtree. No right subtree? → nearest ancestor where you came from the left.
  • Predecessor (next smaller): Has left subtree? → max of left subtree. No left subtree? → nearest ancestor where you came from the right.
Successor is used internally during deletion (two-child case) to fill the removed node's spot.

C Code — Successor & Predecessor
// ── Find Inorder Successor (next greater value) ───────────────
struct Node* findSuccessor(struct Node* root, int target) {
    struct Node* succ = NULL, *cur = root;
    while (cur) {
        if (target < cur->data)   { succ = cur; cur = cur->left; }
        else if (target > cur->data) cur = cur->right;
        else {
            if (cur->right) succ = findMinIter(cur->right);
            break;
        }
    }
    return succ;
}

// ── Find Inorder Predecessor (next smaller value) ─────────────
struct Node* findPredecessor(struct Node* root, int target) {
    struct Node* pred = NULL, *cur = root;
    while (cur) {
        if (target > cur->data)   { pred = cur; cur = cur->right; }
        else if (target < cur->data) cur = cur->left;
        else {
            if (cur->left) pred = findMaxIter(cur->left);
            break;
        }
    }
    return pred;
}
💡
Real-Life Analogy
📅 Like finding next/previous day: If today is Wednesday, successor = Thursday, predecessor = Tuesday. In a BST, the successor is the "next value in sorted order." Has a right subtree? First value there. Otherwise, it's the nearest ancestor you approached from the left.

    if (!r) return 0;
    if (!r->left && !r->right) return 1;
    return countLeaves(r->left) + countLeaves(r->right);
}
int countInternal(struct Node* r) {
    return countNodes(r) - countLeaves(r);
}

// ── Find Parent ──────────────────────────────────────────────
struct Node* findParent(struct Node* root, int target) {
    if (!root || root->data == target) return NULL;
    if ((root->left  && root->left->data  == target) ||
        (root->right && root->right->data == target)) return root;
    if (target < root->data) return findParent(root->left,  target);
    else                     return findParent(root->right, target);
}

// ── Is Ancestor? ─────────────────────────────────────────────
int isAncestor(struct Node* anc, int target) {
    if (!anc) return 0;
    if (anc->data == target) return 1;
    return isAncestor(anc->left, target) || isAncestor(anc->right, target);
}

// ── Find Sibling ─────────────────────────────────────────────
struct Node* findSibling(struct Node* root, int target) {
    struct Node* p = findParent(root, target);
    if (!p) return NULL;
    return (p->left && p->left->data == target) ? p->right : p->left;
}

Update Operations

Update Node ValueDelete + ReinsertRebalance After Update
📚
Theory — Updating Node Values

You cannot simply overwrite node→data in-place — this would break the BST ordering property.

Correct approach: Delete old value → Insert new value.

  • BST property is automatically maintained.
  • AVL automatically rebalances during both operations.
  • Total cost: O(log n) + O(log n) = O(log n).
Never directly change node→data! This corrupts the BST ordering silently.

    // Step 1: Verify old value exists

struct Node* copyTree(struct Node* root) {

    struct Node* n = createNode(root->data);

struct Node* mirrorTree(struct Node* root) {
    if (!root) return NULL;
    struct Node* tmp = root->left;
    root->left  = mirrorTree(root->right);
    root->right = mirrorTree(tmp);
    return root;
}

// ── Clear / Destroy Tree (postorder free) ─────────────────────
void clearTree(struct Node* root) {
    if (!root) return;
    clearTree(root->left);   // Free children first
    clearTree(root->right);
    free(root);              // Then free this node
}
void destroyTree(struct Node** rp) { clearTree(*rp); *rp = NULL; }

📚
Theory — Paths & LCA
  • Path to Node: Sequence of nodes from root down to a target.
  • Root-to-Leaf Paths: All paths from root to every leaf node.
  • LCA: Given two nodes a and b, the deepest node that is an ancestor of both.
In a BST: LCA of a and b is the node where they first "diverge" — if both are smaller go left, if both larger go right, otherwise current node is LCA.

// ── Find Path from Root to Target ────────────────────────────
int findPath(struct Node* root, int target, int* path, int idx) {
    if (!root) return -1;
    path[idx] = root->data;
    if (root->data == target) return idx;
    int f = target < root->data
        ? findPath(root->left,  target, path, idx+1)
        : findPath(root->right, target, path, idx+1);
    return f;
}

// ── Print All Root-to-Leaf Paths ─────────────────────────────
void printPaths(struct Node* root, int* path, int len) {
    if (!root) return;
    path[len++] = root->data;
    if (!root->left && !root->right) {
        for (int i=0; i<len; i++)
            printf("%d%s", path[i], i<len-1?"→":"\n");
        return;
    }
    printPaths(root->left,  path, len);
    printPaths(root->right, path, len);
}

// ── Lowest Common Ancestor (BST approach) ────────────────────
struct Node* findLCA(struct Node* root, int a, int b) {
    if (!root) return NULL;
    if (a < root->data && b < root->data) return findLCA(root->left,  a, b);
    if (a > root->data && b > root->data) return findLCA(root->right, a, b);
    return root; // Split point = LCA
}

struct Node* arrToAVL(int* arr, int lo, int hi) {

    int mid = (lo + hi) / 2;

Recursive Operations

Recursive InsertRecursive DeleteRecursive SearchRecursive Traversal
📚
Theory — Why Recursion Fits Trees

Trees are naturally recursive — every subtree of a tree is itself a tree. Recursive algorithms mirror this structure perfectly.

  • Base case: node is NULL → stop.
  • Recursive case: process left or right subtree.
  • Return path: results bubble up as stack unwinds — AVL balancing happens here!
The unwinding call stack is what makes recursive AVL insertion elegant — each ancestor rebalances automatically as we return from deeper levels.
Stack space used: O(h) = O(log n) for AVL trees. Safe for normal sizes.

// ── Recursive Insert (annotated) ──────────────────────────────
struct Node* insertRec(struct Node* root, int data) {
    // ▼ GOING DOWN: find the insert position
    if (!root) return createNode(data); // BASE CASE

    if (data < root->data)
        root->left  = insertRec(root->left,  data); // dive left
    else if (data > root->data)
        root->right = insertRec(root->right, data); // dive right
    else return root; // duplicate

    // ▲ COMING BACK UP: rebalance each ancestor
    updateHeight(root);
    int bf = getBalance(root);

    if (bf > 1  && data < root->left->data)   return rightRotate(root);
    if (bf < -1 && data > root->right->data)  return leftRotate(root);
    if (bf > 1  && data > root->left->data)
        { root->left  = leftRotate(root->left);  return rightRotate(root); }
    if (bf < -1 && data < root->right->data)
        { root->right = rightRotate(root->right); return leftRotate(root); }
    return root;
}

// ── Recursive Search ─────────────────────────────────────────
struct Node* searchRec(struct Node* root, int target) {
    if (!root || root->data == target) return root;
    return target < root->data
        ? searchRec(root->left,  target)
        : searchRec(root->right, target);
}

Real-Life Analogy
🪆 Like Russian nesting dolls: Each doll opens to reveal a smaller doll inside (recursive call going down). When you reach the smallest doll (base case), you close them back up one by one (return path / stack unwinding). AVL rebalancing is like checking and adjusting each doll as you close it.

Iterative Operations

Iterative InsertIterative SearchIterative Traversal
📚
Theory — Iterative vs Recursive

Iterative versions avoid function call overhead and are safer for very deep trees (no stack overflow risk). They use explicit data structures (stack/queue) to simulate recursion.

  • Iterative Search: Simple while loop — no extra memory needed.
  • Iterative Inorder: Uses an explicit stack to simulate the recursion.
  • Iterative Insert: Find position with a loop, but balancing still requires tracking the path (parent pointers or a stack).
For AVL trees, recursive insert is almost always preferred — the automatic unwinding makes rebalancing elegant and clean.

// ── Iterative Search ─────────────────────────────────────────
struct Node* searchIter(struct Node* root, int target) {
    while (root) {
        if      (target == root->data) return root;
        else if (target  < root->data) root = root->left;
        else                           root = root->right;
    }
    return NULL;
}

// ── Iterative Inorder using explicit stack ────────────────────
void inorderIter(struct Node* root) {
    struct Node* stack[100];
    int top = -1;
    struct Node* curr = root;

    while (curr || top >= 0) {
        // Push all left nodes onto stack
        while (curr) { stack[++top] = curr; curr = curr->left; }

        curr = stack[top--];           // Pop
        printf("%d ", curr->data);   // Visit
        curr = curr->right;            // Go right
    }
}

// ── Iterative Level Order (BFS with queue) ────────────────────
void levelOrderIter(struct Node* root) {
    if (!root) return;
    struct Node* q[100]; int f=0, r=0;
    q[r++] = root;
    while (f < r) {
        struct Node* n = q[f++];
        printf("%d ", n->data);
        if (n->left)  q[r++] = n->left;
        if (n->right) q[r++] = n->right;
    }
}

Advanced AVL Operations

Join Two AVL TreesSplit AVL TreeMerge AVL Trees
📚
Theory — Advanced Structural Operations

These are the most advanced AVL tree operations, used in databases, interval trees, and functional programming languages.

  • Join(T1, k, T2): All keys in T1 < k < all keys in T2. Creates a single AVL tree with all nodes plus k. O(log n) time when heights differ by ≤ 1; O(|h1−h2|) in general.
  • Split(T, k): Splits one AVL tree into two — T1 with keys < k, T2 with keys > k. Uses Join internally. O(log n) time.
  • Merge(T1, T2): Combines two AVL trees (may have overlapping key ranges). O(m log(n/m + 1)) where m ≤ n. Used internally in databases for query result merging.
These operations form the basis of order-statistic trees, augmented BSTs, and functional data structure libraries like those in Haskell, OCaml, and Java's TreeMap.
These are advanced — master sections 1–19 first before implementing these!

C Code — Join, Split & Merge
// ── Join: merge T1, key k, T2 where all(T1) < k < all(T2) ─────
struct Node* joinRight(struct Node* T1, int k, struct Node* T2) {
    if (height(T1) <= height(T2) + 1) {
        struct Node* t = createNode(k);
        t->left = T1; t->right = T2;
        updateHeight(t);
        if (getBalance(t) > 1) return rebalance(t);
        return t;
    }
    struct Node* t = joinRight(T1->right, k, T2);
    T1->right = t;
    updateHeight(T1);
    if (getBalance(T1) < -1) return rebalance(T1);
    return T1;
}

struct Node* join(struct Node* T1, int k, struct Node* T2) {
    if (height(T1) > height(T2) + 1) return joinRight(T1, k, T2);
    if (height(T2) > height(T1) + 1) {
        // symmetric joinLeft
        struct Node* t = createNode(k);
        t->left = T1; t->right = T2;
        return rebalance(t);
    }
    struct Node* t = createNode(k);
    t->left = T1; t->right = T2;
    updateHeight(t);
    return t;
}

// ── Split: split T into (T1 < k) and (T2 > k) ───────────────
void split(struct Node* T, int k,
           struct Node** T1, struct Node** T2) {
    if (!T) { *T1 = *T2 = NULL; return; }
    if (k < T->data) {
        split(T->left, k, T1, T2);
        *T2 = join(*T2, T->data, T->right);
    } else {
        split(T->right, k, T1, T2);
        *T1 = join(T->left, T->data, *T1);
    }
}

// ── Merge (union) of two AVL trees ───────────────────────────
struct Node* merge(struct Node* T1, struct Node* T2) {
    if (!T1) return T2;
    if (!T2) return T1;
    struct Node* L, *R;
    // Use T1's root as the split key
    split(T2, T1->data, &L, &R);
    return join(merge(T1->left, L),
                T1->data,
                merge(T1->right, R));
}

Merge is like combining two city road networks into one, handling any overlapping roads. All operations maintain traffic flow (BST order) and balanced load (AVL property).
