🌐 What is a Directed Graph?

A Directed Graph (Digraph) is a set of vertices (nodes) connected by directed edges (arcs). Each edge has a direction — it goes from one vertex to another. Unlike undirected graphs, if there's an edge from A→B, you cannot automatically go from B→A.

  • Vertex (V): A node/point in the graph (e.g., a city, a person, a webpage).
  • Edge (E): A directed connection between two vertices (e.g., a one-way road).
  • Self-loop: An edge from a vertex to itself.
  • Weight: An optional cost/distance assigned to an edge.
🌐
Real-Life Analogy: The Internet

Think of webpages on the internet. Page A can link to Page B, but Page B may not link back to Page A. Each link is a directed edge. Google's PageRank algorithm models the web as a directed graph!

📋 Adjacency List Representation

An Adjacency List stores a graph as an array of linked lists. Each array index represents a vertex, and its linked list contains all vertices that can be reached directly from it.

Why Adjacency List? It is space-efficient for sparse graphs (few edges). It uses O(V + E) space, while an adjacency matrix uses O(V²).

  • Each node in the linked list stores the destination vertex and optionally a weight.
  • For a directed edge A→B, only B is added to A's list (NOT A to B's list).
  • Insertion of edge: O(1). Finding if edge exists: O(degree).

C — Adjacency List Structure
#include <stdio.h>
#include <stdlib.h>
#define MAX_VERTICES 100

/* Node in the adjacency list (linked list node) */
struct Node {
    int dest;           /* destination vertex */
    int weight;         /* edge weight (0 if unweighted) */
    struct Node* next;  /* pointer to next neighbor */
};

/* Graph structure */
struct Graph {
    int numVertices;               /* total number of vertices */
    struct Node* adjList[MAX_VERTICES]; /* array of linked list heads */
    char label[MAX_VERTICES][20];  /* optional vertex labels */
};

/* Create a new graph with V vertices */
struct Graph* createGraph(int V) {
    struct Graph* g = (struct Graph*) malloc(sizeof(struct Graph));
    g->numVertices = V;
    for (int i = 0; i < V; i++)
        g->adjList[i] = NULL;  /* initialize all lists to NULL */
    return g;
}

/* Create a new adjacency list node */
struct Node* newNode(int dest, int weight) {
    struct Node* node = (struct Node*) malloc(sizeof(struct Node));
    node->dest   = dest;
    node->weight = weight;
    node->next   = NULL;
    return node;
}
Memory Layout: For V=4 vertices with edges 0→1, 0→2, 1→2, 2→3: adjList[0] → [1] → [2] → NULL | adjList[1] → [2] → NULL | adjList[2] → [3] → NULL | adjList[3] → NULL

🔷 Vertex Operations

Insert Vertex

Adding a new vertex means increasing the vertex count and initializing its adjacency list to NULL (no edges yet). In a dynamic graph, this can be done at any time.

🏙️
Analogy: Adding a New City

Imagine a road map. Adding a new city is like inserting a vertex — the city exists but has no roads yet. Roads (edges) are added separately.

C — Insert Vertex
/* Insert a new vertex into the graph */
void insertVertex(struct Graph* g, char* label) {
    if (g->numVertices >= MAX_VERTICES) {
        printf("Graph is full!\n");
        return;
    }
    int id = g->numVertices;         /* new vertex gets next available id */
    g->adjList[id] = NULL;           /* no edges yet */
    strcpy(g->label[id], label);     /* store optional label */
    g->numVertices++;
    printf("Vertex '%s' inserted with ID %d\n", label, id);
}

Delete Vertex

Deleting a vertex requires: (1) freeing its adjacency list, and (2) removing all edges pointing to that vertex from other lists. This is the most expensive vertex operation — O(V + E).

C — Delete Vertex
/* Remove all edges from vertex v's list */
void freeList(struct Node* head) {
    while (head) {
        struct Node* temp = head;
        head = head->next;
        free(temp);
    }
}

void deleteVertex(struct Graph* g, int v) {
    if (v < 0 || v >= g->numVertices) { printf("Invalid vertex\n"); return; }

    /* Step 1: Free v's own adjacency list */
    freeList(g->adjList[v]);
    g->adjList[v] = NULL;

    /* Step 2: Remove all edges pointing TO v */
    for (int i = 0; i < g->numVertices; i++) {
        if (i == v) continue;
        struct Node** cur = &g->adjList[i];
        while (*cur) {
            if ((*cur)->dest == v) {
                struct Node* del = *cur;
                *cur = (*cur)->next;
                free(del);
            } else cur = &(*cur)->next;
        }
    }
    printf("Vertex %d deleted\n", v);
}

Search / Update / Count / Display Vertices

Search Vertex

Check if a vertex with a given ID or label exists. O(V) for label search, O(1) for ID check.

Update Vertex

Rename or reassign a vertex's label. The ID stays the same; only the metadata changes.

Count Vertices

Simply return g→numVertices. O(1) operation.

Display Vertices

Loop from 0 to numVertices-1 and print each vertex ID/label.

int searchVertex(struct Graph* g, char* label) {

int countVertices(struct Graph* g) { return g->numVertices; }

/* Display all vertices */

    printf("Vertices (%d total):\n", g->numVertices);
    for (int i = 0; i < g->numVertices; i++)
        printf("  [%d] %s\n", i, g->label[i]);
}

➡️ Edge Operations

Insert Directed Edge

Adding a directed edge from vertex src to vertex dest means prepending a new node to adjList[src] only. This is O(1). The reverse (dest→src) is NOT added — that's what makes it directed.

🛣️
Analogy: One-Way Street

A one-way street from Main St → Park Ave. Cars can only travel in one direction. Similarly, a directed edge lets you traverse in one direction only.

▶ Edge Operations Demo
Graph has 5 vertices (0-4). Add directed edges...
C — Insert Directed Edge
/* Add directed weighted edge: src → dest */
void insertEdge(struct Graph* g, int src, int dest, int weight) {
    if (src < 0 || src >= g->numVertices ||
        dest < 0 || dest >= g->numVertices) {
        printf("Invalid vertices!\n");
        return;
    }
    /* Create new node and prepend to src's list */
    struct Node* node = newNode(dest, weight);
    node->next = g->adjList[src];   /* point to current head */
    g->adjList[src] = node;         /* new node becomes head */
    printf("Edge %d → %d (weight=%d) added\n", src, dest, weight);
    /* NOTE: We do NOT add dest→src (it's directed!) */
}

Delete Directed Edge

C — Delete Directed Edge
void deleteEdge(struct Graph* g, int src, int dest) {
    struct Node** cur = &g->adjList[src];
    while (*cur) {
        if ((*cur)->dest == dest) {
            struct Node* del = *cur;
            *cur = (*cur)->next;   /* bypass the deleted node */
            free(del);
            printf("Edge %d → %d deleted\n", src, dest);
            return;
        }
        cur = &(*cur)->next;
    }
    printf("Edge not found\n");
}

Search Edge / Update Weight / Count / Display Edges

C — Search, Update Weight, Count, Display Edges
/* Search for edge src→dest, returns 1 if found */
int searchEdge(struct Graph* g, int src, int dest) {
    struct Node* cur = g->adjList[src];
    while (cur) {
        if (cur->dest == dest) return 1; /* found */
        cur = cur->next;
    }
    return 0; /* not found */
}

/* Update weight of edge src→dest */
void updateEdgeWeight(struct Graph* g, int src, int dest, int newW) {
    struct Node* cur = g->adjList[src];
    while (cur) {
        if (cur->dest == dest) { cur->weight = newW; return; }
        cur = cur->next;
    }
    printf("Edge not found\n");
}

/* Count total directed edges */
int countEdges(struct Graph* g) {
    int count = 0;
    for (int i = 0; i < g->numVertices; i++) {
        struct Node* cur = g->adjList[i];
        while (cur) { count++; cur = cur->next; }
    }
    return count;
}

/* Display the entire adjacency list */
void displayGraph(struct Graph* g) {
    printf("Adjacency List:\n");
    for (int i = 0; i < g->numVertices; i++) {
        printf("[%d]", i);
        struct Node* cur = g->adjList[i];
        while (cur) {
            printf(" → %d(w=%d)", cur->dest, cur->weight);
            cur = cur->next;
        }
        printf(" → NULL\n");
    }
}

🔍 Traversal Operations

Breadth First Search (BFS)

BFS visits vertices level by level — first all vertices at distance 1 from the start, then distance 2, and so on. It uses a Queue (FIFO) to track which vertex to visit next.

Algorithm Steps:

Initialize a visited[] array (all false) and a queue.

Enqueue the start vertex and mark it visited.

Dequeue a vertex, print it, then enqueue all its unvisited neighbors and mark them visited.

Repeat step 3 until the queue is empty.

💧
Analogy: Ripples in a Pond

Drop a stone in water — ripples spread outward in circles. BFS works the same way, exploring all nodes at the current "ring" before moving to the next.

C — BFS (Queue-based)
#include <stdio.h>
#define MAX 100

int queue[MAX], front = 0, rear = 0;
void enqueue(int v) { queue[rear++] = v; }
int  dequeue()     { return queue[front++]; }
int  isEmpty()     { return front == rear; }

void BFS(struct Graph* g, int start) {
    int visited[MAX] = {0};  /* 0 = not visited */
    front = rear = 0;        /* reset queue */

    enqueue(start);
    visited[start] = 1;

    printf("BFS Order: ");
    while (!isEmpty()) {
        int v = dequeue();
        printf("%d ", v);

        struct Node* cur = g->adjList[v];
        while (cur) {
            if (!visited[cur->dest]) {
                visited[cur->dest] = 1;
                enqueue(cur->dest);
            }
            cur = cur->next;
        }
    }
    printf("\n");
}

Depth First Search (DFS)

DFS explores as deep as possible along a branch before backtracking. It can be done recursively (using call stack) or iteratively (using an explicit stack).

🗺️
Analogy: Exploring a Maze

In a maze, you pick a path and go as far as you can. If you hit a dead end, you backtrack and try another path. DFS works exactly like this!

C — DFS Recursive & Iterative (Stack)
/* ── Recursive DFS ── */
void DFS_Recursive(struct Graph* g, int v, int visited[]) {
    visited[v] = 1;
    printf("%d ", v);

    struct Node* cur = g->adjList[v];
    while (cur) {
        if (!visited[cur->dest])
            DFS_Recursive(g, cur->dest, visited); /* recurse */
        cur = cur->next;
    }
}

void DFS(struct Graph* g, int start) {
    int visited[MAX] = {0};
    printf("DFS Order: ");
    DFS_Recursive(g, start, visited);
    printf("\n");
}

/* ── Iterative DFS (using explicit stack) ── */
void DFS_Stack(struct Graph* g, int start) {
    int stack[MAX], top = -1;
    int visited[MAX] = {0};

    stack[++top] = start;   /* push start */

    printf("DFS (Stack): ");
    while (top >= 0) {
        int v = stack[top--];  /* pop */
        if (visited[v]) continue;
        visited[v] = 1;
        printf("%d ", v);

        struct Node* cur = g->adjList[v];
        while (cur) {
            if (!visited[cur->dest])
                stack[++top] = cur->dest; /* push neighbor */
            cur = cur->next;
        }
    }
    printf("\n");
}

📊 Degree Operations

In a directed graph, every vertex has two types of degree:

  • In-Degree: Number of edges pointing into the vertex.
  • Out-Degree: Number of edges going out of the vertex.
  • Total Degree: In-Degree + Out-Degree.
  • Max Degree: The highest degree among all vertices.
  • Min Degree: The lowest degree among all vertices.

Sum of all in-degrees = Sum of all out-degrees = Total number of edges.

📧
Analogy: Email Inbox & Outbox

In-degree is like your inbox (emails received from others). Out-degree is like your outbox (emails you sent). A popular person has high in-degree; a chatty person has high out-degree.

C — All Degree Operations
/* Out-Degree: count edges leaving vertex v */
int outDegree(struct Graph* g, int v) {
    int count = 0;
    struct Node* cur = g->adjList[v];
    while (cur) { count++; cur = cur->next; }
    return count;
}

/* In-Degree: count edges pointing TO vertex v */
int inDegree(struct Graph* g, int v) {
    int count = 0;
    for (int i = 0; i < g->numVertices; i++) {
        struct Node* cur = g->adjList[i];
        while (cur) {
            if (cur->dest == v) count++;
            cur = cur->next;
        }
    }
    return count;
}

/* Total Degree */
int totalDegree(struct Graph* g, int v) {
    return inDegree(g, v) + outDegree(g, v);
}

/* Max Out-Degree across all vertices */
int maxDegree(struct Graph* g) {
    int maxD = 0;
    for (int i = 0; i < g->numVertices; i++) {
        int d = outDegree(g, i);
        if (d > maxD) maxD = d;
    }
    return maxD;
}

/* Min Out-Degree across all vertices */
int minDegree(struct Graph* g) {
    int minD = g->numVertices;
    for (int i = 0; i < g->numVertices; i++) {
        int d = outDegree(g, i);
        if (d < minD) minD = d;
    }
    return minD;
}

🛤️ Path Operations

A path is a sequence of vertices where each consecutive pair is connected by a directed edge. Path operations include finding if a path exists, finding all paths, shortest path, and calculating path cost.

  • Find Path: Find any one path from src to dest (DFS-based).
  • All Paths: Find every possible path from src to dest.
  • Shortest Path: Path with fewest edges (BFS) or minimum weight.
  • Reachability: Can we reach dest from src at all?
  • Path Cost: Sum of edge weights along a path.
🗺️
Analogy: GPS Navigation

Finding a path in a graph is like a GPS finding a route from your home to a destination. Multiple routes may exist (All Paths). GPS picks the shortest or fastest one (Shortest Path).

C — Find Path, All Paths, Reachability, Path Cost
/* Check if dest is reachable from src (BFS-based) */
int isReachable(struct Graph* g, int src, int dest) {
    int visited[MAX] = {0};
    int queue[MAX]; int f = 0, r = 0;
    queue[r++] = src; visited[src] = 1;
    while (f < r) {
        int v = queue[f++];
        if (v == dest) return 1; /* found! */
        struct Node* cur = g->adjList[v];
        while (cur) {
            if (!visited[cur->dest]) {
                visited[cur->dest] = 1;
                queue[r++] = cur->dest;
            }
            cur = cur->next;
        }
    }
    return 0; /* not reachable */
}

/* Find ALL paths from src to dest (DFS + backtracking) */
int path[MAX], pathLen = 0;
void findAllPaths(struct Graph* g, int cur, int dest, int visited[]) {
    visited[cur] = 1;
    path[pathLen++] = cur;

    if (cur == dest) {
        for (int i = 0; i < pathLen; i++)
            printf("%d%s", path[i], i < pathLen-1 ? "→" : "\n");
    } else {
        struct Node* n = g->adjList[cur];
        while (n) {
            if (!visited[n->dest])
                findAllPaths(g, n->dest, dest, visited);
            n = n->next;
        }
    }
    /* Backtrack */
    pathLen--;
    visited[cur] = 0;
}

/* Calculate cost of a specific path */
int pathCost(struct Graph* g, int* path, int len) {
    int cost = 0;
    for (int i = 0; i < len - 1; i++) {
        struct Node* cur = g->adjList[path[i]];
        while (cur && cur->dest != path[i+1]) cur = cur->next;
        if (cur) cost += cur->weight;
    }
    return cost;
}

🔗 Connectivity Operations

Connectivity describes how well the graph is "connected":

  • Strongly Connected: Every vertex is reachable from every other vertex following directed edges.
  • Weakly Connected: The graph is connected if we ignore edge directions.
  • Strongly Connected Components (SCC): Maximal subsets of vertices where every vertex can reach every other vertex within the subset. (Kosaraju's or Tarjan's algorithm).
  • Reachable Nodes: All nodes reachable from a given source using BFS/DFS.
🏘️
Analogy: Social Groups

An SCC is like a close-knit friend group where everyone knows everyone else. Two groups might be connected through someone, but not every member knows all members of the other group — that's weak connectivity.

C — Kosaraju's SCC Algorithm
/* ─── Kosaraju's Algorithm for SCCs ─── */

int stack2[MAX], sTop = -1;

/* Step 1: Fill stack by finish time (DFS on original graph) */
void fillOrder(struct Graph* g, int v, int visited[]) {
    visited[v] = 1;
    struct Node* cur = g->adjList[v];
    while (cur) {
        if (!visited[cur->dest])
            fillOrder(g, cur->dest, visited);
        cur = cur->next;
    }
    stack2[++sTop] = v;  /* push after all descendants */
}

/* Step 2: Create transpose graph (reverse all edges) */
struct Graph* transposeGraph(struct Graph* g) {
    struct Graph* t = createGraph(g->numVertices);
    for (int v = 0; v < g->numVertices; v++) {
        struct Node* cur = g->adjList[v];
        while (cur) {
            insertEdge(t, cur->dest, v, cur->weight); /* reverse edge */
            cur = cur->next;
        }
    }
    return t;
}

/* Step 3: Find SCCs */
void findSCCs(struct Graph* g) {
    int visited[MAX] = {0};
    sTop = -1;

    /* Fill finish order */
    for (int i = 0; i < g->numVertices; i++)
        if (!visited[i]) fillOrder(g, i, visited);

    /* Get transpose */
    struct Graph* t = transposeGraph(g);

    /* DFS on transpose in finish-time order */
    int vis2[MAX] = {0};
    while (sTop >= 0) {
        int v = stack2[sTop--];
        if (!vis2[v]) {
            printf("SCC: ");
            DFS_Recursive(t, v, vis2);
            printf("\n");
        }
    }
}

🔄 Cycle Operations

A cycle in a directed graph is a path that starts and ends at the same vertex following directed edges. Detecting cycles is crucial for:

  • Checking if topological sort is possible (cycle → no valid sort).
  • Detecting deadlocks in operating systems.
  • Validating dependency graphs (no circular dependencies).

DFS-based detection: We track a "recursion stack." If we visit a vertex already in the current DFS stack, a cycle exists.

🔁
Analogy: Circular Dependency

Package A requires Package B, Package B requires Package C, and Package C requires Package A. This is a cycle! Your package manager (npm, pip) must detect and report this error.

C — Detect Cycle (DFS with Recursion Stack)
/* Helper: DFS to detect cycle using recursion stack */
int dfsHasCycle(struct Graph* g, int v, int visited[], int recStack[]) {
    visited[v]  = 1;
    recStack[v] = 1;  /* mark as in current DFS path */

    struct Node* cur = g->adjList[v];
    while (cur) {
        int d = cur->dest;
        if (!visited[d] && dfsHasCycle(g, d, visited, recStack))
            return 1;  /* cycle found deeper */
        if (recStack[d])
            return 1;  /* back edge = cycle! */
        cur = cur->next;
    }
    recStack[v] = 0;  /* remove from recursion stack (backtrack) */
    return 0;
}

int hasCycle(struct Graph* g) {
    int visited[MAX]  = {0};
    int recStack[MAX] = {0};
    for (int i = 0; i < g->numVertices; i++)
        if (!visited[i] && dfsHasCycle(g, i, visited, recStack))
            return 1;  /* cycle found */
    return 0;  /* no cycle (DAG) */
}

📐 Topological Operations

Topological Sort orders vertices so that for every directed edge u→v, vertex u comes before v in the ordering. Only possible if the graph has no cycles (DAG — Directed Acyclic Graph).

Two methods:

  • DFS-based (Kahn's variation): Push vertices to stack after all neighbors are processed.
  • BFS-based (Kahn's Algorithm): Repeatedly remove vertices with in-degree 0.

Applications: Build systems (make), course scheduling, task dependency resolution, compilation order.

📚
Analogy: University Course Prerequisite

You must take Math 101 before Calculus, and Calculus before Differential Equations. Topological sort gives you a valid order to take all courses without violating prerequisites.

C — Topological Sort (DFS) & Kahn's Algorithm (BFS)
/* ─── DFS-based Topological Sort ─── */
int topoStack[MAX], topoTop = -1;

void topoUtil(struct Graph* g, int v, int visited[]) {
    visited[v] = 1;
    struct Node* cur = g->adjList[v];
    while (cur) {
        if (!visited[cur->dest])
            topoUtil(g, cur->dest, visited);
        cur = cur->next;
    }
    topoStack[++topoTop] = v; /* push when done with vertex */
}

void topologicalSort(struct Graph* g) {
    int visited[MAX] = {0};
    topoTop = -1;
    for (int i = 0; i < g->numVertices; i++)
        if (!visited[i]) topoUtil(g, i, visited);
    printf("Topological Order: ");
    while (topoTop >= 0) printf("%d ", topoStack[topoTop--]);
}

/* ─── Kahn's Algorithm (BFS) ─── */
void kahnAlgorithm(struct Graph* g) {
    int inDeg[MAX] = {0};
    for (int v = 0; v < g->numVertices; v++) {
        struct Node* cur = g->adjList[v];
        while (cur) { inDeg[cur->dest]++; cur = cur->next; }
    }
    int q[MAX], f = 0, r = 0;
    for (int i = 0; i < g->numVertices; i++)
        if (inDeg[i] == 0) q[r++] = i; /* start with in-degree 0 */

    printf("Kahn's Order: ");
    while (f < r) {
        int v = q[f++];
        printf("%d ", v);
        struct Node* cur = g->adjList[v];
        while (cur) {
            if (--inDeg[cur->dest] == 0) q[r++] = cur->dest;
            cur = cur->next;
        }
    }
    printf("\n");
}

⚡ Shortest Path Algorithms

Dijkstra's Algorithm Non-negative weights only

Dijkstra's finds the shortest path from a single source to all other vertices. It greedily picks the unvisited vertex with the smallest known distance and relaxes its neighbors.

  • Time: O(V²) with simple array, O((V+E) log V) with priority queue.
  • Does NOT work with negative edge weights.
🗺️
Analogy: Google Maps Navigation

Google Maps finds the shortest driving route from your location. It considers road distances (positive weights) and picks the optimal path — exactly what Dijkstra's does.

C — Dijkstra's Algorithm
#define INF 99999

/* Find vertex with minimum distance not yet visited */
int minDistance(int dist[], int visited[], int V) {
    int min = INF, minIdx = -1;
    for (int v = 0; v < V; v++)
        if (!visited[v] && dist[v] <= min) {
            min = dist[v]; minIdx = v;
        }
    return minIdx;
}

void dijkstra(struct Graph* g, int src) {
    int dist[MAX], visited[MAX] = {0};
    for (int i = 0; i < g->numVertices; i++) dist[i] = INF;
    dist[src] = 0;

    for (int iter = 0; iter < g->numVertices - 1; iter++) {
        int u = minDistance(dist, visited, g->numVertices);
        if (u == -1) break;
        visited[u] = 1;

        /* Relax all neighbors of u */
        struct Node* cur = g->adjList[u];
        while (cur) {
            int v = cur->dest, w = cur->weight;
            if (!visited[v] && dist[u] + w < dist[v])
                dist[v] = dist[u] + w;  /* relax edge */
            cur = cur->next;
        }
    }

    printf("Shortest distances from vertex %d:\n", src);
    for (int i = 0; i < g->numVertices; i++)
        printf("  To %d: %d\n", i, dist[i] == INF ? -1 : dist[i]);
}

Bellman-Ford Algorithm Handles negative weights

Bellman-Ford finds shortest paths from a single source and works with negative edge weights. It also detects negative weight cycles. It relaxes all edges V-1 times.

  • Time: O(V × E) — slower than Dijkstra but more general.
  • After V-1 relaxations, if any distance can still be reduced, a negative cycle exists.
💸
Analogy: Currency Arbitrage

In finance, negative weights model currency exchange rates. Bellman-Ford detects if a "negative cycle" exists — meaning you can exchange currencies in a loop and keep making money (arbitrage opportunity)!

C — Bellman-Ford Algorithm
/* Edge list for Bellman-Ford (easier to iterate all edges) */
struct Edge { int src, dest, weight; };

void bellmanFord(struct Edge edges[], int E, int V, int src) {
    int dist[MAX];
    for (int i = 0; i < V; i++) dist[i] = INF;
    dist[src] = 0;

    /* Relax all edges V-1 times */
    for (int iter = 0; iter < V - 1; iter++) {
        for (int e = 0; e < E; e++) {
            int u = edges[e].src, v = edges[e].dest, w = edges[e].weight;
            if (dist[u] != INF && dist[u] + w < dist[v])
                dist[v] = dist[u] + w;
        }
    }

    /* Check for negative weight cycles */
    for (int e = 0; e < E; e++) {
        int u = edges[e].src, v = edges[e].dest, w = edges[e].weight;
        if (dist[u] != INF && dist[u] + w < dist[v]) {
            printf("Negative weight cycle detected!\n");
            return;
        }
    }

    printf("Distances from source %d:\n", src);
    for (int i = 0; i < V; i++)
        printf("  To %d: %d\n", i, dist[i]);
}

Floyd-Warshall Algorithm All-pairs shortest path

Floyd-Warshall finds shortest paths between ALL pairs of vertices. It uses dynamic programming with a 2D distance matrix, trying each vertex as an intermediate node.

  • Time: O(V³) — works for negative weights (but not negative cycles).
  • Space: O(V²) for the distance matrix.
🌍
Analogy: Flight Route Planner

Airlines want to know the shortest route between every pair of airports (with possible stopovers). Floyd-Warshall computes all such pairs at once — like a complete travel distance table.

C — Floyd-Warshall Algorithm
void floydWarshall(int V, int adj[][MAX]) {
    int dist[MAX][MAX];

    /* Initialize distance matrix from adjacency matrix */
    for (int i = 0; i < V; i++)
        for (int j = 0; j < V; j++)
            dist[i][j] = adj[i][j];  /* INF if no edge */

    /* Try each vertex k as an intermediate */
    for (int k = 0; k < V; k++)
        for (int i = 0; i < V; i++)
            for (int j = 0; j < V; j++)
                if (dist[i][k] != INF && dist[k][j] != INF &&
                    dist[i][k] + dist[k][j] < dist[i][j])
                    dist[i][j] = dist[i][k] + dist[k][j];

    /* Print distance matrix */
    printf("All-Pairs Shortest Paths:\n");
    for (int i = 0; i < V; i++) {
        for (int j = 0; j < V; j++)
            dist[i][j] == INF ? printf("%5s", "INF") : printf("%5d", dist[i][j]);
        printf("\n");
    }
}

🧠 Advanced Operations

Transpose Graph

Reverse all directed edges. If original has A→B, transpose has B→A. Used in Kosaraju's SCC algorithm. Time: O(V+E).

🔄

Like reversing all one-way streets in a city — same intersections, opposite traffic flow.

Reverse Graph

Same as transpose — creates a new graph with all edge directions reversed. Essential for bidirectional reachability checks.

↩️

Like playing a video in reverse — same frames, opposite order of events.

Graph Coloring

Assign colors to vertices so no two adjacent vertices share the same color. Minimum colors needed = chromatic number. Used in scheduling and register allocation.

🎨

Like assigning exam time slots so no student has two exams at the same time.

Bipartite Check

A graph is bipartite if vertices can be split into two groups where edges only go between groups (not within). Check using 2-coloring BFS. Used in matching problems.

👥

Like a dating app — users and matches are two groups; connections only exist between groups.

Network Flow Basics Ford-Fulkerson Concept

A flow network is a directed graph where each edge has a capacity. Flow represents data/liquid/traffic moving from a source to a sink. The Max Flow is the maximum amount that can flow from source to sink without exceeding any edge's capacity.

Key Concepts:

  • Capacity: Maximum flow an edge can carry.
  • Flow: Actual amount flowing (≤ capacity).
  • Residual Graph: Shows remaining capacity for more flow.
  • Augmenting Path: A path from source to sink with available capacity.
🚰
Analogy: Water Pipes

Each pipe has a maximum capacity (litres/second). You want to push maximum water from source (reservoir) to sink (city). Network flow finds how much water can flow without bursting any pipe.

C — Transpose Graph & Bipartite Check
/* Transpose: reverse all edges */
struct Graph* transpose(struct Graph* g) {
    struct Graph* t = createGraph(g->numVertices);
    for (int v = 0; v < g->numVertices; v++) {
        struct Node* cur = g->adjList[v];
        while (cur) {
            insertEdge(t, cur->dest, v, cur->weight); /* flip direction */
            cur = cur->next;
        }
    }
    return t;
}

/* Bipartite Check using 2-coloring BFS */
int isBipartite(struct Graph* g) {
    int color[MAX];
    for (int i = 0; i < g->numVertices; i++) color[i] = -1;

    for (int start = 0; start < g->numVertices; start++) {
        if (color[start] != -1) continue;
        int q[MAX], f = 0, r = 0;
        q[r++] = start; color[start] = 0;  /* color 0 */
        while (f < r) {
            int v = q[f++];
            struct Node* cur = g->adjList[v];
            while (cur) {
                if (color[cur->dest] == -1) {
                    color[cur->dest] = 1 - color[v]; /* alternate color */
                    q[r++] = cur->dest;
                } else if (color[cur->dest] == color[v])
                    return 0;  /* same color adjacent → not bipartite */
                cur = cur->next;
            }
        }
    }
    return 1;  /* bipartite! */
}

🛠️ Utility Operations

Utility operations are helper functions that maintain and manage the graph structure:

  • Is Empty: Returns true if the graph has no vertices.
  • Clear Graph: Remove all vertices and edges, free all memory.
  • Copy Graph: Create a deep copy of the graph (all vertices and edges).
  • Display Graph: Print the full adjacency list structure.
  • Graph Size: Return number of vertices and/or edges.
C — All Utility Operations
/* Is the graph empty? */
int isEmpty(struct Graph* g) {
    return g->numVertices == 0;
}

/* Clear all edges and vertices */
void clearGraph(struct Graph* g) {
    for (int i = 0; i < g->numVertices; i++) {
        freeList(g->adjList[i]);
        g->adjList[i] = NULL;
    }
    g->numVertices = 0;
    printf("Graph cleared!\n");
}

/* Deep copy a graph */
struct Graph* copyGraph(struct Graph* src) {
    struct Graph* copy = createGraph(src->numVertices);
    for (int i = 0; i < src->numVertices; i++) {
        strcpy(copy->label[i], src->label[i]);
        /* Copy edges in reverse order to maintain original order */
        struct Node* cur = src->adjList[i];
        while (cur) {
            insertEdge(copy, i, cur->dest, cur->weight);
            cur = cur->next;
        }
    }
    return copy;
}

/* Get number of edges */
int graphSize(struct Graph* g, int* edges) {
    *edges = countEdges(g);
    return g->numVertices;
}

/* Display full graph */
void displayGraph(struct Graph* g) {
    int e;
    int v = graphSize(g, &e);
    printf("=== Graph: %d vertices, %d edges ===\n", v, e);
    for (int i = 0; i < g->numVertices; i++) {
        printf("[%d] %s", i, g->label[i]);
        struct Node* cur = g->adjList[i];
        while (cur) {
            printf(" → %d(w=%d)", cur->dest, cur->weight);
            cur = cur->next;
        }
        printf(" → NULL\n");
    }
}
