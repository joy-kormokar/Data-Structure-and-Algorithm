What is a Priority Queue?

A Priority Queue is a special data structure where every element has a priority, and elements are served based on their priority — not by the order they arrived (like a normal queue). The element with the highest priority is always dequeued first.

A Max Binary Heap is the most efficient way to implement a priority queue. It is a complete binary tree stored in an array, where every parent node has a value greater than or equal to its children. This guarantees the maximum element is always at the root (index 0).

🔺 Always serves highest first
🌳 Complete binary tree
📦 Stored in array
⚡ Insert: O(log n)
⚡ Extract Max: O(log n)
👁 Peek Max: O(1)
🏥
Real-Life Analogy — Hospital Emergency Room
Imagine an emergency room. Patients don't get treated in the order they arrived — they are treated by severity of condition (priority). A patient with a heart attack (priority 100) gets served before a patient with a sprained ankle (priority 5), even if the ankle patient arrived first. The priority queue works exactly like this: highest priority always goes first.

Core Concepts

1. Complete Binary Tree

A complete binary tree is a binary tree where all levels are fully filled except possibly the last level, which is filled from left to right. This structure is critical because it allows the tree to be stored compactly in a flat array without wasting space.

Key Property: In a complete binary tree of n nodes, the height is always ⌊log₂(n)⌋. This is why heap operations are O(log n).
2. Array Representation

A max heap is stored in a simple array (0-indexed). The root is at index 0. This mapping lets us navigate the tree using arithmetic — no pointers needed!

/* Parent-Child Index Formulas (0-based indexing) */ Parent of node i → (i - 1) / 2 Left child of i → 2 * i + 1 Right child of i → 2 * i + 2 /* Example: node at index 3 */ Parent = (3-1)/2 = 1 Left = 2*3+1 = 7 Right = 2*3+2 = 8

3. Max Heap Property

For every node i in the heap:
heap[parent(i)] ≥ heap[i]

This means every parent is greater than or equal to both its children. As a result, the maximum value is always at the root (index 0).

Root
Always Maximum
Leaves
Smallest values (not sorted)
Siblings
No ordering guaranteed
Tree height

/* Define maximum capacity of the heap */
#define MAX_SIZE 100

/* Heap structure: holds array, current size, and capacity */
typedef struct {
    int data[MAX_SIZE];  /* array to store heap elements */

OP 01

Create / Initialize Max Heap

Theory

Initialization sets up an empty heap by defining the storage array, setting capacity (maximum elements allowed) and size = 0 (no elements yet). Think of it as building an empty container before you store anything. No actual data is stored — just the structure is prepared.

  • 1Define the heap array with maximum capacity
  • 2Set capacity to the maximum number of elements
  • 3Set size = 0 — the heap starts empty
  • 4Choose priority rule: larger value = higher priority (max heap)
C Code — Initialize Max Heap
C Language
/* Initialize an empty max heap */
void initHeap(MaxHeap *h, int capacity) {
    h->size     = 0;          /* no elements yet */
    h->capacity = capacity;    /* set max limit */
    /* data[] array already allocated in struct */
    printf("Heap initialized. Capacity: %d\n", capacity);
}

/* Usage in main */
int main() {
    MaxHeap h;
    initHeap(&h, MAX_SIZE);    /* prepare empty heap */
    printf("Heap ready! Size = %d\n", h.size);
    return 0;
}
🗂️
Analogy — Empty Filing Cabinet
Creating a heap is like setting up a brand-new filing cabinet. You decide how many folders it can hold (capacity), and you start with zero files inside (size = 0). The cabinet is ready and waiting, but there's nothing in it yet.
OP 02

Insert Element

Theory

To insert a new element: place it at the last position of the array (to maintain the complete binary tree shape), then bubble it up (heapify-up) to its correct position. This is done by repeatedly comparing the new element with its parent and swapping if it is larger.

  • 1Check if heap is full (overflow check)
  • 2Place new element at index size (last position)
  • 3Increment size
  • 4Perform heapify-up: compare with parent, swap if larger, repeat
Time Complexity: O(log n) — in the worst case, the new element bubbles all the way from leaf to root.

C Code — Insert Element
C Language
/* Swap two integers */
void swap(int *a, int *b) {
    int temp = *a;
    *a = *b;
    *b = temp;
}

/* Insert a new element into the max heap */
void insert(MaxHeap *h, int value) {
    /* Step 1: Check for overflow */
    if (h->size >= h->capacity) {
        printf("Heap is FULL! Cannot insert %d\n", value);
        return;
    }

    /* Step 2: Place new element at last index */
    int i = h->size;
    h->data[i] = value;

    /* Step 3: Increase size */
    h->size++;

    /* Step 4: Heapify-Up — bubble the element up */
    while (i > 0) {
        int parent = PARENT(i);        /* (i-1)/2 */
        if (h->data[i] > h->data[parent]) {
            swap(&h->data[i], &h->data[parent]);
            i = parent;                 /* move up */
        } else {
            break;                      /* heap property satisfied */
        }
    }
    printf("Inserted: %d\n", value);
}
🎖️
Analogy — New Recruit in Army Ranks
When a new soldier joins, they start at the bottom. But if they outrank (have higher priority than) the officer above them, they trade places. They keep moving up until they find someone who outranks them — then they stop. The highest-ranking officer is always at the top.
OP 03

Extract Maximum (Delete Highest Priority)

Theory

Extraction removes and returns the root element (the maximum). To maintain the complete binary tree shape, we replace the root with the last element in the array, remove the last slot, then heapify-down to restore the heap property.

  • 1Check if heap is empty (underflow)
  • 2Store root value (this is the maximum)
  • 3Copy last element to root position
  • 4Decrease size by 1 (removes the last element)
  • 5Heapify-down from root to restore max heap property
  • 6Return the stored maximum value
Time Complexity: O(log n) — heapify-down may traverse full height of tree.
C Code — Extract Maximum
C Language
/* Forward declaration — defined in Section 06 */
void heapifyDown(MaxHeap *h, int i);

/* Extract (remove and return) the maximum element */
int extractMax(MaxHeap *h) {
    /* Step 1: Check underflow */
    if (h->size == 0) {
        printf("Heap is EMPTY!\n");
        return -1;
    }

    /* Step 2: Save the maximum (root) */
    int maxVal = h->data[0];

    /* Step 3: Replace root with last element */
    h->data[0] = h->data[h->size - 1];

    /* Step 4: Reduce size */
    h->size--;

    /* Step 5: Restore heap property from root down */
    if (h->size > 0)
        heapifyDown(h, 0);

    /* Step 6: Return the maximum */
    printf("Extracted max: %d\n", maxVal);
    return maxVal;
}
👑
Analogy — Presidential Election
When the current leader (root = max) steps down, the most recently added person is temporarily given leadership. But they then compete with those below them — whoever is stronger takes their place at the top, and this process cascades downward until the right leader is found again.
OP 04

Peek Maximum / Get Highest Priority

Theory

Peek (or "Get Max") simply reads the root element without removing it. Since the max heap property guarantees the maximum is always at index 0, this operation is instant — O(1). No restructuring needed.

int peekMax(MaxHeap *h) {
    /* Check empty heap */
    if (h->size == 0) {
        printf("Heap is EMPTY! No maximum.\n");
        return -1;

OP 05

Heapify Up (Bubble Up)

Theory

Heapify Up restores the max heap property when an element at position i might be larger than its parent. Starting at index i, we compare with the parent at (i-1)/2. If larger, we swap and move up. This repeats until we reach the root or the parent is ≥ current node.

  • 1Calculate parent index: parent = (i - 1) / 2
  • 2Compare current node value with parent value
  • 3If current > parent → swap them
  • 4Set i = parent and go back to step 1
  • 5Stop when i == 0 (reached root) or parent ≥ current
Used by: Insert and Increase Key operations.
C Code — Heapify Up
C Language
/* Heapify Up: restore heap from index i upward */
void heapifyUp(MaxHeap *h, int i) {
    /* Continue while i is not the root */
    while (i > 0) {
        int parent = PARENT(i);   /* parent = (i-1)/2 */

        /* If current node is greater than parent, swap */
        if (h->data[i] > h->data[parent]) {
            swap(&h->data[i], &h->data[parent]);
            i = parent;           /* move index up to parent */
        } else {
            break;              /* heap property satisfied, stop */
        }
    }
}

/* Recursive version (alternative approach) */
void heapifyUpRecursive(MaxHeap *h, int i) {
    if (i == 0) return;         /* base case: reached root */
    int parent = PARENT(i);
    if (h->data[i] > h->data[parent]) {
        swap(&h->data[i], &h->data[parent]);
        heapifyUpRecursive(h, parent);
    }
}
🚀
Analogy — Floating Balloon
Imagine you drop a helium balloon at the bottom of a tower of rooms. The balloon is lighter (higher priority) than the things above it, so it floats up past each floor, pushing down whatever it's lighter than, until it finds a floor where something lighter (higher priority) is already there — then it stops.
OP 06

Heapify Down (Bubble Down)

Theory

Heapify Down restores the max heap property when an element at position i might be smaller than its children. We find the largest among the node and its left/right children, then swap with the largest child. This repeats downward until the node is the largest in its local group.

  • 1Calculate left child: left = 2*i + 1
  • 2Calculate right child: right = 2*i + 2
  • 3Find largest among data[i], data[left], data[right]
  • 4If largest ≠ i → swap and continue from that child's position
  • 5Stop when current is largest or becomes a leaf
Used by: Extract Max, Decrease Key, Delete Specific Element, and Build Heap.
C Code — Heapify Down
C Language
/* Heapify Down: restore heap from index i downward */
void heapifyDown(MaxHeap *h, int i) {
    int largest = i;              /* assume current is largest */
    int left    = LEFT_CHILD(i);  /* 2*i + 1 */
    int right   = RIGHT_CHILD(i); /* 2*i + 2 */

    /* Check if left child exists and is larger */
    if (left < h->size && h->data[left] > h->data[largest])
        largest = left;

    /* Check if right child exists and is larger */
    if (right < h->size && h->data[right] > h->data[largest])
        largest = right;

    /* If a child is larger, swap and continue down */
    if (largest != i) {
        swap(&h->data[i], &h->data[largest]);
        heapifyDown(h, largest);  /* recurse downward */
    }
    /* If largest == i, heap property is restored */
}
💧
Analogy — Heavy Stone Sinking
Imagine a light pebble placed at the top of a pile of rocks. It's lighter than the rocks below it, so it keeps swapping with the heavier (lower priority) rock below it, sinking further and further down until it finds a spot where it's the heaviest among its neighbors.
OP 07

Build Max Heap

Theory

Build Max Heap converts an arbitrary array into a valid max heap efficiently. Instead of inserting elements one by one (which takes O(n log n)), we start from the last non-leaf node (at index (n/2) - 1) and call heapify-down on each node, moving toward the root. This achieves O(n) time.

Why not insert one by one? Inserting n elements individually takes O(n log n). Building from bottom-up takes only O(n) — that's why Build Heap is important!
  • 1Load all elements into the array (don't worry about order)
  • 2Find last non-leaf node: start = (n/2) - 1
  • 3Call heapify-down on every node from start down to 0
  • 4After all heapify-down calls, array is a valid max heap

C Code — Build Max Heap
C Language
/* Build a max heap from an unsorted array */
void buildMaxHeap(MaxHeap *h, int arr[], int n) {
    /* Copy elements into heap array */
    for (int i = 0; i < n; i++)
        h->data[i] = arr[i];
    h->size = n;

    /* Start from last non-leaf node: (n/2) - 1 */
    /* Leaf nodes don't need heapify (no children) */
    for (int i = (n / 2) - 1; i >= 0; i--) {
        heapifyDown(h, i);   /* fix heap property */
    }
    printf("Max heap built from %d elements.\n", n);
    printf("Maximum element: %d\n", h->data[0]);
}

/* Example usage */
int main() {
    MaxHeap h;
    initHeap(&h, MAX_SIZE);
    int arr[] = {3, 1, 6, 5, 2, 4};
    int n = 6;
    buildMaxHeap(&h, arr, n);
    /* Result: [6, 5, 4, 1, 2, 3] — valid max heap */
    return 0;
}
🏗️
Analogy — Reorganizing a Messy Bookshelf
Imagine a messy bookshelf where you want the tallest books at top positions. Instead of removing all books and re-inserting (slow), you work from the middle shelf downward — at each shelf, you push the shorter book down and pull the taller one up. After going through every shelf once, the bookshelf is perfectly organized.
OP 08

Increase Key / Increase Priority

Theory

Increase Key updates a node's value to a larger value, then restores heap order. Since increasing a value can only violate the heap property upward (the node might now be larger than its parent), we call heapify-up after the update.

  • 1Locate target node at index i
  • 2Verify new value > old value (it must be an increase)
  • 3Update h->data[i] = newValue
  • 4Call heapify-up from index i
Time Complexity: O(log n) — heapify-up can go up to root level.
C Code — Increase Key
C Language
/* Increase the value of node at index i to newValue */
void increaseKey(MaxHeap *h, int i, int newValue) {
    /* Validate index */
    if (i < 0 || i >= h->size) {
        printf("Invalid index!\n");
        return;
    }

    /* New value must be larger */
    if (newValue < h->data[i]) {
        printf("Error: New value is smaller. Use decreaseKey.\n");
        return;
    }

    /* Update the value */
    printf("Increasing data[%d] from %d to %d\n",
           i, h->data[i], newValue);
    h->data[i] = newValue;

    /* Bubble up since value increased */
    heapifyUp(h, i);
}
📈
Analogy — Employee Promotion
An employee gets a promotion (increase key). Their new rank might be higher than their manager's rank, so they move up in the organizational chart, swapping with managers above them until they find a position where the manager above them still outranks them.
OP 09

Decrease Key

Theory

Decrease Key reduces a node's value, then restores heap order. A decreased value might now be smaller than the node's children, so we call heapify-down after the update. This is the opposite of Increase Key.

  • 1Locate target node at index i
  • 2Verify new value < old value (it must be a decrease)
  • 3Update h->data[i] = newValue
  • 4Call heapify-down from index i
Time Complexity: O(log n)
C Code — Decrease Key
C Language
/* Decrease the value of node at index i to newValue */
void decreaseKey(MaxHeap *h, int i, int newValue) {
    /* Validate index */
    if (i < 0 || i >= h->size) {
        printf("Invalid index!\n");
        return;
    }

    /* New value must be smaller */
    if (newValue > h->data[i]) {
        printf("Error: New value is larger. Use increaseKey.\n");
        return;
    }

    /* Update the value */
    printf("Decreasing data[%d] from %d to %d\n",
           i, h->data[i], newValue);
    h->data[i] = newValue;

    /* Bubble down since value decreased */
    heapifyDown(h, i);
}
📉
Analogy — Employee Demotion
An employee is demoted (decrease key). Their new lower rank means some of their subordinates may now outrank them. Those subordinates bubble up past them, pushing the demoted employee downward in the chart until they settle at the right level.
OP 10

Change Priority

Theory

Change Priority is the general operation that handles both increases and decreases. It first searches for the target element, compares the old and new priority, then intelligently delegates to either increaseKey or decreaseKey.

  • 1Search for target element — get its index
  • 2Compare old priority with new priority
  • 3If new > old → call increaseKey → heapify-up
  • 4If new < old → call decreaseKey → heapify-down
  • 5If equal → no action needed
C Code — Change Priority
C Language
/* Search for a value and return its index (-1 if not found) */
int searchIndex(MaxHeap *h, int value) {
    for (int i = 0; i < h->size; i++)
        if (h->data[i] == value) return i;
    return -1;  /* not found */
}

/* Change priority of an element */
void changePriority(MaxHeap *h, int oldVal, int newVal) {
    int idx = searchIndex(h, oldVal);
    if (idx == -1) {
        printf("Element %d not found!\n", oldVal);
        return;
    }
    if (newVal > oldVal) {
        increaseKey(h, idx, newVal);  /* heapify-up */
    } else if (newVal < oldVal) {
        decreaseKey(h, idx, newVal);  /* heapify-down */
    } else {
        printf("Priority unchanged.\n");
    }
}
🔄
Analogy — Hospital Priority Update
A hospital patient's condition changes. If they get worse (higher priority), they move up in the treatment queue. If they improve (lower priority), they move down. The triage system re-evaluates and repositions them automatically.
OP 11

Delete Specific Element

Theory

To delete any specific element (not just the max), we use a trick: increase its value to infinity (or a very large number) so it bubbles to the root, then call extractMax. Alternatively: replace the target with the last element, reduce size, then heapify-up or heapify-down as needed.

  • 1Locate target node at index i
  • 2Replace h->data[i] with last element h->data[size-1]
  • 3Reduce heap size by 1
  • 4If replacement is larger than parent → heapify-up
  • 5If replacement is smaller than a child → heapify-down
Time Complexity: O(n) for search + O(log n) for restructuring = O(n) overall.
C Code — Delete Specific Element
C Language
/* Delete a specific value from the heap */
void deleteElement(MaxHeap *h, int value) {
    /* Step 1: Locate target */
    int i = searchIndex(h, value);
    if (i == -1) {
        printf("Value %d not found!\n", value);
        return;
    }

    /* Step 2: Replace with last element */
    int lastVal = h->data[h->size - 1];
    h->data[i] = lastVal;

    /* Step 3: Reduce size */
    h->size--;

    /* Step 4 & 5: Restore heap property */
    if (h->size > 0) {
        /* Check if we need to go up or down */
        int parent = PARENT(i);
        if (i > 0 && h->data[i] > h->data[parent])
            heapifyUp(h, i);
        else
            heapifyDown(h, i);
    }
    printf("Deleted element: %d\n", value);
}
🪑
Analogy — Removing a Seat at a Conference
If someone in the middle of a priority seating arrangement leaves, the person from the last seat fills the gap. That person then shifts up or down depending on whether they outrank their neighbors — until the seating order is valid again.
OP 12

Search Element

Theory

Search finds the location (index) of an element in the heap. Since a max heap is not fully sorted — only the parent-child relationship is guaranteed — we cannot use binary search. We must do a linear scan through the array. There's no shortcut.

Algorithm
Linear scan
Time Complexity
O(n)
Returns
Index of element, or -1
Binary Search?
❌ Not applicable
C Code — Search Element
C Language
/* Search for a value in the heap array */
int searchElement(MaxHeap *h, int target) {
    /* Linear scan through the heap array */
    for (int i = 0; i < h->size; i++) {
        if (h->data[i] == target) {
            printf("Found %d at index %d\n", target, i);
            return i;   /* return index location */
        }
    }
    /* Not found */
    printf("Element %d NOT found in heap.\n", target);
    return -1;
}
🔍
Analogy — Finding a Person in a Crowd
The crowd is loosely ordered (tallest people tend to be toward the front), but you can't use that ordering to jump directly to someone — you still have to look at people one by one until you find who you're searching for.
OP 13

Merge Max Heaps

Theory

Merge combines two max heaps into a single max heap. The simplest approach: combine both arrays into one, then call buildMaxHeap on the combined array. The result is a valid max heap containing all elements from both.

  • 1Check combined size doesn't exceed capacity
  • 2Copy elements of heap2 into heap1's array (after heap1's elements)
  • 3Update heap1's size to combined total
  • 4Run buildMaxHeap (heapify-down from last non-leaf to root)
Time Complexity: O(n) using buildMaxHeap on the merged array.
C Code — Merge Max Heaps
C Language
/* Merge heap2 into heap1 (heap1 gets all elements) */
void mergeHeaps(MaxHeap *h1, MaxHeap *h2) {
    /* Check if combined fits in capacity */
    if (h1->size + h2->size > h1->capacity) {
        printf("Not enough capacity to merge!\n");
        return;
    }

    /* Copy h2 elements into h1 after h1's existing elements */
    for (int i = 0; i < h2->size; i++) {
        h1->data[h1->size + i] = h2->data[i];
    }

    /* Update total size */
    h1->size = h1->size + h2->size;

    /* Rebuild heap from combined array — O(n) */
    for (int i = (h1->size / 2) - 1; i >= 0; i--) {
        heapifyDown(h1, i);
    }
    printf("Merged! Total elements: %d\n", h1->size);
    printf("New max: %d\n", h1->data[0]);
}
🏢
Analogy — Company Merger
Two companies merge. All employees from both companies are now in one organization. The combined staff needs to be re-organized so that every manager still outranks their direct reports. The new org chart is built efficiently from the bottom up.

Theory

OP 15

Check Full

Theory

Checking if the heap is full prevents insertion overflow. If size == capacity, no more elements can be added without expanding the array (which requires dynamic memory allocation in C).

Condition
size == capacity
Time Complexity
O(1)
C Code — Check Full
C Language
/* Returns 1 if heap is full, 0 otherwise */
int isFull(MaxHeap *h) {
    return (h->size == h->capacity);
}

/* Usage example */
if (isFull(&h)) {
    printf("Heap is FULL! Cannot insert.\n");
} else {
    insert(&h, 42);
}
🧳
Analogy — Full Suitcase
Before packing more clothes, you check if your suitcase is full. If it's at capacity, you can't fit anything else without either removing something or getting a bigger suitcase. Check first, act second.
OP 16

Theory

A bouncer keeps a running count of people in the venue. Every time someone enters, count goes up; every time someone leaves, count goes down. To know how many people are inside, they just check that counter — no need to recount everyone.
OP 17

Traverse Heap

Theory

Since the heap is stored as an array, traversal is straightforward. A sequential array traversal (index 0 to size-1) naturally produces a level-order (BFS) traversal of the binary tree, because of how the array represents tree levels.

Array index 0
Level 0 — root
Indices 1–2

Validate Max Heap Property

Theory

Validation checks whether an array currently satisfies the max heap property — meaning every parent is greater than or equal to both its children. This is useful for debugging or testing. We scan all internal nodes (indices 0 to n/2-1) and verify each against their children.

  • 1Loop through all non-leaf nodes: index 0 to (size/2)-1
  • 2For each node, check left child: data[i] ≥ data[2i+1]
  • 3Check right child if it exists: data[i] ≥ data[2i+2]
  • 4If any violation found → return 0 (invalid)
  • 5If all pass → return 1 (valid max heap)
C Code — Validate Max Heap
C Language
/* Returns 1 if valid max heap, 0 if violated */
int isValidMaxHeap(MaxHeap *h) {
    /* Check all internal (non-leaf) nodes */
    for (int i = 0; i < h->size / 2; i++) {
        int left  = LEFT_CHILD(i);   /* 2*i + 1 */
        int right = RIGHT_CHILD(i);  /* 2*i + 2 */

        /* Parent must be >= left child */
        if (left < h->size &&
            h->data[i] < h->data[left]) {
            printf("Violation at i=%d, left=%d\n", i, left);
            return 0;
        }
        /* Parent must be >= right child */
        if (right < h->size &&
            h->data[i] < h->data[right]) {
            printf("Violation at i=%d, right=%d\n", i, right);
            return 0;
        }
    }
    printf("Valid Max Heap!\n");
    return 1;
}

Convert to Priority Queue View

Theory

Priority Queue View displays elements ordered from highest priority to lowest — essentially simulating the order in which elements would be extracted if you kept calling Extract Max. This is done by repeatedly calling extractMax on a copy of the heap (to preserve the original), collecting results in order.

  • 1Create a copy of the current heap
  • 2Repeatedly call extractMax on the copy
  • 3Each extraction gives the next-highest element
  • 4Collect and display results — highest to lowest
This is essentially Heap Sort! Heap sort uses exactly this technique to sort an array in O(n log n).
