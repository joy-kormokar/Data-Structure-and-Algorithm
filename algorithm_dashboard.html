Function, Recursion & GCD

The Art of Self-Reference

Beginner

What is Recursion?

A recursive function is a function that calls itself to solve a smaller version of the same problem. Every recursive function needs:

  • Base Case — the stopping condition. Without it, the function runs forever (stack overflow!).
  • Recursive Case — the part where the function calls itself with a simpler input.
f(n) = f(n−1) + f(n−2)  [Fibonacci]
GCD(a, b) = GCD(b, a mod b)
🪆

Real-Life Analogy — Russian Dolls

Imagine Russian nesting dolls. To find the smallest doll, you keep opening each doll (recursive call) until you reach one that doesn't open (base case). Recursion works the same way — keep breaking the problem down until it's trivially small.

📞

GCD Analogy — Rope Cutting

You have two ropes of lengths 48 and 18. What's the longest piece you can cut both ropes into evenly? That's GCD! The Euclidean algorithm finds this without trying every possible length.

recursion_gcd.c
// ── FACTORIAL (Recursive) ──
int factorial(int n) {
    // Base case
    if (n == 0 || n == 1) return 1;
    // Recursive case
    return n * factorial(n - 1);
}

// ── GCD (Euclidean Algorithm) ──
int gcd(int a, int b) {
    // Base case: b becomes 0
    if (b == 0) return a;
    // Recursive case: swap & mod
    return gcd(b, a % b);
}

int main() {
    int a = 48, b = 18;
    printf("Factorial(5) = %d\n", factorial(5));
    printf("GCD(%d, %d) = %d\n", a, b, gcd(a, b));
    return 0;
}
/* Output:
   Factorial(5) = 120
   GCD(48, 18) = 6  */

Searching Algorithms

Linear Search: Check every element one by one from start to end. Simple but slow for large arrays.

Time: O(n)  Space: O(1)

Binary Search: Only works on sorted arrays. Cut the search space in half each step. Like guessing a number between 1–100.

Time: O(log n)  Space: O(1)

// LINEAR SEARCH
int linearSearch(int arr[], int n, int target) {
    for (int i = 0; i < n; i++)
        if (arr[i] == target) return i; // found!
    return -1; // not found
}

// BINARY SEARCH (array must be sorted!)
int binarySearch(int arr[], int n, int target) {
    int low = 0, high = n - 1;
    while (low <= high) {
        int mid = (low + high) / 2;
        if (arr[mid] == target) return mid;
        else if (arr[mid] < target) low = mid + 1;
        else high = mid - 1;
    }
    return -1;
}

// BUBBLE SORT — repeatedly swap adjacent wrong-order elements
void bubbleSort(int arr[], int n) {
    for (int i = 0; i < n-1; i++)
        for (int j = 0; j < n-i-1; j++)
            if (arr[j] > arr[j+1]) {
                int tmp = arr[j]; arr[j] = arr[j+1]; arr[j+1] = tmp;
            }
}

// SELECTION SORT — find min, place at front
void selectionSort(int arr[], int n) {
    for (int i = 0; i < n-1; i++) {
        int minIdx = i;
        for (int j = i+1; j < n; j++)
            if (arr[j] < arr[minIdx]) minIdx = j;
        int tmp = arr[minIdx]; arr[minIdx] = arr[i]; arr[i] = tmp;
    }
}

// INSERTION SORT — build sorted portion left to right
void insertionSort(int arr[], int n) {
    for (int i = 1; i < n; i++) {
        int key = arr[i], j = i-1;
        while (j >= 0 && arr[j] > key) { arr[j+1] = arr[j]; j--; }
        arr[j+1] = key;
    }
}

Divide and Conquer

Merge Sort, Quick Sort & Recurrences

Intermediate

The Strategy

Divide: Split the problem into smaller sub-problems of the same type.

Conquer: Solve each sub-problem recursively (until base case).

Combine: Merge the solutions of sub-problems into the final answer.

📚

Analogy — Organizing a Library

You have 1000 unsorted books. Give 500 to your left team and 500 to your right team. Each team splits again... until teams of 1 book (already sorted!). Then merge them in order. That's Merge Sort!

T(n) = 2T(n/2) + O(n) → O(n log n)

merge_sort.c
void merge(int arr[], int l, int m, int r) {
    int n1=m-l+1, n2=r-m;
    int L[n1], R[n2];
    for(int i=0;i<n1;i++) L[i]=arr[l+i];
    for(int j=0;j<n2;j++) R[j]=arr[m+1+j];
    int i=0, j=0, k=l;
    while(i<n1 && j<n2) {
        if(L[i]<=R[j]) arr[k++]=L[i++];
        else arr[k++]=R[j++];
    }
    while(i<n1) arr[k++]=L[i++];
    while(j<n2) arr[k++]=R[j++];
}

void mergeSort(int arr[], int l, int r) {
    if(l < r) {
        int m = (l+r)/2;
        mergeSort(arr, l, m);
        mergeSort(arr, m+1, r);
        merge(arr, l, m, r);
    }
}
quick_sort.c
int partition(int arr[],int low,int high) {
    int pivot=arr[high], i=low-1;
    for(int j=low; j<high; j++) {
        if(arr[j] < pivot) {
            i++;
            // swap arr[i] and arr[j]
            int tmp=arr[i];
            arr[i]=arr[j]; arr[j]=tmp;
        }
    }
    // place pivot in correct spot
    int tmp=arr[i+1];
    arr[i+1]=arr[high]; arr[high]=tmp;
    return i+1;
}

void quickSort(int arr[],int low,int high){
    if(low < high) {
        int pi = partition(arr,low,high);
        quickSort(arr, low, pi-1);
        quickSort(arr, pi+1, high);
    }
}

// Best/Avg: O(n log n)  Worst: O(n²)

Master Theorem — Solving Recurrence Relations

For recurrences of the form T(n) = aT(n/b) + f(n), where a≥1, b>1:

Compare f(n) with n^(log_b a)
Case 1
f(n) < n^log_b(a)
T(n) = Θ(n^log_b a)
Case 2
f(n) = n^log_b(a)
T(n) = Θ(n^log_b a · log n)
Case 3
f(n) > n^log_b(a)
T(n) = Θ(f(n))
Example — Merge Sort: T(n) = 2T(n/2) + n → a=2, b=2, f(n)=n, n^log₂2=n → Case 2 → T(n) = Θ(n log n)

Greedy Approach

Coin Change, Knapsack, Bin Packing & Huffman

Intermediate

Greedy Strategy

A greedy algorithm makes the locally best choice at each step, hoping it leads to a globally optimal solution. It never looks back — once a choice is made, it's final.

Look at all available options

Pick the one that seems best RIGHT NOW

Move on — never reconsider

🪙

Analogy — Making Change

A cashier giving change for $0.67 uses: 2 quarters (50¢) + 1 dime (60¢) + 1 nickel (65¢) + 2 pennies (67¢). At each step, pick the largest coin that doesn't exceed the remaining amount. That's the Greedy Coin Change!

🎒

Knapsack Analogy

A thief with a limited backpack takes items by highest value-per-weight ratio first. (Greedy works for the Fractional Knapsack, but NOT the 0/1 Knapsack — that needs DP!)

int coins[] = {25, 10, 5, 1}; // quarters, dimes, nickels, pennies
void coinChange(int amount) {
    for (int i = 0; i < 4; i++) {
        int count = amount / coins[i];
        amount = amount % coins[i];
        printf("%d coin(s) of %d cents\n", count, coins[i]);
    }
}

// ── FRACTIONAL KNAPSACK (Greedy) ──
typedef struct { int weight, value; } Item;
double fractionalKnapsack(Item items[], int n, int W) {
    // Sort by value/weight ratio (highest first)
    for(int i=0;i<n-1;i++) for(int j=0;j<n-i-1;j++)
        if ((double)items[j].value/items[j].weight <
            (double)items[j+1].value/items[j+1].weight) {
            Item t=items[j]; items[j]=items[j+1]; items[j+1]=t;
        }
    double totalValue=0; int curW=0;
    for(int i=0;i<n;i++) {
        if(curW + items[i].weight <= W) {
            totalValue += items[i].value; curW += items[i].weight;
        } else {
            int rem = W - curW;
            totalValue += (double)items[i].value * rem / items[i].weight;
            break;
        }
    }
    return totalValue;
}

Dynamic Programming

Fibonacci, Coin Change, Knapsack, Memoization vs Tabulation

Intermediate

What is DP?

Dynamic Programming solves problems by breaking them into overlapping subproblems and storing their solutions to avoid redundant computation. It's smarter than brute force and more powerful than greedy.

Two approaches:

  • Memoization (Top-Down): Recursion + cache. Solve big problem, cache sub-answers as you go.
  • Tabulation (Bottom-Up): Fill a table from smallest to largest. Iterative, no recursion stack.
📝

Analogy — Exam Cheat Sheet

Imagine calculating Fibonacci(10) by hand. You'd compute Fibonacci(8) twice, Fibonacci(7) three times... waste! With DP, you write each answer on a cheat sheet (cache/table) and just look it up next time. That's memoization!

🎒

0/1 Knapsack Analogy

A hiker with a 10kg backpack considers each item: "Should I take it or not?" For each item+capacity combo, compute the optimal value. Build a table of decisions — that's the DP table.

Fibonacci — Naive vs DP (See the difference!)

// ── FIBONACCI — Memoization (Top-Down) ──
int memo[100];
int fibMemo(int n) {
    if (n <= 1) return n;
    if (memo[n] != -1) return memo[n]; // cache hit!
    memo[n] = fibMemo(n-1) + fibMemo(n-2);
    return memo[n];
}

// ── FIBONACCI — Tabulation (Bottom-Up) ──
int fibTable(int n) {
    int dp[n+1];
    dp[0]=0; dp[1]=1;
    for(int i=2; i<=n; i++) dp[i] = dp[i-1] + dp[i-2];
    return dp[n];
}

// ── 0/1 KNAPSACK ──
int knapsack(int W, int wt[], int val[], int n) {
    int dp[n+1][W+1];
    for(int i=0; i<=n; i++) {
        for(int w=0; w<=W; w++) {
            if(i==0 || w==0) dp[i][w]=0;
            else if(wt[i-1] <= w)
                dp[i][w] = max(val[i-1]+dp[i-1][w-wt[i-1]], dp[i-1][w]);
            else
                dp[i][w] = dp[i-1][w];
        }
    }
    return dp[n][W];
}

// ── COIN CHANGE — DP (minimum coins) ──
int coinChangeDP(int coins[], int m, int amount) {
    int dp[amount+1];
    dp[0] = 0;
    for(int i=1; i<=amount; i++) dp[i] = amount+1; // "infinity"
    for(int i=1; i<=amount; i++)
        for(int j=0; j<m; j++)
            if(coins[j] <= i)
                dp[i] = min(dp[i], dp[i-coins[j]] + 1);
    return (dp[amount] > amount) ? -1 : dp[amount];
}

Advanced DP Problems

LCS & LIS

Intermediate

Longest Common Subsequence (LCS)

Given two strings, find the length of the longest subsequence present in both. A subsequence doesn't need to be contiguous.

LCS("ABCBDAB", "BDCAB") = 4 → "BCAB"

If characters match: 1 + LCS of remaining

If they don't match: max(skip from s1, skip from s2)

Time: O(m×n)  Space: O(m×n)

Longest Increasing Subsequence (LIS)

Given an array, find the length of the longest subsequence where elements are in increasing order.

LIS([3,10,2,1,20]) = 3 → [3,10,20]

Every element is a subsequence of length at least 1

For each element, find longest subsequence ending at smaller previous elements

Time: O(n²)  Optimized: O(n log n)

// ── LONGEST COMMON SUBSEQUENCE ──
int lcs(char *X, char *Y, int m, int n) {
    int dp[m+1][n+1];
    for(int i=0; i<=m; i++) {
        for(int j=0; j<=n; j++) {
            if(i==0 || j==0) dp[i][j]=0;
            else if(X[i-1]==Y[j-1]) dp[i][j] = 1+dp[i-1][j-1];
            else dp[i][j] = max(dp[i-1][j], dp[i][j-1]);
        }
    }
    return dp[m][n];
}

// ── LONGEST INCREASING SUBSEQUENCE ──
int lis(int arr[], int n) {
    int dp[n], ans=1;
    for(int i=0; i<n; i++) dp[i]=1;
    for(int i=1; i<n; i++)
        for(int j=0; j<i; j++)
            if(arr[j] < arr[i])
                dp[i] = max(dp[i], dp[j]+1);
    for(int i=0; i<n; i++) ans=max(ans, dp[i]);
    return ans;
}

Graph Theory Fundamentals

Representation & Types

Intermediate

What is a Graph?

A graph G = (V, E) consists of Vertices (V) (nodes) and Edges (E) (connections between nodes). Graphs model networks: roads, social connections, internet links, dependencies.

Directed Graph

Edges have direction (A→B ≠ B→A). Like Twitter follows — you can follow someone without them following back.

Undirected Graph

Edges are bidirectional. Like Facebook friends — if you're friends, it goes both ways.

Weighted Graph

Each edge has a cost/distance. Like Google Maps roads — each road has a distance or travel time.

Graph Traversal

BFS & DFS — Exploring Every Node

Intermediate

Breadth-First Search (BFS)

Explore nodes level by level. Use a Queue. Visit all neighbors of the current node before going deeper.

Queue: FIFO — First In, First Out

Start at source, mark visited, enqueue

Dequeue, visit all unvisited neighbors, enqueue them

Repeat until queue is empty

Time: O(V+E)

Depth-First Search (DFS)

Go as deep as possible before backtracking. Use a Stack (or recursion).

Stack: LIFO — Last In, First Out

Start at source, mark visited

Visit an unvisited neighbor, recurse deeper

Backtrack when no unvisited neighbors remain

Time: O(V+E)
🌊

BFS vs DFS Analogy — Exploring a Cave

BFS: You explore a cave by sending scouts out equally in all directions — first all rooms 1 step away, then all 2 steps away. Like ripples on water.

DFS: You walk as far down one tunnel as possible, hitting a dead end, then backtrack and try the next tunnel. Like a spelunker going deep before exploring alternatives.

int adj[V][V]; // adjacency matrix
int visited[V];

// ── BFS ── (uses queue)
void bfs(int start) {
    int queue[V], front=0, rear=0;
    visited[start]=1;
    queue[rear++]=start;
    while(front < rear) {
        int v=queue[front++];
        printf("%d ", v);
        for(int i=0; i<V; i++)
            if(adj[v][i] && !visited[i]) {
                visited[i]=1;
                queue[rear++]=i;
            }
    }
}

// ── DFS ── (uses recursion/stack)
void dfs(int v) {
    visited[v]=1;
    printf("%d ", v);
    for(int i=0; i<V; i++)
        if(adj[v][i] && !visited[i])
            dfs(i); // recurse deeper!
}

Graph Applications

Cycle Detection, Topological Sort & SCC

Advanced

Cycle Detection

Detect if there's a cycle in a graph. Use DFS + a "recursion stack" array for directed graphs, or DFS tracking parent for undirected graphs.

🔄

Analogy: In a project dependency graph, a cycle means A depends on B, B depends on C, and C depends on A — deadlock! We must detect this.

Topological Sort

Order vertices of a Directed Acyclic Graph (DAG) so that for every edge u→v, u comes before v in the ordering.

🎓

Analogy: Course prerequisites! You must take Math 101 before Calculus, and Calculus before Physics. Topological sort gives a valid order to take courses.

Connected Components

Find all disconnected groups of nodes. Run DFS/BFS from each unvisited node — each full traversal = one component.

Use case: Finding isolated subnetworks, clustering users in social graphs.

Strongly Connected Components (SCC)

In a directed graph, a SCC is a group where every node can reach every other node. Kosaraju's or Tarjan's algorithm finds all SCCs.

Use case: Web crawling (finding clusters of mutually linking pages), circuit design.

// TOPOLOGICAL SORT — Kahn's (BFS-based)
#define V 6
int adj[V][V], inDeg[V];

void topoSort() {
    int queue[V], front=0, rear=0;
    for(int i=0;i<V;i++)
        if(inDeg[i]==0) queue[rear++]=i;
    while(front < rear) {
        int v=queue[front++];
        printf("%d ", v);
        for(int i=0;i<V;i++)
            if(adj[v][i]) {
                inDeg[i]--;
                if(inDeg[i]==0) queue[rear++]=i;
            }
    }
}

// CYCLE DETECTION (Directed Graph)
int recStack[V];
int hasCycle(int v) {
    visited[v]=1; recStack[v]=1;
    for(int i=0;i<V;i++) {
        if(adj[v][i] && !visited[i] && hasCycle(i)) return 1;
        if(adj[v][i] && recStack[i]) return 1; // back edge!
    }
    recStack[v]=0; return 0;
}

Minimum Spanning Tree

Kruskal's & Prim's Algorithms

Advanced

What is a Spanning Tree?

A spanning tree of a graph is a subgraph that connects all vertices with the minimum number of edges (no cycles). A Minimum Spanning Tree (MST) has the smallest total edge weight.

💡

Analogy: A power company needs to connect 5 cities with electrical cables. The MST shows the cheapest way to connect all cities without redundant cables.

// ── KRUSKAL'S ALGORITHM (uses Union-Find) ──
typedef struct { int src, dest, weight; } Edge;
int parent[V], rank[V];

int find(int x) { // with path compression
    if(parent[x]!=x) parent[x]=find(parent[x]);
    return parent[x];
}
void unite(int x, int y) {
    int px=find(x), py=find(y);
    if(rank[px]<rank[py]) parent[px]=py;
    else if(rank[px]>rank[py]) parent[py]=px;
    else { parent[py]=px; rank[px]++; }
}
void kruskal(Edge edges[], int E) {
    for(int i=0;i<V;i++) { parent[i]=i; rank[i]=0; }
    // Sort edges by weight first...
    int mstCost=0, cnt=0;
    for(int i=0; i<E && cnt<V-1; i++) {
        int u=edges[i].src, v=edges[i].dest;
        if(find(u) != find(v)) { // no cycle!
            unite(u,v);
            mstCost += edges[i].weight; cnt++;
            printf("%d--%d: %d\n",u,v,edges[i].weight);
        }
    }
}

// ── PRIM'S ALGORITHM ──
void prim(int graph[][V]) {
    int key[V], inMST[V], parent[V];
    for(int i=0;i<V;i++) { key[i]=INT_MAX; inMST[i]=0; }
    key[0]=0; parent[0]=-1;
    for(int cnt=0; cnt<V-1; cnt++) {
        // Pick min key vertex not in MST
        int u=-1;
        for(int v=0;v<V;v++)
            if(!inMST[v]&&(u==-1||key[v]<key[u])) u=v;
        inMST[u]=1;
        for(int v=0;v<V;v++)
            if(graph[u][v]&&!inMST[v]&&graph[u][v]<key[v]) {
                key[v]=graph[u][v]; parent[v]=u;
            }
    }
}

Single Source Shortest Path

Dijkstra's & Bellman-Ford Algorithms

Advanced

Dijkstra's Algorithm

Finds shortest path from a source to all other nodes. Works with non-negative weights. Uses a greedy approach with a priority queue.

dist[v] = min(dist[v], dist[u] + weight(u,v))
O((V + E) log V) with min-heap
🗺️

Google Maps! Find shortest route from home to office. At each intersection, always explore the closest unvisited one next.

Bellman-Ford Algorithm

Handles graphs with negative weights. Relaxes all edges V-1 times. Can also detect negative weight cycles!

Relax V-1 times: dist[v] = dist[u] + w(u,v)
O(V × E) — slower than Dijkstra
💱

Currency exchange arbitrage detection! Negative cycles = infinite profit loop → Bellman-Ford detects this impossibility.

#define INF 99999

// ── DIJKSTRA'S ALGORITHM ──
void dijkstra(int graph[][V], int src) {
    int dist[V], visited[V];
    for(int i=0;i<V;i++) { dist[i]=INF; visited[i]=0; }
    dist[src]=0;
    for(int cnt=0; cnt<V-1; cnt++) {
        // Find unvisited vertex with min distance
        int u=-1;
        for(int v=0;v<V;v++)
            if(!visited[v]&&(u==-1||dist[v]<dist[u])) u=v;
        visited[u]=1;
        // Relax all edges from u
        for(int v=0;v<V;v++)
            if(graph[u][v] && !visited[v] &&
               dist[u]+graph[u][v] < dist[v])
                dist[v] = dist[u]+graph[u][v];
    }
    for(int i=0;i<V;i++)
        printf("%d -> %d: %d\n",src,i,dist[i]);
}

// ── BELLMAN-FORD ALGORITHM ──
void bellmanFord(Edge edges[], int E, int src) {
    int dist[V];
    for(int i=0;i<V;i++) dist[i]=INF;
    dist[src]=0;
    for(int i=1;i<=V-1;i++) // V-1 iterations
        for(int j=0;j<E;j++)
            if(dist[edges[j].src]+edges[j].weight < dist[edges[j].dest])
                dist[edges[j].dest]=dist[edges[j].src]+edges[j].weight;
    // Check for negative cycle
    for(int j=0;j<E;j++)
        if(dist[edges[j].src]+edges[j].weight < dist[edges[j].dest])
            printf("Negative cycle detected!\n");
}

Floyd-Warshall Algorithm

Finds shortest paths between ALL pairs of vertices. Works with both positive and negative weights (but not negative cycles). Uses DP — builds solution through intermediate vertices.

dist[i][j] = min(dist[i][j], dist[i][k] + dist[k][j])

"Can I get from i to j faster by going through k?"

O(V³) time — only practical for small graphs
✈️

Real-Life Analogy — Flight Routes

You're an airline pricing algorithm. You need to know the cheapest flight price between EVERY pair of cities, possibly through layovers. Floyd-Warshall tries every possible intermediate city (k) to see if a route through it is cheaper than the direct route.

🌐

Network routing: Routers use this concept to compute optimal packet routing tables for all source-destination pairs in a network.

#define V 4
#define INF 99999

void floydWarshall(int graph[][V]) {
    int dist[V][V];
    // Initialize dist same as graph
    for(int i=0;i<V;i++)
        for(int j=0;j<V;j++)
            dist[i][j] = graph[i][j];

    // Try each vertex k as intermediate
    for(int k=0;k<V;k++)         // intermediate
        for(int i=0;i<V;i++)     // source
            for(int j=0;j<V;j++) // dest
                if(dist[i][k]+dist[k][j] < dist[i][j])
                    dist[i][j] = dist[i][k]+dist[k][j];

    // Print result matrix
    printf("Shortest distances:\n");
    for(int i=0;i<V;i++) {
        for(int j=0;j<V;j++)
            (dist[i][j]==INF) ? printf("INF ") : printf("%3d ",dist[i][j]);
        printf("\n");
    }
}

int main() {
    int graph[V][V] = {
        {0,   3,   INF, 7  },
        {8,   0,   2,   INF},
        {5,   INF, 0,   1  },
        {2,   INF, INF, 0  }
    };
    floydWarshall(graph);
    return 0;
}
/* Time: O(V³)   Space: O(V²)
   Works with negative weights!
   Cannot handle negative cycles. */
