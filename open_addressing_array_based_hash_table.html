What is a Hash Table?

A Hash Table (also called a Hash Map) is a data structure that stores data as key-value pairs — like a dictionary. You give it a key, it instantly tells you the value. That's the magic.

The secret behind this speed is a hash function — a formula that converts a key (like a word or number) into an array index (a slot number). Instead of searching through all items, it goes directly to the right slot.

index = hash(key) % table_size
⚡ Average Time Complexity: O(1) for Insert, Search, and Delete — the fastest possible!
💡
Real-Life Analogy
🏫
School Locker System
Imagine 100 school lockers numbered 0–99. Each student has an ID. To find locker #, just take: Student ID % 100. Student 1542 → Locker 42. You don't search all 100 lockers — you go directly to #42! That's exactly how a hash table works.

Create Hash Table
Learn how to define the structure of a hash table, allocate memory for the array, and set up the hash function.
📖
Theory: Building the Blueprint

Creating a hash table means deciding three things:

  1. Define table size — How many slots does our array have? Choosing a prime number (like 7, 11, 13, 31) reduces collisions.
  2. Allocate array memory — Reserve actual memory for the slots using malloc() in C.
  3. Initialize array slots — Mark all slots as EMPTY (default state).
  4. Set default values — Each slot should have a valid "null" or empty marker.
  5. Set hash function — Define how keys map to indexes.
💡 Why prime numbers for table size? Primes reduce the chance of multiple keys landing on the same slot, distributing data more evenly.
🎮

Type any key (number) and see how the hash function maps it to an index in a table of size 7:

key = 42 → 42 % 7 = Index: 0

C Code: Create Hash Table
C
#include <stdio.h>
#include <stdlib.h>
#include <string.h>

#define TABLE_SIZE 7   /* Prime number — reduces collisions */
#define EMPTY      -1   /* Marker for an empty slot */
#define DELETED    -2   /* Marker for a deleted slot */

/* Structure for one slot (entry) in the hash table */
typedef struct {
    int key;    /* The key we store (e.g. student ID) */
    int value;  /* The value linked to the key (e.g. grade) */
} Entry;

/* The hash table itself */
typedef struct {
    Entry *table;   /* Pointer to the array of slots */
    int  size;       /* Total number of slots */
    int  count;      /* How many entries are currently stored */
} HashTable;

/* Hash Function: converts key → array index */
int hashFunction(int key, int size) {
    return key % size;  /* Simple modulo operation */
}

/* Step 1: Create and return a new hash table */
HashTable* createHashTable(int size) {
    /* Allocate memory for the HashTable struct */
    HashTable *ht = (HashTable*) malloc(sizeof(HashTable));

    /* Allocate memory for the array of entries */
    ht->table = (Entry*) malloc(size * sizeof(Entry));

    ht->size  = size;   /* Store the size */
    ht->count = 0;     /* Start with 0 entries */

    /* Initialize all slots as EMPTY */
    for (int i = 0; i < size; i++) {
        ht->table[i].key   = EMPTY;
        ht->table[i].value = 0;
    }

    printf("Hash table created! Size: %d slots\n", size);
    return ht;
}

int main() {
    HashTable *ht = createHashTable(TABLE_SIZE);
    printf("Index of key 25: %d\n", hashFunction(25, TABLE_SIZE));
    printf("Index of key 50: %d\n", hashFunction(50, TABLE_SIZE));
    return 0;
}
Output: Hash table created! Size: 7 slots
Index of key 25: 4
Index of key 50: 1

Initialize Table
Learn how to properly set up markers, reset indexes, initialize counters, and configure the load factor for your hash table.
📖
Theory: What Does "Initialize" Mean?

After allocating memory, we need to initialize — set each slot to a known starting state. This is critical because uninitialized memory contains random "garbage" values.

  • Set empty markers — Mark each slot's key as EMPTY (-1) so we know no real data is stored there yet.
  • Reset indexes — All slot positions start at their default state (no key, no value).
  • Initialize counters — Set count = 0 to track how many entries exist.
  • Configure load factor — Set a threshold (typically 0.7) beyond which we resize the table.
📌 Load Factor = (Number of entries) ÷ (Table size). Keep it below 0.7 for best performance.
🎮
Visual: Slot States

A slot in a hash table can be in one of three states:

[ EMPTY ]
→ Never used. Search can stop here.
42→100
→ Has real data (key=42, value=100)
[ DELETED ]
→ Was deleted. Search must continue past it!
⚠️ The DELETED marker is crucial! If we just made it EMPTY after deletion, searches for other keys might stop too early and fail to find existing data.
💻
C Code: Initialize Table
C
#define EMPTY       -1
#define DELETED     -2
#define LOAD_FACTOR  0.7f  /* Resize when 70% full */

typedef struct {
    int   key;
    int   value;
    int   state;   /* EMPTY, DELETED, or FILLED */
} Entry;

typedef struct {
    Entry *table;
    int   size;
    int   count;
    float loadFactor;    /* Threshold to trigger resize */
} HashTable;

/* Initialize all slots to empty state */
void initTable(HashTable *ht) {
    for (int i = 0; i < ht->size; i++) {
        ht->table[i].key   = EMPTY;    /* Mark slot as empty */
        ht->table[i].value = 0;
        ht->table[i].state = EMPTY;    /* Slot state = EMPTY */
    }
    ht->count      = 0;           /* No entries yet */
    ht->loadFactor = LOAD_FACTOR;   /* Set 70% threshold */

    printf("Table initialized: %d slots, all EMPTY\n", ht->size);
}

/* Check current load factor */
float getLoadFactor(HashTable *ht) {
    return (float) ht->count / ht->size;
}

Insert / Put
Learn how to add key-value pairs into the hash table, including handling collisions, updating existing keys, and storing entries in the array.
📖
Theory: How Insert Works
  • Generate hash index — Apply hash function to the key: index = key % size
  • Check slot status — Is the slot EMPTY, FILLED, or DELETED?
  • Insert key & value — If slot is empty or deleted, place the entry there.
  • Handle collision — If slot is already filled with a different key, find next available slot (probing).
  • Update existing key — If the same key already exists, overwrite its value.
  • Store entry in array — Write key, value, and state=FILLED to the slot.
🎮
Interactive Insert Demo

Insert key-value pairs into a hash table of size 7. Watch how the hash function decides the slot!

C Code: Insert / Put
C
/* Insert a key-value pair into the hash table */
void insert(HashTable *ht, int key, int value) {

    /* Step 1: Generate hash index */
    int index = hashFunction(key, ht->size);

    /* Step 2 & 4: Linear probing to find open slot */
    for (int i = 0; i < ht->size; i++) {
        int probe = (index + i) % ht->size;

        /* Step 5: Key already exists — update its value */
        if (ht->table[probe].state == FILLED &&
            ht->table[probe].key == key) {
            ht->table[probe].value = value;
            printf("Updated key %d → value %d\n", key, value);
            return;
        }

        /* Step 3: Slot is empty or deleted — insert here */
        if (ht->table[probe].state != FILLED) {
            ht->table[probe].key   = key;
            ht->table[probe].value = value;
            ht->table[probe].state = FILLED;
            ht->count++;
            printf("Inserted key %d at index %d\n", key, probe);
            return;
        }
    }

    printf("Table is FULL! Cannot insert key %d\n", key);
}

int main() {
    HashTable *ht = createHashTable(7);
    initTable(ht);

    insert(ht, 10, 100);  /* 10 % 7 = 3 → slot 3 */
    insert(ht, 20, 200);  /* 20 % 7 = 6 → slot 6 */
    insert(ht, 15, 150);  /* 15 % 7 = 1 → slot 1 */
    insert(ht, 10, 999);  /* key 10 exists → update! */
    return 0;
}
Output:

Updated key 10 → value 999

Search / Lookup
Learn how to find a value by its key, including traversing the probe sequence and handling both found and not-found cases.
📖
Theory: How Search Works
  • Generate hash index — Same formula as insert: index = key % size
  • Check target slot — Look at the calculated index first.
  • Traverse probe sequence — If the slot has a different key or DELETED marker, keep probing in the same pattern used during insert.
  • Compare keys — At each slot, check if the key matches what we're looking for.
  • Return value — Found the key? Return its value.
  • Return not found — Hit an EMPTY slot? The key doesn't exist. Stop searching.
⚠️ Critical rule: Stop at EMPTY slots (key not found), but continue past DELETED slots (the key might be further ahead)!
🎮
Interactive Search Demo
💻
C Code: Search / Lookup
C
/* Search for a key; returns value or -1 if not found */
int search(HashTable *ht, int key) {

    /* Step 1: Generate starting hash index */
    int index = hashFunction(key, ht->size);

    /* Step 2-4: Traverse probe sequence */
    for (int i = 0; i < ht->size; i++) {
        int probe = (index + i) % ht->size;

        /* Hit an EMPTY slot → key doesn't exist */
        if (ht->table[probe].state == EMPTY) {
            printf("Key %d NOT FOUND (hit empty slot)\n", key);
            return -1;
        }

        /* Skip DELETED slots and keep searching */
        if (ht->table[probe].state == DELETED) {
            continue;
        }

        /* Step 4-5: Compare keys */
        if (ht->table[probe].key == key) {
            printf("Found key %d at index %d → value: %d\n",
                    key, probe, ht->table[probe].value);
            return ht->table[probe].value;
        }
    }

    /* Step 6: Searched all slots, not found */
    printf("Key %d NOT FOUND\n", key);
    return -1;
}

int main() {
    HashTable *ht = createHashTable(7);
    initTable(ht);
    insert(ht, 10, 100);
    insert(ht, 20, 200);

    search(ht, 10);  /* Found at index 3, value: 100 */
    search(ht, 99);  /* NOT FOUND */
    return 0;
}

Analogy

Delete / Remove
Safely remove key-value pairs from the hash table using the DELETED tombstone marker to preserve probe sequence integrity.
📖
Theory: Why Deletion is Tricky

Deletion in a hash table isn't as simple as clearing a slot. If we just make a slot EMPTY, it could break the probe chain — search would incorrectly stop here and miss keys that were inserted past this slot.

  • Find target index — Use hash + probing to locate the key.
  • Verify key — Confirm the found slot's key matches what we want to delete.
  • Mark deleted slot — Set state to DELETED (tombstone), NOT EMPTY!
  • Remove value — Clear the value field.
  • Update counters — Decrement the count.
🪦 Tombstone Pattern: The DELETED marker acts like a gravestone — it says "something was here but is gone now, keep looking past me."
🎮
Interactive Delete Demo
💻
C Code: Delete / Remove
C
/* Delete a key from the hash table */
int deleteKey(HashTable *ht, int key) {

    /* Step 1: Find starting hash index */
    int index = hashFunction(key, ht->size);

    for (int i = 0; i < ht->size; i++) {
        int probe = (index + i) % ht->size;

        /* Hit EMPTY → key was never inserted */
        if (ht->table[probe].state == EMPTY) {
            printf("Key %d not found — cannot delete\n", key);
            return 0;
        }

        /* Skip past tombstones */
        if (ht->table[probe].state == DELETED) {
            continue;
        }

        /* Step 2: Verify key matches */
        if (ht->table[probe].key == key) {
            /* Step 3: Mark as DELETED (tombstone) */
            ht->table[probe].state = DELETED;

            /* Step 4: Clear the value */
            ht->table[probe].value = 0;

            /* Step 5: Decrease count */
            ht->count--;

            printf("Deleted key %d from index %d\n", key, probe);
            return 1;  /* Success */
        }
    }
    printf("Key %d not found\n", key);
    return 0;
}

Update Entry
Modify the value of an existing key without disrupting the hash table's structure or array integrity.
📖
Theory: Update is Search + Overwrite

Updating is the simplest core operation. The steps are:

  • Search existing key — Use the search algorithm to locate the key's slot.
  • Replace old value — Once found, simply overwrite the value field.
  • Maintain array integrity — The key, state, and position remain unchanged. Only the value changes.
✅ Note: Our insert() function already handles updates — if the key exists, it updates instead of inserting a duplicate!
💻
C Code: Update Entry
C
/* Update the value of an existing key */
int update(HashTable *ht, int key, int newValue) {

    /* Step 1: Search for the key */
    int index = hashFunction(key, ht->size);

    for (int i = 0; i < ht->size; i++) {
        int probe = (index + i) % ht->size;

        if (ht->table[probe].state == EMPTY) {
            printf("Key %d not found — nothing to update\n", key);
            return 0;
        }

        if (ht->table[probe].state == FILLED &&
            ht->table[probe].key == key) {

            /* Step 2: Replace old value with new value */
            int oldValue = ht->table[probe].value;
            ht->table[probe].value = newValue;

            printf("Updated key %d: %d → %d\n", key, oldValue, newValue);
            return 1;  /* Step 3: Array integrity maintained */
        }
    }
    return 0;
}

int main() {
    HashTable *ht = createHashTable(7);
    initTable(ht);
    insert(ht, 5, 50);
    update(ht, 5, 500);  /* Updated key 5: 50 → 500 */
    update(ht, 9, 900);  /* Key 9 not found */
    return 0;
}

Collision Handling
Understand what collisions are, why they happen, and the three main strategies to resolve them: linear probing, quadratic probing, and double hashing.
📖
Theory: What is a Collision?

A collision occurs when two different keys hash to the same array index. Since only one value can sit in a slot, we need a strategy to handle this conflict.

Example: With table size 7:

hash(3) = 3 % 7 = 3   |   hash(10) = 10 % 7 = 3 → COLLISION!

Both keys 3 and 10 want slot 3. We need to figure out where key 10 goes.

  • Detect collision — Slot at hash(key) is already FILLED with a different key.
  • Linear probing — Try slots +1, +2, +3... (next-door search)
  • Quadratic probing — Try slots +1², +2², +3²... (jump farther)
  • Double hashing — Use a second hash function to determine jump size
  • Probe sequence generation — The pattern of slots checked when a collision occurs
🎮

C Code: Collision Detection
C
/* Detect if inserting key would cause a collision */
int hasCollision(HashTable *ht, int key) {
    int index = hashFunction(key, ht->size);

    /* Check if the target slot already has a different key */
    if (ht->table[index].state == FILLED &&
        ht->table[index].key != key) {
        printf("COLLISION! key %d → index %d (already has key %d)\n",
               key, index, ht->table[index].key);
        return 1;  /* Collision detected */
    }
    return 0;  /* No collision */
}

/* Example: Both keys hash to index 3 in size-7 table */
int main() {
    HashTable *ht = createHashTable(7);
    initTable(ht);

    insert(ht, 10, 100);         /* 10 % 7 = 3 → slot 3 */
    hasCollision(ht, 17);         /* 17 % 7 = 3 → COLLISION! */
    insert(ht, 17, 170);         /* probe to slot 4 */
    return 0;
}

// SECTION 08
Linear Probing Operations
The simplest collision resolution: check the next slot, then the next, wrapping around the array like a circular buffer.
📖
Theory: Linear Probing

When a collision occurs, linear probing searches for the next open slot by checking one slot at a time in order.

probe(i) = (hash(key) + i) % size   [i = 0, 1, 2, 3...]
  • Check next slot — If slot (hash + 1) is busy, try (hash + 2), etc.
  • Sequential probing — Always moves forward by exactly 1 slot.
  • Wrap-around traversal — When reaching the end, wraps to index 0 (modulo).
  • Empty slot detection — Stops when it finds an EMPTY slot (for insert) or EMPTY slot (for search = not found).
⚠️ Primary Clustering Problem: Linear probing creates "clusters" — long runs of filled slots — which slow down future operations. Quadratic probing and double hashing solve this.
🎮
Linear Probing Step-by-Step Demo

Watch linear probing in action. Insert several keys to see how clustering forms:

C Code: Linear Probing
C
/* Linear Probing: find next slot with +1 step */
int linearProbeInsert(HashTable *ht, int key, int value) {
    int index = hashFunction(key, ht->size);
    int i;

    for (i = 0; i < ht->size; i++) {
        /* Next probe index: moves +1 each time */
        int probe = (index + i) % ht->size;  /* LINEAR: step = 1 */

        printf("Checking slot %d... ", probe);

        /* Update if same key found */
        if (ht->table[probe].state == FILLED &&
            ht->table[probe].key == key) {
            ht->table[probe].value = value;
            printf("Updated!\n");
            return probe;
        }

        /* Insert at empty or deleted slot */
        if (ht->table[probe].state != FILLED) {
            ht->table[probe].key   = key;
            ht->table[probe].value = value;
            ht->table[probe].state = FILLED;
            ht->count++;
            printf("Inserted! (after %d probe(s))\n", i);
            return probe;
        }

        printf("Collision! Try next...\n");
    }

    printf("Table full!\n");
    return -1;
}

/* Wrap-around example: insert at last slot wraps to 0 */
int main() {
    HashTable *ht = createHashTable(7);
    initTable(ht);

    linearProbeInsert(ht, 3,  30);   /* Slot 3 */
    linearProbeInsert(ht, 10, 100); /* 10%7=3 → probe→ slot 4 */
    linearProbeInsert(ht, 17, 170); /* 17%7=3 → probe→ slot 5 */
    return 0;
}

Parking Lot
Linear probing is like finding a parking spot. You go to your preferred spot (hash index). If it's taken, you check the next spot, then the next, going around the lot if needed. The problem: once a cluster of cars forms in one area, more cars keep filling nearby spots, making that area even more crowded!
// SECTION 09
Quadratic Probing Operations
Jump farther with each collision using squared offsets to break up clusters and distribute keys more evenly across the table.
📖
Theory: Quadratic Probing

Instead of jumping by 1 each time, quadratic probing squares the attempt number to jump farther and farther, breaking up clusters.

probe(i) = (hash(key) + i²) % size   [i = 0, 1, 2, 3...]
  • Calculate quadratic offset — Offset = i × i (i squared). For i=1: offset=1, i=2: offset=4, i=3: offset=9.
  • Jump probing — Jumps farther with each attempt, not just one step at a time.
  • Collision resolution — Same goal as linear, but with less clustering.
⚠️ Secondary clustering: Keys that hash to the same slot will follow the exact same probe path (1², 2², 3²...). Double hashing avoids even this.
✅ Important: For quadratic probing to always find an empty slot, the table size must be a prime number and the table must be less than 50% full.
🎮
Quadratic Probing Demo
💻
C Code: Quadratic Probing
C
/* Quadratic Probing: jump by i*i each time */
int quadraticInsert(HashTable *ht, int key, int value) {
    int index = hashFunction(key, ht->size);

    for (int i = 0; i < ht->size; i++) {

        /* QUADRATIC: offset = i squared */
        int offset = i * i;
        int probe  = (index + offset) % ht->size;

        printf("i=%d → offset=%d → slot %d\n", i, offset, probe);

        if (ht->table[probe].state == FILLED &&
            ht->table[probe].key == key) {
            ht->table[probe].value = value;
            return probe;
        }

        if (ht->table[probe].state != FILLED) {
            ht->table[probe].key   = key;
            ht->table[probe].value = value;
            ht->table[probe].state = FILLED;
            ht->count++;
            printf("Inserted key %d at slot %d\n", key, probe);
            return probe;
        }
    }
    return -1;
}

int main() {
    HashTable *ht = createHashTable(11);  /* Prime size */
    initTable(ht);

    /* All three hash to index 3 in size-11 table */
    quadraticInsert(ht, 3,  30);   /* Slot 3 (i=0, offset=0) */
    quadraticInsert(ht, 14, 140); /* Slot 4 (i=1, offset=1) */
    quadraticInsert(ht, 25, 250); /* Slot 7 (i=2, offset=4) */
    return 0;
}

// SECTION 10
Double Hashing Operations
The most sophisticated open addressing technique — use a second hash function to determine unique jump sizes for each key, eliminating clustering entirely.
📖
Theory: Double Hashing

Double hashing uses two hash functions. The second one determines how far to jump when a collision occurs. Different keys get different jump sizes, virtually eliminating clustering.

probe(i) = (h1(key) + i × h2(key)) % size

A classic second hash function is:

h2(key) = PRIME - (key % PRIME)

Where PRIME is a prime number smaller than table size.

  • Primary hash calculation — h1(key) = key % size → gives starting index
  • Secondary hash calculation — h2(key) = PRIME - (key % PRIME) → gives step size
  • Probe jump calculation — Each probe jumps by h2(key) slots
  • Alternative slot traversal — Visits completely different slots than linear/quadratic probing
✅ h2(key) must never return 0 (that would mean no movement). The formula PRIME - (key % PRIME) always gives a value between 1 and PRIME.

C Code: Double Hashing
C
#define TABLE_SIZE  11  /* Primary table size (prime) */
#define PRIME        7  /* Secondary prime (smaller than TABLE_SIZE) */

/* Primary hash function */
int h1(int key) {
    return key % TABLE_SIZE;
}

/* Secondary hash function — determines step size */
int h2(int key) {
    return PRIME - (key % PRIME);  /* Always between 1 and PRIME */
}

/* Double hashing insert */
int doubleHashInsert(HashTable *ht, int key, int value) {
    int start = h1(key);     /* Primary: starting index */
    int step  = h2(key);     /* Secondary: jump size */

    printf("h1(%d)=%d  h2(%d)=%d\n", key, start, key, step);

    for (int i = 0; i < ht->size; i++) {
        /* DOUBLE HASH: jump by h2 instead of 1 */
        int probe = (start + i * step) % ht->size;

        printf("  Probe %d → slot %d\n", i, probe);

        if (ht->table[probe].state == FILLED &&
            ht->table[probe].key == key) {
            ht->table[probe].value = value;
            return probe;
        }
        if (ht->table[probe].state != FILLED) {
            ht->table[probe].key   = key;
            ht->table[probe].value = value;
            ht->table[probe].state = FILLED;
            ht->count++;
            printf("Inserted at slot %d\n", probe);
            return probe;
        }
    }
    return -1;
}

int main() {
    HashTable *ht = createHashTable(TABLE_SIZE);
    initTable(ht);
    doubleHashInsert(ht, 14, 140);
    doubleHashInsert(ht, 25, 250); /* Different step size! */
    return 0;
}

// SECTION 11
Rehashing Operations
When the table gets too full, rehashing rebuilds the entire structure with a larger size and recalculated positions for all existing entries.
📖
Theory: What is Rehashing?

Rehashing is the process of creating a completely new, larger hash table and reinserting all existing entries. It's triggered when the load factor exceeds the threshold (usually 0.7).

  • Detect high load factor — After each insert, check if count/size ≥ loadFactor
  • Create larger array — New size is typically nextPrime(2 × oldSize)
  • Recalculate indexes — Since size changed, ALL keys get new hash values
  • Reinsert entries — Insert every FILLED entry from old table into new table
⚠️ You CANNOT just copy the old array! Every entry must be rehashed because the new table size changes all index calculations.

C Code: Rehashing
C
/* Find the next prime number >= n */
int isPrime(int n) {
    if (n < 2) return 0;
    for (int i = 2; i * i <= n; i++)
        if (n % i == 0) return 0;
    return 1;
}

int nextPrime(int n) {
    while (!isPrime(n)) n++;
    return n;
}

/* Rehash: build a bigger table and reinsert all entries */
void rehash(HashTable *ht) {
    int   oldSize  = ht->size;
    Entry *oldTable = ht->table;

    /* Step 2: New size = next prime after 2x old size */
    int newSize = nextPrime(oldSize * 2);
    printf("Rehashing: size %d → %d\n", oldSize, newSize);

    /* Step 2: Allocate new array */
    ht->table = (Entry*) malloc(newSize * sizeof(Entry));
    ht->size  = newSize;
    ht->count = 0;

    /* Initialize new array */
    for (int i = 0; i < newSize; i++) {
        ht->table[i].state = EMPTY;
    }

    /* Step 3-4: Reinsert all non-deleted entries */
    for (int i = 0; i < oldSize; i++) {
        if (oldTable[i].state == FILLED) {
            insert(ht, oldTable[i].key, oldTable[i].value);
        }
    }

    /* Free old memory */
    free(oldTable);
    printf("Rehash complete! New size: %d\n", newSize);
}

/* Smart insert: rehash automatically if needed */
void smartInsert(HashTable *ht, int key, int value) {
    /* Step 1: Check load factor BEFORE inserting */
    if ((float)ht->count / ht->size >= ht->loadFactor) {
        rehash(ht);  /* Grow the table first */
    }
    insert(ht, key, value);
}

// SECTION 12
Resize Table
The mechanics of growing the underlying array: allocating new memory, copying entries, and recomputing all positions.
📖
Theory: Resize vs Rehash

Resize and rehash are closely related. Resize focuses on the memory operations; rehashing is the broader process including reinsertion.

  • Increase array size — Choose new size: typically next prime after 2× current size
  • Allocate new array — Use malloc(newSize * sizeof(Entry))
  • Copy entries — But NOT by direct memory copy! Must reinsert each FILLED entry
  • Recompute positions — Each key gets a new index: key % newSize
💡 Some implementations use lazy resizing — only resize when an insert would fail, not at a load factor threshold. Both approaches are valid.
💻
C Code: Resize Table
C
/* Resize the hash table to a new larger size */
void resizeTable(HashTable *ht, int newSize) {
    int   oldSize  = ht->size;
    Entry *oldTable = ht->table;
    int   oldCount = ht->count;

    printf("Resizing: %d → %d slots\n", oldSize, newSize);

    /* Step 2: Allocate fresh memory for bigger array */
    ht->table = (Entry*) malloc(newSize * sizeof(Entry));
    ht->size  = newSize;
    ht->count = 0;

    /* Initialize new table */
    for (int i = 0; i < newSize; i++) {
        ht->table[i].key   = EMPTY;
        ht->table[i].state = EMPTY;
        ht->table[i].value = 0;
    }

    /* Step 3 & 4: Reinsert each valid entry */
    for (int i = 0; i < oldSize; i++) {
        if (oldTable[i].state == FILLED) {
            /* Recompute position: key % NEW size */
            insert(ht, oldTable[i].key, oldTable[i].value);
        }
        /* DELETED entries are dropped during resize */
    }

    printf("Resize done: %d entries moved\n", ht->count);

    /* Free old array memory */
    free(oldTable);
}

int main() {
    HashTable *ht = createHashTable(7);
    initTable(ht);

    insert(ht, 1, 10);
    insert(ht, 2, 20);
    insert(ht, 3, 30);
    insert(ht, 4, 40);
    insert(ht, 5, 50);

    /* Table is 5/7 = 71% full → resize! */
    resizeTable(ht, nextPrime(14)); /* New size: 17 */
    return 0;
}

Old vs New Position After Resize
Example: key=5 in old table (size 7): 5 % 7 = 5
Same key=5 in new table (size 17): 5 % 17 = 5

key=10 in old table: 10 % 7 = 3
key=10 in new table: 10 % 17 = 10 ← position changed!

This is why we must reinsert, not copy — every key's home changes with the new table size!

// SECTION 13
Traverse / Display
Walk through every slot in the array, printing keys, values, or both — skipping empty and deleted slots.
📖
Theory: Traversal

Traversal visits every slot in the underlying array from index 0 to size-1. Unlike search (which uses hashing to jump), traversal is purely sequential.

  • Traverse entire array — Loop from i=0 to i=size-1
  • Skip empty slots — If state == EMPTY, skip and continue
  • Print keys — Show the key at each FILLED slot
  • Print values — Show the value at each FILLED slot
  • Print key-value pairs — Show both together (e.g., "key=5 → value=50")

/* Print all key-value pairs in the hash table */
void displayTable(HashTable *ht) {
    printf("\n=== Hash Table Contents ===\n");
    printf("%-6s %-10s %-10s %-10s\n",
           "Index", "State", "Key", "Value");
    printf("------------------------------\n");

    /* Step 1: Traverse entire array (all indexes) */
    for (int i = 0; i < ht->size; i++) {

        if (ht->table[i].state == EMPTY) {
            /* Step 2: Skip empty slots */
            printf("%-6d [EMPTY]\n", i);

            continue;

               i, ht->table[i].key, ht->table[i].value);
    }
    printf("=========================\n");
    printf("Total entries: %d / %d\n\n", ht->count, ht->size);
}

    for (int i = 0; i < ht->size; i++)
        if (ht->table[i].state == FILLED)
            printf("%d ", ht->table[i].value);
    printf("\n");
}
// SECTION 14
Validation Operations
Guard your hash table with checks: is it empty? full? is a key valid? is an index in range? is a slot marked as deleted?
📖
Theory: Why Validate?

Validation prevents crashes and bugs. Before performing any operation, we should verify that inputs and states are valid.

  • Check empty — Is the table completely empty? (count == 0)
  • Check full — Is every slot occupied? (count == size)
  • Validate key — Is the key value within acceptable range? (not EMPTY sentinel)
  • Validate index — Is the computed index within [0, size-1]?
  • Check deleted marker — Is a specific slot marked DELETED vs EMPTY?
💻
C Code: Validation Operations
C
/* Check if the hash table is empty */
int isEmpty(HashTable *ht) {
    return (ht->count == 0);
}

/* Check if the hash table is full */
int isFull(HashTable *ht) {
    return (ht->count == ht->size);
}

/* Validate key: must not be our sentinel values */
int isValidKey(int key) {
    if (key == EMPTY) {
        printf("Invalid key: %d is reserved as EMPTY marker\n", key);
        return 0;
    }
    if (key == DELETED) {
        printf("Invalid key: %d is reserved as DELETED marker\n", key);
        return 0;
    }
    return 1;
}

/* Validate that index is in range */
int isValidIndex(HashTable *ht, int index) {
    return (index >= 0 && index < ht->size);
}

/* Check if a specific slot has a deleted marker */
int isDeleted(HashTable *ht, int index) {
    return (ht->table[index].state == DELETED);
}

/* Safe insert with all validations */
void safeInsert(HashTable *ht, int key, int value) {
    if (!isValidKey(key)) return;   /* Validate key */
    if (isFull(ht)) {                   /* Check if full */
        printf("Table is FULL! Resize first.\n");
        return;
    }
    insert(ht, key, value);
}

int main() {
    HashTable *ht = createHashTable(7);
    initTable(ht);

    printf("Is empty? %d\n", isEmpty(ht));    /* 1 (yes) */
    safeInsert(ht, -1, 100);  /* Invalid! -1 is EMPTY marker */
    safeInsert(ht,  5, 50);   /* Valid — inserted */
    printf("Is empty? %d\n", isEmpty(ht));    /* 0 (no) */
    printf("Is full?  %d\n", isFull(ht));     /* 0 (no) */
    return 0;
}

The load factor (α) measures how "crowded" the hash table is:

α = number of entries / table size

  • Trigger resize — Call rehash/resize when threshold is exceeded

#define MAX_LOAD_FACTOR 0.7f

/* Step 2: Calculate current load factor */
float getLoadFactor(HashTable *ht) {
    return (float) ht->count / ht->size;
}

/* Step 1: Count only occupied (FILLED) slots */
int countOccupied(HashTable *ht) {
    int count = 0;
    for (int i = 0; i < ht->size; i++)
        if (ht->table[i].state == FILLED)
            count++;
    return count;
}

/* Step 3 & 4: Check and trigger resize if needed */
void checkAndResize(HashTable *ht) {
    float lf = getLoadFactor(ht);
    printf("Load factor: %.2f (%d/%d)\n",
           lf, ht->count, ht->size);

    if (lf >= MAX_LOAD_FACTOR) {
        printf("WARNING: Load factor %.2f >= %.2f! Resizing...\n",
               lf, MAX_LOAD_FACTOR);
        rehash(ht);  /* Step 4: Trigger resize */
    } else {
        printf("Load factor OK. No resize needed.\n");
    }
}

int main() {
    HashTable *ht = createHashTable(7);
    initTable(ht);

    for (int i = 1; i <= 6; i++) {
        insert(ht, i * 10, i * 100);
        checkAndResize(ht);  /* Check after every insert */
    }
    return 0;
}
// SECTION 16
Memory Management
Allocate, use, clear, and free memory responsibly — essential for production C programs that avoid memory leaks.
📖
Theory: Memory in C

In C, memory is manually managed. Every malloc() must be paired with a free(). Forgetting to free causes memory leaks — your program eats more and more RAM until the system crashes.

  • Allocate array — malloc(size * sizeof(Entry)) reserves bytes for the array
  • Free array memory — free(ht->table) releases the slot array
  • Clear table — Reset all slots to EMPTY without freeing memory (reuse the table)
  • Destroy table — Free both the array AND the HashTable struct itself
⚠️ Memory management order matters! Free the INNER data (array) BEFORE the OUTER struct. Freeing the struct first loses the pointer to the array — memory leak!
💻
C Code: Memory Management
C
#include <stdlib.h>

/* Step 1: Allocate memory for the hash table */
HashTable* allocateHashTable(int size) {
    /* Outer: allocate the struct */
    HashTable *ht = (HashTable*) malloc(sizeof(HashTable));
    if (ht == NULL) {
        printf("Error: failed to allocate HashTable!\n");
        return NULL;
    }

    /* Inner: allocate the entry array */
    ht->table = (Entry*) malloc(size * sizeof(Entry));
    if (ht->table == NULL) {
        printf("Error: failed to allocate table array!\n");
        free(ht);  /* Free struct if array fails */
        return NULL;
    }

    ht->size  = size;
    ht->count = 0;
    return ht;
}

/* Step 3: Clear table (keep memory, reset contents) */
void clearTable(HashTable *ht) {
    for (int i = 0; i < ht->size; i++) {
        ht->table[i].key   = EMPTY;
        ht->table[i].value = 0;
        ht->table[i].state = EMPTY;
    }
    ht->count = 0;
    printf("Table cleared (memory retained).\n");
}

/* Step 2 & 4: Free all memory — destroy table */
void destroyTable(HashTable *ht) {
    if (ht == NULL) return;

    /* IMPORTANT: Free inner array FIRST */
    free(ht->table);    /* Step 2: Free the slot array */
    ht->table = NULL;   /* Prevent dangling pointer */

    free(ht);           /* Step 4: Free the struct itself */
    printf("Hash table destroyed. All memory freed.\n");
}

int main() {
    HashTable *ht = allocateHashTable(7);
    initTable(ht);

    insert(ht, 10, 100);
    insert(ht, 20, 200);

    clearTable(ht);         /* Wipe data but keep structure */
    insert(ht, 99, 999);   /* Reuse the cleared table */

    destroyTable(ht);       /* Free all memory when done */
    ht = NULL;               /* Avoid using freed pointer */
    return 0;
}

clearTable is like cleaning a house — you empty it out but keep the structure for future use. destroyTable is like demolishing the house completely — you first remove the furniture (free the array), then tear down the walls (free the struct). Trying to demolish the walls first would trap the furniture inside forever (memory leak)!

Iteration Operations
Implement iterators to cycle through all keys, all values, or all entries in sequence — essential for bulk operations and reporting.
📖
Theory: Iteration vs Traversal

While traversal just prints, iteration means stepping through entries programmatically — your code processes each item. This is useful for batch updates, exports, or counting.

  • Iterate keys — Visit every FILLED slot and return/process its key
  • Iterate values — Visit every FILLED slot and return/process its value
  • Iterate entries — Return both key AND value for each filled slot
  • Sequential array traversal — Always goes from index 0 to size-1, never skips
💻
C Code: Iteration Operations
C
/* Iterator structure: tracks current position */
typedef struct {
    HashTable *ht;      /* Pointer to hash table */
    int        current; /* Current index in array */
} Iterator;

/* Create an iterator starting at index 0 */
Iterator createIterator(HashTable *ht) {
    Iterator it;
    it.ht      = ht;
    it.current = 0;
    return it;
}

/* Check if more entries exist */
int hasNext(Iterator *it) {
    /* Step 4: Sequential scan from current position */
    while (it->current < it->ht->size) {
        if (it->ht->table[it->current].state == FILLED)
            return 1;  /* Found a valid entry ahead */
        it->current++;
    }
    return 0;  /* No more entries */
}

/* Step 3: Get next key-value entry and advance */
Entry nextEntry(Iterator *it) {
    Entry entry = it->ht->table[it->current];
    it->current++;  /* Advance to next position */
    return entry;
}

int main() {
    HashTable *ht = createHashTable(7);
    initTable(ht);
    insert(ht, 5, 50);
    insert(ht, 12, 120);
    insert(ht, 3, 30);

    /* Step 1 & 2: Iterate all keys and values */
    Iterator it = createIterator(ht);
    printf("All entries:\n");
    while (hasNext(&it)) {
        Entry e = nextEntry(&it);
        printf("  key=%d → value=%d\n", e.key, e.value);
    }
    return 0;
}
