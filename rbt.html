📚What is a Red-Black Tree?

A Red-Black Tree (RBT) is a self-balancing Binary Search Tree (BST) where every node carries an extra bit of information — its color (RED or BLACK). This coloring enforces balance rules that guarantee the tree's height stays at most 2·log₂(n+1), so all major operations run in O(log n) time even in the worst case.

In a linked-list based implementation, each node is a dynamically allocated struct containing data, color, and three pointers: left, right, and parent. Leaf (NULL) nodes are represented by a special sentinel NIL node, which is always BLACK.

①

Root is BLACK

The root node must always be colored black.

②

RED node → BLACK children

A red node can only have black children (no two reds in a row).

③

Equal Black Height

Every path from a node to its NIL leaves has the same number of black nodes.

④

NIL leaves are BLACK

All NULL / NIL sentinel leaves are considered black nodes.

⑤

Every node is RED or BLACK

No other colors exist — each node stores exactly 1 bit for color.

// REAL-LIFE ANALOGY

Think of an office filing system. Files (nodes) are organized by ID. Every drawer must be balanced — the shortest and tallest stacks can't differ by more than 2×. Red labels are "urgent" — they can only sit next to "normal" (black) files. The rule forces the manager (CPU) to always find any file in the same number of steps, no matter how big the office gets.

01

Basic Operations

// REAL-LIFE ANALOGY

Building the skeleton of a library catalog. Before you store any books, you need the catalog binder (tree struct), the card slots (nodes), and a way to check if the catalog is empty. These are your basic operations.

🔷Node Structure

Every node in a linked-list RBT is a struct with: integer data, color (RED=0 / BLACK=1), and pointers to left child, right child, and parent. A global NIL sentinel node acts as all leaf nodes — this simplifies deletion logic.

C node structure + tree initialization 
#include <stdio.h>
#include <stdlib.h>

/* Color constants */
#define RED   0
#define BLACK 1

/* Node structure (linked-list style) */
typedef struct Node {
    int data;          /* stored value */
    int color;         /* RED or BLACK */
    struct Node *left;  /* left child pointer */
    struct Node *right; /* right child pointer */
    struct Node *parent;/* parent pointer */
} Node;

/* Tree structure */
typedef struct {
    Node *root;  /* pointer to root node */
    Node *NIL;   /* sentinel leaf node (always BLACK) */
} RBTree;

/* Create a new node */
Node* createNode(RBTree *tree, int val) {
    Node *n = (Node*) malloc(sizeof(Node));
    n->data   = val;
    n->color  = RED;       /* new nodes start RED */
    n->left   = tree->NIL; /* leaves point to NIL */
    n->right  = tree->NIL;
    n->parent = tree->NIL;
    return n;
}

/* Initialize a new Red-Black Tree */
RBTree* initTree() {
    RBTree *t = (RBTree*) malloc(sizeof(RBTree));
    /* create the NIL sentinel */
    t->NIL        = (Node*) malloc(sizeof(Node));
    t->NIL->color = BLACK;
    t->NIL->left  = NULL;
    t->NIL->right = NULL;
    t->root       = t->NIL; /* empty tree: root = NIL */
    return t;
}

/* Check if tree is empty */
int isEmpty(RBTree *t) {
    return t->root == t->NIL;
}

02

Insertion Operations

// REAL-LIFE ANALOGY

Adding a new employee to a company hierarchy. A new hire always starts as a "junior" (RED). If the manager (parent) is also junior (RED), the HR policy is violated — we must either restructure the org chart (rotate) or change title levels (recolor) to fix it. The company stays balanced so no one is overloaded.

➕How Insertion Works

Step 1 — BST Insert: Insert the node exactly like a regular BST (compare, go left/right, place at correct leaf position).

Step 2 — Color RED: Every new node starts as RED.

Step 3 — Fix Violations: If the parent is also RED, we have a violation. Fix by checking the uncle's color:
  • Uncle is RED → Recolor (parent & uncle → BLACK, grandparent → RED, move up)
  • Uncle is BLACK → Rotate (left or right depending on case)

Step 4 — Root → BLACK: Always force root to black at the end.

C insert node (iterative BST + RB fix) 
/* Standard BST insert — place node in correct position */
void bstInsert(RBTree *t, Node *z) {
    Node *y = t->NIL; /* will become z's parent */
    Node *x = t->root;

    /* Walk down the tree to find the right spot */
    while (x != t->NIL) {
        y = x;
        if (z->data < x->data)
            x = x->left;
        else
            x = x->right;
    }
    z->parent = y;

    if (y == t->NIL)
        t->root = z;          /* tree was empty */
    else if (z->data < y->data)
        y->left = z;
    else
        y->right = z;
}

/* Insert a new value into the Red-Black Tree */
void insertNode(RBTree *t, int val) {
    Node *z = createNode(t, val); /* new RED node */
    bstInsert(t, z);              /* place it like BST */
    fixInsert(t, z);              /* fix RB violations */
}

/* Fix Red-Black violations after insert */
void fixInsert(RBTree *t, Node *z) {
    while (z->parent->color == RED) {
        if (z->parent == z->parent->parent->left) {
            Node *uncle = z->parent->parent->right;
            if (uncle->color == RED) {
                /* Case 1: Uncle is RED — just recolor */
                z->parent->color         = BLACK;
                uncle->color             = BLACK;
                z->parent->parent->color = RED;
                z = z->parent->parent;  /* move z up */
            } else {
                if (z == z->parent->right) {
                    /* Case 2: z is right child — left rotate parent */
                    z = z->parent;
                    leftRotate(t, z);
                }
                /* Case 3: z is left child — right rotate grandparent */
                z->parent->color         = BLACK;
                z->parent->parent->color = RED;
                rightRotate(t, z->parent->parent);
            }
        } else {
            /* Mirror cases for right subtree */
            Node *uncle = z->parent->parent->left;
            if (uncle->color == RED) {
                z->parent->color         = BLACK;
                uncle->color             = BLACK;
                z->parent->parent->color = RED;
                z = z->parent->parent;
            } else {
                if (z == z->parent->left) {
                    z = z->parent;
                    rightRotate(t, z);
                }
                z->parent->color         = BLACK;
                z->parent->parent->color = RED;
                leftRotate(t, z->parent->parent);
            }
        }
    }
    t->root->color = BLACK; /* rule 1: root must be BLACK */
}
03

Rotation Operations

// REAL-LIFE ANALOGY

Think of a mobile hanging from the ceiling. When one side becomes too heavy, you rotate the hook point to re-distribute weight. A rotation in a tree "pivots" a node, changing who is the parent and who is the child — without breaking the BST ordering property.

🔄Left Rotation

Left rotation is used when the right subtree is heavier. Node x rotates down-left and its right child y takes its place. The BST property is preserved: y's left subtree moves to x's right.

Before: x → y (x is parent, y is right child)
After: y → x (y becomes parent, x becomes left child of y)
🔄Right Rotation

Right rotation is the mirror: used when the left subtree is heavier. Node y rotates down-right and its left child x takes its place.

🔄Double Rotations

Left-Right (LR)

Node is right child of left child. First do left rotate on parent, then right rotate on grandparent.

Right-Left (RL)

Node is left child of right child. First do right rotate on parent, then left rotate on grandparent.

C left rotation + right rotation 
/* ─── Left Rotation ─────────────────────────────
   Before:     x           After:     y
               / \                   / \
              A   y                 x   C
                 / \               / \
                B   C             A   B
──────────────────────────────────────────── */
void leftRotate(RBTree *t, Node *x) {
    Node *y = x->right;    /* y = right child of x */
    x->right = y->left;    /* move B to x's right */

    if (y->left != t->NIL)
        y->left->parent = x;

    y->parent = x->parent; /* link y to x's parent */
    if (x->parent == t->NIL)
        t->root = y;
    else if (x == x->parent->left)
        x->parent->left = y;
    else
        x->parent->right = y;

    y->left   = x;         /* put x on y's left */
    x->parent = y;
}

/* ─── Right Rotation ────────────────────────────
   Before:     y           After:     x
               / \                   / \
              x   C                 A   y
             / \                       / \
            A   B                     B   C
──────────────────────────────────────────── */
void rightRotate(RBTree *t, Node *y) {
    Node *x = y->left;    /* x = left child of y */
    y->left  = x->right;  /* move B to y's left */

    if (x->right != t->NIL)
        x->right->parent = y;

    x->parent = y->parent;
    if (y->parent == t->NIL)
        t->root = x;
    else if (y == y->parent->right)
        y->parent->right = x;
    else
        y->parent->left = x;

    x->right  = y;        /* put y on x's right */
    y->parent = x;
}
04

Recoloring Operations

// REAL-LIFE ANALOGY

Like a traffic light system at an intersection. If two red lights are on at the same time (two consecutive red nodes), we must change the colors to resolve the conflict — make some lights green (black) and promote the problem upward until the whole system is valid again.

🎨When Does Recoloring Happen?

Recoloring occurs when we insert a RED node and its parent is also RED — this violates Rule 2. If the uncle is also RED, we can resolve it by:

1. Set parent → BLACK
2. Set uncle → BLACK
3. Set grandparent → RED
4. Move the "current node" pointer up to the grandparent and repeat if needed.

C recolor helper functions 
/* Change node color to RED */
void setRed(Node *n) {
    if (n != NULL) n->color = RED;
}

/* Change node color to BLACK */
void setBlack(Node *n) {
    if (n != NULL) n->color = BLACK;
}

/* Recolor parent of node z */
void recolorParent(Node *z) {
    if (z && z->parent)
        z->parent->color = BLACK;
}

/* Recolor uncle (sibling of parent) of node z */
void recolorUncle(RBTree *t, Node *z) {
    Node *gp  = z->parent->parent; /* grandparent */
    Node *uncle;
    if (z->parent == gp->left)
        uncle = gp->right; /* uncle is on the right */
    else
        uncle = gp->left;  /* uncle is on the left */
    if (uncle) uncle->color = BLACK;
}

/* Recolor grandparent of node z */
void recolorGrandparent(Node *z) {
    Node *gp = z->parent->parent;
    if (gp) gp->color = RED;
}

/* Combined recolor step (Case 1 of fixInsert) */
Node* recolorCase1(RBTree *t, Node *z, Node *uncle) {
    recolorParent(z);
    uncle->color             = BLACK;
    z->parent->parent->color = RED;
    return z->parent->parent; /* move z up to grandparent */
}
05

Balancing Operations

// REAL-LIFE ANALOGY

Like a seesaw with safety rules. After any change, a referee (fixup function) checks every rule. If one side is too heavy (double-black) or two reds are adjacent, the referee intervenes — rotating or recoloring until all five rules are satisfied again.

⚖️Fix Red-Black Violations

The core balancing function fixInsert (shown in section 02) handles the Double Red problem after insertion. The equivalent for deletion is fixDelete, which handles the Double Black problem.

Double Red — after insert Double Black — after delete Both fixed by rotate + recolor

Handle Double Red

Occurs when a RED node's parent is also RED. Fixed by checking uncle color, then recoloring or rotating.

Handle Double Black

Occurs after deletion. A node "absorbs" an extra blackness. Fixed in 4 cases based on sibling and sibling's children colors.

C rebalance tree (wrapper) 
/* Rebalance after insert — wrapper that calls fixInsert */
void rebalanceAfterInsert(RBTree *t, Node *z) {
    fixInsert(t, z); /* defined in section 2 */
}

/* Check for double red: returns 1 if violation exists */
int hasDoubleRed(Node *z) {
    return (z->color == RED && z->parent->color == RED);
}

/* Fix double black — called from fixDelete */
/* z is the "extra black" node (can be NIL node) */
void handleDoubleBlack(RBTree *t, Node *z) {
    while (z != t->root && z->color == BLACK) {
        if (z == z->parent->left) {
            Node *w = z->parent->right; /* sibling */
            if (w->color == RED) {
                /* Case 1: sibling is RED */
                w->color         = BLACK;
                z->parent->color = RED;
                leftRotate(t, z->parent);
                w = z->parent->right;
            }
            if (w->left->color == BLACK && w->right->color == BLACK) {
                /* Case 2: sibling's children both BLACK */
                w->color = RED;
                z = z->parent;
            } else {
                if (w->right->color == BLACK) {
                    /* Case 3: sibling's right child BLACK */
                    w->left->color = BLACK;
                    w->color       = RED;
                    rightRotate(t, w);
                    w = z->parent->right;
                }
                /* Case 4: sibling's right child RED */
                w->color         = z->parent->color;
                z->parent->color = BLACK;
                w->right->color  = BLACK;
                leftRotate(t, z->parent);
                z = t->root;
            }
        } else {
            /* Mirror of above for right subtree */
            Node *w = z->parent->left;
            if (w->color == RED) {
                w->color         = BLACK;
                z->parent->color = RED;
                rightRotate(t, z->parent);
                w = z->parent->left;
            }
            if (w->right->color == BLACK && w->left->color == BLACK) {
                w->color = RED;
                z = z->parent;
            } else {
                if (w->left->color == BLACK) {
                    w->right->color = BLACK;
                    w->color        = RED;
                    leftRotate(t, w);
                    w = z->parent->left;
                }
                w->color         = z->parent->color;
                z->parent->color = BLACK;
                w->left->color   = BLACK;
                rightRotate(t, z->parent);
                z = t->root;
            }
        }
    }
    z->color = BLACK;
}
06

Deletion Operations

// REAL-LIFE ANALOGY

Removing an employee from a company org chart. If they had no direct reports (leaf), just remove them. If they had one team (one child), their manager directly adopts that team. If they had two teams (two children), the most-qualified next person (inorder successor) steps into their role. After any removal, we check if the balance rules are still met.

🗑️Three Deletion Cases

Case 1: Leaf Node

Node has no children. Simply remove it. If it was BLACK, fix the double-black deficit at its position.

Case 2: One Child

Replace node with its only child. If the removed node was BLACK and the child is BLACK too, we have a double-black problem.

Case 3: Two Children

Find the inorder successor (leftmost node in right subtree), copy its data into this node, then delete the successor (which has at most one child).

// DELETION DEMO

C delete node + transplant helper 
/* Replace subtree rooted at u with subtree rooted at v */
void transplant(RBTree *t, Node *u, Node *v) {
    if (u->parent == t->NIL)
        t->root = v;
    else if (u == u->parent->left)
        u->parent->left = v;
    else
        u->parent->right = v;
    v->parent = u->parent;
}

/* Delete a node with given value */
void deleteNode(RBTree *t, int val) {
    Node *z = searchNode(t, val);
    if (z == t->NIL) { printf("Value not found\n"); return; }

    Node *y = z, *x;
    int yOrigColor = y->color;

    if (z->left == t->NIL) {
        /* Case 1/2a: no left child */
        x = z->right;
        transplant(t, z, z->right);
    } else if (z->right == t->NIL) {
        /* Case 2b: no right child */
        x = z->left;
        transplant(t, z, z->left);
    } else {
        /* Case 3: two children — find inorder successor */
        y = findMin(t, z->right); /* leftmost in right sub */
        yOrigColor = y->color;
        x = y->right;
        if (y->parent == z) {
            x->parent = y;
        } else {
            transplant(t, y, y->right);
            y->right = z->right;
            y->right->parent = y;
        }
        transplant(t, z, y);
        y->left = z->left;
        y->left->parent = y;
        y->color = z->color;
    }
    free(z);
    if (yOrigColor == BLACK)
        handleDoubleBlack(t, x); /* fix RB properties */
}

/* Find minimum node in subtree rooted at node */
Node* findMin(RBTree *t, Node *node) {
    while (node->left != t->NIL)
        node = node->left;
    return node;
}
07

Searching Operations

// REAL-LIFE ANALOGY

Like searching in a phone book. You don't read every name — you go to the middle, decide "is what I want before or after this page?", and jump to that half. A BST search works the same way — compare, go left or right, halving the search space each time.

🔍How Search Works

Because an RBT is a BST, search is O(log n) guaranteed. Start at root: if value matches, return node. If value is less, go left. If value is greater, go right. Stop when you hit NIL (not found).

// SEARCH VISUALIZATION
Insert some nodes first, then search for a value.
C iterative search + recursive search 
/* Iterative search — O(log n) */
Node* searchNode(RBTree *t, int val) {
    Node *curr = t->root;
    while (curr != t->NIL) {
        if (val == curr->data)
            return curr;        /* found! */
        else if (val < curr->data)
            curr = curr->left;   /* go left */
        else
            curr = curr->right;  /* go right */
    }
    return t->NIL; /* not found */
}

/* Recursive search */
Node* searchRecursive(RBTree *t, Node *node, int val) {
    if (node == t->NIL || node->data == val)
        return node;           /* base case */
    if (val < node->data)
        return searchRecursive(t, node->left,  val);
    else
        return searchRecursive(t, node->right, val);
}

/* Exact match search — returns 1 if found, 0 if not */
int exactSearch(RBTree *t, int val) {
    return searchNode(t, val) != t->NIL;
}

/* Example usage */
int main() {
    RBTree *t = initTree();
    insertNode(t, 10);
    insertNode(t, 20);
    insertNode(t, 30);

    Node *found = searchNode(t, 20);
    if (found != t->NIL)
        printf("Found: %d\n", found->data);
    else
        printf("Not found\n");
    return 0;
}

void preorder(RBTree *t, Node *n) {
    if (n == t->NIL) return;
    printf("%d ", n->data);
    preorder(t, n->left);
    preorder(t, n->right);
}

/* Postorder: Left → Right → Root */
void postorder(RBTree *t, Node *n) {
    if (n == t->NIL) return;
    postorder(t, n->left);
    postorder(t, n->right);
    printf("%d ", n->data);
}

/* Level-Order (BFS) using a queue */
void levelOrder(RBTree *t) {
    if (isEmpty(t)) return;
    Node *queue[100];
    int front = 0, rear = 0;
    queue[rear++] = t->root;

    while (front < rear) {
        Node *curr = queue[front++];
        printf("%d ", curr->data);
        if (curr->left  != t->NIL) queue[rear++] = curr->left;
        if (curr->right != t->NIL) queue[rear++] = curr->right;
    }
}

Min / Max Operations

// REAL-LIFE ANALOGY

In a BST, the minimum is always at the far left, maximum at the far right — like a sorted row of books where the cheapest is on the left end and the most expensive is on the right end. Just keep going left (or right) until you can't anymore.

// MIN/MAX HIGHLIGHT
Click buttons to highlight min/max nodes.
C find minimum and maximum 
/* Find minimum — always the leftmost node */
Node* findMinimum(RBTree *t, Node *node) {
    /* Keep going left until we hit NIL */
    while (node->left != t->NIL)
        node = node->left;
    return node; /* leftmost node = smallest */
}

/* Find maximum — always the rightmost node */
Node* findMaximum(RBTree *t, Node *node) {
    /* Keep going right until we hit NIL */
    while (node->right != t->NIL)
        node = node->right;
    return node; /* rightmost node = largest */
}

/* Leftmost node in entire tree */
Node* leftmostNode(RBTree *t) {
    return findMinimum(t, t->root);
}

/* Rightmost node in entire tree */
Node* rightmostNode(RBTree *t) {
    return findMaximum(t, t->root);
}

/* Usage */
int main() {
    RBTree *t = initTree();
    insertNode(t, 50); insertNode(t, 20); insertNode(t, 80);
    insertNode(t, 10); insertNode(t, 90);

    Node *mn = leftmostNode(t);
    Node *mx = rightmostNode(t);
    printf("Min: %d, Max: %d\n", mn->data, mx->data);
    /* Output: Min: 10, Max: 90 */
}

Successor & Predecessor Operations

// REAL-LIFE ANALOGY

In a sorted class roster, the successor of "Alice" is the next student alphabetically ("Bob"), and the predecessor is the previous one. In an RBT, inorder successor = the next larger value, inorder predecessor = the next smaller value.

C successor, predecessor, next greater/smaller 
/* Inorder Successor — next larger node */
Node* inorderSuccessor(RBTree *t, Node *n) {
    /* If right subtree exists, successor = leftmost of right */
    if (n->right != t->NIL)
        return findMinimum(t, n->right);

    /* Otherwise go up until we come from a left child */
    Node *parent = n->parent;
    while (parent != t->NIL && n == parent->right) {
        n      = parent;
        parent = parent->parent;
    }
    return parent; /* NIL if no successor (n was maximum) */
}

/* Inorder Predecessor — next smaller node */
Node* inorderPredecessor(RBTree *t, Node *n) {
    /* If left subtree exists, predecessor = rightmost of left */
    if (n->left != t->NIL)
        return findMaximum(t, n->left);

    /* Otherwise go up until we come from a right child */
    Node *parent = n->parent;
    while (parent != t->NIL && n == parent->left) {
        n      = parent;
        parent = parent->parent;
    }
    return parent;
}

/* Next greater element (same as inorder successor) */
Node* nextGreater(RBTree *t, Node *n) {
    return inorderSuccessor(t, n);
}

/* Next smaller element (same as inorder predecessor) */
Node* nextSmaller(RBTree *t, Node *n) {
    return inorderPredecessor(t, n);
}

/* Usage */
int main() {
    RBTree *t = initTree();
    insertNode(t, 20); insertNode(t, 10);
    insertNode(t, 30); insertNode(t, 25);
    Node *n = searchNode(t, 20);
    Node *succ = inorderSuccessor(t, n);
    printf("Successor of 20: %d\n", succ->data); /* 25 */
}

Height & Black Height Operations

// REAL-LIFE ANALOGY

Height = the tallest floor in a building. Black height = the number of load-bearing floors (black nodes) on any path from roof to ground. By RBT rules, this count must be equal on every path — ensuring the structure is sound everywhere.

📏Key Theorem

For an RBT with n internal nodes, the height is at most 2·log₂(n+1). This guarantees O(log n) for all operations. The black height bh(x) of a node x is the number of black nodes on any path from x to a descendant leaf, not counting x itself.

C height + black height + check balance 
/* Find height of the tree (longest path root to leaf) */
int findHeight(RBTree *t, Node *n) {
    if (n == t->NIL) return 0;
    int leftH  = findHeight(t, n->left);
    int rightH = findHeight(t, n->right);
    /* height = 1 + max(left, right) */
    return 1 + (leftH > rightH ? leftH : rightH);
}

/* Find black-height: count of BLACK nodes on path to NIL */
int findBlackHeight(RBTree *t, Node *n) {
    if (n == t->NIL) return 1; /* NIL counts as black */
    int leftBH = findBlackHeight(t, n->left);
    /* Add 1 if current node is BLACK */
    return leftBH + (n->color == BLACK ? 1 : 0);
}

/* Count black nodes on path from node to NIL */
int countBlackNodes(RBTree *t, Node *n) {
    int count = 0;
    while (n != t->NIL) {
        if (n->color == BLACK) count++;
        n = n->left; /* go left all the way */
    }
    return count;
}

/* Check if tree satisfies black balance (black height equal on all paths) */
int checkBlackBalance(RBTree *t, Node *n, int *bh) {
    if (n == t->NIL) { *bh = 1; return 1; }
    int leftBH, rightBH;
    int leftOK  = checkBlackBalance(t, n->left,  &leftBH);
    int rightOK = checkBlackBalance(t, n->right, &rightBH);
    if (!leftOK || !rightOK || leftBH != rightBH) return 0;
    *bh = leftBH + (n->color == BLACK ? 1 : 0);
    return 1;
}

Structural Operations

Like an HR department that can count employees, find someone's manager, their colleague on the same level, or their department head. These are all structural queries about relationships in the org chart (tree).

/* Count total nodes in tree */
int countNodes(RBTree *t, Node *n) {
    if (n == t->NIL) return 0;
    return 1 + countNodes(t, n->left) + countNodes(t, n->right);
}

/* Count leaf nodes (nodes with no real children) */
int countLeaves(RBTree *t, Node *n) {
    if (n == t->NIL) return 0;
    if (n->left == t->NIL && n->right == t->NIL) return 1;
    return countLeaves(t, n->left) + countLeaves(t, n->right);
}

/* Count internal nodes (non-leaf nodes) */
int countInternal(RBTree *t, Node *n) {
    return countNodes(t, n) - countLeaves(t, n);
}

/* Find parent of a node (already stored in n->parent) */
Node* findParent(Node *n) {
    return n->parent;
}

/* Find uncle = parent's sibling */
Node* findUncle(RBTree *t, Node *n) {
    Node *gp = n->parent->parent;
    if (gp == t->NIL) return t->NIL;
    if (n->parent == gp->left) return gp->right;
    else                        return gp->left;
}

/* Find grandparent = parent's parent */
Node* findGrandparent(Node *n) {
    return n->parent->parent;
}

/* Find sibling = other child of parent */
Node* findSibling(Node *n) {
    Node *p = n->parent;
    if (n == p->left) return p->right;
    else              return p->left;
}

/* Find ancestor — walk up parent chain to root */
void printAncestors(RBTree *t, Node *n) {
    Node *curr = n->parent;

        printf("%d ", curr->data);
        curr = curr->parent;

Property Checking Operations

// REAL-LIFE ANALOGY

Like a building inspector checking all safety codes. After any modification, an inspector verifies: root is stable (black), no adjacent weak spots (red-red), load distribution is equal (black height), and the ordering is correct (BST property). These are your validation functions.

C full RBT validation 
/* Check Rule 1: Root must be BLACK */
int checkRootBlack(RBTree *t) {
    return t->root->color == BLACK;
}

/* Check Rule 2: No two consecutive RED nodes */
int checkRedViolation(RBTree *t, Node *n) {
    if (n == t->NIL) return 1; /* NIL is fine */
    if (n->color == RED) {
        /* RED node must have BLACK children */
        if (n->left->color  == RED) return 0;
        if (n->right->color == RED) return 0;
    }
    return checkRedViolation(t, n->left) &&
           checkRedViolation(t, n->right);
}

/* Check Rule 3: Equal black height on all paths */
int checkBlackHeightRule(RBTree *t, Node *n, int *bh) {
    if (n == t->NIL) { *bh = 0; return 1; }
    int lbh, rbh;
    if (!checkBlackHeightRule(t, n->left,  &lbh)) return 0;
    if (!checkBlackHeightRule(t, n->right, &rbh)) return 0;
    if (lbh != rbh) return 0; /* violation! */
    *bh = lbh + (n->color == BLACK ? 1 : 0);
    return 1;
}

/* Check BST property: left < node < right */
int checkBST(RBTree *t, Node *n, int minV, int maxV) {
    if (n == t->NIL) return 1;
    if (n->data <= minV || n->data >= maxV) return 0;
    return checkBST(t, n->left,  minV,    n->data) &&
           checkBST(t, n->right, n->data, maxV);
}

/* Full validation: check all 5 RBT properties */
int validateRBT(RBTree *t) {
    int bh;
    if (!checkRootBlack(t))                           return 0;
    if (!checkRedViolation(t, t->root))               return 0;
    if (!checkBlackHeightRule(t, t->root, &bh))        return 0;
    if (!checkBST(t, t->root, -2147483648, 2147483647)) return 0;
    return 1; /* All properties satisfied! */
}

Utility Operations

// REAL-LIFE ANALOGY

Like a photocopier, mirror, and shredder for your tree. Copy = make an exact duplicate. Mirror = flip the tree so left and right are swapped (like a reflection). Clear/Destroy = shred all nodes from memory to avoid leaks.

C copy, clone, mirror, clear, destroy 
/* Copy tree — creates a deep copy of the subtree */
Node* copySubtree(RBTree *src, RBTree *dst, Node *n) {
    if (n == src->NIL) return dst->NIL;
    Node *newNode = createNode(dst, n->data);
    newNode->color = n->color;
    newNode->left  = copySubtree(src, dst, n->left);
    newNode->right = copySubtree(src, dst, n->right);
    if (newNode->left  != dst->NIL) newNode->left->parent  = newNode;
    if (newNode->right != dst->NIL) newNode->right->parent = newNode;
    return newNode;
}

RBTree* cloneTree(RBTree *src) {
    RBTree *dst = initTree();
    dst->root   = copySubtree(src, dst, src->root);
    if (dst->root != dst->NIL) dst->root->parent = dst->NIL;
    return dst;
}

/* Mirror tree — swap left and right pointers everywhere */
void mirrorSubtree(RBTree *t, Node *n) {
    if (n == t->NIL) return;
    Node *temp = n->left;  /* swap left and right */
    n->left    = n->right;
    n->right   = temp;
    mirrorSubtree(t, n->left);
    mirrorSubtree(t, n->right);
}

/* Clear all nodes (free memory), keep tree struct */
void clearSubtree(RBTree *t, Node *n) {
    if (n == t->NIL) return;
    clearSubtree(t, n->left);
    clearSubtree(t, n->right);
    free(n);
}

void clearTree(RBTree *t) {
    clearSubtree(t, t->root);
    t->root = t->NIL; /* reset root to NIL */
}

/* Destroy tree — free everything including NIL and struct */
void destroyTree(RBTree *t) {
    clearSubtree(t, t->root);
    free(t->NIL);  /* free sentinel */
    free(t);       /* free tree struct itself */
}

Path Operations

// REAL-LIFE ANALOGY

Like GPS navigation in a city map (tree). "Find path to node X" = navigation route from HQ (root) to a specific office (node). "LCA" = the last common junction before two roads diverge — the first point two employees share on the way to the CEO.

C path to node + root-to-leaf paths + LCA 
/* Find path from root to a target value */
int findPath(RBTree *t, Node *n, int target,
             int *path, int depth) {
    if (n == t->NIL) return 0;
    path[depth] = n->data;          /* record this node */
    if (n->data == target) return depth + 1; /* found! */
    int left  = findPath(t, n->left,  target, path, depth+1);
    if (left  > 0) return left;
    int right = findPath(t, n->right, target, path, depth+1);
    if (right > 0) return right;
    return 0; /* not in this subtree */
}

/* Print all root-to-leaf paths */
void printPaths(RBTree *t, Node *n, int *path, int depth) {
    if (n == t->NIL) return;
    path[depth] = n->data;
    if (n->left == t->NIL && n->right == t->NIL) {
        /* Reached a leaf: print the whole path */
        for (int i = 0; i <= depth; i++)
            printf("%d%s", path[i], i==depth ? "\n" : " → ");
        return;
    }
    printPaths(t, n->left,  path, depth+1);
    printPaths(t, n->right, path, depth+1);
}

/* Lowest Common Ancestor (LCA) of two nodes */
/* Works because RBT is a BST: use value comparison */
Node* findLCA(RBTree *t, Node *root, int a, int b) {
    if (root == t->NIL) return t->NIL;
    /* If both values are smaller, LCA is in left subtree */
    if (a < root->data && b < root->data)
        return findLCA(t, root->left, a, b);
    /* If both are larger, LCA is in right subtree */
    if (a > root->data && b > root->data)
        return findLCA(t, root->right, a, b);
    /* Otherwise current node is the split point = LCA */
    return root;
}

/* Recursive BST insert helper (returns root of modified subtree) */
Node* recursiveBSTInsert(RBTree *t, Node *root, Node *z) {
    if (root == t->NIL) return z; /* base case: place here */
    if (z->data < root->data) {
        root->left         = recursiveBSTInsert(t, root->left, z);
        root->left->parent = root;
    } else if (z->data > root->data) {
        root->right         = recursiveBSTInsert(t, root->right, z);
        root->right->parent = root;
    }
    return root;
}

/* Recursive insert (wraps BST insert + fixup) */
void insertRecursive(RBTree *t, int val) {
    Node *z = createNode(t, val);
    t->root  = recursiveBSTInsert(t, t->root, z);
    t->root->parent = t->NIL;
    fixInsert(t, z); /* same fixup as iterative */
}

/* Recursive search */
Node* searchRec(RBTree *t, Node *node, int val) {
    if (node == t->NIL || node->data == val) return node;
    if (val < node->data) return searchRec(t, node->left,  val);
    else                  return searchRec(t, node->right, val);
}

/* Recursive inorder traversal */
void inorderRec(RBTree *t, Node *n) {
    if (n == t->NIL) return;
    inorderRec(t, n->left);
    printf("%d(%s) ", n->data, n->color==RED?"R":"B");
    inorderRec(t, n->right);
}

Iterative Operations

// REAL-LIFE ANALOGY

Like following a map step by step rather than using GPS recursion. You manually keep track of where you are using a stack (for DFS) or queue (for BFS) instead of relying on the call stack. More memory-efficient and avoids crash on very deep trees.

C iterative search + inorder traversal using stack 
/* Iterative search (simplest iterative op) */
Node* iterativeSearch(RBTree *t, int val) {
    Node *curr = t->root;
    while (curr != t->NIL) {
        if      (val == curr->data) return curr;
        else if (val <  curr->data) curr = curr->left;
        else                       curr = curr->right;
    }
    return t->NIL;
}

/* Iterative inorder traversal using explicit stack */
void iterativeInorder(RBTree *t) {
    Node *stack[100];
    int   top  = -1;
    Node *curr = t->root;

    while (curr != t->NIL || top >= 0) {
        /* Go as far left as possible */
        while (curr != t->NIL) {
            stack[++top] = curr;
            curr = curr->left;
        }
        curr = stack[top--];         /* pop from stack */
        printf("%d ", curr->data);  /* visit */
        curr = curr->right;          /* go right */
    }
}

/* Iterative insert — standard iterative BST insert + fixup */
/* (already shown in Section 2 as insertNode) */

/* Iterative preorder using stack */
void iterativePreorder(RBTree *t) {
    if (isEmpty(t)) return;
    Node *stack[100];
    int top = -1;
    stack[++top] = t->root;
    while (top >= 0) {
        Node *n = stack[top--];
        printf("%d ", n->data);
        /* push right first so left is processed first */
        if (n->right != t->NIL) stack[++top] = n->right;
        if (n->left  != t->NIL) stack[++top] = n->left;
    }
}

/* Iterative delete — uses iterativeSearch + deleteNode logic */
void iterativeDelete(RBTree *t, int val) {
    Node *z = iterativeSearch(t, val);
    if (z != t->NIL)
        deleteNode(t, val); /* deleteNode is already iterative */
    else
        printf("Node %d not found\n", val);
}
