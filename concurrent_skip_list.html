Section 01 · Initialization
Creating a Concurrent Skip List
Before any operation, we must set up the structure — allocating memory, configuring levels, and making it thread-safe from the start.
What is a Skip List?
A Skip List is a probabilistic data structure — like a linked list on steroids. It has multiple layers of linked lists stacked on top of each other, where higher layers "skip" over many elements, giving you O(log n) average search time.

A Concurrent Skip List adds thread-safety so multiple threads can read and write simultaneously without corrupting data.
💡 Think of it as a highway system: the bottom layer (local road) visits every node. Higher layers are express highways that skip blocks to reach distant nodes faster.
Key properties:
  • Probabilistic balancing — no need for complex rotations like AVL trees
  • Average O(log n) for insert, delete, and search
  • Nodes at higher levels are chosen randomly (usually 50% probability each level)
  • Header node acts as entry point to all levels
  • Each level forms a sorted linked list

Section 02 · Insertion
Inserting Elements Safely
Insertion must find the right position, generate a random level, link new nodes, and do all this without breaking concurrent reads or writes.
How Insertion Works
Inserting into a concurrent skip list is trickier than a normal linked list because other threads might be reading or writing at the same time.
🔑 The key insight: we lock nodes we plan to modify (not the whole list), update forward pointers level-by-level, then release locks.
Insertion Steps:
01Search for the correct position — traverse from top level down
02Lock nodes at each level that need updating (predecessors)
03Validate those nodes are still valid (not deleted by another thread)
04Generate a random level for the new node
05Create the new node and link its forward pointers
06Update predecessor forward pointers to point to new node
07Release all locks and increment node count

C Code — Insertion
/* Generate a random level (probabilistic) */ int randomLevel() { int level = 1; while ((double)rand() / RAND_MAX < PROB && level < MAX_LEVEL) level++; return level; /* Returns 1..MAX_LEVEL */ } /* Thread-safe insert */ void insert(SkipList *sl, int key, int value) { /* update[] stores predecessors at each level */ Node *update[MAX_LEVEL]; Node *curr = sl->header; /* Step 1: Search insert position top-to-bottom */ for (int i = sl->currentLevel - 1; i >= 0; i--) { while (curr->forward[i] && curr->forward[i]->key < key) curr = curr->forward[i]; /* move forward on level i */ update[i] = curr; /* save predecessor */ } /* Step 2: Lock for thread safety */ pthread_mutex_lock(&sl->lock); /* Step 3: Validate key doesn't already exist */ curr = curr->forward[0]; if (curr && curr->key == key) { curr->value = value; /* Update if duplicate key */ pthread_mutex_unlock(&sl->lock); return; } /* Step 4: Generate random level for new node */ int newLevel = randomLevel(); /* Step 5: If new node has more levels, update skip list level */ if (newLevel > sl->currentLevel) { for (int i = sl->currentLevel; i < newLevel; i++) update[i] = sl->header; sl->currentLevel = newLevel; } /* Step 6: Create new node */ Node *newNode = createNode(key, value); /* Step 7: Link new node into each level */ for (int i = 0; i < newLevel; i++) { newNode->forward[i] = update[i]->forward[i]; update[i]->forward[i] = newNode; /* splice in */ } sl->size++; pthread_mutex_unlock(&sl->lock); /* Step 8: Release lock */ }

Section 03 · Deletion
Removing Nodes Safely
Deletion in a concurrent environment uses a two-phase approach: first logically mark a node deleted, then physically remove it — preventing use-after-free bugs.
Two-Phase Deletion
Concurrent deletion is dangerous: while you're removing a node, another thread might be reading it. The solution is logical + physical deletion.
⚠️ Logical deletion: mark the node as "deleted" (set a flag). Other threads can see the flag and skip it.
⚠️ Physical deletion: actually unlink and free the node only when it's safe (no threads hold a reference).
Deletion Steps:
01Search for the target node, collecting predecessors
02Lock the target path (predecessor nodes)
03Validate node links are still intact
04Remove forward references at each level
05Mark node as logically deleted
06Physically free the node's memory
07Release locks and decrement count

C Code — Deletion
/* Thread-safe deletion */ int deleteKey(SkipList *sl, int key) { Node *update[MAX_LEVEL]; Node *curr = sl->header; /* Step 1: Search target, collect predecessors */ for (int i = sl->currentLevel - 1; i >= 0; i--) { while (curr->forward[i] && curr->forward[i]->key < key) curr = curr->forward[i]; update[i] = curr; } /* Step 2: Lock for thread safety */ pthread_mutex_lock(&sl->lock); /* Step 3: Get the actual target node */ curr = curr->forward[0]; /* Step 4: Validate it actually exists */ if (!curr || curr->key != key) { pthread_mutex_unlock(&sl->lock); return 0; /* Not found */ } /* Step 5: Remove forward references at each level */ for (int i = 0; i < sl->currentLevel; i++) { if (update[i]->forward[i] != curr) break; update[i]->forward[i] = curr->forward[i]; /* bypass node */ } /* Step 6: Physically free memory */ free(curr); sl->size--; /* Step 7: Lower level if top levels are empty */ while (sl->currentLevel > 1 && sl->header->forward[sl->currentLevel - 1] == NULL) sl->currentLevel--; pthread_mutex_unlock(&sl->lock); return 1; /* Success */ }

Section 04 · Search
Searching Concurrently
Search is the most frequently called operation. In a concurrent skip list, reads can often proceed lock-free, making this O(log n) on average.
How Search Works
Search starts at the highest level of the header node and works down — like taking the express lane until you overshoot, then dropping to local roads.
🔍 Search is often wait-free or lock-free — meaning readers don't need to acquire locks. They just traverse and validate node visibility.
Key suboperations:
  • Start from Highest Level: Begin at the topmost occupied level
  • Move Forward in Level: Advance while next key < target
  • Drop to Lower Level: When next key ≥ target, go down
  • Validate Node Visibility: Check node not marked as deleted
  • Compare Keys: At level 0, do an exact key comparison
  • Return Thread-Safe Result: Copy the value before returning

/* Lock-free concurrent search */ int search(SkipList *sl, int key, int *outValue) { Node *curr = sl->header; /* Start from the highest occupied level */ for (int i = sl->currentLevel - 1; i >= 0; i--) { /* Move forward while next node key < target */ while (curr->forward[i] && curr->forward[i]->key < key) curr = curr->forward[i]; /* If next key > target, drop one level down */ } /* Arrived at level 0 — check exact match */ curr = curr->forward[0]; /* Validate: node exists and key matches */ if (curr && curr->key == key) { *outValue = curr->value; /* Copy value (thread-safe read) */ return 1; /* Found! */ } return 0; /* Not found */ } /* Example usage */ int main() { SkipList *sl = initSkipList(); insert(sl, 10, 100); insert(sl, 20, 200); insert(sl, 30, 300); int val; if (search(sl, 20, &val)) printf("Found: %d\n", val); /* Prints: Found: 200 */ return 0; }

Section 05 · Synchronization
Managing Concurrent Access
Synchronization is the heart of concurrency. Without it, two threads modifying the same node simultaneously can corrupt data permanently.
Three Locking Strategies
Coarse-Grained LockingOne big lock for the whole list. Simple but slow under high contention.
Fine-Grained LockingEach node has its own lock. Allows more parallelism but complex to manage.
Optimistic LockingDon't lock during search; lock only to validate and commit changes.
Lock-Free (CAS)Uses atomic Compare-And-Swap instructions — no locks at all!
🔐 Fine-grained locking is the most common in production skip lists. It locks only the nodes needed for an operation, letting other threads work on unrelated parts simultaneously.

Section 06 · Validation
Ensuring Structural Integrity
After concurrent modifications, the structure must be validated — detecting orphaned nodes, broken links, and logically-deleted nodes still reachable.
Why Validation Matters
In a concurrent environment, between the time you search for an insert position and actually insert, another thread may have:
  • Deleted one of your predecessor nodes
  • Inserted a node with the same key
  • Restructured the links you were planning to use
🛡️ Validation re-checks all invariants while holding locks before committing changes. If validation fails, the operation retries from scratch.
Validation checks:
  • Check Forward Links: Predecessor's forward[i] still points to successor
  • Verify Node Consistency: Node is not marked as logically deleted
  • Detect Marked Nodes: The "deleted" flag is not set on key nodes
  • Confirm Safe Modification: All conditions met — safe to proceed

C Code — Validation
/* Node with a "deleted" flag for logical deletion */ typedef struct VNode { int key, value; int marked; /* 1 = logically deleted */ int fullyLinked; /* 1 = visible at all levels */ pthread_mutex_t lock; struct VNode *forward[MAX_LEVEL]; } VNode; /* Validate before insert/delete — call while holding locks */ int validate(VNode *pred, VNode *curr, int level) { /* Check 1: predecessor is not deleted */ if (pred->marked) return 0; /* FAIL: pred was deleted */ /* Check 2: current node is not deleted */ if (curr->marked) return 0; /* FAIL: target was deleted */ /* Check 3: forward link still intact */ if (pred->forward[level] != curr) return 0; /* FAIL: another thread changed the link */ return 1; /* All checks passed — safe to modify */ } /* Usage inside insert */ void safeInsert(VNode *header, int key, int val) { while (1) { /* Retry loop */ VNode *preds[MAX_LEVEL], *succs[MAX_LEVEL]; /* ... find preds and succs ... */ /* Lock preds */ for (int i = 0; i < MAX_LEVEL; i++) pthread_mutex_lock(&preds[i]->lock); /* Validate all levels */ int valid = 1; for (int i = 0; i < MAX_LEVEL; i++) valid &= validate(preds[i], succs[i], i); if (!valid) { /* Validation failed — retry */ for (int i = 0; i < MAX_LEVEL; i++) pthread_mutex_unlock(&preds[i]->lock); continue; /* Go back to top of while loop */ } /* Safe to insert now */ /* ... link new node ... */ break; } }

Section 07 · Update
Updating Existing Values
Updating is simpler than insert/delete — the node stays in place, but we must lock it to prevent two threads writing different values simultaneously.
Update Strategy
Since the key doesn't change, the node's position in the skip list is unchanged. We only need to:
01Search for the node with the target key (lock-free read)
02Lock the specific node (fine-grained)
03Validate: node not deleted and key matches
04Modify the stored value atomically
05Release the node lock
⚡ Update is the most parallelism-friendly operation — two threads can update different nodes simultaneously with zero contention.

/* Thread-safe value update */ int update(SkipList *sl, int key, int newValue) { Node *curr = sl->header; /* Step 1: Lock-free search for the node */ for (int i = sl->currentLevel - 1; i >= 0; i--) { while (curr->forward[i] && curr->forward[i]->key < key) curr = curr->forward[i]; } curr = curr->forward[0]; /* Step 2: Node must exist and key must match */ if (!curr || curr->key != key) return 0; /* Not found */ /* Step 3: Lock just this node, then update */ pthread_mutex_lock(&sl->lock); /* Step 4: Re-validate after locking (state may have changed) */ if (curr->key == key) { int old = curr->value; curr->value = newValue; /* Atomic write of value */ printf("Updated key %d: %d -> %d\n", key, old, newValue); pthread_mutex_unlock(&sl->lock); return 1; /* Success */ } pthread_mutex_unlock(&sl->lock); return 0; /* Key disappeared (deleted by another thread) */ } /* Atomic replace using CAS (lock-free approach) */ int atomicUpdate(_Atomic int *target, int expected, int newVal) { return atomic_compare_exchange_strong(target, &expected, newVal); }

Section 09 · Level Management
Managing the Layer Structure
Levels are what give skip lists their speed. Managing them properly — growing, shrinking, rebalancing — keeps performance at O(log n).
How Levels Work
When a new node is inserted, it's randomly assigned a level (1 to MAX_LEVEL). Each level appears with probability P = 0.5.
📐 Statistically: ~50% of nodes are at level 1 only, ~25% reach level 2, ~12.5% reach level 3, etc. This distribution gives the O(log n) guarantee.
Level Management rules:
  • Increase Current Level: When a new node's random level exceeds current, raise the list's level
  • Decrease Current Level: After deletion, if top levels are empty, lower the level count
  • Remove Empty Levels: Trim dead levels to avoid wasted pointer traversal
  • Maintain Balanced Levels: Probabilistic approach keeps the structure naturally balanced

Section 10 · Pointer Management
Maintaining Forward Pointers
Forward pointers are the veins of a skip list. Corrupted pointers = corrupted data. Atomic pointer updates ensure thread-safe linking.
Pointer Management Rules
Each node has an array of forward[] pointers — one per level. Managing these safely is critical:
⚡ Atomic pointer updates use CPU-level instructions that guarantee a pointer is updated in a single, uninterruptible operation — no thread can see a "half-updated" pointer.
Key principles:
  • Update Pointer Links: Always update bottom-up (level 0 first) on delete, top-down on insert
  • Redirect References: When bypassing a node, predecessor's forward must point to successor
  • Use Atomic Pointer Update: Prevents torn reads on 64-bit pointer values
  • Prevent Broken Links: Never leave a dangling pointer, even briefly
  • Initialize Pointer Arrays: All forward[] start as NULL
Main & Sub Operations
Maintain Forward PointersKeep all forward[] consistent

/* ── Safe pointer initialization ── */ Node* createNode(int key, int value) { Node *n = (Node*)malloc(sizeof(Node)); n->key = key; n->value = value; for (int i = 0; i < MAX_LEVEL; i++) n->forward[i] = NULL; /* ALWAYS initialize to NULL */ return n; } /* ── Atomic pointer type (C11) ── */ #include <stdatomic.h> typedef struct AtomicNode { int key, value; _Atomic(struct AtomicNode*) fwd[MAX_LEVEL]; } AtomicNode; /* Atomic read of a forward pointer */ AtomicNode* getNext(AtomicNode *node, int level) { return atomic_load(&node->fwd[level]); } /* Atomic write of a forward pointer */ void setNext(AtomicNode *node, int level, AtomicNode *next) { atomic_store(&node->fwd[level], next); } /* CAS-based pointer redirect (lock-free link update) */ int casNext(AtomicNode *node, int level, AtomicNode *expected, AtomicNode *desired) { return atomic_compare_exchange_strong( &node->fwd[level], &expected, desired ); /* Returns 1 if swap succeeded, 0 if expected didn't match */ } /* Prevent broken links — insert new node safely */ void safeLink(AtomicNode *pred, AtomicNode *succ, AtomicNode *newNode, int level) { /* First: new node points to successor */ setNext(newNode, level, succ); /* Then: predecessor points to new node (atomic swap) */ while (!casNext(pred, level, succ, newNode)) ; /* Retry if another thread interfered */ }

Section 11 · Conflict Handling
Resolving Thread Conflicts
When two threads collide — trying to modify the same node simultaneously — conflict handling decides who wins, who retries, and who aborts.
Types of Conflicts
Data RaceTwo threads read-then-write the same memory without synchronization. Result: corrupted data.
ABA ProblemThread A reads value A, Thread B changes it to B then back to A. Thread A thinks nothing changed but the world has changed.
Lock ContentionMany threads waiting for the same lock. Kills performance even with correct logic.
DeadlockThread 1 holds Lock A waiting for B; Thread 2 holds Lock B waiting for A. Both stuck forever.
🔄 Retry strategy: if a conflict is detected, release all locks and restart the operation. This is simple and correct, but may cause starvation under extreme contention.

/* Backoff strategy to reduce lock contention */ #include <time.h> #include <unistd.h> /* Exponential backoff: wait longer each retry */ void backoff(int attempt) { struct timespec ts; ts.tv_sec = 0; ts.tv_nsec = (1 << attempt) * 1000; /* 1us, 2us, 4us... */ nanosleep(&ts, NULL); } /* Insert with retry on conflict */ void insertWithRetry(SkipList *sl, int key, int val) { int attempt = 0; while (1) { int result = pthread_mutex_trylock(&sl->lock); if (result == 0) { /* Got the lock! */ /* ... do insert ... */ pthread_mutex_unlock(&sl->lock); return; } /* Lock was busy — backoff and retry */ backoff(attempt++); if (attempt > 10) attempt = 10; /* Cap backoff */ } } /* Detect ABA problem using version counters */ typedef struct Stamped { Node *ptr; int version; /* Incremented on each change */ } Stamped; /* CAS on stamped pointer: checks BOTH pointer and version */ int stampedCAS(Stamped *target, Stamped expected, Stamped desired) { if (target->ptr == expected.ptr && target->version == expected.version) { *target = desired; /* Atomically in real code */ return 1; } return 0; }

Section 13 · Memory Management
Safe Memory Allocation & Deallocation
In concurrent systems, memory bugs are silent killers — use-after-free, double-free, and memory leaks. Hazard pointers and garbage collection prevent these.
The Memory Danger Zone
☠️ Use-after-free: Thread A deletes node X. Thread B has a pointer to X and reads it — undefined behavior (crash or wrong data).

🛡️ Hazard Pointers: Each thread announces the nodes it's currently using. A node is only freed when no thread has it hazard-listed.
Memory Management Strategies:
Hazard PointersThread registers pointer before use; GC checks all registered pointers before freeing
Epoch-Based ReclamationGroup operations into epochs; free memory from old epochs when all threads advance
RCU (Read-Copy-Update)Readers run lockfree; writers make copies; old copies freed when all readers leave
Reference CountingEach node has a refcount; freed only when count reaches 0
Main & Sub Operations

Section 14 · Analytical Operations
Measuring Performance
Analytics tell you if your concurrent skip list is actually performing well — or just looking correct while being slow.
Key Metrics
Thread ContentionHow often threads block each other. High contention = poor scalability.
ThroughputOperations per second under N threads. Should scale linearly with lock-free designs.
Search ComplexityAverage path length during search. Should approximate log(N) for a balanced skip list.
Lock Efficiency% of time locks are uncontested. >95% means your locking is well-designed.
📊 Amdahl's Law: Speedup = 1 / (S + (1-S)/N) where S = fraction of sequential code. Even 5% sequential code limits speedup to 20× regardless of thread count.

/* ── Range Query ── */ int rangeQuery(SkipList *sl, int lo, int hi, int *results, int maxResults) { Node *curr = sl->header; int count = 0; /* Skip to first node >= lo */ for (int i = sl->currentLevel - 1; i >= 0; i--) while (curr->forward[i] && curr->forward[i]->key < lo) curr = curr->forward[i]; curr = curr->forward[0]; while (curr && curr->key <= hi && count < maxResults) { results[count++] = curr->key; curr = curr->forward[0]; } return count; /* Number of keys found */ } /* ── Snapshot (consistent read) ── */ typedef struct Snapshot { int *keys; int count; } Snapshot; Snapshot takeSnapshot(SkipList *sl) { pthread_mutex_lock(&sl->lock); /* Freeze the world */ int n = sl->size; Snapshot snap = {(int*)malloc(n * sizeof(int)), 0}; Node *curr = sl->header->forward[0]; while (curr) { snap.keys[snap.count++] = curr->key; curr = curr->forward[0]; } pthread_mutex_unlock(&sl->lock); /* Unfreeze */ return snap; /* Caller must free snap.keys */ } /* ── Merge two skip lists ── */ SkipList* merge(SkipList *a, SkipList *b) { SkipList *result = initSkipList(); Node *ca = a->header->forward[0]; Node *cb = b->header->forward[0]; while (ca || cb) { if (!cb || (ca && ca->key <= cb->key)) { insert(result, ca->key, ca->value); ca = ca->forward[0]; } else { insert(result, cb->key, cb->value); cb = cb->forward[0]; } } return result; }
