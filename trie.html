What is a Trie?
A Trie (pronounced "try," from retrieval) is a special type of tree used to store strings where each node represents a single character. Unlike a binary tree, each node in a trie can have up to 26 children (one for each letter of the alphabet).

In a Linked List Based Trie, instead of allocating a fixed array of 26 pointers for every node, each node keeps a linked list of only its actual children — saving memory when the trie is sparse.

The path from the root to any node spells out a prefix. When a node is marked as the end of a word, the full path spells out that complete word.
🏙️ Real-life Analogy
Think of a Trie like a physical dictionary organized in a building. The root is the lobby. Each floor is a letter. To find "cat", you go to floor 'c', then branch to 'a', then to 't'. If the 't' door has a ⭐ sign, it means "cat" is a real word. Multiple words sharing "ca" (cat, car, can) all share the same first two floors — that's the key efficiency of a Trie!

C — Node Structure
#include <stdio.h>
#include <stdlib.h>
#include <string.h>

/* Each trie node stores a character and */
/* a linked list of children nodes       */

typedef struct TrieNode {
    char data;              /* Character this node holds */
    int  is_end_of_word;    /* 1 = complete word ends here */
    struct TrieNode *children; /* First child (linked list) */
    struct TrieNode *next;     /* Next sibling node */
} TrieNode;

/* The Trie itself holds just the root */
typedef struct {
    TrieNode *root;         /* Root node (empty char) */
} Trie;

Section 01

Basic
Operations

The foundation of every Trie — creating the structure, initializing nodes, and checking if the trie is empty. These are the building blocks for every other operation.

🏗️
Create Trie & Initialize Root Node
Creating a Trie means allocating memory for the root node. The root is special — it doesn't hold any real character. It's just the starting point. Every trie begins with exactly one root node whose children are NULL and is_end_of_word = 0.
🏢 Analogy
Creating a Trie is like building the lobby of a hotel. The lobby itself doesn't represent any word — it's just the entrance. From the lobby, corridors (children) branch out to the first letter of every word.
C — Create Trie
/* Step 1: Create a single trie node */
TrieNode* createNode(char ch) {
    TrieNode *node = (TrieNode*) malloc(sizeof(TrieNode));
    node->data         = ch;    /* Store the character */
    node->is_end_of_word = 0;  /* Not end of word yet */
    node->children     = NULL; /* No children yet */
    node->next         = NULL; /* No sibling yet */
    return node;
}

/* Step 2: Create the whole Trie with an empty root */
Trie* createTrie() {
    Trie *t = (Trie*) malloc(sizeof(Trie));
    t->root = createNode('\0'); /* Root has no real char */
    return t;
}

/* Step 3: Check if Trie is empty */
int isEmptyTrie(Trie *t) {
    return (t->root->children == NULL);
}

/* Usage */
int main() {
    Trie *myTrie = createTrie();
    if (isEmptyTrie(myTrie))
        printf("Trie is empty!\n");
    return 0;
}

Section 02

Insertion
Operations

Adding words to the trie one character at a time. Each character either matches an existing node or creates a new child node.

➕
Insert Word — How It Works
Inserting a word like "cat" means:
1
Start at the root node
2
For each character ('c', 'a', 't'), check if a child with that character exists in the current node's children linked list
3
If the child exists, move to it. If it doesn't exist, create a new node and append it to the children linked list
4
After the last character, mark is_end_of_word = 1 to signal a complete word
📂 Analogy
Inserting a word is like filing a document. You open drawer 'C', then sub-folder 'A', then tab 'T'. If any of those don't exist, you create them. Finally, you put a flag on the 'T' tab to say "a complete word ends here."
C — Iterative Insert
/* Find a child with a specific character */
TrieNode* findChild(TrieNode *parent, char ch) {
    TrieNode *child = parent->children;
    while (child != NULL) {
        if (child->data == ch)
            return child; /* Found it! */
        child = child->next; /* Move to next sibling */
    }
    return NULL; /* Not found */
}

/* Add a new child to a parent node */
void addChild(TrieNode *parent, TrieNode *newChild) {
    newChild->next    = parent->children; /* Link to existing list */
    parent->children  = newChild;         /* New node at front */
}

/* Iterative word insertion */
void insertWord(Trie *t, char *word) {
    TrieNode *current = t->root;
    int i;

    for (i = 0; word[i] != '\0'; i++) {
        char ch = word[i];
        TrieNode *child = findChild(current, ch);

        if (child == NULL) {
            /* Character not found — create new node */
            child = createNode(ch);
            addChild(current, child);
        }
        current = child; /* Move down the trie */
    }

    current->is_end_of_word = 1; /* Mark end of word */
    printf("Inserted: %s\n", word);
}
Recursive Insert variant
C — Recursive Insert
/* Recursive version: insert word[index..] into node */
void insertRecursive(TrieNode *node, char *word, int index) {
    /* Base case: all characters placed */
    if (word[index] == '\0') {
        node->is_end_of_word = 1;
        return;
    }
    char ch = word[index];
    TrieNode *child = findChild(node, ch);

    if (child == NULL) {
        child = createNode(ch);
        addChild(node, child);
    }
    insertRecursive(child, word, index + 1);
}

/* Wrapper for recursive insert */
void insertWordRec(Trie *t, char *word) {
    insertRecursive(t->root, word, 0);
}

Section 03

Deletion
Operations

Removing words from a trie requires care — we must only delete nodes that are no longer part of any other word.

🗑️
Delete Word — Rules
Deletion is the trickiest trie operation. You can't just remove a node — that might destroy other words sharing the same path. Three cases to consider:
1
Word doesn't exist — do nothing.
2
Word is a prefix of another word (e.g., deleting "car" when "care" exists) — just unmark is_end_of_word. Don't delete nodes.
3
Word shares prefix with another word (e.g., deleting "cat" when "car" exists) — delete only nodes after the branch point.
4
Word has unique path — safely delete all nodes of the word from the bottom up.
📂 Analogy
Deleting "cat" from a filing cabinet that also holds "can": you remove only the 't' drawer, not the 'c' or 'a' drawers (since 'c→a' is still needed by 'can'). Never remove shared folders!
C — Delete Word
/* Remove child from parent's linked list */
void removeChild(TrieNode *parent, char ch) {
    TrieNode **ptr = &(parent->children);
    while (*ptr != NULL) {
        if ((*ptr)->data == ch) {
            TrieNode *toDelete = *ptr;
            *ptr = toDelete->next; /* Bypass it */
            free(toDelete);
            return;
        }
        ptr = &((*ptr)->next);
    }
}

/* Returns 1 if node can be deleted (no children, not end) */
int canDelete(TrieNode *node) {
    return (node->children == NULL && !node->is_end_of_word);
}

/* Recursive delete — returns 1 if current node should be freed */
int deleteHelper(TrieNode *node, char *word, int index) {
    if (word[index] == '\0') {
        /* Reached end of word */
        if (!node->is_end_of_word) return 0; /* Word not here */
        node->is_end_of_word = 0;
        return (node->children == NULL); /* Delete if no children */
    }
    TrieNode *child = findChild(node, word[index]);
    if (child == NULL) return 0; /* Word not in trie */

    int shouldDelete = deleteHelper(child, word, index + 1);

    if (shouldDelete) {
        removeChild(node, word[index]); /* Clean up */
        return canDelete(node);
    }
    return 0;
}

void deleteWord(Trie *t, char *word) {
    deleteHelper(t->root, word, 0);
}

Section 04

Searching
Operations

Finding words and characters in a trie. This is where the trie really shines — O(L) time, regardless of how many words are stored.

🔍
Search Word
To search for a word, traverse the trie character by character. If at any point a character's child node doesn't exist, the word is not in the trie. If we reach the end and is_end_of_word = 1, the word is found.
C — Search Operations
/* Search for exact word — returns 1 if found */
int searchWord(Trie *t, char *word) {
    TrieNode *current = t->root;
    int i;

    for (i = 0; word[i] != '\0'; i++) {
        TrieNode *child = findChild(current, word[i]);
        if (child == NULL)
            return 0; /* Character missing → word absent */
        current = child;
    }

    return current->is_end_of_word; /* Must be marked as word */
}

/* Search for a single character at depth 'level' */
TrieNode* searchCharAt(Trie *t, char *prefix, int level) {
    TrieNode *current = t->root;
    int i;
    for (i = 0; i <= level && current != NULL; i++)
        current = findChild(current, prefix[i]);
    return current;
}

/* Partial match — checks if any word CONTAINS substring */
int partialSearch(TrieNode *node, char *sub, int idx) {
    if (sub[idx] == '\0') return 1; /* All chars matched */
    if (node == NULL) return 0;

    TrieNode *child = node->children;
    while (child != NULL) {
        if (child->data == sub[idx])
            if (partialSearch(child, sub, idx+1))
                return 1;
        child = child->next;
    }
    return 0;
}

/* Main usage */
int main() {
    Trie *t = createTrie();
    insertWord(t, "apple");
    insertWord(t, "app");

    printf("%d\n", searchWord(t, "apple")); /* 1 - found */
    printf("%d\n", searchWord(t, "ap"));    /* 0 - not a word */
    return 0;
}

Section 05

Prefix
Operations

Prefix operations are what make Tries uniquely powerful. Finding all words with a given prefix, checking if a prefix exists, and finding the longest matching prefix.

🔤
Prefix Operations Explained
A prefix is the beginning part of a word. In a trie, checking for a prefix is similar to word search — but we don't need is_end_of_word to be true. If we can traverse all characters of the prefix without hitting NULL, the prefix exists.

Longest Prefix Match: Walk the trie as far as the input matches stored words — useful in IP routing tables and autocorrect.
📱 Real Use Case
When you type "go" in Google Search, it finds all words starting with "go" — "google", "good", "gone", "golf" — in milliseconds. This is Prefix Search in a Trie at massive scale!
C — Prefix Operations
/* Check if prefix exists in trie */
int checkPrefix(Trie *t, char *prefix) {
    TrieNode *current = t->root;
    int i;
    for (i = 0; prefix[i] != '\0'; i++) {
        TrieNode *child = findChild(current, prefix[i]);
        if (child == NULL) return 0; /* Prefix missing */
        current = child;
    }
    return 1; /* Prefix found */
}

/* Find node where prefix ends */
TrieNode* findPrefix(Trie *t, char *prefix) {
    TrieNode *current = t->root;
    int i;
    for (i = 0; prefix[i] != '\0'; i++) {
        TrieNode *child = findChild(current, prefix[i]);
        if (child == NULL) return NULL;
        current = child;
    }
    return current; /* Node at end of prefix */
}

/* Find longest matching prefix */
int longestPrefixMatch(Trie *t, char *word) {
    TrieNode *current = t->root;
    int i, longest = 0;
    for (i = 0; word[i] != '\0'; i++) {
        TrieNode *child = findChild(current, word[i]);
        if (child == NULL) break;
        current = child;
        if (current->is_end_of_word)
            longest = i + 1; /* Update longest match */
    }
    return longest; /* Length of longest prefix */
}

Section 06

Traversal
Operations

Visiting every node in the trie in a specific order — preorder, postorder, and level-order traversal each have unique purposes.

🌲
Three Traversal Methods
Preorder: Visit the node first, then recurse into children. Best for printing the trie structure or copying it.

Postorder: Recurse into children first, then visit the node. Best for safely deleting the entire trie (free children before parent).

Level-order (BFS): Visit all nodes at depth 1, then depth 2, etc. Uses a queue. Best for visualizing the trie level by level.
🗺️ Analogy
Preorder is like reading a book chapter-by-chapter. Postorder is like packing boxes — pack the smallest boxes first, then the big ones that contain them. Level-order is like scanning a photo row by row.
C — Three Traversals
/* PREORDER: node → children (left to right) */
void preorder(TrieNode *node, char *path, int depth) {
    if (node == NULL) return;

    path[depth] = node->data;
    path[depth+1] = '\0';
    printf("Visit: %c  Path so far: %s\n", node->data, path);

    TrieNode *child = node->children;
    while (child != NULL) {
        preorder(child, path, depth+1);
        child = child->next;
    }
}

/* POSTORDER: children first, then node */
void postorder(TrieNode *node) {
    if (node == NULL) return;

    TrieNode *child = node->children;
    while (child != NULL) {
        postorder(child);      /* Visit children first */
        child = child->next;
    }
    printf("Visit: %c\n", node->data); /* Then self */
}

/* LEVEL-ORDER (BFS) using a queue array */
void levelOrder(TrieNode *root) {
    TrieNode *queue[500];
    int front = 0, back = 0;

    queue[back++] = root;

    while (front < back) {
        TrieNode *node = queue[front++];
        printf("%c ", node->data);

        TrieNode *child = node->children;
        while (child != NULL) {
            queue[back++] = child; /* Enqueue children */
            child = child->next;
        }
    }
}

Section 07

AutoComplete
Operations

The feature that makes search engines and messaging apps feel magical — suggesting words based on what you've typed so far.

💡
How AutoComplete Works
AutoComplete uses three steps:

1. Find the prefix node — traverse the trie to find where the typed prefix ends.
2. Collect all words from that node downward using DFS.
3. Return top-K suggestions — optionally sorted by frequency or alphabetically.
📱 Real Use Case
When you type "prog" in your IDE, it suggests "program", "progress", "programming". Your IDE literally uses a trie (or prefix tree) to do this in microseconds!
C — AutoComplete & Top-K Suggestions
/* Collect all words from a given node */
void collectWords(TrieNode *node, char *prefix,
                  int depth, char results[][50], int *count) {
    if (node == NULL) return;

    prefix[depth] = node->data;
    prefix[depth+1] = '\0';

    if (node->is_end_of_word) {
        strcpy(results[*count], prefix + 1); /* Skip root */
        (*count)++;
    }

    TrieNode *child = node->children;
    while (child != NULL && *count < 10) {
        collectWords(child, prefix, depth+1, results, count);
        child = child->next;
    }
}

/* AutoComplete: given prefix, return matching words */
int autoComplete(Trie *t, char *prefix,
                 char results[][50]) {
    TrieNode *node = findPrefix(t, prefix);
    if (node == NULL) return 0;

    char path[100];
    int plen = strlen(prefix);
    strncpy(path + 1, prefix, plen); /* Build path with prefix */
    int count = 0;

    collectWords(node, path, plen, results, &count);
    return count;
}

/* Top-K: return only k most frequent words */
void topKSuggestions(Trie *t, char *prefix, int k) {
    char results[10][50];
    int n = autoComplete(t, prefix, results);
    int limit = (n < k) ? n : k;
    printf("Top-%d for '%s':\n", k, prefix);
    int i;
    for (i = 0; i < limit; i++)
        printf("  %d. %s\n", i+1, results[i]);
}

Section 08

Counting
Operations

Counting total words, nodes, and prefix matches in a trie — essential for analytics and memory tracking.

🔢
Count Operations
Count Total Words: DFS through the trie; increment counter every time we see is_end_of_word = 1.

Count Total Nodes: Count every node visited (excluding root).

Count Prefix Matches: Navigate to the prefix node, then count all words under it.
C — Counting Operations
/* Count all complete words in trie */
int countWords(TrieNode *node) {
    if (node == NULL) return 0;
    int count = node->is_end_of_word ? 1 : 0;

    TrieNode *child = node->children;
    while (child != NULL) {
        count += countWords(child);
        child = child->next;
    }
    return count;
}

/* Count total nodes (excluding root) */
int countNodes(TrieNode *node) {
    if (node == NULL) return 0;
    int count = 1; /* Count self */

    TrieNode *child = node->children;
    while (child != NULL) {
        count += countNodes(child);
        child = child->next;
    }
    return count;
}

/* Count words that start with prefix */
int countPrefixMatches(Trie *t, char *prefix) {
    TrieNode *node = findPrefix(t, prefix);
    if (node == NULL) return 0;
    return countWords(node); /* Count from prefix node */
}

int main() {
    Trie *t = createTrie();
    insertWord(t, "cat"); insertWord(t, "car"); insertWord(t, "can");
    printf("Total words: %d\n", countWords(t->root));   /* 3 */
    printf("Total nodes: %d\n", countNodes(t->root)-1); /* 5 */
    printf("Words with 'ca': %d\n", countPrefixMatches(t,"ca")); /* 3 */
    return 0;
}

Section 09

Structural
Operations

Navigating the relationships between nodes — finding parents, children, and siblings within the linked-list trie structure.

🔗
Parent, Child & Sibling Nodes
In a linked-list trie, each node connects to:

• Children: The first child via node→children, then siblings via →next
• Parent: Not directly stored — must be found by traversal
• Siblings: Nodes at the same level with the same parent, linked via →next
C — Structural Operations
/* Find the direct child of a node with char ch */
TrieNode* findChildNode(TrieNode *parent, char ch) {
    TrieNode *child = parent->children;
    while (child != NULL) {
        if (child->data == ch) return child;
        child = child->next;
    }
    return NULL;
}

/* Find parent of the node reached by 'word' */
TrieNode* findParentNode(Trie *t, char *word) {
    TrieNode *current = t->root;
    TrieNode *parent  = NULL;
    int i;
    for (i = 0; word[i] != '\0'; i++) {
        parent  = current;
        current = findChild(current, word[i]);
        if (!current) return NULL;
    }
    return parent;
}

/* Find next sibling of a node (in parent's child list) */
TrieNode* findSiblingNode(TrieNode *node) {
    return node->next; /* Direct linked list sibling */
}

/* Print all siblings of a node */
void printSiblings(TrieNode *node) {
    TrieNode *sib = node->next;
    printf("Siblings of '%c': ", node->data);
    while (sib != NULL) {
        printf("%c ", sib->data);
        sib = sib->next;
    }
    printf("\n");
}
🌳 Analogy
In a family tree: children are nodes hanging below you. Siblings are nodes at the same level with the same parent. The parent is one level above. In our linked-list trie, siblings are literally chained together with →next pointers.
Section 10

Update
Operations

Changing words in the trie — updating, replacing, or modifying node data without rebuilding the entire structure.

✏️
Update & Replace Word
Since a trie doesn't store the full word in one place, updating a word = delete old + insert new. However, we can modify individual node data for special cases like case normalization.
C — Update Operations
/* Update word: replace old word with new word */
int updateWord(Trie *t, char *oldWord, char *newWord) {
    if (!searchWord(t, oldWord)) {
        printf("'%s' not found in trie\n", oldWord);
        return 0;
    }
    deleteWord(t, oldWord);  /* Remove old */
    insertWord(t, newWord);  /* Add new */
    printf("Updated '%s' → '%s'\n", oldWord, newWord);
    return 1;
}

/* Replace word (alias — same as update) */
void replaceWord(Trie *t, char *from, char *to) {
    updateWord(t, from, to);
}

/* Modify node data — change character in-place */
/* WARNING: This can break trie structure — use carefully! */
void modifyNodeData(TrieNode *node, char newChar) {
    if (node != NULL)
        node->data = newChar;
}

/* Normalize word to lowercase before inserting */
void insertNormalized(Trie *t, char *word) {
    char lower[100];
    int i;
    for (i = 0; word[i]; i++)
        lower[i] = (word[i] >= 'A' && word[i] <= 'Z')
                 ? word[i] + 32 : word[i];
    lower[i] = '\0';
    insertWord(t, lower);
}
📝 Analogy
Updating a word in a trie is like correcting a name in a physical index card system. You pull the old card out and file a new one in its proper alphabetical spot. You can't erase and rewrite in place because the filing position is determined by the letters themselves.
Section 11

Compression
Operations

Optimizing the trie by collapsing long chains of single-child nodes — creating a Radix Tree (Compressed Trie).

🗜️
Compress Trie — Path Compression
A standard trie wastes memory on long chains where each node has only one child. Path compression merges these chains into a single node that stores a string instead of a character. This is the basis of a Radix Tree (also called Patricia Trie).
🗺️ Analogy
Imagine a road map where a straight highway has 10 exits, but only 1 is important. Instead of listing all 10 stops, you compress it to: "Drive 300km along Highway 1 → reach City X." Path compression does the same — collapses boring single-child paths into one labeled edge.

C — Compressed Trie Node
/* Compressed trie stores STRINGS in nodes */
typedef struct RadixNode {
    char  label[50];          /* Stored string (compressed) */
    int   is_end;
    struct RadixNode *children;
    struct RadixNode *next;
} RadixNode;

/* Path compression: merge single-child chains */
void compressNode(TrieNode *node, char *buf, int depth) {
    if (node == NULL) return;

    /* Single child and not end of word = compress */
    if (node->children != NULL &&
        node->children->next == NULL &&
        !node->is_end_of_word) {
        buf[depth] = node->data;
        buf[depth+1] = '\0';
        compressNode(node->children, buf, depth+1);
    } else {
        /* Multiple children or end of word — output compressed label */
        buf[depth] = node->data;
        buf[depth+1] = '\0';
        printf("Compressed label: [%s]%s\n",
               buf, node->is_end_of_word ? " (WORD)" : "");

        TrieNode *child = node->children;
        while (child) {
            char newBuf[50] = {0};
            compressNode(child, newBuf, 0);
            child = child->next;
        }
    }
}

/* Remove redundant nodes (nodes with single child, not end) */
void removeRedundantNodes(TrieNode *node) {
    while (node->children != NULL &&
           node->children->next == NULL &&
           !node->is_end_of_word) {
        TrieNode *child = node->children;
        node->data = child->data;
        node->children = child->children;
        node->is_end_of_word = child->is_end_of_word;
        free(child); /* Remove the redundant middle node */
    }
}
Section 12

Pattern
Matching

Using wildcards and patterns to search for words — the foundation of search engines and file glob matching.

🃏
Wildcard & Pattern Search
Wildcard Search: Use ? to match any single character, or * to match zero or more characters.

Pattern Search: Match exact sequences with specific unknown positions.

For example, pattern "c?t" matches "cat", "cot", "cut", "cbt" — any 3-letter word starting with 'c' and ending with 't'.
🔎 Real Use Case
When you type ls *.txt in a terminal, the shell uses pattern matching similar to wildcard trie search. The * matches any sequence of characters, so it finds all .txt files. File system search engines use tries for this!
C — Wildcard Pattern Matching
/* Wildcard search: '?' = any one char, no '*' here */
int wildcardSearch(TrieNode *node, char *pattern, int idx) {
    if (pattern[idx] == '\0')
        return node->is_end_of_word;

    char ch = pattern[idx];
    TrieNode *child = node->children;

    while (child != NULL) {
        if (ch == '?' || child->data == ch) {
            /* '?' matches any char */
            if (wildcardSearch(child, pattern, idx+1))
                return 1;
        }
        child = child->next;
    }
    return 0;
}

/* Full wildcard with '*' (matches 0 or more chars) */
int fullWildcard(TrieNode *node, char *pattern, int idx) {
    if (pattern[idx] == '\0')
        return node->is_end_of_word;

    if (pattern[idx] == '*') {
        /* Try matching 0 chars (skip *) */
        if (fullWildcard(node, pattern, idx+1)) return 1;
        /* Try matching 1+ chars */
        TrieNode *c = node->children;
        while (c) {
            if (fullWildcard(c, pattern, idx)) return 1;
            c = c->next;
        }
        return 0;
    }

    TrieNode *child = node->children;
    while (child) {
        if (pattern[idx] == '?' || child->data == pattern[idx])
            if (fullWildcard(child, pattern, idx+1)) return 1;
        child = child->next;
    }
    return 0;
}

Section 13

Utility
Operations

Housekeeping operations — copying, cloning, clearing, and destroying a trie safely to prevent memory leaks.

🧰
Copy, Clone, Clear & Destroy
These operations manage the lifecycle of a trie. In C, forgetting to destroy a trie causes memory leaks. Always free every node when done!
C — Utility Operations
/* Deep copy a single node and all its children */
TrieNode* copyNode(TrieNode *src) {
    if (src == NULL) return NULL;

    TrieNode *newNode = createNode(src->data);
    newNode->is_end_of_word = src->is_end_of_word;

    /* Copy all children recursively */
    TrieNode *child = src->children;
    TrieNode **ptr = &(newNode->children);
    while (child != NULL) {
        *ptr = copyNode(child);
        ptr  = &((*ptr)->next);
        child = child->next;
    }
    return newNode;
}

/* Clone entire Trie */
Trie* cloneTrie(Trie *src) {
    Trie *clone = (Trie*) malloc(sizeof(Trie));
    clone->root = copyNode(src->root);
    return clone;
}

/* Clear all words (postorder deletion) */
void clearNode(TrieNode *node) {
    if (node == NULL) return;
    TrieNode *child = node->children;
    while (child != NULL) {
        TrieNode *next = child->next;
        clearNode(child); /* Recurse first */
        free(child);
        child = next;
    }
    node->children     = NULL;
    node->is_end_of_word = 0;
}

void clearTrie(Trie *t) {
    clearNode(t->root); /* Keep root, clear everything else */
}

/* Destroy trie — free everything including root */
void destroyTrie(Trie **t) {
    if (*t == NULL) return;
    clearNode((*t)->root);
    free((*t)->root);
    free(*t);
    *t = NULL; /* Avoid dangling pointer */
}
🧹 Analogy
Clone = photocopying an entire filing cabinet. Clear = emptying all the folders but keeping the cabinet. Destroy = throwing away the cabinet and all its contents. In C, you must always destroy your trie when you're done — otherwise RAM fills up like a room where you keep buying furniture but never throw any out!
Section 14

Validation
Operations

Ensuring a trie is structurally correct, checking prefix rules, and detecting duplicate words.

✅
Validation Checks
Validation operations ensure the integrity of your trie. A valid trie must: have all nodes reachable from root, have no NULL characters in the middle of a path, and obey prefix rules (no word can be its own prefix unless correctly marked).
C — Validation Operations
/* Validate that all nodes are properly linked */
int checkValidTrie(TrieNode *node) {
    if (node == NULL) return 1; /* NULL is valid (leaf) */

    /* Each node must have a valid printable character */
    if (node->data != '\0' &&
        (node->data < 'a' || node->data > 'z')) {
        printf("Invalid char: %c\n", node->data);
        return 0;
    }

    TrieNode *child = node->children;
    while (child != NULL) {
        if (!checkValidTrie(child)) return 0;
        child = child->next;
    }
    return 1;
}

/* Verify prefix rules: "app" exists → "apple" is valid */
int verifyPrefixRules(Trie *t, char *word) {
    int len = strlen(word);
    char prefix[100];
    int i;
    for (i = 1; i < len; i++) {
        strncpy(prefix, word, i);
        prefix[i] = '\0';
        if (searchWord(t, prefix))
            printf("Prefix word '%s' exists for '%s'\n", prefix, word);
    }
    return 1;
}

/* Check if duplicate word already exists */
int checkDuplicate(Trie *t, char *word) {
    return searchWord(t, word); /* If found = duplicate */
}

/* Safe insert — only inserts if not duplicate */
int safeInsert(Trie *t, char *word) {
    if (checkDuplicate(t, word)) {
        printf("'%s' already exists!\n", word);
        return 0;
    }
    insertWord(t, word);
    return 1;
}
Section 15

Height
& Depth

Measuring the size and depth of the trie — the height tells us the longest word, and depth tells us how deep a specific node is.

📏
Height & Depth Operations
Height: The length of the longest path from root to any leaf = the length of the longest word in the trie.

Depth of a node: How many edges from the root to that node.

Maximum Depth: Same as height — the deepest any node goes.
C — Height & Depth Operations
/* Find height of trie (longest word length) */
int findHeight(TrieNode *node) {
    if (node == NULL) return -1;
    if (node->children == NULL) return 0;

    int maxHeight = -1;
    TrieNode *child = node->children;

    while (child != NULL) {
        int h = findHeight(child);
        if (h > maxHeight) maxHeight = h;
        child = child->next;
    }
    return maxHeight + 1;
}

/* Find depth of node containing character at position i in word */
int findNodeDepth(Trie *t, char *word) {
    TrieNode *current = t->root;
    int depth = 0, i;
    for (i = 0; word[i] != '\0'; i++) {
        TrieNode *child = findChild(current, word[i]);
        if (child == NULL) return -1; /* Not found */
        current = child;
        depth++;
    }
    return depth;
}

/* Maximum depth = height */
int maxDepth(Trie *t) {
    return findHeight(t->root);
}

int main() {
    Trie *t = createTrie();
    insertWord(t, "cat");       /* length 3 */
    insertWord(t, "algorithm"); /* length 9 */
    insertWord(t, "go");        /* length 2 */

    printf("Trie height: %d\n", findHeight(t->root)); /* 9 */
    printf("Depth of 'cat': %d\n", findNodeDepth(t, "cat")); /* 3 */
    return 0;
}
🌲 Analogy
The height of a trie is like the tallest floor in a building. If the longest word has 9 letters, the building has 9 floors. The depth of a node is which floor it's on. "cat" ends on floor 3; "algorithm" ends on floor 9.
Section 16

Recursive
Operations

Implementing Insert, Delete, and Search using recursion — elegant solutions that naturally follow the trie's recursive structure.

🔄
Recursive Implementation
A trie is a naturally recursive structure — each subtree is itself a trie. Recursive solutions are often cleaner and easier to understand for trie operations.
C — Complete Recursive Implementations
/* === RECURSIVE INSERT === */
TrieNode* recInsert(TrieNode *node, char *word, int i) {
    if (node == NULL) node = createNode('\0');

    if (word[i] == '\0') {
        node->is_end_of_word = 1;
        return node;
    }

    TrieNode *child = findChild(node, word[i]);
    if (child == NULL) {
        child = createNode(word[i]);
        addChild(node, child);
    }
    recInsert(child, word, i+1); /* Recurse into child */
    return node;
}

/* === RECURSIVE SEARCH === */
int recSearch(TrieNode *node, char *word, int i) {
    if (node == NULL) return 0;

    if (word[i] == '\0')
        return node->is_end_of_word;

    TrieNode *child = findChild(node, word[i]);
    return recSearch(child, word, i+1); /* Go deeper */
}

/* === RECURSIVE DELETE === */
int recDelete(TrieNode *node, char *word, int i) {
    if (word[i] == '\0') {
        if (!node->is_end_of_word) return 0;
        node->is_end_of_word = 0;
        return (node->children == NULL);
    }

    TrieNode *child = findChild(node, word[i]);
    if (!child) return 0;

    int shouldDelete = recDelete(child, word, i+1);

    if (shouldDelete) {
        removeChild(node, word[i]);
        return (node->children == NULL && !node->is_end_of_word);
    }
    return 0;
}

/* Wrappers to call from main */
void recursiveInsert(Trie *t, char *word) {
    recInsert(t->root, word, 0);
}
int recursiveSearch(Trie *t, char *word) {
    return recSearch(t->root, word, 0);
}
void recursiveDelete(Trie *t, char *word) {
    recDelete(t->root, word, 0);
}
💡 Recursion vs Iteration: Recursive code is shorter and mirrors the trie's natural structure. Iterative code uses less stack space and avoids stack overflow for very long words. Both produce identical results.
Section 17

Iterative
Operations

Implementing trie operations using loops instead of recursion — more memory-efficient for very long words.

🔁
Iterative Implementation
Iterative versions use a while or for loop with an explicit pointer traversing the trie level by level. No recursion = no risk of stack overflow for very long strings.
C — Complete Iterative Implementations
/* === ITERATIVE INSERT (revisited with comments) === */
void iterInsert(Trie *t, char *word) {
    TrieNode *curr = t->root;

    for (int i = 0; word[i]; i++) {
        TrieNode *child = findChild(curr, word[i]);
        if (!child) {
            child = createNode(word[i]);
            addChild(curr, child);
        }
        curr = child; /* Move to next level */
    }
    curr->is_end_of_word = 1;
}

/* === ITERATIVE SEARCH === */
int iterSearch(Trie *t, char *word) {
    TrieNode *curr = t->root;

    for (int i = 0; word[i]; i++) {
        curr = findChild(curr, word[i]);
        if (!curr) return 0; /* Not found */
    }
    return curr->is_end_of_word;
}

/* === ITERATIVE DELETE (uses stack for backtracking) === */
void iterDelete(Trie *t, char *word) {
    int len = strlen(word);
    TrieNode *stack[100];   /* Track visited nodes */
    int top = -1;
    TrieNode *curr = t->root;

    /* Phase 1: traverse and record path */
    for (int i = 0; i < len; i++) {
        stack[++top] = curr;
        curr = findChild(curr, word[i]);
        if (!curr) return; /* Word not in trie */
    }

    if (!curr->is_end_of_word) return; /* Not a complete word */
    curr->is_end_of_word = 0;

    /* Phase 2: backtrack and remove empty nodes */
    for (int i = len - 1; i >= 0; i--) {
        TrieNode *parent = stack[top--];
        TrieNode *child = findChild(parent, word[i]);

        if (child && child->children == NULL && !child->is_end_of_word)
            removeChild(parent, word[i]); /* Safe to delete */
        else
            break; /* Stop — node still needed */
    }
}

C — Merge Two Tries
/* Collect all words from a trie into an array */
void getAllWords(TrieNode *node, char *buf, int depth,
                char words[][50], int *count) {
    if (!node) return;
    buf[depth] = node->data;
    buf[depth+1] = '\0';

    if (node->is_end_of_word)
        strcpy(words[(*count)++], buf+1); /* Skip root */

    TrieNode *child = node->children;
    while (child) {
        getAllWords(child, buf, depth+1, words, count);
        child = child->next;
    }
}

/* Merge: insert all of B's words into A */
void mergeTries(Trie *A, Trie *B) {
    char words[100][50];
    char buf[100] = {0};
    int count = 0, i;

    getAllWords(B->root, buf, 0, words, &count);

    for (i = 0; i < count; i++)
        insertWord(A, words[i]); /* Insert B's words into A */

    printf("Merged %d words from Trie B into Trie A\n", count);
}

/* Split: extract words starting with prefix into new trie */
Trie* splitTrie(Trie *src, char *prefix) {
    Trie *newTrie = createTrie();
    char words[100][50];
    char buf[100] = {0};
    int count = 0, i;
    int plen = strlen(prefix);

    getAllWords(src->root, buf, 0, words, &count);

    for (i = 0; i < count; i++) {
        if (strncmp(words[i], prefix, plen) == 0)
            insertWord(newTrie, words[i]);
    }
    return newTrie;
}

/* Simple spell checker using trie search */
void spellCheck(Trie *dict, char *text) {
    char word[50];
    int wi = 0, i;

    for (i = 0; text[i]; i++) {
        if (text[i] == ' ' || text[i] == '.') {
            word[wi] = '\0';
            if (wi > 0) {
                if (searchWord(dict, word))
                    printf("✓ %s\n", word);
                else
                    printf("✗ %s (MISSPELLED)\n", word);

            wi = 0;
        } else {
            word[wi++] = text[i];
        }
    }
}
