Operation 01
Create Stack
Initialize multiple independent stacks using linked list structure with NULL top pointers.
Theory
Visualization
C Code
Analogy

A stack is a linear data structure that follows the LIFO (Last In, First Out) principle — the last element added is the first one removed. When we use a linked list to implement a stack, each element (node) stores data and a pointer to the next node.

A multiple stack system means we maintain several independent stacks simultaneously. Each stack has its own top pointer (initially NULL), meaning the stack is empty. No fixed memory is pre-allocated — memory grows dynamically as elements are pushed.

Top Pointer

Points to the most recently added (top) node. NULL means the stack is empty.

Dynamic Memory

No fixed size. Each node is allocated on the heap using malloc() when needed.

Multiple Stacks

An array of top pointers lets us manage many stacks with a single array structure.

NULL Assignment

Every stack's top pointer must be set to NULL at creation — mandatory initialization step.

Key idea: Creating a stack just means declaring a pointer variable (or array of pointers) and setting it to NULL. No memory is allocated yet — memory is allocated only when you push elements.

1
Define the number of stacks (e.g., MAX_STACKS = 3)
2
Declare an array of top pointers: Node* top[MAX_STACKS]
3
Loop through all stacks and set top[i] = NULL for each
4
Each stack is now empty and ready for push/pop operations
📊 Multiple Stack Initialization
C
/* ── Linked List Based Multiple Stack: Create Stack ── */

#include <stdio.h>
#include <stdlib.h>

#define MAX_STACKS 3   /* Number of stacks we want */

/* Node structure for linked list */
struct Node {
    int data;              /* Stores the value */
    struct Node* next;    /* Pointer to the next node */
};

/* Array of top pointers — one per stack */
struct Node* top[MAX_STACKS];

/* Function to initialize all stacks */
void createStacks() {
    int i;
    for (i = 0; i < MAX_STACKS; i++) {
        top[i] = NULL;   /* NULL = empty stack */
    }
    printf("All %d stacks created and initialized.\n", MAX_STACKS);
}

int main() {
    createStacks();
    printf("Stack 0 top: %p\n", (void*)top[0]);  /* Will print NULL */
    printf("Stack 1 top: %p\n", (void*)top[1]);
    printf("Stack 2 top: %p\n", (void*)top[2]);
    return 0;
}

SpaceO(n)
n = #stacksLinear

Creating the stacks is like opening the counters — no one is there yet, but the counters are ready to accept customers.

Operation 02
Create Node
Dynamically allocate memory for a new node, assign data and set the next pointer.
Theory
Visualization
C Code
Analogy

A node is the basic building block of a linked list stack. Every time we want to push a new value onto a stack, we must first create a node to hold that value. Node creation involves dynamic memory allocation using malloc().

Each node has two fields: data (stores the actual value) and next (a pointer to the next node in the stack). The newly created node's next pointer is initially set to NULL until it is connected to the stack.

malloc()

Allocates a requested size of bytes on the heap and returns a pointer to it.

sizeof()

Returns the exact byte size of the Node struct — crucial for correct allocation.

Data Field

Holds the actual value you want to store in the stack (e.g., an integer).

Next Pointer

Links this node to the next one. Set to NULL when node is first created.

Always check: malloc() can return NULL if the system is out of memory. Always check if the returned pointer is NULL before using it to prevent crashes (segmentation fault).

🔷 Node Memory Structure
C
/* ── Create Node ── */

struct Node {
    int data;
    struct Node* next;
};

/* Function to create a new node */
struct Node* createNode(int value) {

    /* Step 1: Allocate memory on the heap */
    struct Node* newNode = (struct Node*) malloc(sizeof(struct Node));

    /* Step 2: Always check if malloc succeeded */
    if (newNode == NULL) {
        printf("Memory allocation failed!\n");
        return NULL;
    }

    /* Step 3: Assign the data */
    newNode->data = value;

    /* Step 4: Set next to NULL (not linked yet) */
    newNode->next = NULL;

    return newNode;   /* Return pointer to new node */
}

int main() {
    struct Node* n = createNode(42);
    if (n) {
        printf("Node created: data=%d, next=%p\n", n->data, (void*)n->next);
        free(n);  /* Always free when done */
    }
    return 0;
}
TimeO(1)
SpaceO(1)
📦

Real-Life Analogy: Creating a Box

Think of malloc() as buying a new box from a store. The box has two compartments: one for your item (data) and a hook on the side to attach to another box (next pointer). When you first get the box, the hook is empty (NULL). You'll attach it to the stack when you push.

In a game, when a player picks up a new item, a node is created to store that item. The game doesn't pre-allocate space for all possible items — it creates a node on-the-fly, just like our createNode() function.

Operation 03
Push Operation
Insert a new element at the top of a specified stack. Memory is allocated dynamically.
Theory
Visualization
C Code
Analogy

Push inserts a new value at the top of the stack. In a linked list implementation, this means creating a new node and making it the new top — the old top becomes the second element.

1
Create a new node with the given value using malloc()
2
Set newNode→next = top[stackIndex] (new node points to current top)
3
Update top[stackIndex] = newNode (new node becomes the top)
4
Stack size increases by 1 — no overflow (linked list grows dynamically)

No Overflow: Unlike array-based stacks, a linked list stack never overflows (as long as memory is available). This is one of the biggest advantages of using a linked list for stacks.

⬆️ Push — Interactive Stack Simulator
C
/* ── Push Operation ── */

void push(int stackIndex, int value) {

    /* Step 1: Check valid stack index */
    if (stackIndex < 0 || stackIndex >= MAX_STACKS) {
        printf("Invalid stack index!\n");
        return;
    }

    /* Step 2: Create a new node */
    struct Node* newNode = createNode(value);
    if (newNode == NULL) {
        printf("Memory allocation failed! Cannot push.\n");
        return;
    }

    /* Step 3: Link new node to current top */
    newNode->next = top[stackIndex];

    /* Step 4: Update top pointer */
    top[stackIndex] = newNode;

    printf("Pushed %d onto Stack %d\n", value, stackIndex);
}

int main() {
    createStacks();
    push(0, 10);   /* Stack 0: [10] */
    push(0, 20);   /* Stack 0: [20, 10] */
    push(1, 99);   /* Stack 1: [99] */
    return 0;
}
TimeO(1)
SpaceO(1)
OverflowNever*

Real-Life Analogy: Stack of Plates

Imagine stacking plates in a cafeteria. When you add a plate, you place it on top. The new plate (newNode) points down to the previous top plate (newNode→next = oldTop). The pile's top pointer now points to the new plate. This is exactly how push works!

Operation 04
Pop Operation
Remove and return the top element from a stack, then free the memory used by that node.
Theory
Visualization
C Code
Analogy

Pop removes the topmost element from a stack and returns its value. In a linked list, this means updating the top pointer to the next node and freeing the removed node's memory to prevent memory leaks.

1
Check if stack is empty (top == NULL) — if yes, print underflow error and return
2
Save the top node pointer in a temporary variable: temp = top[i]
3
Store the data: value = temp→data before freeing
4
Update top: top[i] = temp→next
5
Free memory: free(temp) to avoid memory leaks

Underflow: Popping from an empty stack is called underflow. Always check if top == NULL before popping. Attempting to access data from a NULL pointer causes a segmentation fault crash.

⬇️ Pop Visualization
C
/* ── Pop Operation ── */

int pop(int stackIndex) {

    /* Step 1: Check for underflow */
    if (top[stackIndex] == NULL) {
        printf("Stack %d is empty (Underflow)!\n", stackIndex);
        return -1;   /* Sentinel value */
    }

    /* Step 2: Save the top node */
    struct Node* temp = top[stackIndex];

    /* Step 3: Get the data before freeing */
    int poppedValue = temp->data;

    /* Step 4: Move top to next node */
    top[stackIndex] = temp->next;

    /* Step 5: Free the old top node */
    free(temp);

    printf("Popped %d from Stack %d\n", poppedValue, stackIndex);
    return poppedValue;
}

int main() {
    createStacks();
    push(0, 10);
    push(0, 20);
    pop(0);   /* Returns 20 — last in, first out */
    pop(0);   /* Returns 10 */
    pop(0);   /* Underflow! Stack is empty */
    return 0;
}
TimeO(1)
SpaceO(1)
↩️

Real-Life Analogy: Ctrl+Z (Undo)

When you press Ctrl+Z in a text editor, the last action is popped from the undo stack and reversed. The top node (last action) is removed, its data is used to undo the change, and the memory is released. The previous action is now the new top.

🥞

Use Case: Function Call Stack

When you call a function in C, a frame is pushed onto the call stack. When the function returns, its frame is popped — memory is freed and control returns to the caller. This is exactly a pop operation on the system's call stack.

Operation 05
Peek / Top Operation
View the top element without removing it. Non-destructive read of the stack's top value.
Theory
Visualization
C Code
Analogy

Peek (also called Top) allows you to look at the top element of a stack without removing it. This is useful when you need to make a decision based on the top value but don't want to consume it yet.

Implementation is simple: just check if top != NULL and return top→data. No pointers are updated, no memory is freed. The stack remains unchanged.

Non-Destructive

The stack is NOT modified. The top element remains in place after peek.

Empty Check

Must verify top != NULL before accessing data to avoid NULL pointer crash.

O(1) Time

Instant access — just reads the top pointer's data field without traversal.

Common Use

Used in expression parsers, syntax checkers, and decision-making algorithms.

👁️ Peek — View Top Without Removing
C
/* ── Peek / Top Operation ── */

int peek(int stackIndex) {

    /* Check if stack is empty */
    if (top[stackIndex] == NULL) {
        printf("Stack %d is empty! Nothing to peek.\n", stackIndex);
        return -1;
    }

    /* Return top data WITHOUT modifying stack */
    printf("Top of Stack %d: %d\n", stackIndex, top[stackIndex]->data);
    return top[stackIndex]->data;
}

int main() {
    createStacks();
    push(0, 100);
    push(0, 200);

    peek(0);   /* Prints 200 — stack unchanged */
    peek(0);   /* Prints 200 again — same result! */

    printf("Stack still has top = %d\n", top[0]->data);
    return 0;
}
TimeO(1)
SpaceO(1)
DestructiveNo
👀

Real-Life Analogy: Peeking at Top Card

In a card game, you might be allowed to peek at the top card of the draw pile without taking it. You see the value, decide your strategy, but the card stays on top. Peek does exactly this — look but don't touch.

Use Case: Expression Parser

When parsing math expressions like 3 + 4 * 2, an expression evaluator frequently peeks at the operator stack to decide precedence before pushing or popping — without actually modifying the stack until a decision is made.

Operation 06
isEmpty Operation
Check whether a specific stack is empty by verifying if its top pointer is NULL.
Theory
Visualization
C Code
Analogy

isEmpty is a utility/helper function used internally by almost every other stack operation. It checks whether a stack has any elements by testing if the top pointer equals NULL.

A stack is empty when top[stackIndex] == NULL. This condition is checked before pop, peek, display, and other operations to prevent undefined behavior (crashes from NULL dereference).

NULL Checking

The only condition: if top == NULL, the stack is empty. That simple!

Boolean Return

Returns 1 (true) if empty, 0 (false) if not. Used in if-conditions.

Safety Guard

Acts as a guard in pop, peek, and display to prevent crashes on empty stacks.

O(1) Always

Constant time — just one comparison. Doesn't traverse the list.

❓ isEmpty — Visual Check
C
/* ── isEmpty Operation ── */

/* Returns 1 if empty, 0 if not */
int isEmpty(int stackIndex) {
    if (top[stackIndex] == NULL) {
        return 1;   /* True: stack is empty */
    }
    return 0;       /* False: stack has elements */
}

/* Shorter version using ternary */
int isEmptyShort(int stackIndex) {
    return (top[stackIndex] == NULL) ? 1 : 0;
}

int main() {
    createStacks();

    printf("Stack 0 empty? %d\n", isEmpty(0));  /* 1 (true) */

    push(0, 42);
    printf("Stack 0 empty? %d\n", isEmpty(0));  /* 0 (false) */

    /* Used inside pop safely: */
    if (!isEmpty(0)) {
        pop(0);
    }
    printf("Stack 0 empty? %d\n", isEmpty(0));  /* 1 again */

    return 0;
}
TimeO(1)
SpaceO(1)
🪣

Real-Life Analogy: Empty Bucket Check

Before pouring water from a bucket, you check is it empty? If yes, there's nothing to pour. The check is instant — you just look. Similarly, isEmpty just checks the top pointer; it doesn't scan through the stack.

🔋

Use Case: Task Queue Draining

In a job scheduler, a worker thread loops: while (!isEmpty(taskStack)) { pop and execute; }. The isEmpty check prevents the worker from crashing when all tasks are done. It's the most basic safety check in stack-based systems.

Operation 07
Display Operation
Traverse and print all elements in a single stack from top to bottom.
Theory
Visualization
C Code
Analogy

Display traverses a linked list stack from the top node to the last node (where next == NULL), printing each element's data. This is a linear traversal — the only way to see all elements since linked lists don't support random access.

1
Check if stack is empty — print message and return if so
2
Start a temp pointer at top: temp = top[i]
3
Print temp→data, then move: temp = temp→next
4
Repeat until temp == NULL (end of list)

Important: We use a temporary pointer to traverse — never move the actual top pointer, or you'll lose track of the stack permanently!

📋 Display — Traversal Animation
C
/* ── Display Operation ── */

void display(int stackIndex) {

    /* Step 1: Check empty */
    if (top[stackIndex] == NULL) {
        printf("Stack %d is empty.\n", stackIndex);
        return;
    }

    printf("Stack %d (top → bottom): ", stackIndex);

    /* Step 2: Use a temp pointer — DO NOT move top! */
    struct Node* temp = top[stackIndex];

    /* Step 3: Traverse until NULL */
    while (temp != NULL) {
        printf("%d", temp->data);
        if (temp->next != NULL) printf(" → ");
        temp = temp->next;   /* Move to next node */
    }

    printf(" → NULL\n");
}

int main() {
    createStacks();
    push(0, 5); push(0, 15); push(0, 25);
    display(0);  /* Output: 25 → 15 → 5 → NULL */
    return 0;
}
TimeO(n)
SpaceO(1)
n = elementsLinear
🚂

Real-Life Analogy: Reading a Train

Imagine a train of linked wagons. Display starts from the front (top) and walks wagon-by-wagon to the last one, calling out each wagon's cargo. You never detach wagons — you just walk alongside and read.

Analogy

DisplayAllStacks iterates over all stacks (using a loop from 0 to MAX_STACKS-1) and calls the Display function for each. It provides a complete snapshot of all stacks at once, showing which are empty and which have elements.

📊 All Stacks at a Glance
C
/* ── DisplayAllStacks Operation ── */

void displayAllStacks() {
    int i;
    printf("====== All Stacks ======\n");
    for (i = 0; i < MAX_STACKS; i++) {
        printf("Stack %d: ", i);
        if (top[i] == NULL) {
            printf("[EMPTY]\n");
        } else {
            struct Node* temp = top[i];
            while (temp != NULL) {
                printf("%d", temp->data);
                if (temp->next) printf(" → ");
                temp = temp->next;
            }
            printf(" → NULL\n");
        }
    }
    printf("========================\n");
}

int main() {
    createStacks();
    push(0, 10); push(0, 20);
    push(2, 99);
    /* Stack 1 intentionally left empty */
    displayAllStacks();
    return 0;
}

Operation 09
DeleteStack Operation
Destroy an entire stack by freeing all nodes, or delete the top node, or delete from a specific position.
Theory
Visualization
C Code
Analogy

DeleteStack completely destroys a stack by traversing all nodes and freeing each one's memory, then setting the top pointer to NULL. This is critical for preventing memory leaks in C programs.

There are three variants covered here:

A
Delete Entire Stack: Free all nodes one by one, set top = NULL
B
Delete Top Node: Equivalent to Pop — removes just the top node
C
Delete From Specific Stack: Target a particular stack index and delete it

Memory Leak Warning: If you just set top[i] = NULL without freeing nodes, all those nodes are lost in memory — unreachable but not freed. Always traverse and free before nulling the top pointer!

🗑️ Delete Stack — Memory Freeing
C
/* ── DeleteStack Operations ── */

/* A: Delete entire stack — free all nodes */
void deleteStack(int stackIndex) {
    struct Node* temp;
    while (top[stackIndex] != NULL) {
        temp = top[stackIndex];         /* Hold current top */
        top[stackIndex] = temp->next;   /* Advance top */
        free(temp);                     /* Free old top */
    }
    printf("Stack %d deleted successfully.\n", stackIndex);
}

/* B: Delete just the top node (like pop but no return) */
void deleteTopNode(int stackIndex) {
    if (top[stackIndex] == NULL) {
        printf("Stack %d is empty!\n", stackIndex);
        return;
    }
    struct Node* temp = top[stackIndex];
    top[stackIndex] = temp->next;
    free(temp);
    printf("Top node deleted from Stack %d.\n", stackIndex);
}

/* C: Delete from a specific stack (same as deleteStack) */
void deleteFromSpecificStack(int stackIndex) {
    deleteStack(stackIndex);  /* Same logic, named for clarity */
}

int main() {
    createStacks();
    push(0, 10); push(0, 20); push(0, 30);
    deleteTopNode(0);    /* Removes 30 */
    deleteStack(0);       /* Removes 20 and 10, frees all */
    return 0;
}
Full Delete TimeO(n)
Top Delete TimeO(1)
♻️

Real-Life Analogy: Shredding Documents

Deleting a stack is like feeding a pile of documents into a shredder one by one. Each sheet (node) is shredded (freed) and the pile (stack) shrinks until nothing remains. Just setting top = NULL without freeing is like throwing the pile in a locked room — wasted space forever.

Operation 10
ClearStack Operation
Remove all elements from a stack and reset it to empty, freeing all memory.
Theory
Visualization
C Code
Analogy

ClearStack is similar to DeleteStack but the key difference is intent: ClearStack resets the stack for reuse (the stack structure remains, just emptied), while DeleteStack implies the stack itself is being destroyed.

In practice, the implementation is the same — traverse all nodes, free each one, then set top[i] = NULL. After ClearStack, the stack is in the same state as after CreateStack — empty and ready for new pushes.

Delete All Nodes

Every node is freed using free() to reclaim heap memory.

Memory Cleanup

No node is left unreachable — proper cleanup prevents memory leaks.

Reset Top

top[i] = NULL after clearing — stack is now reusable.

Reusable

Unlike deleteStack, clearStack implies the stack will be reused with new pushes.

🧹 ClearStack Animation
C
/* ── ClearStack Operation ── */

void clearStack(int stackIndex) {
    if (top[stackIndex] == NULL) {
        printf("Stack %d is already empty.\n", stackIndex);
        return;
    }

    struct Node* temp;
    int count = 0;

    /* Free all nodes one by one */
    while (top[stackIndex] != NULL) {
        temp = top[stackIndex];
        top[stackIndex] = temp->next;
        free(temp);
        count++;
    }

    /* top is now NULL — stack is reset */
    printf("Stack %d cleared. %d nodes freed.\n", stackIndex, count);
}

int main() {
    createStacks();
    push(1, 5); push(1, 10); push(1, 15);
    clearStack(1);          /* Clears Stack 1 */
    printf("Empty? %d\n", isEmpty(1));  /* Output: 1 */
    push(1, 999);  /* Stack reusable! */
    return 0;
}
TimeO(n)
SpaceO(1)
🧹

Real-Life Analogy: Wiping a Whiteboard

ClearStack is like erasing a whiteboard completely — all written content is gone, but the board still exists and is ready for new content. The board structure (the stack) remains; only the data is wiped and the memory returned.

Operation 11
Search Operation
Perform a linear search through a stack's nodes to find a specific value.
Theory
Visualization
C Code
Analogy

Search traverses the stack from top to bottom comparing each node's data with the target value. Since linked lists don't support random access, we must use linear search — checking one node at a time.

The function returns the position of the element (1 = top, 2 = second from top, etc.) or -1 if not found. The original stack is not modified during search.

Linear Search

Start at top, compare data, move to next — O(n) in worst case.

Compare Values

Each node's data field is compared with the target using ==.

Position Count

Track position from top (1-indexed) to tell the caller where it was found.

Found/Not Found

Return position if found, -1 if the value isn't in the stack.

🔍 Search — Step-by-Step Traversal
C
/* ── Search Operation ── */

int search(int stackIndex, int target) {

    if (top[stackIndex] == NULL) {
        printf("Stack %d is empty.\n", stackIndex);
        return -1;
    }

    struct Node* temp = top[stackIndex];
    int position = 1;   /* Position 1 = top */

    while (temp != NULL) {
        if (temp->data == target) {
            printf("Found %d at position %d in Stack %d\n",
                   target, position, stackIndex);
            return position;   /* Found! */
        }
        temp = temp->next;
        position++;
    }

    printf("%d not found in Stack %d\n", target, stackIndex);
    return -1;   /* Not found */
}

int main() {
    createStacks();
    push(0, 5); push(0, 15); push(0, 25);
    search(0, 15);  /* Found at position 2 */
    search(0, 99);  /* Not found */
    return 0;
}
Best CaseO(1)
Worst CaseO(n)
AverageO(n/2)
🔦

Real-Life Analogy: Searching a Stack of Folders

You have a pile of folders on your desk. To find "Project Alpha," you pick them one by one from the top, check the label, and either stop (found!) or put it aside and check the next one. This is linear search — no skipping allowed.

Operation 12
MoveElement Operation
Pop the top element from a source stack and push it onto a destination stack.
Theory
Visualization
C Code
Analogy

MoveElement transfers the top element from one stack to another. It is a combination of Pop (from source) + Push (to destination). The element's value is popped first, then pushed onto the destination stack.

1
Validate that source and destination stack indices are different and valid
2
Check that the source stack is not empty (cannot move from empty)
3
Pop the top value from the source stack
4
Push the popped value onto the destination stack

Optimization: Instead of popping and pushing (which creates and destroys nodes), we can directly re-link the node from one stack to another — avoiding malloc/free overhead. This is shown in the advanced version.

🔀 MoveElement — Transfer Between Stacks
→
C
/* ── MoveElement Operation ── */

void moveElement(int src, int dst) {

    /* Validate indices */
    if (src == dst) {
        printf("Source and destination are the same!\n");
        return;
    }
    if (src < 0 || dst < 0 || src >= MAX_STACKS || dst >= MAX_STACKS) {
        printf("Invalid stack index!\n");
        return;
    }

    /* Check source is not empty */
    if (top[src] == NULL) {
        printf("Source Stack %d is empty!\n", src);
        return;
    }

    /* Pop from source, push to destination */
    int value = pop(src);
    push(dst, value);

    printf("Moved %d from Stack %d to Stack %d\n", value, src, dst);
}

/* Advanced: direct re-linking (no malloc/free overhead) */
void moveElementFast(int src, int dst) {
    if (top[src] == NULL) { printf("Empty source!\n"); return; }

    struct Node* node = top[src];    /* Grab top node */
    top[src] = node->next;            /* Detach from source */
    node->next = top[dst];            /* Link to destination top */
    top[dst] = node;                  /* Make it new dest top */
}

int main() {
    createStacks();
    push(0, 10); push(0, 20);
    moveElement(0, 1);  /* Moves 20 from Stack 0 to Stack 1 */
    return 0;
}
Pop+PushO(1)
Fast Re-linkO(1)
🎴

Real-Life Analogy: Moving Cards Between Piles

In card games like Solitaire, you pick the top card from one pile (source stack) and place it on another pile (destination stack). The card doesn't change — only which pile it belongs to changes. MoveElement does this between stacks.

/* ── CompareStacks Operation ── */

int compareStacks(int s1, int s2) {

    struct Node* t1 = top[s1];
    struct Node* t2 = top[s2];

    while (t1 != NULL && t2 != NULL) {

        /* Compare node values */
        if (t1->data != t2->data) {
            printf("Stacks %d and %d are NOT equal.\n", s1, s2);
            return 0;
        }

        /* Advance both pointers */
        t1 = t1->next;
        t2 = t2->next;
    }

    /* If both NULL → same size and same values */
    if (t1 == NULL && t2 == NULL) {
        printf("Stacks %d and %d are EQUAL!\n", s1, s2);
        return 1;
    }

    /* One reached NULL first → different sizes */
    printf("Stacks %d and %d have different sizes.\n", s1, s2);
    return 0;
}

int main() {
    createStacks();
    push(0, 5); push(0, 10);
    push(1, 5); push(1, 10);
    compareStacks(0, 1);  /* EQUAL */

    push(2, 99);
    compareStacks(0, 2);  /* NOT EQUAL */
    return 0;
}

C
/* ── GetTotalElements Operation ── */

/* Count elements in a single stack */
int stackSize(int stackIndex) {
    int count = 0;
    struct Node* temp = top[stackIndex];
    while (temp != NULL) {
        count++;
        temp = temp->next;
    }
    return count;
}

/* Count all elements across ALL stacks */
int getTotalElements() {
    int total = 0;
    int i;

    for (i = 0; i < MAX_STACKS; i++) {
        int count = stackSize(i);
        printf("  Stack %d: %d elements\n", i, count);
        total += count;
    }

    printf("Total elements across all stacks: %d\n", total);
    return total;
}

int main() {
    createStacks();
    push(0, 10); push(0, 20);   /* Stack 0: 2 elements */
    push(1, 30);              /* Stack 1: 1 element  */
    push(2, 40); push(2, 50); push(2, 60); /* Stack 2: 3 elements */

    getTotalElements();  /* Total: 6 */
    return 0;
}
